        <div class="d-none d-lg-block ml-2"> <button class="btn-link Link--secondary js-print tooltipped tooltipped-n" aria-label="Print this article"> <!-- From https://heroicons.dev/ -->

       <h3 id="introduction"><a href="#introduction">Introduction</a></h3> 
       <p>GitHub Desktop extends and simplifies your GitHub.com workflow, using a visual interface instead of text commands on the command line. By the end of this guide, you'll have used GitHub Desktop to create a repository, make changes to the repository, and publish the changes to GitHub.</p> 
       <p>After installing GitHub Desktop and signing into GitHub or GitHub Enterprise you can create and clone a tutorial repository. The tutorial will introduce the basics of working with Git and GitHub, including installing a text editor, creating a branch, making a commit, pushing to GitHub.com, and opening a pull request. The tutorial is available if you do not have any repositories on GitHub Desktop yet.</p> 
       <p>We recommend completing the tutorial, but if you want to explore GitHub Desktop by creating a new repository, this guide will walk you through using GitHub Desktop to work on a Git repository.</p> 
       <h3 id="part-1-installing-github-desktop-and-authenticating-your-account"><a href="#part-1-installing-github-desktop-and-authenticating-your-account">Part 1: Installing GitHub Desktop and authenticating your account</a></h3> 
       <p>You can install GitHub Desktop on any supported operating system. After you install the app, you will need to sign in and authenticate your account on GitHub or GitHub Enterprise before you can create and clone a tutorial repository.</p> 
       <p>For more information on installing and authenticating, see "<a href="/en/desktop/installing-and-configuring-github-desktop/setting-up-github-desktop">Setting up GitHub Desktop</a>."</p> 
       <h3 id="part-2-creating-a-new-repository"><a href="#part-2-creating-a-new-repository">Part 2: Creating a new repository</a></h3> 
       <p>If you do not have any repositories associated with GitHub Desktop, you will see a "Let's get started!" view, where you can choose to create and clone a tutorial repository, clone an existing repository from the Internet, create a new repository, or add an existing repository from your hard drive. <img src="/assets/images/help/desktop/lets-get-started.png" alt="The Let's get started! screen"></p> 
       <h4 id="creating-and-cloning-a-tutorial-repository"><a href="#creating-and-cloning-a-tutorial-repository">Creating and cloning a tutorial repository</a></h4> 
       <p>We recommend that you create and clone a tutorial repository as your first project to practice using GitHub Desktop.</p> 
       <ol> 
        <li>Click <strong>Create a tutorial repository and clone it</strong>. <span class="procedural-image-wrapper"><img src="/assets/images/help/desktop/getting-started-guide/create-and-clone-a-tutorial-repository.png" alt="Create and clone a tutorial repository button"></span></li> 
        <li>Follow the prompts in the tutorial to install a text editor, create a branch, edit a file, make a commit, publish to GitHub, and open a pull request.</li> 
       </ol> 
       <h4 id="creating-a-new-repository"><a href="#creating-a-new-repository">Creating a new repository</a></h4> 
       <p>If you do not wish to create and clone a tutorial repository, you can create a new repository.</p> 
       <ol> 
        <li>Click <strong>Create a New Repository on your Hard Drive...</strong>. <span class="procedural-image-wrapper"><img src="/assets/images/help/desktop/getting-started-guide/creating-a-repository.png" alt="Create a new repository"></span></li> 
        <li>Fill in the fields and select your preferred options. <span class="procedural-image-wrapper"><img src="/assets/images/help/desktop/getting-started-guide/create-a-new-repository-options.png" alt="Create a repository options"></span>
         <ul> 
          <li>"Name" defines the name of your repository both locally and on GitHub.</li> 
          <li>"Description" is an optional field that you can use to provide more information about the purpose of your repository.</li> 
          <li>"Local path" sets the location of your repository on your computer. By default, GitHub Desktop creates a <em>GitHub</em> folder inside your <em>Documents</em> folder to store your repositories, but you can choose any location on your computer. Your new repository will be a folder inside the chosen location. For example, if you name your repository <code>Tutorial</code>, a folder named <em>Tutorial</em> is created inside the folder you selected for your local path. GitHub Desktop remembers your chosen location the next time you create or clone a new repository.</li> 
          <li><strong>Initialize this repository with a README</strong> creates an initial commit with a <em>README.md</em> file. READMEs helps people understand the purpose of your project, so we recommend selecting this and filling it out with helpful information. When someone visits your repository on GitHub, the README is the first thing they'll see as they learn about your project. For more information, see "<a href="/en/articles/about-readmes">About READMEs</a>."</li> 
          <li>The <strong>Git ignore</strong> drop-down menu lets you add a custom file to ignore specific files in your local repository that you don't want to store in version control. If there's a specific language or framework that you'll be using, you can select an option from the available list. If you're just getting started, feel free to skip this selection. For more information, see "<a href="/en/github/getting-started-with-github/ignoring-files">Ignoring files</a>."</li> 
          <li>The <strong>License</strong> drop-down menu lets you add an open-source license to a <em>LICENSE</em> file in your repository. You don't need to worry about adding a license right away. For more information about available open-source licenses and how to add them to your repository, see "<a href="/en/articles/licensing-a-repository">Licensing a repository</a>."</li> 
         </ul></li> 
        <li>Click <strong>Create repository</strong>.</li> 
       </ol> 
       <h3 id="part-3-exploring-github-desktop"><a href="#part-3-exploring-github-desktop">Part 3: Exploring GitHub Desktop</a></h3> 
       <p>In the file menu at the top of the screen, you can access settings and actions that you can perform in GitHub Desktop. Most actions also have keyboard shortcuts to help you work more efficiently. For a full list of keyboard shortcuts, see "<a href="/en/desktop/getting-started-with-github-desktop/keyboard-shortcuts">Keyboard shortcuts</a>."</p> 
       <h4 id="the-github-desktop-menu-bar"><a href="#the-github-desktop-menu-bar">The GitHub Desktop menu bar</a></h4> 
       <p>At the top of the GitHub Desktop app, you will see a bar that shows the current state of your repository.</p> 
       <ul> 
        <li> <p><strong>Current repository</strong> shows the name of the repository you're working on. You can click <strong>Current repository</strong> to switch to a different repository in GitHub Desktop.</p> </li> 
        <li> <p><strong>Current branch</strong> shows the name of the branch you're working on. You can click <strong>Current branch</strong> to view all the branches in your repository, switch to a different branch, or create a new branch. Once you create pull requests in your repository, you can also view these by clicking on <strong>Current branch</strong>.</p> </li> 
        <li> <p><strong>Publish repository</strong> appears because you haven't published your repository to GitHub yet, which you'll do later in the next step. This section of the bar will change based on the status of your current branch and repository. Different context dependent actions will be available that let you exchange data between your local and remote repositories.</p> <p><img src="/assets/images/help/desktop/getting-started-guide/explore-github-desktop.png" alt="Explore GitHub Desktop"></p> </li> 
       </ul> 
       <h4 id="changes-and-history"><a href="#changes-and-history">Changes and History</a></h4> 
       <p>In the left sidebar, you'll find the <strong>Changes</strong> and <strong>History</strong> views. <img src="/assets/images/help/desktop/changes-and-history.png" alt="The Changes and History tabs"></p> 
       <ul> 
        <li> <p>The <strong>Changes</strong> view shows changes you've made to files in your current branch but haven't committed to your local repository. At the bottom, there is a box with "Summary" and "Description" text boxes and a <strong>Commit to BRANCH</strong> button. This is where you'll commit new changes. The <strong>Commit to BRANCH</strong> button is dynamic and will display which branch you're committing your changes to. <img src="/assets/images/help/desktop/getting-started-guide/commit-area.png" alt="Commit area"></p> </li> 
        <li> <p>The <strong>History</strong> view shows the previous commits on the current branch of your repository. You should see an "Initial commit" that was created by GitHub Desktop when you created your repository. To the right of the commit, depending on the options you selected while creating your repository, you may see <em>.gitattributes</em>, <em>.gitignore</em>, <em>LICENSE</em>, or <em>README</em> files. You can click each file to see a diff for that file, which is the changes made to the file in that commit. The diff only shows the parts of the file that have changed, not the entire contents of the file. <img src="/assets/images/help/desktop/getting-started-guide/history-view.png" alt="History view"></p> </li> 
       </ul> 
       <h3 id="part-4-publishing-your-repository-to-github"><a href="#part-4-publishing-your-repository-to-github">Part 4: Publishing your repository to GitHub</a></h3> 
       <p>When you create a new repository, it only exists on your computer and you are the only one who can access the repository. You can publish your repository to GitHub to keep it synchronized across multiple computers and allow other people to access it. To publish your repository, push your local changes to GitHub.</p> 
       <ol> 
        <li> <p>Click <strong>Publish repository</strong> in the menu bar. <span class="procedural-image-wrapper"><img src="/assets/images/help/desktop/getting-started-guide/publish-repository.png" alt="Publish repository"></span></p> 
         <ul> 
          <li>GitHub Desktop automatically fills the "Name" and "Description" fields with the information you entered when you created the repository.</li> 
          <li><strong>Keep this code private</strong> lets you control who can view your project. If you leave this option unselected, other users on GitHub will be able to view your code. If you select this option, your code will not be publicly available.</li> 
          <li>The <strong>Organization</strong> drop-down menu, if present, lets you publish your repository to a specific organization that you belong to on GitHub.</li> 
         </ul> <p><span class="procedural-image-wrapper"><img src="/assets/images/help/desktop/getting-started-guide/publish-repository-steps.png" alt="Publish repository steps"></span></p> </li> 
        <li> <p>Click the <strong>Publish Repository</strong> button.</p> </li> 
        <li> <p>You can access the repository on GitHub.com from within GitHub Desktop. In the file menu, click <strong>Repository</strong>, then click <strong>View on GitHub</strong>. This will take you directly to the repository in your default browser.</p> </li> 
       </ol> 
       <h3 id="part-5-making-committing-and-pushing-changes"><a href="#part-5-making-committing-and-pushing-changes">Part 5: Making, committing, and pushing changes</a></h3> 
       <p>Now that you've created and published your repository, you're ready to make changes to your project and start crafting your first commit to your repository.</p> 
       <ol> 
        <li> <p>To launch your external editor from within GitHub Desktop, click <strong>Repository</strong>, then click <strong>Open in <em>EDITOR</em></strong>. For more information, see "<a href="/en/desktop/getting-started-with-github-desktop/configuring-a-default-editor">Configuring a default editor</a>." <span class="procedural-image-wrapper"><img src="/assets/images/help/desktop/getting-started-guide/open-in-editor.png" alt="Open in editor"></span></p> </li> 
        <li> <p>Make some changes to the <em>README.md</em> file that you previously created. You can add information that describes your project, like what it does and why it is useful. When you are satisfied with your changes, save them in your text editor.</p> </li> 
        <li> <p>In GitHub Desktop, navigate to the <strong>Changes</strong> view. In the file list, you should see your <em>README.md</em>. The checkmark to the left of the <em>README.md</em> file indicates that the changes you've made to the file will be part of the commit you make. In the future, you might make changes to multiple files but only want to commit the changes you've made to some of the files. If you click the checkmark next to a file, that file will not be included in the commit. <span class="procedural-image-wrapper"><img src="/assets/images/help/desktop/getting-started-guide/viewing-changes.png" alt="Viewing changes"></span></p> </li> 
        <li> <p>At the bottom of the <strong>Changes</strong> list, enter a commit message. To the right of your profile picture, type a short description of the commit. Since we're changing the <em>README.md</em> file, "Add information about purpose of project" would be a good commit summary. Below the summary, you'll see a "Description" text field where you can type a longer description of the changes in the commit, which is helpful when looking back at the history of a project and understanding why changes were made. Since you're making a basic update of a <em>README.md</em> file, you can skip the description. <span class="procedural-image-wrapper"><img src="/assets/images/help/desktop/getting-started-guide/commit-message.png" alt="Commit message"></span></p> </li> 
        <li> <p>Click <strong>Commit to BRANCH NAME</strong>. The commit button shows your current branch so you can be sure to commit to the branch you want. <span class="procedural-image-wrapper"><img src="/assets/images/help/desktop/getting-started-guide/click-commit-to-master.png" alt="Commit to branch"></span></p> </li> 
        <li> <p>To push your changes to the remote repository on GitHub, click <strong>Push origin</strong>. <span class="procedural-image-wrapper"><img src="/assets/images/help/desktop/getting-started-guide/push-to-origin.png" alt="Push origin"></span></p> </li> 
       </ol> 
       <ul> 
        <li>The <strong>Push origin</strong> button is the same one that you clicked to publish your repository to GitHub. This button changes contextually based on where you are at in the Git workflow. It should now say <code>Push origin</code> with a <code>1</code> next to it, indicating that there is one commit that has not been pushed up to GitHub.</li> 
        <li>The "origin" in <strong>Push origin</strong> means that you are pushing changes to the remote called <code>origin</code>, which in this case is your project's repository on GitHub.com. Until you push any new commits to GitHub, there will be differences between your project's repository on your computer and your project's repository on GitHub.com. This allows you to work locally and only push your changes to GitHub.com when you're ready.</li> 
       </ul> 
       <ol start="7"> 
        <li>In the window to the right of the <strong>Changes</strong> view, you'll see suggestions for actions you can do next. To open the repository on GitHub in your browser, click <strong>View on GitHub</strong>. <span class="procedural-image-wrapper"><img src="/assets/images/help/desktop/available-actions.png" alt="Available actions"></span></li> 
        <li>In your browser, click <strong>2 commits</strong>. You'll see a list of the commits in this repository on GitHub. The first commit should be the commit you just made in GitHub Desktop. <span class="procedural-image-wrapper"><img src="/assets/images/help/desktop/getting-started-guide/click-two-commits.png" alt="Click two commits"></span></li> 
       </ol> 
       <h3 id="conclusion"><a href="#conclusion">Conclusion</a></h3> 
       <p>You've now created a repository, published the repository to GitHub, made a commit, and pushed your changes to GitHub. You can follow this same workflow when contributing to other projects that you create or collaborate on.</p>
