       <div class="lead-mktg">
        <p>You can configure Apache Maven to publish packages to GitHub Packages and to use packages stored on GitHub Packages as dependencies in a Java project.</p>
       </div> 
       <div class="product-callout border rounded-1 mb-4 p-3 color-border-success color-bg-success"> 
        <p>GitHub Packages is available with GitHub Free, GitHub Pro, GitHub Free for organizations, GitHub Team, GitHub Enterprise Cloud, GitHub Enterprise Server, and GitHub AE.</p> 
        <p><br>GitHub Packages is not available for private repositories owned by accounts using legacy per-repository plans. Also, accounts using legacy per-repository plans cannot access the Container registry since these accounts are billed by repository. For more information, see "<a href="/en/articles/github-s-products">GitHub's products</a>."</p>

       <h3 id="authenticating-to-github-packages"><a href="#authenticating-to-github-packages">Authenticating to GitHub Packages</a></h3> 
       <p>You need an access token to publish, install, and delete packages.</p> 
       <p>You can use a personal access token (PAT) to authenticate to GitHub Packages or the GitHub API. When you create a personal access token, you can assign the token different scopes depending on your needs. For more information about packages-related scopes for a PAT, see "<a href="/en/packages/learn-github-packages/about-permissions-for-github-packages#about-scopes-and-permissions-for-package-registries">About permissions for GitHub Packages</a>."</p> 
       <p>To authenticate to a GitHub Packages registry within a GitHub Actions workflow, you can use:</p> 
       <ul> 
        <li><code>GITHUB_TOKEN</code> to publish packages associated with the workflow repository.</li> 
        <li>a PAT to install packages associated with other private repositories (which <code>GITHUB_TOKEN</code> can't access).</li> 
       </ul> 
       <p>For more information about <code>GITHUB_TOKEN</code> used in GitHub Actions workflows, see "<a href="/en/actions/reference/authentication-in-a-workflow#using-the-github_token-in-a-workflow">Authentication in a workflow</a>."</p> 
       <h4 id="authenticating-with-a-personal-access-token"><a href="#authenticating-with-a-personal-access-token">Authenticating with a personal access token</a></h4> 
       <p>You must use a personal access token with the appropriate scopes to publish and install packages in GitHub Packages. For more information, see "<a href="/en/packages/learn-github-packages/about-github-packages#authenticating-to-github-packages">About GitHub Packages</a>."</p> 
       <p>You can authenticate to GitHub Packages with Apache Maven by editing your <em>~/.m2/settings.xml</em> file to include your personal access token. Create a new <em>~/.m2/settings.xml</em> file if one doesn't exist.</p> 
       <p>In the <code>servers</code> tag, add a child <code>server</code> tag with an <code>id</code>, replacing <em>USERNAME</em> with your GitHub username, and <em>TOKEN</em> with your personal access token.</p> 
       <p>In the <code>repositories</code> tag, configure a repository by mapping the <code>id</code> of the repository to the <code>id</code> you added in the <code>server</code> tag containing your credentials. Replace <em>OWNER</em> with the name of the user or organization account that owns the repository. Because uppercase letters aren't supported, you must use lowercase letters for the repository owner even if the GitHub user or organization name contains uppercase letters.</p> 
       <p>If you want to interact with multiple repositories, you can add each repository to separate <code>repository</code> children in the <code>repositories</code> tag, mapping the <code>id</code> of each to the credentials in the <code>servers</code> tag.</p> 
       <p>GitHub Packages supports <code>SNAPSHOT</code> versions of Apache Maven. To use the GitHub Packages repository for downloading <code>SNAPSHOT</code> artifacts, enable SNAPSHOTS in the POM of the consuming project or your <em>~/.m2/settings.xml</em> file.</p>

       <h3 id="publishing-a-package"><a href="#publishing-a-package">Publishing a package</a></h3> 
       <p>By default, GitHub publishes the package to an existing repository with the same name as the package. For example, GitHub will publish a package named <code>com.example:test</code> in a repository called <code>OWNER/test</code>.</p> 
       <p>If you would like to publish multiple packages to the same repository, you can include the URL of the repository in the <code>&lt;distributionManagement&gt;</code> element of the <em>pom.xml</em> file. GitHub will match the repository based on that field. Since the repository name is also part of the <code>distributionManagement</code> element, there are no additional steps to publish multiple packages to the same repository.</p> 
       <p>For more information on creating a package, see the <a href="https://maven.apache.org/guides/getting-started/maven-in-five-minutes.html">maven.apache.org documentation</a>.</p> 
       <ol> 
        <li> <p>Edit the <code>distributionManagement</code> element of the <em>pom.xml</em> file located in your package directory, replacing <code>OWNER</code> with the name of the user or organization account that owns the repository and <code>REPOSITORY</code> with the name of the repository containing your project.</p> <pre><code class="hljs language-xml"><span class="hljs-tag">&lt;<span class="hljs-name">distributionManagement</span>&gt;</span>
   <span class="hljs-tag">&lt;<span class="hljs-name">repository</span>&gt;</span>
     <span class="hljs-tag">&lt;<span class="hljs-name">id</span>&gt;</span>github<span class="hljs-tag">&lt;/<span class="hljs-name">id</span>&gt;</span>
     <span class="hljs-tag">&lt;<span class="hljs-name">name</span>&gt;</span>GitHub OWNER Apache Maven Packages<span class="hljs-tag">&lt;/<span class="hljs-name">name</span>&gt;</span>
     <span class="hljs-tag">&lt;<span class="hljs-name">url</span>&gt;</span>https://maven.pkg.github.com/OWNER/REPOSITORY<span class="hljs-tag">&lt;/<span class="hljs-name">url</span>&gt;</span>

<span class="hljs-tag">&lt;/<span class="hljs-name">distributionManagement</span>&gt;</span>
</code></pre> </li> 
        <li> <p>Publish the package.</p> <pre><code class="hljs language-shell">$ mvn deploy</code></pre> </li> 
       </ol> 
       <p>After you publish a package, you can view the package on GitHub. For more information, see "<a href="/en/packages/manage-packages/viewing-packages">Viewing packages</a>."</p> 
       <h3 id="installing-a-package"><a href="#installing-a-package">Installing a package</a></h3> 
       <p>To install an Apache Maven package from GitHub Packages, edit the <em>pom.xml</em> file to include the package as a dependency. If you want to install packages from more than one repository, add a <code>repository</code> tag for each. For more information on using a <em>pom.xml</em> file in your project, see "<a href="https://maven.apache.org/guides/introduction/introduction-to-the-pom.html">Introduction to the POM</a>" in the Apache Maven documentation.</p> 
       <ol> 
        <li> <p>Authenticate to GitHub Packages. For more information, see "<a href="#authenticating-to-github-packages">Authenticating to GitHub Packages</a>."</p> </li> 
        <li> <p>Add the package dependencies to the <code>dependencies</code> element of your project <em>pom.xml</em> file, replacing <code>com.example:test</code> with your package.</p> <pre><code class="hljs language-xml"><span class="hljs-tag">&lt;<span class="hljs-name">dependencies</span>&gt;</span>

        <li> <p>Install the package.</p> <pre><code class="hljs language-shell">$ mvn install</code></pre> </li> 
       </ol>
