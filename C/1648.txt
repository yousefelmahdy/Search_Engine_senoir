      <p><b>Twinkle</b> is a popular JavaScript <a href="/wiki/Wikipedia:Gadget" title="Wikipedia:Gadget">Wikipedia gadget</a> that gives <a href="/wiki/Wikipedia:User_access_levels#Autoconfirmed_users" title="Wikipedia:User access levels">autoconfirmed registered users</a> many extra options to assist them in common Wikipedia maintenance tasks and to help them deal with acts of <a href="/wiki/Wikipedia:Vandalism" title="Wikipedia:Vandalism">vandalism</a> or unconstructive edits. It provides users with three types of rollback functions and includes a full library of speedy deletion functions, user warnings and welcomes, maintenance tags, semi-automatic reporting of vandals, and more. In addition, it gives <a href="/wiki/Wikipedia:Administrators" title="Wikipedia:Administrators">administrators</a> more tools to help them carry out their duties. </p>

       <h2><span class="mw-headline" id="Quick_info">Quick info</span></h2> 
       <dl>
        <dt>
         Getting started<span class="anchor" id="Installation"></span>
        </dt> 
        <dd>
         To enable Twinkle on your account and receive any future updates automatically, you can simply enable the "Twinkle" gadget in the <a href="/wiki/Special:Preferences#mw-prefsection-gadgets" title="Special:Preferences">Gadgets section of your Preferences page</a>. (Don't forget to click "Save" at the bottom of the list of gadgets!)
        </dd> 
        <dt>
         Customizing Twinkle
        </dt> 
        <dd>
         To modify many aspects of Twinkle's operation, you can visit <a href="/wiki/Wikipedia:Twinkle/Preferences" title="Wikipedia:Twinkle/Preferences">the Twinkle preferences panel</a>. Make your changes, and click "Save changes" at the bottom of the page. Then make sure to <a href="/wiki/Wikipedia:BYPASS#Cache_clearing_and_disabling" class="mw-redirect" title="Wikipedia:BYPASS"><i>completely</i> clear your browser cache</a> so your changes take effect.
        </dd> 
        <dt>
         Getting help
        </dt> 
        <dd>
         There is comprehensive information about Twinkle at <a href="/wiki/Wikipedia:TW/DOC" class="mw-redirect" title="Wikipedia:TW/DOC">the documentation page</a>. If that page does not answer your question, consider asking at <a href="/wiki/Wikipedia_talk:TW" class="mw-redirect" title="Wikipedia talk:TW">the talk page</a>. IRC users can connect to the <a href="irc://irc.libera.chat/wikipedia-userscripts" class="extiw" title="irc:wikipedia-userscripts">#wikipedia-userscripts</a> or <a href="irc://irc.libera.chat/wikipedia-en" class="extiw" title="irc:wikipedia-en">#wikipedia-en</a> IRC channels.
        </dd> 
        <dt>
         Reporting bugs or requesting features
        </dt> 
        <dd>
         Twinkle is under active development, and we generally fix bugs reasonably quickly. If you find a bug, please help out by reporting it at our <a rel="nofollow" class="external text" href="https://github.com/wikimedia-gadgets/twinkle">GitHub repository</a> (you will need to have a GitHub account). If you don't want to create a GitHub account, or you have a request for a new feature that could be added to Twinkle, you can ask at <a href="/wiki/Wikipedia_talk:TW" class="mw-redirect" title="Wikipedia talk:TW">the talk page</a>.
        </dd> 
        <dt>
         Notes
        </dt>
       </dl> 
       <ul>
        <li>If your account is very new, you will not be able to use Twinkle. You need to be <a href="/wiki/Wikipedia:User_access_levels#Autoconfirmed_users" title="Wikipedia:User access levels">autoconfirmed</a> (at least four days old and have ten edits) to enable the Twinkle gadget.</li> 
        <li>It is no longer possible to use Twinkle on <a href="/wiki/Internet_Explorer" title="Internet Explorer">Internet Explorer</a> (IE) versions 10 or earlier. If you are still using one of these IE versions, try switching to another browser such as <a href="/wiki/Mozilla_Firefox" class="mw-redirect" title="Mozilla Firefox">Mozilla Firefox</a> or <a href="/wiki/Google_Chrome" title="Google Chrome">Google Chrome</a>, as Twinkle is functional on most other modern browsers (including modern smartphone browsers).</li> 
        <li>Some browser add-ons may interfere with Twinkle. If Twinkle still does not load after following the instructions above, consider disabling one or more of your add-ons and then restarting your browser.</li> 
        <li>If you're using a <a href="/wiki/Touchscreen" title="Touchscreen">touchscreen</a> device, you may need to tap <i>and hold</i> the "TW" button in the Vector skin to show the Twinkle menu.</li>
       </ul>

      <h2><span class="mw-headline" id="Abuse">Abuse</span></h2> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:TWINKLEABUSE"></span><span id="WP:TWABUSE"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcuts</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:TWINKLEABUSE&amp;redirect=no">WP:TWINKLEABUSE</a></span></li>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:TWABUSE&amp;redirect=no">WP:TWABUSE</a></span></li>
       </ul>
      </div> 
      <p>Never forget that <b>one takes full responsibility for any action performed</b> using Twinkle. One must <b>understand <a href="/wiki/Wikipedia:Policies_and_guidelines" title="Wikipedia:Policies and guidelines">Wikipedia policies</a></b> and use this tool within these policies or risk having one's account <a href="/wiki/Wikipedia:Blocking_policy" title="Wikipedia:Blocking policy"><b>blocked</b></a>. Anti-vandalism tools, such as Twinkle, <a href="/wiki/Wikipedia:Huggle" title="Wikipedia:Huggle">Huggle</a>, and <a href="/wiki/Wikipedia:Rollback" title="Wikipedia:Rollback">rollback</a>, should not be used to undo changes that are constructive and made in good faith. </p>
      <p>If a change is merely "unsatisfactory" in some way, undoing/reverting should not be the first response. Editors should either make a reasonable attempt to <b>improve</b> the change, or should simply leave it in place for future editors to improve. Undo/revert is appropriate in cases where the contribution is arguably "wrong" (consider moving it to the Talk page), or is unreasonably difficult to fix (e.g. incomprehensible, and the author is unresponsive), or is actually harmful to the article (such as vandalism). A plain language edit summary (not merely tags) should be used when reverting changes that appear to have been made in good faith because many contributors will not recognize minimalistic tags and will not learn what the problem was and are likely to repeat it. </p> 
      <h2><span id="Is_Twinkle_loading_unreliably.3F"></span><span class="mw-headline" id="Is_Twinkle_loading_unreliably?">Is Twinkle loading unreliably?</span></h2> 
      <p>In the past, you could install Twinkle by adding the text <code>importScript('User:AzaToth/twinkle.js');</code> to your personal script page; however, this practice can make Twinkle load unreliably (Twinkle might not load on every page you visit, or it might show up but not work properly). </p>
      <p>To rectify this problem, you should remove the line <code>importScript('User:AzaToth/twinkle.js');</code> from <a href="/wiki/Special:MyPage/skin.js" title="Special:MyPage/skin.js">your personal script file</a>, and then turn on the Twinkle gadget using the instructions above. </p>
      <p>If you have any old configuration set up using TwinkleConfig or FriendlyConfig, it will still be effective. However, if you use the <a href="/wiki/Wikipedia:TWPREFS" class="mw-redirect" title="Wikipedia:TWPREFS">preferences panel</a> to modify your configuration, you are advised to remove all old Twinkle/Friendly configuration variables from your skin JavaScript file. </p> 
      <h2><span class="mw-headline" id="History">History</span></h2> 
      <p>Twinkle has its origins as a reversion script based on ideas found in <a href="/wiki/User:Aaron_Schulz" title="User:Aaron Schulz">Aaron Schulz</a>'s script collection. It was developed by <a href="/wiki/User:AzaToth" title="User:AzaToth">AzaToth</a> and released in 2007. It has grown into a tool with many more capabilities than its reversion roots and is used by thousands of <a href="/wiki/Wikipedia:Wikipedians" title="Wikipedia:Wikipedians">Wikipedians</a>. The toolset has expanded over the years: the collection of tools formerly known as Friendly, developed by <a href="/wiki/User:Ioeth" title="User:Ioeth">Ioeth</a>, became part of Twinkle in 2011, and a new module to make user blocking a smoother process for administrators was developed in 2015. Other major contributors include <a href="/wiki/User:Amalthea" title="User:Amalthea">Amalthea</a>, <a href="/wiki/User:Amorymeltzer" title="User:Amorymeltzer">Amorymeltzer</a>, <a href="/wiki/User:MusikAnimal" title="User:MusikAnimal">MusikAnimal</a>, <a href="/wiki/User:SD0001" title="User:SD0001">SD0001</a> and <a href="/wiki/User:This,_that_and_the_other" title="User:This, that and the other">This, that and the other</a>. The gadget continues to be updated and improved, in order to accommodate user requests and changes to Wikipedia processes. </p> 
      <h2><span class="mw-headline" id="Use_on_other_wikis">Use on other wikis</span></h2> 
      <div role="note" class="hatnote navigation-not-searchable">
       See also: <a href="/wiki/Wikipedia:Twinkle/Localisation" title="Wikipedia:Twinkle/Localisation">Wikipedia:Twinkle/Localisation</a>
      </div> 
      <p>UPDATE: As of 2021, the <a rel="nofollow" class="external text" href="https://github.com/wikimedia-gadgets/twinkle-starter">twinkle-starter</a> project provides a streamlined way for localising Twinkle to other wikis. </p> 
      <hr> 
      <p>A common request on the Twinkle talk page comes from those who wish to use Twinkle on other wikis. Some parts of Twinkle are easily portable to other <a href="/wiki/MediaWiki" title="MediaWiki">MediaWiki</a>-based wikis: the revert/rollback module, for example. But most modules have been designed only with English Wikipedia in mind. For instance, the CSD module relies on templates that most likely do not exist on other wikis. </p>
      <p>Users who wish to make Twinkle's functionality available on other wiki projects should consider the following suggestions: </p> 
      <ul>
       <li>The "fluff" (revert/rollback), "diff" (quick access to diff pages), and "unlink" (automated orphaning) modules can be used elsewhere with little or no modification. In the future, it is possible that these scripts will be packaged and made available for a quick setup and update process. For individual users, this can be done by following the instructions <a href="https://meta.wikimedia.org/wiki/User:Ahmad252/How_to_globally_Twinkle" class="extiw" title="m:User:Ahmad252/How to globally Twinkle">here</a>.</li> 
       <li>If you wish to harness more of Twinkle's functionality, you may like to find a willing, <a href="/wiki/JavaScript" title="JavaScript">JavaScript</a>-savvy user on your wiki who can tweak Twinkle scripts to suit your site. The Twinkle developers are happy to help these efforts, to a reasonable degree: just ask at <a href="/wiki/Wikipedia_talk:TW" class="mw-redirect" title="Wikipedia talk:TW">WT:TW</a>, or join the <a href="irc://irc.freenode.net/wikipedia-userscripts" class="extiw" title="freenode:wikipedia-userscripts">#wikipedia-userscripts</a> IRC channel (but be prepared to wait for an answer).</li> 
       <li>The <code>morebits.js</code> library is likely to be of use to script developers, both here on English Wikipedia and other MediaWiki-based wikis. The latest version can be found at the <a rel="nofollow" class="external text" href="https://github.com/wikimedia-gadgets/twinkle/blob/master/morebits.js">Twinkle GitHub repository</a>. The comment at the top of the file should be read to find out more.</li>
      </ul> 
      <hr> 
      <p><a href="https://meta.wikimedia.org/wiki/User:Xiplus/TwinkleGlobal" class="extiw" title="meta:User:Xiplus/TwinkleGlobal">TwinkleGlobal</a> exists as a fork of Twinkle that functions across all WMF wikis. It allows reverting edits, reporting IPs and accounts to <a href="https://meta.wikimedia.org/wiki/SRG" class="extiw" title="meta:SRG">SRG</a>, and requesting speedy deletion of pages. </p>
