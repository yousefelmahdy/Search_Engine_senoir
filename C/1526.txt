      <p><b><a href="/wiki/Transclusion" title="Transclusion">Transclusion</a></b> means the inclusion of the content of one document within another document by <a href="/wiki/Reference_(computer_science)" title="Reference (computer science)">reference</a>. In Wikipedia transclusion, the <a href="/wiki/MediaWiki" title="MediaWiki">MediaWiki</a> software will refer to the content of one page, the <a href="/wiki/Wikipedia:Template" class="mw-redirect" title="Wikipedia:Template">template</a>, for inclusion into the content of any other page, the target page. Similar to the way any <a href="/wiki/Wikipedia:Page_name" title="Wikipedia:Page name">page name</a> can become a link by placing its name in double square brackets, any page name can become a template by placing it in double curly braces, <b><code>{{<i>Namespace</i>:<i><a href="/wiki/Wikipedia:Page_name" title="Wikipedia:Page name">Pagename</a></i>}}</code></b>. Changes made to the template are then automatically reflected on all pages into which that page has been transcluded (though a <a href="/wiki/Wikipedia:Purge" title="Wikipedia:Purge">purge</a> may sometimes be necessary). </p>
      <p>If the template's <a href="/wiki/Wikipedia:Page_name" title="Wikipedia:Page name">page name</a> does not begin with a namespace, it is assumed to be in the Template namespace. To refer to a page in the "Main" (article) namespace, it is necessary to prefix it with a colon (<b>:</b>). Examples: </p> 
      <ul>
       <li><b><code>{{Stochastic processes}}</code></b> will transclude from the page <a href="/wiki/Template:Stochastic_processes" title="Template:Stochastic processes">Template:Stochastic processes</a></li> 
       <li><b><code>{{:Stochastic processes}}</code></b> will transclude from the page <a href="/wiki/Stochastic_processes" class="mw-redirect" title="Stochastic processes">Stochastic processes</a> (an article, in the Main namespace)</li> 
       <li><b><code>{{WP:Assume good faith}}</code></b> will transclude from the page <a href="/wiki/Wikipedia:Assume_good_faith" title="Wikipedia:Assume good faith">Wikipedia:Assume good faith</a></li>
      </ul> 
      <p>What will transclude from a page can be controlled with the tags &lt;noinclude&gt;, &lt;onlyinclude&gt; and &lt;includeonly&gt; placed in the source page wikitext. (See <a href="#Markup">markup</a>, below.) The use of tags enables the template to transclude partially, otherwise the double curly braces will always transclude all content. For <i>selective</i> transclusion the &lt;includeonly&gt; parts of the template are named (with the help of <a href="/wiki/Wikipedia:Parser_functions" class="mw-redirect" title="Wikipedia:Parser functions">parser functions</a>) and that template is then called with the <a href="/wiki/Help:Template#Parameters" title="Help:Template">parameter</a> <code>transcludesection=<i>name</i></code>. </p>
      <p>Transclusion events occur every time the page is <i>loaded</i>, when the template is rendered. Another use of the same mechanism is a one-shot <i><a href="/wiki/Help:Substitution" title="Help:Substitution">substitution</a></i> of the template call itself by its transcluded source page. A template call <code>{{<i><a href="/wiki/Wikipedia:Page_name#Namespace,_pagename,_and_fullpagename" title="Wikipedia:Page name">fullpagename</a></i>}}</code> with the "subst:" prefix <code>{{subst:<i>fullpagename</i>}}</code> is transcluded once but never again, because the wikitext of that call is replaced (substituted) when its page is <i>saved</i>. What was a one-time template call becomes the actual wikitext of the template at the time of its call. </p>

      <h2><span class="mw-headline" id="How_transclusion_works">How transclusion works</span></h2> 
      <div role="note" class="hatnote navigation-not-searchable">
       Note: this section is transcluded from <a href="/wiki/Help:Transclusion/How_Transclusion_Works" title="Help:Transclusion/How Transclusion Works">Help:Transclusion/How Transclusion Works</a>
      </div> 
      <p>To <b>transclude</b> any <b>source</b> page (within a single MediaWiki project, such as en:Wikipedia), include the following code in the <b>target</b> page: </p>
      <p><code>{{SOURCEPAGE}}</code> </p>
      <p>Whenever a target page with this code (<code>{{SOURCEPAGE}}</code>) is rendered, the engine will include the entire content of <code>SOURCEPAGE</code> in the target page. </p>
      <p>In the example below, let us start with target page <b>A</b> and <code>SOURCEPAGE</code> <b>B</b>. <br> If <b>B</b> is transcluded in <b>A</b>, then the engine will include <i>in that place</i> not the code (<code>{{B}}</code>) itself but the content of source page <b>B</b> (which consists of only one word, foo). </p>
      <p>The top row shows how the target page <b>A</b> renders, while the bottom row shows the code required to transclude the source page within the target page. Note that the position of the code determines how each of the target pages <b>A</b>, <b>P</b>, and <b>Q</b> will render. </p>
      <p>The source page content foo will not be highlighted nor boxed on the target page. </p> 
      <div style="margipadding: 0.5em;">
       <a href="/wiki/File:Transclusion-multiple-foo.svg" class="image"><img alt="Transclusion-multiple-foo.svg" src="//upload.wikimedia.org/wikipedia/commons/thumb/2/29/Transclusion-multiple-foo.svg/540px-Transclusion-multiple-foo.svg.png" decoding="async" width="540" height="227" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/2/29/Transclusion-multiple-foo.svg/810px-Transclusion-multiple-foo.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/2/29/Transclusion-multiple-foo.svg/1080px-Transclusion-multiple-foo.svg.png 2x" data-file-width="540" data-file-height="227"></a>
      </div> 
      <p>A typical use case for transclusion would be if you wanted to place the same welcome message on multiple newcomer talk pages. </p>
      <p>Transclusion creates a "live" link between the template&nbsp;page and the target&nbsp;page(s) upon which the template's contents should appear. When the template is edited, the rendering of that template on the target&nbsp;pages is updated as well. Let's say you create a (template) page in Wikipedia with the address of a local Wikimedia event. Next, you transclude that template on your talk page, and also on the talk pages of all local editors to be invited. A week later, the event is moved to a slightly larger place. You would then update the address template, and the <i>revised</i> address will automatically appear on all the other attendees' pages. </p>
      <p>Breaking existing transclusions in a template is called <b>breakage</b>. It is to be avoided if possible, as the break will propagate through every target page in which the template (source) page was transcluded. </p> 
      <h2><span class="mw-headline" id="Transclusion_syntax">Transclusion syntax</span></h2> 
      <p>The general <a href="/wiki/Syntax_(programming_languages)" title="Syntax (programming languages)">syntax</a> for <a href="/wiki/Transclusion" title="Transclusion">transclusion</a> on Wikipedia is <code>{{<a href="/wiki/Wikipedia:Namespace" title="Wikipedia:Namespace">Namespace</a>:<a href="/wiki/Wikipedia:Page_name" title="Wikipedia:Page name">Pagename</a>}}</code>, where <code>Namespace:Pagename</code> describes the title of a Wikipedia page. </p>
      <p>Using transclusion syntax requires some knowledge of page titles. On Wikipedia, non-article pages have titles with a visible namespace and pagename separated by a colon; for example, the page <a href="/wiki/Wikipedia:Tips" title="Wikipedia:Tips">Wikipedia:Tips</a> has <code>Wikipedia</code> as the namespace, and <code>Tips</code> as the pagename. But encyclopedic articles only have a visible pagename in their titles: for example the article <a href="/wiki/Potato" title="Potato">Potato</a> has <code>Potato</code> as the pagename, and the namespace, called <a href="/wiki/Wikipedia:What_is_an_article%3F#Namespace" title="Wikipedia:What is an article?">main namespace</a>, is not visible. </p> 
      <ul>
       <li>The <a href="/wiki/Help:Wikitext" title="Help:Wikitext">wikitext</a> <code>{{Namespace:Pagename}}</code> will transclude the page titled <code>Namespace:Pagename</code>. For example if a page has the wikitext <code>{{Wikipedia:Notability}}</code> in it, it will transclude the page <a href="/wiki/Wikipedia:Notability" title="Wikipedia:Notability">Wikipedia:Notability</a> into it. Please note that <code>{{WP:Notability}}</code> would do the exactly the same thing, as <code>WP:</code> is a <a href="/wiki/Wikipedia:Namespace#Aliases" title="Wikipedia:Namespace">namespace alias</a>, which is <a href="/wiki/Wikipedia:Shortcut#Namespace_aliases" title="Wikipedia:Shortcut">automatically translated</a> by the Wikipedia servers to <code>Wikipedia:</code>.</li> 
       <li>If the namespace is omitted, but the colon is included, like <code>{{:Pagename}}</code>, the encyclopedic article <code>Pagename</code> will be transcluded. For example <code>{{:Notability}}</code> will transclude the article <a href="/wiki/Notability" title="Notability">Notability</a>.</li> 
       <li>If both namepace and colon are omitted, like <code>{{Pagename}}</code>, the <code>Template:Pagename</code> will be transcluded. For example <code>{{Notability}}</code>, and also <code>{{Template:Notability}}</code>, will both transclude the <a href="/wiki/Template:Notability" title="Template:Notability">Template:Notability</a>.</li> 
       <li>For transcluding subpages, see <a href="#Subpages">this section</a>. Also see the <a href="#Template_transclusion">template parameters</a> and <a href="#Transclusion_modifiers">transclusion modifiers</a> section.</li>
      </ul> 
      <p>So, in summary, an example page having the wikitext <code>{{Pagename}}</code> or <code>{{Template:Pagename}}</code> included, will transclude the <code>Template:Pagename</code> into that example page. But <code>{{:Pagename}}</code> will have a different effect: it will transclude the article <code>Pagename</code>. Also, <code>{{Namespace:Pagename}}</code> will transclude the page <code>Namespace:Pagename</code>, for pages with titles of that format. </p> 
      <h3><span class="mw-headline" id="Template_transclusion">Template transclusion</span></h3> 
      <div role="note" class="hatnote navigation-not-searchable">
       Further information: <a href="/wiki/Help:Template#Parameters" title="Help:Template">Help:Template §&nbsp;Parameters</a>
      </div> 
      <div role="note" class="hatnote navigation-not-searchable">
       For a quick guide to templates, see <a href="/wiki/Help:A_quick_guide_to_templates" title="Help:A quick guide to templates">Help:A quick guide to templates</a>.
      </div> 
      <p>The most common application of transclusion is the transclusion of <a href="/wiki/Help:Template" title="Help:Template">templates</a>. Templates are pages that are written with the prime objective of being included in other pages, either through transclusion or <a href="/wiki/Help:Substitution" title="Help:Substitution">substitution</a>. As shown above, the usual syntax for transcluding a template titled Template:Pagename is <code>{{Pagename}}</code>. However, some templates can also use <a href="/wiki/Parameter" title="Parameter">parameters</a>. Parameters are <a href="/wiki/Variable_(computer_science)" title="Variable (computer science)">variables</a> that can be given values (also termed <a href="/wiki/Parameter_(computer_programming)#Parameters_and_arguments" title="Parameter (computer programming)">arguments</a>) that can be passed on to certain templates in order for them to work in different ways. Templates can either use no parameters, a fixed number of parameters, or a variable number of parameters. The number of parameters a template can use is from one upwards. </p>
      <p>The exact syntax for using parameters with templates can vary greatly, depending on the actual template being used. But, for an example template titled Template:Pagename, which happened to use three parameters, the general form would be: </p>
      <p><code>{{Pagename|parameter1|parameter2|parameter3}}</code> </p> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:UNNAMED"></span><span id="WP:POSPARAM"></span><span id="WP:POSITIONAL"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcuts</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:UNNAMED&amp;redirect=no">WP:UNNAMED</a></span></li>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:POSPARAM&amp;redirect=no">WP:POSPARAM</a></span></li>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:POSITIONAL&amp;redirect=no">WP:POSITIONAL</a></span></li>
       </ul>
      </div> 
      <p>with each parameter being substituted with a <code>value</code>, or a <code>parameter name=value</code>, when used in practice. Notice that each parameter is separated with a <a href="/wiki/Vertical_bar" title="Vertical bar">vertical bar</a>. Parameters that take the form <code>value</code> are called <i>unnamed</i> or <i>positional parameters.</i> Parameters that take the form <code>parameter name=value</code> are called <i>named parameters</i>. With unnamed parameters, the first, second, third etc. parameters are equivalent to parameters '1', '2', '3' etc. in <a href="/wiki/Wikipedia:Template_documentation" title="Wikipedia:Template documentation">template documentation.</a> Unnamed parameters must be in the correct order, and best placed before named parameters. An example of the syntax using the <a href="/wiki/Template:Collapse_top" title="Template:Collapse top">Template:Collapse top</a>: </p>
      <p><code>{{Collapse top|This is the title text|This is a custom warning line|left=true}}</code> </p>
      <p>In this particular example, three parameters are used, but Template:Collapse top can actually use a variable number of parameters. <code>This is the title text</code> and <code>This is a custom warning line</code> are the values of unnamed parameters '1' and '2'; and <code>true</code> is the value of the named parameter <code>left</code>. </p>
      <p>For more details, see <a href="/wiki/Help:Template" title="Help:Template">Help:Template</a>. Also, see <a href="/wiki/Wikipedia:Template_index" title="Wikipedia:Template index">Wikipedia:Template index</a> for a list of links to available Main Namespace related templates (usually for encyclopedic articles); and other Namespace related templates (usually for non-article pages); there is also a search function. </p> 
      <h3><span class="mw-headline" id="Substitution">Substitution</span></h3> 
      <p><a href="/wiki/Help:Substitution" title="Help:Substitution">Substitution</a> has a similar double curly-bracket syntax, and its action is similar to transclusion. When you preview a page with substitution wikitext in it, its action is identical to transclusion. But when you save a page with substitution wikitext in it, it differs in that it substitutes that wikitext with the actual wikitext of the template. </p>
      <p>For example when <code>{{subst:Like}}</code> is saved, it would substitute that wikitext with the actual wikitext from the <a href="/wiki/Template:Like" title="Template:Like">Template:Like</a>. In practice, the main difference that this makes is that if an update happened to the wikitext of the Template:Like, it would not update any pages where <code>{{subst:Like}}</code> had already been saved. </p> 
      <h3><span class="mw-headline" id="Magic_words">Magic words</span></h3> 
      <p><a href="/wiki/Help:Magic_words" title="Help:Magic words">Magic words</a> are not examples of transclusion. But some have a similar double curly-bracket syntax and action to transclusion. For example, <code>{{FULLPAGENAME}}</code> renders the <a href="/wiki/Wikipedia:Page_name" title="Wikipedia:Page name">Fullpagename</a> of any Wikipedia page. Like templates, some magic words can also take parameters, which are separated using a colon (:); for example <code>{{FULLPAGENAME:value}}</code>. </p>
      <p>Templates do exist for some magic words, for example <a href="/wiki/Template:FULLPAGENAME" title="Template:FULLPAGENAME">Template:FULLPAGENAME</a>; but these just invoke the related magic word if passing parameters using a vertical bar (|); for example like <code>{{FULLPAGENAME|value}}</code>. But magic words parameters are best passed directly by using a colon, for example like <code>{{FULLPAGENAME:value}}</code>, which bypasses any templates. </p> 
      <h3><span class="mw-headline" id="Transclusion_modifiers">Transclusion modifiers</span></h3> 
      <p>As discussed above, with the example <code>{{subst:Like}}</code>, the <code>subst:</code> code alters the transclusion to substitution. This is a <i>transclusion modifier</i>, a type of specialist magic word for altering transclusion in some manner. Another example is <code>{{:Notability}}</code>, where the <code>:</code> (colon character) forces transclusion to the main namespace. There are additional transclusion modifiers such as <code>safesubst:</code>, <code>int:</code>, <code>msg:</code>, <code>msgnw:</code>, and <code>raw:</code>. For more details on their usage see <a href="https://www.mediawiki.org/wiki/Help:Magic_words#Transclusion_modifiers" class="extiw" title="mw:Help:Magic words">mw:Help:Magic words §&nbsp;Transclusion modifiers</a>. Also see the modified commands <code>#section:</code>, <code>#section-x:</code> and <code>#section-h:</code> used for <a href="/wiki/Help:Labeled_section_transclusion" title="Help:Labeled section transclusion">labeled section transclusion</a>. </p> 
      <h2><span class="mw-headline" id="Applications_of_transclusion">Applications of transclusion</span></h2> 
      <p>The most common application of transclusion is in the use of <a href="/wiki/Help:Template" title="Help:Template">templates</a>. However, other pages are also sometimes transcluded, mainly within project space. </p> 
      <h3><span class="mw-headline" id="Composite_pages">Composite pages</span></h3> 
      <p>Composite pages consist, wholly or partly, of transcluded component pages. The wikitext of such a page may, partly or fully, consist of tags, for the inclusion of component pages. The component pages are usually not in the template namespace, and are often full pages in their own right. Composite pages are intended to gather them into a central location. </p>
      <p>Examples of composite pages: </p> 
      <ul>
       <li><a href="/wiki/Wikipedia:Village_pump" title="Wikipedia:Village pump">Wikipedia:Village pump</a>: mostly consists of the transcluded page <a href="/wiki/Template:Village_pump" title="Template:Village pump">Template:Village pump</a>.</li> 
       <li>A daily page like <a href="/wiki/Wikipedia:Articles_for_deletion/Log/2005_May_31" title="Wikipedia:Articles for deletion/Log/2005 May 31">Wikipedia:Articles for deletion/Log/2005 May 31</a>, where each component page consists of the discussion on the deletion of one Wikipedia page, e.g. <a href="/wiki/Wikipedia:Articles_for_deletion/Sp3tt" title="Wikipedia:Articles for deletion/Sp3tt">Wikipedia:Articles for deletion/Sp3tt</a>. For this day, 75 component pages made up the composite page.</li> 
       <li><a href="https://meta.wikimedia.org/wiki/Meta:Translation/Coordination" class="extiw" title="m:Meta:Translation/Coordination">m:Meta:Translation/Coordination</a>, mainly containing 
        <ul>
         <li><a href="https://meta.wikimedia.org/wiki/Meta:Translation/Coordination/List/Meta" class="extiw" title="m:Meta:Translation/Coordination/List/Meta">m:Meta:Translation/Coordination/List/Meta</a></li> 
         <li><a href="https://meta.wikimedia.org/wiki/Meta:Translation/Coordination/List/Main" class="extiw" title="m:Meta:Translation/Coordination/List/Main">m:Meta:Translation/Coordination/List/Main</a></li>
        </ul></li>
      </ul> 
      <p>This allows the choice between viewing the component pages separately or together. Viewing a composite page is convenient when there are many small, related component pages, in that it allows an overview of all the components without the effort of following numerous links. </p>
      <p>In general, each component page and the composite page are treated separately. While the actual changes on the component pages will be transcluded onto the composite page, the edit history, recent changes, page-watch settings, page protection, TOC, "what links here" links, and other features of the composite page do not reflect, or affect, the histories, watch settings, protection levels, <i>what links here</i> lists of the component pages. The composite page is a page in its own right. The talk page of a composite page is used to talk about the composition and the page in general, not the component pages; although it in turn could be a composite of the talk pages of the component pages. </p>
      <p>Editing a section of a component page can be done directly from the composite page, see <a href="/wiki/Help:Section#Editing_sections_of_included_templates" title="Help:Section">editing sections of included templates</a>. After saving, one ends up editing the component page to which the section belongs. </p>
      <p>On projects with the <a href="/wiki/Help:Interwiki_linking#Interlanguage_links" title="Help:Interwiki linking">interlanguage link feature</a>, the composite page shows the combined interlanguage links of all component pages, hence possibly multiple links for one language or even for one page. </p>
      <p>See also <a href="/wiki/Wikipedia_talk:Template_namespace/Archive_1#transcluding_prose" title="Wikipedia talk:Template namespace/Archive 1">Wikipedia talk:Template namespace/Archive 1#transcluding prose</a>. </p> 
      <h3><span class="mw-headline" id="Pages_with_a_common_section">Pages with a common section</span></h3> 
      <p>When two pages need to discuss the same material in the same way, they can share a section. This involves creating a third page and transcluding that page onto both pages. This third page may be a page in its own right or a subpage of either of the other two, and if the first it may be placed in the same namespace as the other pages or in template namespace. Common sections like this should be marked with an explanatory header, and/or given a special layout, to inform the reader that this section of the page is in a different location, since transcluding shared article sections can easily confuse novice editors and readers alike if left unmarked. </p>
      <p>This can be very useful when two <a href="/wiki/Wikipedia:DAB" class="mw-redirect" title="Wikipedia:DAB">disambiguation</a> pages share content,<sup class="noprint Inline-Template" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Consensus" title="Wikipedia:Consensus"><span title="This claim has reliable sources with contradicting facts (September 2017)">disputed</span></a> <span class="metadata"> – <a href="/wiki/Wikipedia_talk:Manual_of_Style/Disambiguation_pages#Transclusion" title="Wikipedia talk:Manual of Style/Disambiguation pages">discuss</a></span></i>]</sup> or a list page and a disambiguation page share content (see third example below). </p>
      <p>Examples: </p> 
      <ul>
       <li>The <a href="/wiki/Help:Editing_sections_of_included_templates" class="mw-redirect" title="Help:Editing sections of included templates">Help:Editing sections of included templates</a> article is included in <a href="/wiki/Help:Section" title="Help:Section">Help:Section</a> with the markup <code>{{Help:Editing sections of included templates}}</code>. By including a heading in the included article, a user clicking the "Edit" link on that heading in <a href="/wiki/Help:Section" title="Help:Section">Help:Section</a> is automatically directed to edit <a href="/wiki/Help:Editing_sections_of_included_templates" class="mw-redirect" title="Help:Editing sections of included templates">Help:Editing sections of included templates</a>.</li> 
       <li><a href="https://meta.wikimedia.org/wiki/Help:Alphabetic_order" class="extiw" title="m:Help:Alphabetic order">m:Help:Alphabetic order</a> (<a href="https://meta.wikimedia.org/wiki/Help_talk:Alphabetic_order" class="extiw" title="m:Help talk:Alphabetic order">talk</a>, <a href="https://meta.wikimedia.org/wiki/Special:Whatlinkshere/Help:Alphabetic_order" class="extiw" title="m:Special:Whatlinkshere/Help:Alphabetic order">backlinks</a>, <a class="external text" href="https://meta.wikimedia.org//w/index.php?title=Help:Alphabetic_order&amp;action=edit">edit</a>)</li> 
       <li><a href="/wiki/Joseph_Gordon-Levitt" title="Joseph Gordon-Levitt">Joseph Gordon-Levitt</a> transcludes the introduction of <a href="/wiki/HitRecord" title="HitRecord">HitRecord</a> into a summary section of the same name, rather than maintaining two copies of the identical text.</li>
      </ul> 
      <h3><span class="mw-headline" id="Repetition_within_a_page">Repetition within a page</span></h3> 
      <p>On pages where there is a lot of repetitive information&nbsp;— various kinds of lists, usually&nbsp;— it is sometimes useful to make a template that contains the repeating text, and then call that template multiple times. For example, <a href="/wiki/Template:EH_listed_building_row" class="mw-redirect" title="Template:EH listed building row">Template:EH listed building row</a> is used repeatedly to construct tables in many articles. </p>
      <p>Simple repetition of the same text can be handled with repetition of a parameter in a single template: e.g., {{<a href="/wiki/Template:3x" title="Template:3x">3x</a>}}, where <code>{{3x| howdy!}}</code> produces <code> howdy! howdy! howdy!</code>. </p>
      <p>For more information on repetition, see also <a href="https://meta.wikimedia.org/wiki/Help:Recursive_conversion_of_wikitext" class="extiw" title="m:Help:Recursive conversion of wikitext">m:Help:Recursive conversion of wikitext</a>. </p>
      <p>For more information on the current template system, see <a href="/wiki/Wikipedia:Template_namespace" title="Wikipedia:Template namespace">Wikipedia:Template namespace</a>. </p> 
      <h2><span class="mw-headline" id="Partial_transclusion">Partial transclusion</span></h2> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:PARTRANS"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:PARTRANS&amp;redirect=no">WP:PARTRANS</a></span></li>
       </ul>
      </div> 
      <p>By using <code class="nowrap" style="">&lt;noinclude&gt;</code>, <code class="nowrap" style="">&lt;includeonly&gt;</code> and <code class="nowrap" style="">&lt;onlyinclude&gt;</code> markup, it is possible to transclude <i>part</i> of a page, rather than all of it. Such partial transclusions can be achieved by transcluding from any pages, including subpages. It is often useful to exclude part of a page in a transclusion, an example being with template documentation. </p>
      <p>For an example of how this technique can be applied to simplify the creation of <a href="/wiki/Wikipedia:Summary_style" title="Wikipedia:Summary style">summary articles</a>, see how part of the <a href="/wiki/History_of_pathology" title="History of pathology">History of pathology</a> (see the diff <a class="external text" href="https://en.wikipedia.org/w/index.php?title=History_of_pathology&amp;diff=prev&amp;oldid=207668817">here</a>) was transcluded into <a href="/wiki/Pathology" title="Pathology">Pathology</a> (see the diff <a class="external text" href="https://en.wikipedia.org/w/index.php?title=Pathology&amp;diff=prev&amp;oldid=207669487">here</a>) using the <code>{{:History of pathology}}</code> markup. The Pathology article at that time (see <a class="external text" href="https://en.wikipedia.org/w/index.php?title=Pathology&amp;oldid=207669487">here</a>) mainly consisted of transcluded <a href="/wiki/Wikipedia:Manual_of_Style/Lead_section" title="Wikipedia:Manual of Style/Lead section">lead paragraphs</a> and other sections from a number of articles. Look at the source to see how this was done. Since then, the Pathology article has been rewritten, and does not include all these transclusions. </p>
      <p>Another example can be found in the transclusion of part of <a href="/wiki/HitRecord" title="HitRecord">HitRecord</a> (introductory paragraph only) into a same-named summary section in <a href="/wiki/Joseph_Gordon-Levitt#HitRecord" title="Joseph Gordon-Levitt">Joseph Gordon-Levitt</a>. </p>
      <p><span class="anchor" id="Transclusion_markup"></span> </p> 
      <h3><span class="mw-headline" id="Markup">Markup</span></h3> 
      <div role="note" class="hatnote navigation-not-searchable">
       See also: <a href="/wiki/Help:Template#Noinclude,_includeonly,_and_onlyinclude" title="Help:Template">Help:Template §&nbsp;Noinclude, includeonly, and onlyinclude</a>; and <a href="/wiki/Help:Category#Categories_and_templates" title="Help:Category">Help:Category §&nbsp;Categories and templates</a>
      </div> 
      <p>In transclusion, a source page is transcluded into a destination page. But with partial transclusion, only part of that source page will be transcluded into a destination page. But in addition, what is transcluded to a destination page does not have to be visible on the source page. </p>
      <p><i>Page rendering</i> of a source page can be defined as the rendering of that source page when it is saved, which will be the same as the <a href="/wiki/Wikipedia:Preview" class="mw-redirect" title="Wikipedia:Preview">preview</a>. We can call this rendering <i><b>here</b></i>. </p>
      <p><i>Transclusion rendering</i> of a source page can be defined as the rendering of a destination page that has a source page transcluded into it; but only that part of the destination page that was transcluded from the source page. The preview of the transclusion rendering will again be identical. We can call this rendering <i><b>there</b></i>. </p>
      <p>There are three pairs of tags involved in cases where page rendering <i>here</i> should differ from transclusion rendering <i>there</i>. As described earlier, these are <code class="nowrap" style="">&lt;noinclude&gt;</code>, <code class="nowrap" style="">&lt;includeonly&gt;</code> and <code class="nowrap" style="">&lt;onlyinclude&gt;</code>. These tags are invisible, but affect both page rendering <i>here</i> and transclusion rendering <i>there</i>. These tags pair-off to demarcate sections that will create differences. Each tag will describe exceptions to transcluding the whole page named. </p> 
      <pre style="white-space:-moz-pre-wrap; white-space:-pre-wrap; white-space:-o-pre-wrap; white-space:pre-wrap; word-wrap:break-word;">&lt;noinclude&gt;  This section is visible here; but this section is not visible there. Sections outside of these tags will be visible both here and there. &lt;/noinclude&gt;

&lt;onlyinclude&gt; This section is visible here; this section is also visible there. Sections outside of these tags will be visible here, but will not be visible there. &lt;/onlyinclude&gt; 

&lt;includeonly&gt; This section is not visible here; but it is visible there. Sections outside of these tags will be visible both here and there. &lt;/includeonly&gt;</pre> 
      <table class="wikitable" style="text-align: center;"> 
       <tbody>
        <tr> 
         <th>Wikitext </th> 
         <th>What is rendered <i>here</i> (source page) </th> 
         <th>What is transcluded <i>there</i> (destination page) </th>
        </tr> 
        <tr> 
         <td><code class="nowrap" style="">&lt;noinclude&gt;text1&lt;/noinclude&gt;</code> text2 </td> 
         <td><code>text1 text2</code> </td> 
         <td><code>text2</code> </td>
        </tr> 
        <tr> 
         <td><code class="nowrap" style="">&lt;onlyinclude&gt;text1&lt;/onlyinclude&gt;</code> text2 </td> 
         <td><code>text1 text2</code> </td> 
         <td><code>text1</code> </td>
        </tr> 
        <tr> 
         <td><code class="nowrap" style="">&lt;includeonly&gt;text1&lt;/includeonly&gt;</code> text2 </td> 
         <td><code>text2</code> </td> 
         <td><code>text1 text2</code> </td>
        </tr>
       </tbody>
      </table> 
      <p>An important point to note is that <code class="nowrap" style="">&lt;noinclude&gt;</code> and <code class="nowrap" style="">&lt;onlyinclude&gt;</code> do not affect what is page rendered <i>here</i> at all, unlike <code class="nowrap" style="">&lt;includeonly&gt;</code>. The <code class="nowrap" style="">&lt;noinclude&gt;</code> tags stops text inside the tags being transcluded <i>there</i>, while <code class="nowrap" style="">&lt;onlyinclude&gt;</code> has the opposite effect: it stops text outside of the tags from being transcluded <i>there</i>. </p>
      <p>Only <code class="nowrap" style="">&lt;includeonly&gt;</code> stops text from being page rendered <i>here</i>. But naturally enough it is transcluded <i>there</i>. Text outside of the tags will be both rendered <i>here</i> and transcluded <i>there</i>. </p>
      <p>There can be several such sections. Also, they can be nested. All possible differences between <i>here</i> and <i>there</i> are achievable. One example is a content editor who picks an <code class="nowrap" style="">&lt;onlyinclude&gt;</code> section, and then takes a <code class="nowrap" style="">&lt;noinclude&gt;</code> section out of that; but then picks out yet another <code class="nowrap" style="">&lt;onlyinclude&gt;</code> section to append to <i>there</i>; but none of this affects their article in any way. </p>
      <p>The other example is the template programmer, who will <code class="nowrap" style="">&lt;includeonly&gt;</code> the code section and <code class="nowrap" style="">&lt;noinclude&gt;</code> the documentation section of a page. </p>
      <p>A mis-example is, on a the user page, to use <code class="nowrap" style="">&lt;includeonly&gt;</code> to "comment out" a section. While it might comment out sections <i>here</i>, such <a href="/wiki/Invisible_ink" title="Invisible ink">invisible ink</a> may prove to be troublesome; if transcluded, the comments would be revealed. Use <code class="nowrap" style="">&lt;!--...--&gt;</code> instead. </p>
      <p>NoInclude as a name about transcluding is pretty straightforward. A mnemonic for the other two transclusions <i>there</i> is: Inclusion<i>ONLY</i>; and <i>Yet Another ONLY</i>Inclusion. </p> 
      <h2><span class="mw-headline" id="Selective_transclusion">Selective transclusion</span></h2> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:SELECTIVETRANSCLUSION"></span><span id="WP:SELTRANS"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcuts</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:SELECTIVETRANSCLUSION&amp;redirect=no">WP:SELECTIVETRANSCLUSION</a></span></li>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:SELTRANS&amp;redirect=no">WP:SELTRANS</a></span></li>
       </ul>
      </div> 
      <div role="note" class="hatnote navigation-not-searchable">
       See also: <a href="/wiki/Template:Excerpt" title="Template:Excerpt">Template:Excerpt</a> and <a href="/wiki/Template:Transcluded_section" title="Template:Transcluded section">Template:Transcluded section</a>
      </div> 
      <p><i>Selective transclusion</i> is the process of <a href="#Partial_transclusion">partially transcluding</a> one selected section of a document that has more than one transcludable section. As noted above, if only one section of a document is to be transcluded, this can be done by simply surrounding the section of interest with <code>&lt;onlyinclude&gt; … &lt;/onlyinclude&gt;</code> tags, and transcluding the whole page. However, to selectively transclude one section from a template or document into one page, and another section from the same template or document into a second page and/or a different section of the same page, requires a way to: </p> 
      <dl>
       <dd>
        a) uniquely mark each transcludable section in the source document; and
       </dd> 
       <dd>
        b) in the target document(s) (those to show the transcluded sections), a way to specify which section is to be transcluded.
       </dd>
      </dl> 
      <p>This section describes how to accomplish this. There are three ways of doing this: (1) Section header-based transclusion, (2) Labeled section transclusion, and (3) the parametrization method. </p> 
      <h3><span id="Standard_section_transclusion_.28Special_source_document_markup_not_needed.21.29"></span><span class="mw-headline" id="Standard_section_transclusion_(Special_source_document_markup_not_needed!)">Standard section transclusion (Special source document markup not needed!)</span></h3> 
      <p>Using <code>{{#section-h:PAGENAME|SECTIONNAME}}</code>, one can easily transclude the content within a section on one page to another by referring to the standard, ubiquitous headline-based section headers used throughout Wikipedia. To transclude the lead of an article with this method, one can use <span class="nowrap"><code>{{#section-h:PAGENAME}}</code></span>. This is simpler than the already widely used selective transclusion methods of the sections below, which require special source document markup. </p>
      <p><img alt="" src="//upload.wikimedia.org/wikipedia/commons/thumb/4/4b/Exclamationdiamond.svg/30px-Exclamationdiamond.svg.png" decoding="async" width="30" height="30" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/4/4b/Exclamationdiamond.svg/45px-Exclamationdiamond.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/4/4b/Exclamationdiamond.svg/60px-Exclamationdiamond.svg.png 2x" data-file-width="48" data-file-height="48">&nbsp;<b>This selective transclusion method often adds a line break above and/or below the section transclusion, depending upon the source and target document markup; to avoid this issue, wrap the selective transclusion template in a <code>{{<a href="/wiki/Template:Trim" title="Template:Trim">trim</a>}}</code> template.</b> In other words, use the following modifications to the transclusion code listed immediately above: </p> 
      <ul>
       <li>Code for transcluding a section in the body of an article: <code>{{trim|{{#section-h:PAGENAME|SECTIONNAME}}}}</code></li> 
       <li>Code for transcluding the lead of an article: <code>{{trim|{{#section-h:PAGENAME}}}}</code></li>
      </ul> 
      <h4><span class="mw-headline" id="Transcluded_section_hatnote">Transcluded section hatnote</span></h4> 
      <p>For the purpose of indicating where selectively transcluded article content is located, it is helpful to include a <code>{{<a href="/wiki/Template:Transcluded_section" title="Template:Transcluded section">Transcluded section</a>}}</code> hatnote at the top of the corresponding section of an article; i.e., use either <span class="nowrap"><code>{{transcluded section|source=PAGENAME}}</code></span> or <span class="nowrap"><code>{{transcluded section|source=PAGENAME|part=yes}}</code></span>, depending upon whether the transcluded content comprises an entire section of the target page. These templates render as follows. </p> 
      <ul>
       <li><span class="nowrap"><code>{{transcluded section|source=PAGENAME}}</code></span> renders as:</li>
      </ul> 
      <div role="note" class="hatnote navigation-not-searchable relarticle mainarticle selfreference noprint">
       This section is <a class="mw-selflink selflink">transcluded</a> from <a href="/w/index.php?title=PAGENAME&amp;action=edit&amp;redlink=1" class="new" title="PAGENAME (page does not exist)">PAGENAME</a>. <span class="plainlinks" style="font-style: normal"><span style="font-size:85%;">(<a class="external text" href="https://en.wikipedia.org/w/index.php?title=PAGENAME&amp;action=edit">edit</a> | <a class="external text" href="https://en.wikipedia.org/w/index.php?title=PAGENAME&amp;action=history">history</a>)</span></span>
      </div> 
      <ul>
       <li><span class="nowrap"><code>{{transcluded section|source=PAGENAME|part=yes}}</code></span> renders as:</li>
      </ul> 
      <div role="note" class="hatnote navigation-not-searchable relarticle mainarticle selfreference noprint">
       Part of this section is <a class="mw-selflink selflink">transcluded</a> from <a href="/w/index.php?title=PAGENAME&amp;action=edit&amp;redlink=1" class="new" title="PAGENAME (page does not exist)">PAGENAME</a>. <span class="plainlinks" style="font-style: normal"><span style="font-size:85%;">(<a class="external text" href="https://en.wikipedia.org/w/index.php?title=PAGENAME&amp;action=edit">edit</a> | <a class="external text" href="https://en.wikipedia.org/w/index.php?title=PAGENAME&amp;action=history">history</a>)</span></span>
      </div> 
      <h3><span class="mw-headline" id="Using_the_labeled_section_method">Using the labeled section method</span></h3> 
      <div role="note" class="hatnote navigation-not-searchable">
       Main page: <a href="/wiki/Help:Labeled_section_transclusion" title="Help:Labeled section transclusion">Help:Labeled section transclusion</a>
      </div> 
      <p>Labeled-section selective transclusion uses the <a href="/wiki/Wikipedia:PF" class="mw-redirect" title="Wikipedia:PF">parser functions</a> listed in <a href="https://www.mediawiki.org/wiki/Extension:Labeled_Section_Transclusion" class="extiw" title="mw:Extension:Labeled Section Transclusion">mw:Extension:Labeled Section Transclusion</a>, which are enabled on all Wikimedia wikis, to selectively transclude content. See <a href="/wiki/Help:Labeled_section_transclusion" title="Help:Labeled section transclusion">Help:Labeled section transclusion</a> for how labeled section transclusion works. </p> 
      <h3><span class="mw-headline" id="Parametrization_method">Parametrization method</span></h3> 
      <h5><span class="mw-headline" id="Source_document_markup">Source document markup</span></h5> 
      <p>Insert the following line into the "source" document (the one from which text is to be transcluded), immediately preceding the first line of each section to be transcluded, substituting SECTIONNAME (twice) with the unique name of the respective section. The section name can be any identifier and must be unique within that document: </p> 
      <dl>
       <dd>
        <code>&lt;onlyinclude&gt;{{#ifeq:{{{transcludesection|SECTIONNAME}}}|SECTIONNAME|</code>
       </dd>
      </dl> 
      <p>End each such transcludable section with: </p> 
      <dl>
       <dd>
        <code>}}&lt;/onlyinclude&gt;</code>
       </dd>
      </dl> 
      <h5><span class="mw-headline" id="Target_document_markup">Target document markup</span></h5> 
      <p>To transclude a section marked as above into another page (the "target page"), use the following line on that page, substituting PAGENAME for the "source" document from which text to be transcluded, and SECTIONNAME with the name of the section you want to transclude: </p> 
      <dl>
       <dd>
        <code>{{PAGENAME|transcludesection=SECTIONNAME}}</code>
       </dd>
      </dl> 
      <p>Thus each section enclosed within <code>&lt;onlyinclude&gt; … &lt;/onlyinclude&gt; </code> tags will always be rendered when the <code>transcludesection</code> parameter is not set (when the document is viewed ordinarily, or when the document is transcluded without setting the <code>transcludesection</code> parameter as shown below), and will be rendered by transclusion on any page that does set <code>transcludesection</code> to the section's name. It will <i>not</i> be rendered by transclusion that uses the <code>transcludesection</code> parameter but sets it to anything other than the name of the section. </p>
      <p>Also, when providing PAGENAME, without providing a <a href="/wiki/Wikipedia:Namespace" title="Wikipedia:Namespace">Namespace</a>, the wiki will assume that the PAGENAME belongs in the <a href="/wiki/Wikipedia:Template_namespace" title="Wikipedia:Template namespace">Template Namespace</a>. To transclude from a <a href="/wiki/Wikipedia:Main_namespace" class="mw-redirect" title="Wikipedia:Main namespace">Mainspace</a> article, use :PAGENAME. </p> 
      <dl>
       <dd>
        <code>{{:PAGENAME|transcludesection=SECTIONNAME}}</code>
       </dd>
      </dl> 
      <h5><span class="mw-headline" id="Example">Example</span></h5> 
      <p>If we want to make the "Principal Criteria" and "Common Name" sections of <a href="/wiki/Wikipedia:TITLE" class="mw-redirect" title="Wikipedia:TITLE">WP:TITLE</a> be independently transcludable, we edit the WP:TITLE page and enclose the "Principal Criteria" section as follows: </p> 
      <div class="mw-highlight mw-highlight-lang-xml+smarty mw-content-ltr" dir="ltr">
       <pre><span></span><span class="nt">&lt;onlyinclude&gt;</span><span class="cp">{{</span><span class="err">#</span><span class="na">ifeq</span><span class="o">:</span><span class="cp">{{{</span><span class="na">transcludesection</span><span class="o">|</span><span class="na">principalcriteria</span><span class="cp">}}}</span><span class="o">|</span><span class="na">principalcriteria</span><span class="o">|</span>
<span class="o">...</span>
<span class="s1">''</span><span class="o">(</span><span class="na">text</span> <span class="na">of</span> <span class="s2">"Principal Criteria"</span> <span class="na">section</span><span class="o">)</span><span class="s1">''</span>
<span class="o">...</span>
<span class="cp">}}</span><span class="nt">&lt;/onlyinclude&gt;</span>
</pre>
      </div> 
      <p>Similarly, we enclose the "Common Name" section with: </p> 
      <div class="mw-highlight mw-highlight-lang-xml+smarty mw-content-ltr" dir="ltr">
       <pre><span></span><span class="nt">&lt;onlyinclude&gt;</span><span class="cp">{{</span><span class="err">#</span><span class="na">ifeq</span><span class="o">:</span><span class="cp">{{{</span><span class="na">transcludesection</span><span class="o">|</span><span class="na">commonname</span><span class="cp">}}}</span><span class="o">|</span><span class="na">commonname</span><span class="o">|</span>
<span class="o">...</span>
<span class="s1">''</span><span class="o">(</span><span class="na">text</span> <span class="na">of</span> <span class="s2">"Common Name"</span> <span class="na">section</span><span class="o">)</span><span class="s1">''</span>
<span class="o">...</span>
<span class="cp">}}</span><span class="nt">&lt;/onlyinclude&gt;</span>
</pre>
      </div> 
      <p>Then, to transclude the "Principal Criteria" section into another page, we insert into that page: </p> 
      <dl>
       <dd>
        <code>{{WP:TITLE|transcludesection=principalcriteria}}</code>
       </dd>
      </dl> 
      <p>To transclude the "Common Name" section into another page, we insert into that page: </p> 
      <dl>
       <dd>
        <code>{{WP:TITLE|transcludesection=commonname}}</code>
       </dd>
      </dl> 
      <p>Of course, the same page can transclude two or more sections this way by including multiple such lines. </p>
      <p>There is no limit to how many selectable sections for transclusion a document can have. The only requirement is that each <code>transcludesection</code> be given a value that is unique within that page. </p> 
      <h2><span class="mw-headline" id="Subpages">Subpages</span></h2> 
      <p>To transclude a <a href="/wiki/Wikipedia:Subpages" title="Wikipedia:Subpages">subpage</a>: </p> 
      <ul>
       <li>The general syntax is <code>{{Namespace:Pagename/subpagename}}</code>.</li> 
       <li>If it is a subpage of the page you are editing, then it is the general syntax, or <code>{{/subpagename}}</code>.</li> 
       <li>For an encyclopedic article it is <code>{{:Pagename/subpagename}}</code>, although article subpages are normally disabled.</li> 
       <li>For a template, it is the general syntax, or <code>{{Pagename/subpagename}}</code>.</li> 
       <li>You have the option of using <a href="#Transclusion_modifiers">transclusion modifiers</a> and <a href="#Template_transclusion">template parameters</a>, but in practice this is rare.</li>
      </ul> 
      <p>For example, to transclude the <a href="/wiki/Template:Like/doc" title="Template:Like/doc">Template:Like/doc</a>, it would be either <code>{{Template:Like/doc}}</code> or <code>{{Like/doc}}</code>. Subpages are only allowed in certain namespaces, such as User, Talk or Wikipedia pages; currently, subpages <a href="/wiki/Wikipedia:Subpages#Articles_do_not_have_sub-pages_(main_namespace)" title="Wikipedia:Subpages">of main article pages cannot be created</a>. </p>
      <p>Subpages could be used for a shared discussion on talk pages. For example: you want to discuss the deletion and redirecting of the article <a href="/wiki/Pussycat" class="mw-disambig" title="Pussycat">Pussycat</a> to the article <a href="/wiki/Cat" title="Cat">Cat</a> in both talk pages. First, create the subpage <a href="/w/index.php?title=Talk:Pussycat/Let%27s_delete_Pussycat!&amp;action=edit&amp;redlink=1" class="new" title="Talk:Pussycat/Let's delete Pussycat! (page does not exist)">Talk:Pussycat/Let's delete Pussycat!</a>, write your comment into it, then transclude it in <a href="/wiki/Talk:Pussycat" title="Talk:Pussycat">Talk:Pussycat</a> and <a href="/wiki/Talk:Cat" title="Talk:Cat">Talk:Cat</a> using the template <code>{{Talk:Pussycat/Let's delete Pussycat!}}</code>. Comments posted in this subpage will be shown in both talk pages. To ease the editing of the subpage, it may be helpful to precede the transcluded talk with {{<a href="/wiki/Template:Lat" title="Template:Lat">lat</a>}}; for example the wikitext <code>{{lat|Pussycat/Let's delete Pussycat!}}</code> would produce <span class="plainlinks nourlexpansion lx"><a href="/w/index.php?title=Talk:Pussycat/Let%27s_delete_Pussycat!&amp;action=edit&amp;redlink=1" class="new" title="Talk:Pussycat/Let's delete Pussycat! (page does not exist)">Talk:Pussycat/Let's delete Pussycat!</a>&nbsp;<span style="font-size:90%;">(<a class="external text" href="https://en.wikipedia.org/w/index.php?title=Talk:Pussycat/Let%27s_delete_Pussycat!&amp;action=edit">edit</a> | <a href="/w/index.php?title=Pussycat/Let%27s_delete_Pussycat!&amp;action=edit&amp;redlink=1" class="new" title="Pussycat/Let's delete Pussycat! (page does not exist)">article</a> | <a class="external text" href="https://en.wikipedia.org/w/index.php?title=Talk:Pussycat/Let%27s_delete_Pussycat!&amp;action=history">history</a> | <a class="external text" href="https://en.wikipedia.org/wiki/Special:Whatlinkshere/Talk:Pussycat/Let%27s_delete_Pussycat!">links</a> | <a class="external text" href="https://en.wikipedia.org/w/index.php?title=Talk:Pussycat/Let%27s_delete_Pussycat!&amp;action=watch">watch</a> | <a class="external text" href="https://en.wikipedia.org/w/index.php?title=Special:Log&amp;page=Talk%3APussycat%2FLet%27s+delete+Pussycat%21">logs</a>)</span></span>. Transcluding in this way will <i>not</i> preserve a fixed record of the discussion at the time of any archiving, so discussions should be closed in conjunction with archiving. </p>
      <p>An alternative is not to use subpages at all. You could centralise the discussion, in the normal manner, on a single talk page, and point to it on the other talk page by <a href="/wiki/Wikipedia:SUBST" class="mw-redirect" title="Wikipedia:SUBST">substituting</a> the {{<a href="/wiki/Template:Please_see" title="Template:Please see">Please see</a>}} template, as per <a href="/wiki/Wikipedia:MULTI" class="mw-redirect" title="Wikipedia:MULTI">WP:MULTI</a>. As an example, a discussion could take place at <a href="/wiki/Talk:Pussycat#Let's_delete_Pussycat!" title="Talk:Pussycat">Talk:Pussycat#Let's delete Pussycat!</a>; you could then place the wikitext <code>{{subst:Please see|Talk:Pussycat#Let's delete Pussycat!}}</code> in the <a href="/wiki/Talk:Cat" title="Talk:Cat">Talk:Cat</a> page. When the edit was saved, it would substitute it with the text: </p> 
      <pre style="white-space:-moz-pre-wrap; white-space:-pre-wrap; white-space:-o-pre-wrap; white-space:pre-wrap; word-wrap:break-word;">==Discussion at Talk:Pussycat#Let's delete Pussycat!==
[[File:Farm-Fresh eye.png|15px|link=|alt=]] You are invited to join the discussion at [[Talk:Pussycat#Let's delete Pussycat!]]. Thanks. [[User:Example|Example]] ([[User talk:Example|talk]]) 19:03, 20 October 2014 (UTC){{z48}}&lt;!--Template:Please see--&gt;</pre> 
      <h2><span class="mw-headline" id="Special_pages">Special pages</span></h2> 
      <div role="note" class="hatnote navigation-not-searchable">
       See also: <a href="/wiki/Wikipedia:Special_pages#Miscellaneous" class="mw-redirect" title="Wikipedia:Special pages">Wikipedia:Special pages §&nbsp;Miscellaneous</a>
      </div> 
      <p>Some pages on <a href="/wiki/Special:SpecialPages" title="Special:SpecialPages">Special:Specialpages</a> can be transcluded, such as <a href="/wiki/Special:AllPages" title="Special:AllPages">Special:Allpages</a>, <a href="/wiki/Special:PrefixIndex" title="Special:PrefixIndex">Special:Prefixindex</a>, <a href="/wiki/Special:NewFiles" title="Special:NewFiles">Special:Newfiles</a>, <a href="/wiki/Special:NewPages" title="Special:NewPages">Special:Newpages</a>, <a href="/wiki/Special:RecentChanges" title="Special:RecentChanges">Special:Recentchanges</a> and <a href="/wiki/Special:RecentChangesLinked" title="Special:RecentChangesLinked">Special:Recentchangeslinked</a>. Samples: </p> 
      <ul>
       <li><code>{{Special:Allpages/General}}</code> – a list of pages starting at "General".</li> 
       <li><code>{{Special:Prefixindex/General}}</code> – a list of pages with prefix "General".</li> 
       <li><code>{{Special:Newfiles/4}}</code> – a gallery of the four most recently uploaded files.</li> 
       <li><code>{{Special:Newpages/5}}</code> – a list of the five most recently created pages.</li> 
       <li><code>{{Special:Recentchanges/5}}</code> – the five most recent changes.</li> 
       <li><code>{{Special:Recentchangeslinked/General}}</code> – recent changes to the pages linked from "General".</li>
      </ul> 
      <p>Attempting to transclude <code>{{Special:Categories}}</code> will not result in an actual list of categories, but <code>{{Special:Prefixindex/Category:}}</code> can be used for this purpose. </p>
      <p>Except for <code>Special:Recentchangeslinked</code>, the slash, and the word or number after the slash, can be omitted, giving a list of pages without a specific starting point, or a list of the default length. </p>
      <p><a href="/wiki/Help:URL" title="Help:URL">URL</a> parameters can be given like template parameters: </p> 
      <ul>
       <li><code>{{Special:Recentchanges|namespace=10|limit=5}}</code> – the five most recent changes in the "Template" namespace.</li> 
       <li><code>{{Special:Prefixindex/Jimbo Wales/|namespace=2|stripprefix=1}}</code> – the subpages for <a href="/wiki/User:Jimbo_Wales" title="User:Jimbo Wales">User:Jimbo Wales</a>, but without the user page prefix.</li>
      </ul> 
      <p><i>Note:</i> Transcluding certain special pages (such as <a href="/wiki/Special:NewPages" title="Special:NewPages">Special:Newpages</a>) can change the <a href="/wiki/Wikipedia:Page_name" title="Wikipedia:Page name">displayed title</a> of the page. </p>
