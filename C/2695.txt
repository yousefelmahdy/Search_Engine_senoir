      <div id="article-contents" class="article-grid-body"> 
       <p>In this guide, we're going to use the API to fetch information about repositories that we own, and the programming languages that make them up. Then, we'll visualize that information in a couple of different ways using the <a href="http://d3js.org/">D3.js</a> library. To interact with the GitHub API, we'll be using the excellent Ruby library, <a href="https://github.com/octokit/octokit.rb">Octokit</a>.</p> 
       <p>If you haven't already, you should read the <a href="/en/rest/guides/basics-of-authentication">"Basics of Authentication"</a> guide before starting this example. You can find the complete source code for this project in the <a href="https://github.com/github/platform-samples/tree/master/api/ruby/rendering-data-as-graphs">platform-samples</a> repository.</p> 
       <p>Let's jump right in!</p> 
       <h3 id="setting-up-an-oauth-application"><a href="#setting-up-an-oauth-application">Setting up an OAuth application</a></h3> 
       <p>First, <a href="https://github.com/settings/applications/new">register a new application</a> on GitHub. Set the main and callback URLs to <code>http://localhost:4567/</code>.        <h3 id="fetching-repository-information"><a href="#fetching-repository-information">Fetching repository information</a></h3> 
       <p>This time, in order to talk to the GitHub API, we're going to use the <a href="https://github.com/octokit/octokit.rb">Octokit Ruby library</a>. This is much easier than directly making a bunch of REST calls. Plus, Octokit was developed by a GitHubber, and is actively maintained, so you know it'll work.</p> 
       <p>Authentication with the API via Octokit is easy. Just pass your login and token to the <code>Octokit::Client</code> constructor:</p> 
       <pre><code class="hljs language-ruby"><span class="hljs-keyword">if</span> !authenticated?

       <p>Let's do something interesting with the data about our repositories. We're going to see the different programming languages they use, and count which ones are used most often. To do that, we'll first need a list of our repositories from the API. With Octokit, that looks like this:</p> 
       <pre><code class="hljs language-ruby">repos = client.repositories
</code></pre>

       <pre><code class="hljs language-ruby">language_obj = {}

       <h3 id="visualizing-language-counts"><a href="#visualizing-language-counts">Visualizing language counts</a></h3> 
       <p>D3.js, or just D3, is a comprehensive library for creating many kinds of charts, graphs, and interactive visualizations. Using D3 in detail is beyond the scope of this guide, but for a good introductory article, check out <a href="http://www.recursion.org/d3-for-mere-mortals/">"D3 for Mortals"</a>.</p> 
       <p>D3 is a JavaScript library, and likes working with data as arrays. So, let's convert our Ruby hash into a JSON array for use by JavaScript in the browser.</p>

       <p>As the "D3 for Mortals" guide suggests, this isn't necessarily the best use of D3. But it does serve to illustrate how you can use the library, along with Octokit, to make some really amazing things.</p> 
       <h3 id="combining-different-api-calls"><a href="#combining-different-api-calls">Combining different API calls</a></h3> 
       <p>Now it's time for a confession: the <code>language</code> attribute within repositories only identifies the "primary" language defined. That means that if you have a repository that combines several languages, the one with the most bytes of code is considered to be the primary language.</p> 
       <p>Let's combine a few API calls to get a <em>true</em> representation of which language has the greatest number of bytes written across all our code. A <a href="https://www.d3-graph-gallery.com/treemap.html">treemap</a> should be a great way to visualize the sizes of our coding languages used, rather than simply the count. We'll need to construct an array of objects that looks something like this:</p> 
       <pre><code class="hljs language-json">[ { <span class="hljs-attr">"name"</span>: <span class="hljs-string">"language1"</span>, <span class="hljs-attr">"size"</span>: <span class="hljs-number">100</span>},
  { <span class="hljs-attr">"name"</span>: <span class="hljs-string">"language2"</span>, <span class="hljs-attr">"size"</span>: <span class="hljs-number">23</span>}
  ...
]
</code></pre> 
       <p>Since we already have a list of repositories above, let's inspect each one, and call <a href="/en/rest/reference/repos#list-repository-languages">the language listing API method</a>:</p> 
       <pre><code class="hljs language-ruby">repos.each <span class="hljs-keyword">do</span> <span class="hljs-params">|repo|</span>
  repo_name = repo.name
  repo_langs = octokit_client.languages(<span class="hljs-string">"<span class="hljs-subst">#{github_user.login}</span>/<span class="hljs-subst">#{repo_name}</span>"</span>)
<span class="hljs-keyword">end</span>
</code></pre> 
       <p>From there, we'll cumulatively add each language found to a list of languages:</p> 
       <pre><code class="hljs language-ruby">repo_langs.each <span class="hljs-keyword">do</span> <span class="hljs-params">|lang, count|</span>
  <span class="hljs-keyword">if</span> !language_obj[lang]
    language_obj[lang] = count
  <span class="hljs-keyword">else</span>
    language_obj[lang] += count
  <span class="hljs-keyword">end</span>
<span class="hljs-keyword">end</span>
</code></pre> 
       <p>After that, we'll format the contents into a structure that D3 understands:</p> 
       <pre><code class="hljs language-ruby">language_obj.each <span class="hljs-keyword">do</span> <span class="hljs-params">|lang, count|</span>
  language_byte_count.push <span class="hljs-symbol">:name</span> =&gt; <span class="hljs-string">"<span class="hljs-subst">#{lang}</span> (<span class="hljs-subst">#{count}</span>)"</span>, <span class="hljs-symbol">:count</span> =&gt; count

<span class="hljs-comment"># some mandatory formatting for D3</span>

       <p>(For more information on D3 tree map magic, check out <a href="/en/rest/reference/repos#list-repository-languages">this simple tutorial</a>.)</p> 
       <p>To wrap up, we pass this JSON information over to the same ERB template:</p> 
       <pre><code class="hljs language-ruby">erb <span class="hljs-symbol">:lang_freq</span>, <span class="hljs-symbol">:locals</span> =&gt; { <span class="hljs-symbol">:languages</span> =&gt; languages.to_json, <span class="hljs-symbol">:language_byte_count</span> =&gt; language_bytes.to_json}
