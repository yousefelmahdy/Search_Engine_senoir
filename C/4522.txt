      <p><a href="/wiki/Help:Template" title="Help:Template">Templates</a> are a very powerful feature of <a href="/wiki/MediaWiki" title="MediaWiki">MediaWiki</a>, but can be confusing to new users and even experienced users can have difficulty making sense of the more complex ones. Templates should therefore be accompanied by <b><a href="/wiki/Documentation" title="Documentation">documentation</a></b> to improve usability. </p>
      <p>Template documentation should explain what a template does and how to use it. It should be simple enough that a user without complete knowledge of the intricacies of template syntax—which includes many experienced contributors who focus their attention elsewhere—can use it correctly. This is especially true in the case of very widely used templates. </p>
      <p>Editors should defer to official <a href="/wiki/Wikipedia:POLICIES" class="mw-redirect" title="Wikipedia:POLICIES">policies or guidelines</a> when template documentation pages are inconsistent with established community standards and principles. Editors should also avoid "quoting" template documentation pages as though they are policy—including this <a href="/wiki/Wikipedia:INFOPAGES" class="mw-redirect" title="Wikipedia:INFOPAGES">how-to guide</a>. Template documentation pages can be written without much—if any—debate, as opposed to Wikipedia policies that have been <a href="/wiki/Wikipedia:PROPOSAL" class="mw-redirect" title="Wikipedia:PROPOSAL">thoroughly vetted by the community</a> (see <a href="/wiki/Wikipedia:Local_consensus" class="mw-redirect" title="Wikipedia:Local consensus">WP:Local consensus</a> for details). </p>

      <h2><span class="mw-headline" id="What_to_include">What to include</span></h2> 
      <p>Template documentation should cover: </p> 
      <ul>
       <li>The <b>basic purpose</b> of the template: what it does and if it is not immediately obvious, why it needs to be done. If there are other templates with similar names or purposes, it's a good idea to mention those, in order to reduce the chance of the wrong one being used. Include important <b>limitations</b>, such as the lack of {{<a href="/wiki/Template:Navbox_visibility" title="Template:Navbox visibility">Navbox visibility</a>}} on the Mobile web site, if a template should only be used for certain countries or time periods, or other things that editors need to know before choosing to use the template.</li> 
       <li>The <b><a href="/wiki/Help:Template#Parameters" title="Help:Template">parameters</a></b> of the template: whether they are numbered, named or optional, and if so, what the default values are and what effect they have. If a parameter can take only a limited set of values or is constrained in any way, for example, if it can only use "yes", "no", or a number, this should be clearly explained.</li> 
       <li><b>Usage examples</b>: specify the exact wikitext that should be used and the result that it produces. The wikitext can be enclosed in a <code class="nowrap" style="">&lt;code&gt;...&lt;/code&gt;</code> container, to make it clear and <code>easy to copy, like this</code>. If the template can be used in several different ways, with or without optional parameters, for example, provide a range of examples. A good way to do so is to transclude the template itself into the documentation a few times (i.e., use live examples), with different parameters each time and list the parameters used in each case. With <a href="/wiki/Template:Xpd" title="Template:Xpd">Template:Xpd</a> this can be done without putting the template call twice in the wikitext.</li> 
       <li><b>TemplateData</b> tags: See <a href="/wiki/Wikipedia:TemplateData/Tutorial" title="Wikipedia:TemplateData/Tutorial">Wikipedia:TemplateData/Tutorial</a>.</li> 
       <li><b>Related templates</b>: if the template is one of a series of templates, include links to these&nbsp;– in particular, ensure that every template in the series is linked from each of the others, as this makes navigation easier. (A separate navigation template may be useful for this purpose, see: <a href="/wiki/Template:Protection_templates" title="Template:Protection templates">Template:Protection templates</a>).</li> 
       <li><b><a href="/wiki/Wikipedia:Categorization" title="Wikipedia:Categorization">Categories</a></b> where applicable (<a href="/wiki/Help:Interwikimedia_links" class="mw-redirect" title="Help:Interwikimedia links">InterWikimedia links</a> should be listed at <a href="https://www.wikidata.org/wiki/" class="extiw" title="d:">Wikidata</a> – more information at <a href="/wiki/Wikipedia:Wikidata" title="Wikipedia:Wikidata">Wikipedia:Wikidata</a>). Like the documentation, categories must be listed within a <code class="nowrap" style="">&lt;noinclude&gt;...&lt;/noinclude&gt;</code> container on a template, or within <code class="nowrap" style="">&lt;includeonly&gt;{{Sandbox other||...}}&lt;/includeonly&gt;</code> tags if placed on a documentation page. Many template categories are available, see: <a href="/wiki/Category:Wikipedia_templates" title="Category:Wikipedia templates">Category:Wikipedia templates</a> to browse through them.</li>
      </ul> 
      <p>The English Wikipedia is a source of templates for hundreds of other Wikipedias and sister projects. Often, templates are fully self-contained, so the process easy: the contents are simply copied to a new template page at the other wiki, and everything works. However, on more complex templates, the template may invoke a module, transclude other templates, only work if paired with a separate template, or need particular CSS or Javascript code to work. In these cases, it is helpful to include a brief list of templates or other code that this one requires, at the end of the documentation. </p> 
      <h2><span class="mw-headline" id="Where_to_place_it">Where to place it</span></h2> 
      <p>A template's page in the <a href="/wiki/Wikipedia:Template_namespace" title="Wikipedia:Template namespace">Template namespace</a> is the location for the template code that controls the look and behavior of that template. What usually appears underneath the title on the <a href="/wiki/Rendered" class="mw-redirect mw-disambig" title="Rendered">rendered</a> Template: page (as opposed to the <a href="/wiki/Wikipedia:Edit_window" class="mw-redirect" title="Wikipedia:Edit window">edit window on the Edit tab</a> or, in the case of templates whose code is protected, the <a href="/wiki/Wikipedia:View_source" class="mw-redirect" title="Wikipedia:View source">View source tab</a>, is the rendered template itself, followed by a separate section to display the template's rendered documentation, followed by the categories to which the template belongs. </p>
      <p>However, the editable <a href="/wiki/Wikipedia:Wikicode" class="mw-redirect" title="Wikipedia:Wikicode">wikicode</a> for the template's documentation is often placed on a separate <a href="/wiki/Wikipedia:Subpages" title="Wikipedia:Subpages">subpage</a> of the template itself, which is then <a href="/wiki/Wikipedia:Transclusion" class="mw-redirect" title="Wikipedia:Transclusion">transcluded</a> at the end of the template page. This separates the often complex template code from the documentation, making the documentation easier to edit and reducing the number of accidental editing errors in the template code. It also allows templates to be <a href="/wiki/Wikipedia:Protection_policy" title="Wikipedia:Protection policy">protected</a> where necessary, limiting editing access to important templates' code while allowing anyone to edit those templates' documentation. This method is sometimes referred to as the "template-doc page pattern". </p>
      <p>Documentation of any sort on a template page (including <a href="/wiki/Wikipedia:TemplateData" title="Wikipedia:TemplateData">TemplateData</a>) should always be enclosed by a <code class="nowrap" style="">&lt;noinclude&gt;...&lt;/noinclude&gt;</code> container, so that it does not show up when the template is used on another page. </p> 
      <h2><span class="mw-headline" id="How_to_create_a_documentation_subpage">How to create a documentation subpage</span></h2> 
      <h3><span class="mw-headline" id="Put_documentation_in_the_template">Put documentation in the template</span></h3> 
      <p>You must put {{<a href="/wiki/Template:Documentation" title="Template:Documentation">documentation</a>}} when creating (publishing) a template. Then a documentation page is created with <code>[view] [edit] [history] [purge]</code> links. You can create and edit the template documentation clicking in this pane <code>[edit]</code> link. </p>
      <p>Template documentation subpages using {{<a href="/wiki/Template:Documentation" title="Template:Documentation">documentation</a>}} are named and formatted using the following general pattern, for consistency. </p>
      <p>Suppose your template is named <code>Template:X</code>. Edit the template and append the following at the end of the template code, or use {{subst:<a href="/wiki/Template:Doc-code" title="Template:Doc-code">doc-code</a>}}: </p> 
      <pre><span style="background-color: #CCC; color:black;"><var style="padding-right: 1px;">[--last line of your template code--]</var></span>&lt;noinclude&gt;
{{Documentation}}
&lt;!-- Add categories to the /doc subpage and interwikis in Wikidata, not here! --&gt;
&lt;/noinclude&gt;
</pre> 
      <p>This will transclude {{<a href="/wiki/Template:Documentation" title="Template:Documentation">documentation</a>}} at the bottom of the template page. </p>
      <p><b>Important</b>: Make sure the opening <code class="nowrap" style="">&lt;noinclude&gt;</code> begins immediately after the last character of the template code or text and <i>not</i> on a new line, nor with any intervening spaces. Otherwise, extra space will be inserted below the template when it is used, which is usually not wanted. </p>
      <p>If the template is already protected, ask an <a href="/wiki/Wikipedia:Administrators" title="Wikipedia:Administrators">administrator</a> to do this or request an edit by using an <code>{{<a href="/wiki/Template:Edit_protected" class="mw-redirect" title="Template:Edit protected">edit protected</a>}}</code> on the template's talk page. If documentation and categories already exist in a section, enclosed within a <code class="nowrap" style="">&lt;noinclude&gt;...&lt;/noinclude&gt;</code> container, move them into the documentation subpage (where they should be enclosed in <code class="nowrap" style="">&lt;includeonly&gt;...&lt;/includeonly&gt;</code>), as it is best not to have documentation split across two separate pages. </p> 
      <h3><span class="mw-headline" id="Automatic_creation">Automatic creation</span></h3> 
      <p>Use <code>[create]</code> link at the bottom of the empty documentation box to automatically create a <a href="/wiki/Wikipedia:Preload" title="Wikipedia:Preload">preloaded</a> documentation subpage. Insert the documentation after the top line and categories under the appropriate comment line&nbsp;– leaving the comment in place, so that the layout is preserved when the page is edited in future. Related templates, policy page, projects, etc. can be linked to by adding a "See also" section. Save the subpage. </p> 
      <h3><span class="mw-headline" id="Manual_creation">Manual creation</span></h3> 
      <p>To create the documentation subpage manually, create a subpage with the name <code>Template:X/doc</code>. See the details at {{<a href="/wiki/Template:Documentation_subpage" title="Template:Documentation subpage">Documentation subpage</a>}} or start the page by copy-pasting the following standard wikitext: </p> 
      <div style="overflow:auto;" class="mw-highlight mw-highlight-lang-xml+smarty mw-content-ltr" dir="ltr">
       <pre><span></span><span class="cp">{{</span><span class="na">Documentation</span> <span class="na">subpage</span><span class="cp">}}</span>
<span class="c">&lt;!-- Add categories where indicated at the bottom of this page and interwikis at Wikidata --&gt;</span>
== Usage ==


<span class="nt">&lt;includeonly&gt;</span><span class="cp">{{</span><span class="na">sandbox</span> <span class="na">other</span><span class="o">||</span>
<span class="o">&lt;!--</span> <span class="na">Categories</span> <span class="na">below</span> <span class="na">this</span> <span class="na">line</span><span class="o">;</span> <span class="na">interwikis</span> <span class="na">at</span> <span class="na">Wikidata</span> <span class="o">--&gt;</span>

<span class="cp">}}</span><span class="nt">&lt;/includeonly&gt;</span>
</pre>
      </div> 
      <p>The top line will display a message explaining the current page and a link to the template page. Save the subpage and follow instructions in section "<a href="#Use_Template:Documentation">Use Template:Documentation</a>". </p> 
      <h3><span class="mw-headline" id="With_TemplateData">With TemplateData</span></h3> 
      <p>Instead of manually writing a lead graf and a usage table, the {{<a href="/wiki/Template:Format_TemplateData" title="Template:Format TemplateData">Format TemplateData</a>}} template can do most of the job. Simply write your TemplateData in the table interface, and then wrap it into a template call like <code>{{Format TemplateData|1=&lt;templatedata&gt;...&lt;/templatedata&gt;}}</code> at the top of the page. </p> 
      <h3><span class="mw-headline" id="Notes">Notes</span></h3> 
      <p>You may wish to redirect the talk page of the /doc subpage to the talk page of the template itself. Then all talk relating to the template and its documentation will end up on the same talkpage. For example, redirect <code>Template talk:X/doc</code> to <code>Template talk:X</code>. </p>
      <p>A documentation page can also be redirected to the /doc subpage of another template, if that covers the usage for both templates. In this case, clicking the links to view or edit the documentation will directly open the target of the redirect. If it is necessary to access the redirect itself (e.g. to remove the redirect and create a separate doc page), go to the template URL by clicking in the location bar at the top of your browser, and add /doc at the end. </p> 
      <h2><span class="mw-headline" id="Blank">Blank</span></h2> 
      <p>To generate a blank template, which may then be copied from the documentation and pasted into another page, use: </p> 
      <dl>
       <dd>
        <code>{{subst:#tag:pre|{{subst:Parameters|code|base={{subst:BASEPAGENAME}}}}|style=overflow: auto;}}</code>
       </dd>
      </dl> 
      <h2><span class="mw-headline" id="Examples">Examples</span></h2> 
      <p>To generate an instance of the template, populated with its own property names, use: </p> 
      <dl>
       <dd>
        <code>{{subst:Parameters|demo|base={{subst:BASEPAGENAME}}|_base=}}</code>
       </dd>
      </dl> 
      <h2><span class="mw-headline" id="Categories_and_interwiki_links">Categories and interwiki links</span></h2> 
      <ul>
       <li><b>To place the template itself into a category</b>, add the <code>[[Category:Category name]]</code> code inside an <code class="nowrap" style="">&lt;includeonly&gt;...&lt;/includeonly&gt;</code> section on the <i>doc</i> subpage.</li> 
       <li><b>To create an interwiki link for the template itself</b>, go to <a href="/wiki/Wikipedia:Wikidata" title="Wikipedia:Wikidata">Wikidata</a> and follow the instructions for adding links to pages.</li> 
       <li><b>To place the <i>doc</i> subpage into a category</b>, add the <code>[[Category:Category name]]</code> code inside a <code class="nowrap" style="">&lt;noinclude&gt;...&lt;/noinclude&gt;</code> section on the <i>doc</i> subpage.</li> 
       <li><b>To make the template place an article into a category</b> (when the article includes the template), add the <code>[[Category:Category name]]</code> code inside an <code class="nowrap" style="">&lt;includeonly&gt;...&lt;/includeonly&gt;</code> section on the template page. Exact placement within the template code may affect how the category code is executed.</li>
      </ul> 
      <h2><span id=".2Fsandbox_and_.2Ftestcases"></span><span class="mw-headline" id="/sandbox_and_/testcases">/sandbox and /testcases</span></h2> 
      <p>Before doing changes to a template it can be good to first copy the template code to a sandbox and run some testcases, since the template might be visible on thousands or even millions of pages. If you create subpages named exactly "<b>/sandbox</b>" and "<b>/testcases</b>" to a template then the green <span class="nowrap"><code>{{documentation}}</code></span> box on the template auto-detects this and will show links to those pages in its header. See <a href="/wiki/Wikipedia:Template_sandbox_and_test_cases" title="Wikipedia:Template sandbox and test cases">Wikipedia:Template sandbox and test cases</a> for more information. </p> 
      <h2><span id="Several_templates.2C_one_documentation_page"></span><span class="mw-headline" id="Several_templates,_one_documentation_page">Several templates, one documentation page</span></h2> 
      <p>When several templates work together or are very similar then it is often clearer and easier to maintain one single documentation page that documents them together. The simplest way to do this is to make a full documentation page at one of the templates, and then make "soft redirects" from the other templates. See, for instance: {{<a href="/wiki/Template:Wrap" title="Template:Wrap">wrap</a>}}. </p> 
      <h2><span class="mw-headline" id="Directly_on_a_template_page">Directly on a template page</span></h2> 
      <p>When a documentation subpage has not been created, and the {{<a href="/wiki/Template:Documentation" title="Template:Documentation">documentation</a>}} template is being used with a <code class="nowrap" style="">|content=</code> parameter on the actual template page, then to place the template itself into a category, add <code>[[Category:Category name]]</code> inside the documentation template, after the content. For example, for placement on the actual template page: </p> 
      <div style="overflow:auto;" class="mw-highlight mw-highlight-lang-xml+smarty mw-content-ltr" dir="ltr">
       <pre><span></span><span class="c">&lt;!--Last line of your template code--&gt;</span><span class="nt">&lt;noinclude&gt;</span>
<span class="cp">{{</span><span class="na">Documentation</span>
 <span class="o">|</span> <span class="na">content</span> <span class="o">=</span>
<span class="o">&lt;!--</span> <span class="na">template</span> <span class="na">documentation</span> <span class="o">--&gt;</span>

<span class="o">[[</span><span class="na">Category</span><span class="o">:</span><span class="na">Category</span> <span class="na">name</span><span class="o">]]</span>
<span class="o">[[</span><span class="na">Category</span><span class="o">:</span><span class="na">Category</span> <span class="na">name2</span><span class="o">]]</span>
<span class="cp">}}</span><span class="nt">&lt;/noinclude&gt;</span>
</pre>
      </div> 
      <h2><span class="mw-headline" id="When_no_documentation_is_needed"><span class="anchor" id="When_no_documentation_needed"></span> When no documentation is needed</span></h2> 
      <p>When a template as displayed will link to a page that can serve as the documentation, then separate documentation is superfluous and does not need to be created. For instance, a stub template, using the {{<a href="/wiki/Template:Asbox" title="Template:Asbox">asbox</a>}} template as a base, will already display pre-loaded common documentation for all stub templates using that template, and will not need additional documentation. </p> 
      <h2><span class="mw-headline" id="Tips_and_tricks">Tips and tricks</span></h2> 
      <p>Here are some tips to facilitate writing documentations: </p> 
      <ul>
       <li>Links to templates like <span class="nowrap">{{<a href="/wiki/Template:Japanese_year" title="Template:Japanese year">Japanese year</a>|1800}}</span> can be inserted by writing <span class="nowrap">{{<a href="/wiki/Template:Tlp" class="mw-redirect" title="Template:Tlp">tlp</a>|Japanese year|1800}}</span>. See {{<a href="/wiki/Template:Tlp" class="mw-redirect" title="Template:Tlp">tlp</a>}} for similar templates.</li> 
       <li>HTML tags like <code class="nowrap" style="">&lt;ref group="note"&gt;...&lt;/ref&gt;</code> can be easily inserted with {{<a href="/wiki/Template:Tag" title="Template:Tag">tag</a>}}. This example is written as <span class="nowrap">{{<a href="/wiki/Template:Tag" title="Template:Tag">tag</a>|ref|params=group="note"}}</span></li> 
       <li>Equals sign <b>=</b> can be inserted with <a href="/wiki/Template:%3D" title="Template:=">{{=}}</a>. For example, <span class="nowrap">{{<a href="/wiki/Template:Citation_needed" title="Template:Citation needed">Citation needed</a>|date=1900-02-29}}</span> is made by <span class="nowrap">{{<a href="/wiki/Template:Tlp" class="mw-redirect" title="Template:Tlp">tlp</a>|Citation needed|date{{=}}1900-02-29}}</span>. Avoid the equivalent notation &amp;#61; which is not easy to read for other editors.</li> 
       <li>Don't forget the single <code class="nowrap" style="">&lt;nowiki /&gt;</code> tag. [[<code class="nowrap" style="">&lt;nowiki /&gt;</code>example]] becomes [[example]], and [<code class="nowrap" style="">&lt;nowiki /&gt;</code><a class="external free" href="https://en.wikipedia.org">http://en.wikipedia.org</a> no link] is [<a class="external free" href="https://en.wikipedia.org">http://en.wikipedia.org</a> no link]. See <a href="/wiki/Wikipedia:NOWIKI" class="mw-redirect" title="Wikipedia:NOWIKI">WP:NOWIKI</a> for the details.</li> 
       <li>For a link to a category, use {{<a href="/wiki/Template:Cat" title="Template:Cat">cat</a>}}. For example, <span class="nowrap">{{<a href="/wiki/Template:Cat" title="Template:Cat">cat</a>|Templates with incorrect parameter syntax}}</span> becomes <a href="/wiki/Category:Templates_with_incorrect_parameter_syntax" title="Category:Templates with incorrect parameter syntax">Category:Templates with incorrect parameter syntax</a>.</li> 
       <li>To suppress categorization of the /sandbox and /testcases pages of the template (if they exist), use this code at the bottom of the documentation subpage:</li>
      </ul> 
      <div style="overflow:auto;" class="mw-highlight mw-highlight-lang-xml+smarty mw-content-ltr" dir="ltr">
       <pre><span></span><span class="nt">&lt;includeonly&gt;</span><span class="cp">{{</span><span class="na">sandbox</span> <span class="na">other</span><span class="o">||</span><span class="cp">{{</span><span class="na">testcases</span> <span class="na">other</span><span class="o">||</span>
<span class="o">&lt;!--</span> <span class="na">Categories</span> <span class="na">below</span> <span class="na">this</span> <span class="na">line</span><span class="o">,</span> <span class="na">please</span><span class="o">;</span> <span class="na">interwikis</span> <span class="na">at</span> <span class="na">Wikidata</span> <span class="o">--&gt;</span>

<span class="cp">}}}}</span><span class="nt">&lt;/includeonly&gt;</span>
</pre>
      </div> 
      <ul>
       <li>When the template is a <i>child infobox</i>, documentation appears broken. Prevent this by adding the &lt;includeonly&gt; tag:</li>
      </ul> 
      <pre>{{Some infobox (child) template
|  child = &lt;includeonly&gt;yes&lt;/includeonly&gt;
| label1 = Hello
|  data1 = World
}}&lt;noinclude&gt;{{documentation}}&lt;/noinclude&gt;
</pre>
