      <p>The <b>spam blacklist</b> is a control mechanism that prevents an external link from being added to an <a href="/wiki/English_Wikipedia" title="English Wikipedia">English Wikipedia</a> page when the URL matches one listed at <a href="/wiki/MediaWiki:Spam-blacklist" title="MediaWiki:Spam-blacklist">MediaWiki:Spam-blacklist</a>. It mostly lists <a href="/wiki/Wikipedia:Spam" title="Wikipedia:Spam">spammers</a>, but also includes <a href="/wiki/URL_redirection" title="URL redirection">URL redirection</a> services (which could otherwise be used to bypass blacklisting). </p>
      <p>The spam blacklist is supported by MediaWiki's <a href="https://www.mediawiki.org/wiki/Extension:SpamBlacklist" class="extiw" title="mw:Extension:SpamBlacklist">SpamBlacklist extension</a>. </p>

      <h2><span class="mw-headline" id="Introduction">Introduction</span></h2> 
      <p>MediaWiki:Spam-blacklist is a localized version of <a href="https://meta.wikimedia.org/wiki/Spam_blacklist" class="extiw" title="m:Spam blacklist">m:Spam blacklist</a>, a similar blacklist that affects all <a href="/wiki/Wikimedia_Foundation" title="Wikimedia Foundation">Wikimedia Foundation</a> projects. Additions to the local spam blacklist will only affect the English-language edition of Wikipedia, not other projects. Blacklisting requests may be made at <a href="/wiki/MediaWiki_talk:Spam-blacklist" title="MediaWiki talk:Spam-blacklist">MediaWiki talk:Spam-blacklist</a>. However, blacklisting a URL should be used as a last resort against spammers. You should consider the following before requesting that a URL be blacklisted: </p> 
      <ul>
       <li>Can <a href="/wiki/Wikipedia:Protection_policy" title="Wikipedia:Protection policy">protection</a> solve the problem? If so, please make a request at <a href="/wiki/Wikipedia:Requests_for_page_protection" title="Wikipedia:Requests for page protection">Wikipedia:Requests for page protection</a>.</li> 
       <li>Will <a href="/wiki/Wikipedia:Blocking_policy" title="Wikipedia:Blocking policy">blocking</a> a single user solve the problem? If you have given appropriate <a href="/wiki/Wikipedia:Template_messages/User_talk_namespace" class="mw-redirect" title="Wikipedia:Template messages/User talk namespace">warnings</a> to a spammer, you should report them on <a href="/wiki/Wikipedia:Administrator_intervention_against_vandalism" title="Wikipedia:Administrator intervention against vandalism">Wikipedia:Administrator intervention against vandalism</a>, where they can be blocked by an <a href="/wiki/Wikipedia:Administrator" class="mw-redirect" title="Wikipedia:Administrator">administrator</a>. <a href="/wiki/Wikipedia:No_open_proxies" class="mw-redirect" title="Wikipedia:No open proxies">Open proxies</a> used to spam should be reported to <a href="/wiki/Wikipedia:Administrator_intervention_against_vandalism" title="Wikipedia:Administrator intervention against vandalism">Wikipedia:Administrator intervention against vandalism</a> or <a href="/wiki/Wikipedia:WikiProject_on_open_proxies" title="Wikipedia:WikiProject on open proxies">Wikipedia:WikiProject on open proxies</a> so that they can be blocked.</li> 
       <li>Will blocking a small number of users for a short time to allow conversation help?</li> 
       <li>Can the problem be controlled by other means such as <a href="/wiki/User:XLinkBot" title="User:XLinkBot">User:XLinkBot</a>?</li>
      </ul> 
      <p>Note that the blacklist only affects live links. Most non-clickable URLs (such as those contained within a <code>nowiki</code> tag) are not affected. However, URLs in <a href="/wiki/Wikipedia:ES" class="mw-redirect" title="Wikipedia:ES">edit summaries</a> are affected. </p> 
      <h2><span class="mw-headline" id="Requests_for_listing">Requests for listing</span></h2> 
      <p>Compelling evidence of a problem should be given when requesting a page to be listed (preferably by giving <a href="/wiki/Help:Diff" title="Help:Diff">diffs</a>). </p> 
      <ul>
       <li><a href="/wiki/MediaWiki_talk:Spam-blacklist#Proposed_additions" title="MediaWiki talk:Spam-blacklist">Proposed additions</a> – Use this section to request additions.</li> 
       <li><a href="/wiki/MediaWiki_talk:Spam-blacklist#Troubleshooting_and_problems" title="MediaWiki talk:Spam-blacklist">Troubleshooting and problems</a> – Use this section to report problems with the blacklist. <b>This is not the section to request links to be removed!</b></li> 
       <li><a href="/wiki/MediaWiki_talk:Spam-blacklist#Discussion" title="MediaWiki talk:Spam-blacklist">Discussion</a> – Use this section for any other discussion involving the blacklist.</li>
      </ul> 
      <h2><span class="mw-headline" id="Requests_for_delisting">Requests for delisting</span></h2> 
      <p>When requesting that a URL be delisted, you should give compelling evidence as to why it should be delisted. Cases of this could include (but are not limited to) hijacked domains being returned to their rightful owner or a site with content that violates copyrights cleaning up their act. </p>
      <p>Relevant links: </p> 
      <ul>
       <li><a href="/wiki/MediaWiki_talk:Spam-blacklist#Proposed_removals" title="MediaWiki talk:Spam-blacklist">Proposed removals</a> – Use this section to request that links be <b>removed</b> from the spam blacklist.</li> 
       <li><a href="/wiki/MediaWiki_talk:Spam-whitelist" title="MediaWiki talk:Spam-whitelist">MediaWiki talk:Spam-whitelist</a> – Use this page to request that specific pages be whitelisted, while keeping the whole domain blocked.</li>
      </ul>
