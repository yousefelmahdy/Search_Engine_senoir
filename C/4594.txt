        <p>This template is similar to {{<a href="/wiki/Template:WikiProject_banner_shell" title="Template:WikiProject banner shell">WikiProject banner shell</a>}} in that it may be used to add one or more appropriate redirect category (<a href="/wiki/Wikipedia:RCAT" class="mw-redirect" title="Wikipedia:RCAT">rcat</a>) templates, along with their parameters, to redirects. Additionally, this template <i>automatically</i> senses, describes, categorizes and changes <a href="/wiki/Wikipedia:PP" class="mw-redirect" title="Wikipedia:PP">protection levels</a> when appropriate. A third important function is to help editors learn more about redirect categorization by use of the <a href="/wiki/Category:Miscellaneous_redirects" title="Category:Miscellaneous redirects">manifold sort</a>. For more information see the page below and the <a href="/wiki/Wikipedia:TMR" class="mw-redirect" title="Wikipedia:TMR">main rcat <b>index</b></a>. </p>

        <h2><span class="mw-headline" id="Purpose">Purpose</span></h2> 
        <p>As with the <i>This is a redirect</i> template, this template is based on a <a href="/wiki/Wikipedia_talk:WikiProject_Redirect/Archive_1#New_meta-template(s)_idea" title="Wikipedia talk:WikiProject Redirect/Archive 1">proposal</a> to standardize <a href="/wiki/Wikipedia:Redirect_templates" class="mw-redirect" title="Wikipedia:Redirect templates">redirect templates</a> (rcats). Its basic purpose is to simplify the process of tagging and categorizing redirects. </p>
        <p>Like the <i>This is a redirect</i> template, this shell template is also able to help newer contributors learn the redirect category system. Details of this part of the purpose of this shell template may be found in the <a href="#When_used_alone">When used alone</a> section below. </p>
        <p>Protection levels are automatically detected by this template by use of {{<a href="/wiki/Template:Pp-protected" class="mw-redirect" title="Template:Pp-protected">pp-protected</a>}} and {{<a href="/wiki/Template:Pp-move" title="Template:Pp-move">pp-move</a>}}. When edit protection or move protection (or both) is applied to a redirect that has been tagged with this template, the protection level is automatically detected, and the redirect will be tagged with a protection rcat and sorted to the appropriate protection-related category(ies). When protection is altered or removed, then the protection-related rcats and category(ies) will also be (automatically) changed or removed from the redirect. </p>
        <p>This template can categorize and display the text (with parameters) of any number of rcats that are needed (usually from one to seven).<br> <sup id="cite_ref-mbox_1-0" class="reference"><a href="#cite_note-mbox-1">[Mbox note 1]</a></sup> <sup id="cite_ref-printworthiness_2-0" class="reference"><a href="#cite_note-printworthiness-2">[Printworthiness note 1]</a></sup> </p> 
        <h2><span class="mw-headline" id="Usage">Usage</span></h2> 
        <p>Used by itself with no parameters, for example, <span class="nowrap"><code>{{Redirect category shell}}</code></span> (and its alias <span class="nowrap"><code>{{Rcat shell}}</code></span>), the template will appear as it does at the top of this page, and the redirect will populate <a href="/wiki/Category:Miscellaneous_redirects" title="Category:Miscellaneous redirects">Category:Miscellaneous redirects</a>. That category will also be populated if the first parameter is left empty while rcats occupy the second parameter. This is useful when an editor adds rcats and yet feels unsure that there may be at least one other rcat with category that may be used to tag the redirect. Monitors of the <i>Miscellaneous redirects</i> category will then help decide if another rcat(s) is needed. Then the empty first parameter may be excluded by removing either the first or second pipe. That will remove the redirect from <i>Category:Miscellaneous redirects</i>. <a href="#When_used_alone">See below</a> for more information. </p>
        <p>This template may hold any number of different redirect category (<a href="/wiki/Wikipedia:RCAT" class="mw-redirect" title="Wikipedia:RCAT">rcat</a>) templates, usually from one to seven. The choices for editors are listed at <a href="/wiki/Wikipedia:Template_index/Redirect_pages" title="Wikipedia:Template index/Redirect pages">Wikipedia:Template index/Redirect pages</a> both alphabetically and by function. Any of those rcats may be used either with their full template name, e.g., <b>R from alternative language</b>, or with an alias (shortcut redirect), which may be found in each rcat's <a href="/wiki/Wikipedia:TDOC" class="mw-redirect" title="Wikipedia:TDOC">documentation</a>. </p>
        <p>An unnamed parameter, <code class="nowrap" style="">|1=</code>, has been introduced in this template and works as follows: </p> 
        <pre>{{Redirect category shell|1=
{{rcat 1}}
{{rcat 2}}
{{rcat 3}}
{{rcat N}}
}}
</pre> 
        <dl>
         <dd>
          <i>or</i> the <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">1=</code> may be omitted:
         </dd>
        </dl> 
        <pre>{{Redirect category shell|
{{rcat 1}}
{{rcat 2}}
{{rcat 3}}
{{rcat N}}
}}
</pre> 
        <p>As with all templates that have unnamed (numbered) parameters, the numbers and equal signs may be omitted. </p>
        <p>"N" represents the highest number of rcats that can be used in this template, and at this point the only limit would be any wikimarkup ceiling that may exist. Usually, from one to seven rcats suffice; however, more rcats can be used if required. </p>
        <p>The actual sequence of rcats is left to editors – there is no priority as to which rcat should be "first" and which rcats should "follow" in any particular order. </p>
        <p>A second unnamed parameter, <code class="nowrap" style="">|2=</code>, has also been introduced. This gives this template the same "help" functionality that is used in the <i>This is a redirect</i> template. If the first parameter is left "empty", then <a href="/wiki/Category:Miscellaneous_redirects" title="Category:Miscellaneous redirects">Category:Miscellaneous redirects</a> will be populated: </p> 
        <pre>{{Redirect category shell|1=|2=
{{rcat 1}}
{{rcat 2}}
{{rcat N}}
}}
</pre> 
        <dl>
         <dd>
          <i>or</i> if a second pipe is placed directly after the first pipe, then that will accomplish the same manifold sort:
         </dd>
        </dl> 
        <pre>{{Redirect category shell||
{{rcat 1}}
{{rcat 2}}
{{rcat N}}
}}
</pre> 
        <p>Note the double pipe after the word "shell" above, which indicates an empty first parameter. Editors who monitor the <i>Miscellaneous redirects</i> category will come to the redirect and check, add or remove rcats as needed. </p>
        <p>Also, when used by itself this template will appear as it does at the top of this page and populate the <i>Miscellaneous redirects</i> category: </p> 
        <pre>{{Redirect category shell}}
</pre> 
        <p><span class="anchor" id="goodread"></span> This template may be added to a redirect page in the following manner: </p> 
        <pre>#REDIRECT [[(target page name)]]

{{Redirect category shell||
{{R from short name}}
{{R unprintworthy}}
}}
</pre> 
        <dl>
         <dd>
          It can be noted here that the rcats can also go on the same line as the <i>Redirect category shell</i> template:
         </dd>
        </dl> 
        <pre>#REDIRECT [[(target page name)]]

{{Redirect category shell||{{R from short name}}{{R unprintworthy}}}}
</pre> 
        <dl>
         <dd>
          Again, note the double pipe after "shell" above, which indicates an empty first parameter. Leave the first parameter empty to sort a redirect to <a href="/wiki/Category:Miscellaneous_redirects" title="Category:Miscellaneous redirects">Category:Miscellaneous redirects</a> <i>in addition to</i> the categories of the rcats that are used. In this case the <a href="/wiki/Category:Redirects_from_short_names" title="Category:Redirects from short names">Redirects from short names</a> and <a href="/wiki/Category:Unprintworthy_redirects" title="Category:Unprintworthy redirects">Unprintworthy redirects</a> categories will also be populated. Monitors of the "Miscellaneous redirects" category will help editors when needed.
         </dd>
        </dl> 
        <p>Generally for good readability on the edit screen, this template is placed beginning on the third line of that screen as follows: </p> 
        <pre>Line 1: #REDIRECT [[(target page name)]]
</pre> 
        <pre>Line 2: <span style="color: red;"><small>(please leave this line blank for emphasis and ease of reading by editors)</small></span>
</pre> 
        <pre>Line 3: {{Redirect category shell|{{R from move}}{{R to section}}{{R printworthy}}}}
</pre> 
        <dl>
         <dd>
          <i>See also: section titled <b><a href="#Mbox-based">Mbox note</a></b> below.</i>
         </dd>
        </dl>

        <h4><span class="mw-headline" id="Rcat_parameters">Rcat parameters</span></h4> 
        <p>In addition to this template's ability to act as a shell template for rcats, it accepts all of the parameters that rcats accept. Rcat parameters, when needed, will work in this template as follows: </p> 
        <pre>{{Redirect category shell|
{{R from plural|printworthy}}
}}
</pre> 
        <p>This subdues the <a href="/wiki/Category:Unprintworthy_redirects" title="Category:Unprintworthy redirects">Unprintworthy redirects</a> category to which {{<a href="/wiki/Template:R_from_plural" title="Template:R from plural">R from plural</a>}} defaults and populates the <a href="/wiki/Category:Printworthy_redirects" title="Category:Printworthy redirects">Printworthy redirects</a> category instead. The position an rcat holds will have no effect on the parameters of other rcats that are used to tag a redirect: </p> 
        <pre style="font-size:95%;overflow:auto;">{{Redirect category shell|
{{R from move}}
{{R from modification}}
{{R from alternative name}}
{{R from alternative language|en|es}}
{{R for convenience}}
{{R from plural|printworthy}}
{{R printworthy}}
}}
</pre> 
        <p>In the above example, {{<a href="/wiki/Template:R_from_plural" title="Template:R from plural">R from plural</a>}} is the sixth rcat, and its first parameter is used to change from "unprintworthy" to "printworthy". {{<a href="/wiki/Template:R_from_alternative_language" title="Template:R from alternative language">R from alternative language</a>}}) holds the 4th position, and its first parameter is used to house the <a href="/wiki/Wikipedia:Template_messages/Redirect_language_codes" class="mw-redirect" title="Wikipedia:Template messages/Redirect language codes">ISO 639 language code</a> of the source language, in this case <b><code><span class="nowrap">&nbsp;</span>en<span class="nowrap">&nbsp;</span></code></b> for <a href="/wiki/English_language" title="English language">English</a>. Its second parameter houses the language code for the targeted page, in this case <b><code><span class="nowrap">&nbsp;</span>es<span class="nowrap">&nbsp;</span></code></b> for the <a href="/wiki/Spanish_language" title="Spanish language">Spanish language</a>. It does not matter if the relative position of these rcats changes for any reason, their individual parameters will continue to be effective. </p>
        <p>If more rcat parameters are used, then they work the same way. For example, if a redirect exists that is a misspelling of a title, it can be tagged with {{<a href="/wiki/Template:R_from_misspelling" title="Template:R from misspelling">R from misspelling</a>}} in the following manner: </p> 
        <pre>{{Redirect category shell|
{{R from misspelling|(correct spelling if needed)|printworthy}}
{{R printworthy}}
}}
</pre> 
        <p>In this case, the rcat's <i><code class="nowrap" style="">|1=</code></i> parameter is the correct spelling (used only if the target of the redirect is <b>not</b> the correct spelling). The rcat's <i><code class="nowrap" style="">|2=</code></i> parameter can be used to change from the default "unprintworthy" categorization. Some misspellings may be helpful in a printed version of <i>Wikipedia</i>, so the second parameter in <i>R from misspelling</i> above is used to subdue its default "unprintworthy" category sort, and the redirect will populate <a href="/wiki/Category:Printworthy_redirects" title="Category:Printworthy redirects">Category:Printworthy redirects</a> instead. In this shell template, the order of the rcats will not affect the rcats' parameters. If an editor checks the page history and finds that the above redirect is also the result of a page move and inserts {{<a href="/wiki/Template:R_from_move" title="Template:R from move">R from move</a>}} at the top of the order: </p> 
        <pre>{{Redirect category shell|
{{R from move}}
{{R from misspelling|(correct spelling)|printworthy}}
{{R printworthy}}
}}
</pre> 
        <p>...the parameters of the <i>R from misspelling</i> rcat template will not be affected, as they may be when used in the <i>This is a redirect</i> template. </p>
        <p>An example of a "printworthy misspelling" can be found at template {{<a href="/wiki/Template:R_from_misspelling" title="Template:R from misspelling">R from misspelling</a>}}. </p>
        <p><b>Another example:</b> </p> 
        <pre>{{Redirect category shell|
{{R from alternative language|ja|es}}
{{R unprintworthy}}
}}
</pre> 
        <p>The above code will sort the redirect into three categories: </p> 
        <ul>
         <li><a href="/wiki/Category:Redirects_from_Japanese-language_terms" title="Category:Redirects from Japanese-language terms">Redirects from Japanese-language terms</a> (<code class="nowrap" style="">|1=ja</code>),</li> 
         <li><a href="/wiki/Category:Redirects_to_Spanish-language_terms" title="Category:Redirects to Spanish-language terms">Redirects to Spanish-language terms</a> (<code class="nowrap" style="">|2=es</code>) and</li> 
         <li><a href="/wiki/Category:Unprintworthy_redirects" title="Category:Unprintworthy redirects">Unprintworthy redirects</a></li>
        </ul> 
        <p><i>See the lists of <a href="/wiki/List_of_ISO_639-1_codes" title="List of ISO 639-1 codes">ISO 639-1</a> and <a href="/wiki/List_of_ISO_639-2_codes" title="List of ISO 639-2 codes">ISO 639-2</a> codes for two- and three-letter language codes that can be used this way. See also <a href="/wiki/Wikipedia:Template_messages/Redirect_language_codes" class="mw-redirect" title="Wikipedia:Template messages/Redirect language codes">Wikipedia:Template messages/Redirect language codes</a> for common codes that are already used on redirects.</i> </p>

          <ol class="references"> 
           <li id="cite_note-mbox-1"><span class="mw-cite-backlink"><b><a href="#cite_ref-mbox_1-0">^</a></b></span> <span class="reference-text">When the wikimarkup code of this template is opened, it is seen that this is an "mbox-based" template. "Mbox" stands for "multi namespace message box", which is a meta-<a href="/wiki/Help:A_quick_guide_to_templates" title="Help:A quick guide to templates">template</a> that works in all namespaces. There can be a problem with this mbox unless editors are careful to place <i>Redirect category shell</i> <b>on its own line at the <i>very beginning of the line</i></b>. If this is done then all is well; if this is not done, then <a href="/wiki/HTML_Tidy" title="HTML Tidy">HTML Tidy</a> may interfere with the mbox and cause this template to appear in an abnormal and unexpected manner when saved. So be careful where you leave this template placed in a redirect's Edit screen, take another look at the above <a href="#goodread">readability guide</a>, and be sure to preview your edits – all your edits. 
             <dl>
              <dd>
               <b><span style="color:green;">Correct:</span></b>
              </dd>
             </dl> <pre>#REDIRECT [[(target page)]]

{{Redirect category shell||
{{R unprintworthy}}
}}
</pre> 
             <dl>
              <dd>
               <b><span style="color: red;"><i>Not</i> correct:</span></b>
              </dd>
             </dl> <pre>#REDIRECT [[(target page)]] {{Redirect category shell||{{R unprintworthy}}}}
</pre> 
             <dl>
              <dd>
               <i>A blank line is usually placed between the redirect target and this template to make the "Edit" screen easier to read.</i>
              </dd>
             </dl> </span></li> 
          </ol>

        <p><b>Printworthiness note</b> </p> 
        <div role="note" class="hatnote navigation-not-searchable">
         See also: <a href="/wiki/Wikipedia:Printability" title="Wikipedia:Printability">Wikipedia:Printability</a>
        </div> 
        <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1011085734">
        <div class="reflist"> 
         <div class="mw-references-wrap">
          <ol class="references"> 
           <li id="cite_note-printworthiness-2"><span class="mw-cite-backlink"><b><a href="#cite_ref-printworthiness_2-0">^</a></b></span> <span class="reference-text">Where the <a href="/wiki/Wikipedia:Printworthiness" class="mw-redirect" title="Wikipedia:Printworthiness">printworthiness</a> of a <a href="/wiki/Wikipedia:Mainspace" class="mw-redirect" title="Wikipedia:Mainspace">mainspace</a> redirect is concerned, there are three kinds of <a href="/wiki/Wikipedia:RCAT" class="mw-redirect" title="Wikipedia:RCAT">rcats</a>: 
             <ol>
              <li>those that populate the <a href="/wiki/Category:Printworthy_redirects" title="Category:Printworthy redirects">Printworthy redirects</a> category by default,</li> 
              <li>those that populate the <a href="/wiki/Category:Unprintworthy_redirects" title="Category:Unprintworthy redirects">Unprintworthy redirects</a> category by default, and</li> 
              <li>those that do not default to either of the above, and which leave it to the editor to decide based upon best knowledge whether or not a mainspace redirect is suitable for a printed or DVD version of <i>Wikipedia</i>.</li>
             </ol> At present, those that are types 1 and 2 will populate the appropriate printworthiness category, however the notice will not appear within the mbox of this template unless either {{<a href="/wiki/Template:R_printworthy" title="Template:R printworthy">R printworthy</a>}} or {{<a href="/wiki/Template:R_unprintworthy" title="Template:R unprintworthy">R unprintworthy</a>}} is listed as an rcat within this template. For example, if a redirect is tagged with: <pre>{{Redirect category shell|
{{R from ambiguous term}}
}}
</pre> <p>...then the hidden categories populated will be <a href="/wiki/Category:Redirects_from_ambiguous_pages" title="Category:Redirects from ambiguous pages">Redirects from ambiguous pages</a>, <a href="/wiki/Category:Unprintworthy_redirects" title="Category:Unprintworthy redirects">Unprintworthy redirects</a> and any appropriate protection categories. These will appear at the bottom of the redirect page only to registered users who are logged in and who have set their preferences to see hidden categories. The mbox that will appear to all readers would look like: </p>
