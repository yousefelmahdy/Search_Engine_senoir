         <tr>
          <td colspan="2" class="navbox-list navbox-odd vlist" style="width:100%;padding:0px;text-align: left; padding: 0.75em 0;">
           <div style="padding:0em 0.25em">
            <br>
            <div style="text-align:center;">
             <b>This page is for documenting and explaining the English Wikipedia <a href="/wiki/Wikipedia:Policies_and_guidelines" title="Wikipedia:Policies and guidelines">policy</a> on page protection.</b>
            </div> 
            <p><br> </p> 
            <ul>
             <li>Are you trying to request protection of a page? <b><a href="/wiki/Wikipedia:Requests_for_page_protection" title="Wikipedia:Requests for page protection">Request page protection</a></b>.</li> 
             <li>Are you trying to request removal of a page's protection? <b><a href="/wiki/Wikipedia:Requests_for_page_protection" title="Wikipedia:Requests for page protection">Request a change in page protection level</a></b>.</li> 
             <li>Do you want to request an edit be made to a protected page? <b><a href="/wiki/Wikipedia:Edit_requests" title="Wikipedia:Edit requests">Create an edit request</a></b>.</li> 
             <li>Do you need to report persistent vandalism or spam to an <a href="/wiki/Wikipedia:ADMIN" class="mw-redirect" title="Wikipedia:ADMIN">administrator</a>? <b><a href="/wiki/Wikipedia:Administrator_intervention_against_vandalism" title="Wikipedia:Administrator intervention against vandalism">File a vandalism report</a></b>.</li> 
             <li>Do you need to report an inappropriate username or an account with a username that blatantly violates Wikipedia's <a href="/wiki/Wikipedia:Username_policy" title="Wikipedia:Username policy">username policy</a>? <b><a href="/wiki/Wikipedia:Usernames_for_administrator_attention" title="Wikipedia:Usernames for administrator attention">File a username report</a></b>.</li> 
             <li>Are you currently blocked from editing Wikipedia and wish to request an unblock? You can resolve this matter by <b><a href="/wiki/Wikipedia:Appealing_a_block" title="Wikipedia:Appealing a block">appealing your block</a></b>.</li> 
             <li>Do you need to report an <a href="/wiki/Wikipedia:Edit_warring" title="Wikipedia:Edit warring">edit war</a> or <a href="/wiki/Wikipedia:3RR" class="mw-redirect" title="Wikipedia:3RR">three-revert rule</a> violation to an administrator? <b><a href="/wiki/Wikipedia:Administrators%27_noticeboard/Edit_warring" title="Wikipedia:Administrators' noticeboard/Edit warring">File an edit war report</a></b>.</li> 
             <li>Do you need to report or discuss a policy violation or behavioral issue (such as an evasion of a <a href="/wiki/Wikipedia:BLOCK" class="mw-redirect" title="Wikipedia:BLOCK">block</a>, abuse of tools, or other incidents) on Wikipedia that requires administrator intervention? <b><a href="/wiki/Wikipedia:Administrators%27_noticeboard/Incidents" title="Wikipedia:Administrators' noticeboard/Incidents">File an incident report</a></b>.</li>
            </ul>
           </div></td>

      <p>In some circumstances, pages may need to be <b>protected</b> from modification by certain groups of editors. Pages are protected when a specific damaging event has been identified that can not be prevented through other means such as a <a href="/wiki/Wikipedia:Blocking_policy" title="Wikipedia:Blocking policy">block</a>. Otherwise, Wikipedia is built on the principle that <a href="/wiki/Wiki" title="Wiki">anyone can edit it</a>, and it therefore aims to have as many of its pages as possible open for public editing so that anyone can add material and correct errors. This policy explains in detail the protection types and procedures for page protection and unprotection and when each protection should and should not be applied. </p>
      <p>Protection is a technical restriction applied only by <a href="/wiki/Wikipedia:Administrators" title="Wikipedia:Administrators">administrators</a>, although any user may <a href="/wiki/Wikipedia:Requests_for_page_protection" title="Wikipedia:Requests for page protection">request protection</a>. Protection can be <a href="/wiki/Wikipedia:List_of_indefinitely_protected_pages" title="Wikipedia:List of indefinitely protected pages">indefinite</a> or expire after a specified time. The various levels of protection are detailed below, and they can be applied to the page edit, page move, page create, and file upload actions. Even when a page is protected from editing, the <a href="/wiki/Source_code" title="Source code">source code</a> (text) of the page can still be viewed and copied by anyone. </p>
      <p>A protected page is marked at its top right by a padlock icon, usually added by the <code>{{<a href="/wiki/Template:Pp-protected" class="mw-redirect" title="Template:Pp-protected">pp-protected</a>}}</code> template. </p>

      <h2><span class="mw-headline" id="Types_of_protection">Types of protection</span></h2> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:PPLIST"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:PPLIST&amp;redirect=no">WP:PPLIST</a></span></li>
       </ul>
      </div> 
      <p>The following technical options are available to administrators for protecting different actions to pages: </p> 
      <ul>
       <li><b>Edit protection</b> protects the page from being <a href="/wiki/Help:Editing" title="Help:Editing">edited</a>.</li> 
       <li><b>Move protection</b> protects the page from being <a href="/wiki/Help:Moving_a_page" class="mw-redirect" title="Help:Moving a page">moved or renamed</a>.</li> 
       <li><b>Creation protection</b> prevents a page (normally a previously <a href="/wiki/Wikipedia:Deletion_policy" title="Wikipedia:Deletion policy">deleted</a> one) from being created (also known as "<a href="/wiki/Wikipedia:Salt" class="mw-redirect" title="Wikipedia:Salt">salting</a>").</li> 
       <li><b>Upload protection</b> prevents new versions of a file from being <a href="/wiki/Help:Upload" class="mw-redirect" title="Help:Upload">uploaded</a>, but it does not prevent editing to the file's description page (unless edit protection is applied).</li>
      </ul> 
      <p>The following technical options are available to administrators for adding protection levels to the different actions to pages: </p> 
      <ul>
       <li><b>Pending changes protection</b> <i>(only available for edit protection)</i> means edits by unregistered and new contributors are not visible to readers who are not logged-in until the edits are approved by a <a href="/wiki/Wikipedia:User_access_levels#Reviewer" title="Wikipedia:User access levels">reviewer</a> or an administrator.</li> 
       <li><b>Semi-protection</b> prevents the action by unregistered contributors and contributors with accounts that are not <a href="/wiki/Wikipedia:User_access_levels#Autoconfirmed_users" title="Wikipedia:User access levels">confirmed</a>.</li> 
       <li><b>Extended confirmed protection</b>, also known as <b>30/500 protection</b>, prevents the action by users without 30 days' tenure and 500 edits on the English Wikipedia. It is applied to combat any form of disruption where semi-protection has proven to be ineffective. It should not be applied as a protection level of first resort. Its use is logged at the <a href="/wiki/Wikipedia:AN" class="mw-redirect" title="Wikipedia:AN">Administrators' noticeboard</a>.</li> 
       <li><b>Template protection</b> prevents the action by everyone except <a href="/wiki/Wikipedia:Template_editor" title="Wikipedia:Template editor">template editors</a> and <a href="/wiki/Wikipedia:Administrators" title="Wikipedia:Administrators">administrators</a> (who have this right as part of their toolset).</li> 
       <li><b>Full protection</b> prevents the action by everyone except <a href="/wiki/Wikipedia:Administrators" title="Wikipedia:Administrators">administrators</a>.</li>
      </ul> 
      <p>Any type of protection (with the exception of <a href="#Cascading_protection">cascading protection</a>) may be requested at <a href="/wiki/Wikipedia:Requests_for_page_protection" title="Wikipedia:Requests for page protection">Wikipedia:Requests for page protection</a>. Changes to a fully protected page should be proposed on the corresponding <a href="/wiki/Wikipedia:Talk_page" class="mw-redirect" title="Wikipedia:Talk page">talk page</a>, then carried out by an administrator if they are uncontroversial or there is <a href="/wiki/Wikipedia:Consensus" title="Wikipedia:Consensus">consensus</a> for them. </p>
      <p><span class="anchor" id="RemovingProtection"></span>Except in the case of <a href="/wiki/Wikipedia:Office_actions" title="Wikipedia:Office actions">office actions</a> (see <a href="#Office_actions">below</a>), <a href="/wiki/Wikipedia:AC" class="mw-redirect" title="Wikipedia:AC">Arbitration Committee</a> remedies, or pages in the MediaWiki namespace (see below), administrators may unprotect a page if the reason for its protection no longer applies, a reasonable period has elapsed, and there is no consensus that continued protection is necessary. Editors desiring the unprotection of a page should, in the first instance, ask the administrator who applied the protection unless the administrator is inactive or no longer an administrator; thereafter, requests may be made at <a href="/wiki/Wikipedia:RFUP" class="mw-redirect" title="Wikipedia:RFUP">Requests for unprotection</a>. Note that such requests will normally be declined if the protecting administrator is active and was not consulted first. A log of protections and unprotections is available at <a href="/wiki/Special:Log/protect" title="Special:Log/protect">Special:Log/protect</a>. </p>

      <h3><span class="mw-headline" id="Full_protection">Full protection</span></h3> 
      <div class="floatright">
       <img alt="Gold padlock" src="//upload.wikimedia.org/wikipedia/en/thumb/4/44/Full-protection-shackle.svg/64px-Full-protection-shackle.svg.png" decoding="async" width="64" height="64" srcset="//upload.wikimedia.org/wikipedia/en/thumb/4/44/Full-protection-shackle.svg/96px-Full-protection-shackle.svg.png 1.5x, //upload.wikimedia.org/wikipedia/en/thumb/4/44/Full-protection-shackle.svg/128px-Full-protection-shackle.svg.png 2x" data-file-width="512" data-file-height="512">
      </div> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:FULL"></span><span id="WP:GOLDLOCK"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcuts</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:FULL&amp;redirect=no">WP:FULL</a></span></li>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:GOLDLOCK&amp;redirect=no">WP:GOLDLOCK</a></span></li>
       </ul>
      </div> 
      <p>A fully protected page cannot be edited or moved by anyone except <a href="/wiki/Wikipedia:Administrators" title="Wikipedia:Administrators">administrators</a>. The protection may be for a specified time or may be indefinite. </p>
      <p>Modifications to a fully protected page can be proposed on its <a href="/wiki/Wikipedia:Talk_page_guidelines" title="Wikipedia:Talk page guidelines">talk page</a> (or at another appropriate forum) for discussion. Administrators can make changes to the protected article reflecting <a href="/wiki/Wikipedia:Consensus" title="Wikipedia:Consensus">consensus</a>. Placing the <code>{{<a href="/wiki/Template:Edit_fully_protected" class="mw-redirect" title="Template:Edit fully protected">Edit fully protected</a>}}</code> template on the talk page will draw the attention of administrators for implementing uncontroversial changes. </p>
      <p><span class="anchor" id="full-dispute"></span> </p> 
      <h4><span class="mw-headline" id="Content_disputes">Content disputes</span></h4> 
      <div role="note" class="hatnote navigation-not-searchable">
       See also: <a href="/wiki/Wikipedia:Stable_version" title="Wikipedia:Stable version">Wikipedia:Stable version</a>
      </div> 
      <div role="note" class="hatnote navigation-not-searchable">
       "WP:PREFER" redirects here. For what title name should be preferred, see <a href="/wiki/Wikipedia:PRIMARYTOPIC" class="mw-redirect" title="Wikipedia:PRIMARYTOPIC">WP:PRIMARYTOPIC</a>.
      </div> 
      <p>While content disputes and <a href="/wiki/Wikipedia:Edit_war" class="mw-redirect" title="Wikipedia:Edit war">edit warring</a> may be addressed with user <a href="/wiki/Wikipedia:Blocking_policy" title="Wikipedia:Blocking policy">blocks</a> issued by <a href="/wiki/Wikipedia:UNINVOLVED" class="mw-redirect" title="Wikipedia:UNINVOLVED">uninvolved</a> administrators, allowing normal page editing by other editors at the same time, the protection policy provides an alternative approach as administrators have the discretion to temporarily fully protect an article to end an ongoing edit war. This approach may better suit multi-party disputes and contentious content, as it makes talk page <a href="/wiki/Wikipedia:Consensus" title="Wikipedia:Consensus">consensus</a> a requirement for implementation of <a href="/wiki/Wikipedia:Edit_requests" title="Wikipedia:Edit requests">requested edits</a>. </p>
      <p><span class="anchor" id="PREFER"></span><span class="anchor" id="prefer"></span> </p> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:PREFER"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:PREFER&amp;redirect=no">WP:PREFER</a></span></li>
       </ul>
      </div> 
      <p>When protecting a page because of a content dispute, administrators have a duty to avoid protecting a version that contains policy-violating content, such as <a href="/wiki/Wikipedia:Vandalism" title="Wikipedia:Vandalism">vandalism</a>, <a href="/wiki/Wikipedia:COPYVIO" class="mw-redirect" title="Wikipedia:COPYVIO">copyright violations</a>, <a href="/wiki/Wikipedia:Libel" title="Wikipedia:Libel">defamation</a>, or <a href="/wiki/Wikipedia:BLP" class="mw-redirect" title="Wikipedia:BLP">poor-quality coverage of living people</a>. Administrators are deemed to remain uninvolved when exercising discretion on whether to apply protection to the current version of an article, or to an older, <a href="/wiki/Wikipedia:STABLE" class="mw-redirect" title="Wikipedia:STABLE">stable</a>, or pre-edit-war version. </p>
      <p>Protected pages may not be edited except to make changes that are uncontroversial or for which there is clear consensus. Editors convinced that the protected version of an article contains policy-violating content, or that protection has rewarded edit warring or disruption by establishing a contentious revision, may identify a stable version <i>prior to</i> the edit war and <a href="/wiki/Wikipedia:Edit_requests" title="Wikipedia:Edit requests">request</a> reversion to that version. Before making such a request, editors should consider how independent editors might view the suggestion and recognize that continuing an edit war is grounds for being blocked. </p>
      <p>Administrators who have made substantive content changes to an article are considered <a href="/wiki/Wikipedia:INVOLVED" class="mw-redirect" title="Wikipedia:INVOLVED">involved</a> and must not use their advanced permissions to further their own positions. When involved in a dispute, it is almost always wisest to respect the editing policies that bind all editors and call for input from an uninvolved administrator, rather than to invite controversy by acting unilaterally. </p> 
      <h4><span class="mw-headline" id="Vandalism">Vandalism</span></h4> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:NO-PREEMPT"></span><span id="WP:PREEMPTIVE"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcuts</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:NO-PREEMPT&amp;redirect=no">WP:NO-PREEMPT</a></span></li>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:PREEMPTIVE&amp;redirect=no">WP:PREEMPTIVE</a></span></li>
       </ul>
      </div> 
      <p>Applying page protection as a preemptive measure is contrary to the open nature of Wikipedia and is generally not allowed if applied for these reasons. However, brief periods of an appropriate and reasonable protection level are allowed in situations where blatant vandalism or disruption is occurring by multiple users and at a level of frequency that requires its use in order to stop it. The duration of the protection should be set as short as possible, and the protection level should be set to the lowest restriction needed in order to stop the disruption while still allowing productive editors to make changes. </p> 
      <h4><span id=".22History_only.22_review"></span><span class="mw-headline" id="&quot;History_only&quot;_review">"History only" review</span></h4> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:PPDRV"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:PPDRV&amp;redirect=no">WP:PPDRV</a></span></li>
       </ul>
      </div> 
      <p>If a deleted page is going through <a href="/wiki/Wikipedia:Deletion_review" title="Wikipedia:Deletion review">deletion review</a>, only administrators are normally capable of viewing the former content of the page. If they feel it would benefit the discussion to allow other users to view the page content, administrators may <a href="/wiki/Wikipedia:Restore" class="mw-redirect" title="Wikipedia:Restore">restore</a> the page, blank it or replace the contents with <code>{{<a href="/wiki/Template:Temporarily_undeleted" title="Template:Temporarily undeleted">Temporarily undeleted</a>}}</code> or a similar notice, and fully protect the page to prevent further editing. The previous contents of the page are then accessible to everyone via the <a href="/wiki/Help:Page_history" title="Help:Page history">page history</a>. </p> 
      <h4><span class="mw-headline" id="Protected_generic_file_names"><span id="Protected_generic_image_names"></span> Protected generic file names</span></h4> 
      <p>Generic file names such as <a href="/wiki/File:Photo.jpg" title="File:Photo.jpg">File:Photo.jpg</a>, <a href="/w/index.php?title=File:Map.jpg&amp;action=edit&amp;redlink=1" class="new" title="File:Map.jpg (page does not exist)">File:Map.jpg</a>, and <a href="/w/index.php?title=File:Sound.wav&amp;action=edit&amp;redlink=1" class="new" title="File:Sound.wav (page does not exist)">File:Sound.wav</a> are fully protected to prevent new versions being uploaded. Furthermore, <a href="/w/index.php?title=File:Map.jpg&amp;action=edit&amp;redlink=1" class="new" title="File:Map.jpg (page does not exist)">File:Map.jpg</a> and <a href="/w/index.php?title=File:Sound.wav&amp;action=edit&amp;redlink=1" class="new" title="File:Sound.wav (page does not exist)">File:Sound.wav</a> are salted. See <a href="/wiki/Wikipedia:SALT" class="mw-redirect" title="Wikipedia:SALT">WP:SALT</a>. </p>

      <h4><span class="mw-headline" id="Permanent_protection">Permanent protection</span></h4> 
      <p><span class="anchor" id="Interface_protection"></span> </p> 
      <div class="floatright">
       <img alt="Brown padlock" src="//upload.wikimedia.org/wikipedia/en/thumb/2/28/Interface-protection-shackle.svg/64px-Interface-protection-shackle.svg.png" decoding="async" title="Icon for pages that can be edited only by interface administrators" width="64" height="64" srcset="//upload.wikimedia.org/wikipedia/en/thumb/2/28/Interface-protection-shackle.svg/96px-Interface-protection-shackle.svg.png 1.5x, //upload.wikimedia.org/wikipedia/en/thumb/2/28/Interface-protection-shackle.svg/128px-Interface-protection-shackle.svg.png 2x" data-file-width="512" data-file-height="512">

      <div class="module-shortcutanchordiv">
       <span id="WP:PPINDEF"></span><span id="WP:REDLOCK"></span>

      <div class="module-shortcutboxplain plainlist noprint" role="note">

       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:PPINDEF&amp;redirect=no">WP:PPINDEF</a></span></li>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:REDLOCK&amp;redirect=no">WP:REDLOCK</a></span></li>

      <p>Administrators cannot change or remove the protection for some areas on Wikipedia which are permanently protected by the <a href="/wiki/MediaWiki" title="MediaWiki">MediaWiki</a> software: </p> 
      <ul>
       <li>Edits to the <a href="/wiki/Help:MediaWiki_namespace" title="Help:MediaWiki namespace">MediaWiki namespace</a>, which defines parts of the site interface, are restricted to administrators. 
        <ul>
         <li>Edits to system-wide CSS and JavaScript pages such as <a href="/wiki/MediaWiki:Common.js" title="MediaWiki:Common.js">MediaWiki:common.js</a> are further restricted to <a href="/wiki/Wikipedia:IADMIN" class="mw-redirect" title="Wikipedia:IADMIN">interface administrators</a>.</li>
        </ul></li> 
       <li>Edits to personal CSS and JavaScript pages such as <a href="/wiki/User:Example/monobook.css" title="User:Example/monobook.css">User:Example/monobook.css</a> and <a href="/w/index.php?title=User:Example/cologneblue.js&amp;action=edit&amp;redlink=1" class="new" title="User:Example/cologneblue.js (page does not exist)">User:Example/cologneblue.js</a> are restricted to the associated user and interface administrators. Interface administrators may edit these pages, for example, to remove a user script that has been used in an inappropriate way. Administrators may delete (but not edit or restore) these pages.</li> 
       <li>Edits to personal JSON pages such as <a href="/wiki/User:Example/data.json" title="User:Example/data.json">User:Example/data.json</a> are restricted to the associated user and administrators.</li>
      </ul> 
      <p>In addition to hard-coded protection, the following are usually fully protected for an indefinite period of time: </p> 
      <ul>
       <li>Pages that are very visible, such as the <a href="/wiki/Main_Page" title="Main Page">Main Page</a></li> 
       <li>Pages that should not be modified for copyright or legal reasons, such as the <a href="/wiki/Wikipedia:General_disclaimer" title="Wikipedia:General disclaimer">general disclaimer</a> or the <a href="/wiki/Wikipedia:Text_of_Creative_Commons_Attribution-ShareAlike_3.0_Unported_License" title="Wikipedia:Text of Creative Commons Attribution-ShareAlike 3.0 Unported License">local copy of the site copyright license</a>.</li> 
       <li>Pages that are very frequently <a href="/wiki/Wikipedia:Transclusion" class="mw-redirect" title="Wikipedia:Transclusion">transcluded</a>, such as <code>{{<a href="/wiki/Template:Tl" class="mw-redirect" title="Template:Tl">tl</a>}}</code> or <code>{{<a href="/wiki/Template:Citation_needed" title="Template:Citation needed">citation needed</a>}}</code>, to prevent <a href="/wiki/Wikipedia:Vandalism" title="Wikipedia:Vandalism">vandalism</a> or <a href="/wiki/Denial-of-service_attack" title="Denial-of-service attack">denial of service</a> attacks. This includes images or templates used in other highly visible or frequently transcluded pages. See <a href="/wiki/Wikipedia:High-risk_templates" title="Wikipedia:High-risk templates">Wikipedia:High-risk templates</a> for more information.</li>
      </ul>

      <p>A template-protected page can be edited only by administrators or users in the Template editors group. This protection level should be used almost exclusively on <a href="/wiki/Wikipedia:High-risk_templates" title="Wikipedia:High-risk templates">high-risk templates</a> and <a href="/wiki/Wikipedia:LUA" class="mw-redirect" title="Wikipedia:LUA">modules</a>. In cases where pages in other namespaces become transcluded to a very high degree, this protection level is also valid. </p>
      <p>This is a protection level<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup> that replaces <a href="/wiki/Wikipedia:FULL" class="mw-redirect" title="Wikipedia:FULL">full protection</a> on pages that are merely protected due to high transclusion rates, rather than content disputes. It should be used on templates whose risk factor would have otherwise warranted full protection. It should not be used on less risky templates on the grounds that the <a href="/wiki/Wikipedia:TPE" class="mw-redirect" title="Wikipedia:TPE">template editor</a> user right existsthe existence of the right should not result in more templates becoming uneditable for the general editing community. </p>
      <p>Editors may request edits to a template-protected page by proposing them on its talk page, using the <code>{{<a href="/wiki/Template:Edit_template-protected" title="Template:Edit template-protected">Edit template-protected</a>}}</code> template if necessary to gain attention. </p>
      <p><span class="anchor" id="semi"></span><span class="anchor" id="semi-user"></span> </p> 
      <h3><span class="mw-headline" id="Semi-protection">Semi-protection</span></h3> 
      <div role="note" class="hatnote navigation-not-searchable">
       See also: <a href="/wiki/Wikipedia:Rough_guide_to_semi-protection" title="Wikipedia:Rough guide to semi-protection">Wikipedia:Rough guide to semi-protection</a>
      </div> 
      <div class="floatright">
       <img alt="Silver padlock" src="//upload.wikimedia.org/wikipedia/en/thumb/1/1b/Semi-protection-shackle.svg/64px-Semi-protection-shackle.svg.png" decoding="async" width="64" height="64" srcset="//upload.wikimedia.org/wikipedia/en/thumb/1/1b/Semi-protection-shackle.svg/96px-Semi-protection-shackle.svg.png 1.5x, //upload.wikimedia.org/wikipedia/en/thumb/1/1b/Semi-protection-shackle.svg/128px-Semi-protection-shackle.svg.png 2x" data-file-width="512" data-file-height="512">

      <p>Semi-protected pages cannot be edited by <a href="/wiki/Wikipedia:User_access_levels#Unregistered_users" title="Wikipedia:User access levels">unregistered users</a> (<a href="/wiki/IP_address" title="IP address">IP addresses</a>), as well as accounts that are not <a href="/wiki/Wikipedia:User_access_levels#Autoconfirmed_users" title="Wikipedia:User access levels">autoconfirmed</a> (accounts that are at least four days old and have made at least ten edits to Wikipedia) or <a href="/wiki/Wikipedia:User_access_levels#Confirmed_users" title="Wikipedia:User access levels">confirmed</a>. Semi-protection is useful when there is a significant amount of disruption or <a href="/wiki/Wikipedia:Vandalism" title="Wikipedia:Vandalism">vandalism</a> from new or unregistered users, or to prevent <a href="/wiki/Sockpuppet_(Internet)" class="mw-redirect" title="Sockpuppet (Internet)">sockpuppets</a> of blocked or banned users from editing, especially when it occurs on <a href="/wiki/Wikipedia:BLP" class="mw-redirect" title="Wikipedia:BLP">biographies of living persons</a> who have had a recent high level of media interest. An alternative to semi-protection is <a href="/wiki/Wikipedia:Pending_changes" title="Wikipedia:Pending changes">pending changes</a>, which is sometimes favored when an article is being vandalized regularly, but otherwise receives a low amount of editing. </p>
      <p>Such users can request edits to a semi-protected page by proposing them on its talk page, using the {{<a href="/wiki/Template:Edit_semi-protected" title="Template:Edit semi-protected">Edit semi-protected</a>}} template if necessary to gain attention. If the page in question and its talk page are <i>both</i> protected, please make your edit request at <a href="/wiki/Wikipedia:RFED" class="mw-redirect" title="Wikipedia:RFED">Wikipedia:Requests for page protection</a> instead. New users may also request the confirmed user right at <a href="/wiki/Wikipedia:Requests_for_permissions/Confirmed" title="Wikipedia:Requests for permissions/Confirmed">Wikipedia:Requests for permissions/Confirmed</a>. </p> 
      <h4><span class="mw-headline" id="Guidance_for_administrators">Guidance for administrators</span></h4> 
      <p>Administrators may apply <em>indefinite</em> semi-protection to pages that are subject to heavy and persistent <a href="/wiki/Wikipedia:Vandalism" title="Wikipedia:Vandalism">vandalism</a> or violations of <a href="/wiki/Category:Wikipedia_content_policies" title="Category:Wikipedia content policies">content policy</a> (such as <a href="/wiki/Wikipedia:Biographies_of_living_persons" title="Wikipedia:Biographies of living persons">biographies of living persons</a>, <a href="/wiki/Wikipedia:Neutral_point_of_view" title="Wikipedia:Neutral point of view">neutral point of view</a>). Semi-protection should <em>not</em> be used as a preemptive measure against vandalism that has not yet occurred or to privilege registered users over unregistered users in (valid) content disputes. </p>
      <p>In addition, administrators may apply <em>temporary</em> semi-protection on pages that are: </p> 
      <ul>
       <li>Subject to significant but temporary vandalism or disruption (for example, due to <a href="/wiki/Wikipedia:Wikipedia_in_the_media" title="Wikipedia:Wikipedia in the media">media attention</a>) if <a href="/wiki/Wikipedia:Blocking_policy" title="Wikipedia:Blocking policy">blocking</a> individual users is not a feasible option.</li> 
       <li>Subject to <a href="/wiki/Wikipedia:Edit_warring" title="Wikipedia:Edit warring">edit warring</a> if <em>all</em> parties involved are unregistered or new editors (i.e. in cases in which full protection would otherwise be applied). This does <em>not</em> apply when autoconfirmed users are involved.</li> 
       <li>Subject to <a href="/wiki/Wikipedia:Vandalism" title="Wikipedia:Vandalism">vandalism</a> or <a href="/wiki/Wikipedia:Edit_warring" title="Wikipedia:Edit warring">edit warring</a> where unregistered editors are engaging in <a href="/wiki/IP_address#Dynamic_IP" title="IP address">IP hopping</a> by using different computers, obtaining new addresses by using <a href="/wiki/Dynamic_IP" class="mw-redirect" title="Dynamic IP">dynamic IP</a> allocation, or other address-changing schemes.</li> 
       <li>Article discussion pages, if they have been subject to persistent disruption. Such protection should be used sparingly because it prevents unregistered and newly registered users from participating in discussions. A page and its talk page should not normally be protected at the same time. If a page and its talk page are both protected, the talk page should direct affected editors to <a href="/wiki/Wikipedia:RFED" class="mw-redirect" title="Wikipedia:RFED">Wikipedia:Request for edit</a> to ensure that no editor is entirely prevented from contributing.</li> 
       <li>Protection should be used sparingly on the talk pages of blocked users, including IP addresses. Instead the user should be re-blocked with talk page editing disallowed. When required, or when re-blocking without talk page editing allowed is unsuccessful, protection should be implemented for only a brief period not exceeding the duration of the block.</li>
      </ul> 
      <p><a href="/wiki/Wikipedia:Today%27s_featured_article" title="Wikipedia:Today's featured article">Today's featured article</a> may be semi-protected just like any other article. But since that article is subject to sudden spurts of vandalism during certain times of day, administrators should semi-protect it for brief periods in most instances. For the former guideline, see <a href="/wiki/Wikipedia:Main_Page_featured_article_protection" title="Wikipedia:Main Page featured article protection">Wikipedia:Main Page featured article protection</a>. </p>

      <h3><span id="Creation_protection_.28salting.29"></span><span class="mw-headline" id="Creation_protection_(salting)"><span id="Creation_protection"></span>Creation protection (salting)</span></h3>

      <p>Administrators can prevent the creation of pages. This level of protection is useful for bad pages that have been <a href="/wiki/Wikipedia:Deletion_policy" title="Wikipedia:Deletion policy">deleted</a> but repeatedly recreated. Such protection is case-sensitive. There are several levels of creation protection that can be applied to pages, identical to the levels for edit protection. A list of protected titles may be found at <a href="/wiki/Special:ProtectedTitles" title="Special:ProtectedTitles">Special:ProtectedTitles</a> (see also <a href="/wiki/Wikipedia:Protected_titles/Historical" class="mw-redirect" title="Wikipedia:Protected titles/Historical">historical lists</a>). </p>
      <p>Pre-emptive restrictions on new article titles are instituted through the <a href="/wiki/MediaWiki:Titleblacklist" title="MediaWiki:Titleblacklist">title blacklist</a> system, which allows for more flexible protection with support for <a href="/wiki/Substring" title="Substring">substrings</a> and <a href="/wiki/Regular_expression" title="Regular expression">regular expressions</a>. </p>
      <p><span class="anchor" id="Salted"></span>Pages that have been creation-protected are sometimes referred to as "<a href="/wiki/Salting_the_earth" title="Salting the earth">salted</a>". Contributors wishing to re-create a salted title with appropriate content should either contact an <a href="/wiki/Wikipedia:Administrators" title="Wikipedia:Administrators">administrator</a> (preferably the protecting administrator), file a request at <a href="/wiki/Wikipedia:Requests_for_page_protection#Current_requests_for_reduction_in_protection_level" title="Wikipedia:Requests for page protection">Wikipedia:Requests for page protection#Current requests for reduction in protection level</a>, or use the <a href="/wiki/Wikipedia:Deletion_review" title="Wikipedia:Deletion review">deletion review</a> process. To make a convincing case for re-creation, it is helpful to show a <a href="/wiki/Wikipedia:Drafts" title="Wikipedia:Drafts">draft version</a> of the intended article when filing a request. </p>
      <p>Administrators should choose the appropriate level of create protectionautoconfirmed, extended-confirmed,<sup id="cite_ref-ECPRFC2_2-0" class="reference"><a href="#cite_note-ECPRFC2-2">[2]</a></sup> or full. Due to the implementation of <a href="/wiki/Wikipedia:ACPERM" class="mw-redirect" title="Wikipedia:ACPERM">ACPERM</a>, non-confirmed editors cannot create pages in mainspace; thus, semi-creation protection should be rare, used only for protection of pages outside of mainspace. </p>
      <p>While creation-protection is usually permanent, temporary creation protection may be applied if a page is repeatedly recreated by a single user (or <a href="/wiki/Wikipedia:SOCK" class="mw-redirect" title="Wikipedia:SOCK">sockpuppets</a> of that user, if applicable). </p>

      <p>Move protected pages, or more technically, fully move-protected pages, cannot be moved to a new title except by an administrator. Move protection is commonly applied to: </p> 
      <ul>
       <li>Pages subject to persistent page-move <a href="/wiki/Wikipedia:Vandalism" title="Wikipedia:Vandalism">vandalism</a>.</li> 
       <li>Pages subject to a page-name dispute.</li> 
       <li>Highly visible pages that have no reason to be moved, such as the <a href="/wiki/Wikipedia:Administrators%27_noticeboard" title="Wikipedia:Administrators' noticeboard">administrators' noticeboard</a> and articles selected as "<a href="/wiki/Wikipedia:Today%27s_featured_article" title="Wikipedia:Today's featured article">Today's featured article</a>" on the main page.</li>
      </ul> 
      <p>Fully edit-protected pages are also implicitly move-protected. </p>
      <p>As with full edit protection, protection because of edit warring should not be considered an endorsement of the current name. When move protection is applied during a requested move discussion, the page should be protected at the location it was at when the move request was started. </p>
      <p>All <a href="/wiki/Wikipedia:FILE" class="mw-redirect" title="Wikipedia:FILE">files</a> are implicitly move-protected; only <a href="/wiki/Wikipedia:File_mover" title="Wikipedia:File mover">file movers</a> and administrators can rename files. </p>

      <p>Upload protected files, or more technically, fully upload-protected files, cannot be replaced with new versions except by an administrator. Upload protection does not protect file pages from editing. Upload protection may be applied by an administrator to: </p>

      <h3><span class="mw-headline" id="Pending_changes_protection">Pending changes protection</span></h3> 
      <div role="note" class="hatnote navigation-not-searchable">
       Further information: <a href="/wiki/Wikipedia:Pending_changes" title="Wikipedia:Pending changes">Wikipedia:Pending changes</a>
      </div> 
      <div class="floatright">
       <img alt="White padlock" src="//upload.wikimedia.org/wikipedia/en/thumb/b/b7/Pending-protection-shackle.svg/64px-Pending-protection-shackle.svg.png" decoding="async" width="64" height="64" srcset="//upload.wikimedia.org/wikipedia/en/thumb/b/b7/Pending-protection-shackle.svg/96px-Pending-protection-shackle.svg.png 1.5x, //upload.wikimedia.org/wikipedia/en/thumb/b/b7/Pending-protection-shackle.svg/128px-Pending-protection-shackle.svg.png 2x" data-file-width="512" data-file-height="512">
      </div> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:PCPP"></span><span id="WP:WHITELOCK"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcuts</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:PCPP&amp;redirect=no">WP:PCPP</a></span></li>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:WHITELOCK&amp;redirect=no">WP:WHITELOCK</a></span></li>
       </ul>
      </div> 
      <p>Pending changes protection is a tool used to suppress <a href="/wiki/Wikipedia:Vandalism" title="Wikipedia:Vandalism">vandalism</a> and certain other persistent problems while allowing all users to continue to submit edits. Pending changes protection can be used as an alternative to semi-protection to allow unregistered and new users to edit pages, while keeping the edits hidden from the view of most readers until those changes are accepted by a <a href="/wiki/Wikipedia:RPC" class="mw-redirect" title="Wikipedia:RPC">pending changes reviewer</a>. </p>
      <p>When a page under pending changes protection is edited by an <a href="/wiki/Wikipedia:IP" class="mw-redirect" title="Wikipedia:IP">unregistered</a> (IP addresses) editor or a <a href="/wiki/Wikipedia:User_access_levels#New_users" title="Wikipedia:User access levels">new user</a>, the edit is not directly visible to the majority of Wikipedia readers, until it is reviewed and accepted by an editor with the <a href="/wiki/Wikipedia:User_access_levels#Pending_changes_reviewer" title="Wikipedia:User access levels">pending changes reviewer right</a>. When a page under pending changes protection is edited by an <a href="/wiki/Wikipedia:User_access_levels#Autoconfirmed_users" title="Wikipedia:User access levels">autoconfirmed user</a>, the edit will be immediately visible to Wikipedia readers, unless there are pending edits waiting to be reviewed. </p>
      <p>Pending changes are visible in the page history, where they are marked as pending review. Readers that are <i>not logged in</i> (the vast majority of readers) are shown the latest <i>accepted</i> version of the page; <i>logged-in</i> users see the latest version of the page, with all changes (reviewed or not) applied. When editors who are not reviewers make changes to an article with unreviewed pending changes, their edits are also marked as pending and are not visible to most readers. </p>
      <p>A user who clicks "edit this page" is always, at that point, shown the latest version of the page for editing <i>regardless of whether the user is logged in or not</i>. </p> 
      <ul>
       <li>If the editor is not logged in, their changes join any other changes to the article awaiting review&nbsp; for the present they remain hidden from not-logged-in users. (This means that when the editor looks at the article after saving, the editor won't see the change made.)</li> 
       <li>If the editor is logged in and a pending changes reviewer, and there are pending changes, the editor will be prompted to review the pending changes before editing&nbsp; see <a href="/wiki/Wikipedia:Pending_changes" title="Wikipedia:Pending changes">Wikipedia:Pending changes</a>.</li> 
       <li>If the editor is logged in and not a pending changes reviewer, then<span class="nowrap">&nbsp;</span>... 
        <ul>
         <li>If there are no unreviewed pending edits waiting, this editor's edits will be visible to everyone immediately; but</li> 
         <li>If there are unreviewed pending edits waiting, then this editor's edits will be visible only to other logged-in users (including themself) immediately, but not to readers not logged in.</li>
        </ul></li>
      </ul> 
      <p>Reviewing of pending changes should be resolved within reasonable time limits. </p> 
      <h4><span class="mw-headline" id="When_to_apply_pending_changes_protection">When to apply pending changes protection</span></h4> 
      <p>Pending changes may be used to protect articles against: </p> 
      <ul>
       <li>Persistent vandalism</li> 
       <li>Violations of the <a href="/wiki/Wikipedia:Biographies_of_living_persons" title="Wikipedia:Biographies of living persons">biographies of living persons</a> policy</li> 
       <li><a href="/wiki/Wikipedia:CV" class="mw-redirect" title="Wikipedia:CV">Copyright violations</a></li>
      </ul> 
      <p>Pending changes protection should not be used as a preemptive measure against violations that have not yet occurred. Like semi-protection, PC protection should <b>never</b> be used in genuine content disputes, where there is a risk of placing a particular group of editors (unregistered users) at a disadvantage. Pending changes protection should not be used on articles with a very high edit rate, even if they meet the aforementioned criteria. Instead semi-protection should be considered. </p>
      <p>In addition, administrators may apply temporary pending changes protection on pages that are subject to significant but temporary vandalism or disruption (for example, due to media attention) when blocking individual users is not a feasible option. As with other forms of protection, the time frame of the protection should be proportional to the problem. Indefinite PC protection should be used only in cases of severe long-term disruption. </p>
      <p>Removal of pending changes protection can be requested of any administrator, or at <a href="/wiki/Wikipedia:RFUP" class="mw-redirect" title="Wikipedia:RFUP">requests for unprotection</a>. </p>
      <p><span class="anchor" id="Reviewing"></span>The reviewing process is described in detail at <a href="/wiki/Wikipedia:Reviewing_pending_changes" title="Wikipedia:Reviewing pending changes">Wikipedia:Reviewing pending changes</a>. </p>

      <p>Extended confirmed protection, also known as 30/500 protection, only allows edits by editors with the <a href="/wiki/Wikipedia:User_access_levels#Extendedconfirmed" title="Wikipedia:User access levels">extended confirmed</a> user access level, granted automatically to registered users with at least 30 days' tenure and 500 edits. </p>
      <p>Where semi-protection has proven to be ineffective, administrators may use extended confirmed protection to combat disruption (such as <a href="/wiki/Wikipedia:VD" class="mw-redirect" title="Wikipedia:VD">vandalism</a>, abusive <a href="/wiki/Wikipedia:SOCK" class="mw-redirect" title="Wikipedia:SOCK">sockpuppetry</a>, <a href="/wiki/Wikipedia:WAR" class="mw-redirect" title="Wikipedia:WAR">edit wars</a>, etc.) on any topic. Extended confirmed protection should not be used as a preemptive measure against disruption that has not yet occurred, nor should it be used to privilege extended confirmed users over unregistered/new users in valid content disputes on articles not covered by Arbitration Committee 30/500 rulings. Extended confirmed protection may be applied by an administrator at their discretion when creation-protecting a page.<sup id="cite_ref-ECPRFC2_2-1" class="reference"><a href="#cite_note-ECPRFC2-2">[2]</a></sup> </p>
      <p>Until August 12, 2016,<sup id="cite_ref-3" class="reference"><a href="#cite_note-3">[3]</a></sup> 30/500 protection applied only in topic areas determined by the <a href="/wiki/Wikipedia:Arbitration_Committee" title="Wikipedia:Arbitration Committee">Arbitration Committee</a>, which authorized its use on articles reasonably construed as belonging to the <a href="/wiki/Arab%E2%80%93Israeli_conflict" title="ArabIsraeli conflict">ArabIsraeli conflict</a>;<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">[4]</a></sup> as an arbitration enforcement tool by motion or remedy;<sup id="cite_ref-AE-ECP_5-0" class="reference"><a href="#cite_note-AE-ECP-5">[5]</a></sup> or as a result of community consensus.<sup id="cite_ref-6" class="reference"><a href="#cite_note-6">[6]</a></sup> In February 2019, the community authorized uninvolved administrators to place pages reasonably construed as belonging to the <a href="/wiki/Wikipedia:General_sanctions/India%E2%80%93Pakistan_conflict" title="Wikipedia:General sanctions/IndiaPakistan conflict">IndiaPakistan conflict</a> under extended confirmed protection as part of a general sanctions regime.<sup id="cite_ref-7" class="reference"><a href="#cite_note-7">[7]</a></sup> In May 2020 the Arbitration Committee authorized extended confirmed protection to pages related to the history of Jews and <a href="/wiki/Antisemitism_in_Poland" class="mw-redirect" title="Antisemitism in Poland">antisemitism in Poland</a> during World War II (193345).<sup id="cite_ref-8" class="reference"><a href="#cite_note-8">[8]</a></sup> As of September 23, 2016, a bot posts a notification in a subsection of <a href="/wiki/Wikipedia:Administrators%27_noticeboard#Pages_recently_put_under_extended-confirmed_protection" title="Wikipedia:Administrators' noticeboard">AN</a> when this protection level is used.<sup id="cite_ref-9" class="reference"><a href="#cite_note-9">[9]</a></sup> A full list of the 2674 pages under 30/500 protection can be found <span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Special:ProtectedPages&amp;level=extendedconfirmed">here</a></span>. </p>
      <p>Users can request edits to an extended confirmed-protected page by proposing them on its talk page, using the <code>{{<a href="/wiki/Template:Edit_extended-protected" title="Template:Edit extended-protected">Edit extended-protected</a>}}</code> template if necessary to gain attention. </p>

      <h3><span class="mw-headline" id="Cascading_protection">Cascading protection</span></h3> 
      <div role="note" class="hatnote navigation-not-searchable">
       "WP:CASCADE" redirects here.       </div> 
      <div class="floatright">
       <img alt="Turquoise padlock" src="//upload.wikimedia.org/wikipedia/en/thumb/0/0f/Cascade-protection-shackle.svg/64px-Cascade-protection-shackle.svg.png" decoding="async" width="64" height="64" srcset="//upload.wikimedia.org/wikipedia/en/thumb/0/0f/Cascade-protection-shackle.svg/96px-Cascade-protection-shackle.svg.png 1.5x, //upload.wikimedia.org/wikipedia/en/thumb/0/0f/Cascade-protection-shackle.svg/128px-Cascade-protection-shackle.svg.png 2x" data-file-width="512" data-file-height="512">
      </div> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:CASCADE"></span><span id="WP:TURQUOISELOCK"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcuts</a>

       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:CASCADE&amp;redirect=no">WP:CASCADE</a></span></li>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:TURQUOISELOCK&amp;redirect=no">WP:TURQUOISELOCK</a></span></li>
       </ul>
      </div> 
      <p>Cascading protection fully protects a page, and extends that full protection <i>automatically</i> to any page that is <a href="/wiki/Wikipedia:Transclusion" class="mw-redirect" title="Wikipedia:Transclusion">transcluded</a> onto the protected page, whether directly or indirectly. This includes templates, images and other media that are hosted on the English Wikipedia. Files stored on Commons are <i>not</i> protected by any other wiki's cascading protection and, if they are to be protected, must be either temporarily uploaded to the English Wikipedia or explicitly protected at Commons (whether manually or through cascading protection there). When operational, <a href="https://commons.wikimedia.org/wiki/User:KrinkleBot" class="extiw" title="c:User:KrinkleBot">KrinkleBot</a> cascade-protects Commons files transcluded at <a href="/wiki/Wikipedia:Main_Page/Tomorrow" title="Wikipedia:Main Page/Tomorrow">Wikipedia:Main Page/Tomorrow</a>, <a href="/wiki/Wikipedia:Main_Page/Commons_media_protection" title="Wikipedia:Main Page/Commons media protection">Wikipedia:Main Page/Commons media protection</a> and <a href="/wiki/Main_Page" title="Main Page">Main Page</a>. As the bot's response time varies, media should <i>not</i> be transcluded on the main page (or its constituent templates) until after it has been protected. (This is particularly relevant to <a href="/wiki/Template:In_the_news" title="Template:In the news">Template:In the news</a>, for which upcoming images are not queued at <a href="/wiki/Wikipedia:Main_Page/Tomorrow" title="Wikipedia:Main Page/Tomorrow">Wikipedia:Main Page/Tomorrow</a>.) Cascading protection: </p> 
      <ul>
       <li>Should be used only to prevent <a href="/wiki/Wikipedia:Vandalism" title="Wikipedia:Vandalism">vandalism</a> when placed on particularly visible pages, such as the main page.</li> 
       <li>Is available only for fully protected pages; it is disabled for lower levels of protection as it represents a security flaw. See <a href="https://phabricator.wikimedia.org/T10796" class="extiw" title="phabricator:T10796">Phabricator:T10796</a> for more information.</li> 
       <li>Is not instantaneous; it may be several hours before it takes effect. See <a href="https://phabricator.wikimedia.org/T20483" class="extiw" title="phabricator:T20483">Phabricator:T20483</a> for more information.</li> 
       <li>Should generally not be applied directly to templates or <a href="/wiki/Wikipedia:LUA" class="mw-redirect" title="Wikipedia:LUA">modules</a>, as it will not protect transclusions inside <code class="nowrap" style="">&lt;includeonly&gt;</code> tags or transclusions that depend on template parameters, but <i>will</i> protect the <a href="/wiki/Wikipedia:Template_documentation" title="Wikipedia:Template documentation">documentation subpage</a>. See the <a href="#Protection_of_templates">"Protection of templates" section</a> below for alternatives.</li>
      </ul> 
      <p>The list of cascading-protected pages can be found at <a href="/wiki/Wikipedia:Cascade-protected_items" title="Wikipedia:Cascade-protected items">Wikipedia:Cascade-protected items</a>. Requests to add or remove cascading protection on a page should be made at <a href="/wiki/Wikipedia_talk:Cascade-protected_items" title="Wikipedia talk:Cascade-protected items">Wikipedia talk:Cascade-protected items</a> as an <a href="/wiki/Wikipedia:Edit_requests" title="Wikipedia:Edit requests">edit request</a>. </p>

      <p>Superprotect was a level of protection, allowing editing only by <a class="external text" href="https://en.wikipedia.org/w/index.php?title=Special:GlobalUsers&amp;group=staff">Wikimedia Foundation employees who are in the Staff global group</a>. It was <a href="https://gerrit.wikimedia.org/r/153302" class="extiw" title="gerrit:153302">implemented on August 10, 2014</a> and used the same day to override community consensus regarding the use of the Media Viewer on the <a href="/wiki/German_Wikipedia" title="German Wikipedia">German Wikipedia</a>'s primary site JavaScript, <a href="https://de.wikipedia.org/wiki/MediaWiki:Common.js" class="extiw" title="de:MediaWiki:Common.js">common.js</a>. It was never used on the English Wikipedia. On November 5, 2015, the WMF decided to <a href="https://www.mediawiki.org/wiki/WMF_product_development_process/2015-11-05" class="extiw" title="mw:WMF product development process/2015-11-05">remove</a> superprotect from all Wikimedia wikis. </p> 
      <h3><span class="mw-headline" id="Cascading_semi-protection">Cascading semi-protection</span></h3> 
      <p>Cascading semi-protection was formerly possible, but it was disabled in 2007 after <a href="https://phabricator.wikimedia.org/T10796" class="extiw" title="phab:T10796">users noticed</a> that non-administrators could fully protect any page by transcluding it onto the page to which cascading semi-protection had been applied by an administrator. </p>

      <p>Originally, two levels of pending changes protection existed, where level<span class="nowrap">&nbsp;</span>2 required edits by all users who are not <a href="/wiki/Wikipedia:Reviewing_pending_changes" title="Wikipedia:Reviewing pending changes">pending changes reviewers</a> to be reviewed.       <h2><span class="mw-headline" id="Protection_by_namespace">Protection by namespace</span></h2> 
      <h3><span class="mw-headline" id="Article_talk_pages"><span id="Talk_pages"></span>Article talk pages</span></h3> 
      <p><span class="anchor" id="full-talk"></span> Modifications to a protected page can be proposed on its <a href="/wiki/Wikipedia:Talk_page" class="mw-redirect" title="Wikipedia:Talk page">talk page</a> (or at another appropriate forum) for discussion. Administrators can make changes to the protected article reflecting <a href="/wiki/Wikipedia:Consensus" title="Wikipedia:Consensus">consensus</a>. Placing the <code>{{<a href="/wiki/Template:Edit_protected" class="mw-redirect" title="Template:Edit protected">Edit protected</a>}}</code> template on the talk page will draw the attention of administrators for implementing uncontroversial changes. </p>
      <p>Talk pages are not usually protected, and are semi-protected only for a limited duration in the most severe cases of vandalism. </p> 
      <h3><span class="mw-headline" id="User_talk_pages">User talk pages</span></h3> 
      <p>User talk pages are rarely protected. However, protection may be applied if there is severe vandalism or abuse. Users whose talk pages are protected may wish to have an unprotected user talk subpage linked conspicuously from their main talk page to allow good-faith comments from users that the protection restricts editing from. </p>
      <p>A user's request to have their own talk page protected is not a sufficient rationale by itself to protect the page, although requests may be considered if a reason is provided. </p> 
      <h4><span class="mw-headline" id="Blocked_users">Blocked users</span></h4> 
      <p>Blocked users' user talk pages should not ordinarily be protected, as this interferes with the user's ability to contest their block through the normal process. It also prevents others from being able to use the talk page to communicate with the blocked editor. </p>
      <p>In extreme cases of abuse by the blocked user, such as abuse of the {{<a href="/wiki/Template:Unblock" title="Template:Unblock">unblock</a>}} template, re-blocking the user with talk page access removed should be preferred over applying protection to the page. If the user has been blocked and with the ability to edit their user talk page disabled, they should be informed of this in a block notice, subsequent notice, or message, and it should include information and instructions for appealing their block off-wiki, such as through the <a href="/wiki/Wikipedia:UTRS" class="mw-redirect" title="Wikipedia:UTRS">UTRS tool interface</a> or, as a last recourse, <a href="/wiki/Wikipedia:Appealing_a_block#Routes_to_unblock" title="Wikipedia:Appealing a block">the Arbitration Committee</a>. </p>
      <p>When required, protection should be implemented for only a brief period, not exceeding the duration of the block. </p>
      <p>Confirmed socks of registered users should be dealt with in accordance with <a href="/wiki/Wikipedia:Sockpuppetry" title="Wikipedia:Sockpuppetry">Wikipedia:Sockpuppetry</a>; their pages are not normally protected. </p> 
      <h3><span class="mw-headline" id="User_pages">User pages</span></h3> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:UPROT"></span><span id="WP:UPPROT"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcuts</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:UPROT&amp;redirect=no">WP:UPROT</a></span></li>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:UPPROT&amp;redirect=no">WP:UPPROT</a></span></li>
       </ul>
      </div> 
      <p>Base user pages (for example, the page User:Example, and not User:Example/subpage or User talk:Example) are automatically protected from creation or editing by unconfirmed accounts and anonymous IP users. An exception to this includes an unconfirmed registered account attempting to create or edit their own user page. IP editors and unconfirmed accounts are also unable to create or edit user pages that do not belong to a currently-registered account. This protection is enforced by <a href="/wiki/Special:AbuseFilter/803" title="Special:AbuseFilter/803">an edit filter</a>.<sup id="cite_ref-11" class="reference"><a href="#cite_note-11">[11]</a></sup> Users may opt-out of this protection by placing <code>{{<a href="/wiki/Template:Unlocked_userpage" title="Template:Unlocked userpage">unlocked userpage</a>}}</code> anywhere on their own user page. </p>
      <p><a href="/wiki/Wikipedia:User_pages" title="Wikipedia:User pages">User pages</a> and subpages within their own user space may be protected upon a request from the user, as long as a need exists. Pages within the user space should not be automatically or pre-emptively protected without good reason or cause.<sup id="cite_ref-12" class="reference"><a href="#cite_note-12">[12]</a></sup><sup id="cite_ref-13" class="reference"><a href="#cite_note-13">[13]</a></sup> Requests for protection specifically at uncommon levels (such as template protection) may be granted if the user has expressed a genuine and realistic need. </p>
      <p>When a filter is insufficient to stop user page vandalism, a user may choose to create a ".css" subpage (ex. User:Example/Userpage.css), copy all the contents of their user page onto the subpage, transclude the subpage by putting <span class="nowrap">{{User:Example/Userpage.css}}</span> on their user page, and then ask an administrator to fully protect their user page. Because user space pages that end in ".css", ".js", and ".json" are editable only by the user to which that user space belongs (and interface administrators), this will protect your user page from further vandalism. </p> 
      <h4><span class="mw-headline" id="Deceased_users">Deceased users</span></h4> 
      <div role="note" class="hatnote navigation-not-searchable">
       See also: <a href="/wiki/Wikipedia:Deceased_Wikipedians/Guidelines" title="Wikipedia:Deceased Wikipedians/Guidelines">Wikipedia:Deceased Wikipedians/Guidelines</a>
      </div> 
      <p>In the event of the confirmed death of a user, the user's user page (but not the user talk page) should be fully protected. </p> 
      <h3><span class="mw-headline" id="Protection_of_templates">Protection of templates</span></h3> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:PTPROT"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:PTPROT&amp;redirect=no">WP:PTPROT</a></span></li>
       </ul>
      </div> 
      <div role="note" class="hatnote navigation-not-searchable">
       See also: <a href="/wiki/Wikipedia:High-risk_templates" title="Wikipedia:High-risk templates">Wikipedia:High-risk templates</a> and <a href="/wiki/Wikipedia:Template_documentation" title="Wikipedia:Template documentation">Wikipedia:Template documentation</a>
      </div> 
      <p>Highly visible templates, which are used on an extremely large number of pages or substituted with great frequency, are often semi-, template-, or fully-protected based on the degree of visibility, type of use, content, etc. </p>
      <p>Protected templates should normally have the {{<a href="/wiki/Template:Documentation" title="Template:Documentation">documentation</a>}} template. It loads the unprotected <code>/doc</code> page, so that non-admins and IP-users can edit the documentation, categories and interwiki links. It also automatically adds {{<a href="/wiki/Template:Pp-template" title="Template:Pp-template">pp-template</a>}} to protected templates, which displays a small padlock in the top right corner and categorizes the template as protected. Only manually add <span class="nowrap">{{pp-template}}</span> to protected templates that don't use <span class="nowrap">{{documentation}}</span> (mostly the flag templates). </p>
      <p>Cascading protection should generally not be applied directly to templates, as it will not protect transclusions inside <code class="nowrap" style="">&lt;includeonly&gt;</code> tags or transclusions that depend on template parameters, but <i>will</i> protect the template's <a href="/wiki/Wikipedia:Template_documentation" title="Wikipedia:Template documentation">documentation subpage</a>. Instead, consider any of the following: </p> 
      <ul>
       <li>If the set of subtemplates is static (even if large), protect them using normal protection mechanisms.</li> 
       <li>If the set of subtemplates is unbounded, use <a href="/wiki/MediaWiki:Titleblacklist" title="MediaWiki:Titleblacklist">MediaWiki:Titleblacklist</a> to protect all subtemplates using a particular naming format (as is done for <a href="/wiki/Wikipedia:Editnotice" title="Wikipedia:Editnotice">editnotice templates</a> and subtemplates of <a href="/wiki/Template:TFA_title" title="Template:TFA title">Template:TFA title</a>).</li>
      </ul> 
      <p>Note: All <a href="/wiki/Wikipedia:Editnotice" title="Wikipedia:Editnotice">editnotice templates</a> (except those in userspace) are already protected via <a href="/wiki/MediaWiki:Titleblacklist" title="MediaWiki:Titleblacklist">MediaWiki:Titleblacklist</a>. They can be edited by admins, <a href="/wiki/Wikipedia:Template_editor" title="Wikipedia:Template editor">template editors</a> and <a href="/wiki/Wikipedia:Page_mover" title="Wikipedia:Page mover">page movers</a> only. </p>

      <p>Sandboxes should not ordinarily be protected since their purpose is to let new users test and experiment with <a href="/wiki/Help:Wiki_markup" class="mw-redirect" title="Help:Wiki markup">wiki syntax</a>. Most sandboxes are automatically cleaned every 12 hours, although they are frequently overwritten by other testing users. The <a href="/wiki/Wikipedia:Sandbox" title="Wikipedia:Sandbox">Wikipedia:Sandbox</a> is cleaned every hour. Those who use sandboxes for malicious purposes, or to violate policies such as <a href="/wiki/Wikipedia:No_personal_attacks" title="Wikipedia:No personal attacks">no personal attacks</a>, <a href="/wiki/Wikipedia:Civility" title="Wikipedia:Civility">civility</a>, or <a href="/wiki/Wikipedia:Copyrights" title="Wikipedia:Copyrights">copyrights</a>, should instead be warned and/or blocked. </p>
