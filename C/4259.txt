       Wikipedia's guideline on user pages
      </div>
      <div role="note" class="hatnote navigation-not-searchable">
       "WP:UP" redirects here. For the File Upload Wizard, see <a href="/wiki/Wikipedia:FUW" class="mw-redirect" title="Wikipedia:FUW">WP:FUW</a>.

      <div role="note" class="hatnote navigation-not-searchable">
       This page is a <a href="/wiki/Wikipedia:Policies_and_guidelines" title="Wikipedia:Policies and guidelines">guideline</a> for user and user talk pages and should not be confused with <b><a href="/wiki/Special:MyPage" title="Special:MyPage">your own user page</a></b>. 
       <br>See also <a href="/wiki/Wikipedia:User_page_design_center" title="Wikipedia:User page design center">user page design center</a>, our <a href="/wiki/Wikipedia:Username_policy" title="Wikipedia:Username policy">user name policy</a>, and more about <a href="/wiki/Wikipedia:Wikipedians" title="Wikipedia:Wikipedians">Wikipedia users</a>.
      </div>

          <div style="font-size: 90%;">
            It is a generally accepted standard that editors should attempt to follow, though it is best treated with <a href="/wiki/Wikipedia:What_%22Ignore_all_rules%22_means#Use_common_sense" title="Wikipedia:What &quot;Ignore all rules&quot; means">common sense</a>, and <a href="/wiki/Wikipedia:Ignore_all_rules" title="Wikipedia:Ignore all rules">occasional exceptions</a> may apply. Any substantive edit to this page should reflect <a href="/wiki/Wikipedia:Consensus" title="Wikipedia:Consensus">consensus</a>.          <td class="mbox-text"><b>This guideline in a nutshell:</b> <p>User pages are for communication and collaboration. While considerable leeway is allowed in personalizing and managing your user pages, they are community project pages, <a href="/wiki/Wikipedia:What_Wikipedia_is_not#WEBHOST" title="Wikipedia:What Wikipedia is not">not a personal website, blog, social networking medium</a>, or <a href="/wiki/Wikipedia:FAKEARTICLE" class="mw-redirect" title="Wikipedia:FAKEARTICLE">a Wikipedia article</a>. They should be used to better participate in the community, and not used to excess for unrelated purposes nor to bring the project into disrepute.</p></td>

      <p><b>User pages</b> are <a href="/wiki/Wikipedia:Administration" title="Wikipedia:Administration">administration pages</a> in the User and User talk <a href="/wiki/Wikipedia:Namespace" title="Wikipedia:Namespace">namespaces</a> that are useful for organizing and aiding the work users do on <a href="/wiki/Wikipedia" title="Wikipedia">Wikipedia</a>, as well as facilitating interaction and sharing between users. User pages are mainly for interpersonal discussion, notices, testing and drafts (see: <a href="/wiki/Wikipedia:About_the_Sandbox" class="mw-redirect" title="Wikipedia:About the Sandbox">Sandboxes</a>), and, if desired, limited autobiographical and personal content. </p>
      <p>User pages are available to Wikipedia users personally for purposes compatible with the Wikipedia project and acceptable to the community; <a href="/wiki/Wikipedia:What_Wikipedia_is_not#BLOG" title="Wikipedia:What Wikipedia is not">Wikipedia is not a blog, webspace provider, or social networking site</a>. Wikipedia <a href="/wiki/Wikipedia:List_of_policies#Content" title="Wikipedia:List of policies">policies concerning the content of pages</a> can and generally do apply to user pages, and users must observe these policies. Users believed to be in violation of these policies should first be advised on their talk page using <code>{{<a href="/wiki/Template:Uw-userpage" title="Template:Uw-userpage">subst:uw-userpage</a>}}</code> when immediate action is not otherwise necessary. </p>

      <h2><span class="mw-headline" id="Terminology_and_page_locations"><span id="User_pages_and_user_space"></span>Terminology and page locations</span></h2> 
      <div role="note" class="hatnote navigation-not-searchable">
       See also: <a href="/wiki/Wikipedia:Username_policy" title="Wikipedia:Username policy">Wikipedia:Username policy</a>, <a href="/wiki/Wikipedia:How_to_archive_a_talk_page" class="mw-redirect" title="Wikipedia:How to archive a talk page">Wikipedia:How to archive a talk page</a>, and <a href="/wiki/Wikipedia:Wikipedians" title="Wikipedia:Wikipedians">Wikipedia:Wikipedians</a>
      </div> 
      <p><i>Your</i> in this context means associated with you, not belonging to you. </p> 
      <dl>
       <dt>
        User page
       </dt> 
       <dd>
        Your user page has a name like this: User:<i>Example</i>. (<a href="/wiki/Special:MyPage" title="Special:MyPage">This link is to yours</a>.) Its normal use is to give basic information, <i>if you wish</i>, about yourself or your Wikimedia-related activities. If you prefer to put nothing here, then you can <a href="/wiki/Wikipedia:Redirect" title="Wikipedia:Redirect">redirect</a> it to your user talk page for the convenience of other editors. You may also wish to create a <a href="/wiki/Wikipedia:GUP" class="mw-redirect" title="Wikipedia:GUP">global user page</a> that will display on all Wikimedia projects where you have not created a local user page.
       </dd> 
       <dt>
        User talk page
       </dt> 
       <dd>
        Your user talk page (sometimes abbreviated to "your talk page" or "your user talk") has a name like this: User talk:<i>Example</i>. (<a href="/wiki/Special:MyTalk" title="Special:MyTalk">This link is to yours</a>.) Its normal use is for messages from, and discussion with, other editors. For more information see <a href="/wiki/Help:Using_talk_pages" class="mw-redirect" title="Help:Using talk pages">Help:Using talk pages</a>.
       </dd> 
       <dt>
        Subpages<span class="anchor" id="SUB"></span><span class="anchor" id="What_about_user_subpages?"></span>
       </dt> 
       <dd>
        <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
        <div class="module-shortcutanchordiv">
         <span id="WP:UP#SUB"></span><span id="WP:USERSUBPAGE"></span><span id="WP:CREATEUSER#SUB"></span>
        </div>
        <div class="module-shortcutboxplain plainlist noprint" role="note">
         <div class="module-shortcutlist">
          <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcuts</a>
         </div>
         <ul>
          <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:UP&amp;redirect=no#SUB">WP:UP#SUB</a></span></li>
          <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:USERSUBPAGE&amp;redirect=no">WP:USERSUBPAGE</a></span></li>
          <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:CREATEUSER&amp;redirect=no#SUB">WP:CREATEUSER#SUB</a></span></li>
         </ul>
        </div> <a href="/wiki/Wikipedia:Subpages" title="Wikipedia:Subpages">Subpages</a> in user space can be used to <a href="#What_may_I_have_in_my_user_pages?">store sandboxes, essays about Wikipedia, and drafts of Wikipedia articles, among other things</a>. <a href="#Creating_a_subpage">You can create these subpages yourself</a>.
       </dd> 
       <dt>
        User pages or user space
       </dt> 
       <dd>
        All of these pages are your user pages or user space. While you <a href="/wiki/Wikipedia:OWN" class="mw-redirect" title="Wikipedia:OWN">do not "own" them</a>, by custom you may manage them as you wish, so long as you do so reasonably and within these guidelines.
       </dd> 
       <dd>
        You also have subpages ending <span class="nowrap">in <tt>.js</tt></span> <span class="nowrap">and <tt>.css</tt></span> to store any <a href="/wiki/Wikipedia:User_scripts" title="Wikipedia:User scripts">user scripts</a> and <a href="/wiki/Wikipedia:CSS" class="mw-redirect" title="Wikipedia:CSS">skin customizations</a> that you may wish to have when you edit Wikipedia. Only you and interface administrators can edit such pages, although anyone can view them.
       </dd>
      </dl>

      <p>You can create subpages of your <i>User page</i> and your <i>Talk</i> page. To create a subpage write the following text into any editable text area: </p> 
      <pre>User:Your_Wikipedia_Name/subpage
</pre> 
      <p>then copy and paste this text into the Wikipedia search box and press ENTER. This will bring you to a page with the title <b>User:Your_Wikipedia_Name/subpage</b>. Now click the CREATE-button next to the Wikipedia search box and your new subpage will be created for you. Click on EDIT, enter a few test words and save the new page. You will notice that different from your <i>User page</i> a subpage contains a backlink to your <i>User page</i> looking like this: </p> 
      <pre>&lt;User:Your_Wikipedia_Name
</pre> 
      <p>Clicking on the backlink will bring you to your <i>User page</i>. But, contrary to what you might expect, no new tab has been created for "subpages", for example, containing a list of all of your subpages; everything on your <i>User page</i> is unchanged. So how do you navigate to your subpage? As a method of last resort, you can always go back to your subpage by adding the title of your subpage to the url of your <i>User page</i>: </p> 
      <pre>If your home page url is:
https://en.wikipedia.org/wiki/User:Your_Wikipedia_Name
just add the name of your subpage:

              https://en.wikipedia.org/wiki/User:Your_Wikipedia_Name/subpage

we used "subpage" as our first subpage name, and therefore the above url will get us to our newly created subpage.
</pre> 
      <p>There is, of course, an easier method, but it has to be done manually. Copy and modify the following text and put it on your <i>User page</i>: </p> 
      <pre>[[Special:Prefixindex/User:Your_Wikipedia_Name]]
</pre> 
      <p>After saving your <i>User page</i>, clicking on this link will provide you with a list of all pages, containing the string "User:Your_Wikipedia_Name", which in our test case includes the sandbox, the <i>User page</i> and, of course, the newly created subpage. </p>

      <p>There is no fixed use for user pages, except that usually one's user page has something about oneself, and one's talk page is used for messaging. Provided other users can quickly and easily find the pages they need, users may, within reason, freely organize their user pages as they choose. </p>
      <p>Users may include a user page notice on their own user pages, user talk pages, or both. Placing the template <code>{{<a href="/wiki/Template:User_page" title="Template:User page">User page</a>}}</code> at the start of a user page clearly identifies the nature of the page for readers, and also helps if people find the labeled page in <a href="/wiki/Wikipedia:Mirrors_and_forks" title="Wikipedia:Mirrors and forks">copies of Wikipedia elsewhere</a> (more about this below) and want to locate the original. </p>
      <p>Contributions can also be given a wider license  for example releasing them into the <a href="/wiki/Public_domain" title="Public domain">public domain</a> or <a href="/wiki/Wikipedia:Multi-licensing" title="Wikipedia:Multi-licensing">multi-licensing</a> them  by putting a notice to this effect on one's user page, or on a subpage linked from it. Note that it is not possible to give them <b>narrower</b> licensing: <b>all</b> edits on Wikipedia, including all userspace edits, are licensed for use under the <a href="/wiki/Creative_Commons_license" title="Creative Commons license">Creative Commons Attribution/Share-Alike License</a> and in most cases the <a href="/wiki/GNU_Free_Documentation_License" title="GNU Free Documentation License">GNU Free Documentation License</a> as part of Wikipedia. </p>
      <p>User pages may be <a href="/wiki/Wikipedia:Mirrors_and_forks" title="Wikipedia:Mirrors and forks">mirrored</a> by other sites. If there is material you do not want copied, reposted, or reused, <b>do not post it on the site</b>. </p>
      <p>Certain kinds of material must not linger indefinitely in user space; see <a href="#NOT">below</a> for details. </p>

      <p>You are also welcome to include a simple link to your personal home page, although you should not surround it with any promotional language. However, if a link to your home page is the <i>only</i> thing on your userpage, this may be seen as an attempt at self-promotion. </p>
      <p>User pages are also used for administrative purposes, to make users aware of <a href="/wiki/Wikipedia:Blocking_policy" title="Wikipedia:Blocking policy">blocks</a>, warnings, or other sanctions if they happen, and to notify of matters that may affect articles you have worked on or editorial issues you have been involved with. Others may also edit your user pages, for instance awarding you a <a href="/wiki/Wikipedia:Barnstars" title="Wikipedia:Barnstars">barnstar</a> or leaving notes and images for you, or adding comments and questions. Although you have wide leeway to edit your user pages, a few of these matters should not be removed (see below). </p> 
      <h3><span class="mw-headline" id="Userspace_and_mainspace">Userspace and mainspace</span></h3> 
      <p>Details about yourself should <b>not</b> normally go in the main encyclopedia <a href="/wiki/Wikipedia:Namespace" title="Wikipedia:Namespace">namespace</a> (reserved for encyclopedia articles <b>only</b>), and encyclopedia articles should <b>never</b> link to or <a href="/wiki/Wikipedia:Transclusion" class="mw-redirect" title="Wikipedia:Transclusion">transclude</a> any userspace pages. </p>
      <p>In the rare case that you or something closely connected to you may have an article in the encyclopedia, that is always treated as <b>completely separate</b> from you <b>as an editor</b>. You should very carefully read the guidance on <a href="/wiki/Wikipedia:COI" class="mw-redirect" title="Wikipedia:COI">conflict of interest</a> and generally <b>avoid</b> editing about yourself or matters closely related to you in any article. </p>
      <p>If you would like to draft a new article, <a href="/wiki/Help:Userspace_draft" title="Help:Userspace draft">Help:Userspace draft</a> provides a standard template and useful guidance to help you create a draft in your userspace, and the <a href="/wiki/Wikipedia:Article_wizard" title="Wikipedia:Article wizard">article wizard</a> can walk you through all stages of creating an article with the option to save as a userspace draft too. You can use the template {{<a href="/wiki/Template:Userspace_draft" title="Template:Userspace draft">userspace draft</a>}} to tag a userspace draft if it is not automatically done for you. </p> 
      <h3><span class="mw-headline" id="Personal_and_privacy-breaching_material">Personal and privacy-breaching material</span></h3> 
      <p>Some people add personal information such as contact details (email, instant messaging, etc.), a photograph, their real name, their location, information about their areas of expertise and interest, likes and dislikes, etc. Once added, this information is unlikely to ever become private again. It could be copied elsewhere or even used to <a href="/wiki/Wikipedia:HARASS" class="mw-redirect" title="Wikipedia:HARASS">harass</a> you in the future. <b>You are cautioned to think carefully before adding non-public information to your user page because you are unlikely to be able to retract it later, even if you change your mind</b>. </p>
      <p>Privacy-breaching non-public material, whether added by yourself or others, <b>may be removed</b> from any page upon request, either by administrators or (unless impractical) by purging from the page history and any logs by <a href="/wiki/Wikipedia:SIGHT" class="mw-redirect" title="Wikipedia:SIGHT">oversighters</a> (see <a href="/wiki/Wikipedia:RFO" class="mw-redirect" title="Wikipedia:RFO">requests for oversight</a>). </p>

      <h2><span id="What_may_I_not_have_in_my_user_pages.3F"></span><span class="mw-headline" id="What_may_I_not_have_in_my_user_pages?"><span id="NOT"></span><span id="What_may_I_not_have_on_my_user_page?"></span>What may I <i>not</i> have in my user pages?</span></h2>

      <p>Generally, <b>you should avoid substantial content on your user page that is unrelated to Wikipedia.</b> <a href="/wiki/Wikipedia:What_Wikipedia_is_not#Wikipedia_is_not_a_blog,_webspace_provider,_social_network,_or_memorial_site" title="Wikipedia:What Wikipedia is not">Wikipedia is not</a> a general hosting service, so your user page is not a personal website. Your user page is about you as a Wikipedian, and pages in your user space should be used as part of your efforts to contribute to the project. </p>
      <p>In addition, there is broad agreement that <b>you may not include in your user space material that is likely to bring the project into disrepute</b>, or which is likely to give widespread offense (<i>e.g.</i> <a href="/wiki/Wikipedia:Child_protection" title="Wikipedia:Child protection">pro-pedophilia advocacy</a>). Whether serious or <a href="/wiki/Internet_troll" title="Internet troll">trolling</a>, "<a href="/wiki/Wikipedia:SOAP" class="mw-redirect" title="Wikipedia:SOAP">Wikipedia is not a soapbox</a>" is usually interpreted as applying to user space as well as the encyclopedia itself, and "<a href="/wiki/Wikipedia:NOT#CENSORED" class="mw-redirect" title="Wikipedia:NOT">Wikipedia is not censored</a>" relates to article pages and images; in other namespaces there <b>are</b> restrictions aimed at ensuring relevance, value, and non-disruption to the community. You do have more latitude in user space than elsewhere, but <a href="/wiki/Wikipedia:Don%27t_be_inconsiderate" title="Wikipedia:Don't be inconsiderate">don't be inconsiderate</a>. <i>Extremely offensive material may be removed on sight by any editor.</i> </p>
      <p>The Wikipedia community is generally tolerant and offers fairly wide latitude in applying these guidelines to regular participants. Particularly, community-building activities that are not strictly "on topic" may be allowed, especially when initiated by committed Wikipedians with good edit histories. At their best, such activities help us to build the community, and this helps to build the encyclopedia. But at the same time, if user page activity becomes disruptive to the community or gets in the way of the task of building an encyclopedia, it must be modified to prevent disruption. </p>

      <h3><span class="mw-headline" id="Advocacy_or_support_of_grossly_improper_behaviors_with_no_project_benefit"><span id="Statements_of_violence"></span>Advocacy or support of grossly improper behaviors with no project benefit</span></h3> 
      <p>Statements or pages that seem to advocate, encourage, or condone these behaviors:<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup> <a href="/wiki/Wikipedia:VANDAL" class="mw-redirect" title="Wikipedia:VANDAL">vandalism</a>, <a href="/wiki/Wikipedia:COPYVIO" class="mw-redirect" title="Wikipedia:COPYVIO">copyright violation</a>, <a href="/wiki/Wikipedia:EW" class="mw-redirect" title="Wikipedia:EW">edit warring</a>, <a href="/wiki/Wikipedia:HARASS" class="mw-redirect" title="Wikipedia:HARASS">harassment</a>, <a href="/wiki/Wikipedia:Harassment#Posting_of_personal_information" title="Wikipedia:Harassment">privacy breach</a>, <a href="/wiki/Wikipedia:Libel" title="Wikipedia:Libel">defamation</a>, and <a href="/wiki/Wikipedia:Responding_to_threats_of_harm" title="Wikipedia:Responding to threats of harm">acts of violence</a>. ("Acts of violence" includes all forms of violence but does <i>not</i> include mere statements of support for controversial groups or regimes that some may interpret as an encouragement of violence.) </p>
      <p>These may be removed, redacted or <a href="/wiki/Template:Collapse_top" title="Template:Collapse top">collapsed</a> by any user to avoid the appearance of acceptability for Wikipedia, and existing <a href="/wiki/Wikipedia:CSD" class="mw-redirect" title="Wikipedia:CSD">speedy deletion criteria</a> may apply. To preserve traditional leeway over userspace, other kinds of material should be handled as described <a href="#Handling_inappropriate_content">below</a> unless otherwise agreed by consensus. </p> 
      <h3><span id="Categories.2C_templates_that_add_categories.2C_and_redirects"></span><span class="mw-headline" id="Categories,_templates_that_add_categories,_and_redirects">Categories, templates that add categories, and redirects</span></h3> 
      <div role="note" class="hatnote navigation-not-searchable">
       Further information: <a href="/wiki/Wikipedia:Categorisation#Categorizing_user_pages" class="mw-redirect" title="Wikipedia:Categorisation">WP:Categorisation §&nbsp;Categorizing user pages</a>
      </div> 
      <p>Do <b>not</b> put your userpage or subpages, including draft articles, into <a href="/wiki/Wikipedia:PROJCATS" class="mw-redirect" title="Wikipedia:PROJCATS">content categories</a>. Userpages and subpages may be placed in appropriate administrative categories, such as <a href="/wiki/Category:User_essays" title="Category:User essays">Category:User essays</a>. </p>
      <p>Especially note that templates often add categories themselves. You can prevent this while the article is being drafted, by putting <b><tt>tlx|</tt></b> between the {{ and the template name, like this: <b><code>{{<a href="/wiki/Template:Tlx" class="mw-redirect" title="Template:Tlx">tlx</a>|stub|any parameters}}</code></b>. </p>
      <p>You can also force a portion of text to be ignored by adding <code>&lt;!--</code> in front of it and <code>--&gt;</code> after it, or by adding a colon before "Category", like this: <code>[[:Category:Bridges]]</code> to force a category link to act like a plain wikilink. </p>
      <p>User talk pages <b>should not redirect to anything other than</b> the talk page of another account controlled by the same user. However, redirects from userspace subpages to mainspace are common and acceptable. Soft redirects are allowed on userpages. </p> 
      <h3><span class="mw-headline" id="User_pages_that_look_like_project_pages">User pages that look like project pages</span></h3> 
      <p>Userspace is also not a substitute for project space (<i>Wikipedia:...</i>), nor should a userspace page be used as primary documentation for any Wikipedia policy, guideline, practice, or established concept. If your user page related to the project becomes widely used or linked in project space, or has functional use similar to a project page, consider moving it into project space or merging it with other similar pages already existing there. </p>

       <li>Userspace drafts which do not meet article content standards <b>should not</b> be moved to mainspace in order to seek deletion.</li> 
       <li>In case a userspace draft was moved to mainspace but is found not fit to be in mainspace, it <b>must be</b> returned to the previous location in userspace.</li>

      <h3><span class="mw-headline" id="Non-free_files"><span id="Non-free_images"></span>Non-free files</span></h3> 
      <p>Do <b>not</b> include non-free files (<i>copyrighted files lacking a free content license</i>) on your user page or on any subpage thereof, per the <a href="/wiki/Wikipedia:Non-free_content_criteria" title="Wikipedia:Non-free content criteria">non-free content policy</a>. Non-free files found on user or user talk pages will be removed without warning and, if unused in a Wikipedia article, will be <a href="/wiki/Wikipedia:CSD#F5" class="mw-redirect" title="Wikipedia:CSD">deleted entirely</a>. Links to non-free files are acceptableplace a colon before the word "File" as in <code>[[:File:Example.jpg]]</code>. </p> 
      <h3><span class="mw-headline" id="Images_that_would_bring_the_project_into_disrepute">Images that would bring the project into disrepute</span></h3> 
      <p>There is broad consensus that you should not have any image in your userspace that would bring the project into disrepute and you may be asked to remove such images. Content clearly intended as sexually provocative (images and in some cases text) or to <a href="/wiki/Shock_image" class="mw-redirect" title="Shock image">cause distress and shock</a> that appears to have little or no project benefit or using Wikipedia only as a <a href="/wiki/Wikipedia:NOT#WEBHOST" class="mw-redirect" title="Wikipedia:NOT">web host or personal pages</a> or for <a href="/wiki/Wikipedia:NOT#SOAPBOX" class="mw-redirect" title="Wikipedia:NOT">advocacy</a>, may be removed by any user (or deleted), subject to appeal at <a href="/wiki/Wikipedia:DRV" class="mw-redirect" title="Wikipedia:DRV">deletion review</a>.<sup id="cite_ref-7" class="reference"><a href="#cite_note-7">[7]</a></sup> Context should be taken into account. Simple personal disclosures of a non-provocative nature (such as userboxes or statements about sexuality and relationship status) are unaffected. </p> 
      <h3><span class="mw-headline" id="Copyright_violations">Copyright violations</span></h3> 
      <p>The same rules for <a href="/wiki/Wikipedia:COPYVIO" class="mw-redirect" title="Wikipedia:COPYVIO">copyright</a> apply on userpages as in article space. Text must be either freely licensed or out of copyright; otherwise only a short quote can be used. If you use text from another source on your userpage, it should still be credited to the author, whether or not it is currently copyrighted. </p>

        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:SMI&amp;redirect=no">WP:SMI</a></span></li>
       </ul>
      </div> 
      <p>The Wikipedia community <a href="/wiki/Wikipedia_talk:User_pages/UI_spoofing" title="Wikipedia talk:User pages/UI spoofing">strongly discourages</a> simulating the <a href="/wiki/MediaWiki" title="MediaWiki">MediaWiki</a> interface, except on the rare occasion when it is necessary for testing purposes. Fake <a href="/wiki/Wikipedia:User_pages#User_talk_notification" title="Wikipedia:User pages">user talk notification banners</a> that mislead readers into thinking they have new messages are also prohibited.<sup id="cite_ref-8" class="reference"><a href="#cite_note-8">[8]</a></sup> </p>
      <p>CSS and other formatting codes that disrupt the MediaWiki interface, for example by preventing important links or controls from being easily seen or used, making text on the page hard to read or unreadable (other than by way of commenting out), or replacing the expected interface with a disruptive simulation, may be removed or remedied by any user. Inappropriate internal or external links that unexpectedly direct the reader to unreasonable locations or violate <a href="/wiki/Wikipedia:EL" class="mw-redirect" title="Wikipedia:EL">prohibitions on linking</a> may also be removed or remedied by any user. Text, images, and non-disruptive formatting should be left as intact as possible. Users of such code should consider possible disruption to <i>other</i> <a href="/wiki/Wikipedia:SKIN" class="mw-redirect" title="Wikipedia:SKIN">skins</a>, <a href="/wiki/Wikipedia:DIFF" class="mw-redirect" title="Wikipedia:DIFF">diffs</a>, and <a href="/wiki/Help:Page_history" title="Help:Page history">old revisions</a>. </p> 
      <h2><span class="mw-headline" id="Ownership_and_editing_of_user_pages"><span id="OWN"></span><span id="Ownership_and_editing_of_pages_in_the_user_space"></span>Ownership and editing of user pages</span></h2> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">

      <p>Traditionally, Wikipedia offers wide latitude to users to manage their user space as they see fit. However, pages in user space belong to the wider community. They are <b>not</b> a personal homepage, and do not belong to the user. They are part of Wikipedia, and exist to make collaboration among editors easier. </p>
      <p>Bots and other users may edit pages in your user space or leave messages for you, though by convention others will not usually edit your user page itself, other than (rarely) to address significant concerns or place project-related tags. Material that clearly does not somehow further the goals of the project may be removed (see below), as may edits from <a href="/wiki/Wikipedia:BAN" class="mw-redirect" title="Wikipedia:BAN">banned</a> users. Most community policies including <a href="/wiki/Wikipedia:No_personal_attacks" title="Wikipedia:No personal attacks">No personal attacks</a> and <a href="/wiki/Wikipedia:BLP" class="mw-redirect" title="Wikipedia:BLP">Biographies of living persons</a> will apply to your user space, just as elsewhere. (Purely content policies such as <a href="/wiki/Wikipedia:Original_Research" class="mw-redirect" title="Wikipedia:Original Research">original research</a> and <a href="/wiki/Wikipedia:NPOV" class="mw-redirect" title="Wikipedia:NPOV">neutral point of view</a> generally do not apply unless the material is moved into mainspace.) </p>
      <p>As with all other edits, user space contributions are irrevocably licensed for copying and reuse under the <a href="/wiki/Wikipedia:CC-BY-SA" class="mw-redirect" title="Wikipedia:CC-BY-SA">Creative Commons Attribution/Share-Alike License</a> and <a href="/wiki/Wikipedia:GFDL" class="mw-disambig" title="Wikipedia:GFDL">GNU Free documentation license</a>. </p>
      <p>Finally, a few specific notices and tags, if placed, may not be moved to a less visible subpage or deleted without discussion. </p>

      <p>Policy does not prohibit users, whether registered or unregistered, from removing comments from <strong>their own</strong> talk pages, although <a href="/wiki/Help:Archiving_a_talk_page" title="Help:Archiving a talk page">archiving</a> is preferred. If a user removes material from their talk page, it is normally taken to mean that the user <strong>has read</strong> and <strong>is aware</strong> of its contents. There is no need to keep them on display, and usually users should not be forced to do so. It is often best to simply let the matter rest if the issues stop. If they do not, or they recur, then any record of past warnings and discussions can be found in the <a href="/wiki/Help:Page_history" title="Help:Page history">page history</a> if ever needed, and these <a href="/wiki/Help:Diff" title="Help:Diff">diffs</a> are just as good evidence of previous matters. </p>

       <li>Declined <a href="/wiki/Wikipedia:Appealing_a_block" title="Wikipedia:Appealing a block">unblock requests</a> regarding a currently active block.

        </div></li>

      <h3><span id="Editing_of_other_editors.27_user_and_user_talk_pages"></span><span class="mw-headline" id="Editing_of_other_editors'_user_and_user_talk_pages">Editing of other editors' user and user talk pages</span></h3> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:NOBAN"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">

       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:NOBAN&amp;redirect=no">WP:NOBAN</a></span></li>

      </div> 
      <p>In general, one should avoid substantially editing another's user and user talk pages, except when it is likely edits are expected and/or will be helpful. If unsure, ask. If an editor asks you not to edit their user pages, such requests should, within reason, be respected. However, editors should not make such requests lightly, especially concerning their talk pages, as doing so can impede the ordinary communication which is important for the improvement and smooth running of the project. Also, a user cannot avoid administrator attention or notices and communications that policies or guidelines require to be posted merely by demanding their talk page not be posted to. Still, repeatedly posting on a user's page after being asked not to, without good reason, may be seen as <a href="/wiki/Wikipedia:HARASSMENT" class="mw-redirect" title="Wikipedia:HARASSMENT">harassment</a> or similar kind of disruptive behavior. When in doubt, ask for help from another experienced editor or uninvolved administrator. See also <a href="/wiki/Wikipedia:DR" class="mw-redirect" title="Wikipedia:DR">WP:DR</a> for tips on resolving disputes. </p> 
      <div role="note" class="hatnote navigation-not-searchable">
       Further information on this area is at <a href="/wiki/Wikipedia:Talk_page_guidelines" title="Wikipedia:Talk page guidelines">Wikipedia:Talk page guidelines</a>, in particular the sections on <a href="/wiki/Wikipedia:TPG#YES" class="mw-redirect" title="Wikipedia:TPG">appropriate conduct</a>, <a href="/wiki/Wikipedia:TPG#NO" class="mw-redirect" title="Wikipedia:TPG">inappropriate conduct</a>, and norms related to <a href="/wiki/Wikipedia:TPG#EDIT" class="mw-redirect" title="Wikipedia:TPG">editing your own and others' comments</a>. These apply to user talk pages as well as all other discussion pages on Wikipedia.
      </div>

      <h4><span class="mw-headline" id="On_your_user_pages">On your user pages</span></h4> 
      <p>If the community lets you know that they would rather you delete some content from your user space, you should consider doing sosuch content is only permitted with the consent of the community. Alternatively, you could move the content to another site, and link to it. </p>
      <p>Although other editors will aim to respect your user space, if corrective action is needed and not undertaken the inappropriate content will eventually be removed, either by editing the page (if only part is inappropriate), by redirecting the page to your main user page (if entirely inappropriate), by community discussion at <a href="/wiki/Wikipedia:Miscellany_for_deletion" title="Wikipedia:Miscellany for deletion">Wikipedia:Miscellany for deletion</a>, or possibly by the application of <a href="/wiki/Wikipedia:CSD#U5._Blatant_misuse_of_Wikipedia_as_a_web_host" class="mw-redirect" title="Wikipedia:CSD">WP:CSD#U5. Blatant misuse of Wikipedia as a web host</a>. Inappropriate content may be removed from any page in your user space, including your user talk page. </p> 
      <h4><span id="On_others.27_user_pages"></span><span class="mw-headline" id="On_others'_user_pages">On others' user pages</span></h4> 
      <p>The best option if there is a concern with a user's page is to draw their attention to the matter via their talk page and let them edit it themselves, if they are agreeable. In some cases a more experienced editor may make substantive edits to another user's user space, in which case that editor should leave a note explaining why this was done. This should not be done for trivial reasons. If the user does not agree, or does not effectively remedy the concerns, or the matter is unsure or controversial, then other steps in this section can be taken including <a href="/wiki/Wikipedia:UNINVOLVED" class="mw-redirect" title="Wikipedia:UNINVOLVED">uninvolved user</a> opinions or proposing the page for deletion. </p>
      <p>If the material must be addressed urgently (for example, unambiguous copyright, attack, defamation, or <a href="/wiki/Wikipedia:BLP" class="mw-redirect" title="Wikipedia:BLP">BLP</a> reasons, etc.), the user appears inactive, the edit appears unlikely to cause problems, and you are quite sure the material is inappropriate, then <a href="/wiki/Wikipedia:BRD" class="mw-redirect" title="Wikipedia:BRD">remove or fix</a> the problem material minimally and leave a note explaining what you have done, why you have done so, and inviting the user to discuss if needed. If the entire page is inappropriate, consider <strong>blanking</strong> it, or <strong>redirecting</strong> the subpage to the userpage, or to the most relevant existing mainspace or project space page. </p>
      <p>Unsuitable pages, media and images in userspace may also be nominated for deletion or (if appropriate) <a href="/wiki/Wikipedia:Criteria_for_speedy_deletion" title="Wikipedia:Criteria for speedy deletion">speedy deleted</a>, but special care should be taken as the user may be expecting leeway and take it personally, and there are a few exceptions. Users with a strong editing record and/or most of their contribution edits outside their user space should be given a little more leeway in this regard than users whose edits consist <a href="/wiki/Wikipedia:SPA" class="mw-redirect" title="Wikipedia:SPA">solely or mostly of user space edits or promotional-style activity</a>. See <a href="#DELETE">Deleting user pages</a> below. </p> 
      <h3><span class="mw-headline" id="User_pages_and_leaving_Wikipedia">User pages and leaving Wikipedia</span></h3> 
      <p>When a user leaves Wikipedia, their user and user talk pages are usually unaffected and may be edited again at any future time. Some users place the {{<a href="/wiki/Template:Retired" title="Template:Retired">retired</a>}} template on their user and talk page to let others know that they are away for an extended period or permanently. A user may blank their user and user talk pages (i.e. overwriting with a blank page) provided non-removable notices (if any) are left intact. </p> 
      <h4><span class="mw-headline" id="Courtesy_vanishing">Courtesy vanishing</span></h4> 
      <p>Wikipedia's community traditionally offers a <a href="/wiki/Wikipedia:RTV" class="mw-redirect" title="Wikipedia:RTV">courtesy vanishing</a> to users who are <strong>permanently departing Wikipedia and will sever all ties with the site</strong>. </p>
      <p>Summary of key points: While some help can be given, it is <strong>not</strong> possible for your edits to be removed entirely and account deletion would potentially violate <a href="/wiki/Wikipedia:LICENSE" class="mw-redirect" title="Wikipedia:LICENSE">copyrights</a> by allowing for inaccurate attribution and authorship claims. Certain important templates may need to be retained on user and user talk pages. Also pages that may be of value to the wider community or whose deletion is opposed by other users might be undeleted during a <a href="/wiki/Wikipedia:Miscellany_for_deletion" title="Wikipedia:Miscellany for deletion">deletion discussion</a>. Pages remain licensed for reuse even after deletion, and may occasionally be cited or deemed to contain useful content. If a "vanished" user returns, old pages associated with them may be undeleted or unblanked, and could be linked to any new account they create and disclosed at <a href="/wiki/Wikipedia:RFA" class="mw-redirect" title="Wikipedia:RFA">RFA</a>; if this would be a problem, consult <a href="/wiki/Wikipedia:AC" class="mw-redirect" title="Wikipedia:AC">ArbCom</a> by email <em>beforehand</em>. Of course the return of users in good standing or reformed "problem users" is welcomed if they happen to change their mind. </p>
      <p>Users who have left Wikipedia may be added to <a href="/wiki/Wikipedia:Missing_Wikipedians" title="Wikipedia:Missing Wikipedians">Wikipedia:Missing Wikipedians</a> after a certain amount of time, usually one year without an edit. </p> 
      <h2><span class="mw-headline" id="Protection_of_user_pages"><span id="PROTECT">Protection of user pages</span></span></h2> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:UP#PROTECT"></span>

       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:UP&amp;redirect=no#PROTECT">WP:UP#PROTECT</a></span></li>
       </ul>
      </div> 
      <div role="note" class="hatnote navigation-not-searchable">
       See also: <a href="/wiki/Wikipedia:Protection_policy#User_pages" title="Wikipedia:Protection policy">Wikipedia:Protection policy §&nbsp;User pages</a>
      </div> 
      <p>As with article pages, user pages are occasionally the targets of <a href="/wiki/Wikipedia:Vandalism" title="Wikipedia:Vandalism">vandalism</a>, or, more rarely, <a href="/wiki/Wikipedia:Edit_warring" title="Wikipedia:Edit warring">edit wars</a>. When edit wars or vandalism persist, the affected page should be protected from editing. </p>
      <p>Most user page vandalism occurs in retaliation for a contributor's efforts to deal with vandalism. Administrators may protect their own user pages when appropriate, and are permitted to edit protected pages in user space. Sometimes a non-administrator's user page may be targeted for vandalism. Some of this vandalism is prevented through <a href="/wiki/Special:AbuseFilter/803" title="Special:AbuseFilter/803">a filter</a>, as unregistered and <a href="/wiki/Wikipedia:CONFIRMED" class="mw-redirect" title="Wikipedia:CONFIRMED">unconfirmed</a> editors are not permitted to modify other editors' primary user pages.<sup id="cite_ref-9" class="reference"><a href="#cite_note-9">[9]</a></sup> </p>
      <p>In cases in which the filter is insufficient in preventing vandalism to a non-administrator's user page, an editor may create a .css suffixed sub-page containing their user page content within their user space, transclude the sub-page into their main user page, then request that an administrator fully protect their user page. (For instance, create User:Example User as {{User:Example User/userpage.css}}.) This method will completely prevent further vandalism by limiting user page editing to yourself, and <a href="/wiki/Wikipedia:Interface_administrators" title="Wikipedia:Interface administrators">interface administrators</a> since ".js" and ".css" pages in userspace can only be edited by them. Note that the addition of inappropriate content to your user page after locking other editors out is considered a serious offense. </p>
      <p>Repeatedly inserting copyrighted content or other inappropriate material on your own user pages after being notified not to do so, or misusing user space following a block (e.g., for <a href="/wiki/Wikipedia:NPA" class="mw-redirect" title="Wikipedia:NPA">personal attacks</a> or <a href="/wiki/Wikipedia:TE" class="mw-redirect" title="Wikipedia:TE">tendentious editing</a>) are both considered disruptive and may lead to the pages being <a href="/wiki/Wikipedia:PROTECT" class="mw-redirect" title="Wikipedia:PROTECT">protected</a> to prevent further disruption. User pages may also routinely be protected in the event of a <a href="/wiki/Wikipedia:BAN" class="mw-redirect" title="Wikipedia:BAN">ban</a>. </p>
      <p>Vandalism of talk pages is less common. Usually such vandalism should merely be reverted. <a href="/wiki/Wikipedia:Blocking_policy" title="Wikipedia:Blocking policy">Blocks</a> should be used for repeated vandalism of talk pages, where policy permits. In rare cases, protection may be used but is considered a last resort given the importance of talk page discussions to the project. </p>

      <h3><span id="Deleting_others.27_user_pages"></span><span class="mw-headline" id="Deleting_others'_user_pages">Deleting others' user pages</span></h3>

      <p>In general, other users' user pages are managed by that user. Except for blatant or serious matters, it is preferable to try contacting the user before deletion (see above). However, <b>unambiguous</b> copyright violations, attack pages, promotional text, and privacy or <a href="/wiki/Wikipedia:BLP" class="mw-redirect" title="Wikipedia:BLP">BLP</a> violations can be <a href="/wiki/Wikipedia:CSD" class="mw-redirect" title="Wikipedia:CSD">speedy deleted</a> using a suitable <a href="/wiki/Wikipedia:Criteria_for_speedy_deletion#Marking_an_article_for_speedy_deletion" title="Wikipedia:Criteria for speedy deletion">template</a>, such as {{<a href="/wiki/Template:Db-attack" class="mw-redirect" title="Template:Db-attack">db-attack</a>}}, {{<a href="/wiki/Template:Db-copyvio" class="mw-redirect" title="Template:Db-copyvio">db-copyvio</a>}} or {{<a href="/wiki/Template:Db-spamuser" title="Template:Db-spamuser">db-spamuser</a>}}; other pages likely to require deletion (or where remedial action is not taken) may be submitted to <a href="/wiki/Wikipedia:MFD" class="mw-redirect" title="Wikipedia:MFD">deletion discussion</a>. </p>
      <p>Take special care to <a href="/wiki/Wikipedia:BITE" class="mw-redirect" title="Wikipedia:BITE">speak appropriately</a> and explain the concern; many users will take it as a personal affront or <a href="/wiki/Wikipedia:NPA" class="mw-redirect" title="Wikipedia:NPA">attack</a> if an unknown user announces they are going to delete a userspace image or page and an <a href="/wiki/Wikipedia:CIVIL" class="mw-redirect" title="Wikipedia:CIVIL">uncivil</a> or <a href="/wiki/Wikipedia:AGF" class="mw-redirect" title="Wikipedia:AGF">heavy duty</a> approach can discourage new users who are unaware of expectations and might enjoy contributing. Remember that a limited amount of personal information (perhaps a <i>short</i> biography) and a freely licensed tasteful personal photograph or two are usually allowed if the page reasonably complies with other requirements. </p>
      <p>Use of a user page as a personal web page unconnected with Wikipedia's mission may be a <a href="/wiki/Wikipedia:CSD" class="mw-redirect" title="Wikipedia:CSD">speedy deletion criterion</a>, as is clear advertising and promotional use. <b>A user's contributions that consist solely of a lone edit to their user page should not normally be speedy deleted unless it consists solely of spam or other speedy deletable material.</b> Test edits and the re-creation of deleted material (within limits) are permitted in user space. </p> 
      <h3><span class="mw-headline" id="Deleting_your_user_page_or_subpages"><span id="Deleting_your_own_sub_pages"></span><span id="Deleting_your_user_subpages"></span>Deleting your user page or subpages</span></h3> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:DELETEOWN"></span><span id="WP:DELETEYOURS"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcuts</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:DELETEOWN&amp;redirect=no">WP:DELETEOWN</a></span></li>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:DELETEYOURS&amp;redirect=no">WP:DELETEYOURS</a></span></li>
       </ul>
      </div> 
      <p>You can freely <a href="/wiki/Wikipedia:BLANKING" class="mw-redirect" title="Wikipedia:BLANKING">blank</a> any pages in your user space yourself (other than the few items that must not be removed) and request the deletion of your user page or subpages, by adding {{<a href="/wiki/Template:Db-user" class="mw-redirect" title="Template:Db-user">db-user</a>}} to the top of the page. Alternatively, you might consider simply making the page <a href="/wiki/Wikipedia:Redirect" title="Wikipedia:Redirect">redirect</a> to your user page. This is normally sufficient for most people's needs. Subpages tagged for deletion will be deleted if there is no overriding reason the page must be kept. </p>
      <p>Blanking of user subpages is not interpreted as a deletion request, see criteria for speedy deletion <a href="/wiki/Wikipedia:G7" class="mw-redirect" title="Wikipedia:G7">G7</a>. If you want it deleted completely then use {{<a href="/wiki/Template:Db-user" class="mw-redirect" title="Template:Db-user">db-user</a>}}. </p>
      <p>Your talk page, pages which were moved into your user space from somewhere else, and user talk archives created by <a href="/wiki/Wikipedia:MOVE" class="mw-redirect" title="Wikipedia:MOVE">page move</a>, may <b>not</b> be deleted in this way. These must be listed at <a href="/wiki/Wikipedia:Articles_for_deletion" title="Wikipedia:Articles for deletion">Articles for deletion</a> if they originated as articles, or <a href="/wiki/Wikipedia:Miscellany_for_deletion" title="Wikipedia:Miscellany for deletion">Miscellany for deletion</a> for anything else. To move them back where they came from, ask at <a href="/wiki/Wikipedia:Requested_moves" title="Wikipedia:Requested moves">Wikipedia:Requested moves</a>. </p> 
      <h3><span class="mw-headline" id="Deletion_of_user_talk_pages">Deletion of user talk pages</span></h3>

      <p>User <i>talk</i> pages and user talk archives created by <a href="/wiki/Wikipedia:MOVE" class="mw-redirect" title="Wikipedia:MOVE">page move</a> are generally <b>not</b> deleted; they are usually needed for reference by other users. Individual revisions, log entries, and other user space material <b>may</b> be deleted or <a href="https://en.wiktionary.org/wiki/redaction" class="extiw" title="wikt:redaction">redacted</a> for privacy reasons or because of harassment, threats, gross offensiveness, and other serious violations. Exceptions to this can be and are made on occasion for good reason. In addition, nonpublic personal information and potentially libelous information posted to your talk page may be removed as described above. </p>
