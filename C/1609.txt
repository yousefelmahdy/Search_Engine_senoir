      <p>A <b>redirect</b> is a page which automatically sends visitors to another page, usually an article or section of an article. For example, if you type "UK" in the search box or click on the wikilink <a href="/wiki/UK" class="mw-redirect" title="UK">UK</a>, you will be taken to the article <a href="/wiki/United_Kingdom" title="United Kingdom">United Kingdom</a> with a note at the top of the page (or on mobile, in a black message bar at the bottom): "<span style="font-size: 84%; line-height: 1.2em; color: rgb(84, 84, 84);">(Redirected from <span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=UK&amp;redirect=no">UK</a></span>)</span>". This is because the page <span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=UK&amp;redirect=no">UK</a></span> contains special wikitext which defines it as a redirect page and indicates the target article. It is also possible to redirect to a specific <a href="/wiki/Help:Section" title="Help:Section">section</a> of the target page, using more advanced syntax. </p>
      <p>Redirect pages can contain other content below the redirect, such as <a href="/wiki/Template:R_template_index" title="Template:R template index">redirect category templates</a>, and category links (which provide a way to list article sections in categories). </p>
      <p>Redirects are used to help people arrive more quickly at the page they want to read; this page contains guidance on how to use them properly. For technical help relating to how redirects work, see <b><a href="/wiki/Help:Redirect" title="Help:Redirect">Help:Redirect</a></b>. Other relevant pages are <a href="/wiki/Wikipedia:Double_redirects" title="Wikipedia:Double redirects">Wikipedia:Double redirects</a>, <a href="/wiki/Wikipedia:Hatnote#Redirect" title="Wikipedia:Hatnote">Wikipedia:Hatnote §&nbsp;Redirect</a> and <a href="/wiki/Wikipedia:WikiProject_Redirect" title="Wikipedia:WikiProject Redirect">WikiProject Redirect</a>. </p>

      <h2><span class="mw-headline" id="Purposes_of_redirects">Purposes of redirects</span></h2> 
      <p><span class="anchor" id="reasons"></span><span class="anchor" id="What_do_we_use_redirects_for?"></span> </p> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:POFR"></span><span id="WP:RPURPOSE"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcuts</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:POFR&amp;redirect=no">WP:POFR</a></span></li>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:RPURPOSE&amp;redirect=no">WP:RPURPOSE</a></span></li>
       </ul>
      </div> 
      <p>Reasons for creating and maintaining redirects include: </p> 
      <ul>
       <li>Alternative names redirect to the <a href="/wiki/Wikipedia:Article_titles" title="Wikipedia:Article titles">most appropriate article title</a> (for example, <a href="/wiki/Edson_Arantes_do_Nascimento" class="mw-redirect" title="Edson Arantes do Nascimento">Edson Arantes do Nascimento</a> redirects to <a href="/wiki/Pel%C3%A9" title="Pelé">Pelé</a>).</li> 
       <li>Plurals (for example, <a href="/wiki/Greenhouse_gases" class="mw-redirect" title="Greenhouse gases">Greenhouse gases</a> redirects to <a href="/wiki/Greenhouse_gas" title="Greenhouse gas">Greenhouse gas</a>).</li> 
       <li>Closely related words (for example, <a href="/wiki/Symbiont" class="mw-redirect" title="Symbiont">Symbiont</a> redirects to <a href="/wiki/Symbiosis" title="Symbiosis">Symbiosis</a>).</li> 
       <li>Adjectives or adverbs point to noun forms (e.g., <a href="/wiki/Treasonous" class="mw-redirect" title="Treasonous">Treasonous</a> redirects to <a href="/wiki/Treason" title="Treason">Treason</a>)</li> 
       <li>Less specific forms of names, for which the article subject is still the <a href="/wiki/Wikipedia:PRIMARYTOPIC" class="mw-redirect" title="Wikipedia:PRIMARYTOPIC">primary topic</a>. For example, <a href="/wiki/Einstein" class="mw-redirect" title="Einstein">Einstein</a> redirects to <a href="/wiki/Albert_Einstein" title="Albert Einstein">Albert Einstein</a>, whereas <a href="/wiki/Johnson" title="Johnson">Johnson</a> is a disambiguation page rather than a redirect, since no Johnson is regarded as the primary topic for that name.</li> 
       <li>More specific forms of names (for example, <a href="/wiki/Articles_of_Confederation_and_Perpetual_Union" class="mw-redirect" title="Articles of Confederation and Perpetual Union">Articles of Confederation and Perpetual Union</a> redirects to <a href="/wiki/Articles_of_Confederation" title="Articles of Confederation">Articles of Confederation</a>).</li> 
       <li>Abbreviations and initialisms (for example, <a href="/wiki/ADHD" class="mw-redirect" title="ADHD">ADHD</a> redirects to <a href="/wiki/Attention_deficit_hyperactivity_disorder" title="Attention deficit hyperactivity disorder">Attention deficit hyperactivity disorder</a>). But often an abbreviation will have multiple meanings, none of which is a <a href="/wiki/Wikipedia:PRIMARYTOPIC" class="mw-redirect" title="Wikipedia:PRIMARYTOPIC">primary topic</a>—in that case a <a href="/wiki/Wikipedia:Disambiguation" title="Wikipedia:Disambiguation">disambiguation</a> page should be created rather than a redirect.</li> 
       <li>Alternate forms of a name as found in reliable sources and common databases. For example, <a href="/wiki/Cellular_and_Molecular_Life_Sciences_:_CMLS" class="mw-redirect" title="Cellular and Molecular Life Sciences : CMLS">Cellular and Molecular Life Sciences&nbsp;: CMLS</a> redirects to <a href="/wiki/Cellular_and_Molecular_Life_Sciences" title="Cellular and Molecular Life Sciences">Cellular and Molecular Life Sciences</a>, based on its <a rel="nofollow" class="external text" href="https://www.ncbi.nlm.nih.gov/nlmcatalog?term=1420-682X">PUBMED</a> entry.</li> 
       <li>Alternative spellings or punctuation. For example, <a href="/wiki/Colour" class="mw-redirect" title="Colour">Colour</a> redirects to <a href="/wiki/Color" title="Color">Color</a>, and <a href="/wiki/Al-Jazeera" class="mw-redirect" title="Al-Jazeera">Al-Jazeera</a> redirects to <a href="/wiki/Al_Jazeera" title="Al Jazeera">Al Jazeera</a>.</li> 
       <li>Punctuation issues—some titles containing <a href="/wiki/Dash" title="Dash">dashes</a> should have redirects using <a href="/wiki/Hyphen-minus" title="Hyphen-minus">hyphens</a>, and vice versa. The proper title depends on official spelling (in the case of a name, such as <a href="/wiki/Olivia_Newton-John" title="Olivia Newton-John">Olivia Newton-John</a>), or established Wikipedia policy and naming conventions (such as <a href="/wiki/Spanish%E2%80%93American_War" title="Spanish–American War">Spanish–American War</a>).</li> 
       <li>Representations using ASCII characters, that is, common transliterations (for example, <a href="/wiki/Pele" class="mw-redirect" title="Pele">Pele</a> also redirects to <a href="/wiki/Pel%C3%A9" title="Pelé">Pelé</a> while <a href="/wiki/Kurt_Goedel" class="mw-redirect" title="Kurt Goedel">Kurt Goedel</a> and <a href="/wiki/Kurt_Godel" class="mw-redirect" title="Kurt Godel">Kurt Godel</a> redirect to <a href="/wiki/Kurt_G%C3%B6del" title="Kurt Gödel">Kurt Gödel</a>).</li> 
       <li>Likely misspellings (for example, <a href="/wiki/Condoleeza_Rice" class="mw-redirect" title="Condoleeza Rice">Condoleeza Rice</a> redirects to <a href="/wiki/Condoleezza_Rice" title="Condoleezza Rice">Condoleezza Rice</a>). Note: this criterion typically does not apply to redirects from typos in template space; consensus is that such typos, unless they are very common, should remain as red links until they are fixed.</li> 
       <li>Likely alternative capitalizations (for example, <a href="/wiki/Natural_Selection" class="mw-redirect" title="Natural Selection">Natural Selection</a> redirects to <a href="/wiki/Natural_selection" title="Natural selection">Natural selection</a>). This is not necessary for user searches made via Wikipedia's search engine, but may aid linking from other articles and external sites, as well as direct URL entry.</li> 
       <li>To comply with the maintenance of nontrivial edit history, pursuant to <a href="/wiki/Wikipedia:Merging#PROMERGE" title="Wikipedia:Merging">Wikipedia:Merging#PROMERGE</a> for copyright licensing requirements.</li> 
       <li>Sub-topics or other topics which are described or listed within a wider article. (Such redirects are often targeted to a particular section of the article.)</li> 
       <li>Redirects to disambiguation pages that do not contain "(disambiguation)" in the title (for example, <a href="/wiki/Durham_(disambiguation)" class="mw-redirect mw-disambig" title="Durham (disambiguation)">Durham (disambiguation)</a> redirects to <a href="/wiki/Durham" class="mw-disambig" title="Durham">Durham</a>). These help maintenance by allowing deliberate links to disambiguation pages to be distinguished from links that need to be disambiguated. See <a href="/wiki/Wikipedia:INTDAB" class="mw-redirect" title="Wikipedia:INTDAB">WP:INTDAB</a> for a fuller explanation.</li> 
       <li><a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcuts</a> (for example, <a href="/wiki/Wikipedia:V" class="mw-redirect" title="Wikipedia:V">Wikipedia:V</a> redirects to <a href="/wiki/Wikipedia:Verifiability" title="Wikipedia:Verifiability">Wikipedia:Verifiability</a>). This is commonly done in <a href="/wiki/Wikipedia:Project_namespace" title="Wikipedia:Project namespace">project space</a>, but not in article space.</li> 
       <li>Old-style <a href="/wiki/Wikipedia:CamelCase_and_Wikipedia" title="Wikipedia:CamelCase and Wikipedia">CamelCase</a> links (if already in existence) (<a href="/wiki/AnnaKournikova" class="mw-redirect" title="AnnaKournikova">AnnaKournikova</a> redirects to <a href="/wiki/Anna_Kournikova" title="Anna Kournikova">Anna Kournikova</a>).</li> 
       <li>Links auto-generated from <a href="/wiki/Exchangeable_image_file_format" class="mw-redirect" title="Exchangeable image file format">Exchangeable image file format</a> information (<a href="/wiki/Adobe_Photoshop_CS_Windows" class="mw-redirect" title="Adobe Photoshop CS Windows">Adobe Photoshop CS Windows</a> redirects to <a href="/wiki/Adobe_Photoshop" title="Adobe Photoshop">Adobe Photoshop</a>).</li> 
       <li>Finding <a href="/wiki/Help:What_links_here#Limitations" title="Help:What links here">what links to a section</a>, when links are made to the redirect rather than the section.</li>
      </ul> 
      <p>There are <a href="/wiki/Wikipedia:Template_messages/Redirect_pages" class="mw-redirect" title="Wikipedia:Template messages/Redirect pages">redirect templates</a> to explain the <a href="/wiki/Wikipedia:Redirect#PLA" title="Wikipedia:Redirect">reason</a> for a redirect. </p>
      <p><span class="anchor" id="note"></span> Note that redirects to other Wikimedia projects, other websites, or <a href="/wiki/Special:SpecialPages" title="Special:SpecialPages">special pages</a> do not work. These should be avoided or replaced with a {{<a href="/wiki/Template:Soft_redirect" title="Template:Soft redirect">soft redirect</a>}} template. Soft redirects are also used in <a href="/wiki/Help:Category" title="Help:Category">category space</a> (using the {{<a href="/wiki/Template:Category_redirect" title="Template:Category redirect">category redirect</a>}} template). Redirects from list titles to categories (e.g. a redirect from [[List of things]] to [[Category:Things]]) are highly discouraged.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup> </p> 
      <h2><span class="mw-headline" id="How_to_make_a_redirect">How to make a redirect</span></h2> 
      <div role="note" class="hatnote navigation-not-searchable">
       Main pages: <a href="/wiki/Help:Redirect" title="Help:Redirect">Help:Redirect</a> and <a href="/wiki/Wikipedia:How_to_make_a_redirect" title="Wikipedia:How to make a redirect">Wikipedia:How to make a redirect</a>
      </div> 
      <h3><span class="mw-headline" id="Editing_the_source_directly">Editing the source directly</span></h3> 
      <p>To create a basic redirect using the source editor, type <code>#REDIRECT&nbsp;[[target page name here]]</code> as the only text on the page. The capitalization of the word <code>REDIRECT</code> doesn't matter. For instance, if you were redirecting from "<span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=UK&amp;redirect=no">UK</a></span>" to "<a href="/wiki/United_Kingdom" title="United Kingdom">United Kingdom</a>", this would be the entire body of <span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=UK&amp;redirect=no">the "UK" page</a></span>: </p>
      <p><code>#REDIRECT&nbsp;[[United Kingdom]]</code> </p> 
      <h3><span class="mw-headline" id="Using_VisualEditor">Using VisualEditor</span></h3> 
      <p>To create a redirect using the <a href="/wiki/Wikipedia:VisualEditor" title="Wikipedia:VisualEditor">VisualEditor</a>: </p> 
      <ol>
       <li>Open the "page options" menu (icon with three parallel horizontal bars) at the top right of the editor</li> 
       <li>Select "Page settings"</li> 
       <li>Check the box marked "Redirect this page to"</li> 
       <li>Enter the name of the target page in the text box below the checkbox</li> 
       <li>Click on the blue "Apply changes" button</li> 
       <li>Save the page. You may enter an edit summary, or an automatic summary will be generated.</li>
      </ol> 
      <h3><span class="mw-headline" id="When_moving_a_page">When moving a page</span></h3> 
      <p>Redirects can also be automatically created when you <a href="/wiki/Wikipedia:MOVE" class="mw-redirect" title="Wikipedia:MOVE">move (rename) an existing page</a>. </p> 
      <h2><span class="mw-headline" id="How_to_edit_a_redirect_or_convert_it_into_an_article">How to edit a redirect or convert it into an article</span></h2> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:EDRED"></span><span id="WP:RTOA"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcuts</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:EDRED&amp;redirect=no">WP:EDRED</a></span></li>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:RTOA&amp;redirect=no">WP:RTOA</a></span></li>
       </ul>
      </div> 
      <p>Sometimes an existing redirect should really be handled by a full article, per <a href="/wiki/Category:Redirects_with_possibilities" title="Category:Redirects with possibilities">Category:Redirects with possibilities</a>. For example, the name of a notable musician (who does not yet have an article) may instead be a redirect to an existing article about a band of which the musician is a member. In this case, you can edit the redirect to make it into an article. Also, if an existing redirect points to the wrong page, you can edit the redirect to point to a different page. </p>
      <p>If you want to edit a redirect page you must use a special technique in order to get to the redirect page itself. This is because when you try to go straight to the redirect page and edit it, the redirect page will automatically redirect you to its target page (because this is what a redirect page is meant to do). Below is an example of why you might need to go to a redirect page itself (to do a small edit) and how to actually get there. </p>
      <p>For example, say <a href="/wiki/Trygve_Halvdan_Lie" class="mw-redirect" title="Trygve Halvdan Lie">Trygve Halvdan Lie</a> did not have his own article, and so this link was a redirect to the page <a href="/wiki/Secretary-General_of_the_United_Nations" title="Secretary-General of the United Nations">Secretary-General of the United Nations</a>. If, later on, the page <a href="/wiki/Trygve_Lie" title="Trygve Lie">Trygve Lie</a> was created as a biography, the page <a href="/wiki/Trygve_Halvdan_Lie" class="mw-redirect" title="Trygve Halvdan Lie">Trygve Halvdan Lie</a> should be changed to redirect to <a href="/wiki/Trygve_Lie" title="Trygve Lie">Trygve Lie</a> per <a href="/wiki/Wikipedia:COMMONNAME" class="mw-redirect" title="Wikipedia:COMMONNAME">WP:COMMONNAME</a>. To do this, go to the redirect page by clicking the existing redirect note on the target page, which in this case would read "(Redirected from <span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Trygve_Halvdan_Lie&amp;redirect=no">Trygve Halvdan Lie</a></span>)". Once there, you may click the "Edit" tab, and change the page from </p>
      <p><code>#REDIRECT&nbsp;[[Secretary-General of the United Nations]]</code> </p> 
      <dl>
       <dd>
        to
       </dd>
      </dl> 
      <p><code>#REDIRECT&nbsp;[[Trygve Lie]]</code> </p>
      <p>When adding or changing a redirect, always verify the links that already point there. For instance, if another person named Trygve Lie becomes very well known, it would make sense to make <a href="/wiki/Trygve_Lie" title="Trygve Lie">Trygve Lie</a> a redirect to his page (after renaming the existing <a href="/wiki/Trygve_Lie" title="Trygve Lie">Trygve Lie</a> page). Such a change cannot be made without changing all the preexisting links to <a href="/wiki/Trygve_Lie" title="Trygve Lie">Trygve Lie</a>; these links can be found by clicking on <a href="/wiki/Special:WhatLinksHere/Trygve_Lie" title="Special:WhatLinksHere/Trygve Lie">What links here</a> in the left hand menu. </p> 
      <h2><span class="mw-headline" id="Targeted_and_untargeted_redirects">Targeted and untargeted redirects</span></h2> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:TARGET"></span><span id="WP:RSECT"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcuts</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:TARGET&amp;redirect=no">WP:TARGET</a></span></li>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:RSECT&amp;redirect=no">WP:RSECT</a></span></li>
       </ul>
      </div> 
      <div role="note" class="hatnote navigation-not-searchable">
       See also: <a href="/wiki/MOS:LINK2SECT" class="mw-redirect" title="MOS:LINK2SECT">MOS:LINK2SECT</a>
      </div> 
      <p>Most redirects are <i>untargeted</i>, i.e. they lead simply to a page, not to any specific section of the page. This is usually done when there is more than one possible name under which an article might be sought (for example, <a href="/wiki/Cellphone" class="mw-redirect" title="Cellphone">Cellphone</a> redirects to the article <a href="/wiki/Mobile_phone" title="Mobile phone">Mobile phone</a>). For deciding which should be the actual title of the article, see <a href="/wiki/Wikipedia:Article_titles" title="Wikipedia:Article titles">Article titles</a>. </p>
      <p>It is also possible to create a <i>targeted redirect</i>, i.e. a redirect to a particular point on the target page—either a <a href="/wiki/Help:Section" title="Help:Section">section header</a> or an <a href="/wiki/Help:Anchor" class="mw-redirect" title="Help:Anchor">anchor</a>. For example, the page <a href="/wiki/Malia_Obama" class="mw-redirect" title="Malia Obama">Malia Obama</a> contains the code <code>#REDIRECT&nbsp;[[Family of Barack Obama#Malia and Sasha Obama]]</code>, which redirects to the <a href="/wiki/Family_of_Barack_Obama#Malia_and_Sasha_Obama" title="Family of Barack Obama">Malia and Sasha Obama</a> section in the article <a href="/wiki/Family_of_Barack_Obama" title="Family of Barack Obama">Family of Barack Obama</a>. Therefore, entering "<i>Malia Obama</i>" will bring the searcher straight to the content that deals with "Malia and Sasha Obama". </p>
      <p>Consider that when the target page is displayed, it is likely that the top of the page will not be shown, so the user may not see the helpful "(redirected from... )" text unless they know to scroll back to the top. This is less likely to cause confusion if the redirect is to a heading with the same name as the redirect. </p>
      <p>The text given in the link on a targeted redirect page must exactly match the target section heading or anchor text, including capitalization and punctuation. (While spaces and underscores are interchangeable in the current implementation of the Wikimedia software, it is generally good practice and aids maintenance to use exactly the same spelling in links as is used in the corresponding targets also for these characters.) (In the absence of a match, the reader will simply be taken to the top of the target page.) It is often helpful to leave a <a href="/wiki/Wikipedia:HIDDEN" class="mw-redirect" title="Wikipedia:HIDDEN">hidden comment</a> in the target text, to inform other editors that a section title is linked, so that if the title is altered, the redirect can be changed. For example: </p> 
      <pre> ==Vaccine overload==
 &lt;!-- linked from redirect [[Vaccine overload]] --&gt;
</pre> 
      <p>To ensure that a redirect will not break if a section title gets altered, or to create a redirect to a point on the page other than a section heading, create an explicit target anchor in the page, e.g., by using the {{<a href="/wiki/Template:Anchor" title="Template:Anchor">anchor</a>}} template. Alternative anchors for section headings are ideally placed directly in front of the name of the heading (but after the equals signs): </p> 
      <pre>=={{subst:Anchor|anchor name}}Section title==</pre> 
      <p><code>{{subst:Anchor}}</code> is preferable to simply using <code>{{<a href="/wiki/Template:Anchor" title="Template:Anchor">Anchor</a>}}</code> because otherwise, when the section is edited via its own <span class="nowrap">"[ edit ]"</span> link, the anchor markup and alternative section title(s) will appear as undesirable clutter at the beginning of revision history entries. Please see <a href="/wiki/MOS:RENAMESECTION" class="mw-redirect" title="MOS:RENAMESECTION">MOS:RENAMESECTION</a> for further discussion of this. </p>
      <p>The anchor text will not be visible on the page (unless the {{<a href="/wiki/Template:Visible_anchor" title="Template:Visible anchor">Visible anchor</a>}} template is used), but it will serve as a permanent marker of that place on the page. Editors should generally not remove or alter such anchors without checking all incoming links and redirects. If several logically independent aspects of a topic are discussed under a single section header and should be linked to, it is sometimes useful to define separate anchors for them, if the current amount of information doesn't justify a division into multiple sections already. This makes it easier to rearrange contents on a page as it develops since those anchors can be moved with their corresponding contents without a need to fix up incoming links. </p>
      <p>For example, in the <a href="/wiki/Google_Search" title="Google Search">Google Search</a> article, the text <code>{{Anchor|calculator}}</code> is placed at the point where <i>Google Calculator</i> is discussed. The title <a href="/wiki/Google_Calculator" class="mw-redirect" title="Google Calculator">Google Calculator</a> can then be redirected to <a href="/wiki/Google_Search#calculator" title="Google Search">Google Search#calculator</a>. </p>
      <p>When a section title is known to be the target of incoming links, the Wikipedia Manual of Style <a href="/wiki/MOS:RENAMESECTION" class="mw-redirect" title="MOS:RENAMESECTION">suggests creating a redundant anchor with the same name as the section title</a>, so that such links will continue to work even if someone renames the section without creating an anchor with the old name. Technically, doing so results in invalid HTML.<a rel="nofollow" class="external autonumber" href="https://www.w3.org/TR/html51/dom.html#the-id-attribute">[1]</a> However, when a document contains multiple tags with the same <code>id</code> value, browsers are required to return the first one, so in practice, this is not a problem.<a rel="nofollow" class="external autonumber" href="https://dom.spec.whatwg.org/#dom-nonelementparentnode-getelementbyid">[2]</a> </p>
      <p>Be careful with anchor capitalization, as redirects are case-sensitive in standards-compliant browsers.<a rel="nofollow" class="external autonumber" href="http://www.w3.org/TR/html4/struct/links.html#h-12.2.1">[3]</a> </p> 
      <h2><span class="mw-headline" id="Double_redirects">Double redirects</span></h2> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:2R"></span><span id="WP:DblR"></span><span id="WP:2RE"></span><span id="WP:2REDIR"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcuts</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:2R&amp;redirect=no">WP:2R</a></span></li>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:DblR&amp;redirect=no">WP:DblR</a></span></li>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:2RE&amp;redirect=no">WP:2RE</a></span></li>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:2REDIR&amp;redirect=no">WP:2REDIR</a></span></li>
       </ul>
      </div> 
      <div role="note" class="hatnote navigation-not-searchable">
       Main page: <a href="/wiki/Wikipedia:Double_redirects" title="Wikipedia:Double redirects">Wikipedia:Double redirects</a>
      </div> 
      <p>The software will not follow chains of more than one redirect—this is called a double redirect. A redirect should not be left pointing to another redirect page. </p>
      <p>Double redirects often arise after a page is <a href="/wiki/Wikipedia:MOVE" class="mw-redirect" title="Wikipedia:MOVE">moved</a> (renamed)—after moving a page, check whether there are any redirects to the old title (using the link on the move result page, or using <a href="/wiki/Wikipedia:WHATLINKSHERE" class="mw-redirect" title="Wikipedia:WHATLINKSHERE">"What links here"</a>), and change them to redirect straight to the new title. Double redirects are usually fixed by a <a href="/wiki/Wikipedia:Bots" title="Wikipedia:Bots">bot</a> in a few days; however, an editor should not leave behind any self-created double redirects. </p> 
      <h2><span class="mw-headline" id="Linking_to_a_redirect">Linking to a redirect</span></h2> 
      <div role="note" class="hatnote navigation-not-searchable">
       Main page: <a href="/wiki/Wikipedia:Manual_of_Style_(linking)#Redirects" class="mw-redirect" title="Wikipedia:Manual of Style (linking)">MoS linking redirects</a>
      </div>
      <p> You can link to a redirect page just as you can <a href="/wiki/Help:Link" title="Help:Link">link</a> to an article page by placing the redirect page name within a set of double brackets, such as: </p>
      <div style="margin-left: 2em">
       <code>[[<i>Redirect page name</i>]]</code>
      </div>
      <p> replacing <i>Redirect page name</i> with the name of the redirect page to link. </p>
      <p>To link to a redirect page without following the underlying redirect, use: <code>{{<a href="/wiki/Template:No_redirect" title="Template:No redirect">No redirect</a>|<i>Redirect page name</i>}}</code> replacing <i>Redirect page name</i> with the name of the redirect page to link. Clicking on a no-redirect link will send the reader to the redirect page rather than the final redirect destination. </p> 
      <h2><span class="mw-headline" id="Categorizing_redirect_pages">Categorizing redirect pages</span></h2> 
      <div role="note" class="hatnote navigation-not-searchable">
       Main page: <a href="/wiki/Wikipedia:Categorizing_redirects" title="Wikipedia:Categorizing redirects">Wikipedia:Categorizing redirects</a>
      </div> 
      <p>Most redirect pages are not placed in article <a href="/wiki/Wikipedia:Categorization" title="Wikipedia:Categorization">categories</a>. There are three types of redirect categorization that are helpful and useful: </p> 
      <ul>
       <li>Maintenance categories are in use for particular types of redirects, such as <a href="/wiki/Category:Redirects_from_initialisms" title="Category:Redirects from initialisms">Category:Redirects from initialisms</a>, in which a redirect page may be sorted using the {{<a href="/wiki/Template:R_from_initialism" title="Template:R from initialism">R from initialism</a>}} template. One major use of these categories is to determine which redirects are fit for inclusion in a <a href="/wiki/Wikipedia:Printability" title="Wikipedia:Printability">printed subset</a> of Wikipedia. See <a href="/wiki/Wikipedia:Template_messages/Redirect_pages" class="mw-redirect" title="Wikipedia:Template messages/Redirect pages">Wikipedia:Template messages/Redirect pages</a> for functional and alphabetical lists of these templates. A brief functional list of redirect category (<a href="/wiki/Wikipedia:RCAT" class="mw-redirect" title="Wikipedia:RCAT">rcat</a>) templates is also found in the {{<a href="/wiki/Template:R_template_index" title="Template:R template index">R template index</a>}} navbar.</li> 
       <li>Sometimes a redirect is placed in an article category because the form of the redirected title is more appropriate to the context of that category, e.g. <span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Honey_Lantree&amp;redirect=no">Honey Lantree</a></span>. (Redirects appear in italics in category listings.)</li> 
       <li>Discussion pages. If a discussion/talk page exists for a redirect, please ensure (1) that the talk page's Wikiproject banners are tagged with the "class=Redirect" parameter and (2) that the talk page is tagged at the TOP with the {{<a href="/wiki/Template:Talk_page_of_redirect" title="Template:Talk page of redirect">Talk page of redirect</a>}} template. If the discussion page is a redirect, then it may be tagged with appropriate redirect categorization templates (rcats).</li>
      </ul> 
      <h3><span class="mw-headline" id="Redirects_from_moves">Redirects from moves</span></h3> 
      <p>When a page is renamed/moved, a redirect that is titled with the replaced page name is created and is automatically tagged with the {{<a href="/wiki/Template:R_from_move" title="Template:R from move">R from move</a>}} template. This sorts the redirect into <a href="/wiki/Category:Redirects_from_moves" title="Category:Redirects from moves">Category:Redirects from moves</a>. </p> 
      <h2><span id="When_should_we_delete_a_redirect.3F"></span><span class="mw-headline" id="When_should_we_delete_a_redirect?">When should we delete a redirect? <span id="CRD"></span><span id="crd"></span><span id="Criteria_for_redirect_deletion"></span></span></h2> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:R#CRD"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:R&amp;redirect=no#CRD">WP:R#CRD</a></span></li>
       </ul>
      </div> 
      <p>To delete a redirect <b>without replacing it with a new article</b>, list it on <a href="/wiki/Wikipedia:Redirects_for_discussion" title="Wikipedia:Redirects for discussion">redirects for discussion</a>. See the <a href="/wiki/Wikipedia:Deletion_policy" title="Wikipedia:Deletion policy">deletion policy</a> for details on how to nominate pages for deletion. </p>
      <p>Listing is not necessary if you just want to replace a redirect with an article, or change where it points: see <a href="https://meta.wikimedia.org/wiki/Help:Redirect#Changing_a_redirect" class="extiw" title="meta:Help:Redirect">these instructions</a> for help doing this. If you want to <i>swap</i> a redirect and an article, but are not able to move the article to the location of the redirect please use <a href="/wiki/Wikipedia:Requested_moves" title="Wikipedia:Requested moves">Wikipedia:Requested moves</a> to request help from an <a href="/wiki/Wikipedia:Administrators" title="Wikipedia:Administrators">admin</a> in doing that. </p> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:R#HARMFUL"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:R&amp;redirect=no#HARMFUL">WP:R#HARMFUL</a></span></li>
       </ul>
      </div>
      <div id="harmful"> 
       <p><span class="anchor" id="HARMFUL"></span> </p> 
       <div role="note" class="hatnote navigation-not-searchable">
        Further information: <a href="/wiki/Wikipedia:Redirects_for_discussion/Common_outcomes" title="Wikipedia:Redirects for discussion/Common outcomes">Wikipedia:Redirects for discussion/Common outcomes</a>
       </div> 
       <div role="note" class="hatnote navigation-not-searchable">
        Further information: <a href="/wiki/Wikipedia:Moving_a_page#Moving_over_a_redirect" title="Wikipedia:Moving a page">Wikipedia:Moving a page §&nbsp;Moving over a redirect</a>
       </div> 
       <p>The major reasons why deletion of redirects is <b>harmful</b> are: </p> 
       <ul>
        <li>a redirect may contain non-trivial edit history;</li> 
        <li>if a redirect is reasonably old (or is the result of moving a page that has been there for quite some time), then it is possible that its deletion will break incoming links (such links coming from older revisions of Wikipedia pages, from edit summaries, from other Wikimedia projects or <a href="/wiki/Wikipedia:EXTERNALROT" class="mw-redirect" title="Wikipedia:EXTERNALROT">from elsewhere on the internet</a>, do not show up in <a href="/wiki/Help:What_links_here" title="Help:What links here">"What links here"</a>).</li>
       </ul> 
       <p>Therefore consider the deletion only of either harmful redirects or of recent ones. </p> 
      </div> 
      <p><span class="anchor" id="DELETE"></span> </p> 
      <h3><span class="mw-headline" id="Reasons_for_deleting">Reasons for deleting</span></h3> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:R#DELETE"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:R&amp;redirect=no#DELETE">WP:R#DELETE</a></span></li>
       </ul>
      </div>
      <div id="delete"> 
       <div role="note" class="hatnote navigation-not-searchable">
        See also: <a href="/wiki/Wikipedia:Criteria_for_speedy_deletion#Redirects" title="Wikipedia:Criteria for speedy deletion">Wikipedia:Criteria for speedy deletion §&nbsp;Redirects</a>
       </div> 
       <div role="note" class="hatnote navigation-not-searchable">
        "WP:RDELETE" redirects here. For revision deletion, see <a href="/wiki/Wikipedia:Revision_deletion" title="Wikipedia:Revision deletion">Wikipedia:Revision deletion</a>. For deletion review, see <a href="/wiki/Wikipedia:Deletion_review" title="Wikipedia:Deletion review">Wikipedia:Deletion review</a>.
       </div> 
       <p>You might want to <b>delete</b> a redirect if one or more of the following conditions is met (<i>but note also the exceptions listed below this list</i>): </p> 
      </div> 
      <ol>
       <li><span class="anchor" id="D1"></span><span class="anchor" id="d1"></span>The redirect page makes it <i>unreasonably</i> difficult for users to locate similarly named articles via the search engine. For example, if the user searches for "New Articles", and is redirected to a disambiguation page for "Articles", it would take much longer to get to the newly added articles on Wikipedia.</li> 
       <li><span class="anchor" id="D2"></span><span class="anchor" id="d2"></span>The redirect might cause confusion. For example, if "Adam B. Smith" was redirected to "Andrew B. Smith", because Andrew was accidentally called Adam in one source, this could cause confusion with the article on <a href="/wiki/Adam_Smith" title="Adam Smith">Adam Smith</a>, so the redirect should be deleted.</li> 
       <li><span class="anchor" id="D3"></span><span class="anchor" id="d3"></span>The redirect is offensive or abusive, such as redirecting "Joe Bloggs is a Loser" to "Joe Bloggs" (unless "Joe Bloggs is a Loser" is legitimately discussed in the article), or "Joe Bloggs" to "Loser". (<a href="/wiki/Wikipedia:CSD#G10" class="mw-redirect" title="Wikipedia:CSD">Speedy deletion criteria G10</a> and <a href="/wiki/Wikipedia:CSD#G3" class="mw-redirect" title="Wikipedia:CSD">G3</a> may apply.) 
        <div role="note" class="hatnote navigation-not-searchable">
         See also: <a href="#Neutrality_of_redirects">§&nbsp;Neutrality of redirects</a>
        </div></li> 
       <li><span class="anchor" id="D4"></span><span class="anchor" id="d4"></span>The redirect constitutes self-promotion or spam. (<a href="/wiki/Wikipedia:CSD#G11" class="mw-redirect" title="Wikipedia:CSD">Speedy deletion criterion G11</a> may apply.)</li> 
       <li><span class="anchor" id="D5"></span><span class="anchor" id="d5"></span>The redirect makes no sense, such as redirecting "Apple" to "Orange". (<a href="/wiki/Wikipedia:CSD#G1" class="mw-redirect" title="Wikipedia:CSD">Speedy deletion criterion G1</a> may apply.)</li> 
       <li><span class="anchor" id="D6"></span><span class="anchor" id="d6"></span>It is a <a href="/wiki/Wikipedia:Cross-namespace_redirects" title="Wikipedia:Cross-namespace redirects">cross-namespace redirect</a> out of article space, such as one pointing into the User or Wikipedia namespace. The major exception to this rule are the <a href="/wiki/Wikipedia:Pseudo-namespace" class="mw-redirect" title="Wikipedia:Pseudo-namespace">pseudo-namespace</a> shortcut redirects, which technically are in the main article space. Some long-standing cross-namespace redirects are also kept because of their long-standing history and potential usefulness. "<a href="/wiki/Wikipedia:Manual_of_Style" title="Wikipedia:Manual of Style">MOS:</a>" redirects, for example, are an exception to this rule. (Note "WP:" redirects are in the Wikipedia namespace, <i>WP:</i> being an <i><a href="/wiki/Wikipedia:Alias" class="mw-redirect" title="Wikipedia:Alias">alias</a></i> for <i>Wikipedia:</i>. <a href="/wiki/Wikipedia:CSD#R2" class="mw-redirect" title="Wikipedia:CSD">Speedy deletion criterion R2</a> may also apply unless if the redirect from the main article namespace points to the <b>Category:</b>, <b>Template:</b>, <b>Wikipedia:</b>, <b>Help:</b>, and <b>Portal:</b> namespaces.)</li> 
       <li><span class="anchor" id="D7"></span><span class="anchor" id="d7"></span>If the redirect is broken, meaning it redirects to an article that does not exist, it can be immediately deleted under <a href="/wiki/Wikipedia:CSD#G8" class="mw-redirect" title="Wikipedia:CSD">speedy deletion criterion G8</a>, though you should check that there is not an alternative place it could be appropriately redirected to first.</li> 
       <li><span class="anchor" id="D8"></span><span class="anchor" id="d8"></span>If the redirect is a novel or very obscure synonym for an article name, it is unlikely to be useful. In particular, <a href="/wiki/Wikipedia:Redirects_in_languages_other_than_English" title="Wikipedia:Redirects in languages other than English">redirects in a language other than English</a> to a page whose subject is unrelated to that language (or a culture that speaks that language) should generally not be created. Implausible typos or misnomers are <a href="/wiki/Wikipedia:CSD#R3" class="mw-redirect" title="Wikipedia:CSD">candidates for speedy deletion criterion R3</a>, if recently created.</li> 
       <li><span class="anchor" id="D9"></span><span class="anchor" id="d9"></span>If the target article needs to be moved to the redirect title, but the redirect has been edited before and has a history of its own, then the title needs to be freed up to make way for the move. If the move is uncontroversial, tag the redirect for <a href="/wiki/Wikipedia:Criteria_for_speedy_deletion#G6" title="Wikipedia:Criteria for speedy deletion">G6 speedy deletion</a>, or alternatively (with the <code>suppressredirect</code> user right; available to <a href="/wiki/Wikipedia:Page_mover" title="Wikipedia:Page mover">page movers</a> and admins), perform a <a href="/wiki/Wikipedia:ROUNDROBIN" class="mw-redirect" title="Wikipedia:ROUNDROBIN">round-robin move</a>. If not, take the article to <a href="/wiki/Wikipedia:Requested_moves" title="Wikipedia:Requested moves">Requested moves</a>.</li> 
       <li><span class="anchor" id="D10"></span><span class="anchor" id="d10"></span>If the redirect could plausibly be expanded into an article, and the target article contains virtually no information on the subject.</li>
      </ol> 
      <p><span class="anchor" id="KEEP"></span> </p> 
      <h3><span class="mw-headline" id="Reasons_for_not_deleting">Reasons for not deleting</span></h3> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:R#KEEP"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:R&amp;redirect=no#KEEP">WP:R#KEEP</a></span></li>
       </ul>
      </div>
      <div id="keep"> 
       <p>However, <b>avoid deleting</b> such redirects if: </p> 
      </div> 
      <ol>
       <li><span class="anchor" id="K1"></span><span class="anchor" id="k1"></span>They have a potentially useful page history, or an edit history that should be kept to comply with the licensing requirements for a merge (see <a href="/wiki/Wikipedia:Merge_and_delete" title="Wikipedia:Merge and delete">Wikipedia:Merge and delete</a>). On the other hand, if the redirect was created by renaming a page with that name, and the page history just mentions the renaming, and for one of the reasons above you want to delete the page, copy the page history to the Talk page of the article it redirects to. The act of renaming is useful page history, and even more so if there has been discussion on the page name.</li> 
       <li><span class="anchor" id="K2"></span><span class="anchor" id="k2"></span>They would aid <a href="http://meatballwiki.org/wiki/AccidentalLinking" class="extiw" title="meatball:AccidentalLinking">accidental linking</a> and make the creation of <a href="/wiki/Wikipedia:Proposed_mergers" class="mw-redirect" title="Wikipedia:Proposed mergers">duplicate articles</a> less likely, whether by redirecting a plural to a singular, by redirecting a frequent misspelling to a correct spelling, by redirecting a misnomer to a correct term, by redirecting to a synonym, etc. In other words, redirects with no incoming links are <i>not</i> candidates for deletion <i>on those grounds</i> because they are of benefit to the browsing user. Some extra vigilance by editors will be required to minimize the occurrence of those frequent misspellings in the article texts because the linkified misspellings will not appear as broken links.</li> 
       <li><span class="anchor" id="K3"></span><span class="anchor" id="k3"></span>They aid searches on certain terms. For example, if someone sees the "<a href="/wiki/Keystone_State" class="mw-redirect" title="Keystone State">Keystone State</a>" mentioned somewhere but does not know what that refers to, then he or she will be able to find out at the <a href="/wiki/Pennsylvania" title="Pennsylvania">Pennsylvania</a> (target) article.</li> 
       <li><span class="anchor" id="K4"></span><span class="anchor" id="k4"></span>You risk breaking incoming or internal links by deleting the redirect. For example, redirects resulting from page moves should not normally be deleted without good reason. Links that have existed for a significant length of time, including <a href="/wiki/Wikipedia:CamelCase_and_Wikipedia" title="Wikipedia:CamelCase and Wikipedia">CamelCase</a> links and old <a href="/wiki/Wikipedia:Subpages" title="Wikipedia:Subpages">subpage</a> links, should be left alone in case there are any existing links on external pages pointing to them. 
        <div role="note" class="hatnote navigation-not-searchable">
         See also: <a href="/wiki/Wikipedia:Link_rot#Link_rot_on_non-Wikimedia_sites" title="Wikipedia:Link rot">Wikipedia:Link rot §&nbsp;Link rot on non-Wikimedia sites</a>
        </div></li> 
       <li><span class="anchor" id="K5"></span><span class="anchor" id="k5"></span>Someone finds them useful. Hint: If someone says they find a redirect useful, they probably do. You might not find it useful—this is not because the other person is being untruthful, but because you browse Wikipedia in different ways. The <a class="external text" href="https://tools.wmflabs.org/pageviews/">pageviews tool</a> can also provide evidence of outside utility.</li> 
       <li><span class="anchor" id="K6"></span><span class="anchor" id="k6"></span>The redirect is to a closely related word form, such as a <a href="/wiki/Wikipedia:Naming_conventions_(plurals)#Redirect" title="Wikipedia:Naming conventions (plurals)">plural form</a> to a <a href="/wiki/Wikipedia:Redirect#What_do_we_use_redirects_for?" title="Wikipedia:Redirect">singular form</a>.</li>
      </ol> 
      <h3><span class="mw-headline" id="Neutrality_of_redirects">Neutrality of redirects</span></h3> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:RNEUTRAL"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:RNEUTRAL&amp;redirect=no">WP:RNEUTRAL</a></span></li>
       </ul>
      </div> 
      <p>Just as article titles using non-neutral language <a href="/wiki/Wikipedia:Article_titles#Non-neutral_but_common_names" title="Wikipedia:Article titles">are permitted in some circumstances</a>, so are such redirects. Because redirects are less visible to readers, more latitude is allowed in their names. Perceived lack of neutrality in redirect names is therefore not a sufficient reason for their deletion. In most cases, non-neutral but <a href="/wiki/Wikipedia:V" class="mw-redirect" title="Wikipedia:V">verifiable</a> redirects should point to neutrally titled articles about the subject of the term. Non-neutral redirects may be tagged with <code>{{<a href="/wiki/Template:R_from_non-neutral_name" title="Template:R from non-neutral name">R from non-neutral name</a>}}</code>. </p>
      <p>Non-neutral redirects are commonly created for three reasons: </p> 
      <ol>
       <li>Articles that are created using non-neutral titles are routinely moved to a new neutral title, which leaves behind the old non-neutral title as a working redirect (e.g. <span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Climategate&amp;redirect=no">Climategate</a></span> ? <a href="/wiki/Climatic_Research_Unit_email_controversy" title="Climatic Research Unit email controversy">Climatic Research Unit email controversy</a>).</li> 
       <li>Articles created as <a href="/wiki/Wikipedia:POV_fork" class="mw-redirect" title="Wikipedia:POV fork">POV forks</a> may be deleted and replaced by a redirect pointing towards the article from which the fork originated (e.g. <span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Barack_Obama_Muslim_rumor&amp;redirect=no">Barack Obama Muslim rumor</a></span> ? deleted and now redirected to <a href="/wiki/Barack_Obama_religion_conspiracy_theories" title="Barack Obama religion conspiracy theories">Barack Obama religion conspiracy theories</a>).</li> 
       <li>The subject matter of articles may be represented by some sources outside Wikipedia in non-neutral terms. Such terms are generally avoided in Wikipedia article titles, per the <a href="/wiki/Wikipedia:WTA" class="mw-redirect" title="Wikipedia:WTA">words to avoid</a> guidelines and the general <a href="/wiki/Wikipedia:NPOV" class="mw-redirect" title="Wikipedia:NPOV">neutral point of view policy</a>. For instance the non-neutral expression "<span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Attorneygate&amp;redirect=no">Attorneygate</a></span>" is used to redirect to the neutrally titled <a href="/wiki/Dismissal_of_U.S._attorneys_controversy" title="Dismissal of U.S. attorneys controversy">Dismissal of U.S. attorneys controversy</a>. The article in question has never used that title, but the redirect was created to provide an alternative means of reaching it because a number of press reports use the term.</li>
      </ol> 
      <p>The exceptions to this rule would be redirects that are not <a href="/wiki/Wikipedia:UNDUE" class="mw-redirect" title="Wikipedia:UNDUE">established terms</a> and are unlikely to be useful, and therefore may be nominated for deletion, perhaps under <a href="#D3">deletion reason #3</a>. However, if a redirect represents an established term that is used in multiple mainstream <a href="/wiki/Wikipedia:SOURCES" class="mw-redirect" title="Wikipedia:SOURCES">reliable sources</a>, it should be kept even if non-neutral, as it will facilitate searches on such terms. Please keep in mind that <a href="/wiki/Wikipedia:Redirects_for_discussion" title="Wikipedia:Redirects for discussion">RfD</a> is not the place to <a href="/wiki/Wikipedia:Dispute_resolution" title="Wikipedia:Dispute resolution">resolve</a> most editorial disputes. </p>
      <p><span class="anchor" id="PLA"></span><span class="anchor" id="pla"></span><span class="anchor" id="ASTONISH"></span><span class="anchor" id="SURPRISE"></span> </p> 
      <h2><span id="What_needs_to_be_done_on_pages_that_are_targets_of_redirects.3F"></span><span class="mw-headline" id="What_needs_to_be_done_on_pages_that_are_targets_of_redirects?">What needs to be done on pages that are targets of redirects?</span></h2> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:R#PLA"></span><span id="WP:R#ASTONISH"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcuts</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:R&amp;redirect=no#PLA">WP:R#PLA</a></span></li>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:R&amp;redirect=no#ASTONISH">WP:R#ASTONISH</a></span></li>
       </ul>
      </div> 
      <div role="note" class="hatnote navigation-not-searchable">
       Further information: <a href="/wiki/Wikipedia:Manual_of_Style/Text_formatting#Other_uses" title="Wikipedia:Manual of Style/Text formatting">Wikipedia:Manual of Style/Text formatting §&nbsp;Other uses</a>, and <a href="/wiki/Wikipedia:Manual_of_Style/Lead_section#Alternative_names" title="Wikipedia:Manual of Style/Lead section">Wikipedia:Manual of Style/Lead section §&nbsp;Alternative names</a>
      </div> 
      <p>Wikipedia follows the "<a href="/wiki/Wikipedia:Guide_to_writing_better_articles#Principle_of_least_astonishment" class="mw-redirect" title="Wikipedia:Guide to writing better articles">principle of least astonishment</a>"; after following a redirect, the reader's first question is likely to be: "Hang on ... I wanted to read about <i>this</i>. Why has the link taken me to <i>that</i><span style="padding-left:0.15em;">?</span>" Make it clear to the reader that they <i>have</i> arrived in the right place. </p>
      <p>Normally, we try to make sure that all "inbound redirects" other than misspellings or other obvious close variants of the article title are mentioned in the first couple of paragraphs of the article or section to which the redirect goes. It will often be appropriate to <b>bold</b> the redirected term. For example: </p> 
      <ul>
       <li><b>Alice Bradley Sheldon</b> (August 24, 1915 – May 19, 1987) was an American science fiction author better known as <b>James Tiptree Jr.</b> ... 
        <ul>
         <li><a href="/wiki/James_Tiptree_Jr." title="James Tiptree Jr.">James Tiptree Jr.</a>, redirects from <a href="/wiki/Alice_Sheldon" class="mw-redirect" title="Alice Sheldon">Alice Sheldon</a></li>
        </ul></li>
      </ul> 
      <p>But insignificant or minor redirects can skip this: </p> 
      <ul>
       <li><a href="/wiki/Density_of_water" class="mw-redirect" title="Density of water">Density of water</a> redirects to <a href="/wiki/Properties_of_water" title="Properties of water">Properties of water</a>. There is no need to insert a bolded <b>density of water</b> sentence in the lead section; it is a minor subtopic of the article.</li>
      </ul> 
      <p>If the redirected term could have other meanings, a <a href="/wiki/Wikipedia:Hatnote" title="Wikipedia:Hatnote">hatnote</a> (<a href="/wiki/Wikipedia:HATTEST#Redirect" class="mw-redirect" title="Wikipedia:HATTEST">examples</a>) should be placed at the top of the target article or targeted section that will direct readers to the other meanings or to a relevant <a href="/wiki/Wikipedia:DAB" class="mw-redirect" title="Wikipedia:DAB">disambiguation</a> page. This is usually done using one of the <a href="/wiki/Template:Redirect#&quot;…_redirects_here._For_other_uses,_see_…&quot;" title="Template:Redirect">redirect disambiguation templates</a> (<a href="/wiki/Wikipedia:HATTEST#Redirect" class="mw-redirect" title="Wikipedia:HATTEST">examples</a>). </p>
      <p>It may also be helpful to <a href="/wiki/Help:Searching" title="Help:Searching">search</a> the <a href="/wiki/Help:Category#Listing_all_categories" title="Help:Category">List of Categories</a> for <a href="/wiki/Wikipedia:FAQ/Categories#Can_I_specify_categories_when_searching?" title="Wikipedia:FAQ/Categories">related terms</a>. </p> 
      <h2><span class="mw-headline" id="Redirects_that_replace_previous_articles">Redirects that replace previous articles</span></h2> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:BLANKANDREDIRECT"></span><span id="WP:BLAR"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcuts</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:BLANKANDREDIRECT&amp;redirect=no">WP:BLANKANDREDIRECT</a></span></li>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:BLAR&amp;redirect=no">WP:BLAR</a></span></li>
       </ul>
      </div> 
      <div role="note" class="hatnote navigation-not-searchable">
       See also: <a href="/wiki/Wikipedia:Deletion_policy#Redirection" title="Wikipedia:Deletion policy">Wikipedia:Deletion policy §&nbsp;Redirection</a>
      </div> 
      <p><a href="/wiki/Wikipedia:BLANK" class="mw-redirect" title="Wikipedia:BLANK">Removing all content</a> in a problematic article and replacing it with a redirect is common practice, known as <b>blank-and-redirect</b>. If other editors <a href="/wiki/Wikipedia:BRD" class="mw-redirect" title="Wikipedia:BRD">disagree</a> with this blanking, its contents can be recovered from <a href="/wiki/Help:Page_history" title="Help:Page history">page history</a>, as the article has not been <a href="/wiki/Wikipedia:DEL" class="mw-redirect" title="Wikipedia:DEL">deleted</a>. If editors cannot agree, the content issues should be discussed at the relevant talk page, and other methods of <a href="/wiki/Wikipedia:Resolving_disputes" class="mw-redirect" title="Wikipedia:Resolving disputes">dispute resolution</a> should be used, such as restoring the article and nominating the article for <a href="/wiki/Wikipedia:Articles_for_deletion" title="Wikipedia:Articles for deletion">Wikipedia:Articles for deletion</a> or listing on <a href="/wiki/Wikipedia:Requests_for_comment" title="Wikipedia:Requests for comment">Wikipedia:Requests for comments</a> for further input. </p>
      <p>To make it easier for other editors to find the history of the blanked article, it's good practice to add a short notice at the talk page of the target article, even if no content has been <a href="/wiki/Wikipedia:Merging" title="Wikipedia:Merging">merged</a> there. This is specially useful if the blanked article had few visits and infrequent edits. If the redirect replaces an article that has been deleted by an <a href="/wiki/Wikipedia:Administrators" title="Wikipedia:Administrators">administrator</a>, this notice is the only way for editors to know that a previous version of the article existed at all. </p> 
      <h3><span class="mw-headline" id="Content_of_the_replaced_article">Content of the replaced article</span></h3> 
      <p>If the topic of the article can be reasonably thought to describe a <a href="/wiki/Wikipedia:N" class="mw-redirect" title="Wikipedia:N">notable</a> topic, mark the redirect with the template {{<a href="/wiki/Template:Redirect_with_possibilities" class="mw-redirect" title="Template:Redirect with possibilities">Redirect with possibilities</a>}} to indicate that <a href="/wiki/Wikipedia:Redirect#How_to_edit_a_redirect_or_convert_it_into_an_article" title="Wikipedia:Redirect">it could be expanded in the future</a>. You may also consider <a href="/wiki/Wikipedia:Stubbing#Stubbing_existing_articles" class="mw-redirect" title="Wikipedia:Stubbing">turning the article into a stub</a> by removing all unsourced content and keeping the valid references, instead of blanking it. </p>
      <p>Note that certain forms of blanking are not allowed. <a href="/wiki/Wikipedia:Vandalism#Blanking,_illegitimate" title="Wikipedia:Vandalism">Illegitimate blanking</a> of valid content without reason is considered <a href="/wiki/Wikipedia:Vandalism" title="Wikipedia:Vandalism">vandalism</a>, a form of <a href="/wiki/Wikipedia:Disruptive_editing" title="Wikipedia:Disruptive editing">disruptive editing</a>. Other forms of blank-and-redirect, although not vandalism, are still undesirable. If you want to <a href="/wiki/Wikipedia:CUTPASTE" class="mw-redirect" title="Wikipedia:CUTPASTE">rename the article by cutting and pasting text</a> to a new article with a different title, you should instead <a href="/wiki/Wikipedia:Moving_a_page" title="Wikipedia:Moving a page">move the page</a> with the <i>Move</i> option. If you want to keep some content from the blanked article and add it to the target article, you should follow the instructions at <a href="/wiki/Wikipedia:Merging#How_to_merge" title="Wikipedia:Merging">Wikipedia:Merging §&nbsp;How to merge</a>. Both processes will create proper links to the edit history, which is required by the <a href="/wiki/Wikipedia:Copyrights" title="Wikipedia:Copyrights">Wikipedia license</a> for legal reasons to <a href="/wiki/Wikipedia:Copying_within_Wikipedia" title="Wikipedia:Copying within Wikipedia">preserve attribution of content to its authors</a>. </p> 
      <h2><span id="Do_not_.22fix.22_links_to_redirects_that_are_not_broken"></span><span class="mw-headline" id="Do_not_&quot;fix&quot;_links_to_redirects_that_are_not_broken">Do not "fix" links to redirects that are not broken</span></h2> 
      <p><span class="anchor" id="NOTBROKEN"></span><span class="anchor" id="notbroken"></span>
       <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855"></p>
      <div class="module-shortcutanchordiv">
       <span id="WP:NOTBROKEN"></span><span id="WP:DONOTFIXIT"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcuts</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:NOTBROKEN&amp;redirect=no">WP:NOTBROKEN</a></span></li>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:DONOTFIXIT&amp;redirect=no">WP:DONOTFIXIT</a></span></li>
       </ul>
      </div> 
      <div role="note" class="hatnote navigation-not-searchable">
       "WP:NOTBROKEN" redirects here. You may be looking for <a href="/wiki/Wikipedia:Typo_Team#Some_typos_do_not_need_to_be_corrected" title="Wikipedia:Typo Team">Wikipedia:Typo Team §&nbsp;Some typos do not need to be corrected</a> or <a href="/wiki/Wikipedia:If_it_ain%27t_broke,_don%27t_fix_it" title="Wikipedia:If it ain't broke, don't fix it">Wikipedia:If it ain't broke, don't fix it</a>.
      </div> 
      <div role="note" class="hatnote navigation-not-searchable">
       Further information: <a href="/wiki/Wikipedia:Manual_of_Style/Linking#MOS:NOPIPE" title="Wikipedia:Manual of Style/Linking">Wikipedia:Manual of Style/Linking §&nbsp;MOS:NOPIPE</a>
      </div> 
      <p>There is usually nothing wrong with linking to redirects to articles. Some editors are tempted, upon finding a link to a redirect page, to bypass the redirect and point the link directly at the target page. However, changing to a piped link is beneficial only in a few cases. Piping links solely to avoid redirects is generally a time-wasting exercise that can actually be detrimental. It is almost never helpful to replace <code>[[redirect]]</code> with <code>[[target|redirect]]</code>. </p>
      <p>That is, editors should not change, for instance, <code>[[Franklin Roosevelt]]</code> to <code>[[Franklin D. Roosevelt]]</code> or <code>[[Franklin D. Roosevelt|Franklin Roosevelt]]</code> <i>just</i> to "fix a redirect". However, it is perfectly acceptable to change it to <code>[[Franklin D. Roosevelt]]</code> if for some reason it is preferred that "Franklin D. Roosevelt" actually appear in the visible text. Editors should also not change <a href="/wiki/Category:Redirects_with_possibilities" title="Category:Redirects with possibilities">redirects with possibilities</a> like <code>[[Journal of the Franklin Institute]]</code> to <code>[[Franklin Institute#Journal of the Franklin Institute|Journal of the Franklin Institute]]</code>, so that readers arrive at the more pertinent article in the eventuality that it is created. </p>
      <p>Reasons <i>not</i> to bypass redirects include: </p> 
      <ul>
       <li>Redirects can indicate <a href="/wiki/Category:Redirects_with_possibilities" title="Category:Redirects with possibilities">possible future articles</a> (see {{<a href="/wiki/Template:R_with_possibilities" title="Template:R with possibilities">R with possibilities</a>}}).</li> 
       <li>Introducing unnecessary invisible text makes the article more difficult to read in page source form.</li> 
       <li>Non-piped links make better use of the "what links here" tool, making it easier to track how articles are linked and helping with large-scale changes to links.</li> 
       <li>Shortcuts or redirects to embedded anchors or sections of articles or of <a href="/wiki/Wikipedia:Policies_and_guidelines" title="Wikipedia:Policies and guidelines">Wikipedia's advice pages</a> should never be bypassed, as the anchors or section headings on the page may change over time. Updating one redirect is far more efficient than updating dozens of piped links. (The <a href="https://iw.toolforge.org/dispenser/cgi-bin/rdcheck.py" class="extiw" title="toollabs:dispenser/cgi-bin/rdcheck.py">Rdcheck</a> tool is extremely useful in such cases for finding which redirects need to be changed after an article is updated.)</li> 
       <li>Intentional links to <a href="/wiki/Wikipedia:Disambiguation" title="Wikipedia:Disambiguation">disambiguation pages</a> always use the title with "(disambiguation)", even if that is a redirect.</li> 
       <li>If editors persistently use a redirect instead of an article title, it may be that the article needs to be moved rather than the redirect changed. As such the systematic "fixing of redirects" may eradicate useful information which can be used to help decide on the "best" article title.</li>
      </ul> 
      <p><span class="anchor" id="Bypass_redirects_in_navigational_templates"></span><span class="anchor" id="template_link_fix"></span><span class="anchor" id="template_linkfix"></span><span class="anchor" id="templatelinkfix"></span><span class="anchor" id="TLF"></span>Good reasons to bypass redirects include:
       <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855"></p>
      <div class="module-shortcutanchordiv">
       <span id="WP:BRINT"></span><span id="WP:NAVNOREDIRECT"></span><span id="WP:MPNOREDIRECT"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcuts</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:BRINT&amp;redirect=no">WP:BRINT</a></span></li>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:NAVNOREDIRECT&amp;redirect=no">WP:NAVNOREDIRECT</a></span></li>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:MPNOREDIRECT&amp;redirect=no">WP:MPNOREDIRECT</a></span></li>
       </ul>
      </div> 
      <ul>
       <li>It is usually preferable not to use redirected links in navigational templates, such as those found at the bottom of many articles (e.g., {{<a href="/wiki/Template:US_Presidents" class="mw-redirect" title="Template:US Presidents">US Presidents</a>}} at the end of <a href="/wiki/George_Washington" title="George Washington">George Washington</a>). When the template is placed on an article and contains a <i>direct</i> link to the same article (rather than a redirect), the direct link will display in <b>bold</b> (and not as a link), making it easier to navigate through a series of articles using the template. There are exceptions to this exception: where a redirect represents a distinct sub-topic within a larger article and is not merely a variant name, it is preferable to leave the redirect in the template.</li> 
       <li>It may be appropriate to make this kind of change if the hint that appears when a user hovers over the link is misleading (see <a href="/wiki/Wikipedia:ASTONISH" class="mw-redirect" title="Wikipedia:ASTONISH">Principle of least astonishment</a>).</li> 
       <li>Spelling errors and other mistakes should be corrected. Don't link to a misspelled redirect. This does not necessarily mean that the misspelled redirect should be deleted (see {{<a href="/wiki/Template:R_from_misspelling" title="Template:R from misspelling">R from misspelling</a>}}).</li> 
       <li>Links on disambiguation pages. See <a href="/wiki/Wikipedia:Manual_of_Style/Disambiguation_pages#Piping_and_redirects" title="Wikipedia:Manual of Style/Disambiguation pages">Wikipedia:Manual of Style/Disambiguation pages §&nbsp;Piping and redirects</a> for rationale and exceptions.</li> 
       <li>Radio and TV station call letters, since call letters given up by one station can be used later by a different station.</li> 
       <li>In other namespaces, particularly the template and portal namespaces in which <a href="/wiki/Wikipedia:Subpages" title="Wikipedia:Subpages">subpages</a> are common, any link or transclusion to a former page title that has become a redirect following a page move or merge should be updated to the new title for naming consistency.</li> 
       <li>Links on the <a href="/wiki/Main_Page" title="Main Page">Main Page</a>. (But note, as above, that redirects to article sections should never be bypassed.)</li>
      </ul> 
      <h3><span class="mw-headline" id="Self-redirects">Self-redirects</span></h3> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:SELFRED"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:SELFRED&amp;redirect=no">WP:SELFRED</a></span></li>
       </ul>
      </div> 
      <div role="note" class="hatnote navigation-not-searchable">
       See also: <a href="/wiki/Help:Self_link" title="Help:Self link">Help:Self link</a>
      </div> 
      <p>Avoid linking to titles that redirect straight back to the page on which the link is found. This situation may arise if a redirect is created from a red link on the page, or if the title was once a separate page but was merged. </p>
      <p>However, linking to a title that redirects to a section or anchor within the article (redirects with {{<a href="/wiki/Template:R_to_section" title="Template:R to section">R to section</a>}} or {{<a href="/wiki/Template:R_to_anchor" title="Template:R to anchor">R to anchor</a>}}) is acceptable, as it facilitates navigation in particular on long articles that cannot be viewed all at once on an average-sized computer screen. In addition to readability benefits, when such redirects are marked with {{<a href="/wiki/Template:R_with_possibilities" title="Template:R with possibilities">R with possibilities</a>}}, they have the potential to become independent articles in the future. However, consider using <a href="/wiki/Wikipedia:Manual_of_Style/Linking#Section_links" title="Wikipedia:Manual of Style/Linking">section links</a> instead, when such redirects do not already exist. </p> 
      <h2><span class="mw-headline" id="Template_redirects">Template redirects</span></h2> 
      <p>A template can be redirected to another template in the same way, e.g., by entering the following markup at the top of a template T2: </p> 
      <pre>#REDIRECT [[Template:T1]]
</pre> 
      <p>This allows the template name T2 to be used instead of the actual template name T1. All the parameters of T1 will be respected by T2. </p>
      <p>A redirect categorisation (<a href="/wiki/Wikipedia:Categorizing_redirects" title="Wikipedia:Categorizing redirects">rcat</a>) template such as {{<a href="/wiki/Template:R_from_move" title="Template:R from move">R from move</a>}} may be added to T2 (on the third line below the <code>#REDIRECT</code> line) as follows: </p> 
      <pre>#REDIRECT [[Template:T1]]

{{Redirect category shell|
{{R from move}}
}}
</pre> 
      <p>While template shortcut/alias redirects are common, they may infrequently cause confusion and make updating template calls more complicated. For example, if calls to T1 are to be changed to some new template NT1, articles must be searched for <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">{{T1}}</code> and a separate search must also be made for each of its aliases (including T2 in this example). Moreover, changes to syntax, corrections, scans and other processes (for example tag dating) must take into account <i>all</i> applicable redirects. </p> 
      <h2><span class="mw-headline" id="Redirect_protection">Redirect protection</span></h2> 
      <p><span class="anchor" id="protection"></span> </p> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:REDIRECTPROTECTION"></span><span id="WP:RPROTECTION"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcuts</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:REDIRECTPROTECTION&amp;redirect=no">WP:REDIRECTPROTECTION</a></span></li>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:RPROTECTION&amp;redirect=no">WP:RPROTECTION</a></span></li>
       </ul>
      </div> 
      <p>Sometimes, a redirect to an article pertaining to a very controversial topic will be fully or, more rarely, semi-protected indefinitely. This is done when any of the following criteria are met: </p> 
      <ol>
       <li>There is no reason for it to be edited</li> 
       <li>It is frequently expanded into whole articles</li> 
       <li>It is an obvious vandalism target</li> 
       <li>It redirects and/or refers to a very controversial topic</li>
      </ol> 
      <p>Redirects that are protected include <a href="/wiki/Obama" class="mw-redirect" title="Obama">Obama</a>, <a href="/wiki/Hitler" class="mw-redirect" title="Hitler">Hitler</a>, and <a href="/wiki/9/11" class="mw-redirect" title="9/11">9/11</a>. <b><a href="/wiki/Wikipedia:SRD" class="mw-redirect" title="Wikipedia:SRD">Soft redirects</a></b> that are protected include obvious vandalism targets like <a href="/wiki/Dumbass" title="Dumbass">dumbass</a> and <a href="/wiki/Fatass" title="Fatass">fatass</a>. </p>
      <p>Redirects in other namespaces may be protected for technical reasons or are protected under existing guidelines. For example, a template redirect (shorthand) used thousands of times qualifies it as a <a href="/wiki/Wikipedia:High-risk_template" class="mw-redirect" title="Wikipedia:High-risk template">highly visible template</a>, eligible for <a href="/wiki/Wikipedia:TEMP-P" class="mw-redirect" title="Wikipedia:TEMP-P">template protection</a>. </p> 
      <h2><span class="mw-headline" id="Category_redirects">Category redirects</span></h2> 
      <p><span class="anchor" id="CATEGORY"></span><span class="anchor" id="category"></span> </p> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:R#CATEGORY"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:R&amp;redirect=no#CATEGORY">WP:R#CATEGORY</a></span></li>
       </ul>
      </div> 
      <div role="note" class="hatnote navigation-not-searchable">
       This section is about technical issue on category redirects. For making "soft" category redirects, see <a href="/wiki/Wikipedia:Categories_for_discussion#Redirecting_categories" title="Wikipedia:Categories for discussion">Wikipedia:Categories for discussion §&nbsp;Redirecting categories</a>.
      </div> 
      <p>Do <i>not</i> create inter-<a href="/wiki/Help:Category" title="Help:Category">category</a> redirects, by adding a line <code>#REDIRECT [[:Category:<i>target category</i>]]</code> to a category page. Articles added to a "redirected" category do not show up in the target category, preventing proper categorization. What's worse, since redirected categories do not become "red links", editors won't be aware even when they add an article to a redirected category. </p>
      <p>For an attempt to fix this issue in <a href="/wiki/MediaWiki" title="MediaWiki">MediaWiki</a>, see <a href="https://phabricator.wikimedia.org/T5311" class="extiw" title="phabricator:T5311">T5311</a>. </p>
      <p>Instead, <a href="/wiki/Wikipedia:Soft_redirect" title="Wikipedia:Soft redirect">"soft" redirects</a> are used. It can be created by placing <code>{{<a href="/wiki/Template:Category_redirect" title="Template:Category redirect">Category redirect</a>|target}}</code> in the category page. See <a href="/wiki/Wikipedia:Categories_for_discussion#Redirecting_categories" title="Wikipedia:Categories for discussion">Wikipedia:Categories for discussion#Redirecting categories</a>. </p> 
      <h2><span class="mw-headline" id="Suppressing_redirects">Suppressing redirects</span></h2> 
      <p><span class="anchor" id="SUPPRESS"></span>
       <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855"></p>
      <div class="module-shortcutanchordiv">
       <span id="WP:R#SUPPRESS"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:R&amp;redirect=no#SUPPRESS">WP:R#SUPPRESS</a></span></li>
       </ul>
      </div> 
      <p>When a page is moved, a redirect is automatically left behind. Some groups of users (those who possess a <code>suppressredirect</code> <a href="/wiki/Special:ListGroupRights" title="Special:ListGroupRights">right</a>) have the ability to prevent the redirect being created, by unchecking the box labelled "Leave a redirect behind." Currently these groups are <a href="/wiki/Wikipedia:Administrators" title="Wikipedia:Administrators">administrators</a>, <a href="/wiki/Wikipedia:Bots" title="Wikipedia:Bots">bots</a>, <a href="/wiki/Wikipedia:Page_mover" title="Wikipedia:Page mover">page movers</a>, and <a href="https://meta.wikimedia.org/wiki/Global_rollback" class="extiw" title="meta:Global rollback">global rollbackers</a>. In some circumstances, a page should be moved, but a redirect from its current name is inappropriate, such as reverting <a href="/wiki/Wikipedia:Vandalism#Page-move_vandalism" title="Wikipedia:Vandalism">page-move vandalism</a>. Suppressing the redirect can avoid an extra action (page removal) and save time in these cases. </p>
      <p>However, in general, the redirect will be a useful entry in the history, and it is best to leave it behind, unless there is a good reason to suppress the redirect, such as vandalism, <a href="/wiki/Wikipedia:Userfication" title="Wikipedia:Userfication">userfying</a> <i>recently created</i> malplaced items or freeing a title to be occupied immediately by another page (e.g., moving <u>term</u> to <u>accurate term</u> and <u>term (disambiguation)</u> to <u>term</u>). Redirects leave a trail to help readers find the old article, in case a new article is created at its previous location, and to prevent <a href="/wiki/Wikipedia:Linkrot" class="mw-redirect" title="Wikipedia:Linkrot">linkrot</a>. Therefore, we usually neither suppress nor <a href="#When_should_we_delete_a_redirect?">delete</a> redirects. As Brion Vibber <a href="https://phabricator.wikimedia.org/T17842#205934" class="extiw" title="phab:T17842">said</a>, "Not breaking links helps everyone, <i>especially us first and foremost</i>". He also <a href="https://phabricator.wikimedia.org/T17842#205878" class="extiw" title="phab:T17842">said</a> that the removal of (file) redirects is "extremely user-hostile and makes the project less useful". </p> 
      <h2><span class="mw-headline" id="Technical_notes">Technical notes</span></h2> 
      <p>A Wikipedia redirect is not the same as an <a href="/wiki/HTTP_redirect" class="mw-redirect" title="HTTP redirect">HTTP redirect</a>—it does not generate an <a href="/wiki/HTTP_302" title="HTTP 302">HTTP 302</a> (or other 30<i>x</i>) response. Instead, a page with almost the same content as the target of the redirect is generated by the MediaWiki software, differing in that a small-text note appears below the title of the page, identifying the name of the redirect used to get there (and linking to it in such a way that it can be accessed without the redirect, e.g. so it can be changed). When a user clicks on a redirect such as <a href="/wiki/Housecat" class="mw-redirect" title="Housecat">housecat</a>, the page URL initially will be <a class="external free" href="https://en.wikipedia.org/wiki/Housecat">https://en.wikipedia.org/wiki/Housecat</a>, but the URL shown by the browser will change to <a class="external free" href="https://en.wikipedia.org/wiki/Cat">https://en.wikipedia.org/wiki/Cat</a> after the page loads. </p>
      <p>On one hand, this allows links like <a href="/wiki/Housecat#Anatomy" class="mw-redirect" title="Housecat">housecat#Anatomy</a> to work as expected, but it also requires redirects to anchors to be implemented as a piece of JavaScript that jumps to an appropriate section after the page has loaded. For example, <a href="/wiki/Second-stage_boot_loader" class="mw-redirect" title="Second-stage boot loader">second-stage boot loader</a>, which is rendered as the URL <a class="external free" href="https://en.wikipedia.org/wiki/Second-stage_boot_loader">https://en.wikipedia.org/wiki/Second-stage_boot_loader</a>, is a page defined as a #REDIRECT to <a href="/wiki/Booting#SECOND-STAGE" title="Booting">Booting#SECOND-STAGE</a>. "SECOND-STAGE", in this case, is a manually defined anchor (using the markup "=== {{anchor|SECOND-STAGE}}Second-stage boot loader ===") which will persist even if the section is renamed. However, whether a redirect points to a manually defined anchor, or an anchor defined implicitly via a section name, the behavior will be the same: the page will automatically be scrolled down to the pointed-to anchor only after the page finishes loading (at which point the URL bar will also change to reflect the redirected-to URL, including "#<i>anchor</i>" portion, rather than the redirected-from URL). </p>
