       <h3 id="community-conversations-in-github-discussions"><a href="#community-conversations-in-github-discussions">Community conversations in GitHub Discussions</a></h3> 
       <p>Since GitHub Discussions is an open forum, there is an opportunity to bring non-code collaboration into a project's repository and gather diverse feedback and ideas more quickly. You can help drive a productive conversation by:</p> 
       <ul> 
        <li>Asking pointed questions and follow-up questions to garner specific feedback</li> 
        <li>Capture a diverse experience and distill it down to main points</li> 
        <li>Open an issue to take action based on the conversation, where applicable</li> 
       </ul> 
       <p>For more information about opening an issue and cross-referencing a discussion, see "<a href="/en/github/managing-your-work-on-github/opening-an-issue-from-a-comment">Opening an issue from a comment</a>."</p> 
       <h3 id="learning-about-conversations-on-github"><a href="#learning-about-conversations-on-github">Learning about conversations on GitHub</a></h3> 
       <p>You can create and participate in discussions, issues, and pull requests, depending on the type of conversation you'd like to have.</p> 
       <p>You can use GitHub Discussions to discuss big picture ideas, brainstorm, and spike out a project's specific details before committing it to an issue, which can then be scoped. Discussions are useful for teams if:</p> 
       <ul> 
        <li>You are in the discovery phase of a project and are still learning which director your team wants to go in</li> 
        <li>You want to collect feedback from a wider community about a project</li> 
        <li>You want to keep bug fixes, feature requests, and general conversations separate</li> 
       </ul> 
       <p>Issues are useful for discussing specific details of a project such as bug reports and planned improvements. For more information, see "<a href="/en/articles/about-issues">About issues</a>." Pull requests allow you to comment directly on proposed changes. For more information, see "<a href="/en/articles/about-pull-requests">About pull requests</a>" and "<a href="/en/articles/commenting-on-a-pull-request">Commenting on a pull request</a>."</p> 
       <p>On your team's page, you can use team discussions for conversations that span across projects or repositories and don't belong in a specific issue or pull request. Instead of opening an issue in a repository to discuss an idea, you can include the entire team by having a conversation in a team discussion. For more information, see "<a href="/en/organizations/collaborating-with-your-team/about-team-discussions">About team discussions</a>."</p>
