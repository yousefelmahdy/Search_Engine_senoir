      <div id="article-contents" class="article-grid-body"> 
       <p>The Events API can return different types of events triggered by activity on GitHub. Each event response contains shared properties, but has a unique <code>payload</code> object determined by its event type. The <a href="#event-object-common-properties">Event object common properties</a> describes the properties shared by all events, and each event type describes the <code>payload</code> properties that are unique to the specific event.</p> 
       <h3 id="event-object-common-properties"><a href="#event-object-common-properties">Event object common properties</a></h3> 
       <p>The event objects returned from the Events API endpoints have the same structure.</p> 
       <table>
        <thead>
         <tr>
          <th>Event API attribute name</th>
          <th>Description</th>
         </tr>
        </thead>
        <tbody>
         <tr>
          <td><code>id</code></td>
          <td>Unique identifier for the event.</td>
         </tr>
         <tr>
          <td><code>type</code></td>
          <td>The type of event. Events uses PascalCase for the name.</td>
         </tr>
         <tr>
          <td><code>actor</code></td>
          <td>The user that triggered the event.</td>
         </tr>
         <tr>
          <td><code>actor.id</code></td>
          <td>The unique identifier for the actor.</td>
         </tr>
         <tr>
          <td><code>actor.login</code></td>
          <td>The username of the actor.</td>
         </tr>
         <tr>
          <td><code>actor.display_login</code></td>
          <td>The specific display format of the username.</td>
         </tr>
         <tr>
          <td><code>actor.gravatar_id</code></td>
          <td>The unique identifier of the Gravatar profile for the actor.</td>
         </tr>
         <tr>
          <td><code>actor.url</code></td>
          <td>The REST API URL used to retrieve the user object, which includes additional user information.</td>
         </tr>
         <tr>
          <td><code>actor.avatar_url</code></td>
          <td>The URL of the actor's profile image.</td>
         </tr>
         <tr>
          <td><code>repo</code></td>
          <td>The repository object where the event occurred.</td>
         </tr>
         <tr>
          <td><code>repo.id</code></td>
          <td>The unique identifier of the repository.</td>
         </tr>
         <tr>
          <td><code>repo.name</code></td>
          <td>The name of the repository, which includes the owner and repository name. For example, <code>octocat/hello-world</code> is the name of the <code>hello-world</code> repository owned by the <code>octocat</code> user account.</td>
         </tr>
         <tr>
          <td><code>repo.url</code></td>
          <td>The REST API URL used to retrieve the repository object, which includes additional repository information.</td>
         </tr>
         <tr>
          <td><code>payload</code></td>
          <td>The event payload object is unique to the event type. See the event type below for the event API <code>payload</code> object.</td>
         </tr>
        </tbody>
       </table>

       <h3 id="commitcommentevent"><a href="#commitcommentevent">CommitCommentEvent</a></h3> 
       <p>A commit comment is created. The type of activity is specified in the <code>action</code> property of the payload object. For more information, see the "<a href="/en/rest/reference/repos#comments">commit comment</a>" REST API.</p> 
       <p>The <a href="#event-object-common-properties">event object</a> includes properties that are common for all events. Each event object includes a <code>payload</code> property and the value is unique to each event type. The <code>payload</code> object for this event is described below.</p>

          <td><code>string</code></td>
          <td>The action performed. Can be <code>created</code>.</td>

          <td><code>comment</code></td>
          <td><code>object</code></td>

         </tr>
        </tbody>
       </table>

       <p>The <a href="#event-object-common-properties">event object</a> includes properties that are common for all events. Each event object includes a <code>payload</code> property and the value is unique to each event type. The <code>payload</code> object for this event is described below.</p>

       <h3 id="deleteevent"><a href="#deleteevent">DeleteEvent</a></h3> 
       <p>A Git branch or tag is deleted. For more information, see the "<a href="/en/rest/reference/git">Git data</a>" REST API.</p> 
       <p>The <a href="#event-object-common-properties">event object</a> includes properties that are common for all events. Each event object includes a <code>payload</code> property and the value is unique to each event type. The <code>payload</code> object for this event is described below.</p>
