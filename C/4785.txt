         <h1 id="user-docs">User Docs <span class="badge-trigger free"></span><a href="#user-docs" title="Permalink" class="anchor"></a> </h1> 
         <p>Welcome to GitLab! Were glad to have you here! </p>
         <p>As a GitLab user you have access to all the features your <a href="https://about.gitlab.com/pricing/">subscription</a> includes, except <a href="../administration/index.html">GitLab administrator</a> settings, unless you have admin privileges to install, configure, and upgrade your GitLab instance. </p>
         <p>Admin privileges for <a href="https://gitlab.com/">GitLab.com</a> are restricted to the GitLab team. </p>
         <p>For more information on configuring GitLab self-managed instances, see the <a href="../administration/index.html">Administrator documentation</a>. </p>
         <h2 id="overview">Overview<a href="#overview" title="Permalink" class="anchor"></a> </h2> 
         <p>GitLab is a fully integrated software development platform that enables your team to be transparent, fast, effective, and cohesive from discussion on a new idea to production, all on the same platform. </p>
         <p>For more information, see <a href="https://about.gitlab.com/features/">All GitLab Features</a>. </p>
         <h3 id="concepts">Concepts<a href="#concepts" title="Permalink" class="anchor"></a> </h3> 
         <p>To get familiar with the concepts needed to develop code on GitLab, read the following articles: </p>
         <ul> 
          <li> <a href="https://about.gitlab.com/blog/2017/03/17/demo-mastering-code-review-with-gitlab/">Demo: Mastering Code Review With GitLab</a>. </li>
          <li> <a href="https://about.gitlab.com/topics/version-control/what-is-gitlab-workflow/#gitlab-workflow-a-use-case-scenario">GitLab Workflow: An Overview</a>. </li>
          <li> <a href="https://about.gitlab.com/blog/2016/03/08/gitlab-tutorial-its-all-connected/">Tutorial: Its all connected in GitLab</a>: an overview on code collaboration with GitLab. </li>
          <li> <a href="https://about.gitlab.com/blog/2016/08/16/trends-in-version-control-land-microservices/">Trends in Version Control Land: Microservices</a>. </li>
          <li> <a href="https://about.gitlab.com/topics/version-control/what-is-innersource/">Trends in Version Control Land: Innersourcing</a>. </li>
         </ul> 
         <h2 id="use-cases">Use cases<a href="#use-cases" title="Permalink" class="anchor"></a> </h2> 
         <p>GitLab is a Git-based platform that integrates a great number of essential tools for software development and deployment, and project management: </p>
         <ul> 
          <li>Hosting code in repositories with version control. </li>
          <li>Tracking proposals for new implementations, bug reports, and feedback with a fully featured <a href="project/issues/index.html#issues-list">Issue Tracker</a>. </li>
          <li>Organizing and prioritizing with <a href="project/issue_board.html">Issue Boards</a>. </li>
          <li>Reviewing code in <a href="project/merge_requests/index.html">Merge Requests</a> with live-preview changes per branch with <a href="../ci/review_apps/index.html">Review Apps</a>. </li>
          <li>Building, testing, and deploying with built-in <a href="../ci/README.html">Continuous Integration</a>. </li>
          <li>Deploying personal and professional static websites with <a href="project/pages/index.html">GitLab Pages</a>. </li>
          <li>Integrating with Docker by using <a href="packages/container_registry/index.html">GitLab Container Registry</a>. </li>
          <li>Tracking the development lifecycle by using <a href="analytics/value_stream_analytics.html">GitLab Value Stream Analytics</a>. </li>
          <li>Provide support with <a href="project/service_desk.html">Service Desk</a>. </li>
          <li> <a href="project/issues/csv_export.html">Export issues as CSV</a>. </li>
         </ul> 
         <p>With GitLab Enterprise Edition, you can also: </p>
         <ul> 
          <li>Improve collaboration with: 
           <ul> 
            <li> <a href="project/merge_requests/merge_request_approvals.html">Merge Request Approvals</a>. </li>
            <li> <a href="project/issues/multiple_assignees_for_issues.html">Multiple Assignees for Issues</a>. </li>
            <li> <a href="project/issue_board.html#multiple-issue-boards">Multiple Issue Boards</a>. </li>
           </ul> </li>
          <li>Create formal relationships between issues with <a href="project/issues/related_issues.html">Related Issues</a>. </li>
          <li>Use <a href="project/milestones/burndown_and_burnup_charts.html">Burndown Charts</a> to track progress during a sprint or while working on a new version of their software. </li>
          <li>Leverage <a href="../integration/elasticsearch.html">Elasticsearch</a> with <a href="search/advanced_search.html">Advanced Search</a> for faster, more advanced code search across your entire GitLab instance. </li>
          <li> <a href="../integration/kerberos.html">Authenticate users with Kerberos</a>. </li>
          <li> <a href="project/repository/repository_mirroring.html">Mirror a repository</a> from elsewhere on your local server. </li>
          <li>View your entire CI/CD pipeline involving more than one project with <a href="../ci/multi_project_pipelines.html">Multiple-Project Pipelines</a>. </li>
          <li> <a href="project/file_lock.html">Lock files</a> to prevent conflicts. </li>
          <li>View the current health and status of each CI environment running on Kubernetes with <a href="project/deploy_boards.html">Deploy Boards</a>. </li>
          <li>Leverage continuous delivery method with <a href="project/canary_deployments.html">Canary Deployments</a>. </li>
          <li>Scan your code for vulnerabilities and <a href="application_security/sast/index.html">display them in merge requests</a>. </li>
         </ul> 
         <p>You can also <a href="project/integrations/overview.html">integrate</a> GitLab with numerous third-party applications, such as Mattermost, Microsoft Teams, HipChat, Trello, Slack, Bamboo CI, Jira, and a lot more. </p>
         <h2 id="user-types">User types<a href="#user-types" title="Permalink" class="anchor"></a> </h2> 
         <p>There are several types of users in GitLab: </p>
         <ul> 
          <li>Regular users and GitLab.com users. </li>
          <li> <a href="group/index.html">Groups</a> of users. </li>
          <li>GitLab <a href="admin_area/index.html">admin area</a> user. </li>
          <li> <a href="../administration/index.html">GitLab Administrator</a> with full access to self-managed instances features and settings. </li>
          <li> <a href="../development/internal_users.html">Internal users</a>. </li>
         </ul> 
         <h2 id="user-activity">User activity<a href="#user-activity" title="Permalink" class="anchor"></a> </h2> 
         <p>You can follow or unfollow other users from their <a href="profile/index.html#access-your-user-profile">user profiles</a>. To see their activity in the top-level Activity view, select Follow or Unfollow, and select the Followed Users tab: </p>
         <p><a href="img/activity_followed_users_v13_9.png" target="_blank" rel="noopener noreferrer"><img src="img/activity_followed_users_v13_9.png" alt="Follow users"></a> </p>
         <h2 id="projects">Projects<a href="#projects" title="Permalink" class="anchor"></a> </h2> 
         <p>In GitLab, you can create <a href="project/index.html">projects</a> to host your code, track issues, collaborate on code, and continuously build, test, and deploy your app with built-in GitLab CI/CD. Or, you can do it all at once, from one single project. </p>
         <ul> 
          <li> <a href="project/repository/index.html">Repositories</a>: Host your codebase in repositories with version control and as part of a fully integrated platform. </li>
          <li> <a href="project/issues/index.html">Issues</a>: Explore the best of GitLab Issues features. </li>
          <li> <a href="project/merge_requests/index.html">Merge Requests</a>: Collaborate on code, reviews, live preview changes per branch, and request approvals with Merge Requests. </li>
          <li> <a href="project/milestones/index.html">Milestones</a>: Work on multiple issues and merge requests towards the same target date with Milestones. </li>
         </ul> 
         <h2 id="account">Account<a href="#account" title="Permalink" class="anchor"></a> </h2> 
         <p>There is a lot you can customize and configure to enjoy the best of GitLab. </p>
         <ul> 
          <li> <a href="profile/index.html">Settings</a>: Manage your user settings to change your personal information, personal access tokens, authorized applications, etc. </li>
          <li> <a href="../topics/authentication/index.html">Authentication</a>: Read through the authentication methods available in GitLab. </li>
          <li> <a href="permissions.html">Permissions</a>: Learn the different set of permissions levels for each user type (guest, reporter, developer, maintainer, owner). </li>
          <li> <a href="feature_highlight.html">Feature highlight</a>: Learn more about the little blue dots around the app that explain certain features. </li>
          <li> <a href="abuse_reports.html">Abuse reports</a>: Report abuse from users to GitLab administrators. </li>
         </ul> 
         <h2 id="groups">Groups<a href="#groups" title="Permalink" class="anchor"></a> </h2> 
         <p>With GitLab <a href="group/index.html">Groups</a> you can assemble related projects together and grant members access to several projects at once. </p>
         <p>Groups can also be nested in <a href="group/subgroups/index.html">subgroups</a>. </p>
         <h2 id="discussions">Discussions<a href="#discussions" title="Permalink" class="anchor"></a> </h2> 
         <p>In GitLab, you can comment and mention collaborators in issues, merge requests, code snippets, and commits. </p>
         <p>When performing inline reviews to implementations to your codebase through merge requests you can gather feedback through <a href="discussions/index.html#resolvable-comments-and-threads">resolvable threads</a>. </p>
         <h3 id="gitlab-flavored-markdown-gfm">GitLab Flavored Markdown (GFM)<a href="#gitlab-flavored-markdown-gfm" title="Permalink" class="anchor"></a> </h3> 
         <p>Read through the <a href="markdown.html">GFM documentation</a> to learn how to apply the best of GitLab Flavored Markdown in your threads, comments, issues and merge requests descriptions, and everywhere else GFM is supported. </p>
         <h2 id="to-do-list">To-Do List<a href="#to-do-list" title="Permalink" class="anchor"></a> </h2> 
         <p>Never forget to reply to your collaborators. <a href="todos.html">GitLab To-Do List</a> is a tool for working faster and more effectively with your team, by listing all user or group mentions, as well as issues and merge requests youre assigned to. </p>
         <h2 id="search">Search<a href="#search" title="Permalink" class="anchor"></a> </h2> 
         <p><a href="search/index.html">Search and filter</a> through groups, projects, issues, merge requests, files, code, and more. </p>
         <h2 id="snippets">Snippets<a href="#snippets" title="Permalink" class="anchor"></a> </h2> 
         <p><a href="snippets.html">Snippets</a> are code blocks that you want to store in GitLab, from which you have quick access to. You can also gather feedback on them through <a href="#discussions">Discussions</a>. </p>
         <h2 id="gitlab-cicd">GitLab CI/CD<a href="#gitlab-cicd" title="Permalink" class="anchor"></a> </h2> 
         <p>Use built-in <a href="../ci/README.html">GitLab CI/CD</a> to test, build, and deploy your applications directly from GitLab. No third-party integrations needed. </p>
         <h2 id="features-behind-feature-flags">Features behind feature flags<a href="#features-behind-feature-flags" title="Permalink" class="anchor"></a> </h2> 
         <p>Understand what <a href="feature_flags.html">features behind feature flags</a> mean. </p>
         <h2 id="keyboard-shortcuts">Keyboard shortcuts<a href="#keyboard-shortcuts" title="Permalink" class="anchor"></a> </h2> 
         <p>There are many <a href="shortcuts.html">keyboard shortcuts</a> in GitLab to help you navigate between pages and accomplish tasks faster. </p>
         <h2 id="integrations">Integrations<a href="#integrations" title="Permalink" class="anchor"></a> </h2> 
         <p><a href="../integration/README.html">Integrate GitLab</a> with your preferred tool, such as Trello, Jira, etc. </p>
         <h2 id="webhooks">Webhooks<a href="#webhooks" title="Permalink" class="anchor"></a> </h2> 
         <p>Configure <a href="project/integrations/webhooks.html">webhooks</a> to listen for specific events like pushes, issues or merge requests. GitLab sends a POST request with data to the webhook URL. </p>
         <h2 id="api">API<a href="#api" title="Permalink" class="anchor"></a> </h2> 
         <p>Automate GitLab via <a href="../api/README.html">API</a>. </p>
         <h2 id="git-and-gitlab">Git and GitLab<a href="#git-and-gitlab" title="Permalink" class="anchor"></a> </h2> 
         <p>Learn what is <a href="../topics/git/index.html">Git</a> and its best practices. </p>
         <h2 id="instance-level-analytics">Instance-level analytics<a href="#instance-level-analytics" title="Permalink" class="anchor"></a> </h2> 
         <p>See <a href="admin_area/analytics/index.html">various statistics</a> of your GitLab instance. </p>
         <h2 id="operations-dashboard">Operations Dashboard<a href="#operations-dashboard" title="Permalink" class="anchor"></a> </h2> 
         <p>See <a href="operations_dashboard/index.html">Operations Dashboard</a> for a summary of each projects operational health. </p>
