      <p>A Wikipedia <b>namespace</b> is a set of Wikipedia pages whose <a href="/wiki/Help:Page_name" title="Help:Page name">names</a> begin with a particular reserved word recognized by the <a href="/wiki/MediaWiki" title="MediaWiki">MediaWiki</a> software (followed by a colon). For example, in the <i>user namespace</i> all titles begin with the prefix <i>User:</i>. In the case of the article (or main) namespace, in which encyclopedia articles appear, the reserved word and colon are absent. (Note: main space articles can optionally include a colon at the beginning with no reserved word, so <code>[[Article]]</code> and <code>[[:Article]]</code> are equivalent. This is usually only necessary for the advanced technique of <a href="/wiki/Wikipedia:Transclusion" class="mw-redirect" title="Wikipedia:Transclusion">transcluding</a> a main space article into another page.) </p>
      <p>Wikipedia has 34 current namespaces: 16 subject namespaces, 16 corresponding talk namespaces, and 2 virtual namespaces. These are all listed in the box to the right. While surfing Wikipedia, a list of namespaces is available in two clicks: do an empty search, then click <i>Advanced</i> under the search box. The list also makes itself available where needed in the dropdown menus of, for example, <a href="/wiki/Special:PrefixIndex" title="Special:PrefixIndex">Special:Prefixindex</a>, <a href="/wiki/Special:AllPages" title="Special:AllPages">Special:Allpages</a>, <a href="/wiki/Special:RecentChanges" title="Special:RecentChanges">Special:Recentchanges</a>, and <a href="/wiki/Special:Contributions" title="Special:Contributions">Special:Contributions</a>. </p>
      <p>The aliases <b>WP</b> for <b>Wikipedia</b>, and <b>WT</b> for <b>Wikipedia&nbsp;talk</b>, work for page names in the search box, and making links as well as inclusions. See <a href="#Aliases">Aliases</a> below for other aliases. (The pseudo-namespaces such as <b>H</b> for <b>Help</b>, and <b>CAT</b> for <b>Category</b>, are extremely limited. See <i><a href="#Pseudo-namespaces">Pseudo-namespaces</a></i> below.) </p>
      <p>The table on the right shows what number to use when you want to hide pages that are on your watchlist. See: <a href="/wiki/Wikipedia:Hide_Pages_in_Watchlist" class="mw-redirect" title="Wikipedia:Hide Pages in Watchlist">Wikipedia:Hide Pages in Watchlist</a> for details. </p>
      <p>Previously, the Topic namespace was available for the <a href="/wiki/Wikipedia:Flow" title="Wikipedia:Flow">Flow project</a> but it has since been turned off on this wiki. </p>

      <h2><span class="mw-headline" id="Subject_namespaces">Subject namespaces</span></h2> 
      <div role="note" class="hatnote navigation-not-searchable">
       Further information: <a href="/wiki/Wikipedia:Administration#Data_structure_and_development" title="Wikipedia:Administration">Wikipedia:Administration §&nbsp;Data structure and development</a>

      <p>Namespaces allow for the organization and separation of <a href="/wiki/Wikipedia:Administration#Content_namespaces" title="Wikipedia:Administration">content pages</a> from <a href="/wiki/Wikipedia:Administration#Administrative_namespaces" title="Wikipedia:Administration">administration pages</a>. Namespaces separate data into core sets, those intended for public viewing, and those intended for the editing community. Wikipedia's subject namespaces and their functions are listed below. A brief description is provided, but see the link for more information. </p>
      <p><span class="anchor" id="subjectpage"></span>A namespace is sometimes called a <i>space</i>, for short, as in "Project space". A <i>subject page</i> and its <i>talk page</i> form a pair. </p> 
      <h3><span class="mw-headline" id="In_use">In use</span></h3> 
      <ul>
       <li><b><a href="/wiki/Wikipedia:Main_namespace" class="mw-redirect" title="Wikipedia:Main namespace">Main namespace</a></b> (no prefix): contains all encyclopedia articles, lists, <a href="/wiki/Wikipedia:Disambiguation" title="Wikipedia:Disambiguation">disambiguation pages</a>, and encyclopedia <a href="/wiki/Wikipedia:Redirect" title="Wikipedia:Redirect">redirects</a>. Sometimes referred to as "mainspace".</li> 
       <li><b><a href="/wiki/Wikipedia:User_namespace" class="mw-redirect" title="Wikipedia:User namespace">User namespace</a></b> (prefix <b>User:</b>): contains user pages and other pages created by individual users for their own personal use. Pages under this namespace can still be viewed and modified by others, so do not keep any of your sensitive data here.</li> 
       <li><b><a href="/wiki/Wikipedia:Wikipedia_namespace" class="mw-redirect" title="Wikipedia:Wikipedia namespace">Wikipedia namespace</a></b> or <b><a href="/wiki/Wikipedia:Project_namespace" title="Wikipedia:Project namespace">Project namespace</a></b> (prefix <b>Wikipedia:</b>): contains many types of pages connected with the Wikipedia project itself: information, policies, guidelines, essays, processes, discussion, etc. <a href="/wiki/Wikipedia:Namespace#Aliases" title="Wikipedia:Namespace">Namespace aliases</a>: <b>WP:</b> or <b>Project:</b></li> 
       <li><span class="anchor" id="File"></span><span class="anchor" id="File_namespace"></span><b><a href="/wiki/Wikipedia:File_namespace" class="mw-redirect" title="Wikipedia:File namespace">File namespace</a></b> (prefix <b>File:</b>): contains <a href="/wiki/Help:File_description_page" title="Help:File description page">description pages for media files</a> (images, videos, and audio files). A link starting with <code>[[File:</code> will display the media at that position on the page instead of showing a wikilink that leads to the specified file description page, so if you want the latter effect, use a link beginning with <code>[[:File:</code> (the <a href="/wiki/Help:Colon_trick" title="Help:Colon trick">colon trick</a>). Namespace alias: <b>Image:</b></li> 
       <li><b><a href="/wiki/Help:MediaWiki_namespace" title="Help:MediaWiki namespace">MediaWiki namespace</a></b> (prefix <b>MediaWiki:</b>): a namespace containing interface texts, such as the links and messages that appear on automatically generated pages. Pages in this namespace are permanently <a href="/wiki/Wikipedia:Protection_policy" title="Wikipedia:Protection policy">protected</a> against editing by regular users. For a list of these messages, see <a href="/wiki/Special:AllMessages" title="Special:AllMessages">Special:AllMessages</a>. (The prefix <i>cannot</i> be shortened to "MW" because <b>mw</b> is an interwiki prefix. See <i><a href="#Interwiki_links">Interwiki links</a></i> below.)</li> 
       <li><b><a href="/wiki/Wikipedia:Template_namespace" title="Wikipedia:Template namespace">Template namespace</a></b> (prefix <b>Template:</b>): contains templates&nbsp;– pages that are intended primarily to be <a href="/wiki/Wikipedia:Transclusion" class="mw-redirect" title="Wikipedia:Transclusion">transcluded</a> or <a href="/wiki/Wikipedia:Substitution" title="Wikipedia:Substitution">substituted</a> onto other pages to insert standard text or boxes such as infoboxes and navigation boxes.</li> 
       <li><b><a href="/wiki/Wikipedia:Help_namespace" title="Wikipedia:Help namespace">Help namespace</a></b> (prefix <b>Help:</b>): contains pages which provide help in using Wikipedia and its software, both for users of the encyclopedia and for editors.</li> 
       <li><b><a href="/wiki/Wikipedia:Category_namespace" class="mw-redirect" title="Wikipedia:Category namespace">Category namespace</a></b> (prefix <b>Category:</b>): contains category pages, which are curated lists of related pages and subcategories, along with optional additional text. A link beginning with <code>[[Category:</code> has the effect of adding the page the link appears on to the specified category; it will not result in a wikilink to the category page itself. If you want the latter, use the <a href="/wiki/Help:Colon_trick" title="Help:Colon trick">colon trick</a>: <code>[[:Category:</code>.</li> 
       <li><b><a href="/wiki/Wikipedia:Portal_namespace" class="mw-redirect" title="Wikipedia:Portal namespace">Portal namespace</a></b> (prefix <b>Portal:</b>): for reader-oriented portals that help readers find articles related to a specific topic, and may contain links to encourage contributions to relevant <a href="/wiki/Wikipedia:WikiProject" title="Wikipedia:WikiProject">WikiProjects</a>. See, for example, <a href="/wiki/Portal:Cricket" title="Portal:Cricket">Portal:Cricket</a> and <a href="/wiki/Portal:Spaceflight" title="Portal:Spaceflight">Portal:Spaceflight</a>.</li> 
       <li><b><a href="/wiki/Wikipedia:Draft_namespace" class="mw-redirect" title="Wikipedia:Draft namespace">Draft namespace</a></b> (prefix <b>Draft:</b>): hosts drafts of new (potential) Wikipedia articles.</li> 
       <li><b><a href="https://www.mediawiki.org/wiki/Extension:TimedMediaHandler" class="extiw" title="mw:Extension:TimedMediaHandler">TimedText namespace</a></b> (prefix <b>TimedText:</b>): synchronized subtitles for media files.</li> 
       <li><b><a href="/wiki/Wikipedia:Module_namespace" class="mw-redirect" title="Wikipedia:Module namespace">Module namespace</a></b> (prefix <b>Module:</b>): contains <a href="https://www.mediawiki.org/wiki/Extension:Scribunto" class="extiw" title="mw:Extension:Scribunto">Scribunto</a> modules&nbsp;– Lua scripts implementing special-purpose <a href="https://www.mediawiki.org/wiki/Manual:Parser_functions" class="extiw" title="mw:Manual:Parser functions">parser functions</a> for use in templates, beyond the <a href="/wiki/Help:Magic_words#Parser_functions" title="Help:Magic words">basic set</a> included with MediaWiki and its <a href="https://www.mediawiki.org/wiki/Extensions_FAQ" class="extiw" title="mw:Extensions FAQ">extensions</a>.</li>
      </ul>

      <p>On <a href="/wiki/MediaWiki" title="MediaWiki">MediaWiki</a> installations other than the English Wikipedia, additional namespaces may be installed by <a href="https://www.mediawiki.org/wiki/Manual:Extensions" class="extiw" title="mw:Manual:Extensions">MediaWiki extensions</a>; the list of default IDs for these namespaces is at <a href="https://www.mediawiki.org/wiki/Extension_default_namespaces" class="extiw" title="mw:Extension default namespaces">Extension default namespaces</a>. </p>
      <p>The <b><a href="/wiki/Wikipedia:Flow" title="Wikipedia:Flow">Flow namespace</a></b> (prefix <b>Topic:</b>) was a proposed talk page replacement, formerly installed on English Wikipedia by <a href="https://www.mediawiki.org/wiki/Extension:StructuredDiscussions" class="extiw" title="mw:Extension:StructuredDiscussions">Extension:StructuredDiscussions</a>. It is no longer installed after being rejected by the community. </p> 
      <h3><span class="mw-headline" id="Aliases_and_pseudo-namespaces">Aliases and pseudo-namespaces</span></h3> 
      <p>An <i>alias</i> of a namespace is its namespace. For example, <b>WP:</b> is <b>Wikipedia:</b>, and so <code>[[Wikipedia:<a href="/wiki/Wikipedia:Page_name" title="Wikipedia:Page name"><i>Page name</i></a>]]</code> can be abbreviated <code>[[WP:<a href="/wiki/Wikipedia:Page_name" title="Wikipedia:Page name"><i>Page name</i></a>]]</code>. See <i><a href="#Aliases">Aliases</a></i> below, for a list of aliases, such as <b>WT:</b> for <b>Wikipedia talk:</b>. A <i>pseudo-namespace</i> is an agreement to create redirect pages named that way, each of which is a shortcut from mainspace to <i>a single page in its namespace</i>. So an alias refers to <i>any page in its namespace</i>, but a pseudo-namespace has far fewer pages than its real namespace. There are a few pseudo-namespace names like <b>H:</b> for <b>Help:</b>, and <b>CAT:</b> for <b>Category:</b>, and these can be discovered below in the section <i><a href="#Pseudo-namespaces">Pseudo-namespaces</a></i>. </p> 
      <h2><span class="mw-headline" id="Talk_namespaces"><span id="Talk"></span><span id="Wikipedia_talk"></span><span id="WT"></span> Talk namespaces</span></h2> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:TALKSPACE"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:TALKSPACE&amp;redirect=no">WP:TALKSPACE</a></span></li>
       </ul>
      </div> 
      <div role="note" class="hatnote navigation-not-searchable">
       Main pages: <a href="/wiki/Help:Using_talk_pages" class="mw-redirect" title="Help:Using talk pages">Help:Using talk pages</a> and <a href="/wiki/Wikipedia:Talk_page_guidelines" title="Wikipedia:Talk page guidelines">Wikipedia:Talk page guidelines</a>
      </div> 
      <p>Each of the above namespaces (but not the virtual namespaces and the former Topic: namespace) has an associated <a href="/wiki/Help:Talk_page" class="mw-redirect" title="Help:Talk page">talk namespace</a>—these are also known as <i>discussion pages</i>. The talk namespaces are designated by appending the word <code>talk</code> to the namespace name. For example, the talk namespace associated with the user namespace has the prefix <b>User talk:</b>. The talk namespace associated with the article namespace is <b>Talk:</b>. </p>
      <p>Most of the pages in the talk namespaces are used to discuss changes to the corresponding page in the associated namespace. Pages in the <i>user talk</i> namespace are used to leave messages for a particular user. The user talk namespace is special in that, whenever a user's talk page is edited, that user will see an orange box saying "You have new messages" on the top of every page that they view until they visit their talk page. By default, logged-in users will see <a href="/wiki/Wikipedia:Notifications" class="mw-redirect" title="Wikipedia:Notifications">a red notification square</a> <i>and</i> a small orange box on the top right hand corner of the page; IP users will only see a wide orange box spanning the top of the page. (Minor edits made by <a href="/wiki/Wikipedia:Bot_policy" title="Wikipedia:Bot policy">bots</a> no longer set off the new message bar. For logged-in users, the new-message bar may be disabled in <a href="/wiki/Special:Preferences" title="Special:Preferences">Special:Preferences</a>.) </p>
      <p>Note that the prefix <b>WT:</b> expands to <b>Wikipedia talk:</b>, just as <b>WP:</b> expands to <b>Wikipedia:</b> (see <a href="#Aliases">Aliases</a> below). For example, [[WT:Verifiability]] links to <a href="/wiki/Wikipedia_talk:Verifiability" title="Wikipedia talk:Verifiability">Wikipedia talk:Verifiability</a>. </p> 
      <h2><span class="mw-headline" id="Virtual_namespaces">Virtual namespaces<span id="Special"></span></span></h2> 
      <p>In addition to subject namespaces, and their corresponding talk pages, there are two virtual namespaces (without corresponding talk pages) for specialist purposes: </p> 
      <h3><span class="mw-headline" id="Special">Special</span></h3> 
      <div role="note" class="hatnote navigation-not-searchable">
       Main page: <a href="/wiki/Help:Special_page" title="Help:Special page">Help:Special page</a>
      </div> 
      <p>The <i>Special:</i> namespace consists of pages (called <i>special pages</i>) that are created by the software on demand, such as <a href="/wiki/Special:RecentChanges" title="Special:RecentChanges">Special:RecentChanges</a>. These pages can be linked as usual, as with [[Special:RecentChanges]], except when they have parameters. To use parameters, the full URL must be given as an external link. For example, <a class="external free" href="https://en.wikipedia.org/w/index.php?title=Special:Recentchanges&amp;days=3&amp;limit=10">https://en.wikipedia.org/w/index.php?title=Special:Recentchanges&amp;days=3&amp;limit=10</a>, which returns the last ten changes in the last three days. </p>
      <p>You can create a <a href="/wiki/Wikipedia:Redirect" title="Wikipedia:Redirect">redirect</a> to a Special page, but the redirect will not be followed automatically. </p>
      <p>For a list of special pages, see <a href="/wiki/Special:SpecialPages" title="Special:SpecialPages">Special:SpecialPages</a>. </p>
      <p><span class="anchor" id="Media"></span><span class="anchor" id="Media_namespace"></span> </p> 
      <h3><span class="mw-headline" id="Media">Media</span></h3> 
      <div role="note" class="hatnote navigation-not-searchable">
       Main page: <a href="/wiki/Help:Files" title="Help:Files">Help:Files</a>
      </div> 
      <p>The <i>Media:</i> namespace can be used to link directly to a file, rather than to the file description page. </p> 
      <h2><span class="mw-headline" id="Aliases_and_pseudo-namespaces_2">Aliases and pseudo-namespaces</span></h2> 
      <div role="note" class="hatnote navigation-not-searchable">
       Further information: <a href="/wiki/Wikipedia:Shortcut#List_of_prefixes" title="Wikipedia:Shortcut">Wikipedia:Shortcut §&nbsp;List of prefixes</a>
      </div> 
      <p>Aliases and pseudo-namespaces are not namespaces in themselves; their main role is to provide handy ways to link to actual namespaces, by providing shortened links. </p>
      <p>The main difference between them being the way they work for this purpose; while aliases directly link to namespaces, pseudo-namespaces can only link to other namespaces by using specialist <a href="/wiki/Wikipedia:Redirect" title="Wikipedia:Redirect">redirect pages</a> in the main namespace called <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">shortcuts</a>. While aliases are also used for shortcuts, this isn't necessary if the pagename is not shortened. </p>
      <p>Pseudo-namespaces all actually reside in main namespace; but a pseudo-namespace called <a href="https://meta.wikimedia.org/wiki/Help:Transwiki" class="extiw" title="m:Help:Transwiki">Transwiki:</a> exists as a pseudo-namespace before being moved to another namespace. </p> 
      <h3><span class="mw-headline" id="Aliases">Aliases</span></h3> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:ALIAS"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:ALIAS&amp;redirect=no">WP:ALIAS</a></span></li>
       </ul>
      </div> 
      <p>There are six aliases defined for namespaces, all of which are case-insensitive: </p> 
      <table class="wikitable"> 
       <tbody>
        <tr> 
         <th>Alias </th> 
         <th>Namespace </th>
        </tr> 
        <tr> 
         <td>WP</td>

        </tr>

         <td rowspan="2">Wikipedia talk </td>
        </tr> 
        <tr>

        </tr> 
        <tr>

        </tr>

      <p>These aliases are automatically translated by the Wikipedia servers to the proper namespace prefix. So if a link is made to a page title beginning with one of these aliases, or else entered in the search box or used in a URL in a browser address bar, the alias links to the page with the corresponding true namespace prefix. Hence entering <a href="/wiki/Wikipedia:External_links" title="Wikipedia:External links">WP:External links</a> or <a href="/wiki/Wikipedia:External_links" title="Wikipedia:External links">Project:External links</a> is equivalent to entering <a href="/wiki/Wikipedia:External_links" title="Wikipedia:External links">Wikipedia:External links</a>; and entering <a href="/wiki/Wikipedia_talk:External_links" title="Wikipedia talk:External links">WT:External links</a> or <a href="/wiki/Wikipedia_talk:External_links" title="Wikipedia talk:External links">Project talk:External links</a> is equivalent to entering <a href="/wiki/Wikipedia_talk:External_links" title="Wikipedia talk:External links">Wikipedia talk:External links</a>. </p>
      <p>An alias cannot be used as the actual namespace in a page title. For example, you cannot create page <i>WP:123</i>; if tried, the page name is instantly converted to <i>Wikipedia:123</i>. It <i>is</i>, however, possible to <a href="/wiki/Wikipedia:Page_name#Changing_the_displayed_title" title="Wikipedia:Page name">change the <i>displayed</i> title</a> on page <i>Wikipedia:123</i> to make it <i>WP:123</i> or <i>Project:123</i>. </p>
      <p><span class="plainlinks">This is very handy for the creation of <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">shortcuts</a>. For example, the shortcut for this section is <a href="/wiki/Wikipedia:ALIAS" class="mw-redirect" title="Wikipedia:ALIAS">WP:ALIAS</a>, as shown in the right-side box. When the link is clicked, it automatically translated to <a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:ALIAS&amp;redirect=no">Wikipedia:ALIAS</a>. At this page is a <a href="/wiki/Help:Redirect" title="Help:Redirect">redirect</a> to <a href="/wiki/Wikipedia:Namespace#Aliases" title="Wikipedia:Namespace">Wikipedia:Namespace#Aliases</a>. This means that the shortcut goes directly to this section from anywhere in Wikipedia. It can also be entered as a <a href="/wiki/URL" title="URL">URL</a> in the browser address bar as <a class="external free" href="https://en.wikipedia.org/wiki/WP:ALIAS">https://en.wikipedia.org/wiki/WP:ALIAS</a> .</span> </p> 
      <h3><span class="mw-headline" id="Pseudo-namespaces">Pseudo-namespaces</span></h3> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:PNS"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:PNS&amp;redirect=no">WP:PNS</a></span></li>
       </ul>
      </div> 
      <p>?part from Wikipedia's 32 regular namespaces and their aliases, and 2 virtual ones, there are several title prefixes appearing in <a href="/wiki/Help:Shortcut" class="mw-redirect" title="Help:Shortcut">shortcut redirects</a> pointing to frequently-referred-to Wikipedia pages. These prefixes are referred to as <b>pseudo-namespaces</b> and include the following: </p> 
      <ul>
       <li><b><a href="/wiki/Special:PrefixIndex/CAT:" title="Special:PrefixIndex/CAT:">CAT:</a></b> for shortcuts to the <a href="/wiki/Wikipedia:Category_namespace" class="mw-redirect" title="Wikipedia:Category namespace">Category namespace</a>. See also <a href="/wiki/Category:Redirects_to_category_space" title="Category:Redirects to category space">Category:Redirects to category space</a></li> 
       <li><b><a href="/wiki/Special:PrefixIndex/H:" title="Special:PrefixIndex/H:">H:</a></b> for shortcuts to the <a href="/wiki/Wikipedia:Help_namespace" title="Wikipedia:Help namespace">Help namespace</a>. See also <a href="/wiki/Category:Redirects_to_help_namespace" title="Category:Redirects to help namespace">Category:Redirects to help namespace</a></li> 
       <li><b><a href="/wiki/Special:PrefixIndex/MOS:" title="Special:PrefixIndex/MOS:">MOS:</a></b> for shortcuts to the <a href="/wiki/Wikipedia:Manual_of_Style" title="Wikipedia:Manual of Style">Manual of Style</a>. See also <a href="/wiki/Category:Redirects_to_project_space" title="Category:Redirects to project space">Category:Redirects to project space</a></li> 
       <li><b><a href="/wiki/Special:PrefixIndex/P:" title="Special:PrefixIndex/P:">P:</a></b> for shortcuts to the <a href="/wiki/Wikipedia:Portal_namespace" class="mw-redirect" title="Wikipedia:Portal namespace">Portal namespace</a>. See also <a href="/wiki/Category:Redirects_to_portal_space" title="Category:Redirects to portal space">Category:Redirects to portal space</a></li> 
       <li><b><a href="/wiki/Special:PrefixIndex/T:" title="Special:PrefixIndex/T:">T:</a></b> for shortcuts to the <a href="/wiki/Wikipedia:Template_namespace" title="Wikipedia:Template namespace">Template namespace</a>. See also <a href="/wiki/Category:Redirects_to_template_namespace" title="Category:Redirects to template namespace">Category:Redirects to template namespace</a></li>
      </ul> 
      <p>A more complete list is available at <a href="/wiki/Wikipedia:Shortcut#List_of_prefixes" title="Wikipedia:Shortcut">Wikipedia:Shortcut#List of prefixes</a>. </p>
      <p>Pseudo-namespaces are not in any way recognised by the wiki software; they are purely a community custom. Titles in pseudo-namespaces actually belong technically in the main (article) namespace and are treated as such by the software: they are case-sensitive and appear in <a href="/wiki/Help:Search#Syntax" class="mw-redirect" title="Help:Search">search results</a> restricted to the main namespace. An alias is treated like a real namespace, resulting in a search for the <a href="/wiki/Wikipedia:Page_name" title="Wikipedia:Page name">pagename</a> in its namespace, but the "pseudo-namespace:pagename" search is in mainspace, not its pseudo-namespace. For example, searching for "H:S" will not search <b>Help</b>. </p>
      <p>To learn some of these shortcuts, first follow the pseudo-namespace link above, then follow one of the redirects listed on that page. It will take you to the target, where you then notice any {{<a href="/wiki/Template:Shortcut" title="Template:Shortcut">shortcut</a>}} boxes to the right of the page, which may also list some additional shortcuts to that same area. By observing these steps repeatedly, you can discover ways to shorten your typing. You will use pseudo-namespace names more often if you are an advanced editor, highly active in many namespaces, or constantly linking to pages that have a pseudo-namespace shorcut. </p>
      <p>To understand the appropriateness of redirects of this type, see <a href="/wiki/Wikipedia:Cross-namespace_redirects" title="Wikipedia:Cross-namespace redirects">Wikipedia:Cross-namespace redirects</a>. All shortcuts are discoverable by <a href="/wiki/Help:Searching" title="Help:Searching">searching</a> for redirects. There is also the category <a href="/wiki/Category:Wikipedia_redirects" title="Category:Wikipedia redirects">Wikipedia redirects</a>. </p>
      <p>As a final example, every <a href="/wiki/Wikimedia_Foundation" title="Wikimedia Foundation">Wikimedia Foundation</a> project has a "<a href="https://meta.wikimedia.org/wiki/Help:Transwiki" class="extiw" title="m:Help:Transwiki">Transwiki:</a>" pseudo-namespace (sometimes a real namespace) for importing articles manually. A wiki project that would manually move many pages between <a href="/wiki/Wikipedia:Sister" class="mw-redirect" title="Wikipedia:Sister">sister projects</a> uses this tag. Redirects/shortcuts are not directly involved here, as described above. Instead the page title is <a href="/wiki/Wikipedia:Move" class="mw-redirect" title="Wikipedia:Move">moved</a> to "Transwiki:<i>namespace:pagename</i>", and that content is then copied and pasted into a new page name of that temporary title on the target wiki. When it is accepted the title is then moved out of the <b>Transwiki</b> pseudo-namespace there and becomes a real page name in its respective namespace. </p>

       Further information: <a href="/wiki/Help:Interwiki_linking" title="Help:Interwiki linking">Help:Interwiki linking</a> and <a href="/wiki/Help:Interlanguage_links" title="Help:Interlanguage links">Help:Interlanguage links</a>
      </div> 
      <p><a href="https://meta.wikimedia.org/wiki/Table_of_Wikimedia_projects" class="extiw" title="m:Table of Wikimedia projects">Interwiki</a> and <a href="https://meta.wikimedia.org/wiki/List_of_Wikipedias" class="extiw" title="m:List of Wikipedias">interlanguage</a> prefixes do not define namespaces, but point to pages in other Wikimedia projects, outside the en.wikipedia.org website. They are mentioned here because they use a namespace-like prefix technique for linking. Examples of interwiki prefixes include <b>w:</b> for Wikipedia; <b>m:</b> (or <b>meta:</b>) for <a href="/wiki/Meta-Wiki" class="mw-redirect" title="Meta-Wiki">Meta-Wiki</a>, <b>mw:</b> for <a href="/wiki/MediaWiki" title="MediaWiki">MediaWiki</a>, and <b>wikt:</b> for <a href="/wiki/Wiktionary" title="Wiktionary">Wiktionary</a>. Examples of interlanguage prefixes include <b>en:</b> for English language, <b>fr:</b> for French language, and <b>de:</b> for German language. </p>
      <p>Important points to note: </p> 
      <ul>
       <li>In the presence of an interwiki prefix and the absence of an interlanguage prefix, the link will be to a specified project in the current language. So <code>[[w:Internet]]</code> links to "<a href="/wiki/Internet" title="Internet">Internet</a>" on the English Wikipedia, if the link is on any English-language Wikimedia project.</li> 
       <li>In the presence of a interlanguage prefix and the absence of an interwiki prefix, the link will be to a specified language in the current project. So <code>[[:en:Internet]]</code> will link to "<a href="/wiki/Internet" title="Internet">Internet</a>" on the English Wikipedia, if the link is on a Wikipedia project in any language.</li>
      </ul> 
      <p>When making links to other language Wikipedias, add an initial colon if you want the link to appear inline rather than in the skin's sidebar under "Languages"; see <a href="/wiki/Help:Colon_trick" title="Help:Colon trick">Help:Colon trick</a> for an explanation. So <code>[[:de:Internet]]</code> is an inline link to "<a href="https://de.wikipedia.org/wiki/Internet" class="extiw" title="de:Internet">Internet</a>" in the German Wikipedia; but if using an interwiki prefix this is unnecessary, for example <code>[[w:de:Internet]]</code> is the same inline link. </p>
      <p>Further examples: </p> 
      <ul>
       <li><a href="https://meta.wikimedia.org/wiki/Meta:Babel" class="extiw" title="m:Meta:Babel">m:Meta:Babel</a> and <a href="https://meta.wikimedia.org/wiki/Meta:Babel" class="extiw" title="meta:Meta:Babel">meta:Meta:Babel</a> both link to the "Babel" page in the project namespace (Meta:, the 2nd prefix) of Meta-Wiki (m: or meta:, the 1st prefix).</li> 
       <li><a href="https://en.wiktionary.org/wiki/fr:Discuter:pied/fusion_daf" class="extiw" title="wikt:fr:Discuter:pied/fusion daf">wikt:fr:Discuter:pied/fusion daf</a> links to the "pied/fusion daf" page in the talk namespace (Discuter:, the 3rd prefix) of French (fr:, the 2nd prefix) Wiktionary (wikt:, the 1st prefix).</li>
      </ul> 
      <p>Interwiki prefixes are not just for Wikimedia projects. For example, <a href="http://www.dict.org/bin/Dict?Database=*&amp;Form=Dict1&amp;Strategy=*&amp;Query=" class="extiw" title="dictionary:">Dictionary:</a> is the interwiki link for <a href="/wiki/Dict.org" class="mw-redirect" title="Dict.org">dict.org</a> (the DICT Development Group). For a complete list of interwiki prefixes, see the <a href="https://meta.wikimedia.org/wiki/Interwiki_map" class="extiw" title="m:Interwiki map">interwiki map</a> and <a href="https://meta.wikimedia.org/wiki/Special:Interwiki" class="extiw" title="m:Special:Interwiki">interwiki table</a>. </p>
      <p>As with aliases, it is not possible to create a page with a title beginning with an interwiki or interlanguage prefix. It is also not possible to create redirects to interwiki pages, unless it is a <a href="/wiki/Wikipedia:Soft_redirect" title="Wikipedia:Soft redirect">soft redirect</a>. </p> 
      <h2><span class="mw-headline" id="Case_insensitivity">Case insensitivity</span></h2> 
      <p>Standard namespace names, aliases and interwiki prefixes are case insensitive. The same applies to the first character after the colon. Hence the following all link to the same page: </p> 
      <ul>
       <li><a href="/wiki/Wikipedia:Afd" class="mw-redirect" title="Wikipedia:Afd">Wikipedia:Afd</a>, <a href="/wiki/Wikipedia:Afd" class="mw-redirect" title="Wikipedia:Afd">WIKIPEDIA:Afd</a>, <a href="/wiki/Wikipedia:Afd" class="mw-redirect" title="Wikipedia:Afd">wIKIpedia:Afd</a>, <a href="/wiki/Wikipedia:Afd" class="mw-redirect" title="Wikipedia:Afd">wikipedia:afd</a></li> 
       <li><a href="/wiki/Wikipedia:Afd" class="mw-redirect" title="Wikipedia:Afd">wP:Afd</a>, <a href="/wiki/Wikipedia:Afd" class="mw-redirect" title="Wikipedia:Afd">wp:Afd</a>, <a href="/wiki/Wikipedia:Afd" class="mw-redirect" title="Wikipedia:Afd">WP:afd</a></li> 
       <li><a href="/wiki/Wikipedia:Afd" class="mw-redirect" title="Wikipedia:Afd">PROJECT:Afd</a>, <a href="/wiki/Wikipedia:Afd" class="mw-redirect" title="Wikipedia:Afd">project:Afd</a>, <a href="/wiki/Wikipedia:Afd" class="mw-redirect" title="Wikipedia:Afd">ProjecT:Afd</a>, <a href="/wiki/Wikipedia:Afd" class="mw-redirect" title="Wikipedia:Afd">Project:afd</a></li>
      </ul> 
      <h2><span class="mw-headline" id="Namespace-dependent_functionality">Namespace-dependent functionality</span></h2> 
      <p>The namespace name is a useful variable for searching in and reporting on sets of <a href="/wiki/Wikipedia:Page_name" title="Wikipedia:Page name">pages</a>. It is also used to apply features that configure the sets of pages in one namespace differently from another namespace. Using the namespace name MediaWiki can: </p> 
      <ul>
       <li><a href="/wiki/Wikipedia:Search" class="mw-redirect" title="Wikipedia:Search">Search</a> for pages in a particular namespace only.</li> 
       <li><a href="/wiki/Help:Special_page" title="Help:Special page">Report</a> on <a href="/wiki/Special:AllPages" title="Special:AllPages">content</a> or <a href="/wiki/Wikipedia:Recent_changes" class="mw-redirect" title="Wikipedia:Recent changes">changes in content</a> of a namespace.</li> 
       <li>Enable or disable the <a href="/wiki/Wikipedia:Subpage" class="mw-redirect" title="Wikipedia:Subpage">subpage</a> feature, per namespace.</li> 
       <li>Offer a <a href="/wiki/Wikipedia:Random_article" class="mw-redirect" title="Wikipedia:Random article">random article</a> for the main page (<kbd>[[Special:Random]]</kbd>), or for a namespace <kbd>[[Special:Random/<i>Namespace</i>]]</kbd>.</li> 
       <li>Change the functionality of <kbd>[[<i>wikilink</i>]]</kbd> if it targets the <b>Category</b> namespace or <b>File</b> namespace. Normally wikilinks are activated by the mouse, but <kbd>[[Category:<i>Pagename</i>]]</kbd> activates software functionality when the <a href="/wiki/Wikipedia:Category" class="mw-redirect" title="Wikipedia:Category">page is saved</a>, and <kbd>[[File:<i>Page name</i>]]</kbd> activates software functionality when the <a href="/wiki/Wikipedia:Image" class="mw-redirect" title="Wikipedia:Image">page is rendered</a>.</li> 
       <li>Change the functionality of <kbd>{{<a href="/wiki/Wikipedia:Transclusion" class="mw-redirect" title="Wikipedia:Transclusion">pagename</a>}}</kbd> so that it defaults to the set of pagenames in <b>Template</b>.</li> 
       <li>Tailor <i><a href="/wiki/Wikipedia:Edit_notices" class="mw-redirect" title="Wikipedia:Edit notices">edit notices</a></i> to a particular namespace's edit page, informing an editor of certain aspects about the content of that namespace.</li> 
       <li>Allow <i><a href="/wiki/Wikipedia:Templates" title="Wikipedia:Templates">templates</a></i> to behave differently on pages of different namespaces.</li>
      </ul> 
      <p>The namespace functionality is often visible in the URL, where you will see a namespace number. These are given in the table below. </p>
      <p>Subpages are configured for all namespaces except Main, File, MediaWiki and Category namespaces. Subpages are configured in the software by enabling or disabling the subpage feature per namespace. This further extends the function of a namespace by extending the types of pages or topics of pages a namespace can home. For example, talk page <i>/archives</i> are subpages, and they are both namespace searchable and page name searchable from the search box, unlike <a href="/wiki/Wikipedia:Page_history" class="mw-redirect" title="Wikipedia:Page history">history pages</a>, which are in no namespace. Another example is in the Template namespace where it is routine to put the documentation in a subpage called <i>/doc</i>. </p>
      <p>Random pages are enabled, in the default software configuration, only for the mainspace. </p>
      <p>Searching and reporting can operate over the entire database, or they can be limited to a namespace, and so they will offer a list of all the namespaces currently configured. <a href="/wiki/Special:WhatLinksHere" title="Special:WhatLinksHere">Special:WhatLinksHere</a> also offers a list of namespaces. </p>

      <p>Alternative variables are available for most namespaces, for example {{ns:talk}}, {{ns:user}}, and {{ns:user_talk}} for {{ns:1}}, {{ns:2}} and {{ns:3}}. The portal and book namespaces are numbered in the 100-plus range as are all MediaWiki nonstandard namespaces. </p>
      <p>All namespaces with a positive number or zero in this list are searchable. Specifically, the Media and Special namespaces are not searchable. </p>
      <p>The <a href="/wiki/Help:Magic_words" title="Help:Magic words">magic word</a> <code>{{NAMESPACE}}</code> returns the namespace prefix of the current page. This can be used in combination with other magic words and <a href="https://meta.wikimedia.org/wiki/Help:ParserFunctions" class="extiw" title="meta:Help:ParserFunctions">parser functions</a> to change the appearance of <a href="/wiki/Wikipedia:Template" class="mw-redirect" title="Wikipedia:Template">templates</a> depending on the namespace in which they are transcluded. </p>
      <p>Meta-templates have been created to simplify this process. These include the simple template {{<a href="/wiki/Template:Main_other" title="Template:Main other">main other</a>}}, and the more complex and versatile {{<a href="/wiki/Template:Namespace_detect" title="Template:Namespace detect">namespace detect</a>}}. </p>
      <p>For information on changing the appearance of a page or template by namespace based on CSS, see <a href="https://meta.wikimedia.org/wiki/Help:Namespace#CSS_based_namespace_detection" class="extiw" title="m:Help:Namespace">CSS-based namespace detection</a> on the Meta help page. </p>
