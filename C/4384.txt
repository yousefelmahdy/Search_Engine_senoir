      <h2><span id="Heavy_use_.2F_visible_files.28.21.29_protection"></span><span class="mw-headline" id="Heavy_use_/_visible_files(!)_protection">Heavy use / visible files(!) protection</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Commons_talk:Protection_policy&amp;action=edit&amp;section=2" title="Edit section: Heavy use / visible files(!) protection">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>[move=sysop] and [upload=sysop] should be enough, isn't it? edit=sysop isn't needed. It came to my mind, that someone malicious could place redir on a file page to get another content displayed at file inclusions. But as far as I tested (temporarily) at <a href="/w/index.php?title=File:AXMPtiffbeach.jpg&amp;action=edit&amp;redlink=1" class="new" title="File:AXMPtiffbeach.jpg (page does not exist)">File:AXMPtiffbeach.jpg</a> redirs do not fire if there are file version - neither for the file content (thumb URL links) and |thumb| inclusions nor even for the file page <a href="/w/index.php?title=File:AXMPtiffbeach.jpg&amp;action=edit&amp;redlink=1" class="new" title="File:AXMPtiffbeach.jpg (page does not exist)">File:AXMPtiffbeach.jpg</a>. Can someone confirm this - or even has a link to the relevant MediaWiki code / doc? </p>
      <p>If we are sure that edits cannot do harm the action=edit can be free. As such protections probably occur often at Commons I suggest to add one or two short sentences of advice/help how to protect (not for editing) such files. Cheers --<a href="/wiki/User:Saibo" title="User:Saibo">Saibo</a>&nbsp;(<a href="/wiki/User_talk:Saibo" title="User talk:Saibo"><small>?</small></a>) 01:04, 14 November 2011 (UTC) </p> 
      <dl>
       <dd>
        I agree. edit=sysop restriction is not necessary for highly visoble files. Edit restriction can be set individually when there is excessive vandalism or edit warring about file's description. – <a href="/wiki/User:Kwj2772" title="User:Kwj2772">Kwj2772</a> (<a href="/wiki/User_talk:Kwj2772" title="User talk:Kwj2772">msg</a>) 08:40, 15 November 2011 (UTC)

      <h2><span class="mw-headline" id="Strange_semi-protected_templates">Strange semi-protected templates</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Commons_talk:Protection_policy&amp;action=edit&amp;section=4" title="Edit section: Strange semi-protected templates">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>Hi,<br> Many templates (<a href="/wiki/Template:Wikispecies" title="Template:Wikispecies"><span style="font-family:monospace,monospace;">{{Wikispecies}}</span></a>, <a href="/wiki/Template:WikispeciesCompact" title="Template:WikispeciesCompact"><span style="font-family:monospace,monospace;">{{WikispeciesCompact}}</span></a>, <a href="/wiki/Template:Taxonavigation" title="Template:Taxonavigation"><span style="font-family:monospace,monospace;">{{Taxonavigation}}</span></a>...) that I maintain are protected but their protection is incoherent:<br> </p> 
      <ul>
       <li>When I click on "See source code", they display a banner&nbsp;: "This page is semi-protected and can only be modified by registered users" (approximate translation as my language is french).</li> 
       <li>I am a long-registered User, but I cannot edit theses templates.</li>
      </ul> 
      <p>So either, there is <b>a bug blocking my edition</b> or <b>the banner is buggy</b> and should say that this template is fully protected and only an admin can edit it.<br> PS: I still dream of a <b> 'authorized to change templates and modules' status</b> not related to 'admin' status. An admin is not meant to develop templates and modules + A templates and modules editor may no wish to become admin.<br> Best regards, <a href="/wiki/User:Lin%C3%A91" title="User:Liné1">Liné1</a> (<a href="/wiki/User_talk:Lin%C3%A91" title="User talk:Liné1"><span class="signature-talk">talk</span></a>) 12:48, 7 October 2013 (UTC) </p>
