      <p>Within the Toolbox section on the left-hand side of every page is a link labeled "<b>What links here</b>". This is used to see a list of the pages that <a href="/wiki/Help:Link" title="Help:Link">link</a> to (or <a href="/wiki/Wikipedia:Redirect" title="Wikipedia:Redirect">redirect</a> to, or <a href="/wiki/Wikipedia:Transclusion" class="mw-redirect" title="Wikipedia:Transclusion">transclude</a>) the current page. These are sometimes referred to as <b><a href="/wiki/Backlink" title="Backlink">backlinks</a></b>. </p>
      <p>It is possible to make a wikilink to the "What links here" list for a particular page; to do this type <code>[[Special:WhatLinksHere/<i>Page&nbsp;name</i>]]</code>, replacing <i>Page name</i> with the title of the target page. (The same text – without the brackets – can also be entered in the search box, to access "What links here" for any page title.) </p>
      <p>It is also possible to list the pages with <code>{{Special:Whatlinkshere/<i>Page&nbsp;name</i>|namespace=<i>number</i>}}</code>, where <i>Page name</i> is the name of the page, and <code>namespace</code> (optional) is <a href="/wiki/Help:Namespace" class="mw-redirect" title="Help:Namespace">the number of the namespace</a>. E.g. <code>{{Special:Whatlinkshere/Help:What links here|namespace=0}}</code> lists all pages from article space that link to this page. </p>
      <p>To use the tool, click <b><a href="/wiki/Special:WhatLinksHere" title="Special:WhatLinksHere">Special:WhatLinksHere</a></b> and type in the page title. </p>
      <p>"What links here" is limited to showing actual backlinks. Pages which contain the title being searched for, but where the title has not been linked, are excluded from the results. An <a href="/wiki/User:Lourdes/Backlinks" title="User:Lourdes/Backlinks">alternative script</a> exists to search out such pages, where possible unlinked backlinks may exist for the respective article. </p>

      <h2><span class="mw-headline" id="Overview">Overview</span></h2> 
      <p>The "What links here" facility lists the pages on the same site (English Wikipedia) which link to (or <a href="/wiki/Wikipedia:Redirect" title="Wikipedia:Redirect">redirect</a> to, or <a href="/wiki/Wikipedia:Transclusion" class="mw-redirect" title="Wikipedia:Transclusion">transclude</a>) a given page. It is possible to limit the search to pages in a specified <a href="/wiki/Wikipedia:Namespace" title="Wikipedia:Namespace">namespace</a>. To see this information, click the "What links here" link while looking at any page. The list is sorted by page ID, i.e., by date of creation of the page. </p>
      <p>This information comes from the <a href="https://www.mediawiki.org/wiki/pagelinks_table" class="extiw" title="mw:pagelinks table">pagelinks table</a> and the <a href="https://www.mediawiki.org/wiki/templatelinks_table" class="extiw" title="mw:templatelinks table">templatelinks table</a>. Pages redirected to the given page are marked "redirect". Pages transcluding the given page are marked "transclusion"; for these pages it is not shown whether they also <i>link</i> to the given page. For image and other file pages, the pages using the image or file appear on the list and are marked "image link". </p>
      <p>The parser function <a href="https://www.mediawiki.org/wiki/Help:Extension:ParserFunctions##ifexist" class="extiw" title="mw:Help:Extension:ParserFunctions">#ifexist:</a> causes a listing in "What links here" among the normal links even though no link is produced. </p>
      <p>The list of links to an article is useful in a number of ways: </p> 
      <ul>
       <li>The number of incoming links gives a rough indication of how important or popular a page is.</li> 
       <li>Where the intended subject material of an article is unclear, the list of articles linking to it might provide useful context.</li> 
       <li>It facilitates proper <a href="/wiki/Wikipedia:Disambiguation" title="Wikipedia:Disambiguation">disambiguation</a>.</li>
      </ul> 
      <p>The function works even for a page title that does not exist (recording <a href="/wiki/Wikipedia:Redlink" class="mw-redirect" title="Wikipedia:Redlink">redlinks</a> to that title). The "What links here" link appears on the edit page on which one arrives when following a broken link. Thus, for example, if film articles are linked to the actors, one can find the films an actor has played in even if there is no article about the actor. </p>
      <p>To invoke a "What links here" list directly (in the search box, browser address bar, or wikilinks) use the syntax <a href="/wiki/Special:WhatLinksHere/John_Smith" title="Special:WhatLinksHere/John Smith">Special:WhatLinksHere/John Smith</a> (replacing "John Smith" with the desired target article title). </p> 
      <h2><span class="mw-headline" id="Limitations">Limitations</span></h2> 
      <p>The following are <i>not</i> listed at "What links here": </p> 
      <ul>
       <li>automatically generated links from <a href="/wiki/Help:Category" title="Help:Category">categories</a> to their subcategories and member pages (and vice versa)</li> 
       <li>automatically generated links from <a href="/wiki/Help:Subpage" class="mw-redirect" title="Help:Subpage">subpages</a> to their parent pages</li> 
       <li>links in <a href="/wiki/Help:Edit_summary" title="Help:Edit summary">edit summaries</a></li> 
       <li>links in <a href="/wiki/Help:Special_page" title="Help:Special page">special pages</a>, for example <a href="/wiki/Special:LongPages" title="Special:LongPages">Special:LongPages</a></li> 
       <li>links made with <a href="/wiki/Help:Link#External_links" title="Help:Link">external link</a> syntax instead of wikilinks, for example <a class="external free" href="https://en.wikipedia.org/wiki/Example">https://en.wikipedia.org/wiki/Example</a> instead of the wikilink <a href="/wiki/Example" class="mw-disambig" title="Example">Example</a> made with <code>[[Example]]</code></li> 
       <li>links from outside Wikipedia</li>
      </ul> 
      <p>In the case of links to <a href="/wiki/Help:Section" title="Help:Section">sections</a> or other anchors, the precise target is not shown. "What links here" cannot list the backlinks of a specific section/anchor only. (It may be possible to work round this by making a new title that redirects to a particular section, and encouraging people to make links to the redirect rather than the section. Another possibility is to use <a href="/wiki/Help:Linksearch" title="Help:Linksearch">Linksearch</a>, but here users would have to add hidden external links.) </p>
      <p>As mentioned, backlinks which have not yet been linked are excluded from the results. An <a href="/wiki/User:Lourdes/Backlinks" title="User:Lourdes/Backlinks">alternative script</a> exists to search out such pages. </p>
      <p>Also note that if a page's links change because of a change in a <a href="/wiki/Help:Template" title="Help:Template">template</a>, the backlinks for that page are <a href="https://www.mediawiki.org/wiki/Manual:Job_queue#Updating_links_tables_when_a_template_changes" class="extiw" title="mw:Manual:Job queue">not updated immediately</a>, but via the job queue. </p> 
      <h2><span class="mw-headline" id="Redirects">Redirects</span></h2> 
      <p>The What links here report will list a <a href="/wiki/Help:Redirect" title="Help:Redirect">redirect</a> to a page, and indented under it any incoming links to the page that go through that redirect. </p>
      <p>For a double redirect, the indent is double, and any incoming wikilinks to the page that go through the two intervening redirects are then shown triple indented. Although they are shown, double redirects are cleaned up; any triple redirects will become doubles, so triple redirects are not shown, just doubles. </p>
      <p>For a demonstration, see the <a href="/wiki/Special:WhatLinksHere/Help:What_links_here" title="Special:WhatLinksHere/Help:What links here">What links here</a> report for this page. Double redirects and redirects to sections are set up at <a href="/wiki/Help:What_links_here/demo_rdcheck" title="Help:What links here/demo rdcheck">/demo rdcheck</a>. </p> 
      <h3><span class="mw-headline" id="Only_show_incoming_redirects">Only show incoming redirects</span></h3> 
      <p>To only show incoming links, click on "What links here" and then click both "Hide transclusions" and "Hide links". </p>
      <p>To hide incoming redirects, click on "Hide redirects". </p> 
      <h2><span class="mw-headline" id="Transclusions">Transclusions</span></h2> 
      <p>The backlinks list includes <a href="/wiki/Wikipedia:Transclusion" class="mw-redirect" title="Wikipedia:Transclusion">transclusions</a> of the current page. </p>
      <p>It also includes links which exist on certain pages because the page transcludes another page (template). For example, if page A transcludes template B, and B contains a link to C (not contained within &lt;noinclude&gt; tags), then the link to C will appear on page A, and A will be listed among the backlinks of C. </p>
      <p>It is possible that a page contains a link to C because it transcludes a template even though the template does not itself contain such a link (the link may be produced in the template wikitext by the application of various <a href="/wiki/Help:Parser_function" class="mw-redirect" title="Help:Parser function">parser functions</a>, rather than explicitly). In this case the template will not show up in the backlinks of C, although pages on which it is transcluded will (if they have the links to C). It may be convenient to make the template appear on the list by placing a link to C on the template, within &lt;noinclude&gt; tags. </p> 
      <h2><span class="mw-headline" id="Number_of_links">Number of links</span></h2> 
      <p>The "What links here" page does not display how many backlinks exist in total. The number of links displayed at one time is limited – initially to 50, although there are links to change this to certain other values. (Different values can be obtained by editing the URL resulting from clicking these links, but the maximum possible value is 5,000.) </p> 
      <h2><span class="mw-headline" id="Extensions_and_programming">Extensions and programming</span></h2> 
      <h3><span class="mw-headline" id="Number_of_backlinks">Number of backlinks</span></h3> 
      <p>The "What links here" page body has selector body.page-Special_Whatlinkshere, so for example we can use the <a href="/wiki/Help:Cascading_Style_Sheets" title="Help:Cascading Style Sheets">CSS</a> <code>body.page-Special_Whatlinkshere ul { list-style: decimal }</code> to number the backlinks. </p>
      <p>It is possible to obtain the list of backlinks (with additional information) programmatically using the MediaWiki API. For information on how to do this, see the <a href="https://www.mediawiki.org/wiki/API:Backlinks" class="extiw" title="mw:API:Backlinks">API documentation</a>. </p> 
      <h3><span class="mw-headline" id="Semantic_relations">Semantic relations</span></h3> 
      <p>With the extension <a href="/wiki/Semantic_MediaWiki" title="Semantic MediaWiki">Semantic MediaWiki</a> (SMW) links are categorized by specifying relations. For a given relation the backlinks of a page can be produced in-page. A series of queries, one for each relation (which seems cumbersome but can be put in a template like <a rel="nofollow" class="external autonumber" href="http://semanticweb.org/wiki/Template:Ask">[1]</a>), provides an in-page list of backlinks sorted by relation. Moreover, forward links and attributes of the resulting pages can also be provided, and also backlinks of backlinks. </p> 
      <h3><span class="mw-headline" id="DPL_method">DPL method</span></h3> 
      <p>A simpler extension that can embed backlinks (without the semantic functionality) is <a href="https://www.mediawiki.org/wiki/Dynamic_Page_List" class="extiw" title="mw:Dynamic Page List">Dynamic Page List</a>. <b>EXAMPLE NEEDED</b> </p> 
      <h3><span class="mw-headline" id="Suppress_links">Suppress links</span></h3> 
      <p>"(? links | edit)" can be suppressed by placing the following in Common.css </p> 
      <div class="mw-highlight mw-highlight-lang-css mw-content-ltr" dir="ltr">
       <pre><span></span><span class="p">.</span><span class="nc">mw-whatlinkshere-tools</span><span class="p">{</span>
    <span class="k">display</span><span class="p">:</span><span class="kc">none</span><span class="p">;</span>
<span class="p">}</span>
</pre>
      </div> 
      <p><b>However, "(transclusion)" can't be suppressed with that CSS. Maybe with CSS# positional-selection.</b> </p> 
      <h3><span class="mw-headline" id="Unclear_warning">Unclear warning</span></h3> 
      <p>It is <i>not</i> possible to simply transclude a "Special:WhatLinksHere/<i>Page_name</i>" page, as only a certain sub-group of special pages can be transcluded: <a href="/wiki/Wikipedia:Transclusion#Special_pages" class="mw-redirect" title="Wikipedia:Transclusion">Wikipedia:Transclusion#Special pages</a>. </p> 
      <h3><span class="mw-headline" id="Workaround_to_hide_transcluded_links">Workaround to hide transcluded links</span></h3> 
      <p>It is a known limitation that there is no way to filter out pages that link to the current page via a transcluded template. This limits the usefulness of "What links here" in cases where a page is linked to from a widely transcluded navbox. A workaround is to perform a <a href="/wiki/Help:Searching" title="Help:Searching">search</a> using the <a href="/wiki/Help:Searching#insource:" title="Help:Searching">insource</a> parameter, e.g. use <code>insource:/"[[Foo]]"/</code> to search for articles that contain direct links to <a href="/wiki/Foo" class="mw-redirect" title="Foo">Foo</a>, and <code>insource:/"[[Foo|"/</code> for articles that contain piped links. A user script, <a href="/wiki/User:PrimeHunter/Source_links.js" title="User:PrimeHunter/Source links.js">User:PrimeHunter/Source links.js</a>, simplifies this process by providing a link to automatically perform the search. </p>
