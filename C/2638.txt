       <h3 id="about-personalizing-codespaces"><a href="#about-personalizing-codespaces">About personalizing Codespaces</a></h3> 
       <p>When using any development environment, customizing the settings and tools to your preferences and workflows is an important step. Codespaces allows for two main ways of personalizing your codespaces. </p> 
       <ul> 
        <li><a href="#settings-sync">Settings Sync</a> - You can use and share Visual Studio Code settings between Codespaces and other instances of Visual Studio Code.</li> 
        <li><a href="#dotfiles">Dotfiles</a> – You can use a public <code>dotfiles</code> repository to specify scripts, shell preferences, and other configurations.</li> 
       </ul> 
       <p>Codespaces personalization applies to any codespace you create.</p> 
       <p>Project maintainers can also define a default configuration that applies to every codespace for a repository, created by anyone. For more information, see "<a href="/en/github/developing-online-with-codespaces/configuring-codespaces-for-your-project">Configuring Codespaces for your project</a>." </p>

       <h3 id="dotfiles"><a href="#dotfiles">Dotfiles</a></h3> 
       <p>Dotfiles are files and folders on Unix-like systems starting with <code>.</code> that control the configuration of applications and shells on your system. You can store and manage your dotfiles in a repository on GitHub. For advice and tutorials about what to include in your <code>dotfiles</code> repository, see <a href="https://dotfiles.github.io/">GitHub does dotfiles</a>.</p> 
       <p>If your user account on GitHub owns a public repository named <code>dotfiles</code>, GitHub automatically uses this repository to personalize your codespace environment. Private <code>dotfiles</code> repositories are not currently supported.</p> 
       <p>Your <code>dotfiles</code> repository might include your shell aliases and preferences, any tools you want to install, or any other codespace personalization you want to make.</p> 
       <p>When you create a new codespace, GitHub clones your <code>dotfiles</code> repository to the codespace environment, and looks for one of the following files to set up the environment.</p> 
       <ul> 
        <li><em>install.sh</em></li> 
        <li><em>install</em></li> 
        <li><em>bootstrap.sh</em></li> 
        <li><em>bootstrap</em></li> 
        <li><em>script/bootstrap</em></li> 
        <li><em>setup.sh</em></li> 
        <li><em>setup</em></li> 
        <li><em>script/setup</em></li> 
       </ul> 
       <p>If none of these files are found, then any files or folders in <code>dotfiles</code> starting with <code>.</code> are symlinked to the codespace's <code>~</code> or <code>$HOME</code> directory.</p> 
       <p>Any changes to your <code>dotfiles</code> repository will apply only to each new codespace, and do not affect any existing codespace.</p> 
       <div class="extended-markdown note border rounded-1 mb-4 p-3 color-border-info color-bg-info f5"> 
        <p><strong>Note:</strong> Currently, Codespaces does not support personalizing the <em>User</em> settings for the Visual Studio Code editor with your <code>dotfiles</code> repository. You can set default <em>Workspace</em> and <em>Remote [Codespaces]</em> settings for a specific project in the project's repository. For more information, see "<a href="/en/github/developing-online-with-codespaces/configuring-codespaces-for-your-project#creating-a-custom-codespace-configuration">Configuring Codespaces for your project</a>."</p> 
       </div>

For more information, see "<a href="/en/github/developing-online-with-codespaces/managing-encrypted-secrets-for-codespaces">Managing encrypted secrets for Codespaces</a>", "<a href="/en/github/developing-online-with-codespaces/managing-gpg-verification-for-codespaces">Managing GPG verification for Codespaces</a>", and "<a href="/en/github/developing-online-with-codespaces/managing-access-and-security-for-codespaces">Managing access and security for Codespaces</a>."</p>
