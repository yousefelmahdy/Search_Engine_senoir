       <h3 id="about-the-dependabotyml-file"><a href="#about-the-dependabotyml-file">About the <em>dependabot.yml</em> file</a></h3> 
       <p>The Dependabot configuration file, <em>dependabot.yml</em>, uses YAML syntax. If you're new to YAML and want to learn more, see "<a href="https://www.codeproject.com/Articles/1214409/Learn-YAML-in-five-minutes">Learn YAML in five minutes</a>."</p> 
       <p>You must store this file in the <code>.github</code> directory of your repository. When you add or update the <em>dependabot.yml</em> file, this triggers an immediate check for version updates. Any options that also affect security updates are used the next time a security alert triggers a pull request for a security update. For more information, see "<a href="/en/github/administering-a-repository/enabling-and-disabling-version-updates">Enabling and disabling version updates</a>" and "<a href="/en/github/managing-security-vulnerabilities/configuring-dependabot-security-updates">Configuring Dependabot security updates</a>."</p> 
       <p>The <em>dependabot.yml</em> file has two mandatory top-level keys: <code>version</code>, and <code>updates</code>. You can, optionally, include a top-level <code>registries</code> key. The file must start with <code>version: 2</code>.</p> 
       <h3 id="configuration-options-for-updates"><a href="#configuration-options-for-updates">Configuration options for updates</a></h3> 
       <p>The top-level <code>updates</code> key is mandatory. You use it to configure how Dependabot updates the versions or your project's dependencies. Each entry configures the update settings for a particular package manager. You can use the following options.</p>

       <p>These options fit broadly into the following categories.</p>

        <li>Essential set up options that you must include in all configurations: <a href="#package-ecosystem"><code>package-ecosystem</code></a>, <a href="#directory"><code>directory</code></a>,<a href="#scheduleinterval"><code>schedule.interval</code></a>.</li> 
        <li>Options to customize the update schedule: <a href="#scheduletime"><code>schedule.time</code></a>, <a href="#scheduletimezone"><code>schedule.timezone</code></a>, <a href="#scheduleday"><code>schedule.day</code></a>.</li> 
        <li>Options to control which dependencies are updated: <a href="#allow"><code>allow</code></a>, <a href="#ignore"><code>ignore</code></a>, <a href="#vendor"><code>vendor</code></a>.</li> 
        <li>Options to add metadata to pull requests: <a href="#reviewers"><code>reviewers</code></a>, <a href="#assignees"><code>assignees</code></a>, <a href="#labels"><code>labels</code></a>, <a href="#milestone"><code>milestone</code></a>.</li> 
        <li>Options to change the behavior of the pull requests: <a href="#target-branch"><code>target-branch</code></a>, <a href="#versioning-strategy"><code>versioning-strategy</code></a>, <a href="#commit-message"><code>commit-message</code></a>, <a href="#rebase-strategy"><code>rebase-strategy</code></a>, <a href="#pull-request-branch-nameseparator"><code>pull-request-branch-name.separator</code></a>.</li> 
       </ul> 
       <p>In addition, the <a href="#open-pull-requests-limit"><code>open-pull-requests-limit</code></a> option changes the maximum number of pull requests for version updates that Dependabot can open.</p> 
       <div class="extended-markdown note border rounded-1 mb-4 p-3 color-border-info color-bg-info f5"> 
        <p><strong>Note:</strong> Some of these configuration options may also affect pull requests raised for security updates of vulnerable package manifests.</p> 
        <p>Security updates are raised for vulnerable package manifests only on the default branch.        <h4 id="ignore"><a href="#ignore"><code>ignore</code></a></h4> 
       <p>By default all dependencies that are explicitly defined in a manifest or lock file are kept up to date. You can use <code>allow</code> and <code>ignore</code> to customize which dependencies to maintain with version updates. Dependabot checks for all allowed dependencies and then filters out any ignored dependencies or versions. So a dependency that is matched by both an <code>allow</code> and an <code>ignore</code> will be ignored.</p> 
       <p>Dependencies can be ignored either by adding them to <code>ignore</code> or by using the <code>@dependabot ignore</code> command on a pull request opened by Dependabot.</p> 
       <h5 id="creating-ignore-conditions-from-dependabot-ignore"><a href="#creating-ignore-conditions-from-dependabot-ignore">Creating <code>ignore</code> conditions from <code>@dependabot ignore</code></a></h5> 
       <p>Dependencies ignored by using the <code>@dependabot ignore</code> command are stored centrally for each package manager. If you start ignoring dependencies in the <code>dependabot.yml</code> file, these existing preferences are considered alongside the <code>ignore</code> dependencies in the configuration.</p> 
       <p>You can check whether a repository has stored <code>ignore</code> preferences by searching the repository for <code>"@dependabot ignore" in:comments</code>. If you wish to un-ignore a dependency ignored this way, re-open the pull request.</p> 
       <p>For more information about the <code>@dependabot ignore</code> commands, see "<a href="/en/github/administering-a-repository/managing-pull-requests-for-dependency-updates#managing-dependabot-pull-requests-with-comment-commands">Managing pull requests for dependency updates</a>."</p> 
       <h5 id="specifying-dependencies-and-versions-to-ignore"><a href="#specifying-dependencies-and-versions-to-ignore">Specifying dependencies and versions to ignore</a></h5> 
       <p>You can use the <code>ignore</code> option to customize which dependencies are updated. The <code>ignore</code> option supports the following options.</p> 
       <ul> 
        <li><code>dependency-name</code>use to ignore updates for dependencies with matching names, optionally using <code>*</code> to match zero or more characters. For Java dependencies, the format of the <code>dependency-name</code> attribute is: <code>groupId:artifactId</code> (for example: <code>org.kohsuke:github-api</code>).</li> 
        <li><code>versions</code>use to ignore specific versions or ranges of versions. If you want to define a range, use the standard pattern for the package manager (for example: <code>^1.0.0</code> for npm, or <code>~&gt; 2.0</code> for Bundler).</li> 
        <li><code>update-types</code>use to ignore types of updates, such as semver <code>major</code>, <code>minor</code>, or <code>patch</code> updates on version updates (for example: <code>version-update:semver-patch</code> will ignore patch updates). You can combine this with <code>dependency-name: *</code> to ignore particular <code>update-types</code> for all dependencies. Currently, <code>version-update:semver-major</code>, <code>version-update:semver-minor</code>, and <code>version-update:semver-patch</code> are the only supported options. Security updates are unaffected by this setting.</li> 
       </ul> 
       <p>If <code>versions</code> and <code>update-types</code> are used together, Dependabot will ignore any update in either set.</p> 
       <p>Setting this option will also affect pull requests for security updates to the manifest files of this package manager, unless you use <code>target-branch</code> to check for version updates on a non-default branch.</p> 
       <pre><code class="hljs language-yaml"><span class="hljs-comment"># Use `ignore` to specify dependencies that should not be updated </span>

<span class="hljs-attr">updates:</span>
  <span class="hljs-bullet">-</span> <span class="hljs-attr">package-ecosystem:</span> <span class="hljs-string">"npm"</span>
    <span class="hljs-attr">directory:</span> <span class="hljs-string">"/"</span>

      <span class="hljs-attr">interval:</span> <span class="hljs-string">"daily"</span>

        <span class="hljs-comment"># For Express, ignore all updates for version 4 and 5</span>

       <h4 id="insecure-external-code-execution"><a href="#insecure-external-code-execution"><code>insecure-external-code-execution</code></a></h4> 
       <p>Package managers with the <code>package-ecosystem</code> values <code>bundler</code>, <code>mix</code>, and <code>pip</code> may execute external code in the manifest as part of the version update process. This might allow a compromised package to steal credentials or gain access to configured registries. When you add a <a href="#registries"><code>registries</code></a> setting within an <code>updates</code> configuration, Dependabot automatically prevents external code execution, in which case the version update may fail. You can choose to override this behavior and allow external code execution for <code>bundler</code>, <code>mix</code>, and <code>pip</code> package managers by setting <code>insecure-external-code-execution</code> to <code>allow</code>.</p> 
       <p>You can explicitly deny external code execution, irrespective of whether there is a <code>registries</code> setting for this update configuration, by setting <code>insecure-external-code-execution</code> to <code>deny</code>.</p>

       <h4 id="versioning-strategy"><a href="#versioning-strategy"><code>versioning-strategy</code></a></h4> 
       <p>When Dependabot edits a manifest file to update a version, it uses the following overall strategies:</p> 
       <ul> 
        <li>For apps, the version requirements are increased, for example: npm, pip and Composer.</li> 
        <li>For libraries, the range of versions is widened, for example: Bundler and Cargo.</li> 
       </ul> 
       <p>Use the <code>versioning-strategy</code> option to change this behavior for supported package managers.</p> 
       <p>Setting this option will also affect pull requests for security updates to the manifest files of this package manager, unless you use <code>target-branch</code> to check for version updates on a non-default branch.</p> 
       <p>Available update strategies</p> 
       <table>
        <thead>
         <tr>
          <th>Option</th>
          <th>Supported by</th>
          <th>Action</th>
         </tr>
        </thead>
        <tbody>
         <tr>
          <td><code>lockfile-only</code></td>
          <td><code>bundler</code>, <code>cargo</code>, <code>composer</code>, <code>mix</code>, <code>npm</code>, <code>pip</code></td>
          <td>Only create pull requests to update lockfiles. Ignore any new versions that would require package manifest changes.</td>
         </tr>
         <tr>
          <td><code>auto</code></td>
          <td><code>bundler</code>, <code>cargo</code>, <code>composer</code>, <code>mix</code>, <code>npm</code>, <code>pip</code></td>
          <td>Follow the default strategy described above.</td>
         </tr>
         <tr>
          <td><code>widen</code></td>
          <td><code>composer</code>, <code>npm</code></td>
          <td>Relax the version requirement to include both the new and old version, when possible.</td>
         </tr>
         <tr>
          <td><code>increase</code></td>
          <td><code>bundler</code>, <code>composer</code>, <code>npm</code></td>
          <td>Always increase the version requirement to match the new version.</td>
         </tr>
         <tr>
          <td><code>increase-if-necessary</code></td>
          <td><code>bundler</code>, <code>composer</code>, <code>npm</code></td>
          <td>Increase the version requirement only when required by the new version.</td>
         </tr>
        </tbody>
       </table>

<span class="hljs-attr">version:</span> <span class="hljs-number">2</span>
<span class="hljs-attr">updates:</span>
  <span class="hljs-bullet">-</span> <span class="hljs-attr">package-ecosystem:</span> <span class="hljs-string">"npm"</span>

       <h3 id="configuration-options-for-private-registries"><a href="#configuration-options-for-private-registries">Configuration options for private registries</a></h3> 
       <p>The top-level <code>registries</code> key is optional. It allows you to specify authentication details that Dependabot can use to access private package registries. </p> 
       <div class="extended-markdown note border rounded-1 mb-4 p-3 color-border-info color-bg-info f5"> 
        <p><strong>Note:</strong> Private registries behind firewalls on private networks are not supported.</p> 
       </div> 
       <p>The value of the <code>registries</code> key is an associative array, each element of which consists of a key that identifies a particular registry and a value which is an associative array that specifies the settings required to access that registry. The following <em>dependabot.yml</em> file, configures a registry identified as <code>dockerhub</code> in the <code>registries</code> section of the file and then references this in the <code>updates</code> section of the file.</p> 
       <pre><code class="hljs language-yaml"><span class="hljs-comment"># Minimal settings to update dependencies in one private registry</span>

<span class="hljs-attr">version:</span> <span class="hljs-number">2</span>
<span class="hljs-attr">registries:</span>
  <span class="hljs-attr">dockerhub:</span> <span class="hljs-comment"># Define access for a private registry </span>
    <span class="hljs-attr">type:</span> <span class="hljs-string">docker-registry</span>
    <span class="hljs-attr">url:</span> <span class="hljs-string">registry.hub.docker.com</span>
    <span class="hljs-attr">username:</span> <span class="hljs-string">octocat</span>
    <span class="hljs-attr">password:</span> <span class="hljs-string">${{secrets.DOCKERHUB_PASSWORD}}</span>
<span class="hljs-attr">updates:</span>
  <span class="hljs-bullet">-</span> <span class="hljs-attr">package-ecosystem:</span> <span class="hljs-string">"docker"</span>
    <span class="hljs-attr">directory:</span> <span class="hljs-string">"/docker-registry/dockerhub"</span>
    <span class="hljs-attr">registries:</span>
      <span class="hljs-bullet">-</span> <span class="hljs-string">dockerhub</span> <span class="hljs-comment"># Allow version updates for dependencies in this registry</span>
    <span class="hljs-attr">schedule:</span>
      <span class="hljs-attr">interval:</span> <span class="hljs-string">"monthly"</span>
</code></pre> 
       <p>You use the following options to specify access settings. Registry settings must contain a <code>type</code> and a <code>url</code>, and typically either a <code>username</code> and <code>password</code> combination or a <code>token</code>.</p> 
       <table>
        <thead>
         <tr>
          <th align="left">Option&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;</th>
          <th align="left">Description</th>
         </tr>
        </thead>
        <tbody>
         <tr>
          <td align="left"><code>type</code></td>
          <td align="left">Identifies the type of registry. See the full list of types below.</td>
         </tr>
         <tr>
          <td align="left"><code>url</code></td>
          <td align="left">The URL to use to access the dependencies in this registry. The protocol is optional. If not specified, <code>https://</code> is assumed. Dependabot adds or ignores trailing slashes as required.</td>
         </tr>
         <tr>
          <td align="left"><code>username</code></td>
          <td align="left">The username that Dependabot uses to access the registry.</td>
         </tr>
         <tr>
          <td align="left"><code>password</code></td>
          <td align="left">A reference to a Dependabot secret containing the password for the specified user. For more information, see "<a href="/en/github/administering-a-repository/managing-encrypted-secrets-for-dependabot">Managing encrypted secrets for Dependabot</a>."</td>
         </tr>
         <tr>
          <td align="left"><code>key</code></td>
          <td align="left">A reference to a Dependabot secret containing an access key for this registry. For more information, see "<a href="/en/github/administering-a-repository/managing-encrypted-secrets-for-dependabot">Managing encrypted secrets for Dependabot</a>."</td>
         </tr>
         <tr>
          <td align="left"><code>token</code></td>
          <td align="left">A reference to a Dependabot secret containing an access token for this registry. For more information, see "<a href="/en/github/administering-a-repository/managing-encrypted-secrets-for-dependabot">Managing encrypted secrets for Dependabot</a>."</td>
         </tr>
         <tr>
          <td align="left"><code>replaces-base</code></td>
          <td align="left">For registries with <code>type: python-index</code>, if the boolean value is <code>true</code>, pip resolves dependencies by using the specified URL rather than the base URL of the Python Package Index (by default <code>https://pypi.org/simple</code>).</td>
         </tr>
        </tbody>
       </table>
