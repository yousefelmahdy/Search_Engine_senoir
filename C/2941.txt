        <h1 class="border-bottom-0">Creating a project board</h1> 
        <div class="d-none d-lg-block ml-2"> <button class="btn-link Link--secondary js-print tooltipped tooltipped-n" aria-label="Print this article"> <!-- From https://heroicons.dev/ --> 
          <svg fill="none" height="18" width="18" stroke-linecap="round" stroke-linejoin="round" stroke-width="2" viewbox="0 0 24 24" stroke="currentColor">
           <path d="M17 17h2a2 2 0 002-2v-4a2 2 0 00-2-2H5a2 2 0 00-2 2v4a2 2 0 002 2h2m2 4h6a2 2 0 002-2v-4a2 2 0 00-2-2H9a2 2 0 00-2 2v4a2 2 0 002 2zm8-12V5a2 2 0 00-2-2H9a2 2 0 00-2 2v4h10z"></path>
          </svg> </button>

       </div> 
       <div class="lead-mktg">
        <p>Project boards can be used to create customized workflows to suit your needs, like tracking and prioritizing specific feature work, comprehensive roadmaps, or even release checklists.</p>
       </div> 
      </div> 
      <div class="article-grid-toc border-bottom border-xl-0 pb-4 mb-5 pb-xl-0 mb-xl-0"> 
       <div class="article-grid-toc-content"> 
        <h2 id="in-this-article" class="f5 mb-2"><a class="Link--primary" href="#in-this-article">In this article</a></h2> 
        <ul class="list-style-none pl-0 f5 mb-0"> 
         <li class="ml-0  mb-2 lh-condensed"><a href="#creating-a-user-owned-project-board">Creating a user-owned project board</a></li> 
         <li class="ml-0  mb-2 lh-condensed"><a href="#creating-an-organization-wide-project-board">Creating an organization-wide project board</a></li> 
         <li class="ml-0  mb-2 lh-condensed"><a href="#creating-a-repository-project-board">Creating a repository project board</a></li> 
         <li class="ml-0  mb-2 lh-condensed"><a href="#further-reading">Further reading</a></li> 
        </ul> 
       </div> 
      </div> 
      <div id="article-contents" class="article-grid-body"> 
       <p>You can use a project board template to create a project board with automation already configured. For more information, see "<a href="/en/articles/about-project-boards#templates-for-project-boards">About project boards</a>."</p> 
       <p>You can also copy a project board to reuse its customizations for similar projects. For more information, see "<a href="/en/articles/copying-a-project-board">Copying a project board</a>."</p> 
       <p>You can link up to twenty-five repositories to your organization or user-owned project board. Linking repositories makes it easier to add issues and pull requests from those repositories to your project board using <strong>
         <svg version="1.1" width="16" height="16" viewbox="0 0 16 16" class="octicon octicon-plus" aria-label="The Plus icon" role="img">
          <path fill-rule="evenodd" d="M7.75 2a.75.75 0 01.75.75V7h4.25a.75.75 0 110 1.5H8.5v4.25a.75.75 0 11-1.5 0V8.5H2.75a.75.75 0 010-1.5H7V2.75A.75.75 0 017.75 2z"></path>
         </svg> Add cards</strong> or from the issue or pull requests sidebar. For more information, see "<a href="/en/articles/linking-a-repository-to-a-project-board">Linking a repository to a project board</a>."</p> 
       <p>Once you've created your project board, you can add issues, pull requests, and notes to it. For more information, see "<a href="/en/articles/adding-issues-and-pull-requests-to-a-project-board">Adding issues and pull requests to a project board</a>" and "<a href="/en/articles/adding-notes-to-a-project-board">Adding notes to a project board</a>."</p> 
       <p>You can also configure workflow automations to keep your project board in sync with the status of issues and pull requests. For more information, see "<a href="/en/articles/about-automation-for-project-boards">About automation for project boards</a>."</p> 
       <p>You can use GitHub's API to import a project board. For more information, see "<a href="/en/graphql/reference/mutations#importproject">importProject</a>."</p> 
       <h3 id="creating-a-user-owned-project-board"><a href="#creating-a-user-owned-project-board">Creating a user-owned project board</a></h3> 
       <ol> 
        <li>In the top right corner of GitHub, click your profile photo, then click <strong>Your profile</strong>. <span class="procedural-image-wrapper"><img src="/assets/images/help/profile/top_right_avatar.png" alt="Profile photo"></span></li> 
        <li>On the top of your profile page, in the main navigation, click 
         <svg version="1.1" width="16" height="16" viewbox="0 0 16 16" class="octicon octicon-project" aria-label="The project board icon" role="img">

        <li>Click <strong>New Project</strong>. <span class="procedural-image-wrapper"><img src="/assets/images/help/projects/new-project-button.png" alt="New Project button"></span></li> 
        <li>Type a name and description for your project board. <span class="procedural-image-wrapper"><img src="/assets/images/help/projects/name-description-create-button.png" alt="Fields for project name and description and Create project button"></span></li> 
        <li>Optionally, to add a template to your project board, use the <strong>Template:</strong> drop-down menu and click a template. <span class="procedural-image-wrapper"><img src="/assets/images/help/projects/project_board_template_drop_down_menu.png" alt="Drop-down menu showing project board template options"></span></li>

        <li>Add columns to best suit your workflow.</li>
