      <p>A <a href="/wiki/Wikipedia:Redirect" title="Wikipedia:Redirect">redirect</a> is a special type of page that automatically causes another page to be displayed in its place. A redirect that points to another redirect is called a <b>double redirect</b>. These pages are unwanted, because Wikipedia's MediaWiki software is currently configured to not follow the second redirect. The MediaWiki feature which will allow it <a href="https://phabricator.wikimedia.org/T32359" class="extiw" title="phab:T32359">is not mature yet</a>. If someone is redirected to a redirect, the chain stops after the first redirect, like in <a href="/wiki/Wikipedia:Example_of_a_double_redirect" class="mw-redirect" title="Wikipedia:Example of a double redirect">the example</a>. These situations create unpleasant experiences for the reader and make the navigational structure of the site confusing. </p>
      <p>Double redirects are usually created after a <a href="/wiki/Wikipedia:Moving_a_page" title="Wikipedia:Moving a page">page move</a>, when old redirects are left unchanged and pointing towards an old name. Although bots fix most of these within a few days from creation automatically, editors should take care not to manually create double redirects. </p>

      <ul>
       <li>Suppose page title A (<i>Morchella esculenta</i> in the example to the right) redirects to B (<i>Morel</i>), which in turn redirects to C (<i>Morchella</i>).</li> 
       <li>Upon following the link to A, you will see, as illustrated, a page containing: 
        <ul>
         <li>the page title B;</li> 
         <li>a large link to C;</li> 
         <li>a small notice saying "Redirected from A"</li>
        </ul></li> 
       <li>Click the "A" in "redirected from A".</li> 
       <li>You will see a page containing: 
        <ul>
         <li>the page title A;</li> 
         <li>a large link to B.</li>
        </ul></li> 
       <li>Click "Edit source" (or "Edit", if no "Edit source" tab is available) and change B to C.</li>
      </ul> 
      <h2><span class="mw-headline" id="Double_redirects_and_bots">Double redirects and bots</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Wikipedia:Double_redirects&amp;action=edit&amp;section=2" title="Edit section: Double redirects and bots">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>Double redirects are easily and automatically fixed by bots, and most are fixed within a few days of creation. Because of this, human editors would be best off putting their time on other tasks that can't be automated. However, bots will not fix a double redirect if the redirect page is <a href="/wiki/Wikipedia:Protection_policy#Full_protection" title="Wikipedia:Protection policy">fully protected</a>. The following bots fix double redirects: </p>

      <h2><span class="mw-headline" id="Checking_for_double_redirects">Checking for double redirects</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Wikipedia:Double_redirects&amp;action=edit&amp;section=4" title="Edit section: Checking for double redirects">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p>Suppose you want to check whether there are any double redirects to page C. Go to page C and click "What links here" (usually the first link in the "Toolbox" on the lefthand side of the page). Double (or multiple) redirects are those pages which appear in the list with both of these properties: </p> 
      <ul>
       <li>Indented at least one level in comparison to the page at the top of the list, <em><strong>AND</strong></em></li> 
       <li>Labelled "<i>(redirect page)</i>".</li>
      </ul> 
      <p>Note that if you've just moved page C, then there might be only one page which is not indented, (the page at the top of the list), and everything else might be indented at least one level. </p>
      <p>Once all double redirects have been fixed, the "What links here" page will have only three types of pages listed: </p> 
      <ol>
       <li>Direct links from ordinary pages (not indented, <em><strong>not</strong></em> labelled "<i>(redirect page)</i>").</li> 
       <li>Direct links from redirect pages (not indented; labelled "<i>(redirect page)</i>").</li> 
       <li>Indirect links from ordinary pages (indented one level, <em><strong>not</strong></em> labelled "<i>(redirect page)</i>").</li>
      </ol> 
      <p>The first and third categories do not need to be fixed – see <a href="/wiki/Wikipedia:Redirect#Do_not_&quot;fix&quot;_links_to_redirects_that_are_not_broken" title="Wikipedia:Redirect">Wikipedia:Redirect#Do not "fix" links to redirects that are not broken</a>. </p> 
      <h2><span id="When_MediaWiki_won.27t_automatically_follow_a_redirect"></span><span class="mw-headline" id="When_MediaWiki_won't_automatically_follow_a_redirect">When MediaWiki won't automatically follow a redirect</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Wikipedia:Double_redirects&amp;action=edit&amp;section=5" title="Edit section: When MediaWiki won't automatically follow a redirect">edit</a><span class="mw-editsection-bracket">]</span></span></h2> 
      <p><span class="anchor" id="When_your_computer_won't_follow_a_redirect"></span> MediaWiki will not follow a redirect if: </p> 
      <ul>
       <li>there are double redirects</li> 
       <li>it is a redirect to a Special page</li> 
       <li>it is a redirect to a non-existent page</li> 
       <li>it is a redirect to an interwiki page</li>
      </ul>
