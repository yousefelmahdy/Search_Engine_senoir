       <h3 id="about-queries"><a href="#about-queries">About queries</a></h3> 
       <p>Every GraphQL schema has a root type for both queries and mutations. The <a href="https://graphql.github.io/graphql-spec/June2018/#sec-Type-System">query type</a> defines GraphQL operations that retrieve data from the server.</p> 
       <p>For more information, see "<a href="/en/graphql/guides/forming-calls-with-graphql#about-queries">About queries</a>."</p> 
       <div class="extended-markdown note border rounded-1 mb-4 p-3 color-border-info color-bg-info f5"> 
        <p><strong>Note:</strong> For <a href="/en/developers/apps/identifying-and-authorizing-users-for-github-apps#user-to-server-requests">user-to-server</a> GitHub App requests, you should use separate queries for issues and pull requests. For example, use the <code>is:issue</code> or <code>is:pull-request</code> filters and their equivalents. Using the <code>search</code> connection to return a combination of issues and pull requests in a single query will result in an empty set of nodes.</p> 
       </div>
