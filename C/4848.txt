      <p><b>Talk pages</b> (also known as <b>discussion pages</b>) are <a href="/wiki/Wikipedia:Administration#Data_structure_and_development" title="Wikipedia:Administration">administration pages</a> where editors can discuss improvements to articles or other Wikipedia pages. The talk page associated with an article is named "Talk:Example", where "Example" is the name of the article. For example, the talk page for discussion of improvements to the article <a href="/wiki/Australia" title="Australia">Australia</a> is named <a href="/wiki/Talk:Australia" title="Talk:Australia">Talk:Australia</a>. The talk page associated with a page in another namespace is named by adding "talk" after the namespace label; for example, the talk page for <a href="/wiki/Wikipedia:About" title="Wikipedia:About">Wikipedia:About</a> is called <a href="/wiki/Wikipedia_talk:About" title="Wikipedia talk:About">Wikipedia talk:About</a>. </p>
      <p>When viewing an article (or any other non-talk page) on the desktop Wikipedia, a link to the corresponding talk page appears on the "Talk" tab at the top of the page. Click this tab to switch to the talk page. On the mobile Wikipedia, type "Talk:" and the article's name in the search bar. (There are <a href="https://www.mediawiki.org/wiki/Reading/Web/Advanced_mobile_contributions" class="extiw" title="mw:Reading/Web/Advanced mobile contributions">plans</a> to provide easier talk access on mobile.) </p>
      <p><a href="/wiki/Wikipedia:User_pages" title="Wikipedia:User pages">User pages</a> also have associated talk pages (for example, <a href="/wiki/User_talk:Jimbo_Wales" title="User talk:Jimbo Wales">User talk:Jimbo Wales</a> for <a href="/wiki/User:Jimbo_Wales" title="User:Jimbo Wales">Jimbo Wales' userpage</a>.). When other editors need to contact you, they will usually do this by leaving a message on <a href="/wiki/Special:MyTalk" title="Special:MyTalk">your talk page</a>. When someone has left you a message that way, you will either see an <a href="#You_have_new_messages">orange information notice</a> the next time you log in or view a page on Wikipedia if you are editing as an IP address, or a <a href="#You_have_new_messages">red badge next to your username</a> if you are logged in. </p>

      <h2><span class="mw-headline" id="Where_to_find_talk_pages">Where to find talk pages</span></h2> 
      <div class="thumb tleft">
       <div class="thumbinner" style="width:352px;">
        <a href="/wiki/File:Where_to_find_talk_pages.png" class="image"><img alt="" src="//upload.wikimedia.org/wikipedia/commons/thumb/c/c3/Where_to_find_talk_pages.png/350px-Where_to_find_talk_pages.png" decoding="async" width="350" height="133" class="thumbimage" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/c/c3/Where_to_find_talk_pages.png/525px-Where_to_find_talk_pages.png 1.5x, //upload.wikimedia.org/wikipedia/commons/c/c3/Where_to_find_talk_pages.png 2x" data-file-width="576" data-file-height="219"></a> 
        <div class="thumbcaption">
         <div class="magnify">
          <a href="/wiki/File:Where_to_find_talk_pages.png" class="internal" title="Enlarge"></a>
         </div>Location of the talk page tab (here, on the page <a href="/wiki/Encyclopedia" title="Encyclopedia">Encyclopedia</a>)
        </div>
       </div>
      </div> 
      <p>When viewing an article (or any other non-talk page), a link to the corresponding talk page appears on the "Talk" tab at the top of the page. (The <a href="/wiki/Help:Mobile_access" title="Help:Mobile access">mobile version</a> has a button at the bottom for logged-in users, while <a href="/wiki/Mobile_app" title="Mobile app">apps</a> may have no link.) Click this tab to switch to the talk page; you can then view the talk page and its <a href="/wiki/Wikipedia:Page_history" class="mw-redirect" title="Wikipedia:Page history">history</a>, and <a href="/wiki/Wikipedia:How_to_edit_a_page" class="mw-redirect" title="Wikipedia:How to edit a page">edit</a> it if you want to add a question or comment. </p>
      <p>If the "Talk" link is <a href="/wiki/Wikipedia:Red_link" title="Wikipedia:Red link">red</a>, it means no talk page has been started yet. Click the red link to begin a talk page for that article and follow the instructions in <b>Starting a new thread</b> below. (It is also possible for a talk page to exist while the corresponding non-talk page is a red link; this often occurs in <a href="/wiki/Wikipedia:User_page" class="mw-redirect" title="Wikipedia:User page">User space</a>, when a user has received talk page messages but has not started a user page yet.) </p>
      <p>To go back to the article page from its talk page, use the leftmost tab at the top of the page, labeled "article". For pages other than articles, this tab may say something different, like "user page" or "project page". </p> 
      <h2><span class="mw-headline" id="Identifying_yourself">Identifying yourself</span></h2> 
      <p>When you post a message on a talk page you should always <b>sign and date your comment</b> so other editors can follow the thread of the conversation. To do this easily, type four tildes (~~~~) at the end of your comment, or just click the signature button <a href="/wiki/File:Signature_icon_april_2018.png" class="image"><img alt="Signature icon april 2018.png" src="//upload.wikimedia.org/wikipedia/commons/1/1a/Signature_icon_april_2018.png" decoding="async" width="32" height="34" data-file-width="32" data-file-height="34"></a> on the row of buttons above the edit box. Once you publish the edit, this will be <a href="/wiki/Help:Automatic_conversion_of_wikitext" class="mw-redirect" title="Help:Automatic conversion of wikitext">automatically converted</a> into a user signature with a link to your user page, your user talk page, and the date and time that you save your edit. (You can change the form of your signature using your <a href="/wiki/Help:Preferences" title="Help:Preferences">user preferences</a>.) </p> 
      <table class="wikitable" style="margin-left:auto; margin-right:auto; border:none;"> 
       <tbody>
        <tr> 
         <th>Code</th> 
         <th>Result </th>
        </tr> 
        <tr> 
         <td><code>~~~~</code> </td> 
         <td><a href="/wiki/User:Example" title="User:Example">Username</a> (<a href="/wiki/User_talk:Example" title="User talk:Example">talk</a>) 08:35, 4 June 2021 (UTC) </td>
        </tr>
       </tbody>
      </table> 
      <p>If you choose to contribute without logging in, regardless of whether you have an account, you should still sign your posts. In this case your IP address will take the place of your username, and will link to your contributions history. </p>

      <p>To discuss a topic that's not already covered on the article or user talk page, start a new topic. </p> 
      <ul>
       <li>Click on the <b>"New section"</b> link at the top of the talk page screen.</li> 
       <li>Be sure to enter a section header in the "Subject/headline" box with a suitable title, preferably not something generic like "Question" or "Problem".</li> 
       <li>Sign. At the end of your post, type four tildes (~~~~), which will automatically add your user name and the date.</li> 
       <li>Click "Publish changes"</li>
      </ul> 
      <p>A new section can also be started by editing the whole page or an existing section, going to a new line and typing <code>== Heading ==</code>, replacing "Heading" with a suitable title, but make sure to add new discussions at the <i>bottom</i> of the page. A new section automatically adds the heading to the "Contents" box on pages with at least four sections. </p>

      <ul>
       <li>Click the <b>"Edit"</b> link on the right end of the bar of the section you want to reply to.</li> 
       <li>Add your comment below the last entry in the discussion. If you want to respond to a specific comment, you can place your response directly below it. Use a colon (:) to indent your message to create a threaded message. See <b>Indentation</b> below for more information on indenting talk pages with colons.</li> 
       <li>Sign. Type four tildes (~~~~), which will automatically add your user name and the date.</li>

      <h3><span class="mw-headline" id="Indentation">Indentation</span></h3> 
      <div role="note" class="hatnote navigation-not-searchable">
       See also: <a href="/wiki/Wikipedia:Indentation" title="Wikipedia:Indentation">Wikipedia:Indentation</a> and <a href="/wiki/Help:List" title="Help:List">Help:List</a>
      </div> 
      <p>Indentation is used to keep talk pages readable. Comments are indented using one or more initial colons (<code>:</code>), each colon representing one level of indentation. Each comment should be indented one more level than the comment it replies to, which may or may not be the preceding comment. For example: </p> 
      <table class="wikitable" style="margin-left:auto; margin-right:auto; border:none; width:90%;"> 
       <tbody>
        <tr> 
         <th scope="col">Code </th> 
         <th scope="col">Result </th>
        </tr> 
        <tr> 
         <td> <pre>== Header ==

The first comment in a section has no colons before it. ~~~~

:The reply to the first comment is indented one level. ~~~~
::The reply to the second comment should be indented one more level. ~~~~
::Another reply to the second comment is also indented one more level than the comment being replied to. ~~~~
:A subsequent reply to the first comment is indented one level. ~~~~
</pre> </td> 
         <td> 
          <div style="color: #000000; background: none; overflow: hidden; page-break-after: avoid; font-size: 1.5em; font-family: Georgia,Times,serif; margin-top: 1em; margin-bottom: 0.25em; line-height: 1.3; padding: 0; border-bottom: 1px solid #AAAAAA;">
           Header
          </div> <p>The first comment in a section has no colons before it. <a href="/wiki/User:Example" title="User:Example">Editor 1</a> (<a href="/wiki/User_talk:Example" title="User talk:Example">talk</a>) 10:44, 21 September 2016 (UTC) </p> 
          <dl>
           <dd>
            The reply to the first comment is indented one level. <a href="/wiki/User:Example" title="User:Example">Editor 2</a> (<a href="/wiki/User_talk:Example" title="User talk:Example">talk</a>) 16:40, 21 September 2016 (UTC) 
            <dl>
             <dd>
              The reply to the second comment should be indented one more level. <a href="/wiki/User:Example" title="User:Example">Editor 1</a> (<a href="/wiki/User_talk:Example" title="User talk:Example">talk</a>) 16:57, 23 September 2016 (UTC)
             </dd> 
             <dd>
              Another reply to the second comment is also indented one more level than the comment being replied to. <a href="/wiki/User:Example" title="User:Example">Editor 4</a> (<a href="/wiki/User_talk:Example" title="User talk:Example">talk</a>) 08:27, 25 September 2016 (UTC)
             </dd>
            </dl>
           </dd> 
           <dd>
            A subsequent reply to the first comment is indented one level. <a href="/wiki/User:Example" title="User:Example">Editor 3</a> (<a href="/wiki/User_talk:Example" title="User talk:Example">talk</a>) 03:15, 24 September 2016 (UTC)
           </dd>
          </dl> </td>
        </tr>
       </tbody>
      </table> 
      <p><span class="anchor" id="mixed_indents"></span><span class="anchor" id="mixthread"></span>Some pages (deletion discussions, for example) use asterisks (<code>*</code>) rather than colons for indentation. Generally colons and asterisks should not be mixed; if you see asterisks are being used in a page, use them as well. Complex discussions may mix them (and numbered lists, too); in such a case avoid <a href="/wiki/Wikipedia:Manual_of_Style/Accessibility#Lists" title="Wikipedia:Manual of Style/Accessibility">mangled list formatting</a> with this simple rule of thumb: </p>
      <p><strong>Use the same indentation and list formatting as what you are replying to, plus one level at the end of the indent/list code.</strong> E.g., if you are replying to something in a complicated discussion that starts with <code>#:::*</code>, just copy-paste that and add a <code>:</code>, resulting in <code>#:::*:</code> in front of your reply (or use <code>#:::**</code> if you feel it is necessary for your reply to begin with a bullet point). </p>
      <p>Avoid placing double line breaks between indented lines of text, since this can create problems for users of screen reader software (see <a href="/wiki/Wikipedia:Manual_of_Style/Accessibility#Indentation" title="Wikipedia:Manual of Style/Accessibility">Wikipedia:Manual of Style/Accessibility §&nbsp;Indentation</a>). </p>
      <p>If you practice these techniques, be sure to practice on a talk page, such as <code>User talk:<var style="padding-right: 1px;">your user name</var>/sandbox</code>. The <a href="/wiki/Cascading_Style_Sheets" class="mw-redirect" title="Cascading Style Sheets">Cascading Style Sheets</a> for talk pages are different from articles, and the visual appearance of list-formatted text can be different. Also, <code>:</code> should not be used for visual indentation in articles, as it is actually markup <a href="/wiki/Wikipedia:Manual_of_Style/Lists#Description_(definition,_association)_lists" title="Wikipedia:Manual of Style/Lists">specifically for description lists</a>. <style data-mw-deduplicate="TemplateStyles:r1015928993">.mw-parser-output div.crossreference{padding-left:0}</style><span role="note" class="hatnote navigation-not-searchable crossreference">(See <a href="/wiki/Wikipedia:Manual_of_Style#Indentation" title="Wikipedia:Manual of Style">Wikipedia:Manual of Style §&nbsp;Indentation</a> for how to indent single lines and blocks of text in articles; for block-quoting, see <a href="/wiki/Wikipedia:Manual_of_Style#Block_quotations" title="Wikipedia:Manual of Style">Wikipedia:Manual of Style §&nbsp;Block quotations</a>.)</span> </p>

      <h3><span class="mw-headline" id="Subpages_and_archiving">Subpages and archiving</span></h3> 
      <div role="note" class="hatnote navigation-not-searchable">
       Main page: <a href="/wiki/Help:Archiving_a_talk_page" title="Help:Archiving a talk page">Help:Archiving a talk page</a>
      </div> 
      <p>Sometimes particular topics generate a disproportionate amount of traffic on a talk page. It may be decided to remove discussion of those topics to a <a href="/wiki/Help:Subpage" class="mw-redirect" title="Help:Subpage">subpage</a> of the talk page. To do this, create a page titled "Talk:Xxx/Yyy", where "Talk:Xxx" is the name of the main talk page, and "Yyy" indicates the topic of the subpage. Leave a note at the top of the main talk page linking to any subpages. </p>
      <p>On talk pages that generate significant amounts of discussion, old discussions are often archived to keep the size of the talk page at a manageable level. This may be done either manually or with the help of a <a href="/wiki/Help:Bot" class="mw-redirect" title="Help:Bot">bot</a>. An archive box with links to the discussion archives is normally placed at the top of the current talk page. </p> 
      <h3><span class="mw-headline" id="Assorted_talk_page_boxes_and_graphics">Assorted talk page boxes and graphics</span></h3> 
      <h4><span class="mw-headline" id="Quotations">Quotations</span></h4> 
      <p>Sometimes it is necessary to display a sentence or paragraph from the article on the talk page so that other editors can easily understand what is being discussed. An easy way to do this is to use the {{<a href="/wiki/Template:Quote_frame" title="Template:Quote frame">Quote frame</a>}} or {{<a href="/wiki/Template:Quote" class="mw-redirect" title="Template:Quote">Quote</a>}} templates: </p>
      <p>Use <code>{{Quote frame|Quoted passage from the article.}}</code> to produce: </p> 
      <blockquote class="quote-frame pullquote toccolours" style="float:none; display:table; border: 1px solid #aaa;">
       <div style="padding: 10px 15px 10px 15px;">
        Quoted passage from the article.
       </div>
      </blockquote> 
      <p>Alternatively, use <code>{{Quote frame |Quoted text |Author |Source}}</code> to produce: </p> 
      <blockquote class="quote-frame pullquote toccolours" style="float:none; display:table; border: 1px solid #aaa;">
       <div style="padding: 10px 15px 10px 15px;">
        Quoted text 
        <br><cite style="display: block; text-align: right;"> —?Author , <i>Source</i></cite>
       </div>
      </blockquote> 
      <p>The {{<a href="/wiki/Template:Quote" class="mw-redirect" title="Template:Quote">Quote</a>}} template produces an indented quotation with no box; use <code>{{Quote |Quoted text |''Source''}}</code> to produce: </p> 
      <style data-mw-deduplicate="TemplateStyles:r996844942">.mw-parser-output .templatequote{overflow:hidden;margin:1em 0;padding:0 40px}.mw-parser-output .templatequote .templatequotecite{line-height:1.5em;text-align:left;padding-left:1.6em;margin-top:0}</style>
      <blockquote class="templatequote">
       <p>Quoted text</p>
       <div class="templatequotecite">
        —?<cite><i>Source</i></cite>
       </div>
      </blockquote> 
      <p>Alternatively, use <code>{{Quote|Quoted text |Author |''Source'', page XX}}</code> to add an author and page number: </p> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r996844942">
      <blockquote class="templatequote">
       <p>Quoted text </p>
       <div class="templatequotecite">
        —?<cite>Author, <i>Source</i>, page XX</cite>
       </div>
      </blockquote> 
      <p>For directly quoting another user, the {{<a href="/wiki/Template:Talkquote" class="mw-redirect" title="Template:Talkquote">Talkquote</a>}} template places quoted text inside a green box, as in: <code>{{Talkquote |Quoted text |Example |ts=12:00, 01 January 2000 (UTC)}}</code>, which produces: </p>
