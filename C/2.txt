       <p>A&nbsp;<strong><code>DOMString</code></strong> is a sequence of&nbsp;16-bit unsigned integers, typically interpreted as&nbsp;UTF-16 <a href="https://www.unicode.org/glossary/#code_unit" class="external" rel=" noopener">code units</a>. This corresponds exactly to the JavaScript&nbsp;<a href="https://tc39.es/ecma262/#sec-ecmascript-language-types-string-type" class="external" rel=" noopener">primitive String type</a>. When&nbsp;a <code>DOMString</code> is provided to JavaScript, it maps&nbsp;directly to the corresponding&nbsp;<a href="/en-US/docs/Web/JavaScript/Reference/Global_Objects/String"><code>String</code></a>.</p> 
       <p>When a Web API accepts a&nbsp;<code>DOMString</code>, the value provided will be stringified, using&nbsp;the <code><a href="https://tc39.es/ecma262/#sec-tostring" class="external" rel=" noopener">ToString</a></code> abstract operation. (For types other than Symbol, this has the same behavior as the <a href="/en-US/docs/Web/JavaScript/Reference/Global_Objects/String/String"><code>String()</code></a> function.)</p> 
       <p>Certain Web APIs accepting a <code>DOMString</code>&nbsp;have an additional legacy behavior, where passing <a href="/en-US/docs/Web/JavaScript/Reference/Global_Objects/null"><code>null</code></a> stringifies to the&nbsp;empty string instead of the usual&nbsp;<code>"null"</code>.</p>
      </div>
