This type of archiving is done by <a href="/wiki/User:SpBot" title="User:SpBot">SpBot</a> <small><i>(not by <a href="/wiki/User:ArchiverBot" title="User:ArchiverBot">ArchiverBot</a>)</i></small>. </p>

       <h2><span class="mw-headline" id="Usage">Usage</span></h2> 
       <div class="templatebox-usagesample"> 
        <div class="poem"> 
         <p>{{Autoarchive resolved section<br> &nbsp;|age = <br> &nbsp;|archive = <br> &nbsp;|show = <br> &nbsp;|level = <br> &nbsp;|timecompare = <br> &nbsp;|timeout = <br> &nbsp;|width = <br> }} </p> 
        </div> 
       </div> 
       <p><small></small> </p> 
       <div class="templatebox-languagedependant"> 
        <p>SpBot archiving requires the use of either <a href="/wiki/Template:Section_resolved" title="Template:Section resolved">{{Section resolved}}</a> or the <code>timeout</code> parameter of this template (usual archiving like by <a href="/wiki/User:ArchiverBot" title="User:ArchiverBot">ArchiverBot</a>). </p>
        <p>It does not matter where the template <code><span style="white-space:nowrap;">{{Section resolved|…}}</span></code> is placed within the section. It will always refer to the last/previous heading of the section to be archived (see parameter <code>level</code>). </p>
        <p>The template relies on a certain time stamp format, for more see in <a href="#Signature_format">Signature format</a>. </p>
        <p>To quickly use a standard setup to archive your user talk page, see <a href="/wiki/Template:Autoarchive_resolved_section/usertalksetup" title="Template:Autoarchive resolved section/usertalksetup">Template:Autoarchive resolved section/usertalksetup</a> (see own description). </p> 
       </div> 
       <h3><span class="mw-headline" id="Template_parameters">Template parameters</span></h3> 
       <table class="wikitable templatebox-table">
        <tbody>
         <tr>
          <th style="width:15%">Parameter</th>
          <th style="width:65%" colspan="2">Description</th>
          <th style="width:10%">Default</th>
          <th style="width:10%">Status</th>
         </tr>
         <tr>
          <td><code>age</code></td>
          <td colspan="2">The parameter <i><code>age</code></i> defines the required age of the template block <code><span style="white-space:nowrap;">{{Section resolved|…}}</span></code> in days. After this amount of days the section will be archived.</td>
          <td><span class="mw-templatedata-doc-muted" style="color:#777; font-variant:small-caps">empty</span></td>
          <td><b>required</b></td>
         </tr>
         <tr>
          <td><code>archive</code></td>
          <td colspan="2">Path to archive. You can use variables in your <i><code>archive</code></i> parameter as <a href="#Archive_variables">listed below</a>.<br><small>Path inclusion with ASCII apostrophes (') is not necessary, but allowed for historic reasons.</small></td>
          <td><span class="mw-templatedata-doc-muted" style="color:#777; font-variant:small-caps">empty</span></td>
          <td><b>required</b></td>
         </tr>
         <tr>
          <td><code>overview</code></td>
          <td colspan="2">The parameter <i><code>overview</code></i> can be used to display a link to the location of the archive page. Example code: <code>overview=[[Commons:Village pump/Archive]]/</code> results in<br>“For the archive overview, see <b><a href="/wiki/Commons:Village_pump/Archive" title="Commons:Village pump/Archive">Commons:Village pump/Archive</a></b>.”<br><small>An alternative to this parameter is the use of template <a href="/wiki/Template:Archive_box" title="Template:Archive box">{{Archive box}}</a>.</small></td>
          <td><span class="mw-templatedata-doc-muted" style="color:#777; font-variant:small-caps">empty</span></td>
          <td>optional</td>
         </tr>
         <tr>
          <td><code>latest archive</code></td>
          <td colspan="2">The parameter <i><code>latest archive</code></i> can be used to display a link to the current archive. The <a href="#Archive_variables"><code>archive</code> parameter variables</a> can be used. Example code: <code>latest archive=[[Commons:Village pump/Archive/2011/02]]</code> results in<br>“The latest archive is located under <b><a href="/wiki/Commons:Village_pump/Archive/2011/02" title="Commons:Village pump/Archive/2011/02">Commons:Village pump/Archive/2011/02</a></b>.”</td>
          <td><span class="mw-templatedata-doc-muted" style="color:#777; font-variant:small-caps">empty</span></td>
          <td>optional</td>
         </tr>
         <tr>
          <td><code>show</code></td>
          <td colspan="2">The parameter <i><code>show</code></i> defines, whether the template should display anything or not: <code>show=yes</code> (default) or <code>show=no</code>. This parameter can be used, if the template is used on a page which has an own intro explaining the archiving parameters.</td>
          <td>yes</td>
          <td>optional</td>
         </tr>
         <tr>
          <td><code>level</code></td>
          <td colspan="2">Parameter <i><code>level</code></i> defines which heading level should be archived. Deeper heading levels are treated as part of the specified level. 
           <dl>
            <dt>
             Examples
            </dt> 
            <dd></dd>
           </dl> <p><small>(<i>hint:</i> Count the equal signs)</small> </p> <pre>== talk page section level II ==
</pre> 
           <dl>
            <dd>
             Such a heading is level 2.
            </dd>
           </dl> <pre>==== talk page section level IV ====
</pre> 
           <dl>
            <dd>
             Such a heading is level 4.
            </dd>
           </dl></td>
          <td>2</td>
          <td>optional</td>
         </tr>
         <tr>
          <td><code>timecompare</code></td>
          <td colspan="2">The parameter <i><code>timecompare</code></i> specifies which timestamp should be used for distribution into the archives. In the parameter <i>archive</i> you can use variables referring to the date. <i><code>timecompare</code></i> defines which timestamp is used for archiving. 
           <ul>
            <li><code>timecompare = resolved</code>: timestamp given by <a href="/wiki/Template:Section_resolved" title="Template:Section resolved">{{Section resolved}}</a> block</li> 
            <li>Any other value or empty: the latest timestamp of the whole section</li>
           </ul> Compare template usage in <a href="/wiki/Commons:Graphic_Lab/Illustration_workshop" title="Commons:Graphic Lab/Illustration workshop">Commons:Graphic Lab/Illustration workshop</a>.</td>
          <td><span class="mw-templatedata-doc-muted" style="color:#777; font-variant:small-caps">empty</span></td>
          <td>optional</td>
         </tr>
         <tr>
          <td><code>timeout</code></td>
          <td colspan="2">The parameter <i><code>timeout</code></i> specifies the number of days having to pass by to consider a section as “resolved” automatically, even if no <a href="/wiki/Template:Section_resolved" title="Template:Section resolved">{{Section resolved}}</a> block was set. Archiving will be only proceeded, if the parameter is set and the specified period of time (<i>timeout</i>) has passed since the last comment (<i>note:</i> with value <code>0</code> it gets inactive). The template <a href="/wiki/Template:DNAU" title="Template:DNAU">{{DNAU}}</a> is supported.<br> By setting this parameter you can get almost the same service as given by <i>ArchiverBot</i> (see <a href="/wiki/User:MiszaBot/Archive_HowTo" title="User:MiszaBot/Archive HowTo">User:MiszaBot/Archive HowTo</a>, different name for bot for historic reasons). In this case, it is no longer necessary to use this template and additional <code>{{User:MiszBot/config}}</code>, which are processed by two different bots. Instead, only <a class="mw-selflink selflink">{{Autoarchive resolved section}}</a> for <a href="/wiki/User:SpBot" title="User:SpBot">SpBot</a> is needed.</td>
          <td><span class="mw-templatedata-doc-muted" style="color:#777; font-variant:small-caps">empty</span></td>
          <td>optional</td>
         </tr>
         <tr>
          <td><code>width</code></td>
          <td colspan="2">Width of the archive box</td>
          <td>100%</td>
          <td>optional</td>
         </tr>

       <h2><span id=".E2.80.9CArchive.E2.80.9D_parameter_variables"></span><span class="mw-headline" id="“Archive”_parameter_variables"><span id="Archive_variables"></span>“Archive” parameter variables</span></h2> 
       <p>You can use the following variables in the <code>archive</code> parameter: </p> 
       <table class="wikitable"> 
        <tbody>
         <tr> 
          <th>variable</th> 
          <th>meaning </th>
         </tr> 
         <tr> 
          <td>((year))</td> 
          <td>year, e.g. <i>2021</i> </td>
         </tr> 
         <tr> 
          <td>((month:long))</td> 
          <td>full monthname, e.g. <i>February</i> </td>
         </tr> 
         <tr> 
          <td>((month:short))</td> 
          <td>short monthname, e.g. <i>Feb</i> </td>
         </tr> 
         <tr> 
          <td>((month:##))</td> 
          <td>month as two digits, e.g. <i>02</i> </td>
         </tr> 
         <tr> 
          <td>((week))</td> 
          <td>week date as a number, e.g. <i>4</i> or <i>15</i> (per <a href="https://en.wikipedia.org/wiki/ISO8601#Week_dates" class="extiw" title="en:ISO8601">ISO 8601</a>) </td>
         </tr> 
         <tr> 
          <td>((week:##))</td> 
          <td>week date as a two digits, e.g. <i>04</i> or <i>15</i> (per <a href="https://en.wikipedia.org/wiki/ISO8601#Week_dates" class="extiw" title="en:ISO8601">ISO 8601</a>) </td>
         </tr> 
         <tr> 
          <td>((day:##))</td> 
          <td>day of month as two digits, e.g. <i>07</i> or <i>31</i> </td>
         </tr> 
         <tr> 
          <td>((quarter))</td> 
          <td>quarter, e.g. <i>3</i> </td>
         </tr> 
         <tr> 
          <td>((quarter:##))</td> 
          <td>quarter as two digits, e.g. <i>03</i> </td>
         </tr> 
         <tr> 
          <td>((quarter:i))</td> 
          <td>quarter as lowercase Roman numerals, e.g. <i>iii</i> or <i>iv</i> </td>
         </tr> 
         <tr> 
          <td>((quarter:I))</td> 
          <td>quarter as uppercase Roman numerals, e.g. <i>III</i> or <i>IV</i> </td>
         </tr> 
         <tr> 
          <td>((half-year))</td> 
          <td>half-year, e.g. <i>2</i> </td>
         </tr> 
         <tr> 
          <td>((half-year:i))</td> 
          <td>half-year as lowercase Roman numeral, e.g. <i>ii</i> </td>
         </tr> 
         <tr> 
          <td>((half-year:I))</td> 
          <td>half-year as uppercase Roman numeral, e.g. <i>II</i> </td>
         </tr> 
         <tr> 
          <td>((FULLPAGENAME))</td> 
          <td>The full page name (namespace + actual page) name of the current page. Same as <code><a href="https://en.wikipedia.org/wiki/Help:Magic_words#Variables" class="extiw" title="en:Help:Magic words">{{FULLPAGENAME}}</a></code>, but with normal brackets. Use this instead of hard coding your page name. </td>
         </tr>
        </tbody>
       </table> 
       <p><i>Examples</i>: </p> 
       <pre>archive=((FULLPAGENAME))/Archive/((year))/((month:short))
archive=Commons:Village pump/Archive/((year))/((month:##))
</pre> 
       <h2><span class="mw-headline" id="Example"><span id="Example"></span>Example</span></h2> 
       <p><i>See also:</i> <a href="/wiki/Template:Autoarchive_resolved_section/usertalksetup" title="Template:Autoarchive resolved section/usertalksetup">Template:Autoarchive resolved section/usertalksetup</a> </p>
       <p>The following example is derived from the archiving setup of dewiki’s <a href="https://de.wikipedia.org/wiki/Wikipedia:Redaktion_Chemie" class="extiw" title="de:Wikipedia:Redaktion Chemie">Wikipedia:Redaktion Chemie</a>. All marked/tagged sections with level 2 (that is <code>==</code>) will be archived seven days after placement of a <a href="/wiki/Template:Section_resolved" title="Template:Section resolved">{{Section resolved}}</a> block. The archive is at <a href="https://de.wikipedia.org/wiki/Wikipedia:Redaktion_Chemie/Archiv" class="extiw" title="de:Wikipedia:Redaktion Chemie/Archiv">Wikipedia:Redaktion Chemie/Archiv</a>. The archiving will be done to <a href="https://de.wikipedia.org/wiki/Wikipedia:Redaktion_Chemie/Archiv/2007/Januar" class="extiw" title="de:Wikipedia:Redaktion Chemie/Archiv/2007/Januar">Wikipedia:Redaktion Chemie/Archiv/2007/Januar</a> if the <b>first</b> comment in this section was made in <i>January</i>. </p> 
       <pre>{{Autoarchive resolved section
 |age=7
 |archive='((FULLPAGENAME))/Archive/((year))'
 |overview=[[{{FULLPAGENAME}}/Archive|Archive]]
}}
</pre>

       <h2><span class="mw-headline" id="Signature_format"><span id="Signature_format"></span>Signature format</span></h2> 
       <p>The date and time part of the signature is important for recognition of the timestamp. The default format specified by MediaWiki and automatically created, when <code>~~~~</code> or <code>~~~~~</code> is entered, is sufficient: <i>15:11, 5 June 2021 (UTC)</i>, also with abbreviated month: <i>Jun</i>. This corresponds to the following structure with <a href="https://en.wikipedia.org/wiki/Help:Magic_words#Other_variables_by_type" class="extiw" title="en:Help:Magic words">Mediawiki variables</a>: <code>{{CURRENTTIME}}, {{CURRENTDAY}} {{CURRENTMONTHNAME}} {{CURRENTYEAR}} (UTC)</code> (short month name: <code>{{CURRENTMONTHABBREV}}</code>. When directly entered these variables must be <a href="https://en.wikipedia.org/wiki/Wikipedia:Substitution" class="extiw" title="en:Wikipedia:Substitution">substituted</a> (the according <code>{{LOCAL…}}</code> variants can be used, as well). </p>
       <p>Other formats are not supported. </p>
