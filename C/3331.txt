      <p>The <b><a href="/wiki/MediaWiki:Spam-blacklist" title="MediaWiki:Spam-blacklist">MediaWiki:Spam-blacklist</a></b> exists primarily to control widespread spamming and <a href="/wiki/Wikipedia:BLOCK#Disruption" class="mw-redirect" title="Wikipedia:BLOCK">disruption</a> of Wikimedia Foundation projects. It is intended as a last resort for persistent spamming on the project, by multiple individuals or IP addresses. <a href="/wiki/MediaWiki:Spam-blacklist" title="MediaWiki:Spam-blacklist">Mediawiki:Spam-blacklist</a> is meant to be used by the <a href="https://www.mediawiki.org/wiki/Extension:SpamBlacklist" class="extiw" title="mw:Extension:SpamBlacklist">spam blacklist extension</a>. Unlike the <a href="https://meta.wikimedia.org/wiki/Spam_blacklist" class="extiw" title="m:Spam blacklist">Meta spam blacklist</a>, this blacklist affects pages on the English Wikipedia only. The MediaWiki:Spam-blacklist is a page in the <a href="/wiki/Wikipedia:MediaWiki_namespace" class="mw-redirect" title="Wikipedia:MediaWiki namespace">MediaWiki namespace</a>, which only <a href="/wiki/Wikipedia:ADMIN" class="mw-redirect" title="Wikipedia:ADMIN">administrators</a> may edit. </p>

      <p>Before blacklisting, please consider the following: </p> 
      <ol>
       <li>Does the site have any validity to the project?</li> 
       <li>Have links been placed after warnings/blocks?</li> 
       <li>Would referring this to our anti-spam bot, <a href="/wiki/User_talk:XLinkBot/RevertList" title="User talk:XLinkBot/RevertList">XLinkBot</a> be a more appropriate step?</li> 
       <li>Is there a <a href="/wiki/Wikipedia_talk:WikiProject_Spam" title="Wikipedia talk:WikiProject Spam">WikiProject Spam report</a>?</li> 
       <li>Have <b>all</b> other avenues for dealing with the problem been explored or tried? 
        <ol>
         <li>Can the problem be controlled by page protection?</li> 
         <li>Can the problem be controlled by blocking a single user? Accounts which exist solely to spam can and will be blocked uncontroversially.</li> 
         <li>Fixed IPs with solely external link contributions can probably be blocked uncontroversially.</li> 
         <li>Open proxies should be blocked outright.</li>
        </ol></li> 
       <li>Ensure <b>all links have been removed</b> from articles and discussion pages <b>before blacklisting</b>.</li>
      </ol> 
      <h2><span class="mw-headline" id="Overview">Overview</span></h2> 
      <p>Admins, if you seek to blacklist an external link then please ensure you proceed in the following order (full details of each step given in the rest of this help page): </p> 
      <ol>
       <li><b>Evidence</b>- There should be clear <b>evidence of disruption</b>, <b>persistent spamming</b> or otherwise simply violates Wikipedia's policies or guidelines: 
        <ul>
         <li><b>Request details</b> - A clear request already in existence that describes the situation. If this has not yet been created by another editor (eg at <a href="/wiki/MediaWiki_talk:Spam-blacklist#Proposed_additions" title="MediaWiki talk:Spam-blacklist">MediaWiki talk:Spam-blacklist#Proposed additions</a>), then you will need to create your own full description that shows persistent spamming and the involved registered or IP-anon users. This is required so that others may later review the reasons for the blacklisting. So please add a request to <a href="/wiki/MediaWiki_talk:Spam-blacklist#Proposed_additions" title="MediaWiki talk:Spam-blacklist">MediaWiki talk:Spam-blacklist#Proposed additions</a>. (Then click on the "History" tab as you will later need the version number of the article as evidence&nbsp;– see part 3).</li> 
         <li>Blatantly inappropriate (porn, shock, etc. links of this nature are self evident), refer to a <a href="/wiki/Special:Contributions" title="Special:Contributions">Special:Contributions</a> of an account adding this link.</li> 
         <li><a href="/wiki/Special:Contributions" title="Special:Contributions">Special:Contributions</a> of several accounts (socks) pushing the link.</li> 
         <li>Link-report (e.g. from <a href="/wiki/User:COIBot" title="User:COIBot">User:COIBot</a>) showing widespread abuse and little or no proper use.</li> 
         <li>OTRS ticket (off-wiki proof).</li> 
         <li>Mail to the foundation (for extreme cases, off-wiki proof) .</li>
        </ul></li> 
       <li><b>Blacklist</b> - Add the external link to the <a href="/wiki/MediaWiki:Spam-blacklist" title="MediaWiki:Spam-blacklist">MediaWiki:Spam-blacklist</a>. To do this you <b>must</b> have some competence with regex or innocent sites may be blocked as well. If you require help please run through the history on the list &amp; ask someone for help.</li> 
       <li><b>Log your action</b> - Finally add a log of your action to <a href="/wiki/MediaWiki_talk:Spam-blacklist/log" title="MediaWiki talk:Spam-blacklist/log">MediaWiki talk:Spam-blacklist/log</a>. You will need in this log entry to link to the original request details (i.e. link to request of part 1 above).</li>
      </ol> 
      <h3><span class="mw-headline" id="Handling_additions"><a href="/wiki/File:Admin_mop.PNG" class="image"><img alt="Admin mop.PNG" src="//upload.wikimedia.org/wikipedia/commons/thumb/9/96/Admin_mop.PNG/25px-Admin_mop.PNG" decoding="async" width="25" height="25" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/9/96/Admin_mop.PNG/38px-Admin_mop.PNG 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/9/96/Admin_mop.PNG/50px-Admin_mop.PNG 2x" data-file-width="117" data-file-height="118"></a> Handling additions</span></h3> 
      <p><i><b>Listing</b></i> should include compelling evidence of a problem. Admins, if you yourself wish to blacklist an external link, you too will need to ensure that there is a description of the evidence why blacklisting is required. <a href="/wiki/Wikipedia:SBL#Proposed_additions" class="mw-redirect" title="Wikipedia:SBL">WP:SBL#Proposed additions</a> is usually used by admin if no thread at <a href="/wiki/Wikipedia:AN/I" class="mw-redirect" title="Wikipedia:AN/I">WP:AN/I</a>, <a href="/wiki/Wikipedia_talk:WPSPAM" class="mw-redirect" title="Wikipedia talk:WPSPAM">WT:WPSPAM</a>, talk pages etc, but remember to close your own proposal with a {{<a href="/wiki/Template:Done" title="Template:Done">done</a>}}. </p>
      <p>Links and diffs from multiple projects and multiple articles should be described at <a href="/wiki/MediaWiki_talk:Spam-blacklist" title="MediaWiki talk:Spam-blacklist">MediaWiki talk:Spam-blacklist</a>: </p> 
      <ul>
       <li><a href="/wiki/MediaWiki_talk:Spam-blacklist#Proposed_additions" title="MediaWiki talk:Spam-blacklist">Proposed additions</a>&nbsp;— request a link to be added</li> 
       <li><a href="/wiki/MediaWiki_talk:Spam-blacklist#Proposed_removals" title="MediaWiki talk:Spam-blacklist">Proposed removals</a>&nbsp;— request a link to be <b>removed</b>.</li> 
       <li><a href="/wiki/MediaWiki_talk:Spam-blacklist#Troubleshooting_and_problems" title="MediaWiki talk:Spam-blacklist">Troubleshooting and problems</a>&nbsp;— having problems editing an article, other problem with the spam blacklist. This is <b>not for requesting removal, for that see <a href="/wiki/MediaWiki_talk:Spam-blacklist#Proposed_removals" title="MediaWiki talk:Spam-blacklist">Proposed removals</a>.</b></li>
      </ul> 
      <p>Requests may include the following templated details (do not use "subst:" with these templates): </p> 
      <dl>
       <dd>
        <ul>
         <li><b>{{<a href="/wiki/Template:IPSummary" class="mw-redirect" title="Template:IPSummary">IPSummary</a>}}</b> - to report anonymous editors suspected of spamming: e.g. <tt>{{IPSummary|127.0.0.1}}</tt></li> 
         <li><b>{{<a href="/wiki/Template:UserSummary" class="mw-redirect" title="Template:UserSummary">UserSummary</a>}}</b> - to report registered users suspected of spamming: e.g. <tt>{{UserSummary|Jimbo Wales}}</tt></li> 
         <li><b>{{<a href="/wiki/Template:LinkSummary" class="mw-redirect" title="Template:LinkSummary">LinkSummary</a>}}</b> - to report spam domains: e.g. <tt>{{LinkSummary|spammydomain.com}}</tt> -<small><i>Do not include the "<a rel="nofollow" class="external free" href="http://www">http://www</a>." portion of an <a href="/wiki/URL" title="URL">URL</a> inside this template</i></small></li> 
         <li>Also, please include links (<a href="/wiki/Wikipedia:DIFF" class="mw-redirect" title="Wikipedia:DIFF">"diffs"</a>) to sample spam edits.</li>
        </ul>
       </dd>
      </dl> 
      <p><br> <i><b>Delisting</b></i> should include compelling evidence that the original problem no longer exists or is no longer a risk. For example, a hijacked or cybersquatted domain may have been returned to its rightful owner, or a site which was being used to link copyright-violating content may have cleaned up its act. </p>
      <p>Before approving or denying a listing, please familiarise yourself with the original reasons for Blacklisting the domain. Archived debates can be found with the <a rel="nofollow" class="external text" href="http://toolserver.org/~eagle/spamArchiveSearch.php">Archive Search tool</a>. To locate a specific url, in the <a href="/wiki/MediaWiki_talk:Spam-blacklist/log" title="MediaWiki talk:Spam-blacklist/log"><i>Log</i></a> Use the <a rel="nofollow" class="external text" href="http://toolserver.org/~seth/grep_regexp_from_url.cgi">"<i>Search spamlists tool</i>"</a>. When using the "<i>Search spamlists</i>" tool, be sure to type in the user-defined language, e.g. <i>en</i> (English Wikipedia), <i>fr</i> (French Wikipedia)...etc. </p>

      <h3><span class="mw-headline" id="Adding_entries_to_the_blacklist"><a href="/wiki/File:Admin_mop.PNG" class="image"><img alt="Admin mop.PNG" src="//upload.wikimedia.org/wikipedia/commons/thumb/9/96/Admin_mop.PNG/25px-Admin_mop.PNG" decoding="async" width="25" height="25" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/9/96/Admin_mop.PNG/38px-Admin_mop.PNG 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/9/96/Admin_mop.PNG/50px-Admin_mop.PNG 2x" data-file-width="117" data-file-height="118"></a> Adding entries to the blacklist</span></h3> 
      <p><font color="red"><i><b>Warning</b></i></font>: <b>Please do not do this unless you are familiar with <a href="/wiki/Regular_expressions" class="mw-redirect" title="Regular expressions">regex</a>&nbsp;– the disruption that can be caused is substantial</b>. </p> 
      <ul>
       <li>Make the entry at the bottom of the page <a href="/wiki/MediaWiki:Spam-blacklist" title="MediaWiki:Spam-blacklist">MediaWiki:Spam-blacklist</a> This is the actual blacklist.</li> 
       <li>Write a <a href="/wiki/Regular_expression" title="Regular expression">regular expression</a> to match the target domain. For example <i><b>http://www.spammydomain.com</b></i> would read;</li>
      </ul> 
      <p><br> </p> 
      <dl>
       <dd>
        <b>\bspammydomain\.com\b</b>
       </dd>
      </dl> 
      <p><br> Having blacklisted the url you will next need to: </p> 
      <ol>
       <li><b>Remove any remaining <i>urls</i></b> from articles and remove the <i>http://</i> from discussion pages (archived or urls on non active pages need not be removed).</li> 
       <li><b>Log the entry</b> as described in the next section</li>
      </ol> 
      <h3><span class="mw-headline" id="Logging_entries_on_the_blacklist"><a href="/wiki/File:Admin_mop.PNG" class="image"><img alt="Admin mop.PNG" src="//upload.wikimedia.org/wikipedia/commons/thumb/9/96/Admin_mop.PNG/25px-Admin_mop.PNG" decoding="async" width="25" height="25" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/9/96/Admin_mop.PNG/38px-Admin_mop.PNG 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/9/96/Admin_mop.PNG/50px-Admin_mop.PNG 2x" data-file-width="117" data-file-height="118"></a> Logging entries on the blacklist</span></h3> 
      <dl>
       <dd>
        <dl>
         <dd>
          <i><b>Note:</b></i> <i><span style="background:#fff990">if you do not log your entries it may be removed if someone appeals and no valid reasons can be found.</span></i>
         </dd>
        </dl>
       </dd>
      </dl> 
      <p>You must bear in mind that if someone challenges a blacklisting in 6 or 12 months time any admin dealing with such a request <b>must</b> be able to find the reason for the listing. The admin who places the entry on the list may not be around and, if there was no "actual" request the archives will be no use. Indeed they will only be helpful if there is full detail in the request including permanent links for example. <i><b>Logging</b></i> a request is essential for future reference. </p>
      <p>Add your log entry to the relevant dated section of <a href="/wiki/MediaWiki_talk:Spam-blacklist/log" title="MediaWiki talk:Spam-blacklist/log">MediaWiki talk:Spam-blacklist/log</a>. <font color="red"><big><b>Warning:</b></big></font> <b>if you do not log any entry you make on the blacklist, it may well be removed if someone appeals and no valid reasons can be found</b>. </p> 
      <ul>
       <li>Example (see below re use of {{<a href="/wiki/Template:WPSPAM" title="Template:WPSPAM">WPSPAM</a>}} link to evidence at <a href="/wiki/Wikipedia_talk:WikiProject_Spam" title="Wikipedia talk:WikiProject Spam">Wikipedia talk:WikiProject Spam</a>):</li>
      </ul> 
      <pre>\bspammydomain\.com\b           # <i><b>your username</b></i>   # See (<i><b>your link to evidence</b></i>)
</pre> 
      <ul>
       <li><b>Logging</b> multiple domains</li>
      </ul> 
      <pre>#:                             # <i><b>your username</b></i>   # See (<i><b>your link to evidence</b></i>)
  \bspammydomain\.com\b <br>
  \bspammydomain\.net\b <br>
  \bbadsite\.com\b <br>
  [badsite]{3}\.com
</pre> 
      <p>For single entries, write the entry, name, and reason on one line in columns seperated by the '#' symbol. For multiple entries, write your name and reason on one line beginning with " #:", then list the entries on new lines below indented by three. Note the space <i>before</i> the regexed urls (and #), these spaces are only added on the log page. </p>
