        <h1 class="border-bottom-0">Using content attachments</h1> 
        <div class="d-none d-lg-block ml-2"> <button class="btn-link Link--secondary js-print tooltipped tooltipped-n" aria-label="Print this article"> <!-- From https://heroicons.dev/ --> 
          <svg fill="none" height="18" width="18" stroke-linecap="round" stroke-linejoin="round" stroke-width="2" viewbox="0 0 24 24" stroke="currentColor">
           <path d="M17 17h2a2 2 0 002-2v-4a2 2 0 00-2-2H5a2 2 0 00-2 2v4a2 2 0 002 2h2m2 4h6a2 2 0 002-2v-4a2 2 0 00-2-2H9a2 2 0 00-2 2v4a2 2 0 002 2zm8-12V5a2 2 0 00-2-2H9a2 2 0 00-2 2v4h10z"></path>

       <div class="lead-mktg">
        <p>Content attachments allow a GitHub App to provide more information in GitHub for URLs that link to registered domains. GitHub renders the information provided by the app under the URL in the body or comment of an issue or pull request.</p>
       </div>

         <li class="ml-0  mb-2 lh-condensed"><a href="#implementing-the-content-attachment-flow">Implementing the content attachment flow</a></li> 
         <li class="ml-0  mb-2 lh-condensed"><a href="#using-content-attachments-in-graphql">Using content attachments in GraphQL</a></li>

       <div class="extended-markdown note border rounded-1 mb-4 p-3 color-border-info color-bg-info f5"> 
        <p><strong>Note:</strong> The Content Attachments API is currently in public beta and only available for use with GitHub Apps. Features and requirements may change at any time during this period.</p> 
       </div> 
       <h3 id="about-content-attachments"><a href="#about-content-attachments">About content attachments</a></h3> 
       <p>A GitHub App can register domains that will trigger <code>content_reference</code> events. When someone includes a URL that links to a registered domain in the body or comment of an issue or pull request, the app receives the <a href="/en/webhooks/event-payloads/#content_reference"><code>content_reference</code> webhook</a>. You can use content attachments to visually provide more context or data for the URL added to an issue or pull request. The URL must be a fully-qualified URL, starting with either <code>http://</code> or <code>https://</code>. URLs that are part of a markdown link are ignored and don't trigger the <code>content_reference</code> event.</p> 
       <p>Before you can use the Content Attachments API, you'll need to configure content references for your GitHub App:</p> 
       <ul> 
        <li>Give your app <code>Read &amp; write</code> permissions for "Content references."</li> 
        <li>Register up to 5 valid, publicly accessible domains when configuring the "Content references" permission. Do not use IP addresses when configuring content reference domains. You can register a domain name (example.com) or a subdomain (subdomain.example.com).</li> 
        <li>Subscribe your app to the "Content reference" event.</li> 
       </ul> 
       <p>Once your app is installed on a repository, issue or pull request comments in the repository that contain URLs for your registered domains will generate a content reference event. The app must create a content attachment within six hours of the content reference URL being posted.</p> 
       <p>Content attachments will not retroactively update URLs. It only works for URLs added to issues or pull requests after you configure the app using the requirements outlined above and then someone installs the app on their repository.</p> 
       <p>See "<a href="/en/apps/building-github-apps/creating-a-github-app">Creating a GitHub App</a>" or "<a href="/en/apps/managing-github-apps/editing-a-github-app-s-permissions">Editing a GitHub App's permissions</a>" for the steps needed to configure GitHub App permissions and event subscriptions.</p> 
       <h3 id="implementing-the-content-attachment-flow"><a href="#implementing-the-content-attachment-flow">Implementing the content attachment flow</a></h3> 
       <p>The content attachment flow shows you the relationship between the URL in the issue or pull request, the <code>content_reference</code> webhook event, and the REST API endpoint you need to call to update the issue or pull request with additional information:</p> 
       <p><strong>Step 1.</strong> Set up your app using the guidelines outlined in <a href="#about-content-attachments">About content attachments</a>. You can also use the <a href="#example-using-probot-and-github-app-manifests">Probot App example</a> to get started with content attachments.</p> 
       <p><strong>Step 2.</strong> Add the URL for the domain you registered to an issue or pull request. You must use a fully qualified URL that starts with <code>http://</code> or <code>https://</code>.</p> 
       <p><img src="/assets/images/github-apps/github_apps_content_reference.png" alt="URL added to an issue"></p> 
       <p><strong>Step 3.</strong> Your app will receive the <a href="/en/webhooks/event-payloads/#content_reference"><code>content_reference</code> webhook</a> with the action <code>created</code>.</p> 
       <pre><code class="hljs language-json">{
  <span class="hljs-attr">"action"</span>: <span class="hljs-string">"created"</span>,
  <span class="hljs-attr">"content_reference"</span>: {
    <span class="hljs-attr">"id"</span>: <span class="hljs-number">17</span>,
    <span class="hljs-attr">"node_id"</span>: <span class="hljs-string">"MDE2OkNvbnRlbnRSZWZlcmVuY2UxNjA5"</span>,
    <span class="hljs-attr">"reference"</span>: <span class="hljs-string">"errors.ai"</span>
  },
  <span class="hljs-attr">"repository"</span>: {
    <span class="hljs-attr">"full_name"</span>: <span class="hljs-string">"Codertocat/Hello-World"</span>,
  },
  <span class="hljs-attr">"sender"</span>: {...},
  <span class="hljs-attr">"installation"</span>: {
    <span class="hljs-attr">"id"</span>: <span class="hljs-number">371641</span>,
    <span class="hljs-attr">"node_id"</span>: <span class="hljs-string">"MDIzOkludGVncmF0aW9uSW5zdGFsbGF0aW9uMzcxNjQx"</span>
  }
}
</code></pre> 
       <p><strong>Step 4.</strong> The app uses the <code>content_reference</code> <code>id</code> and <code>repository</code> <code>full_name</code> fields to <a href="/en/rest/reference/apps#create-a-content-attachment">Create a content attachment</a> using the REST API. You'll also need the <code>installation</code> <code>id</code> to authenticate as a <a href="/en/apps/building-github-apps/authenticating-with-github-apps/#authenticating-as-an-installation">GitHub App installation</a>.</p> 
       <div class="extended-markdown note border rounded-1 mb-4 p-3 color-border-info color-bg-info f5"> 
        <p><strong>Note:</strong> To access the Content Attachments API during the preview period, you must provide a custom <a href="/en/rest/overview/media-types">media type</a> in the <code>Accept</code> header:</p> 
        <pre><code>application/vnd.github.corsair-preview+json
</code></pre> 
       </div> 
       <div class="extended-markdown warning border rounded-1 mb-4 p-3 color-border-danger color-bg-danger f5"> 
        <p><strong>Warning:</strong> The API may change without advance notice during the preview period. Preview features are not supported for production use. If you experience any issues, contact <a href="https://support.github.com/contact">GitHub Support</a> or <a href="https://premium.githubsupport.com">GitHub Premium Support</a>.</p> 
       </div> 
       <p>The <code>body</code> parameter can contain markdown:</p> 
       <pre><code>&lt;pre&gt;&lt;code class="hljs language-shell"&gt;curl -X POST \
  https://api.github.com/repos/Codertocat/Hello-World/content_references/17/attachments \
  -H 'Accept: application/vnd.github.corsair-preview+json' \
  -H 'Authorization: Bearer $INSTALLATION_TOKEN' \
  -d '{
    "title": "[A-1234] Error found in core/models.py file",
    "body": "You have used an email that already exists for the user_email_uniq field.\n ## DETAILS:\n\nThe (email)=(Octocat@github.com) already exists.\n\n The error was found in core/models.py in get_or_create_user at line 62.\n\n self.save()"
}'&lt;/code&gt;&lt;/pre&gt;
</code></pre> 
       <p>For more information about creating an installation token, see "<a href="/en/apps/building-github-apps/authenticating-with-github-apps/#authenticating-as-an-installation">Authenticating as a GitHub App</a>."</p> 
       <p><strong>Step 5.</strong> You'll see the new content attachment appear under the link in a pull request or issue comment:</p> 
       <p><img src="/assets/images/github-apps/content_reference_attachment.png" alt="Content attached to a reference in an issue"></p> 
       <h3 id="using-content-attachments-in-graphql"><a href="#using-content-attachments-in-graphql">Using content attachments in GraphQL</a></h3> 
       <p>We provide the <code>node_id</code> in the <a href="/en/webhooks/event-payloads/#content_reference"><code>content_reference</code> webhook</a> event so you can refer to the <code>createContentAttachment</code> mutation in the GraphQL API.</p> 
       <div class="extended-markdown note border rounded-1 mb-4 p-3 color-border-info color-bg-info f5"> 
        <p><strong>Note:</strong> To access the Content Attachments API during the preview period, you must provide a custom <a href="/en/rest/overview/media-types">media type</a> in the <code>Accept</code> header:</p> 
        <pre><code>application/vnd.github.corsair-preview+json
</code></pre> 
       </div> 
       <div class="extended-markdown warning border rounded-1 mb-4 p-3 color-border-danger color-bg-danger f5"> 
        <p><strong>Warning:</strong> The API may change without advance notice during the preview period. Preview features are not supported for production use. If you experience any issues, contact <a href="https://support.github.com/contact">GitHub Support</a> or <a href="https://premium.githubsupport.com">GitHub Premium Support</a>.</p> 
       </div> 
       <p>For example:</p> 
       <pre><code class="hljs language-graphql"><span class="hljs-keyword">mutation</span> {

    body:<span class="hljs-string">"You have used an email that already exists for the user_email_uniq field.\n ## DETAILS:\n\nThe (email)=(Octocat@github.com) already exists.\n\n The error was found in core/models.py in get_or_create_user at line 62.\n\n self.save()"</span>
  }) {
    contentAttachment {
      ... <span class="hljs-keyword">on</span><span class="hljs-literal"> ContentAttachment</span> {
        id
        title
        body
      }
    }
  }
}
</code></pre> 
       <p>Example cURL:</p> 
       <pre><code class="hljs language-shell">curl -X "POST" "https://api.github.com/graphql" \

       <p>For more information on <code>node_id</code>, see "<a href="/en/graphql/guides/using-global-node-ids">Using Global Node IDs</a>."</p> 
       <h3 id="example-using-probot-and-github-app-manifests"><a href="#example-using-probot-and-github-app-manifests">Example using Probot and GitHub App Manifests</a></h3> 
       <p>To quickly setup a GitHub App that can use the Content Attachments API, you can use <a href="https://probot.github.io/">Probot</a>. See "<a href="/en/apps/building-github-apps/creating-github-apps-from-a-manifest">Creating GitHub Apps from a manifest</a>" to learn how Probot uses GitHub App Manifests.</p> 
       <p>To create a Probot App, follow these steps:</p> 
       <ol> 
        <li> <p><a href="https://probot.github.io/docs/development/#generating-a-new-app">Generate a new GitHub App</a>.</p> </li> 
        <li> <p>Open the project you created, and customize the settings in the <code>app.yml</code> file. Subscribe to the <code>content_reference</code> event and enable <code>content_references</code> write permissions:</p> <pre><code class="hljs language-yml"> <span class="hljs-attr">default_events:</span>
