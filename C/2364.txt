       <header style="z-index: 1;" class="container-xl position-sticky top-0 color-bg-secondary border-bottom px-3 pt-4 pb-2 js-release-notes-patch-heading" data-version="3.1.0"> 
        <div class="d-flex flex-items-center"> 
         <h2 class="border-bottom-0 m-0 p-0"> Enterprise Server 3.1.0 </h2> <a href="https://enterprise.github.com/releases/3.1.0/download" class="ml-3 text-small text-bold"> Download </a> <button class="js-print btn-link ml-3 text-small text-bold"> Print </button>

        <div class="mt-3">
         <p>For minimum infrastructure requirements, see "<a href="/en/enterprise-server@3.1/admin/enterprise-management/upgrading-github-enterprise-server#about-minimum-requirements-for-github-enterprise-server-30-and-later">About minimum requirements for GitHub Enterprise Server 3.0 and later</a>."</p>
        </div> 
        <div class="release-notes-section-features py-6 d-block d-xl-flex gutter-xl flex-items-baseline border-bottom"> 
         <div class="col-12 col-xl-3 mb-5"> <span class="px-3 py-2 text-small text-bold text-uppercase text-mono color-text-inverse release-notes-section-label">Features</span> 
         </div> 
         <ul class="col-12 col-xl-9 release-notes-list"> 
          <li class="release-notes-list-item list-style-none"> <h4 id="github-advanced-security-secret-scanning" class="release-notes-section-heading text-uppercase text-bold"> <a href="#github-advanced-security-secret-scanning" class="text-inherit">GitHub Advanced Security Secret Scanning</a> </h4> 
           <ul class="pl-0 pb-4 mt-2 release-notes-list"> 
            <li class="release-notes-list-item"><p><a href="https://github.com/features/security">Secret Scanning</a> is now generally available on GitHub Enterprise Server 3.1+. Scan public and private repositories for committed credentials, find secrets, and notify the secret provider or admin the moment they are committed into a repository.</p> <p>This release includes several improvements from the beta of Secret Scanning on GitHub Enterprise Server:</p> 
             <ul> 
              <li>Expanded our <a href="/en/enterprise-server@3.1/code-security/secret-security/about-secret-scanning#about-secret-scanning-for-private-repositories">pattern coverage</a> from 24 partners to 37</li> 
              <li>Added an <a href="/en/enterprise-server@3.1/rest/reference/secret-scanning">API</a> and <a href="/en/enterprise-server@3.1/developers/webhooks-and-events/webhook-events-and-payloads#secret_scanning_alert">webhooks</a></li> 
              <li>Added <a href="https://github.blog/changelog/2021-03-05-secret-scanning-notifications-for-commit-authors-on-private-repositories/">notifications for commit authors</a> when they commit secrets</li> 
              <li>Updated the index view to made it easy to triage secrets in bulk</li> 
              <li>Reduced the false positive rate on many patterns</li> 
             </ul> <p>Administrators using GitHub Advanced Security can <a href="/en/enterprise-server@3.1/admin/configuration/configuring-secret-scanning-for-your-appliance">enable and configure</a> GitHub Advanced Security secret scanning. You can review the <a href="/en/enterprise-server@3.1/admin/installation/setting-up-a-github-enterprise-server-instance">updated minimum requirements for your platform</a> before you turn on GitHub Advanced Security secret scanning.</p></li> 
           </ul> </li> 
          <li class="release-notes-list-item list-style-none"> <h4 id="github-advanced-security-billing-improvements" class="release-notes-section-heading text-uppercase text-bold"> <a href="#github-advanced-security-billing-improvements" class="text-inherit">GitHub Advanced Security billing improvements</a> </h4> 
           <ul class="pl-0 pb-4 mt-2 release-notes-list"> 
            <li class="release-notes-list-item"><p>This release includes several improvements to GitHub Advanced Security billing in GitHub Enterprise Server:</p> 
             <ul> 
              <li>GitHub Advanced Security customers can now view their active committer count and the remaining number of unused committer seats on their organization or enterprise account’s Billing page. If Advanced Security is purchased for an enterprise, administrators can also view the active committer seats which are being used by other organizations within their enterprise. For more information, see "<a href="/en/enterprise-server@3.1/admin/advanced-security/about-licensing-for-github-advanced-security">About GitHub Advanced Security licensing</a>" and "<a href="/en/enterprise-server@3.1/admin/advanced-security/viewing-your-github-advanced-security-usage">Viewing your GitHub Advanced Security usage</a>."</li> 
              <li>GitHub Advanced Security customers can now view their active committer count for any Advanced Security enabled repositories on their organization or enterprise account's Billing page. These changes help billing administrators track their usage against how many committer licenses they purchased. For more information see "<a href="/en/enterprise-server@3.1/organizations/keeping-your-organization-secure/managing-security-and-analysis-settings-for-your-organization">Managing security and analysis settings for your organization</a>."</li> 
             </ul></li> 
           </ul> </li> 
          <li class="release-notes-list-item list-style-none"> <h4 id="dependabot-improvements" class="release-notes-section-heading text-uppercase text-bold"> <a href="#dependabot-improvements" class="text-inherit">Dependabot improvements</a> </h4> 
           <ul class="pl-0 pb-4 mt-2 release-notes-list"> 
            <li class="release-notes-list-item"><p>This release includes improvements to Dependabot alerts in GitHub Enterprise Server:</p> 
             <ul> 
              <li>Users with Dependabot alerts enabled can see which of their repositories are impacted by a given vulnerability by navigating to its entry in the <a href="https://github.com/advisories">GitHub Advisory Database</a>. This feature is available in public beta. For more information, see "<a href="/en/enterprise-server@3.1/code-security/supply-chain-security/viewing-and-updating-vulnerable-dependencies-in-your-repository">Viewing and updating vulnerable dependencies in your repository</a>."</li> 
              <li>When a vulnerability is added to GitHub Advisory Database, you will no longer receive <a href="https://github.com/notifications">email and web notifications</a> for Dependabot alerts on low and moderate severity vulnerabilities. These alerts are still accessible from the repository's Security tab. For more information, see "<a href="/en/enterprise-server@3.1/code-security/supply-chain-security/viewing-and-updating-vulnerable-dependencies-in-your-repository">Viewing and updating vulnerable dependencies in your repository</a>."</li> 
              <li>You can now give people instructions on how to responsibly report security vulnerabilities in your project by adding a <code>SECURITY.md</code> file to your repository's <code>root</code>, <code>docs</code>, or <code>.github</code> folder. When someone creates an issue in your repository, they will see a link to your project's security policy. For more information, see "<a href="/en/enterprise-server@3.1/code-security/getting-started/adding-a-security-policy-to-your-repository">Adding a security policy to your repository</a>."</li> 
             </ul></li> 
           </ul> </li> 
          <li class="release-notes-list-item list-style-none"> <h4 id="github-actions-workflow-visualization-beta" class="release-notes-section-heading text-uppercase text-bold"> <a href="#github-actions-workflow-visualization-beta" class="text-inherit">GitHub Actions Workflow Visualization beta</a> </h4> 
           <ul class="pl-0 pb-4 mt-2 release-notes-list"> 
            <li class="release-notes-list-item"><p>GitHub Actions can now generate a visual graph of your workflow on every run. With workflow visualization, you can:</p> 
             <ul> 
              <li>View and understand complex workflows</li> 
              <li>Track progress of workflows in real-time</li> 
              <li>Troubleshoot runs quickly by easily accessing logs and jobs metadata</li> 
              <li>Monitor progress of deployment jobs and easily access deployment targets</li> 
             </ul> <p>For more information, see "<a href="/en/enterprise-server@3.1/actions/managing-workflow-runs/using-the-visualization-graph">Using the visualization graph</a>."</p></li> 
           </ul> </li> 
          <li class="release-notes-list-item list-style-none"> <h4 id="oauth-20-device-authorization-grant" class="release-notes-section-heading text-uppercase text-bold"> <a href="#oauth-20-device-authorization-grant" class="text-inherit">OAuth 2.0 Device Authorization Grant</a> </h4> 
           <ul class="pl-0 pb-4 mt-2 release-notes-list"> 
            <li class="release-notes-list-item"><p><a href="https://github.com/login/device">OAuth 2.0 Device Authorization Grant</a> allows any CLI client or developer tool to authenticate using a secondary system with a browser.</p> <p>Administrators using <a href="/en/enterprise-server@3.1/developers/apps/authorizing-oauth-apps#device-flow">OAuth Apps</a> and <a href="/en/enterprise-server@3.1/developers/apps/authorizing-oauth-apps#device-flow">GitHub Apps</a> can enable and configure OAuth 2.0 Device Authorization Flow, in addition to the existing Web Application Flow. You can review the <a href="/en/enterprise-server@3.1/admin/installation/setting-up-a-github-enterprise-server-instance">updated minimum requirements for your platform</a> before you enable OAuth 2.0 Device Authorization Flow.</p></li> 
           </ul> </li> 
          <li class="release-notes-list-item list-style-none"> <h4 id="pull-request-auto-merge" class="release-notes-section-heading text-uppercase text-bold"> <a href="#pull-request-auto-merge" class="text-inherit">Pull request auto-merge</a> </h4> 
           <ul class="pl-0 pb-4 mt-2 release-notes-list"> 
            <li class="release-notes-list-item"><p>With auto-merge, pull requests can be set to merge automatically when all merge requirements have been satisfied. This saves users from needing to constantly check the state of their pull requests just to merge them. Auto-merge can be enabled by a user with permission to merge and on pull requests that have unsatisfied merge requirements. For more information, see "<a href="/en/enterprise-server@3.1/github/collaborating-with-issues-and-pull-requests/automatically-merging-a-pull-request">Automatically merging a pull request</a>."</p></li> 
           </ul> </li> 
          <li class="release-notes-list-item list-style-none"> <h4 id="custom-notifications" class="release-notes-section-heading text-uppercase text-bold"> <a href="#custom-notifications" class="text-inherit">Custom notifications</a> </h4> 
           <ul class="pl-0 pb-4 mt-2 release-notes-list"> 
            <li class="release-notes-list-item"><p>You can customize the types of notifications you want to receive from individual repositories. For more information, see "<a href="/en/enterprise-server@3.1/github/managing-subscriptions-and-notifications-on-github/configuring-notifications#configuring-your-watch-settings-for-an-individual-repository">Configuring notifications</a>."</p></li> 
           </ul> </li> 
          <li class="release-notes-list-item list-style-none"> <h4 id="github-mobile-filtering" class="release-notes-section-heading text-uppercase text-bold"> <a href="#github-mobile-filtering" class="text-inherit">GitHub Mobile filtering</a> </h4> 
           <ul class="pl-0 pb-4 mt-2 release-notes-list"> 
            <li class="release-notes-list-item"><p><a href="https://github.com/mobile">GitHub for mobile</a> filtering allows you to search for and find issues, pull requests, and discussions from your device. New metadata for issues and pull request list items allow you to filter by assignees, checks status, review states, and comment counts.</p> <p>GitHub for mobile beta is available for GitHub Enterprise Server. Sign in with our <a href="https://play.google.com/store/apps/details?id=com.github.android">Android</a> and <a href="https://apps.apple.com/app/github/id1477376905">iOS</a> apps to triage notifications and manage issues and pull requests on the go. Administrators can disable mobile support for their Enterprise using the management console or by running <code>ghe-config app.mobile.enabled false</code>. For more information, see "<a href="/en/enterprise-server@3.1/github/getting-started-with-github/using-github/github-for-mobile">GitHub for mobile</a>."</p></li> 
           </ul> </li> 
         </ul> 
        </div> 
        <div class="release-notes-section-changes py-6 d-block d-xl-flex gutter-xl flex-items-baseline border-bottom"> 
         <div class="col-12 col-xl-3 mb-5"> <span class="px-3 py-2 text-small text-bold text-uppercase text-mono color-text-inverse release-notes-section-label">Changes</span> 
         </div> 
         <ul class="col-12 col-xl-9 release-notes-list"> 
          <li class="release-notes-list-item list-style-none"> <h4 id="administration-changes" class="release-notes-section-heading text-uppercase text-bold"> <a href="#administration-changes" class="text-inherit">Administration Changes</a> </h4> 
           <ul class="pl-0 pb-4 mt-2 release-notes-list"> 
            <li class="release-notes-list-item"><p>By precomputing checksums, the amount of time a repository is under the lock has reduced dramatically, allowing more write operations to succeed immediately and improving monorepo performance.</p></li> 
            <li class="release-notes-list-item"><p>The latest release of the CodeQL CLI supports uploading analysis results to GitHub. This makes it easier to run code analysis for customers who wish to use CI/CD systems other than GitHub Actions. Previously, such users had to use the separate CodeQL runner, which will continue to be available. For more information, see "<a href="/en/enterprise-server@3.1/code-security/secure-coding/about-codeql-code-scanning-in-your-ci-system">About CodeQL code scanning in your CI system</a>."</p></li> 
            <li class="release-notes-list-item"><p>GitHub Actions now supports skipping <code>push</code> and <code>pull_request</code> workflows by looking for some common keywords in your commit message.</p></li> 
            <li class="release-notes-list-item"><p>Check annotations older than four months will be archived.</p></li> 
           </ul> </li> 
          <li class="release-notes-list-item list-style-none"> <h4 id="security-changes" class="release-notes-section-heading text-uppercase text-bold"> <a href="#security-changes" class="text-inherit">Security Changes</a> </h4> 
           <ul class="pl-0 pb-4 mt-2 release-notes-list"> 
            <li class="release-notes-list-item"><p>Following feedback, display of Code Scanning results on a pull request without submitting with a pull request ID will remain supported. For more information, see "<a href="/en/enterprise-server@3.1/code-security/secure-coding/configuring-code-scanning#scanning-pull-requests">Configuring code scanning</a>" and "<a href="/en/enterprise-server@3.1/code-security/secure-coding/configuring-codeql-code-scanning-in-your-ci-system#scanning-pull-requests">Configuring CodeQL code scanning in your CI system</a>.</p></li> 
            <li class="release-notes-list-item"><p>SARIF upload support increased to a maximum of 5000 results per upload.</p></li> 
           </ul> </li> 
          <li class="release-notes-list-item list-style-none"> <h4 id="developer-changes" class="release-notes-section-heading text-uppercase text-bold"> <a href="#developer-changes" class="text-inherit">Developer Changes</a> </h4> 
           <ul class="pl-0 pb-4 mt-2 release-notes-list"> 
            <li class="release-notes-list-item"><p>You can specify multiple callback URLs while configuring a GitHub App. This can be used in services with multiple domains or subdomains. GitHub will always deny authorization if the callback URL from the request is not in the authorization callback URL list.</p></li> 
            <li class="release-notes-list-item"><p>The GitHub App file permission has been updated to allow an app developer to specify up to 10 files for read-only or read-write access that their app can request access to.</p></li> 
            <li class="release-notes-list-item"><p>CodeQL now supports more <a href="https://codeql.github.com/docs/codeql-overview/supported-languages-and-frameworks/">libraries and frameworks</a> for a variety of languages (<a href="https://github.com/github/codeql/tree/main/cpp">C++</a>, <a href="https://github.com/github/codeql/tree/main/javascript">JavaScript</a>, <a href="https://github.com/github/codeql/tree/main/python">Python</a>,<a href="https://github.com/github/codeql/tree/main/java">Java</a>, <a href="https://github.com/github/codeql-go/tree/main">Go</a>). The CodeQL engine can now detect more sources of untrusted user data, which improves the quality and depth of the code scanning alerts. For more information, see "<a href="https://codeql.github.com/docs/codeql-overview/about-codeql/">About CodeQL</a>."</p></li> 
            <li class="release-notes-list-item"><p>When configuring a GitHub App, the authorization callback URL is a required field. Now, we allow the developer to specify multiple callback URLs. This can be used in services with multiple domains or subdomains. GitHub will always deny authorization if the callback URL from the request is not in the authorization callback URL list.</p></li> 
            <li class="release-notes-list-item"><p>Delete an entire directory of files, including subdirectories, from your web browser. For more information, see "<a href="/en/enterprise-server@3.1/github/managing-files-in-a-repository/deleting-files-in-a-repository#deleting-a-directory">Deleting a file or directory</a>."</p></li> 
            <li class="release-notes-list-item"><p>Include multiple words after the <code>#</code> in an issue, discussion, or pull request comment to further narrow your search.</p></li> 
            <li class="release-notes-list-item"><p>When you’re writing an issue, pull request, or discussion comment the list syntax for bullets, numbers, and tasks autocompletes after you press <code>return</code> or <code>enter</code>.</p></li> 
           </ul> </li> 
          <li class="release-notes-list-item list-style-none"> <h4 id="api-changes" class="release-notes-section-heading text-uppercase text-bold"> <a href="#api-changes" class="text-inherit">API Changes</a> </h4> 
           <ul class="pl-0 pb-4 mt-2 release-notes-list"> 
            <li class="release-notes-list-item"><p>The code scanning API allows users to upload data about static analysis security testing results, or export data about alerts. For more information, see the <a href="/en/enterprise-server@3.1/rest/reference/code-scanning">code scanning API reference</a>.</p></li> 
            <li class="release-notes-list-item"><p>The <a href="/en/enterprise-server@3.1/rest/reference/apps">GitHub Apps API</a> for managing installations has now graduated from an API preview to a generally available API. The <a href="/en/enterprise-server@3.1/rest/overview/api-previews">preview header</a> is no longer required to access these endpoints.</p></li> 
           </ul> </li> 
         </ul> 
        </div> 
        <div class="release-notes-section-security_fixes py-6 d-block d-xl-flex gutter-xl flex-items-baseline border-bottom"> 
         <div class="col-12 col-xl-3 mb-5"> <span class="px-3 py-2 text-small text-bold text-uppercase text-mono color-text-inverse release-notes-section-label">Security fixes</span> 
         </div> 
         <ul class="col-12 col-xl-9 release-notes-list"> 
          <li class="release-notes-list-item "> <p><strong>MEDIUM</strong> Under certain circumstances, users who were removed from a team or organization could retain write access to branches they had existing pull requests opened for.</p> </li> 
          <li class="release-notes-list-item "> <p>Packages have been updated to the latest security versions.</p> </li> 
         </ul>

         <div class="col-12 col-xl-3 mb-5"> <span class="px-3 py-2 text-small text-bold text-uppercase text-mono color-text-inverse release-notes-section-label">Bug fixes</span>

         <ul class="col-12 col-xl-9 release-notes-list"> 
          <li class="release-notes-list-item list-style-none"> <h4 id="fixes-for-known-issues-from-release-candidate" class="release-notes-section-heading text-uppercase text-bold"> <a href="#fixes-for-known-issues-from-release-candidate" class="text-inherit">Fixes for known issues from Release Candidate</a> </h4> 
           <ul class="pl-0 pb-4 mt-2 release-notes-list">

           </ul> </li> 
          <li class="release-notes-list-item list-style-none"> <h4 id="fixes-for-other-issues" class="release-notes-section-heading text-uppercase text-bold"> <a href="#fixes-for-other-issues" class="text-inherit">Fixes for other issues</a> </h4> 
           <ul class="pl-0 pb-4 mt-2 release-notes-list"> 
            <li class="release-notes-list-item"><p>On the "Configure Actions and Packages" page of the initial installation process, clicking on the "Test domain settings" button did not complete the test.</p></li> 
            <li class="release-notes-list-item"><p>Running <code>ghe-btop</code> failed with an error and cannot find a <code>babeld</code> container.</p></li> 
            <li class="release-notes-list-item"><p>MySQL could reload and cause downtime if you change auto failover settings.</p></li> 
            <li class="release-notes-list-item"><p>After upgrading, a mismatch of internal and external timeout values created service unavailability.</p></li> 
            <li class="release-notes-list-item"><p>Expected replication delays in MSSQL generated warnings.</p></li> 
            <li class="release-notes-list-item"><p>Link to "<a href="/en/enterprise-server@3.1/admin/enterprise-management/configuring-clustering">Configuring clustering</a>" on the Management Console was incorrect.</p></li> 
            <li class="release-notes-list-item"><p>When creating or editing a pre-receive hook, a race condition in the user interface meant that after selecting a repository, files within the repository were sometimes not populated in files dropdown.</p></li> 
            <li class="release-notes-list-item"><p>When an IP address is added to a whitelist using "Create Whitelist Entry" button, it could still be shown as locked out.</p></li> 
            <li class="release-notes-list-item"><p>References to the "Dependency graph" and "Dependabot alerts" features were not shown as disabled on some repositories.</p></li> 
            <li class="release-notes-list-item"><p>Setting an announcement in the enterprise account settings could result in a 500 Internal Server Error.</p></li> 
            <li class="release-notes-list-item"><p>HTTP POST requests to the <code>/hooks</code> endpoint could fail with a 401 response due to an incorrectly configured <code>hookID</code>.</p></li> 
            <li class="release-notes-list-item"><p>The <code>build-server</code> process failed to clean up processes, leaving them in the <code>defunct</code> state.</p></li> 
            <li class="release-notes-list-item"><p><code>spokesd</code> created excessive log entries, including the phrase "fixing placement skipped".</p></li> 
            <li class="release-notes-list-item"><p>While upgrading Actions the upgrade could fail if the instance could not make self-requests via its configured hostname.</p></li> 
            <li class="release-notes-list-item"><p>Upgrading from 2.22.x to 3.1.0.rc1 could result in a database migration error relating to the <code>BackfillIntegrationApplicationCallbackUrlsTransition</code> data transition.</p></li> 
           </ul> </li> 
         </ul>

        <div class="release-notes-section-known_issues py-6 d-block d-xl-flex gutter-xl flex-items-baseline border-bottom"> 
         <div class="col-12 col-xl-3 mb-5"> <span class="px-3 py-2 text-small text-bold text-uppercase text-mono color-text-inverse release-notes-section-label">Known issues</span> 
         </div> 
         <ul class="col-12 col-xl-9 release-notes-list"> 
          <li class="release-notes-list-item "> <p>Access to a repository through the administrative shell using <code>ghe-repo &lt;owner&gt;/&lt;reponame&gt;</code> will hang. As a workaround, use <code>ghe-repo &lt;owner&gt;/&lt;reponame&gt; -c "bash -i"</code> until a fix is available in the next version.</p> </li> 
          <li class="release-notes-list-item "> <p>The GitHub Packages npm registry no longer returns a time value in metadata responses. This was done to allow for substantial performance improvements. We continue to have all the data necessary to return a time value as part of the metadata response and will resume returning this value in the future once we have solved the existing performance issues.</p> </li> 
          <li class="release-notes-list-item "> <p>On a freshly set up GitHub Enterprise Server without any users, an attacker could create the first admin user.</p> </li> 
          <li class="release-notes-list-item "> <p>Custom firewall rules are not maintained during an upgrade.</p> </li> 
          <li class="release-notes-list-item "> <p>Git LFS tracked files <a href="https://github.com/blog/2105-upload-files-to-your-repositories">uploaded through the web interface</a> are incorrectly added directly to the repository.</p> </li> 
          <li class="release-notes-list-item "> <p>Issues cannot be closed if they contain a permalink to a blob in the same repository where the file path is longer than 255 characters.</p> </li> 
          <li class="release-notes-list-item "> <p>When "Users can search GitHub.com" is enabled with GitHub Connect, issues in private and internal repositories are not included in GitHub.com search results.</p> </li> 
          <li class="release-notes-list-item "> <p>Upgrading an instance that has previously ran a 2.13 release, but not a 2.14 release, results in a database migration error relating to the <code>AddRepositoryIdToCheckRuns</code> data transition.</p> </li> 
         </ul>

        <div class="release-notes-section-deprecations py-6 d-block d-xl-flex gutter-xl flex-items-baseline border-bottom"> 
         <div class="col-12 col-xl-3 mb-5"> <span class="px-3 py-2 text-small text-bold text-uppercase text-mono color-text-inverse release-notes-section-label">Deprecations</span> 
         </div> 
         <ul class="col-12 col-xl-9 release-notes-list"> 
          <li class="release-notes-list-item list-style-none"> <h4 id="deprecation-of-github-enterprise-server-220" class="release-notes-section-heading text-uppercase text-bold"> <a href="#deprecation-of-github-enterprise-server-220" class="text-inherit">Deprecation of GitHub Enterprise Server 2.20</a> </h4> 
           <ul class="pl-0 pb-4 mt-2 release-notes-list"> 
            <li class="release-notes-list-item"><p><strong>GitHub Enterprise Server 2.20 was discontinued on March 2, 2021</strong>. That means that no patch releases will be made, even for critical security issues, after this date. For better performance, improved security, and new features, <a href="/en/enterprise-server@3.1/admin/enterprise-management/upgrading-github-enterprise-server">upgrade to the newest version of GitHub Enterprise Server</a> as soon as possible.</p></li> 
           </ul> </li> 
          <li class="release-notes-list-item list-style-none"> <h4 id="deprecation-of-github-enterprise-server-221" class="release-notes-section-heading text-uppercase text-bold"> <a href="#deprecation-of-github-enterprise-server-221" class="text-inherit">Deprecation of GitHub Enterprise Server 2.21</a> </h4> 
           <ul class="pl-0 pb-4 mt-2 release-notes-list"> 
            <li class="release-notes-list-item"><p><strong>GitHub Enterprise Server 2.21 will be discontinued on June 9, 2021</strong>. That means that no patch releases will be made, even for critical security issues, after this date. For better performance, improved security, and new features, <a href="/en/enterprise-server@3.1/admin/enterprise-management/upgrading-github-enterprise-server">upgrade to the newest version of GitHub Enterprise Server</a> as soon as possible.</p></li> 
           </ul> </li> 
          <li class="release-notes-list-item list-style-none"> <h4 id="deprecation-of-legacy-github-app-webhook-events" class="release-notes-section-heading text-uppercase text-bold"> <a href="#deprecation-of-legacy-github-app-webhook-events" class="text-inherit">Deprecation of Legacy GitHub App Webhook Events</a> </h4> 
           <ul class="pl-0 pb-4 mt-2 release-notes-list"> 
            <li class="release-notes-list-item"><p>Starting with GitHub Enterprise Server 2.21.0 two legacy GitHub Apps-related webhook events have been deprecated and will be removed in GitHub Enterprise Server 3.2.0. The deprecated events <code>integration_installation</code> and <code>integration_installation_repositories</code> have equivalent events which will be supported. More information is available in the <a href="https://developer.github.com/changes/2020-04-15-replacing-the-installation-and-installation-repositories-events/">deprecation announcement blog post</a>.</p></li> 
           </ul> </li> 
          <li class="release-notes-list-item list-style-none"> <h4 id="deprecation-of-legacy-github-apps-endpoint" class="release-notes-section-heading text-uppercase text-bold"> <a href="#deprecation-of-legacy-github-apps-endpoint" class="text-inherit">Deprecation of Legacy GitHub Apps Endpoint</a> </h4> 
           <ul class="pl-0 pb-4 mt-2 release-notes-list"> 
            <li class="release-notes-list-item"><p>Starting with GitHub Enterprise Server 2.21.0 the legacy GitHub Apps endpoint for creating installation access tokens was deprecated and will be removed in GitHub Enterprise Server 3.2.0. More information is available in the <a href="https://developer.github.com/changes/2020-04-15-replacing-create-installation-access-token-endpoint/">deprecation announcement blog post</a>.</p></li> 
           </ul> </li> 
          <li class="release-notes-list-item list-style-none"> <h4 id="deprecation-of-oauth-application-api" class="release-notes-section-heading text-uppercase text-bold"> <a href="#deprecation-of-oauth-application-api" class="text-inherit">Deprecation of OAuth Application API</a> </h4> 
           <ul class="pl-0 pb-4 mt-2 release-notes-list"> 
            <li class="release-notes-list-item"><p>GitHub no longer supports the OAuth application endpoints that contain <code>access_token</code> as a path parameter. We have introduced new endpoints that allow you to securely manage tokens for OAuth Apps by moving <code>access_token</code> to the request body. While deprecated, the endpoints are still accessible in this version. We intend to remove these endpoints on GitHub Enterprise Server 3.4. For more information, see the <a href="https://developer.github.com/changes/2020-02-14-deprecating-oauth-app-endpoint/">deprecation announcement blog post</a>.</p></li> 
           </ul> </li> 
          <li class="release-notes-list-item list-style-none"> <h4 id="deprecation-of-github-actions-short-sha-support" class="release-notes-section-heading text-uppercase text-bold"> <a href="#deprecation-of-github-actions-short-sha-support" class="text-inherit">Deprecation of GitHub Actions short SHA support</a> </h4> 
           <ul class="pl-0 pb-4 mt-2 release-notes-list"> 
            <li class="release-notes-list-item"><p>GitHub Actions will remove support for referencing actions using the shortened version of a git commit SHA. This may cause some workflows in your repository to break. To fix these workflows, you will need to update the action reference to use the full commit SHA. For more information, see "<a href="/en/enterprise-server@3.1/actions/learn-github-actions/security-hardening-for-github-actions#using-third-party-actions">Security hardening for GitHub Actions</a>."</p></li> 
           </ul> </li> 
          <li class="release-notes-list-item list-style-none"> <h4 id="deprecation-of-xenserver-hypervisor-support" class="release-notes-section-heading text-uppercase text-bold"> <a href="#deprecation-of-xenserver-hypervisor-support" class="text-inherit">Deprecation of XenServer Hypervisor support</a> </h4> 
           <ul class="pl-0 pb-4 mt-2 release-notes-list"> 
            <li class="release-notes-list-item"><p>Beginning in GitHub Enterprise Server 3.1, we will begin discontinuing support for Xen Hypervisor. The complete deprecation is scheduled for GitHub Enterprise Server 3.3, following the standard one year deprecation window.</p></li> 
           </ul> </li> 
         </ul> 
        </div>
