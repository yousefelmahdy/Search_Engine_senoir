       <h3 id="about-the-codeql-cli"><a href="#about-the-codeql-cli">About the CodeQL CLI</a></h3> 
       <p>You can use the CodeQL CLI to run code scanning on code that you're processing in a third-party continuous integration (CI) system. Code scanning is a feature that you use to analyze the code in a GitHub repository to find security vulnerabilities and coding errors. Any problems identified by the analysis are shown in GitHub. For information, see "<a href="/en/code-security/secure-coding/about-code-scanning">About code scanning</a>."</p> 
       <p>The CodeQL CLI is a standalone product that you can use to analyze code. Its main purpose is to generate a database representation of a codebase, a CodeQL database. Once the database is ready, you can query it interactively, or run a suite of queries to generate a set of results in SARIF format and upload the results to GitHub.</p> 
       <p>Alternatively, you can use CodeQL runner in your CI system, or GitHub Actions to run code scanning within GitHub. For an overview of the options for CI systems, see "<a href="/en/code-security/secure-coding/about-codeql-code-scanning-in-your-ci-system">About CodeQL code scanning in your CI system</a>". For information about code scanning using actions, see "<a href="/en/code-security/secure-coding/setting-up-code-scanning-for-a-repository">Setting up code scanning for a repository</a>."</p> 
       <div class="extended-markdown note border rounded-1 mb-4 p-3 color-border-info color-bg-info f5"> 
        <p><strong>Note:</strong> The CodeQL CLI is free to use on public repositories that are maintained on GitHub.com, and available to use on private repositories that are owned by customers with an Advanced Security license. For information, see "<a href="https://securitylab.github.com/tools/codeql/license">GitHub CodeQL Terms and Conditions</a>" and "<a href="https://codeql.github.com/docs/codeql-cli/">CodeQL CLI</a>."</p> 
       </div> 
       <h3 id="downloading-the-codeql-cli"><a href="#downloading-the-codeql-cli">Downloading the CodeQL CLI</a></h3> 
       <p>You should download the CodeQL bundle from <a href="https://github.com/github/codeql-action/releases">https://github.com/github/codeql-action/releases</a>. The bundle contains:</p> 
       <ul> 
        <li>CodeQL CLI product</li> 
        <li>A compatible version of the queries and libraries from <a href="https://github.com/github/codeql">https://github.com/github/codeql</a></li> 
        <li>Precompiled versions of all the queries included in the bundle</li> 
       </ul> 
       <p>You should always use the CodeQL bundle as this ensures compatibility and also gives much better performance than a separate download of the CodeQL CLI and checkout of the CodeQL queries. If you will only be running the CLI on one specific platform, download the appropriate <code>codeql-bundle-PLATFORM.tar.gz</code> file. Alternatively, you can download <code>codeql-bundle.tar.gz</code>, which contains the CLI for all supported platforms.</p> 
       <h3 id="setting-up-the-codeql-cli-in-your-ci-system"><a href="#setting-up-the-codeql-cli-in-your-ci-system">Setting up the CodeQL CLI in your CI system</a></h3> 
       <p>You need to make the full contents of the CodeQL CLI bundle available to every CI server that you want to run CodeQL code scanning analysis on. For example, you might configure each server to copy the bundle from a central, internal location and extract it. Alternatively, you could use the REST API to get the bundle directly from GitHub, ensuring that you benefit from the latest improvements to queries. Updates to the CodeQL CLI are released every 2-3 weeks. For example:</p> 
       <pre><code class="hljs language-shell">$ wget https://github.com/github/codeql-action/releases/latest/download/codeql-bundle-linux64.tar.gz
$ tar -xvzf ../codeql-bundle-linux64.tar.gz</code></pre> 
       <p>After you extract the CodeQL CLI bundle, you can run the <code>codeql</code> executable on the server:</p> 
       <ul> 
        <li>By executing <code>/extraction-root/codeql/codeql</code>, where <code>&lt;extraction-root&gt;</code> is the folder where you extracted the CodeQL CLI bundle.</li> 
        <li>By adding <code>/extraction-root/codeql</code> to your <code>PATH</code>, so that you can run the executable as just <code>codeql</code>.</li> 
       </ul> 
       <h3 id="testing-the-codeql-cli-set-up"><a href="#testing-the-codeql-cli-set-up">Testing the CodeQL CLI set up</a></h3> 
       <p>After you extract the CodeQL CLI bundle, you can run the following command to verify that the CLI is correctly set up to create and analyze databases.</p> 
       <ul> 
        <li><code>codeql resolve languages</code> if <code>/extraction-root/codeql</code> is on the <code>PATH</code>.</li> 
        <li><code>/extraction-root/codeql/codeql resolve languages</code> otherwise.</li> 
       </ul> 
       <p><strong>Example of successful output:</strong></p> 
       <pre><code>cpp (/extraction-root/codeql/cpp)
csharp (/extraction-root/codeql/csharp)
csv (/extraction-root/codeql/csv)
go (/extraction-root/codeql/go)
html (/extraction-root/codeql/html)
java (/extraction-root/codeql/java)
javascript (/extraction-root/codeql/javascript)
properties (/extraction-root/codeql/properties)
python (/extraction-root/codeql/python)
xml (/extraction-root/codeql/xml)
</code></pre> 
       <p>If the CodeQL CLI is unable to resolve the expected languages, check that you downloaded the CodeQL bundle and not a standalone copy of the CodeQL CLI.</p> 
       <h3 id="generating-a-token-for-authentication-with-github"><a href="#generating-a-token-for-authentication-with-github">Generating a token for authentication with GitHub</a></h3> 
       <p>Each CI server needs a GitHub App or personal access token for the CodeQL CLI to use to upload results to GitHub. You must use an access token or a GitHub App with the <code>security_events</code> write permission. If CI servers already use a token with this scope to checkout repositories from GitHub, you could potentially allow the CodeQL CLI to use the same token. Otherwise, you should create a new token with the <code>security_events</code> write permission and add this to the CI system's secret store. For information, see "<a href="/en/developers/apps/building-github-apps">Building GitHub Apps</a>" and "<a href="/en/github/authenticating-to-github/creating-a-personal-access-token">Creating a personal access token</a>."</p> 
       <h3 id="using-the-codeql-cli-to-generate-data-and-upload-it-to-github"><a href="#using-the-codeql-cli-to-generate-data-and-upload-it-to-github">Using the CodeQL CLI to generate data and upload it to GitHub</a></h3> 
       <p>You call the CodeQL CLI to analyze the codebase in three steps:</p> 
       <ol> 
        <li>Create a CodeQL database to represent a single programming language in the repository using: <code>codeql database create</code></li> 
        <li>Run queries to analyze the CodeQL database and summarize the results in a SARIF file using: <code>codeql database analyze</code></li> 
        <li>Upload the SARIF file to GitHub where the results are matched to a branch or pull request and displayed as code scanning alerts using: <code>codeql github upload-results</code></li> 
       </ol> 
       <p>Each command has a few mandatory options with additional options that you can use to modify the behavior of the command. You can display the command-line help for any command using the <nobr>
         <code>--help</code>
        </nobr> option.</p> 
       <div class="extended-markdown note border rounded-1 mb-4 p-3 color-border-info color-bg-info f5"> 
        <p><strong>Note:</strong> Uploading SARIF data to display as code scanning results in GitHub is supported for organization-owned repositories with GitHub Advanced Security enabled, and public repositories on GitHub.com. For more information, see "<a href="/en/github/administering-a-repository/managing-security-and-analysis-settings-for-your-repository">Managing security and analysis settings for your repository</a>."</p> 
       </div> 
       <h4 id="creating-a-codeql-database-to-analyze"><a href="#creating-a-codeql-database-to-analyze">Creating a CodeQL database to analyze</a></h4> 
       <ol> 
        <li> <p>Check out the code that you want to analyze:</p> 
         <ul> 
          <li>For a branch checkout the head of the branch that you want to analyze.</li> 
          <li>For a pull request checkout either the head commit of the pull request, or check out a GitHub-generated merge commit of the pull request.</li> 
         </ul> </li> 
        <li> <p>Set up the environment for the codebase, making sure that any dependencies are available. For more information, see <a href="https://codeql.github.com/docs/codeql-cli/creating-codeql-databases/#creating-databases-for-non-compiled-languages">Creating databases for non-compiled languages</a> and <a href="https://codeql.github.com/docs/codeql-cli/creating-codeql-databases/#creating-databases-for-compiled-languages">Creating databases for compiled languages</a> in the documentation for the CodeQL CLI.</p> </li> 
        <li> <p>Run <code>codeql database create</code> from the checkout root of your repository.</p> <pre><code class="hljs language-shell">codeql database create &lt;database&gt; --language=&lt;language-identifier&gt;</code></pre>

       <h4 id="uploading-results-to-github"><a href="#uploading-results-to-github">Uploading results to GitHub</a></h4> 
       <div class="extended-markdown note border rounded-1 mb-4 p-3 color-border-info color-bg-info f5"> 
        <p><strong>Note:</strong> SARIF upload supports a maximum of 5000 results per upload. Any results over this limit are ignored. If a tool generates too many results, you should update the configuration to focus on results for the most important rules or queries.</p> 
       </div> 
       <p>Before you can upload results to GitHub, you must determine the best way to pass the GitHub App or personal access token you created earlier to the CodeQL CLI (see <a href="#generating-a-token-for-authentication-with-github">Generating a token for authentication with GitHub</a> above). We recommend that you review your CI system's guidance on the secure use of the secret store. The CodeQL CLI supports:</p> 
       <ul> 
        <li>Passing the token to the CLI via standard input using the <code>--github-auth-stdin</code> option (recommended).</li> 
        <li>Saving the secret in the environment variable <code>GITHUB_TOKEN</code> and running the CLI without including the <code>--github-auth-stdin</code> option.</li> 
       </ul> 
       <p>When you have decided on the most secure and reliable method for your CI server, run <code>codeql github upload-results</code> on the SARIF results file and include <code>--github-auth-stdin</code> unless the token is available in the environment variable <code>GITHUB_TOKEN</code>.</p>

      --github-auth-stdin</code></pre>
