   <section id="faq" class="section" aria-label="Ofte stillede spørgsmål" data-module="faq-01"> 
    <div class="row"> 
     <div class="column medium-3"> 
      <h2>Ofte stillede spørgsmål</h2> 
     </div> 
     <div class="column medium-8 medium-offset-1"> 
      <ul class="faq"> 
       <li class="faq__item"> <button type="button" class="faq__question" aria-expanded="false" aria-controls="07bab68a" data-bi-slot="1" data-bi-id="faq-1" data-bi-an="faq-01"> Hvad er Azure Cosmos DB? </button> 
        <div class="faq__answer" id="07bab68a">
         <p>Azure Cosmos DB er en globalt distribueret databasetjeneste med flere modeller til en hvilken som helst skala. Tjenesten er udviklet fra bunden med global distribution og horisontal skalering som kernen.</p>
         <p>Med nøglefærdig global distribution på tværs af et hvilket som helst antal Azure-områder skalerer og replikerer Azure Cosmos DB dine data på gennemsigtig vis, uanset hvor brugerne er. Skaler dine skrivninger og læsninger elastisk i hele verden, og betal kun for det, du har brug for. Azure Cosmos DB sikrer oprindelig understøttelse til API'er til NoSQL og OSS, herunder MongoDB, Cassandra, Gremlin, etcd, Spark og SQL. Tjenesten tilbyder også flere veldefinerede modeller til ensartethed, garanterede læse- og skriveventetider på encifret millisekunder ved den 99. percentil og garanteret høj tilgængelighed på 99,999&nbsp;% med understøttelse af flere websteder overalt i verden – alt sammen understøttet af brancheførende, omfattende SLA'er.</p>
         <p>Azure Cosmos DB er en fuldt administreret tjeneste, der giver dig mulighed for at aflaste den administrative byrde ved at drive og skalere distribuerede databaser til Azure, så du ikke skal bekymre dig om at administrere VM'er, klargøre hardware, konfiguration, kapacitet, replikering, reparation af software eller skalering af klynger.</p>
        </div> </li> 
       <li class="faq__item"> <button type="button" class="faq__question" aria-expanded="false" aria-controls="dbd66c33" data-bi-slot="2" data-bi-id="faq-2" data-bi-an="faq-01"> Hvad er nøglefærdig global distribution? </button> 
        <div class="faq__answer" id="dbd66c33">
         <p>Med Azure Cosmos DB kan du konfigurere dine databaser, så de kan distribueres globalt og være tilgængelige i et hvilket som helst Azure-område. Du kan minimere ventetiden ved at placere dataene tæt på dine brugere.</p>
         <p>Azure Cosmos DB replikerer på gennemsigtig vis dataene til alle områder, der er knyttet til din Azure Cosmos DB-konto. Tjenesten leverer en enkelt systemafbildning af din globalt distribuerede Azure Cosmos DB-database og objektbeholdere, som dit program kan læse og skrive til lokalt.</p>
         <p>Med nøglefærdig global distribution kan du tilføje eller fjerne områder, der er knyttet til din konto når som helst. Du behøver ikke at afbryde programmet midlertidigt eller genudrulle det for at tilføje eller fjerne et område. Høj tilgængelighed bevares via egenskaber for flere websteder, som leveres oprindeligt af tjenesten. <a href="https://docs.microsoft.com/da-dk/azure/cosmos-db/distribute-data-globally">Få mere at vide</a>.</p>
         <p></p>
        </div> </li> 
       <li class="faq__item"> <button type="button" class="faq__question" aria-expanded="false" aria-controls="50b91778" data-bi-slot="3" data-bi-id="faq-3" data-bi-an="faq-01"> Hvordan administrerer Azure Cosmos DB ensartethed? </button> 
        <div class="faq__answer" id="50b91778">
         <p>Azure Cosmos DB ser ensartede data som et spektrum af valg med flere muligheder end de to ekstremer for høj ensartethed og eventuel ensartethed.</p>
         <p>Vælg mellem fem veldefinerede modeller på spektret for ensartethed:</p> 
         <ul aria-label="Fem veldefinerede modeller">
          <li>Høj</li>
          <li>Afgrænset forældelse</li>
          <li>Session</li>
          <li>Ensartet præfiks</li>
          <li>Eventuel</li>
         </ul>
         <p>Hver model understøttes af omfattende SLA'er. <a href="https://docs.microsoft.com/da-dk/azure/cosmos-db/consistency-levels-tradeoffs">Få mere at vide.</a></p>
        </div> </li> 
       <li class="faq__item"> <button type="button" class="faq__question" aria-expanded="false" aria-controls="bd444223" data-bi-slot="4" data-bi-id="faq-4" data-bi-an="faq-01"> Hvilke SLA'er findes der til Azure Cosmos DB? </button> 
        <div class="faq__answer" id="bd444223">
         <p>Som en globalt distribueret database understøttes Azure Cosmos DB af omfattende SLA'er, der omfatter gennemløb, ventetid ved den 99. percentil, ensartethed og høj tilgængelighed. <a href="/da-dk/support/legal/sla/cosmos-db/">Hent flere oplysninger</a>.</p>
        </div> </li> 
       <li class="faq__item"> <button type="button" class="faq__question" aria-expanded="false" aria-controls="b2a625c0" data-bi-slot="5" data-bi-id="faq-5" data-bi-an="faq-01"> Hvordan understøtter Azure Cosmos DB tilgængelighed, skalerbarhed og holdbarhed? </button> 
        <div class="faq__answer" id="b2a625c0">
         Azure Cosmos DB replikerer på gennemsigtig vis dine data på tværs af alle Azure-områder, der er knyttet til din Azure Cosmos DB-konto. Tjenesten anvender flere redundanslag på dine data. <a href="https://docs.microsoft.com/da-dk/azure/cosmos-db/high-availability">Få flere detaljer</a>.
        </div> </li> 
       <li class="faq__item"> <button type="button" class="faq__question" aria-expanded="false" aria-controls="59b33434" data-bi-slot="6" data-bi-id="faq-6" data-bi-an="faq-01"> Hvad er Azure Cosmos DB API til MongoDB? </button> 
        <div class="faq__answer" id="59b33434">
         Azure Cosmos DB API til MongoDB giver dig mulighed for at interagere med Azure Cosmos DB, som om det var en MongoDB-database, uden at skulle administrere selve infrastrukturen af databasen. Det betyder, at næsten alle dine foretrukne værktøjer, SDK'er og programmer til MongoDB fortsat vil fungere. Du kan finde detaljerede oplysninger om kompatibilitet <a href="https://aka.ms/cosmosdb-mongo-latest-compat">i vores dokumentation</a>.
        </div> </li> 
       <li class="faq__button-container faq__item"> <button type="button" class="faq__question faq__load-more-button">Indlæs flere ofte stillede spørgsmål</button> </li> 
       <li class="faq__item"> <button type="button" class="faq__question" aria-expanded="false" aria-controls="285b0739" data-bi-slot="7" data-bi-id="faq-7" data-bi-an="faq-01"> Hvad er understøttelse af v4.0 til Azure Cosmos DB API til MongoDB? </button> 
        <div class="faq__answer" id="285b0739">
         Med understøttelse af serverversion 4.0 i Azure Cosmos DB API til MongoDB er det nemmere at udforme kompleks transaktionsforretningslogik uden den komplekse logik til håndtering af fejl. Med understøttelsen af v4.0 introduceres nye funktioner såsom transaktioner i flere dokumenter, skrivninger, der kan forsøges igen m.m.
        </div> </li> 
       <li class="faq__item"> <button type="button" class="faq__question" aria-expanded="false" aria-controls="8f79bf90" data-bi-slot="8" data-bi-id="faq-8" data-bi-an="faq-01"> Hvad er Azure Cosmos DB Cassandra-API? </button> 
        <div class="faq__answer" id="8f79bf90">
         Med Azure Cosmos DB Cassandra-API kan du interagere med lagrede data i Azure Cosmos DB ved hjælp af Cassandra Query Language (CQL), Cassandra-baserede værktøjer (f.eks. cqlsh) og eksisterende Apache-drivere, der er kompatible med CQLv4. Det betyder, at dit eksisterende Cassandra-program nu kan kommunikere med Azure Cosmos DB Cassandra-API. Snart kan API'en oprette forbindelse til Azure Managed Instance til Apache Cassandra (når den bliver generelt tilgængelig).
        </div> </li> 
      </ul> 
     </div>
