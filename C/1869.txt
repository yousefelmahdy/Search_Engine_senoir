       <h3 id="about-github-connect"><a href="#about-github-connect">About GitHub Connect</a></h3> 
       <p>To enable GitHub Connect, you must configure the connection in both your GitHub Enterprise Server instance and in your GitHub Enterprise Cloud organization or enterprise account.</p> 
       <p>To configure a connection, your proxy configuration must allow connectivity to <code>github.com</code> and <code>api.github.com</code>. For more information, see "<a href="/en/enterprise-server@3.1/admin/guides/installation/configuring-an-outbound-web-proxy-server">Configuring an outbound web proxy server</a>."</p> 
       <p>After enabling GitHub Connect, you will be able to enable features such as unified search and unified contributions. For more information about all of the features available, see "<a href="/en/enterprise-server@3.1/admin/installation/managing-connections-between-github-enterprise-server-and-github-enterprise-cloud">Managing connections between GitHub Enterprise Server and GitHub Enterprise Cloud</a>."</p> 
       <p>When you connect your GitHub Enterprise Server instance to GitHub Enterprise Cloud, a record on GitHub.com stores information about the connection:</p> 
       <ul> 
        <li>The public key portion of your GitHub Enterprise Server license</li> 
        <li>A hash of your GitHub Enterprise Server license</li> 
        <li>The customer name on your GitHub Enterprise Server license</li> 
        <li>The hostname of your GitHub Enterprise Server instance</li> 
        <li>The version of your GitHub Enterprise Server instance</li> 
        <li>The organization or enterprise account on GitHub.com that's connected to your GitHub Enterprise Server instance</li> 
        <li>The authentication token that's used by your GitHub Enterprise Server instance to make requests to GitHub.com</li> 
       </ul> 
       <p>Enabling GitHub Connect also creates a GitHub App owned by your GitHub Enterprise Cloud organization or enterprise account. GitHub Enterprise Server uses the GitHub App's credentials to make requests to GitHub.com.</p> 
       <p>GitHub Enterprise Server stores credentials from the GitHub App. These credentials will be replicated to any high availability or clustering environments, and stored in any backups, including snapshots created by GitHub Enterprise Server Backup Utilities.</p> 
       <ul> 
        <li>An authentication token, which is valid for one hour</li> 
        <li>A private key, which is used to generate a new authentication token</li> 
       </ul> 
       <p>Enabling GitHub Connect will not allow GitHub.com users to make changes to GitHub Enterprise Server.</p> 
       <p>For more information about managing enterprise accounts using the GraphQL API, see "<a href="/en/enterprise-server@3.1/graphql/guides/managing-enterprise-accounts">Enterprise accounts</a>."</p>
