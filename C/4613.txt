   <div class="solutions-block solution-1"> 
    <h2 class="title"> Seeing is believing </h2> 
    <div class="subtitle">
      Continuous integration (CI) delivers better software faster. 
    </div> 
    <div class="description">
      Build, test, and deploy on day one with GitLab’s <a href="https://about.gitlab.com/stages-devops-lifecycle/continuous-delivery/">built-in CI/CD</a> and Auto DevOps. From scalable pipelines to integrated security—see it all in one tab, believe in a better workflow. 
    </div> <a class="btn cta-btn orange-light" href="/resources/ebook-single-app-cicd/">Get to know CI/CD</a>

   <div class="solutions-block solution-2"> 
    <h2 class="title"> People-powered automation </h2> 
    <div class="description"> <a href="https://about.gitlab.com/topics/ci-cd/">Continious integration (CI)</a> starts with you. You share new code in a merge request (MR), you trigger a pipeline. Build, test, and validate—GitLab does the rest. 
    </div> 
    <div class="description">
      Continuous delivery (CD) puts your hard work into action, handing off CI-validated code to your application via structured deployment pipelines. 
    </div> 
   </div> 
  </div>
