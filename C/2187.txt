        <div class="d-none d-lg-block ml-2"> <button class="btn-link Link--secondary js-print tooltipped tooltipped-n" aria-label="Print this article"> <!-- From https://heroicons.dev/ -->

       <h3 id="options-for-setting-up-code-scanning"><a href="#options-for-setting-up-code-scanning">Options for setting up code scanning</a></h3> 
       <p>You decide how to generate code scanning alerts, and which tools to use, at a repository level. GitHub provides fully integrated support for CodeQL analysis, and also supports analysis using third-party tools. For more information, see "<a href="/en/code-security/secure-coding/about-code-scanning#about-codeql">About CodeQL</a>."</p>

       <h3 id="setting-up-code-scanning-using-actions"><a href="#setting-up-code-scanning-using-actions">Setting up code scanning using actions</a></h3> 
       <p>Using actions to run code scanning will use minutes. For more information, see "<a href="/en/billing/managing-billing-for-github-actions/about-billing-for-github-actions">About billing for GitHub Actions</a>."</p> 
       <ol> 
        <li> <p>On GitHub, navigate to the main page of the repository.</p> </li> 
        <li> <p>Under your repository name, click <strong>
           <svg version="1.1" width="16" height="16" viewbox="0 0 16 16" class="octicon octicon-shield" aria-label="The shield symbol" role="img">
            <path fill-rule="evenodd" d="M7.467.133a1.75 1.75 0 011.066 0l5.25 1.68A1.75 1.75 0 0115 3.48V7c0 1.566-.32 3.182-1.303 4.682-.983 1.498-2.585 2.813-5.032 3.855a1.7 1.7 0 01-1.33 0c-2.447-1.042-4.049-2.357-5.032-3.855C1.32 10.182 1 8.566 1 7V3.48a1.75 1.75 0 011.217-1.667l5.25-1.68zm.61 1.429a.25.25 0 00-.153 0l-5.25 1.68a.25.25 0 00-.174.238V7c0 1.358.275 2.666 1.057 3.86.784 1.194 2.121 2.34 4.366 3.297a.2.2 0 00.154 0c2.245-.956 3.582-2.104 4.366-3.298C13.225 9.666 13.5 8.36 13.5 7V3.48a.25.25 0 00-.174-.237l-5.25-1.68zM9 10.5a1 1 0 11-2 0 1 1 0 012 0zm-.25-5.75a.75.75 0 10-1.5 0v3a.75.75 0 001.5 0v-3z" />
           </svg> Security</strong>. <span class="procedural-image-wrapper"><img src="/assets/images/help/repository/security-tab.png" alt="Security tab"></span></p> </li> 
        <li> <p>To the right of "Code scanning alerts", click <strong>Set up code scanning</strong>. If code scanning is missing, you need to ask an organization owner or repository administrator to enable GitHub Advanced Security. For more information, see "<a href="/en/organizations/keeping-your-organization-secure/managing-security-and-analysis-settings-for-your-organization">Managing security and analysis settings for your organization</a>" or "<a href="/en/github/administering-a-repository/managing-security-and-analysis-settings-for-your-repository">Managing security and analysis settings for your repository</a>." <span class="procedural-image-wrapper"><img src="/assets/images/help/security/overview-set-up-code-scanning.png" alt="&quot;Set up code scanning&quot; button to the right of &quot;Code scanning&quot; in the Security Overview"></span></p> </li> 
        <li> <p>Under "Get started with code scanning", click <strong>Set up this workflow</strong> on the CodeQL analysis workflow or on a third-party workflow. <span class="procedural-image-wrapper"><img src="/assets/images/help/repository/code-scanning-set-up-this-workflow.png" alt="&quot;Set up this workflow&quot; button under &quot;Get started with code scanning&quot; heading"></span>Workflows are only displayed if they are relevant for the programming languages detected in the repository. The CodeQL analysis workflow is always displayed, but the "Set up this workflow" button is only enabled if CodeQL analysis supports the languages present in the repository.</p> </li> 
        <li> <p>To customize how code scanning scans your code, edit the workflow.</p> <p>Generally you can commit the CodeQL analysis workflow without making any changes to it. However, many of the third-party workflows require additional configuration, so read the comments in the workflow before committing.</p> <p>For more information, see "<a href="/en/code-security/secure-coding/configuring-code-scanning">Configuring code scanning</a>."</p> </li> 
        <li> <p>Use the <strong>Start commit</strong> drop-down, and type a commit message. <span class="procedural-image-wrapper"><img src="/assets/images/help/repository/start-commit-commit-new-file.png" alt="Start commit"></span></p> </li> 
        <li> <p>Choose whether you'd like to commit directly to the default branch, or create a new branch and start a pull request. <span class="procedural-image-wrapper"><img src="/assets/images/help/repository/start-commit-choose-where-to-commit.png" alt="Choose where to commit"></span></p> </li> 
        <li> <p>Click <strong>Commit new file</strong> or <strong>Propose new file</strong>.</p> </li> 
       </ol> 
       <p>In the default CodeQL analysis workflow, code scanning is configured to analyze your code each time you either push a change to the default branch or any protected branches, or raise a pull request against the default branch. As a result, code scanning will now commence.</p> 
       <h3 id="bulk-set-up-of-code-scanning"><a href="#bulk-set-up-of-code-scanning">Bulk set up of code scanning</a></h3> 
       <p>You can set up code scanning in many repositories at once using a script. For an example of a script that raises pull requests to add a GitHub Actions workflow to multiple repositories, see the <a href="https://github.com/jhutchings1/Create-ActionsPRs"><code>jhutchings1/Create-ActionsPRs</code></a> repository.</p> 
       <h3 id="viewing-the-logging-output-from-code-scanning"><a href="#viewing-the-logging-output-from-code-scanning">Viewing the logging output from code scanning</a></h3> 
       <p>After setting up code scanning for your repository, you can watch the output of the actions as they run.</p> 
       <ol> 
        <li> <p>Under your repository name, click <strong>Actions</strong>. <span class="procedural-image-wrapper"><img src="/assets/images/help/repository/actions-tab.png" alt="Actions tab in the main repository navigation"></span></p> <p>You'll see a list that includes an entry for running the code scanning workflow. The text of the entry is the title you gave your commit message.</p> <p><span class="procedural-image-wrapper"><img src="/assets/images/help/repository/code-scanning-actions-list.png" alt="Actions list showing code scanning workflow"></span></p> </li> 
        <li> <p>Click the entry for the code scanning workflow.</p> </li> 
        <li> <p>Click the job name on the left. For example, <strong>Analyze (LANGUAGE)</strong>.</p> <p><span class="procedural-image-wrapper"><img src="/assets/images/help/repository/code-scanning-logging-analyze-action.png" alt="Log output from the code scanning workflow"></span></p> </li> 
        <li> <p>Review the logging output from the actions in this workflow as they run.</p> </li> 
        <li> <p>Once all jobs are complete, you can view the details of any code scanning alerts that were identified. For more information, see "<a href="/en/code-security/secure-coding/managing-code-scanning-alerts-for-your-repository#viewing-the-alerts-for-a-repository">Managing code scanning alerts for your repository</a>."</p> </li> 
       </ol> 
       <div class="extended-markdown note border rounded-1 mb-4 p-3 color-border-info color-bg-info f5"> 
        <p><strong>Note:</strong> If you raised a pull request to add the code scanning workflow to the repository, alerts from that pull request aren't displayed directly on the Code scanning page until the pull request is merged. If any alerts were found you can view these, before the pull request is merged, by clicking the <strong><em>n</em> alerts found</strong> link in the banner on the Code scanning page.</p> 
        <p> <img src="/assets/images/help/repository/code-scanning-alerts-found-link.png" alt="Click the &quot;n alerts found&quot; link"></p> 
       </div> 
       <h3 id="understanding-the-pull-request-checks"><a href="#understanding-the-pull-request-checks">Understanding the pull request checks</a></h3> 
       <p>Each code scanning workflow you set to run on pull requests always has at least two entries listed in the checks section of a pull request. There is one entry for each of the analysis jobs in the workflow, and a final one for the results of the analysis.</p> 
       <p>The names of the code scanning analysis checks take the form: "TOOL NAME / JOB NAME (TRIGGER)." For example, for CodeQL, analysis of C++ code has the entry "CodeQL / Analyze (cpp) (pull_request)." You can click <strong>Details</strong> on a code scanning analysis entry to see logging data. This allows you to debug a problem if the analysis job failed. For example, for code scanning analysis of compiled languages, this can happen if the action can't build the code.</p> 
       <p> <img src="/assets/images/help/repository/code-scanning-pr-checks.png" alt="code scanning pull request checks"></p> 
       <p>When the code scanning jobs complete, GitHub works out whether any alerts were added by the pull request and adds the "Code scanning results / TOOL NAME" entry to the list of checks. After code scanning has been performed at least once, you can click <strong>Details</strong> to view the results of the analysis. If you used a pull request to add code scanning to the repository, you will initially see a "Missing analysis" message when you click <strong>Details</strong> on the "Code scanning results / TOOL NAME" check.</p> 
       <p> <img src="/assets/images/help/repository/code-scanning-missing-analysis.png" alt="Missing analysis for commit message"></p> 
       <h4 id="reasons-for-the-missing-analysis-message"><a href="#reasons-for-the-missing-analysis-message">Reasons for the "missing analysis" message</a></h4> 
       <p>After code scanning has analyzed the code in a pull request, it needs to compare the analysis of the topic branch (the branch you used to create the pull request) with the analysis of the base branch (the branch into which you want to merge the pull request). This allows code scanning to compute which alerts are newly introduced by the pull request, which alerts were already present in the base branch, and whether any existing alerts are fixed by the changes in the pull request. Initially, if you use a pull request to add code scanning to a repository, the base branch has not yet been analyzed, so it's not possible to compute these details. In this case, when you click through from the results check on the pull request you will see the "Missing analysis for base commit SHA-HASH" message.</p> 
       <p>There are other situations where there may be no analysis for the latest commit to the base branch for a pull request. These include:</p> 
       <ul> 
        <li> <p>The pull request has been raised against a branch other than the default branch, and this branch hasn't been analyzed.</p> <p>To check whether a branch has been scanned, go to the Code scanning page, click the <strong>Branch</strong> drop-down and select the relevant branch.</p> <p><img src="/assets/images/help/repository/code-scanning-branch-dropdown.png" alt="Choose a branch from the Branch drop-down menu"></p> <p>The solution in this situation is to add the name of the base branch to the <code>on:push</code> and <code>on:pull_request</code> specification in the code scanning workflow on that branch and then make a change that updates the open pull request that you want to scan.</p> </li> 
        <li> <p>The latest commit on the base branch for the pull request is currently being analyzed and analysis is not yet available.</p> <p>Wait a few minutes and then push a change to the pull request to retrigger code scanning.</p> </li> 
        <li> <p>An error occurred while analyzing the latest commit on the base branch and analysis for that commit isn't available.</p> <p>Merge a trivial change into the base branch to trigger code scanning on this latest commit, then push a change to the pull request to retrigger code scanning.</p> </li> 
       </ul> 
       <h3 id="next-steps"><a href="#next-steps">Next steps</a></h3> 
       <p>After setting up code scanning, and allowing its actions to complete, you can:</p> 
       <ul> 
        <li>View all of the code scanning alerts generated for this repository. For more information, see "<a href="/en/code-security/secure-coding/managing-code-scanning-alerts-for-your-repository">Managing code scanning alerts for your repository</a>."</li> 
        <li>View any alerts generated for a pull request submitted after you set up code scanning. For more information, see "<a href="/en/code-security/secure-coding/triaging-code-scanning-alerts-in-pull-requests">Triaging code scanning alerts in pull requests</a>."</li> 
        <li>Set up notifications for completed runs. For more information, see "<a href="/en/github/managing-subscriptions-and-notifications-on-github/configuring-notifications#github-actions-notification-options">Configuring notifications</a>."</li> 
        <li>Investigate any problems that occur with the initial setup of CodeQL code scanning. For more information, see "<a href="/en/code-security/secure-coding/troubleshooting-the-codeql-workflow">Troubleshooting the CodeQL workflow</a>."</li> 
        <li>Customize how code scanning scans the code in your repository. For more information, see "<a href="/en/code-security/secure-coding/configuring-code-scanning">Configuring code scanning</a>."</li> 
       </ul>
