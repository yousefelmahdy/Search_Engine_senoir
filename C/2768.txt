       <div class="d-flex flex-items-baseline flex-justify-between mt-3"> 
        <h1 class="border-bottom-0">Opening an issue from code</h1> 
        <div class="d-none d-lg-block ml-2"> <button class="btn-link Link--secondary js-print tooltipped tooltipped-n" aria-label="Print this article"> <!-- From https://heroicons.dev/ --> 
          <svg fill="none" height="18" width="18" stroke-linecap="round" stroke-linejoin="round" stroke-width="2" viewbox="0 0 24 24" stroke="currentColor">
           <path d="M17 17h2a2 2 0 002-2v-4a2 2 0 00-2-2H5a2 2 0 00-2 2v4a2 2 0 002 2h2m2 4h6a2 2 0 002-2v-4a2 2 0 00-2-2H9a2 2 0 00-2 2v4a2 2 0 002 2zm8-12V5a2 2 0 00-2-2H9a2 2 0 00-2 2v4h10z"></path>
          </svg> </button> 
        </div> 
       </div> 
       <div class="lead-mktg">
        <p>You can open a new issue from a specific line or lines of code in a file or pull request.</p>
       </div> 
       <div class="permissions-statement">
        <p>People with read permissions can create an issue in a repository where issues are enabled.</p>
       </div> 
      </div> 
      <div class="article-grid-toc border-bottom border-xl-0 pb-4 mb-5 pb-xl-0 mb-xl-0"> 
       <div class="article-grid-toc-content"> 
       </div> 
      </div> 
      <div id="article-contents" class="article-grid-body"> 
       <p>When you open an issue from code, the issue contains a snippet showing the line or range of code you chose. You can only open an issue in the same repository where the code is stored.</p> 
       <p><img src="/assets/images/help/repository/issue-opened-from-code.png" alt="Code snippet rendered in an issue opened from code"></p> 
       <p>Repository administrators can disable issues for a repository. For more information, see "<a href="/en/github/managing-your-work-on-github/disabling-issues">Disabling issues</a>."</p> 
       <ol> 
        <li>On GitHub, navigate to the main page of the repository.</li> 
        <li>Locate the code you want to reference in an issue:
         <ul> 
          <li>To open an issue about code in a file, navigate to the file.</li> 
          <li>To open an issue about code in a pull request, navigate to the pull request and click 
           <svg version="1.1" width="16" height="16" viewbox="0 0 16 16" class="octicon octicon-diff" aria-label="The file diff icon" role="img">
            <path fill-rule="evenodd" d="M8.75 1.75a.75.75 0 00-1.5 0V5H4a.75.75 0 000 1.5h3.25v3.25a.75.75 0 001.5 0V6.5H12A.75.75 0 0012 5H8.75V1.75zM4 13a.75.75 0 000 1.5h8a.75.75 0 100-1.5H4z"></path>
           </svg> <strong>Files changed</strong>. Then, browse to the file that contains the code you want include in your comment, and click <strong>View</strong>.</li> 
         </ul></li> 
        <li>Choose whether to select a single line or a range:
         <ul> 
          <li>To select a single line of code, click the line number to highlight the line. <span class="procedural-image-wrapper"><img src="/assets/images/help/repository/highlight-line-of-code.png" alt="File with a single line of code selected"></span>
           <ul> 
            <li>To select a range of code, click the number of the first line in the range to highlight the line of code. Then, hover over the last line of the code range, press <kbd>Shift</kbd>, and click the line number to highlight the range. <span class="procedural-image-wrapper"><img src="/assets/images/help/repository/highlight-range-of-code.png" alt="File with a range of code selected"></span></li> 
           </ul></li> 
         </ul></li> 
        <li>To the left of the code range, click 
         <svg version="1.1" width="16" height="16" viewbox="0 0 16 16" class="octicon octicon-kebab-horizontal" aria-label="The horizontal kebab octicon" role="img">
          <path d="M8 9a1.5 1.5 0 100-3 1.5 1.5 0 000 3zM1.5 9a1.5 1.5 0 100-3 1.5 1.5 0 000 3zm13 0a1.5 1.5 0 100-3 1.5 1.5 0 000 3z"></path>
         </svg>. In the drop-down menu, click <strong>Reference in new issue</strong>. <span class="procedural-image-wrapper"><img src="/assets/images/help/repository/open-new-issue-specific-line.png" alt="Kebab menu with option to open a new issue from a selected line"></span></li> 
        <li>Type a title and description for your issue. <span class="procedural-image-wrapper"><img src="/assets/images/help/issues/sample_issue.png" alt="A sample issue"></span></li> 
        <li>If you're a project maintainer, you can <a href="/en/articles/assigning-issues-and-pull-requests-to-other-github-users">assign the issue to someone</a>, <a href="/en/articles/adding-issues-and-pull-requests-to-a-project-board/#adding-issues-and-pull-requests-to-a-project-board-from-the-sidebar">add it to a project board</a>, <a href="/en/articles/associating-milestones-with-issues-and-pull-requests">associate it with a milestone</a>, or <a href="/en/articles/applying-labels-to-issues-and-pull-requests">apply a label</a>.</li> 
        <li>When you're finished, click <strong>Submit new issue</strong>.</li> 
       </ol>
