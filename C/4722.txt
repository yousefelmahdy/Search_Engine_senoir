     <h1 id="omnibus-gitlab-documentation">Omnibus GitLab documentation<a href="#omnibus-gitlab-documentation" title="Permalink" class="anchor"></a> </h1> 
     <p>Omnibus is a way to package different services and tools required to run GitLab, so that most users can install it without laborious configuration. </p>

     <h2 id="installation">Installation<a href="#installation" title="Permalink" class="anchor"></a> </h2> 
     <h5 id="prerequisites">Prerequisites<a href="#prerequisites" title="Permalink" class="anchor"></a> </h5> 
     <ul> 
      <li><a href="/11.11/ce/install/requirements.html">Installation Requirements</a> </li>
      <li>If you want to access your GitLab instance via a domain name, like mygitlabinstance.com, make sure the domain correctly points to the IP of the server where GitLab is being installed. You can check this using the command <code class="highlighter-rouge">host mygitlabinstance.com</code> </li>
      <li>If you want to use HTTPS on your GitLab instance, make sure you have the SSL certificates for the domain ready. (Note that certain components like Container Registry which can have their own subdomains requires certificates for those subdomains also) </li>
      <li>If you want to send notification emails, install and configure a mail server (MTA) like sendmail. Alternatively, you can use other third party SMTP servers, which is described below. </li>
     </ul>

     <h4 id="using-docker-image">Using docker image<a href="#using-docker-image" title="Permalink" class="anchor"></a> </h4> 
     <p>You can also use the docker images provided by GitLab to install and configure a GitLab instance. Check the <a href="docker/README.html">documentation</a> to know more. </p>
     <h4 id="running-on-a-low-resource-device-like-a-raspberry-pi">Running on a low-resource device (like a Raspberry Pi)<a href="#running-on-a-low-resource-device-like-a-raspberry-pi" title="Permalink" class="anchor"></a> </h4> 
     <p>You can run GitLab on supported low-resource computers like the Raspberry Pi 3, but you will need to tune the settings to work best with the available resources. Check out the <a href="settings/rpi.html">documentation</a> for suggestions on what to adjust. </p>
