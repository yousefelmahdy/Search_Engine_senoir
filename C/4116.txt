       <p>This template generates a box that marks a thread on a talk page as resolved. It enables automatic archiving of selected threads. The delay between marking a section and archiving has to be set up using <a href="/wiki/Template:Autoarchive_resolved_section" title="Template:Autoarchive resolved section"><span style="font-family:monospace,monospace;">{{Autoarchive resolved section}}</span></a>. The archiving is performed by <a href="/wiki/User:SpBot" title="User:SpBot">SpBot</a>. </p>

      <h3><span class="mw-headline" id="Template_parameters">Template parameters</span></h3>

        <a class="mw-selflink selflink"><span class="submit ui-button ui-widget ui-state-default ui-corner-all ui-button-icon-only" role="button"><span class="ui-button-icon-primary ui-icon ui-icon-help">&nbsp;</span><span class="ui-button-text">Parameter <b>text</b> is empty</span></span></a>

          <dt>
           TemplateData
          </dt>
         </dl> 
         <p><a href="https://www.mediawiki.org/wiki/Special:MyLanguage/Extension:TemplateData" class="extiw" title="mw:Special:MyLanguage/Extension:TemplateData">TemplateData</a> is a way to store information about template parameters (the description of those and of the whole template) for both humans and machines. It is used by <a href="https://www.mediawiki.org/wiki/Special:MyLanguage/:VisualEditor" class="extiw" title="mw:Special:MyLanguage/:VisualEditor">VisualEditor</a> and possibly other tools like <a href="/wiki/Special:MyLanguage/Commons:Upload_Wizard" title="Special:MyLanguage/Commons:Upload Wizard">Upload Wizard</a>. </p> 
         <hr> 
         <p><u>Existing template documentation</u><br> At Wikimedia Commons, it is recommended to use <a href="/wiki/Template:TemplateBox" title="Template:TemplateBox"><span style="font-family:monospace,monospace;">{{TemplateBox}}</span></a> with either <code>useTemplateData=1</code> or <code>useTemplateData=only</code> on the <code>/doc</code> subpage and transcluding it with <a href="/wiki/Template:Documentation" title="Template:Documentation"><span style="font-family:monospace,monospace;">{{Documentation}}</span></a> into the template. <code>&lt;nowiki&gt;</code>-tags can be wrapped around the arguments, if required, to avoid templates being expanded.<br> </p>
         <p><u>Newly created template documentation and imports</u><br> Another option, especially for imported templates, or for users with JSON experience, is placing raw &lt;templatedata&gt;-tags into the Wikitext of the template, as described in various Wikipediae. </p>
         <p><u>Discussion</u><br> <a href="/wiki/Commons:Requests_for_comment/How_Commons_should_deal_with_TemplateData" title="Commons:Requests for comment/How Commons should deal with TemplateData">There is an ongoing discussion about that matter. Feel invited to comment if you are experienced in either way, your time permits and if you like to share your opinion or to make a suggestion.</a> </p>

The delay between marking a section and archiving has to be set up using {{Autoarchive resolved section}}. The archiving is performed by SpBot.</p>

      <h2><span class="mw-headline" id="Own_design">Own design</span></h2> 
      <p>If you don't like the <a href="/wiki/Template:Section_resolved/standard_module" title="Template:Section resolved/standard module">default template</a>, you can use your own design.        <li>For <tt>Commons:Example/Subpage A/Subsubpage AB</tt> the root is <tt>Commons:Example</tt>. So, you must create <tt>Commons:Example/section resolved module</tt> to make it work.</li>
