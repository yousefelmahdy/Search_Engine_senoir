       <h3 id="configuring-the-template-chooser"><a href="#configuring-the-template-chooser">Configuring the template chooser</a></h3> 
       <p>You can customize the issue template chooser that people see when creating a new issue in your repository by adding a <em>config.yml</em> file to the <code>.github/ISSUE_TEMPLATE</code> folder.</p> 
       <p>You can encourage contributors to use issue templates by setting <code>blank_issues_enabled</code> to <code>false</code>. If you set <code>blank_issues_enabled</code> to <code>true</code>, people will have the option to open a blank issue.</p> 
       <div class="extended-markdown note border rounded-1 mb-4 p-3 color-border-info color-bg-info f5"> 
        <p><strong>Note:</strong> If you used the legacy workflow to manually create an <code>issue_template.md</code> file and enable blank issues in your <em>config.yml</em> file, the template in <code>issue_template.md</code> will be used when people chose to open a blank issue. If you disable blank issues, the template will never be used.</p> 
       </div> 
       <p>If you prefer to receive certain reports outside of GitHub, you can direct people to external sites with <code>contact_links</code>.</p> 
       <p>Here is an example <em>config.yml</em> file.</p> 
       <pre><code class="hljs language-shell">blank_issues_enabled: false
contact_links:
  - name: GitHub Community Support
    url: https://github.community/
    about: Please ask and answer questions here.
  - name: GitHub Security Bug Bounty
    url: https://bounty.github.com/
    about: Please report security vulnerabilities here.</code></pre> 
       <p>Your configuration file will customize the template chooser when the file is merged into the repository's default branch.</p>
