      <p>Wikipedia's <a href="/wiki/Wikipedia:Verifiability" title="Wikipedia:Verifiability">verifiability</a> policy requires <a href="/wiki/Wikipedia:Inline_citation" title="Wikipedia:Inline citation">inline citations</a> for any material <a href="/wiki/Wikipedia:CHALLENGED" class="mw-redirect" title="Wikipedia:CHALLENGED">challenged or likely to be challenged</a>, and for all quotations, anywhere in <a href="/wiki/Wikipedia:Mainspace" class="mw-redirect" title="Wikipedia:Mainspace">article space</a>. </p>
      <p>A citation or reference in an article usually has two parts. In the first part, each section of text that is either based on, or quoted from, an outside source is marked as such with an <a href="/wiki/Wikipedia:INCITE" class="mw-redirect" title="Wikipedia:INCITE">inline citation</a>. The inline citation may be a superscript <a href="/wiki/Help:Footnotes" title="Help:Footnotes">footnote</a> number, or an abbreviated version of the citation called a <a href="#Short_citations"><i>short citation</i></a>. The second necessary part of the citation or reference is the list of full references, which provides complete, formatted detail about the source, so that anyone reading the article can find it and verify it. </p>
      <p>This page explains how to place and format both parts of the citation. Each article should use one citation method or style throughout. If an article already has citations, preserve consistency by using that method or seek consensus on the talk page before changing it <style data-mw-deduplicate="TemplateStyles:r1015928993">.mw-parser-output div.crossreference{padding-left:0}</style><span role="note" class="hatnote navigation-not-searchable crossreference">(the principle is reviewed at <a href="#Variation_in_citation_methods">§&nbsp;Variation in citation methods</a>)</span>. While you should try to write citations correctly, what matters most is that you provide enough information to identify the source. Others will improve the formatting if needed. See: "<a href="/wiki/Help:Referencing_for_beginners" title="Help:Referencing for beginners">Help:Referencing for beginners</a>", for a brief introduction on how to put references in Wikipedia articles; and <a href="/wiki/Wikipedia:VisualEditor/User_guide#Using_standard_cite_templates" title="Wikipedia:VisualEditor/User guide">cite templates in Visual Editor</a>, about a graphical way for citation, included in Wikipedia. </p>

      <h2><span class="mw-headline" id="Types_of_citation">Types of citation</span></h2> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:CITETYPE"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:CITETYPE&amp;redirect=no">WP:CITETYPE</a></span></li>
       </ul>
      </div> 
      <ul>
       <li>A <b>full citation</b> fully identifies a <a href="/wiki/Wikipedia:SOURCES" class="mw-redirect" title="Wikipedia:SOURCES">reliable source</a> and, where applicable, the place in that source (such as a page number) where the information in question can be found. For example: <span class="example" style="font-family: Georgia, 'DejaVu Serif', serif; color: #006400;">Rawls, John. <i>A Theory of Justice</i>. Harvard University Press, 1971, p. 1.</span> This type of citation is usually given as a <a href="/wiki/Help:Footnotes" title="Help:Footnotes">footnote</a>, and is the most commonly used citation method in Wikipedia articles.</li> 
       <li>An <b>inline citation</b> means any citation added close to the material it supports, for example after the sentence or paragraph, normally in the form of a footnote.</li> 
       <li>A <b>short citation</b> is an inline citation that identifies the place in a source where specific information can be found, but without giving full details of the source – these will have been provided in a full bibliographic citation either in an earlier footnote, or in a separate section. For example: <span class="example" style="font-family: Georgia, 'DejaVu Serif', serif; color: #006400;">Rawls 1971, p. 1.</span> This system is used in some articles.</li> 
       <li><b>In-text attribution</b> involves adding the source of a statement to the article text, such as <span class="example" style="font-family: Georgia, 'DejaVu Serif', serif; color: #006400;">Rawls argues that X.<sup>[5]</sup></span> This is done whenever a writer or speaker should be credited, such as with quotations, <a href="/wiki/Wikipedia:Close_paraphrasing" title="Wikipedia:Close paraphrasing">close paraphrasing</a>, or statements of opinion or uncertain fact. The in-text attribution does not give full details of the source – this is done in a footnote in the normal way. See <a href="#In-text_attribution">In-text attribution</a> below.</li> 
       <li>A <b>general reference</b> is a citation that supports content, but is not linked to any particular piece of material in the article through an inline citation. General references are usually listed at the end of the article in a References section. They are usually found in underdeveloped articles, especially when all article content is supported by a single source. They may also be listed in more developed articles as a supplement to inline citations.</li>
      </ul> 
      <h2><span class="mw-headline" id="When_and_why_to_cite_sources">When and why to cite sources</span></h2> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:WHYCITE"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:WHYCITE&amp;redirect=no">WP:WHYCITE</a></span></li>
       </ul>
      </div> 
      <p>By citing sources for Wikipedia content, you enable users to <a href="/wiki/Wikipedia:V" class="mw-redirect" title="Wikipedia:V">verify</a> that the information given is supported by reliable sources, thus improving the <a href="/wiki/Reliability_of_Wikipedia" title="Reliability of Wikipedia">credibility of Wikipedia</a> while showing that the content is <a href="/wiki/Wikipedia:NOR" class="mw-redirect" title="Wikipedia:NOR">not original research</a>. You also help users find <a href="/wiki/Wikipedia:LINK" class="mw-redirect" title="Wikipedia:LINK">additional information</a> on the subject; and by giving attribution you avoid <a href="/wiki/Wikipedia:PLAGFORM" class="mw-redirect" title="Wikipedia:PLAGFORM">plagiarising</a> the source of your words or ideas. </p>
      <p>In particular, sources are required for material that is <a href="/wiki/Wikipedia:V" class="mw-redirect" title="Wikipedia:V">challenged or likely to be challenged</a> – if reliable sources cannot be found for challenged material, it is likely to be removed from the article. Sources are also required when <a href="/wiki/Wikipedia:QUOTE" class="mw-redirect" title="Wikipedia:QUOTE">quoting</a> someone, with or without quotation marks, or <a href="/wiki/Wikipedia:Close_paraphrasing" title="Wikipedia:Close paraphrasing">closely paraphrasing</a> a source. However, the citing of sources is not limited to those situations – editors are always encouraged to add or improve citations for any information contained in an article. </p>
      <p>Citations are especially desirable for statements about living persons, particularly when the statements are contentious or potentially defamatory. In accordance with the <a href="/wiki/Wikipedia:BLP" class="mw-redirect" title="Wikipedia:BLP">biography of living persons policy</a>, unsourced information of this type is likely to be removed on sight. </p> 
      <h3><span class="mw-headline" id="Multimedia">Multimedia</span></h3> 
      <p>For an image or other media file, details of its origin and copyright status should appear on its <a href="/wiki/Help:File_page" class="mw-redirect" title="Help:File page">file page</a>. Image <a href="/wiki/Wikipedia:Caption" class="mw-redirect" title="Wikipedia:Caption">captions</a> should be referenced as appropriate just like any other part of the article. A citation is not needed for descriptions such as <a href="/wiki/Wikipedia:Alternative_text_for_images" class="mw-redirect" title="Wikipedia:Alternative text for images">alt text</a> that are verifiable directly from the image itself, or for text that merely identifies a source (e.g., the caption "<i><a href="/wiki/Belshazzar%27s_Feast_(Rembrandt)" title="Belshazzar's Feast (Rembrandt)">Belshazzar's Feast</a></i> (1635)" for <i><a href="/wiki/File:Rembrandt-Belsazar.jpg" title="File:Rembrandt-Belsazar.jpg">File:Rembrandt-Belsazar.jpg</a></i>). </p> 
      <h3><span class="mw-headline" id="When_not_to_cite">When <em>not</em> to cite</span></h3> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:WHENNOTCITE"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:WHENNOTCITE&amp;redirect=no">WP:WHENNOTCITE</a></span></li>
       </ul>
      </div> 
      <p>Citations are <em>not</em> used on <a href="/wiki/Wikipedia:MOSDAB" class="mw-redirect" title="Wikipedia:MOSDAB">disambiguation pages</a> (sourcing for the information given there should be done in the target articles). Citations are often omitted from the <a href="/wiki/Wikipedia:LEAD" class="mw-redirect" title="Wikipedia:LEAD">lead section</a> of an article, insofar as the lead summarizes information for which sources are given later in the article, although quotations and controversial statements, particularly if about living persons, should be supported by citations even in the lead. See <a href="/wiki/Wikipedia:LEADCITE" class="mw-redirect" title="Wikipedia:LEADCITE">WP:LEADCITE</a> for more information. </p>

      <h3><span class="mw-headline" id="Examples">Examples</span></h3> 
      <h4><span class="mw-headline" id="Books">Books</span></h4> 
      <div role="note" class="hatnote navigation-not-searchable">
       See also the template {{<a href="/wiki/Template:Cite_book" title="Template:Cite book">cite book</a>}}.
      </div> 
      <p>Citations for books typically include: </p> 
      <ul>
       <li>name of author(s)</li> 
       <li>title of book</li> 
       <li>volume when appropriate</li> 
       <li>name of publisher</li> 
       <li>place of publication</li> 
       <li>date of publication of the edition</li> 
       <li>chapter or page numbers cited, if appropriate</li> 
       <li>edition, if not the first edition</li> 
       <li><a href="/wiki/Wikipedia:ISBN" title="Wikipedia:ISBN">ISBN</a> (optional)</li>
      </ul> 
      <p>Citations for individually authored chapters in books typically include: </p> 
      <ul>
       <li>name of author(s)</li> 
       <li>title of the chapter</li> 
       <li>name of book's editor</li> 
       <li>name of book and other details as above</li> 
       <li>chapter number or page numbers for the chapter (optional)</li>
      </ul> 
      <p>In some instances, the <a href="/wiki/Recto_and_verso" title="Recto and verso">verso</a> of a book's title page may record, "Reprinted with corrections XXXX" or similar, where 'XXXX' is a year. This is a different version of a book in the same way that different editions are different versions. In such a case, record: the year of the particular reprint, the edition immediately prior to this particular reprint (if not the first edition) and a note to say "Reprint with corrections". If {{<a href="/wiki/Template:Cite" class="mw-redirect" title="Template:Cite">cite</a>}} (or similar) is being used, the notation, "Reprint with corrections", can be added immediately following the template. <a href="#Dates_and_reprints_of_older_publications">§&nbsp;Dates and reprints of older publications</a> gives an example of appending a similar textual note. </p> 
      <h4><span class="mw-headline" id="Journal_articles">Journal articles</span></h4> 
      <div role="note" class="hatnote navigation-not-searchable">
       See also the template {{<a href="/wiki/Template:Cite_journal" title="Template:Cite journal">cite journal</a>}}.
      </div> 
      <p>Citations for journal articles typically include: </p> 
      <ul>
       <li>name of the author(s)</li> 
       <li>year and sometimes month of publication</li> 
       <li>title of the article</li> 
       <li>name of the journal</li> 
       <li>volume number, issue number, and page numbers (article numbers in some electronic journals)</li> 
       <li><a href="/wiki/Wikipedia:DOI" class="mw-redirect" title="Wikipedia:DOI">DOI</a> and/or other <a href="/wiki/List_of_academic_databases_and_search_engines" title="List of academic databases and search engines">identifiers</a> are optional</li>
      </ul> 
      <h4><span class="mw-headline" id="Newspaper_articles">Newspaper articles</span></h4> 
      <div role="note" class="hatnote navigation-not-searchable">
       See also the template {{<a href="/wiki/Template:Cite_news" title="Template:Cite news">cite news</a>}}.
      </div> 
      <p>Citations for newspaper articles typically include: </p> 
      <ul>
       <li><a href="/wiki/Byline" title="Byline">byline</a> (author's name), if any</li> 
       <li>title of the article</li> 
       <li>name of the newspaper in italics</li> 
       <li>city of publication (if not included in name of newspaper)</li> 
       <li>date of publication</li> 
       <li>page number(s) are optional and may be substituted with negative number(s) on microfilm reels</li>
      </ul> 
      <h4><span class="mw-headline" id="Web_pages">Web pages</span></h4> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:CITEWEB"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:CITEWEB&amp;redirect=no">WP:CITEWEB</a></span></li>
       </ul>
      </div> 
      <div role="note" class="hatnote navigation-not-searchable">
       See also the template {{<a href="/wiki/Template:Cite_web" title="Template:Cite web">cite web</a>}}.
      </div> 
      <p>Citations for World Wide Web pages typically include: </p> 
      <ul>
       <li>URL of the specific web page <em>where the referenced content can be found</em></li> 
       <li>name of the author(s)</li> 
       <li>title of the article</li> 
       <li>title or domain name of the website</li> 
       <li>publisher, if known</li> 
       <li>date of publication</li> 
       <li>page number(s) (if applicable)</li> 
       <li>the date you retrieved (or accessed) the web page (required if the publication date is unknown)</li>
      </ul> 
      <h4><span class="mw-headline" id="Sound_recordings">Sound recordings</span></h4> 
      <div role="note" class="hatnote navigation-not-searchable">
       Further information: <a href="/wiki/Help:References_and_page_numbers#Other_in-source_locations" title="Help:References and page numbers">Help:References and page numbers §&nbsp;Other in-source locations</a>
      </div> 
      <p>Citations for sound recordings typically include: </p> 
      <ul>
       <li>name of the composer(s), songwriter(s), script writer(s) or the like</li> 
       <li>name of the performer(s)</li> 
       <li>title of the song or individual track</li> 
       <li>title of the album (if applicable)</li> 
       <li>name of the record label</li> 
       <li>year of release</li> 
       <li>medium (for example: LP, audio cassette, CD, MP3 file)</li> 
       <li>approximate time at which event or point of interest occurs, where appropriate</li>
      </ul> 
      <p>Do not cite an entire body of work by one performer. Instead, make one citation for each work your text relies on. </p> 
      <h4><span id="Film.2C_television.2C_or_video_recordings"></span><span class="mw-headline" id="Film,_television,_or_video_recordings">Film, television, or video recordings</span></h4> 
      <div role="note" class="hatnote navigation-not-searchable">
       See also the template {{<a href="/wiki/Template:Cite_AV_media" title="Template:Cite AV media">cite AV media</a>}}.
      </div> 
      <p>Citations for films, TV episodes, or video recordings typically include: </p> 
      <ul>
       <li>name of the director</li> 
       <li>name of the producer, if relevant</li> 
       <li>names of major performers</li> 
       <li>the title of a TV episode</li> 
       <li>title of the film or TV series</li> 
       <li>name of the studio</li> 
       <li>year of release</li> 
       <li>medium (for example: film, videocassette, DVD)</li> 
       <li>approximate time at which event or point of interest occurs, where appropriate</li>
      </ul> 
      <h4><span class="mw-headline" id="Wikidata">Wikidata</span></h4> 
      <div role="note" class="hatnote navigation-not-searchable">
       See also the template {{<a href="/wiki/Template:Cite_Q" title="Template:Cite Q">cite Q</a>}}.
      </div> 
      <p>Wikidata is largely user-generated, and articles should not directly cite Wikidata as a source (just as it would be inappropriate to cite other Wikipedias' articles as sources). </p>
      <p>Wikidata's statements, however, can be directly transcluded into articles; this is usually done to provide external links or infobox data. For example, more than two million external links from Wikidata are shown through the {{<a href="/wiki/Template:Authority_control" title="Template:Authority control">Authority control</a>}} template. There has been controversy over the use of Wikidata in the English Wikipedia due to vandalism and its own sourcing. While there is no consensus on whether information from Wikidata should be used at all, there is general agreement that any Wikidata statements that are transcluded need to be just as – or more – reliable compared to Wikipedia content. As such, <a href="/wiki/Module:WikidataIB" title="Module:WikidataIB">Module:WikidataIB</a> and some related modules and templates filter unsourced Wikidata statements by default; however, other modules and templates, such as <a href="/wiki/Module:Wikidata" title="Module:Wikidata">Module:Wikidata</a>, do not. </p>
      <p>In order to transclude an item from Wikidata, the <a href="https://www.wikidata.org/wiki/Wikidata:Glossary#QID" class="extiw" title="d:Wikidata:Glossary">QID (Q number)</a> of an item in Wikidata needs to be known. QID can by found by searching for an item by the name or <a href="/wiki/Digital_object_identifier" title="Digital object identifier">DOI</a> in Wikidata. A book, a journal article, a musical recording, sheet music or any other item can be represented by a structured item in Wikidata. </p>
      <p>As of December 2020, {{<a href="/wiki/Template:Cite_Q" title="Template:Cite Q">Cite Q</a>}} does not support "last, first" or Vancouver-style author name lists, so it should not be used in articles in which "last, first" or Vancouver-style author names are the <a href="/wiki/Wikipedia:CITEVAR" class="mw-redirect" title="Wikipedia:CITEVAR">dominant citation style</a>. </p>

      <h3><span class="mw-headline" id="Identifying_parts_of_a_source">Identifying parts of a source</span></h3> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:PAGENUM"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:PAGENUM&amp;redirect=no">WP:PAGENUM</a></span></li>
       </ul>
      </div> 
      <div role="note" class="hatnote navigation-not-searchable">
       Further information: <a href="/wiki/Help:References_and_page_numbers" title="Help:References and page numbers">Help:References and page numbers</a>
      </div> 
      <p>When citing lengthy sources, you should identify which part of a source is being cited. </p> 
      <h4><span class="mw-headline" id="Books_and_print_articles">Books and print articles</span></h4> 
      <p>Specify the page number or range of page numbers. Page numbers are not required for a reference to the book or article as a whole. When you specify a page number, it is helpful to specify the version (date and edition for books) of the source because the layout, pagination, length, etc. can change between editions. </p>
      <p>If there are no page numbers, whether in <a href="/wiki/Ebook" title="Ebook">ebooks</a> or print materials, then you can use other means of identifying the relevant section of a lengthy work, such as the chapter number or the section title. </p>
      <p>In some works, such as plays and ancient works, there are standard methods of referring to sections, such as "Act 1, scene 2" for plays and <a href="/wiki/Bekker_numbering" title="Bekker numbering">Bekker numbers</a> for Aristotle's works. Use these methods whenever appropriate. </p> 
      <h4><span class="mw-headline" id="Audio_and_video_sources">Audio and video sources</span></h4> 
      <p>Specify the time at which the event or other point of interest occurs. Be as precise as possible about the version of the source that you are citing; for example, movies are often released in different editions or "cuts". Due to variations between formats and playback equipment, precision may not be accurate in some cases. However, many government agencies do not publish minutes and transcripts but do post video of official meetings online; generally the subcontractors who handle audio-visual are quite precise. </p> 
      <h3><span class="mw-headline" id="Links_and_ID_numbers">Links and ID numbers</span></h3> 
      <p>A citation ideally includes a link or ID number to help editors locate the source. If you have a URL (web page) link, you can add it to the title part of the citation, so that when you add the citation to Wikipedia the URL becomes hidden and the title becomes clickable. To do this, enclose the URL and the title in square brackets—the URL first, then a space, then the title. For example: </p> 
      <div class="mw-highlight mw-highlight-lang-moin mw-content-ltr" dir="ltr">
       <pre><span></span><span class="c">''</span><span class="k">[http://monographs.iarc.fr/ENG/Monographs/vol66/mono66-7.pdf</span><span class="s"> IARC Monographs On The Evaluation Of Carcinogenic Risks To Humans – Doxefazepam</span><span class="k">]</span><span class="c">''</span>. International Agency For Research On Cancer (IARC). 66: 97–104. 13–20 February 1996.
</pre>
      </div> 
      <p>For web-only sources with no publication date, the "Retrieved" date (or the date you accessed the web page) should be included, in case the web page changes in the future. For example: <span class="example" style="font-family: Georgia, 'DejaVu Serif', serif; color: #006400;">Retrieved 15 July 2011</span> or you can use the <a href="/wiki/Wikipedia:RefToolbar_2.0#Automatic_date_insertion" class="mw-redirect" title="Wikipedia:RefToolbar 2.0">access-date</a> parameter in the automatic <a href="/wiki/Wikipedia:RefToolbar_2.0" class="mw-redirect" title="Wikipedia:RefToolbar 2.0">Wikipedia:refToolbar 2.0</a> editing window feature. </p>
      <p>You can also add an ID number to the end of a citation. The ID number might be an <a href="/wiki/ISBN" class="mw-redirect" title="ISBN">ISBN</a> for a book, a <a href="/wiki/Digital_object_identifier" title="Digital object identifier">DOI</a> (digital object identifier) for an article or some e-books, or any of several ID numbers that are specific to particular article databases, such as a PMID number for articles on <a href="/wiki/PubMed" title="PubMed">PubMed</a>. It may be possible to format these so that they are automatically activated and become clickable when added to Wikipedia, for example by typing ISBN (or PMID) followed by a space and the ID number. </p>
      <p>If your source is not available online, it should be available in reputable libraries, archives, or collections. If a citation without an external link is challenged as unavailable, any of the following is sufficient to show the material to be reasonably available (though not necessarily <a href="/wiki/Wikipedia:Verifiability#Reliable_sources" title="Wikipedia:Verifiability">reliable</a>): providing an <a href="/wiki/ISBN" class="mw-redirect" title="ISBN">ISBN</a> or <a href="/wiki/OCLC" title="OCLC">OCLC</a> number; linking to an established Wikipedia article about the source (the work, its author, or its publisher); or directly quoting the material on the talk page, <a href="/wiki/Wikipedia:COPYVIO" class="mw-redirect" title="Wikipedia:COPYVIO">briefly</a> and in context. </p> 
      <h4><span class="mw-headline" id="Linking_to_pages_in_PDF_files">Linking to pages in PDF files</span></h4> 
      <p><span class="anchor" id="PDF"></span><span class="anchor" id="pdf"></span> </p> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:PAGELINKS"></span><span id="WP:BOOKLINKS"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcuts</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:PAGELINKS&amp;redirect=no">WP:PAGELINKS</a></span></li>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:BOOKLINKS&amp;redirect=no">WP:BOOKLINKS</a></span></li>
       </ul>
      </div> 
      <p>Links to long PDF documents can be made more convenient by taking readers to a specific page with the addition of <code>#page<var>n</var></code> to the document URL, where <code>n</code> is the page number. For example, using <code>http://www.domain.com/document.pdf#page5</code> as the citation URL displays page five of the document in any PDF viewer that supports this feature. If the viewer or browser does not support it, it will display the first page instead. </p>

       Further information: <a href="/wiki/Wikipedia:Google_Books_and_Wikipedia" title="Wikipedia:Google Books and Wikipedia">Wikipedia:Google Books and Wikipedia</a> and <a href="/wiki/Wikipedia_talk:Citing_sources/Archive_30#Linking_to_Google_Books_pages" title="Wikipedia talk:Citing sources/Archive 30">Wikipedia talk:Citing sources/Archive 30 §&nbsp;Linking to Google Books pages</a>

Page links should only be added when the book is available for preview; they will not work with snippet view. Keep in mind that availability varies by location. No editor is required to add page links, but if another editor adds them, they should not be removed without cause; see the <a href="/wiki/Wikipedia_talk:Citing_sources/Archive_30#Linking_to_Google_Books_pages" title="Wikipedia talk:Citing sources/Archive 30">October 2010 RfC</a> for further information. </p>

      <h3><span class="mw-headline" id="Say_where_you_read_it">Say where <em>you</em> read it <span id="SAYWHEREYOUGOTIT"></span></span></h3> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:SAYWHEREYOUREADIT"></span><span id="WP:SAYWHEREYOUGOTIT"></span><span id="WP:SWYRT"></span><span id="WP:SWYGT"></span><span id="WP:SAYWHERE"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcuts</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:SAYWHEREYOUREADIT&amp;redirect=no">WP:SAYWHEREYOUREADIT</a></span></li>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:SAYWHEREYOUGOTIT&amp;redirect=no">WP:SAYWHEREYOUGOTIT</a></span></li>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:SWYRT&amp;redirect=no">WP:SWYRT</a></span></li>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:SWYGT&amp;redirect=no">WP:SWYGT</a></span></li>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:SAYWHERE&amp;redirect=no">WP:SAYWHERE</a></span></li>
       </ul>
      </div> 
      <p>"Say where <em>you</em> read it" follows the practice in academic writing of citing sources directly only if you have read the source yourself. If your knowledge of the source is secondhand—that is, if you have read Jones (2010), who cited Smith (2009), and you want to use what Smith (2009) said—make clear that your knowledge of Smith is based on your reading of Jones. </p>
      <p>When citing the source, write the following (this formatting is just an example): </p> 
      <style data-mw-deduplicate="TemplateStyles:r996844942">.mw-parser-output .templatequote{overflow:hidden;margin:1em 0;padding:0 40px}.mw-parser-output .templatequote .templatequotecite{line-height:1.5em;text-align:left;padding-left:1.6em;margin-top:0}</style>
      <blockquote class="templatequote">
       <p>John Smith (2009). <i>Name of Book I Haven't Seen</i>, Cambridge University Press, p. 99, cited in Paul Jones (2010). <i>Name of Encyclopedia I Have Seen</i>, Oxford University Press, p. 29.</p>
      </blockquote> 
      <p>Or if you are using short citations: </p> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r996844942">
      <blockquote class="templatequote">
       <p>Smith (2009), p. 99, cited in Jones (2010), p. 29.</p>
      </blockquote> 
      <p><b>Note</b>: The advice to "say where <em>you</em> read it" does <em>not</em> mean that you have to give credit to any search engines, websites, libraries, library catalogs, archives, subscription services, bibliographies, or other sources that led you to Smith's book. If you have read a book or article yourself, that's all you have to cite. You do not have to specify <em>how</em> you obtained and read it. </p>
      <p>So long as you are confident that you read a true and accurate copy, it does not matter whether you read the material using an online service like Google Books; using preview options at a bookseller's website like Amazon; through your library; via online <a href="/wiki/Wikipedia:PAYWALL" class="mw-redirect" title="Wikipedia:PAYWALL">paid databases of scanned publications</a>, such as <a href="/wiki/JSTOR" title="JSTOR">JSTOR</a>; using <a href="/wiki/Reading_machine" title="Reading machine">reading machines</a>; on an <a href="/wiki/E-reader" title="E-reader">e-reader</a> (except to the extent that this affects page numbering); or any other method. </p> 
      <h3><span class="mw-headline" id="Dates_and_reprints_of_older_publications">Dates and reprints of older publications</span></h3> 
      <p>Editors should be aware that older sources (especially those in the public domain) are sometimes reprinted with modern publication dates. When this occurs and the citation style being used requires it, cite both the <em>original</em> publication date, as well as the date of the re-publication, e.g.: </p> 
      <ul>
       <li>
        <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r999302996"><cite id="CITEREFDarwin1964" class="citation book cs1">Darwin, Charles (1964) [1859]. <i>On the Origin of Species</i> (facsimile of 1st&nbsp;ed.). Harvard University Press.</cite><span title="ctx_ver=Z39.88-2004&amp;rft_val_fmt=info%3Aofi%2Ffmt%3Akev%3Amtx%3Abook&amp;rft.genre=book&amp;rft.btitle=On+the+Origin+of+Species&amp;rft.edition=facsimile+of+1st&amp;rft.pub=Harvard+University+Press&amp;rft.date=1964&amp;rft.aulast=Darwin&amp;rft.aufirst=Charles&amp;rfr_id=info%3Asid%2Fen.wikipedia.org%3AWikipedia%3ACiting+sources" class="Z3988"></span></li>
      </ul> 
      <p>This is done automatically in the {{<a href="/wiki/Template:Citation" title="Template:Citation">citation</a>}} and {{<a href="/wiki/Template:Cite_book" title="Template:Cite book">cite book</a>}} templates when you use the <code class="nowrap" style="">|orig-date=</code> parameter. </p>
      <p>Alternately, information about the reprint can be appended as a textual note: </p> 
      <ul>
       <li>
        <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r999302996"><cite id="CITEREFBoole1854" class="citation book cs1">Boole, George (1854). <i>An Investigation of the Laws of Thought on Which Are Founded the Mathematical Theories of Logic and Probabilities</i>. Macmillan.</cite><span title="ctx_ver=Z39.88-2004&amp;rft_val_fmt=info%3Aofi%2Ffmt%3Akev%3Amtx%3Abook&amp;rft.genre=book&amp;rft.btitle=An+Investigation+of+the+Laws+of+Thought+on+Which+Are+Founded+the+Mathematical+Theories+of+Logic+and+Probabilities&amp;rft.pub=Macmillan&amp;rft.date=1854&amp;rft.aulast=Boole&amp;rft.aufirst=George&amp;rfr_id=info%3Asid%2Fen.wikipedia.org%3AWikipedia%3ACiting+sources" class="Z3988"></span> Reprinted with corrections, Dover Publications, New York, NY, 1958.</li>
      </ul> 
      <h3><span class="mw-headline" id="Seasonal_publication_dates_and_differing_calendar_systems">Seasonal publication dates and differing calendar systems</span></h3> 
      <p>Publication dates, for both older and recent sources, should be written with the goal of helping the reader find the publication and, once found, confirm that the correct publication has been located. For example, if the publication date bears a date in the Julian calendar, it should not be converted to the Gregorian calendar. </p>
      <p>If the publication date was given as a season or holiday, such as "Winter" or "Christmas" of a particular year or two-year span, it should not be converted to a month or date, such as <span class="example deprecated-content example-bad" style="font-family: Georgia, 'DejaVu Serif', serif; color: #8B0000;">July–August</span> or <span class="example deprecated-content example-bad" style="font-family: Georgia, 'DejaVu Serif', serif; color: #8B0000;">December 25</span>. If a publication provided both seasonal and specific dates, prefer the specific one. </p> 
      <h3><span class="mw-headline" id="Additional_annotation">Additional annotation</span></h3> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:ANNOTATION"></span><span id="WP:FOOTQUOTE"></span><span id="WP:FQ"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcuts</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:ANNOTATION&amp;redirect=no">WP:ANNOTATION</a></span></li>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:FOOTQUOTE&amp;redirect=no">WP:FOOTQUOTE</a></span></li>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:FQ&amp;redirect=no">WP:FQ</a></span></li>
       </ul>
      </div> 
      <p>In most cases it is sufficient for a citation footnote simply to identify the source (as described in the sections above); readers can then consult the source to see how it supports the information in the article. Sometimes, however, it is useful to include additional <a href="/wiki/Annotation" title="Annotation">annotation</a> in the footnote, for example to indicate precisely which information the source is supporting (particularly when a single footnote lists more than one source – 
       <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1015928993"><span role="note" class="hatnote navigation-not-searchable crossreference">see <a href="#Bundling_citations">§&nbsp;Bundling citations</a> and <a href="#Text–source_integrity">§&nbsp;Text–source integrity</a>, below</span>). </p>
      <p>A footnote may also contain a relevant exact quotation from the source. This is especially helpful when the cited text is long or dense. A quotation allows readers to immediately identify the applicable portion of the reference. Quotes are also useful if the source is not easily accessible. </p>
      <p>In the case of non-English sources, it may be helpful to quote from the original text and then give an English translation. If the article itself contains a translation of a quote from such a source (without the original), then the original should be included in the footnote. 
       <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1015928993"><span role="note" class="hatnote navigation-not-searchable crossreference">(See the <a href="/wiki/Wikipedia:Verifiability#Non-English_sources" title="Wikipedia:Verifiability">WP:Verifiability §&nbsp;Non-English sources</a> policy for more information.)</span> </p> 
      <h2><span class="mw-headline" id="Inline_citations">Inline citations</span></h2> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:INCITE"></span><span id="WP:INLINECITE"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcuts</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:INCITE&amp;redirect=no">WP:INCITE</a></span></li>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:INLINECITE&amp;redirect=no">WP:INLINECITE</a></span></li>
       </ul>
      </div> 
      <div role="note" class="hatnote navigation-not-searchable">
       Further information: <a href="/wiki/Wikipedia:Inline_citation" title="Wikipedia:Inline citation">Wikipedia:Inline citation</a>
      </div> 
      <p>Inline citations allow the reader to associate a given bit of material in an article with the specific reliable source(s) that support it. Inline citations are added using either <a href="#Footnotes">footnotes</a> (long or <a href="#Shortened_footnotes">short</a>) or <a href="#Parenthetical_referencing">parenthetical references</a>. This section describes how to add either type, and also describes how to create a list of full bibliography citations to support shortened footnotes. </p>
      <p>The first editor to add footnotes to an article must create a <a href="/wiki/Help:Section" title="Help:Section">section</a> where those citations are to appear. </p> 
      <h3><span class="mw-headline" id="Footnotes">Footnotes</span></h3> 
      <div role="note" class="hatnote navigation-not-searchable">
       See also: <a href="/wiki/Help:Footnotes" title="Help:Footnotes">Help:Footnotes</a>
      </div> 
      <h4><span class="mw-headline" id="How_to_create_the_list_of_citations">How to create the list of citations</span></h4> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:REFLIST"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:REFLIST&amp;redirect=no">WP:REFLIST</a></span></li>
       </ul>
      </div> 
      <p>This section, if needed, is usually titled "Notes" or "References", and is placed at or near the bottom of the article. For more about the order and titles of sections at the end of an article (which may also include "Further reading" and "External links" sections), see <a href="/wiki/Wikipedia:Footers" class="mw-redirect" title="Wikipedia:Footers">Wikipedia:Footers</a>. </p>
      <p>With some exceptions discussed below, citations appear in a single section containing only the <code>&lt;references /&gt;</code> tag or the <code>{{<a href="/wiki/Template:Reflist" title="Template:Reflist">Reflist</a>}}</code> template. For example: </p> 
      <pre>== References ==
{{Reflist}}
</pre> 
      <p>The footnotes will then automatically be listed under that section heading. Each numbered footnote marker in the text is a clickable link to the corresponding footnote, and each footnote contains a <a href="/wiki/Caret" title="Caret">caret</a> that links back to the corresponding point in the text. </p> 
      <p><span class="anchor" id="Avoid_scrolling_lists"></span><span class="anchor" id="Scrolling_lists"></span><span class="anchor" id="ASL"></span>
       <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855"></p>
      <div class="module-shortcutanchordiv">
       <span id="WP:ASL"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:ASL&amp;redirect=no">WP:ASL</a></span></li>
       </ul>
      </div> 
      <p>Scrolling lists, or lists of citations appearing within a <a href="/wiki/Scroll_box" class="mw-redirect" title="Scroll box">scroll box</a>, should never be used. This is because of issues with readability, browser compatibility, <a href="/wiki/Wikipedia:Accessibility" class="mw-redirect" title="Wikipedia:Accessibility">accessibility</a>, printing, and <a href="/wiki/Wikipedia:Mirrors_and_forks" title="Wikipedia:Mirrors and forks">site mirroring</a>.<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">[note 2]</a></sup> </p>
      <p>If an article contains a list of <a href="#General_reference">general references</a>, this is usually placed in a separate section, titled (for example) "References". This usually comes immediately after the section(s) listing footnotes, if any. (If the general references section is called "References", then the citations section is usually called "Notes".) </p> 
      <h4><span class="mw-headline" id="How_to_place_an_inline_citation_using_ref_tags"><span id="Adding_the_citation"></span><span id="Inline_reference"></span><span id="Footnotes_and_references"></span>How to place an inline citation using ref tags</span></h4> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:CITEFOOT"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:CITEFOOT&amp;redirect=no">WP:CITEFOOT</a></span></li>
       </ul>
      </div> 
      <div role="note" class="hatnote navigation-not-searchable">
       Further information: <a href="/wiki/Help:Footnotes#Footnotes:_the_basics" title="Help:Footnotes">Footnotes: the basics</a>
      </div> 
      <p>To create a footnote, use the <code>&lt;ref&gt;...&lt;/ref&gt;</code> syntax at the appropriate place in the article text, for example: </p> 
      <ul>
       <li><code>Justice is a human invention.&lt;ref&gt;Rawls, John. ''A Theory of Justice''. Harvard University Press, 1971, p. 1.&lt;/ref&gt; It&nbsp;...</code></li>
      </ul> 
      <p>which will be displayed as something like: </p> 
      <ul>
       <li>Justice is a human invention.<sup class="reference nowrap ltr" style="color:#002bb8;">[1]</sup> It ...</li>
      </ul> 
      <p>It will also be necessary to generate the list of footnotes (where the citation text is actually displayed); for this, see the previous section. </p>
      <p>As in the above example, citation markers are normally placed <em>after</em> adjacent punctuation such as periods (full stops) and commas. For exceptions, see the <a href="/wiki/Wikipedia:Manual_of_Style#Punctuation_and_footnotes" title="Wikipedia:Manual of Style">WP:Manual of Style §&nbsp;Punctuation and footnotes</a>. Note also that no space is added before the citation marker. Citations should not be placed within, or on the same line as, section headings. </p>
      <p>The citation should be added close to the material it supports, offering <a href="#Text–source_integrity">text–source integrity</a>. If a word or phrase is particularly contentious, an inline citation may be added next to that word or phrase within the sentence, but it is usually sufficient to add the citation to the end of the clause, sentence, or paragraph, so long as it's clear which source supports which part of the text. </p> 
      <h4><span class="mw-headline" id="Separating_citations_from_explanatory_footnotes">Separating citations from explanatory footnotes</span></h4> 
      <div role="note" class="hatnote navigation-not-searchable">
       See also: <a href="/wiki/Wikipedia:Manual_of_Style/Layout#Notes_and_references" title="Wikipedia:Manual of Style/Layout">Wikipedia:Manual of Style/Layout §&nbsp;Notes and references</a>, and <a href="/wiki/Help:Explanatory_notes" title="Help:Explanatory notes">Help:Explanatory notes</a>
      </div> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:EXPLNOTESECT"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:EXPLNOTESECT&amp;redirect=no">WP:EXPLNOTESECT</a></span></li>
       </ul>
      </div> 
      <p>If an article contains both footnoted citations and other (explanatory) footnotes, then it is possible (but not necessary) to divide them into two separate lists using <a href="/wiki/Help:Footnotes#Footnotes:_groups" title="Help:Footnotes">footnotes groups</a>. The explanatory footnotes and the citations are then placed in separate sections, called (for example) "Notes" and "References" respectively. </p>
      <p>Another method of separating explanatory footnotes from footnoted references is using {{<a href="/wiki/Template:Efn" title="Template:Efn">efn</a>}} for the explanatory footnotes. The advantage of this system is that the content of an explanatory footnote can in this case be referenced with a footnoted citation. When explanatory footnotes and footnoted references are not in separate lists, {{<a href="/wiki/Template:Refn" title="Template:Refn">refn</a>}} can be used for explanatory footnotes containing footnoted citations. </p> 
      <h4><span class="mw-headline" id="Avoiding_clutter">Avoiding clutter</span></h4> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:ILCLUTTER"></span><span id="WP:INLINECLUTTER"></span><span id="WP:INLINECITECLUTTER"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcuts</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:ILCLUTTER&amp;redirect=no">WP:ILCLUTTER</a></span></li>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:INLINECLUTTER&amp;redirect=no">WP:INLINECLUTTER</a></span></li>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:INLINECITECLUTTER&amp;redirect=no">WP:INLINECITECLUTTER</a></span></li>
       </ul>
      </div> 
      <p>Inline references can significantly bloat the wikitext in the edit window and can become difficult to manage and confusing. There are two main methods to avoid <a href="/wiki/Visual_pollution" title="Visual pollution">clutter</a> in the edit window: </p> 
      <ul>
       <li>Using <a href="/wiki/Help:Footnotes#List-defined_references" title="Help:Footnotes">list-defined references</a> by collecting the full citation code within the <a href="/wiki/Template:Reflist#List-defined_references" title="Template:Reflist">reference list template</a> {{reflist}}, and then inserting them in the text with a shortened reference tag, for example <code>&lt;ref name="Smith 2001, p99" /&gt;</code>.</li> 
       <li>Inserting <a href="#Short_citations">short citations</a> (see below) that then refer to a full list of source texts</li>
      </ul> 
      <p>As with other citation formats, articles should not undergo large-scale conversion between formats without consensus to do so. </p>
      <p>Note, however, that references defined in the reference list template can no longer be edited with the <a href="/wiki/Wikipedia:VisualEditor" title="Wikipedia:VisualEditor">VisualEditor</a>. </p> 
      <h4><span class="mw-headline" id="Repeated_citations">Repeated citations</span></h4> 
      <div role="note" class="hatnote navigation-not-searchable">
       Further information: <a href="/wiki/Help:Footnotes#Footnotes:_using_a_source_more_than_once" title="Help:Footnotes">Footnotes: using a source more than once</a>
      </div> 
      <p>For multiple use of the same inline citation or footnote, you can use the <a href="/wiki/Wikipedia:NAMEDREFS" class="mw-redirect" title="Wikipedia:NAMEDREFS">named references</a> feature, choosing a name to identify the inline citation, and typing <code class="nowrap" style="">&lt;ref name="<var>name</var>"&gt;text of the citation&lt;/ref&gt;</code>. Thereafter, the same named reference may be reused any number of times either before or after the defining use by typing the previous reference name, like this: <code class="nowrap" style="">&lt;ref name="<var>name</var>" /&gt;</code>. The use of the slash before the <code>&gt;</code> means that the tag is self-closing, and the <code class="nowrap" style="">&lt;/ref&gt;</code> used to close other references must not be used in addition. </p>
      <p>The text of the <code>name</code> can be almost anything?—?apart from being completely numeric. If spaces are used in the text of the <code>name</code>, the text must be placed within double quotes. Placing all named references within double quotes may be helpful to future editors who do not know that rule. To help with page maintenance, it is recommended that the text of the <code>name</code> have a connection to the inline citation or footnote, for example "author year page": <code class="nowrap" style="">&lt;ref name="<var>Smith 2005 p94</var>"&gt;text of the citation&lt;/ref&gt;</code>. </p>
      <p>Use straight quotation marks <code>"</code> to enclose the reference name. Do not use curly quotation marks <code>“”</code>. Curly marks are treated as another character, not as delimiters. The page will display an error if one style of quotation marks is used when first naming the reference, and the other style is used in a repeated reference, or if a mix of styles is used in the repeated references. </p> 
      <h4><span class="mw-headline" id="Citing_multiple_pages_of_the_same_source"><span id="MultiPages"></span>Citing multiple pages of the same source</span></h4> 
      <p><span class="anchor" id="IBID"></span><span class="anchor" id="op.cit."></span>
       <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855"></p>
      <div class="module-shortcutanchordiv">
       <span id="WP:IBID"></span><span id="WP:OPCIT"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcuts</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:IBID&amp;redirect=no">WP:IBID</a></span></li>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:OPCIT&amp;redirect=no">WP:OPCIT</a></span></li>
       </ul>
      </div> 
      <div role="note" class="hatnote navigation-not-searchable">
       Further information: <a href="/wiki/Help:References_and_page_numbers" title="Help:References and page numbers">Help:References and page numbers</a>
      </div> 
      <p>When an article cites many different pages from the same source, to avoid the redundancy of many big, nearly identical full citations, most Wikipedia editors use one of these options: </p> 
      <ul>
       <li><a href="/wiki/Wikipedia:NAMEDREFS" class="mw-redirect" title="Wikipedia:NAMEDREFS">Named references</a> in conjunction with a combined list of page numbers using the <a href="/wiki/Help:Citation_Style_1#Pages" title="Help:Citation Style 1"><code class="nowrap" style="">|pages=</code></a> parameter of the {{<a href="/wiki/Template:Cite" class="mw-redirect" title="Template:Cite">cite</a>}} templates (most commonly used, but can become confusing for large number of pages)</li> 
       <li>Named references in conjunction with the <code>{{<a href="/wiki/Template:Rp" title="Template:Rp">rp</a>}}</code> or <code>{{<a href="/wiki/Template:R" title="Template:R">r</a>}}</code> templates to specify the page</li> 
       <li><a href="/wiki/Wikipedia:CITESHORT" class="mw-redirect" title="Wikipedia:CITESHORT">Short citations</a></li>
      </ul> 
      <p>The use of <i><a href="/wiki/Ibid." title="Ibid.">ibid.</a></i>, <i><a href="/wiki/Id." class="mw-redirect" title="Id.">id.</a></i>, or similar abbreviations is discouraged, as they may become broken as new references are added (<i><a href="/wiki/Op._cit." title="Op. cit.">op. cit.</a></i> is less problematic in that it should refer explicitly to a citation contained in the article; however, not all readers are familiar with the meaning of the terms). If the use of <i>ibid</i> is extensive, tag the article using the {{<a href="/wiki/Template:Ibid" title="Template:Ibid">ibid</a>}} template. </p> 
      <h4><span class="mw-headline" id="Duplicate_citations">Duplicate citations</span></h4> 
      <p><span class="anchor" id="DupCites"></span><span class="anchor" id="DUPCITES"></span>
       <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855"></p>
      <div class="module-shortcutanchordiv">
       <span id="WP:DUPCITES"></span><span id="WP:DUPREF"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcuts</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:DUPCITES&amp;redirect=no">WP:DUPCITES</a></span></li>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:DUPREF&amp;redirect=no">WP:DUPREF</a></span></li>
       </ul>
      </div> 
      <p>Combine <i>precisely duplicated full citations</i>, in keeping with the existing citation style (if any). In this context "precisely duplicated" means having the same content, not necessarily identical strings ("The New York Times" is the same as "NY Times"; different access-dates are not significant). Do not discourage editors, particularly <a href="/wiki/Wikipedia:BITE" class="mw-redirect" title="Wikipedia:BITE">inexperienced</a> ones, from adding duplicate citations when the use of the source is appropriate, because a duplicate is better than no citation. But any editor should feel free to combine them, and doing so is the best practice on Wikipedia. </p>
      <p><i>Citations to different pages or parts of the same source</i> can also be combined (preserving the distinct parts of the citations), as described in the previous section. Any method that is consistent with the existing citation style (if any) may be used, or consensus can be sought to change the existing style. </p>
      <p>Finding duplicate citations by examining reference lists is difficult. There are some tools that can help: </p> 
      <ul>
       <li><a href="/wiki/Wikipedia:AutoWikiBrowser" title="Wikipedia:AutoWikiBrowser">AutoWikiBrowser</a> (AWB) will identify and (usually) correct <i>exact duplicates</i> between &lt;ref&gt;...&lt;/ref&gt; tags. See <a href="/wiki/Wikipedia:HTADC" class="mw-redirect" title="Wikipedia:HTADC">the documentation</a>.</li> 
       <li><a rel="nofollow" class="external text" href="https://www.convertcsv.com/url-extractor.htm">URL Extractor For Web Pages and Text</a> can identify Web citations with the <i>exact same URL</i> but otherwise possibly different. Occasionally references to the same Web page might be followed by different non-significant tracking parameters (<code>?utm ...</code>, <code>#ixzz...</code>), and will not be listed as duplicates. 
        <ul>
         <li>Step 1: enter the URL of the Wikipedia article and click "Load",</li> 
         <li>Step 2: tick "Only Display duplicate URL addresses" (which unticks "Remove duplicate addresses")</li> 
         <li>Step 3: Click Extract.</li> 
         <li>Then the duplicates (possibly with false positives, in particular for references with an archive-url) will be listed, and must be manually merged.</li>
        </ul></li>
      </ul> 
      <h4><span class="mw-headline" id="Short_citations">Short citations <span id="Shortened_footnotes"></span></span></h4> 
      <p><span class="anchor" id="CITESHORT"></span><span class="anchor" id="SFN"></span>
       <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855"></p>
      <div class="module-shortcutanchordiv">
       <span id="WP:CITESHORT"></span><span id="WP:SFN"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcuts</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:CITESHORT&amp;redirect=no">WP:CITESHORT</a></span></li>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:SFN&amp;redirect=no">WP:SFN</a></span></li>
       </ul>
      </div> 
      <div role="note" class="hatnote navigation-not-searchable">
       Main page: <a href="/wiki/Help:Shortened_footnotes" title="Help:Shortened footnotes">Help:Shortened footnotes</a>
      </div> 
      <p>Some Wikipedia articles use <i>short citations</i>, giving summary information about the source together with a page number, as in <code class="nowrap" style="">&lt;ref&gt;Smith 2010, p. 1.&lt;/ref&gt;</code>. These are used together with <i>full citations</i>, which give full details of the sources, but without page numbers, and are listed in a separate "References" section. </p>
      <p>Forms of short citations used include author-date referencing (<a href="/wiki/APA_style" title="APA style">APA style</a>, <a href="/wiki/Harvard_style" class="mw-redirect" title="Harvard style">Harvard style</a>, or <a href="/wiki/The_Chicago_Manual_of_Style" title="The Chicago Manual of Style">Chicago style</a>), and author-title or author-page referencing (<a href="/wiki/The_MLA_Style_Manual" class="mw-redirect" title="The MLA Style Manual">MLA style</a> or Chicago style). As before, the list of footnotes is automatically generated in a "Notes" or "Footnotes" section, which immediately precedes the "References" section containing the full citations to the source. Short citations can be written manually, or by using the <a href="/wiki/Template:Harvard_citation_documentation#Shortened_footnote" title="Template:Harvard citation documentation"><code>{{sfn}}</code> or <code>{{harvnb}}</code></a> templates. (Note that templates should not be added without consensus to an article that already uses a consistent referencing style.) The short citations and full citations may be linked so that the reader can click on the short note to find full information about the source. See the <a href="/wiki/Template:Harvard_citation_documentation" title="Template:Harvard citation documentation">template documentation</a> for details and solutions to common problems. For variations with and without templates, see <a href="/wiki/Wikipedia:Citing_sources/Further_considerations#Wikilinks_to_full_references" title="Wikipedia:Citing sources/Further considerations">wikilinks to full references</a>. For a set of realistic examples, see <a href="/wiki/Wikipedia:Citing_sources/Example_edits_for_different_methods#Shortened_notes" title="Wikipedia:Citing sources/Example edits for different methods">these</a>. </p>
      <p>This is how short citations look in the edit box: </p> 
      <pre>The Sun is pretty big,<span style="color:black;"><b>&lt;ref&gt;</b>Miller 2005, p. 23.<b>&lt;/ref&gt;</b></span> but the Moon is not so big.<span style="color:black;"><b>&lt;ref&gt;</b>Brown 2006, p. 46.<b>&lt;/ref&gt;</b></span> The Sun is also quite hot.<span style="color:black;"><b>&lt;ref&gt;</b>Miller 2005, p. 34.<b>&lt;/ref&gt;</b></span>

<span style="color:#666;">== Notes ==</span>
<span style="color:black;"><b>{{reflist}}</b></span>

<span style="color:#666;">== References ==</span>
* <span style="color:black;"><b>Brown, Rebecca (2006). "Size of the Moon", ''Scientific American'', 51 (78)</b></span>.
* <span style="color:black;"><b>Miller, Edward (2005). ''The Sun''. Academic Press</b></span>.
</pre> 
      <p>This is how they look in the article:<span class="anchor" id="refnote"></span> </p> 
      <blockquote style="background:white; padding:1em; border:1px solid #999;"> 
       <p>The Sun is pretty big,<sup id="nbFoot01b" class="reference"><a href="#noteFoot01b">[1]</a></sup> but the Moon is not so big.<sup id="nbFoot02b" class="reference"><a href="#noteFoot02b">[2]</a></sup> The Sun is also quite hot.<sup id="nbFoot03b" class="reference"><a href="#noteFoot03b">[3]</a></sup><br><br> <span style="font-size:medium">Notes</span> </p> 
       <hr> 
       <ol class="references"> 
        <li id="noteFoot01b"><b><a href="#nbFoot01b">^</a></b> Miller 2005, p. 23.</li> 
        <li id="noteFoot02b"><b><a href="#nbFoot02b">^</a></b> Brown 2006, p. 46.</li> 
        <li id="noteFoot03b"><b><a href="#nbFoot03b">^</a></b> Miller 2005, p. 34.</li> 
       </ol>
       <p><br> </p>
       <p><span style="font-size:medium">References</span> </p> 
       <hr> 
       <ul>
        <li>Brown, Rebecca (2006). "Size of the Moon", <i>Scientific American</i>, 51 (78).</li> 
        <li>Miller, Edward (2005). <i>The Sun</i>. Academic Press.</li>
       </ul> 
      </blockquote> 
      <p>Shortened notes using titles rather than publication dates would look like this in the article: </p> 
      <blockquote style="background:white; padding:1em; border:1px solid #999;"> 
       <p><span style="font-size:medium">Notes</span> </p> 
       <hr> 
       <ol class="references"> 
        <li id="noteFoot01b"><b><a href="#nbFoot01b">^</a></b> Miller, <i>The Sun</i>, p. 23.</li> 
        <li id="noteFoot02b"><b><a href="#nbFoot02b">^</a></b> Brown, "Size of the Moon", p. 46.</li> 
        <li id="noteFoot03b"><b><a href="#nbFoot03b">^</a></b> Miller, <i>The Sun</i>, p. 34.</li> 
       </ol> 
      </blockquote> 
      <p>When using manual links it is easy to introduce errors such as duplicate anchors and unused references. The script <a href="/wiki/User:Ucucha/HarvErrors" title="User:Ucucha/HarvErrors">User:Ucucha/HarvErrors</a> will show many related errors. Duplicate anchors may be found by using the <a href="/wiki/W3C_Markup_Validation_Service" title="W3C Markup Validation Service">W3C Markup Validation Service</a>. </p>

      <h2><span class="mw-headline" id="Citation_style">Citation style <span id="Citation_styles"></span></span></h2> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:CITESTYLE"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:CITESTYLE&amp;redirect=no">WP:CITESTYLE</a></span></li>
       </ul>
      </div> 
      <p>While citations should aim to provide the information listed above, Wikipedia does not have a single <a href="/wiki/Style_guide" title="Style guide">house style</a>, though citations within any given article should follow a consistent style. A number of citation styles exist including those described in the Wikipedia articles for <a href="/wiki/Citation" title="Citation">Citation</a>, <a href="/wiki/APA_style" title="APA style">APA style</a>, <a href="/wiki/ASA_style" title="ASA style">ASA style</a>, <a href="/wiki/MLA_style" class="mw-redirect" title="MLA style">MLA style</a>, <i><a href="/wiki/The_Chicago_Manual_of_Style" title="The Chicago Manual of Style">The Chicago Manual of Style</a></i>, <a href="/wiki/Author-date_referencing" class="mw-redirect" title="Author-date referencing">Author-date referencing</a>, the <a href="/wiki/Vancouver_system" title="Vancouver system">Vancouver system</a> and <i><a href="/wiki/Bluebook" title="Bluebook">Bluebook</a></i>. </p>
      <p>Although nearly any consistent style may be used, <em>avoid</em> all-numeric date formats other than YYYY-MM-DD, because of the ambiguity concerning which number is the month and which the day. For example, <span class="example" style="font-family: Georgia, 'DejaVu Serif', serif; color: #006400;">2002-06-11</span> may be used, but not <span class="example deprecated-content example-bad" style="font-family: Georgia, 'DejaVu Serif', serif; color: #8B0000;">11/06/2002</span>. The YYYY-MM-DD format should in any case be limited to <a href="/wiki/Gregorian_calendar" title="Gregorian calendar">Gregorian calendar</a> dates where the year is after 1582. Because it could easily be confused with a range of years, the format YYYY-MM (for example: <span class="example deprecated-content example-bad" style="font-family: Georgia, 'DejaVu Serif', serif; color: #8B0000;">2002-06</span>) is not used. </p>
      <p>For more information on the capitalization of cited works, see <a href="/wiki/Wikipedia:Manual_of_Style/Capital_letters#All_caps_and_small_caps" title="Wikipedia:Manual of Style/Capital letters">Wikipedia:Manual of Style/Capital letters §&nbsp;All caps and small caps</a>. </p> 
      <h3><span class="mw-headline" id="Variation_in_citation_methods">Variation in citation methods <span id="Style_variation_and_consistency"></span></span></h3> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:CITEVAR"></span><span id="WP:WHENINROME"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcuts</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:CITEVAR&amp;redirect=no">WP:CITEVAR</a></span></li>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:WHENINROME&amp;redirect=no">WP:WHENINROME</a></span></li>
       </ul>
      </div> 
      <p>Editors should not attempt to change an article's established citation style merely on the grounds of personal preference, to make it match other articles, or without first seeking <a href="/wiki/Wikipedia:Consensus" title="Wikipedia:Consensus">consensus</a> for the change. The arbitration committee <a href="/wiki/Wikipedia:Requests_for_arbitration/Sortan#Preferred_styles" title="Wikipedia:Requests for arbitration/Sortan">ruled in 2006</a>: </p> 
      <blockquote>
       <p>Wikipedia does not mandate styles in many different areas; these include (but are not limited to) American vs. British spelling, date formats, and citation style. Where Wikipedia does not mandate a specific style, editors should not attempt to convert Wikipedia to their own preferred style, nor should they edit articles for the sole purpose of converting them to their preferred style, or removing examples of, or references to, styles which they dislike.</p>
      </blockquote> 
      <p>As with <a href="/wiki/Wikipedia:ENGVAR" class="mw-redirect" title="Wikipedia:ENGVAR">spelling differences</a>, it is normal practice to defer to the style used by the first major contributor or adopted by the consensus of editors already working on the page, unless a change in consensus has been achieved. If the article you are editing is already using a particular citation style, you should follow it; if you believe it is inappropriate for the needs of the article, seek consensus for a change on the talk page. If you are the first contributor to add citations to an article, you may choose whichever style you think best for the article. However, <a href="/wiki/Special:PermanentLink/976895063#Deprecate_parenthetical_citations" title="Special:PermanentLink/976895063">as of 5 September 2020</a>, inline <a href="#Parenthetical_referencing">parenthetical referencing</a> is a deprecated citation style on English-language Wikipedia. </p>
      <p>If all or most of the citations in an article consist of bare URLs, or otherwise fail to provide needed bibliographic data – such as the name of the source, the title of the article or web page consulted, the author (if known), the publication date (if known), and the page numbers (where relevant) – then that would not count as a "consistent citation style" and can be changed freely to insert such data. The data provided should be sufficient to uniquely identify the source, allow readers to find it, and allow readers to initially evaluate it without retrieving it. </p> 
      <h4><span class="mw-headline" id="To_be_avoided">To be avoided</span></h4> 
      <p>When an article is already consistent, avoid: </p> 
      <ul>
       <li>switching between major citation styles or replacing the preferred style of one academic discipline with another's – except when moving away from <a href="/wiki/Wikipedia:PAREN" class="mw-redirect" title="Wikipedia:PAREN">parenthetical referencing</a>;</li> 
       <li>adding citation templates to an article that already uses a consistent system without templates, or removing citation templates from an article that uses them consistently;</li> 
       <li>changing where the references are defined, e.g., moving <a href="/wiki/Wikipedia:LDRHOW" class="mw-redirect" title="Wikipedia:LDRHOW">reference definitions in the reflist</a> to the prose, or moving reference definitions from the prose into the reflist.</li>
      </ul> 
      <h4><span class="mw-headline" id="Generally_considered_helpful">Generally considered helpful</span></h4> 
      <p>The following are standard practice: </p> 
      <ul>
       <li>improving existing citations by adding missing information, such as by replacing bare URLs with full bibliographic citations: an improvement because it aids <a href="/wiki/Wikipedia:Verifiabilty" class="mw-redirect" title="Wikipedia:Verifiabilty">verifiability</a>, and fights <a href="/wiki/Wikipedia:Link_rot" title="Wikipedia:Link rot">link rot</a>;</li> 
       <li>replacing some or all general references with inline citations: an improvement because it provides more verifiable information to the reader, and helps maintain text–source integrity;</li> 
       <li>imposing one style on an article with inconsistent citation styles (e.g., some of the citations in footnotes and others as parenthetical references): an improvement because it makes the citations easier to understand and edit;</li> 
       <li>fixing errors in citation coding, including incorrectly used template parameters, and <code>&lt;ref&gt;</code> markup problems: an improvement because it helps the citations to be parsed correctly;</li> 
       <li>combining duplicate citations 
        <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1015928993"><span role="note" class="hatnote navigation-not-searchable crossreference">(see <a href="#Duplicate_citations">§&nbsp;Duplicate citations</a>, above)</span>.</li> 
       <li>converting <a href="/wiki/Wikipedia:PAREN" class="mw-redirect" title="Wikipedia:PAREN">parenthetical referencing</a> to an acceptable referencing style.</li>
      </ul> 
      <h2><span class="mw-headline" id="Handling_links_in_citations">Handling links in citations</span></h2> 
      <p>As noted above under <a href="#What_information_to_include">"What information to include"</a>, it is helpful to include hyperlinks to source material, when available. Here we note some issues concerning these links. </p> 
      <h3><span class="mw-headline" id="Avoid_embedded_links">Avoid embedded links</span></h3> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:CS:EMBED"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:CS:EMBED&amp;redirect=no">WP:CS:EMBED</a></span></li>
       </ul>
      </div> 
      <p>Embedded links to external websites should not be used as a form of inline citation, because they are highly susceptible to <a href="/wiki/Wikipedia:Linkrot" class="mw-redirect" title="Wikipedia:Linkrot">linkrot</a>. Wikipedia allowed this in its early years—for example by adding a link after a sentence, like this: [http://media.guardian.co.uk/site/story/0,14173,1601858,00.html], which is rendered as: <a rel="nofollow" class="external autonumber" href="http://media.guardian.co.uk/site/story/0,14173,1601858,00.html">[1]</a>. This is no longer recommended. Raw links are not recommended in lieu of properly written out citations, even if placed between ref tags, like this <code>&lt;ref&gt;[http://media.guardian.co.uk/site/story/0,14173,1601858,00.html]&lt;/ref&gt;</code>. Since any citation that accurately identifies the source is better than none, do not revert the good-faith addition of partial citations. They should be considered temporary, and replaced with more complete, properly formatted citations as soon as possible. </p>
      <p>Embedded links should never be used to place <a href="/wiki/Wikipedia:EL" class="mw-redirect" title="Wikipedia:EL">external links</a> in the content of an article, like this: <span class="example deprecated-content example-bad" style="font-family: Georgia, 'DejaVu Serif', serif; color: #8B0000;">"<a rel="nofollow" class="external text" href="http://apple.com"><span class="example deprecated-content example-bad" style="font-family: Georgia, 'DejaVu Serif', serif; color: #8B0000;">Apple, Inc.</span></a> announced their latest product ..."</span>. </p> 
      <h3><span class="mw-headline" id="Convenience_links">Convenience links</span></h3> 
      <div role="note" class="hatnote navigation-not-searchable">
       Further information: <a href="/wiki/Wikipedia:Copyrights#Linking_to_copyrighted_works" title="Wikipedia:Copyrights">Wikipedia:Copyrights §&nbsp;Linking to copyrighted works</a>, and <a href="/wiki/Help:Citation_Style_1#Online_sources" title="Help:Citation Style 1">Help:Citation Style 1 §&nbsp;Online sources</a>
      </div> 
      <p>A <i>convenience link</i> is a link to a copy of your source on a web page provided by someone other than the original publisher or author. For example, a copy of a newspaper article no longer available on the newspaper's website may be hosted elsewhere. When offering convenience links, it is important to be reasonably certain that the convenience copy is a true copy of the original, without any changes or inappropriate commentary, and that it does not infringe the original publisher's copyright. Accuracy can be assumed when the hosting website appears reliable. </p>
      <p>For <a href="/wiki/Scholarly_communication" title="Scholarly communication">academic sources</a>, the convenience link is typically a reprint provided by an <a href="/wiki/Open-access_repository" title="Open-access repository">open-access repository</a>, such as the author's university's library or <a href="/wiki/Institutional_repository" title="Institutional repository">institutional repository</a>. Such <a href="/wiki/Green_open_access" class="mw-redirect" title="Green open access">green open access</a> links are generally preferable to <a href="/wiki/Paywall" title="Paywall">paywalled</a> or otherwise commercial and <a href="/wiki/Free_content" title="Free content">unfree</a> sources. </p>
      <p>Where several sites host a copy of the material, the site selected as the convenience link should be the one whose general content appears most in line with <a href="/wiki/Wikipedia:Neutral_point_of_view" title="Wikipedia:Neutral point of view">Wikipedia:Neutral point of view</a> and <a href="/wiki/Wikipedia:Verifiability" title="Wikipedia:Verifiability">Wikipedia:Verifiability</a>. </p> 
      <h3><span class="mw-headline" id="Indicating_availability">Indicating availability</span></h3> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:INDICATEAVAIL"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:INDICATEAVAIL&amp;redirect=no">WP:INDICATEAVAIL</a></span></li>
       </ul>
      </div> 
      <p>If your source is not available online, it should be available in reputable libraries, archives, or collections. If a citation without an external link is challenged as unavailable, any of the following is sufficient to show the material to be reasonably available (though not necessarily <a href="/wiki/Wikipedia:Verifiability#Reliable_sources" title="Wikipedia:Verifiability">reliable</a>): providing an <a href="/wiki/ISBN" class="mw-redirect" title="ISBN">ISBN</a> or <a href="/wiki/OCLC" title="OCLC">OCLC</a> number; linking to an established Wikipedia article about the source (the work, its author, or its publisher); or directly quoting the material on the talk page, <a href="/wiki/Wikipedia:COPYVIO" class="mw-redirect" title="Wikipedia:COPYVIO">briefly</a> and in context. </p> 
      <h3><span class="mw-headline" id="Links_to_sources">Links to sources</span></h3> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:SOURCELINKS"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:SOURCELINKS&amp;redirect=no">WP:SOURCELINKS</a></span></li>
       </ul>
      </div> 
      <p>For a source available in <a href="/wiki/Hardcopy" class="mw-redirect" title="Hardcopy">hardcopy</a>, <a href="/wiki/Microform" title="Microform">microform</a>, and/or <a href="/wiki/Online_and_offline" title="Online and offline">online</a>, omit, in most cases, which one you read. While it is useful to cite author, title, edition (1st, 2nd, etc.), and similar information, it generally is not important to cite a database such as <a href="/wiki/ProQuest" title="ProQuest">ProQuest</a>, <a href="/wiki/EBSCOhost" class="mw-redirect" title="EBSCOhost">EBSCOhost</a>, or <a href="/wiki/JSTOR" title="JSTOR">JSTOR</a> (see the <a href="/wiki/List_of_academic_databases_and_search_engines" title="List of academic databases and search engines">list of academic databases and search engines</a>) or to link to such a database requiring a subscription or a third party's login. The basic bibliographic information you provide should be enough to search for the source in any of these databases that have the source. Don't add a URL that has a part of a password embedded in the URL. However, you may provide the <a href="/wiki/Wikipedia:Digital_Object_Identifier" title="Wikipedia:Digital Object Identifier">DOI</a>, <a href="/wiki/Wikipedia:ISBN" title="Wikipedia:ISBN">ISBN</a>, or another uniform identifier, if available. If the publisher offers a link to the source or its abstract that does not require a payment or a third party's login for access, you may provide the URL for that link. If the source only exists online, give the link even if access is restricted (see <a href="/wiki/Wikipedia:PAYWALL" class="mw-redirect" title="Wikipedia:PAYWALL">WP:PAYWALL</a>). </p>

      </div>

       </ul>
      </div> 
      <p>To help prevent <a href="/wiki/Wikipedia:Dead_external_links" class="mw-redirect" title="Wikipedia:Dead external links">dead links</a>, persistent identifiers are available for some sources. Some journal articles have a <a href="/wiki/Digital_object_identifier" title="Digital object identifier">digital object identifier</a> (DOI); some online newspapers and blogs, and also Wikipedia, have <a href="/wiki/Permalink" title="Permalink">permalinks</a> that are stable. When permanent links aren't available, consider archiving the referenced document when writing the article; on-demand web archiving services such as the <a href="/wiki/Wayback_Machine" title="Wayback Machine">Wayback Machine</a> (<a rel="nofollow" class="external free" href="https://web.archive.org/save">https://web.archive.org/save</a>) or <a href="/wiki/Archive.today" title="Archive.today">archive.today</a> (<a rel="nofollow" class="external free" href="https://archive.today">https://archive.today</a>) are fairly easy to use (see <a href="/wiki/Wikipedia:Citing_sources/Further_considerations#Pre-emptive_archiving" title="Wikipedia:Citing sources/Further considerations">pre-emptive archiving</a>). </p>
      <p><b>Do not delete a citation merely because the URL is not working</b>. Dead links should be repaired or replaced if possible. If you encounter a dead URL being used as a reliable source to support article content, follow these steps prior to deleting it: </p> 
      <ol>
       <li><b>Confirm status</b>: First, check the link to confirm that it is dead and not temporarily down. Search the website to see whether it has been rearranged. The online service <a rel="nofollow" class="external text" href="https://www.isitdownrightnow.com/">"Is it down right now?"</a> can help to determine if a <i>site</i> is down, and any information known.</li> 
       <li><b>Check for a changed URL on the same Web site</b>: Pages are frequently moved to different location on the same site as they become archive content rather than news. The site's error page may have a "Search" box; alternatively, in both the Google and DuckDuckGo search engines – among others – the keyterm "site:" can be used. For instance: <i>site:en.wikipedia.org "New Zealand police vehicle markings and livery"</i>.</li>

Examples:

         <li><a href="/wiki/Archive.today" title="Archive.today">archive.today</a> See <a href="/wiki/Wikipedia:Using_archive.today" class="mw-redirect" title="Wikipedia:Using archive.today">Wikipedia:Using archive.today</a></li> 
         <li><a href="/wiki/WebCite" title="WebCite">WebCite</a> has billions of archived web pages. See <a href="/wiki/Wikipedia:Using_WebCite" class="mw-redirect" title="Wikipedia:Using WebCite">Wikipedia:Using WebCite</a>.</li> 
         <li>The <a href="/wiki/UK_Government_Web_Archive" title="UK Government Web Archive">UK Government Web Archive</a> (<a rel="nofollow" class="external free" href="https://www.nationalarchives.gov.uk/webarchive/">https://www.nationalarchives.gov.uk/webarchive/</a>) preserves 1500 UK central government websites.</li> 
         <li>The <a rel="nofollow" class="external text" href="https://www.webarchive.org.uk/mementos/search">Mementos interface</a> allows you to search multiple <a href="/wiki/Web_archiving" title="Web archiving">archiving services</a> with a single request using the <a href="/wiki/Memento_Project" title="Memento Project">Memento</a> protocol. If the URL contains a "?" it is unlikely to work properly. When entering the URL into the Mementos interface manually, the most common change needed is to change "?" to "%3F". While making only this change will not be sufficient in all cases, it will work most of the time. The bookmarklet in the table below will properly encode URLs such that searches will work.</li>
        </ul></li>
      </ol>

         </dd>

      <h2><span id="Text.E2.80.93source_integrity"></span><span class="mw-headline" id="Text–source_integrity">Text–source integrity</span></h2> 
      <p><span class="anchor" id="INTEGRITY"></span>
       <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855"></p>
      <div class="module-shortcutanchordiv">
       <span id="WP:INTEGRITY"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:INTEGRITY&amp;redirect=no">WP:INTEGRITY</a></span></li>
       </ul>
      </div> 
      <div role="note" class="hatnote navigation-not-searchable">
       "WP:INTEGRITY" redirects here. For WikiProject Integrity, see <a href="/wiki/Wikipedia:WPINTEGRITY" class="mw-redirect" title="Wikipedia:WPINTEGRITY">WP:WPINTEGRITY</a>.
      </div> 
      <p>When using inline citations, it is important to maintain text–source integrity. The point of an inline citation is to allow readers and other editors to check that the material is sourced; that point is lost if the citation is not clearly placed. The distance between material and its source is a matter of editorial judgment, but adding text without clearly placing its source may lead to allegations of <a href="/wiki/Wikipedia:NOR" class="mw-redirect" title="Wikipedia:NOR">original research</a>, of violations of the <a href="/wiki/Wikipedia:V" class="mw-redirect" title="Wikipedia:V">sourcing policy</a>, and even of <a href="/wiki/Wikipedia:PLAGFORM" class="mw-redirect" title="Wikipedia:PLAGFORM">plagiarism</a>. </p> 
      <h3><span class="mw-headline" id="Keeping_citations_close">Keeping citations close</span></h3> 
      <p>Editors should exercise caution when rearranging or inserting material to ensure that text–source relationships are maintained. References need not be moved solely to maintain the chronological order of footnotes as they appear in the article, and should not be moved if doing so might break the text–source relationship. </p>
      <p>If a sentence or paragraph is footnoted with a source, adding new material that is not supported by the existing source to the sentence/paragraph, without a source for the new text, is highly misleading if placed to appear that the cited source supports it. When new text is inserted into a paragraph, make sure it is supported by the existing or a new source. For example, when editing text originally reading </p> 
      <blockquote style="background:white; padding:1em; border:1px solid #999;"> 
       <p>The sun is pretty big.<sup id="nbFoot01a" class="reference"><a href="#noteFoot01a">[1]</a></sup> <br><br> <span style="font-size:medium">Notes</span> </p> 
       <hr> 
       <ol class="references"> 
        <li id="noteFoot01a"><b><a href="#nbFoot01a">^</a></b> Miller, Edward. <i>The Sun</i>. Academic Press, 2005, p. 1.</li> 
       </ol> 
      </blockquote> 
      <p>an edit that does not imply that the new material is sourced by the same reference is </p> 
      <blockquote style="background:white; padding:1em; border:1px solid #999;"> 
       <p>The sun is pretty big.<sup id="nbFoot01a" class="reference"><a href="#noteFoot01a">[1]</a></sup> The sun is also quite hot.<sup id="nbFoot03a" class="reference"><a href="#noteFoot03a">[2]</a></sup> <br><br> <span style="font-size:medium">Notes</span> </p> 
       <hr> 
       <ol class="references"> 
        <li id="noteFoot01a"><b><a href="#nbFoot01a">^</a></b> Miller, Edward. <i>The Sun</i>. Academic Press, 2005, p. 1.</li> 
        <li id="noteFoot03a"><b><a href="#nbFoot03a">^</a></b> Smith, John. <i>The Sun's Heat</i>. Academic Press, 2005, p. 2.</li> 
       </ol> 
      </blockquote> 
      <p>Do not add other facts or assertions into a fully cited paragraph or sentence: </p> 
      <blockquote style="background:white; padding:1em; border:1px solid #999;">
       <p><img alt="?" src="//upload.wikimedia.org/wikipedia/commons/thumb/a/a2/X_mark.svg/20px-X_mark.svg.png" decoding="async" width="20" height="23" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/a/a2/X_mark.svg/30px-X_mark.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/a/a2/X_mark.svg/40px-X_mark.svg.png 2x" data-file-width="525" data-file-height="600"><span style="display:none">N</span> </p>
       <p>The sun is pretty big, but the moon is not so big.<sup id="nbFoot01a" class="reference"><a href="#noteFoot01a">[1]</a></sup> The sun is also quite hot.<sup id="nbFoot03a" class="reference"><a href="#noteFoot03a">[2]</a></sup> <br><br> <span style="font-size:medium">Notes</span> </p> 
       <hr> 
       <ol class="references"> 
        <li id="noteFoot01a"><b><a href="#nbFoot01a">^</a></b> Miller, Edward. <i>The Sun</i>. Academic Press, 2005, p. 1.</li> 
        <li id="noteFoot03a"><b><a href="#nbFoot03a">^</a></b> Smith, John. <i>The Sun's Heat</i>. Academic Press, 2005, p. 2.</li> 
       </ol> 
      </blockquote> 
      <p>Include a source to support the new information. There are several ways to write this, including: </p> 
      <blockquote style="background:white; padding:1em; border:1px solid #999;">
       <p><img alt="check" src="//upload.wikimedia.org/wikipedia/en/thumb/f/fb/Yes_check.svg/20px-Yes_check.svg.png" decoding="async" width="20" height="20" srcset="//upload.wikimedia.org/wikipedia/en/thumb/f/fb/Yes_check.svg/30px-Yes_check.svg.png 1.5x, //upload.wikimedia.org/wikipedia/en/thumb/f/fb/Yes_check.svg/40px-Yes_check.svg.png 2x" data-file-width="600" data-file-height="600"><span style="display:none">Y</span> </p>
       <p>The sun is pretty big,<sup id="nbFoot01a" class="reference"><a href="#noteFoot01a">[1]</a></sup> but the moon is not so big.<sup id="nbFoot02a" class="reference"><a href="#noteFoot02a">[2]</a></sup> The sun is also quite hot.<sup id="nbFoot03a" class="reference"><a href="#noteFoot03a">[3]</a></sup> <br><br> <span style="font-size:medium">Notes</span> </p> 
       <hr> 
       <ol class="references"> 
        <li id="noteFoot01a"><b><a href="#nbFoot01a">^</a></b> Miller, Edward. <i>The Sun</i>. Academic Press, 2005, p. 1.</li> 
        <li id="noteFoot02a"><b><a href="#nbFoot02a">^</a></b> Brown, Rebecca. "Size of the Moon", <i>Scientific American</i>, 51 (78): 46.</li> 
        <li id="noteFoot03a"><b><a href="#nbFoot03a">^</a></b> Smith, John. <i>The Sun's Heat</i>. Academic Press, 2005, p. 2.</li> 
       </ol> 
      </blockquote> 
      <h3><span class="mw-headline" id="Bundling_citations">Bundling citations</span></h3> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:CITEBUNDLE"></span><span id="WP:BUNDLING"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcuts</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:CITEBUNDLE&amp;redirect=no">WP:CITEBUNDLE</a></span></li>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:BUNDLING&amp;redirect=no">WP:BUNDLING</a></span></li>
       </ul>
      </div> 
      <div role="note" class="hatnote navigation-not-searchable">
       Main page: <a href="/wiki/Help:Citation_merging" title="Help:Citation merging">Help:Citation merging</a>
      </div> 
      <div role="note" class="hatnote navigation-not-searchable">
       Further information: <a href="/wiki/Wikipedia:Citation_overkill" title="Wikipedia:Citation overkill">Wikipedia:Citation overkill</a>
      </div> 
      <p>Sometimes the article is more readable if multiple citations are bundled into a single footnote. For example, when there are multiple sources for a given sentence, and each source applies to the entire sentence, the sources can be placed at the end of the sentence, like this.<sup class="reference nowrap ltr" style="color:#002bb8;">[4]</sup><sup class="reference nowrap ltr" style="color:#002bb8;">[5]</sup><sup class="reference nowrap ltr" style="color:#002bb8;">[6]</sup><sup class="reference nowrap ltr" style="color:#002bb8;">[7]</sup> Or they can be bundled into one footnote at the end of the sentence or paragraph, like this.<sup class="reference nowrap ltr" style="color:#002bb8;">[4]</sup> </p>
      <p>Bundling is also useful if the sources each support a different portion of the preceding text, or if the sources all support the same text. Bundling has several advantages: </p> 
      <dl>
       <dd>
        <ul>
         <li>It helps readers and other editors see at a glance which source supports which point, maintaining <a href="#Text–source_integrity">text–source integrity</a>;</li> 
         <li>It avoids the visual clutter of multiple clickable footnotes inside a sentence or paragraph;</li> 
         <li>It avoids the confusion of having multiple sources listed separately after sentences, with no indication of which source to check for each part of the text, such as this.<sup class="reference nowrap ltr" style="color:#002bb8;">[1]</sup><sup class="reference nowrap ltr" style="color:#002bb8;">[2]</sup><sup class="reference nowrap ltr" style="color:#002bb8;">[3]</sup><sup class="reference nowrap ltr" style="color:#002bb8;">[4]</sup></li> 
         <li>It makes it less likely that inline citations will be moved inadvertently when text is re-arranged, because the footnote states clearly which source supports which point.</li>
        </ul>
       </dd>
      </dl> 
      <p>To concatenate multiple citations for the same content, semicolons (or another character appropriate to the article's style) can be used. Alternatively, the template {{<a href="/wiki/Template:Multiref" title="Template:Multiref">multiref</a>}} may be used. </p> 
      <blockquote style="background:white; padding:1em; border:1px solid #999;"> 
       <p>The sun is pretty big, bright and hot.<sup id="nbFoot04a" class="reference"><a href="#noteFoot04a">[1]</a></sup> <br><br> <span style="font-size:medium">Notes</span> </p> 
       <hr> 
       <ol class="references"> <b>Semicolons</b> 
        <li id="noteFoot04a" value="1"> <b><a href="#nbFoot04a">^</a></b> Miller, Edward. <i>The Sun</i>. Academic Press, 2005, p. 1; Brown, Rebecca. "The Solar System", <i>Scientific American</i>, 51 (78): 46; Smith, John. <i>The Earth's Star</i>. Academic Press, 2005, p. 2</li> 
       </ol> 
      </blockquote> 
      <p>For multiple citations in a single footnote, each in reference to specific statements, there are several layouts available, as illustrated below. Within a given article only a single layout should be used. </p> 
      <blockquote style="background:white; padding:1em; border:1px solid #999;"> 
       <p>The sun is pretty big, but the moon is not so big. The sun is also quite hot.<sup id="nbFoot01a" class="reference"><a href="#noteFoot01a">[1]</a></sup> <br><br> <span style="font-size:medium">Notes</span> </p> 
       <hr> 
       <ol class="references"> <b>Bullets</b> 
        <li id="noteFoot01a" value="1"> <b><a href="#nbFoot01a">^</a></b> 
         <ul>
          <li>For the sun's size, see Miller, Edward. <i>The Sun</i>. Academic Press, 2005, p. 1.</li> 
          <li>For the moon's size, see Brown, Rebecca. "Size of the Moon", <i>Scientific American</i>, 51 (78): 46.</li> 
          <li>For the sun's heat, see Smith, John. <i>The Sun's Heat</i>. Academic Press, 2005, p. 2.</li>
         </ul> <b>Line breaks</b> </li>
        <li id="noteFoot02a" value="1"> <b><a href="#nbFoot02a">^</a></b> For the sun's size, see Miller, Edward. <i>The Sun</i>. Academic Press, 2005, p. 1. <br>For the moon's size, see Brown, Rebecca. "Size of the Moon", <i>Scientific American</i>, 51 (78): 46. <br>For the sun's heat, see Smith, John. <i>The Sun's Heat</i>. Academic Press, 2005, p. 2.</li> <b>Paragraph</b> 
        <li id="noteFoot03a" value="1"> <b><a href="#nbFoot03a">^</a></b> For the sun's size, see Miller, Edward. <i>The Sun</i>. Academic Press, 2005, p. 1. For the moon's size, see Brown, Rebecca. "Size of the Moon", <i>Scientific American</i>, 51 (78): 46. For the sun's heat, see Smith, John. <i>The Sun's Heat</i>. Academic Press, 2005, p. 2.</li> 
       </ol> 
      </blockquote> 
      <p>However, using line breaks to separate list items breaches <a href="/wiki/Wikipedia:Accessibility#Nobreaks" class="mw-redirect" title="Wikipedia:Accessibility">WP:Accessibility §&nbsp;Nobreaks</a>: <style data-mw-deduplicate="TemplateStyles:r989825325">.mw-parser-output .inline-quote-talk{font-family:Georgia,"DejaVu Serif",serif;color:#008560;quotes:none}.mw-parser-output .inline-quote-talk-italic{font-family:inherit;font-style:italic}.mw-parser-output .inline-quote-talk-marks{quotes:"\"""\""}</style><q class="inline-quote-talk">"Do not separate list items with line breaks (<code class="nowrap" style="">&lt;br&gt;</code>). Use {{<a href="/wiki/Template:Plainlist" title="Template:Plainlist">plainlist</a>}} / {{<a href="/wiki/Template:Unbulleted_list" title="Template:Unbulleted list">unbulleted list</a>}} ..."</q>. </p> 
      <h2><span class="mw-headline" id="In-text_attribution">In-text attribution</span></h2> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:INTEXT"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:INTEXT&amp;redirect=no">WP:INTEXT</a></span></li>
       </ul>
      </div> 
      <div role="note" class="hatnote navigation-not-searchable">
       Further information: <a href="/wiki/Wikipedia:Neutral_point_of_view#Attributing_and_specifying_biased_statements" title="Wikipedia:Neutral point of view">Wikipedia:Neutral point of view §&nbsp;Attributing and specifying biased statements</a>, and <a href="/wiki/Wikipedia:Manual_of_Style#Point_of_view" title="Wikipedia:Manual of Style">Wikipedia:Manual of Style §&nbsp;Point of view</a>
      </div> 
      <p><b>In-text attribution</b> is the attribution inside a sentence of material to its source, in addition to an <a href="/wiki/Wikipedia:INCITE" class="mw-redirect" title="Wikipedia:INCITE">inline citation</a> after the sentence. In-text attribution should be used with <a href="/wiki/Direct_speech" title="Direct speech">direct speech</a> (a source's words between quotation marks or as a <a href="/wiki/Block_quotation" title="Block quotation">block quotation</a>); <a href="/wiki/Indirect_speech" title="Indirect speech">indirect speech</a> (a source's words modified without quotation marks); and <a href="/wiki/Wikipedia:Close_paraphrasing" title="Wikipedia:Close paraphrasing">close paraphrasing</a>. It can also be used when loosely summarizing a source's position in your own words, and it should always be used for <a href="/wiki/Wikipedia:ATTRIBUTEPOV" class="mw-redirect" title="Wikipedia:ATTRIBUTEPOV">biased statements of opinion</a>. It avoids inadvertent plagiarism and helps the reader see where a position is coming from. An inline citation should follow the attribution, usually at the end of the sentence or paragraph in question. </p>
      <p>For example: </p> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r996844942">
      <blockquote class="templatequote">
       <p><img alt="?" src="//upload.wikimedia.org/wikipedia/commons/thumb/a/a2/X_mark.svg/20px-X_mark.svg.png" decoding="async" width="20" height="23" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/a/a2/X_mark.svg/30px-X_mark.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/a/a2/X_mark.svg/40px-X_mark.svg.png 2x" data-file-width="525" data-file-height="600"><span style="display:none">N</span> To reach fair decisions, parties must consider matters as if behind a <a href="/wiki/Veil_of_ignorance" class="mw-redirect" title="Veil of ignorance">veil of ignorance</a>.<sup class="reference nowrap ltr" style="color:#002bb8;">[2]</sup> </p>
      </blockquote> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r996844942">
      <blockquote class="templatequote">
       <p><img alt="check" src="//upload.wikimedia.org/wikipedia/en/thumb/f/fb/Yes_check.svg/20px-Yes_check.svg.png" decoding="async" width="20" height="20" srcset="//upload.wikimedia.org/wikipedia/en/thumb/f/fb/Yes_check.svg/30px-Yes_check.svg.png 1.5x, //upload.wikimedia.org/wikipedia/en/thumb/f/fb/Yes_check.svg/40px-Yes_check.svg.png 2x" data-file-width="600" data-file-height="600"><span style="display:none">Y</span> <a href="/wiki/John_Rawls" title="John Rawls">John Rawls</a> argues that, to reach fair decisions, parties must consider matters as if behind a <a href="/wiki/Veil_of_ignorance" class="mw-redirect" title="Veil of ignorance">veil of ignorance</a>.<sup class="reference nowrap ltr" style="color:#002bb8;">[2]</sup> </p>
      </blockquote> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r996844942">
      <blockquote class="templatequote">
       <p><img alt="check" src="//upload.wikimedia.org/wikipedia/en/thumb/f/fb/Yes_check.svg/20px-Yes_check.svg.png" decoding="async" width="20" height="20" srcset="//upload.wikimedia.org/wikipedia/en/thumb/f/fb/Yes_check.svg/30px-Yes_check.svg.png 1.5x, //upload.wikimedia.org/wikipedia/en/thumb/f/fb/Yes_check.svg/40px-Yes_check.svg.png 2x" data-file-width="600" data-file-height="600"><span style="display:none">Y</span> <a href="/wiki/John_Rawls" title="John Rawls">John Rawls</a> argues that, to reach fair decisions, parties must consider matters as if "situated behind a <a href="/wiki/Veil_of_ignorance" class="mw-redirect" title="Veil of ignorance">veil of ignorance</a>".<sup class="reference nowrap ltr" style="color:#002bb8;">[2]</sup> </p>
      </blockquote> 
      <p>When using in-text attribution, make sure it doesn't lead to an inadvertent <a href="/wiki/Wikipedia:UNDUE" class="mw-redirect" title="Wikipedia:UNDUE">neutrality</a> violation. For example, the following implies parity between the sources, without making clear that the position of Darwin is the <a href="/wiki/Wikipedia:WEIGHT" class="mw-redirect" title="Wikipedia:WEIGHT">majority view</a>: </p> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r996844942">
      <blockquote class="templatequote">
       <p><img alt="?" src="//upload.wikimedia.org/wikipedia/commons/thumb/a/a2/X_mark.svg/20px-X_mark.svg.png" decoding="async" width="20" height="23" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/a/a2/X_mark.svg/30px-X_mark.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/a/a2/X_mark.svg/40px-X_mark.svg.png 2x" data-file-width="525" data-file-height="600"><span style="display:none">N</span> <a href="/wiki/Charles_Darwin" title="Charles Darwin">Charles Darwin</a> says that human beings evolved through <a href="/wiki/Natural_selection" title="Natural selection">natural selection</a>, but John Smith writes that we arrived here in pods from Mars.</p>
      </blockquote> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r996844942">
      <blockquote class="templatequote">
       <p><img alt="check" src="//upload.wikimedia.org/wikipedia/en/thumb/f/fb/Yes_check.svg/20px-Yes_check.svg.png" decoding="async" width="20" height="20" srcset="//upload.wikimedia.org/wikipedia/en/thumb/f/fb/Yes_check.svg/30px-Yes_check.svg.png 1.5x, //upload.wikimedia.org/wikipedia/en/thumb/f/fb/Yes_check.svg/40px-Yes_check.svg.png 2x" data-file-width="600" data-file-height="600"><span style="display:none">Y</span> Humans evolved through <a href="/wiki/Natural_selection" title="Natural selection">natural selection</a>, as first explained in <a href="/wiki/Charles_Darwin" title="Charles Darwin">Charles Darwin</a>'s <i><a href="/wiki/The_Descent_of_Man,_and_Selection_in_Relation_to_Sex" title="The Descent of Man, and Selection in Relation to Sex">The Descent of Man, and Selection in Relation to Sex</a></i>.</p>
      </blockquote> 
      <p>Neutrality issues apart, there are other ways in-text attribution can mislead. The sentence below suggests <i>The New York Times</i> has alone made this important discovery: </p> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r996844942">
      <blockquote class="templatequote">
       <p><img alt="?" src="//upload.wikimedia.org/wikipedia/commons/thumb/a/a2/X_mark.svg/20px-X_mark.svg.png" decoding="async" width="20" height="23" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/a/a2/X_mark.svg/30px-X_mark.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/a/a2/X_mark.svg/40px-X_mark.svg.png 2x" data-file-width="525" data-file-height="600"><span style="display:none">N</span> According to <i>The New York Times</i>, the sun will set in the west this evening.</p>
      </blockquote> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r996844942">
      <blockquote class="templatequote">
       <p><img alt="check" src="//upload.wikimedia.org/wikipedia/en/thumb/f/fb/Yes_check.svg/20px-Yes_check.svg.png" decoding="async" width="20" height="20" srcset="//upload.wikimedia.org/wikipedia/en/thumb/f/fb/Yes_check.svg/30px-Yes_check.svg.png 1.5x, //upload.wikimedia.org/wikipedia/en/thumb/f/fb/Yes_check.svg/40px-Yes_check.svg.png 2x" data-file-width="600" data-file-height="600"><span style="display:none">Y</span> The sun sets in the west each evening.</p>
      </blockquote> 
      <p>It is preferable not to clutter articles with information best left to the references. Interested readers can click on the ref to find out the publishing journal: </p> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r996844942">
      <blockquote class="templatequote">
       <p><img alt="?" src="//upload.wikimedia.org/wikipedia/commons/thumb/a/a2/X_mark.svg/20px-X_mark.svg.png" decoding="async" width="20" height="23" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/a/a2/X_mark.svg/30px-X_mark.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/a/a2/X_mark.svg/40px-X_mark.svg.png 2x" data-file-width="525" data-file-height="600"><span style="display:none">N</span> In an article published in <i>The Lancet</i> in 2012, researchers announced the discovery of the new tissue type.<sup class="reference nowrap ltr" style="color:#002bb8;">[3]</sup> </p>
      </blockquote> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r996844942">
      <blockquote class="templatequote">
       <p><img alt="check" src="//upload.wikimedia.org/wikipedia/en/thumb/f/fb/Yes_check.svg/20px-Yes_check.svg.png" decoding="async" width="20" height="20" srcset="//upload.wikimedia.org/wikipedia/en/thumb/f/fb/Yes_check.svg/30px-Yes_check.svg.png 1.5x, //upload.wikimedia.org/wikipedia/en/thumb/f/fb/Yes_check.svg/40px-Yes_check.svg.png 2x" data-file-width="600" data-file-height="600"><span style="display:none">Y</span> The discovery of the new tissue type was first published by researchers in 2012.<sup class="reference nowrap ltr" style="color:#002bb8;">[3]</sup> </p>
      </blockquote> 
      <p>Simple facts such as this can have inline citations to reliable sources as an aid to the reader, but normally the text itself is best left as a plain statement without in-text attribution: </p> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r996844942">
      <blockquote class="templatequote">
       <p><img alt="check" src="//upload.wikimedia.org/wikipedia/en/thumb/f/fb/Yes_check.svg/20px-Yes_check.svg.png" decoding="async" width="20" height="20" srcset="//upload.wikimedia.org/wikipedia/en/thumb/f/fb/Yes_check.svg/30px-Yes_check.svg.png 1.5x, //upload.wikimedia.org/wikipedia/en/thumb/f/fb/Yes_check.svg/40px-Yes_check.svg.png 2x" data-file-width="600" data-file-height="600"><span style="display:none">Y</span> By mass, oxygen is the third most abundant element in the universe after hydrogen and helium.<sup class="reference nowrap ltr" style="color:#002bb8;">[4]</sup> </p>
      </blockquote> 
      <h2><span class="mw-headline" id="General_references"><span id="General_reference"></span>General references</span></h2> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:GENREF"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:GENREF&amp;redirect=no">WP:GENREF</a></span></li>
       </ul>
      </div> 
      <p>A general reference is a citation to a reliable source that supports content, but is not linked to any particular text in the article through an <a href="/wiki/Wikipedia:Inline_citation" title="Wikipedia:Inline citation">inline citation</a>. General references are usually listed at the end of the article in a "References" section, and are usually sorted by the last name of the author or the editor. General reference sections are most likely to be found in underdeveloped articles, especially when all article content is supported by a single source. The disadvantage of general references is that <a href="#Text–source_integrity">text–source integrity</a> is lost, unless the article is very short. They are frequently reworked by later editors into inline citations. </p>
      <p>The appearance of a general references section is the same as those given above in the sections on <a href="#Short_citations">short citations</a> and <a href="#Parenthetical_referencing">parenthetical references</a>. If both cited and uncited references exist, their distinction can be highlighted with separate section names, e.g., "References" and "General references". </p>

       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:NOCITE&amp;redirect=no">WP:NOCITE</a></span></li>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:BLPCITE&amp;redirect=no">WP:BLPCITE</a></span></li>
       </ul>
      </div> 
      <p>If an article has no references at all, then: </p> 
      <ul>
       <li>If the entire article is <a href="/wiki/Wikipedia:PN" class="mw-redirect" title="Wikipedia:PN">"Patent Nonsense"</a>, tag it for <a href="/wiki/Wikipedia:CSD" class="mw-redirect" title="Wikipedia:CSD">speedy deletion</a> using criterion G1.</li> 
       <li>If the article is a biography of a living person, it can be tagged with {{subst:prod blp}} to propose deletion. If it's a biography of a living person and is an attack page, then it should be tagged for speedy deletion using criterion G10, which will blank the page.</li> 
       <li>If the article doesn't fit into the above two categories, then consider finding references yourself, or commenting on the article talk page or the talk page of the article creator. You may also tag the article with the <code>{{<a href="/wiki/Template:Unreferenced" title="Template:Unreferenced">unreferenced</a>}}</code> template and consider nominating it for <a href="/wiki/Wikipedia:AFD" class="mw-redirect" title="Wikipedia:AFD">deletion</a>.</li>
      </ul> 
      <p>For individual unreferenced claims in an article: </p> 
      <ul>
       <li>If the article is a biography of a living person, then any <em>contentious</em> material must be removed immediately: see <a href="/wiki/Wikipedia:Biographies_of_living_persons" title="Wikipedia:Biographies of living persons">Biographies of living persons</a>. If the unreferenced material is seriously inappropriate, it may need to be hidden from general view, in which case <a href="/wiki/Wikipedia:RFO" class="mw-redirect" title="Wikipedia:RFO">request admin assistance</a>.</li> 
       <li>If the material added appears to be false or an expression of opinion, remove it and inform the editor who added the unsourced material. The <code>{{<a href="/wiki/Template:Uw-unsourced1" title="Template:Uw-unsourced1">uw-unsourced1</a>}}</code> template may be placed on their talk page.</li> 
       <li>In any other case consider finding references yourself, or commenting on the article talk page or the talk page of the editor who added the unsourced material. You may place a <code>{{<a href="/wiki/Template:Citation_needed" title="Template:Citation needed">citation needed</a>}}</code> or <code>{{<a href="/wiki/Template:Dubious" title="Template:Dubious">dubious</a>}}</code> tag against the added text.</li>
      </ul> 
      <h2><span class="mw-headline" id="Citation_templates_and_tools">Citation templates and tools</span></h2> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:CITECONSENSUS"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:CITECONSENSUS&amp;redirect=no">WP:CITECONSENSUS</a></span></li>
       </ul>
      </div> 
      <div role="note" class="hatnote navigation-not-searchable">
       Further information: <a href="/wiki/Wikipedia:Citation_templates" title="Wikipedia:Citation templates">Wikipedia:Citation templates</a> and <a href="/wiki/Help:Citation_tools" title="Help:Citation tools">Help:Citation tools</a>
      </div> 
      <div role="note" class="hatnote navigation-not-searchable">
       For a comparison of citations using templates with citations written freehand, see <a href="/wiki/Wikipedia:Citing_sources/Example_edits_for_different_methods#Footnotes" title="Wikipedia:Citing sources/Example edits for different methods">Wikipedia:Citing sources/Example edits for different methods §&nbsp;Footnotes</a>.
      </div> 
      <p><a href="/wiki/Wikipedia:Citation_templates" title="Wikipedia:Citation templates">Citation templates</a> can be used to format citations in a consistent way. The use of citation templates is neither encouraged nor discouraged: an article should not be switched between templated and non-templated citations without good reason and consensus – see <a href="#Variation_in_citation_methods">"Variation in citation methods"</a>, above. </p>
      <p>If citation templates are used in an article, the parameters should be accurate. It is inappropriate to set parameters to false values to cause the template to render as if it were written in some style other than the style normally produced by the template (e.g., <a href="/wiki/The_MLA_Style_Manual" class="mw-redirect" title="The MLA Style Manual">MLA style</a>). </p> 
      <h3><span class="mw-headline" id="Metadata">Metadata</span></h3> 
      <p>Citations may be accompanied by metadata, though it is not mandatory. Most citation templates on Wikipedia use the <a href="/wiki/COinS" title="COinS">COinS</a> standard. Metadata such as this allow browser plugins and other automated software to make citation data accessible to the user, for instance by providing links to their library's online copies of the cited works. In articles that format citations manually, metadata may be added manually in a span, according to the <a rel="nofollow" class="external text" href="http://ocoins.info/">COinS specification</a>. </p>
