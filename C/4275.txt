      <h2><span class="mw-headline" id="Using_access_keys">Using access keys</span></h2> 
      <p>An <a href="/wiki/Access_key" title="Access key">access key</a> allows a computer user to immediately jump to a specific part of a web page via the keyboard. In Wikipedia, access keys allow you to do a lot more—protect a page, show page history, publish your changes, show preview text, and so on. See the next section for the full list. </p>
      <p>Most web browsers require holding down one or two "modifier" keys to use an access key. Different web browsers invoke access keys in different ways. Please use the list below to determine the appropriate key combination for your browser and system: </p>

      <p>Keyboard shortcuts, in the way they are implemented on Wikipedia, are useful for some users with disabilities and do not require JavaScript to work. At the same time, they are bothersome for some screen reader users because it interferes with the shortcuts of their screen reader. The best solution would be to have customizable keyboard shortcuts. </p>
      <p>The accessibility of keyboard shortcuts (access keys) is a complicated matter. This feature, in its current implementation, was temporarily abandoned in <a href="/wiki/WCAG" class="mw-redirect" title="WCAG">WCAG</a> 2.0, notably because it conflicts with the shortcuts of assistive technologies. Usage of access keys is currently discouraged in the online contents and applications. </p>
      <p>But presence of access keys is also an <a href="/wiki/Authoring_Tool_Accessibility_Guidelines" class="mw-redirect" title="Authoring Tool Accessibility Guidelines">Authoring Tool Accessibility Guidelines</a> (ATAG) requirement—the ATAG approach is particularly relevant in Wikipedia's case—where this matter is looked into thoroughly. In particular, ATAG requires the possibility to customize shortcuts. This should be requested in <a href="https://bugzilla.wikimedia.org/show_bug.cgi?id=477" class="extiw" title="bugzilla:477">bugzilla:477</a>. </p>
      <p>Currently, their presence in Wikipedia's interface can be problematic for some users and can be useful for others. But more importantly, it misleads developers into thinking that the interface is keyboard accessible. Requirements that need to be fulfilled for a website to be keyboard accessible are not related to keyboard shortcuts (<a rel="nofollow" class="external text" href="http://www.w3.org/TR/2008/NOTE-WCAG20-TECHS-20081211/G90">G90</a>, <a rel="nofollow" class="external text" href="http://www.w3.org/TR/WCAG-TECHS/F42.html">F42</a>, <a rel="nofollow" class="external text" href="http://www.w3.org/TR/WCAG-TECHS/SCR35.html">SCR35</a>, <a rel="nofollow" class="external text" href="http://www.w3.org/TR/2008/NOTE-WCAG20-TECHS-20081211/F58">F58</a>, <a rel="nofollow" class="external text" href="http://www.w3.org/TR/2008/NOTE-WCAG20-TECHS-20081211/F61">F61</a>). </p>
