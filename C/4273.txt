      <p>A <b>shortcut</b> is a specialized type of <a href="/wiki/Wikipedia:Redirect" title="Wikipedia:Redirect">redirect page</a> that provides an abbreviated <a href="/wiki/Wikilink" class="mw-redirect" title="Wikilink">wikilink</a> to an <a href="/wiki/Wikipedia:Administration#Data_structure_and_development" title="Wikipedia:Administration">administrative page</a> or one of its sections, usually from the <strong><a href="/wiki/Wikipedia:Project_namespace" title="Wikipedia:Project namespace">Wikipedia namespace</a></strong> or <strong><a href="/wiki/Wikipedia:Help_namespace" title="Wikipedia:Help namespace">Help namespace</a></strong>. These shortcuts' abbreviations in all-caps text are commonly used (linked or unlinked) on community pages and talk pages, but should not be used in articles themselves. If there are one or more shortcuts for a page or section, one or more will usually be chosen to be displayed in an information box on the right labelled <b>Shortcut</b> or <b>Shortcuts</b>, as can be seen at the top of this page. </p>
      <p>Shortcuts are created for the convenience of editors. It is possible to create a shortcut for any page at all. <strong><a href="/wiki/Wikipedia:Policies_and_guidelines#Adherence" title="Wikipedia:Policies and guidelines">The existence of a shortcut does not imply or prove that the linked page is a policy or guideline</a>.</strong> </p>

      <h2><span class="mw-headline" id="How_to_use_a_Wikipedia_shortcut">How to use a Wikipedia shortcut</span></h2> 
      <p>A Wikipedia shortcut can be entered into the Wikipedia search box to quickly bring you to a project page. For example, you can type <b>WP:R</b> or <b>H:R</b> into the search box and then press the enter key (or click the magnifying glass icon), to get to the <b>Wikipedia:Redirect</b> page, instead of having to type in the complete phrase. </p>
      <p>Alternatively, you can use a shortcut in the <a href="/wiki/URL" title="URL">URL</a> in your browser's address bar. For example, you are currently viewing the <b>Wikipedia:Shortcut</b> page. The URL for this page is <span class="plainlinks"><a class="external free" href="https://en.wikipedia.org/wiki/Wikipedia:Shortcut">https://en.wikipedia.org/wiki/Wikipedia:Shortcut</a></span>. You could then get from here to the <b>Wikipedia:Redirect</b> page by replacing <b>Wikipedia:Shortcut</b> with <b>WP:R</b> in the address bar, followed by pressing the enter key. (If your browser has relative URL entry, simply prefix the shortcut with <tt>./</tt> instead of having to edit the existing URL.) </p>
      <p>Shortcuts are presented in all capital letters (<a href="/wiki/ALL_CAPS" class="mw-redirect" title="ALL CAPS">ALL CAPS</a>); however, the search box is <a href="/wiki/Case-insensitive" class="mw-redirect" title="Case-insensitive">case-insensitive</a>. For example, in the search box, you can type <b>wp:r</b>, instead of <b>WP:R</b>. However, when using the URL method (or when making <a href="/wiki/Help:Link" title="Help:Link">links</a>) it is necessary to match the capitalization of the shortcut itself. </p> 
      <h2><span class="mw-headline" id="Readability">Readability</span></h2> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:SCJARGON"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a class="mw-selflink selflink">Shortcut</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:SCJARGON&amp;redirect=no">WP:SCJARGON</a></span></li>
       </ul>
      </div> 
      <p>Shortcuts are often used on talk pages in their abbreviated form, decreasing <a href="/wiki/Readability" title="Readability">readability</a> for the general reader. For example, some editors are familiar with the bulk of the <a href="/wiki/Wikipedia:WP" class="mw-redirect" title="Wikipedia:WP">Wikipedia namespace shortcuts</a>, recognizing what they stand for on sight. Others, however, are faced with pages full of incomprehensible jargon, the meaning of which is not immediately clear. Shortcuts also are sometimes (ab)used to make a <a href="/wiki/Wikipedia:POINT" class="mw-redirect" title="Wikipedia:POINT">WP:POINT</a>, best described by <a href="/wiki/Wikipedia:WOTTA" class="mw-redirect" title="Wikipedia:WOTTA">WP:WOTTA</a>. </p>
      <p>To avoid these problems, a good practice when creating shortcuts is to choose common English words that are easily identifiable and memorable. Another good practice is to be mindful of the general reader and use meaningful terms when citing an obscure shortcut. For example, the <a href="/wiki/Wikipedia:Piped_link" title="Wikipedia:Piped link">piped link</a> <code>[[WP:SHC|shortcut]]</code> gives readers an idea of the subject of the target page, while the bare abbreviation <code>[[WP:SHC]]</code> is unintelligible to those unfamiliar with the term. </p>
      <p>It should also be remembered that shortcuts are a convenience, not a substitute for titles. Shortcuts should not appear in <a href="/wiki/Wikipedia:SEEALSO" class="mw-redirect" title="Wikipedia:SEEALSO">"See also" sections</a>, <a href="/wiki/Wikipedia:Hatnote" title="Wikipedia:Hatnote">hatnotes</a> or any other place where a page title is expected. </p>

       <div class="module-shortcutlist">
        <a class="mw-selflink selflink">Shortcut</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:LINKBOXES&amp;redirect=no">WP:LINKBOXES</a></span></li>
       </ul>
      </div> 
      <p>Small link boxes, listing the names of the page's shortcuts, appear at the top of many pages, especially those on policies and guidelines. A shortcut link box can be added to a page, by placing the template <b><code>{{<a href="/wiki/Template:Shortcut" title="Template:Shortcut">shortcut</a>|WP:&lt;x&gt;}}</code></b> (for <a href="/wiki/Wikipedia:Project_namespace" title="Wikipedia:Project namespace">project namespace</a>) or <b><code>{{<a href="/wiki/Template:Shortcut" title="Template:Shortcut">shortcut</a>|H:&lt;x&gt;}}</code></b> (for Help pages) at the top of the page's text, while editing. </p>
      <p>The point of these template boxes is not to list every single redirect for any given page (indeed, that's what <a href="/wiki/Special:WhatLinksHere" title="Special:WhatLinksHere">Special:WhatLinksHere</a> is for); instead, they generally should list only the most common and easily remembered redirects. </p>
      <p><a href="/wiki/Wikipedia:Policies_and_guidelines" title="Wikipedia:Policies and guidelines">Guideline</a> pages with shortcuts pointing to them typically use <b><code>{{<a href="/wiki/Template:Guideline" title="Template:Guideline">guideline</a>|WP:&lt;x&gt;}}</code></b> or the <b>{{<a href="/wiki/Template:Subcat_guideline" title="Template:Subcat guideline">subcat guideline</a>}}</b> template, such as on this page, instead of the shortcut template <b><code>{{<a href="/wiki/Template:Shortcut" title="Template:Shortcut">shortcut</a>}}</code></b>. Another frequently used template is: <b><code>{{<a href="/wiki/Template:MoS-guideline" class="mw-redirect" title="Template:MoS-guideline">MoS-guideline</a>|MOS:&lt;x&gt;}}</code></b>. </p>
      <p>To learn more about the different shortcut box templates and their functions, see documentation at {{<a href="/wiki/Template:Shortcut" title="Template:Shortcut">shortcut</a>}}. Among other things, there are templates for making boxes flow to the left and now, anchors are automatically added, making it much simpler to link to a page section. </p> 
      <h2><span class="mw-headline" id="Shortcut_names">Shortcut names</span></h2> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:SCNAMES"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a class="mw-selflink selflink">Shortcut</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:SCNAMES&amp;redirect=no">WP:SCNAMES</a></span></li>
       </ul>
      </div> 
      <p>Shortcut names are almost always in uppercase. </p>
      <p>A shortcut is typically named using the following convention: </p> 
      <dl>
       <dd>
        <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">PREFIX:SUFFIX</code>
       </dd>
      </dl> 
      <p>In the above form, the prefix is typically one of the prefixes in the following section, and the suffix is an <a href="/wiki/Acronym_and_initialism" class="mw-redirect" title="Acronym and initialism">acronym</a> or <a href="/wiki/Wikipedia:Manual_of_Style_(abbreviations)" class="mw-redirect" title="Wikipedia:Manual of Style (abbreviations)">abbreviation</a> of the redirect target. However quite a lot are simply <a href="/wiki/Wikipedia:BAD" class="mw-redirect" title="Wikipedia:BAD">WP:BAD</a>. </p>
      <p>If the target is a <a href="/wiki/Help:Subpage" class="mw-redirect" title="Help:Subpage">sub-page</a>, the following conventions are common (where 'Y' is an additional sequence of uppercase characters or numerals): </p> 
      <dl>
       <dd>
        <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">PREFIX:SUFFIX/Y</code>
       </dd>
      </dl> 
      <dl>
       <dd>
        <dl>
         <dd>
          For example, <a href="/wiki/P:HVNY/DYK" class="mw-redirect" title="P:HVNY/DYK">P:HVNY/DYK</a> is a shortcut to <a href="/wiki/Portal:Hudson_Valley/Did_you_know" title="Portal:Hudson Valley/Did you know">Portal:Hudson Valley/Did you know</a>, which is a subpage of <a href="/wiki/Portal:Hudson_Valley" title="Portal:Hudson Valley">Portal:Hudson Valley</a>, which has a shortcut <a href="/wiki/P:HVNY" class="mw-redirect" title="P:HVNY">P:HVNY</a>.
         </dd>
        </dl>
       </dd>
      </dl> 
      <dl>
       <dd>
        <dl>
         <dd>
          <a href="/wiki/Wikipedia:ACC/P" class="mw-redirect" title="Wikipedia:ACC/P">WP:ACC/P</a> is a shortcut to <a href="/wiki/Wikipedia:Request_an_account/Procedures" title="Wikipedia:Request an account/Procedures">Wikipedia:Request an account/Procedures</a>, which is a subpage of <a href="/wiki/Wikipedia:Request_an_account" title="Wikipedia:Request an account">Wikipedia:Request an account</a>, which has a shortcut <a href="/wiki/Wikipedia:ACC" class="mw-redirect" title="Wikipedia:ACC">WP:ACC</a>.
         </dd>
        </dl>
       </dd>
      </dl> 
      <dl>
       <dd>
        <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">PREFIX:SUFFIXY</code>
       </dd>
      </dl> 
      <dl>
       <dd>
        <dl>
         <dd>
          For example, <a href="/wiki/Wikipedia:AFDLT" class="mw-redirect" title="Wikipedia:AFDLT">WP:AFDLT</a> is a shortcut to <a href="/wiki/Wikipedia:Articles_for_deletion/Log/Today" title="Wikipedia:Articles for deletion/Log/Today">Wikipedia:Articles for deletion/Log/Today</a>, which is a subpage of <a href="/wiki/Wikipedia:Articles_for_deletion" title="Wikipedia:Articles for deletion">Wikipedia:Articles for deletion</a>, which has the shortcut <a href="/wiki/Wikipedia:AFD" class="mw-redirect" title="Wikipedia:AFD">WP:AFD</a>.
         </dd>
        </dl>
       </dd>
      </dl> 
      <p>Shortcuts that do not follow the above naming convention are discouraged, and are typically deleted. </p>
      <p>The directory of shortcuts contains several conflicts for prefixes of subpages, often caused by a shortcut being 're-targeted' (modified to point to a new target page) without the shortcuts to subpages also being re-targeted. </p> 
      <dl>
       <dd>
        <dl>
         <dd>
          For example, <a href="/wiki/Wikipedia:CJ" class="mw-redirect" title="Wikipedia:CJ">WP:CJ</a> was originally used for <a href="/wiki/Wikipedia:Community_Justice" class="mw-redirect" title="Wikipedia:Community Justice">Wikipedia:Community Justice</a>, and several shortcuts for subpages were created (<a href="/wiki/Wikipedia:CJ/E" class="mw-redirect" title="Wikipedia:CJ/E">WP:CJ/E</a> &amp; <a href="/wiki/Wikipedia:CJ/M/1" class="mw-redirect" title="Wikipedia:CJ/M/1">WP:CJ/M/1</a>). After several years, 'WP:CJ' was updated to point to <a href="/wiki/Wikipedia:Centijimbos" title="Wikipedia:Centijimbos">Wikipedia:Centijimbos</a>, but the subpage shortcuts have not changed.
         </dd>
        </dl>
       </dd>
      </dl> 
      <p>Before creating a shortcut, use <a href="/wiki/Special:PrefixIndex" title="Special:PrefixIndex">Special:PrefixIndex</a> to ensure that the same prefix is not being used for different purposes. e.g. <a href="/wiki/Special:PrefixIndex/WP:CJ" title="Special:PrefixIndex/WP:CJ">WP:CJ</a> has several existing uses. No additional confusion would be caused by creating a shortcut WP:CJQ, however creating a shortcut WP:CJ/Q would be adding to the existing mess. There is a long-standing feature request to resolve shortcut subpages automatically. (See <a href="https://phabricator.wikimedia.org/T14980" class="extiw" title="phabricator:T14980">T14980</a>.) </p>
      <p>Prefixes that shouldn't be used for other purposes include: </p> 
      <ul>
       <li><a href="/wiki/Special:PrefixIndex/WP:NC" title="Special:PrefixIndex/WP:NC">WP:NC</a>  naming conventions</li> 
       <li><a href="/wiki/Special:PrefixIndex/WP:AFD" title="Special:PrefixIndex/WP:AFD">WP:AFD</a>  articles for deletion</li>
      </ul> 
      <p>Other prefixes have become common for a set of pages include: </p> 
      <ul>
       <li><a href="/wiki/Special:PrefixIndex/WP:N" title="Special:PrefixIndex/WP:N">WP:N</a>  notability</li>
      </ul> 
      <h2><span class="mw-headline" id="List_of_prefixes">List of prefixes</span></h2> 
      <h3><span class="mw-headline" id="Namespace_aliases">Namespace aliases</span></h3> 
      <p>The following <a href="/wiki/Wikipedia:Namespace#Aliases" title="Wikipedia:Namespace">namespace aliases</a> are automatically translated by the Wikipedia servers. </p> 
      <table class="wikitable"> 
       <tbody>
        <tr> 
         <td style="width: 8em">WP:</td> 
         <td style="width: 15em">Wikipedia: </td>
        </tr> 
        <tr> 
         <td>WT:</td> 
         <td>Wikipedia talk: </td>
        </tr> 
        <tr> 
         <td>Project:</td> 
         <td>Wikipedia: </td>
        </tr> 
        <tr> 
         <td>Project talk:</td> 
         <td>Wikipedia talk: </td>
        </tr> 
        <tr> 
         <td>Image:</td> 
         <td>File: </td>
        </tr> 
        <tr> 
         <td>Image talk:</td> 
         <td>File talk: </td>
        </tr>
       </tbody>
      </table> 
      <p>They are case-insensitive (so <b>wp:</b> or <b>wP:</b> or <b>Wp:</b> have the same meaning as <b>WP:</b>). For example, both <a href="/wiki/Wikipedia:TS" class="mw-redirect" title="Wikipedia:TS">WP:TS</a> and <a href="/wiki/Wikipedia:TS" class="mw-redirect" title="Wikipedia:TS">wp:TS</a> link directly to <a href="/wiki/Wikipedia:TS" class="mw-redirect" title="Wikipedia:TS">Wikipedia:TS</a>. </p>
      <p>One unique feature of the WP: alias is its utility in searching for vast number of project pages. This is because most project pages outside the Wikipedia namespace, will have a <a href="/wiki/Wikipedia:Redirect" title="Wikipedia:Redirect">redirect</a> for the Wikipedia namespace prefix appended to their title and the same for most major shortcuts to them (e.g., Help:Watchlist, with the shortcut H:W, has redirects at <i>Wikipedia:Watchlist</i> and at <i>WP:W</i>. Thus, by extension, most project pages may be found by searching using the WP: alias alone, e.g. <code>WP:<i>Name</i></code>. For more, see <a href="/wiki/Help:WP_search_protocol" title="Help:WP search protocol">Help:WP search protocol</a>. </p> 
      <h3><span class="mw-headline" id="Pseudo-namespaces">Pseudo-namespaces</span></h3> 
      <p>When the shortcut does not use one of the namespace aliases listed above (WP or WT), it is a <a href="/wiki/Wikipedia:Cross-namespace_redirect" class="mw-redirect" title="Wikipedia:Cross-namespace redirect">cross-namespace redirect</a> and is case-sensitive. </p>
      <p>However, when a prefix is used by multiple shortcuts, and has broad community support, it is referred to as a <a href="/wiki/Wikipedia:Namespace#Pseudo-namespaces" title="Wikipedia:Namespace">pseudo-namespace</a> to distinguish these shortcuts from other cross-namespace redirects. (Note that the term "<a href="https://www.mediawiki.org/wiki/pseudo-namespace" class="extiw" title="mw:pseudo-namespace">pseudo-namespace</a>" is also used to refer to namespaces without content that are provided by the MediaWiki software and extensions, such as Special:, Media:, Feedback:) </p>
      <p>The following prefixes may be used freely for the target listed. </p> 
      <table class="wikitable"> 
       <tbody>
        <tr> 
         <th style="width: 8em">Prefix</th> 
         <th style="width: 15em">Target </th>
        </tr> 
        <tr> 
         <td><a href="/wiki/Special:PrefixIndex/CAT:" title="Special:PrefixIndex/CAT:">CAT:</a></td> 
         <td>Category: </td>
        </tr> 
        <tr> 
         <td><a href="/wiki/Special:PrefixIndex/H:" title="Special:PrefixIndex/H:">H:</a></td> 
         <td>Help: </td>
        </tr> 
        <tr> 
         <td><a href="/wiki/Special:PrefixIndex/MOS:" title="Special:PrefixIndex/MOS:">MOS:</a></td> 
         <td>Wikipedia:Manual of Style </td>
        </tr> 
        <tr> 
         <td><a href="/wiki/Special:PrefixIndex/P:" title="Special:PrefixIndex/P:">P:</a></td> 
         <td>Portal: </td>
        </tr>
       </tbody>
      </table> 
      <p>Other prefixes do not enjoy the same level of community support. It is disputed whether they should be deemed to be pseudo-namespaces. </p>
      <p>The following prefixes are used for limited or specific uses only. </p> 
      <table class="wikitable"> 
       <tbody>
        <tr> 
         <td style="width: 8em"><a href="/wiki/Special:PrefixIndex/T:" title="Special:PrefixIndex/T:">T:</a></td>

        </tr> 
        <tr> 
         <td><a href="/wiki/Special:PrefixIndex/MP:" title="Special:PrefixIndex/MP:">MP:</a></td> 
         <td>Main Page </td>
        </tr>
       </tbody>
      </table> 
      <p>The following prefixes do not enjoy broad community support. </p> 
      <table class="wikitable"> 
       <tbody>
        <tr> 
         <td><a href="/wiki/Special:PrefixIndex/WikiProject:" title="Special:PrefixIndex/WikiProject:">WikiProject:</a><br><a href="/wiki/Special:PrefixIndex/Wikiproject:" title="Special:PrefixIndex/Wikiproject:">Wikiproject:</a></td> 
         <td>Wikipedia:WikiProject </td>
        </tr> 
        <tr> 
         <td><a href="/wiki/Special:PrefixIndex/MoS:" title="Special:PrefixIndex/MoS:">MoS:</a><br><a href="/wiki/Special:PrefixIndex/Mos:" title="Special:PrefixIndex/Mos:">Mos:</a></td> 
         <td>Wikipedia:Manual of Style </td>
        </tr>
       </tbody>
      </table> 
      <p>In addition to those two, a handful of <a href="/wiki/Wikipedia:WikiProjects" class="mw-redirect" title="Wikipedia:WikiProjects">wikiprojects</a> have mainspace shortcuts (as of 2018)<sup class="plainlinks noexcerpt noprint asof-tag update" style="display:none;"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Shortcut&amp;action=edit">[update]</a></sup> using a space (which can also be an underscore) after the word, instead of a colon: </p> 
      <table class="wikitable"> 
       <tbody>
        <tr> 
         <td style="width: 8em"><a href="/wiki/Special:PrefixIndex/WikiProject" title="Special:PrefixIndex/WikiProject">WikiProject_</a><br><a href="/wiki/Special:PrefixIndex/Wikiproject" title="Special:PrefixIndex/Wikiproject">Wikiproject_</a></td> 
         <td style="width: 15em">Wikipedia:WikiProject </td>
        </tr>
       </tbody>
      </table> 
      <p>The second of these is no longer in use at all. </p>
      <p>Portals have occasionally also used spaces in shortcuts, e.g. "<a href="/wiki/Special:PrefixIndex/Portal" title="Special:PrefixIndex/Portal">Portal&nbsp;<var style="padding-right: 1px;">X</var></a>", or "<a href="/wiki/Special:Search/InTitle:Portal" title="Special:Search/InTitle:Portal"><var style="padding-right: 1px;">X</var>&nbsp;portal</a>". Most mainspace pages containing the word are articles or redirects to articles. </p>
      <p>Technical note: The links in the table go to a page where the term <i>prefix</i> is not used in the meaning as it has in this section. Rather, the code <i>PrefixIndex</i> there means, as it does for <a href="/wiki/Help:Search#Syntax" class="mw-redirect" title="Help:Search">the search parameter</a>, "the beginning string of characters in a title", whether that string ends before, during or after the colon character in a page name. </p> 
      <h2><span class="mw-headline" id="How_to_create_a_shortcut">How to create a shortcut</span></h2> 
      <div role="note" class="hatnote navigation-not-searchable">
       Note: The following is an example. The shortcut WP:TS already exists.
      </div> 
      <p>Say you want to create the shortcut <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">WP:TS</code> to an existing page, <a href="/wiki/Wikipedia:Template_standardisation" title="Wikipedia:Template standardisation">Wikipedia:Template standardisation</a>. Create a new page, <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">Wikipedia:TS</code>, and in it place the following: </p> 
      <div class="mw-highlight mw-highlight-lang-moin mw-content-ltr" dir="ltr">
       <pre><span></span><span class="c">#REDIRECT [[Wikipedia:Template standardisation]]</span>
{{R from shortcut}}
</pre>
      </div> 
      <p>The resulting shortcut will be <a href="/wiki/Wikipedia:TS" class="mw-redirect" title="Wikipedia:TS">WP:TS</a>. </p>
      <p>If you're creating a shortcut to a <b>category</b>, you need to insert an additional colon (":") at the beginning of the link, as in the following. </p> 
      <div class="mw-highlight mw-highlight-lang-moin mw-content-ltr" dir="ltr">
       <pre><span></span><span class="c">#REDIRECT [[:Category:Redirects from shortcuts]]</span>
{{R from shortcut}}
</pre>
      </div> 
      <p>The template <code>{{<a href="/wiki/Template:R_from_shortcut" title="Template:R from shortcut">R from shortcut</a>}}</code> must always be included in this kind of redirect. It records that the redirect is a shortcut, and auto-categorizes it into <a href="/wiki/Category:Redirects_from_shortcuts" title="Category:Redirects from shortcuts">Category:Redirects from shortcuts</a>. <style data-mw-deduplicate="TemplateStyles:r1015928993">.mw-parser-output div.crossreference{padding-left:0}</style><span role="note" class="hatnote navigation-not-searchable crossreference">(See <a href="/wiki/Wikipedia:Template_messages/Redirect_pages" class="mw-redirect" title="Wikipedia:Template messages/Redirect pages">Wikipedia:Template messages/Redirect pages</a> for more information on categories of redirects.)</span> </p>
      <p>Many shortcuts are <a href="/wiki/Acronym_and_initialism" class="mw-redirect" title="Acronym and initialism">acronyms or initialisms</a>; others are <a href="/wiki/Wikipedia:Manual_of_Style_(abbreviations)" class="mw-redirect" title="Wikipedia:Manual of Style (abbreviations)">abbreviations</a> or single but memorable words from a longer page title. </p> 
      <h2><span class="mw-headline" id="Changing_shortcuts">Changing shortcuts</span></h2> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:CHANGESC"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a class="mw-selflink selflink">Shortcut</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:CHANGESC&amp;redirect=no">WP:CHANGESC</a></span></li>
       </ul>
      </div> 
      <p>Technically that's easy. All it takes is to click on the shortcut wherever it is, then on the page it leads to (a shortcut is a redirect) click on the "redirected from" link near the top of the page (forcing <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">&amp;redirect=no</code> in the URL when you go back to the shortcut page); edit the target <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">#REDIRECT [[Wikipedia:Whatever]]</code>; then add <code>{{<a href="/wiki/Template:R_from_shortcut" title="Template:R from shortcut">R from shortcut</a>}}</code> if that template doesn't already appear. </p>
      <p>However, before changing a shortcut, consider its existing usage: </p> 
      <ol>
       <li>If you wish to retarget a shortcut that's already in use, you should take into account how active its target page is. For example, if you just created a WikiProject, and want a specific shortcut for it, but the one you want is already in use by another project, it's generally recommended to post a message on that project's talk page asking the members if you may change the link, even if the project is tagged as <span class="nowrap">{{<a href="/wiki/Template:WikiProject_status" title="Template:WikiProject status">WikiProject status</a>|inactive}}</span>.</li> 
       <li>Above all, check the shortcut backlinks with "what links here"; changing a shortcut used elsewhere can be highly disruptive. If you do change a shortcut under these circumstances, you should be sure to change it in the pages that linked to it as well, so that they link to the old target. If they were used in any <a href="/wiki/Wikipedia:XFD" class="mw-redirect" title="Wikipedia:XFD">closed deletion discussions</a>, it is not a good idea then to change the redirect because these discussions cannot be edited.</li> 
       <li>If it has no backlink, folks might still use it directly when searching. If you're not 100% sure that the old target is unused, ask about it on the shortcut's talk page.</li> 
       <li>For controversial cases, go to <a href="/wiki/Wikipedia:Requests_for_comment" title="Wikipedia:Requests for comment">Wikipedia:Requests for comment</a>.</li>
      </ol> 
      <p>After changing a shortcut, there are other pages which may need to be updated: </p> 
      <ol>
       <li>If the shortcut is already listed on <a href="/wiki/Wikipedia:List_of_shortcuts" class="mw-redirect" title="Wikipedia:List of shortcuts">Wikipedia:List of shortcuts</a> or a similar list like <a href="/wiki/Wikipedia:Shortcuts_to_talk_pages" title="Wikipedia:Shortcuts to talk pages">Wikipedia:Shortcuts to talk pages</a>, please update that entry showing the new target page.</li> 
       <li>If the old target page mentions its shortcut, then that also should be updated, typically in <code>{{<a href="/wiki/Template:Shortcut" title="Template:Shortcut">Shortcut</a>}}</code> or similar <a href="/wiki/Category:Wikipedia_header_templates" title="Category:Wikipedia header templates">Wikipedia header templates</a>.</li>
      </ol> 
      <h2><span class="mw-headline" id="Limitations">Limitations</span></h2> 
      <p>Redirects (including shortcuts) to pages on other projects and <a href="/wiki/Wikipedia:Special_pages" class="mw-redirect" title="Wikipedia:Special pages">special pages</a> won't workthis formerly supported feature was abused and hence it has been disabled. The software will display the redirect page instead of redirecting the user to the target. </p>
      <p>Some redirects to other projects and special pages use the template {{<a href="/wiki/Template:Soft_redirect" title="Template:Soft redirect">soft redirect</a>}}; however, shortcuts that are a soft redirects will appear in the pool of pages that are selected by <a href="/wiki/Special:Random" title="Special:Random">Special:Random</a>. Soft redirects are categorised into <a href="/wiki/Category:Wikipedia_soft_redirects" title="Category:Wikipedia soft redirects">Category:Wikipedia soft redirects</a>. </p>
