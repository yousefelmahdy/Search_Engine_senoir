       <h3 id="about-classrooms"><a href="#about-classrooms">About classrooms</a></h3> 
       <p>A classroom in GitHub Classroom is where teachers and students interact during a course. Teachers can create a student roster for the classroom, then create, assign, review, and grade assignments within the classroom.</p> 
       <p><img src="/assets/images/help/classroom/classroom-hero.png" alt="Classroom"></p> 
       <h3 id="about-management-of-classrooms"><a href="#about-management-of-classrooms">About management of classrooms</a></h3> 
       <p>GitHub Classroom uses organization accounts on GitHub to manage permissions, administration, and security for each classroom that you create. Each organization can have multiple classrooms.</p> 
       <p>After you create a classroom, GitHub Classroom will prompt you to invite teaching assistants (TAs) and admins to the classroom. Each classroom can have one or more admins. Admins can be teachers, TAs, or any other course administrator who you'd like to have control over your classrooms on GitHub Classroom.</p> 
       <p>Invite TAs and admins to your classroom by inviting the user accounts on GitHub to your organization as organization owners and sharing the URL for your classroom. Organization owners can administer any classroom for the organization. For more information, see "<a href="/en/organizations/managing-peoples-access-to-your-organization-with-roles/permission-levels-for-an-organization">Permission levels for an organization</a>" and "<a href="/en/organizations/managing-membership-in-your-organization/inviting-users-to-join-your-organization">Inviting users to join your organization</a>."</p> 
       <p>When you're done using a classroom, you can archive the classroom and refer to the classroom, roster, and assignments later, or you can delete the classroom if you no longer need the classroom.</p> 
       <h3 id="about-classroom-rosters"><a href="#about-classroom-rosters">About classroom rosters</a></h3> 
       <p>Each classroom has a roster. A roster is a list of identifiers for the students who participate in your course.</p> 
       <p>When you first share the URL for an assignment with a student, the student must sign into GitHub with a user account to link the user account to an identifier for the classroom. After the student links a user account, you can see the associated user account in the roster. You can also see when the student accepts or submits an assignment.</p> 
       <p><img src="/assets/images/help/classroom/roster-hero.png" alt="Classroom roster"></p> 
       <h3 id="prerequisites"><a href="#prerequisites">Prerequisites</a></h3> 
       <p>You must have an organization account on GitHub to manage classrooms on GitHub Classroom. For more information, see "<a href="/en/github/getting-started-with-github/types-of-github-accounts#organization-accounts">Types of GitHub accounts</a>" and "<a href="/en/organizations/collaborating-with-groups-in-organizations/creating-a-new-organization-from-scratch">Creating a new organization from scratch</a>."</p> 
       <p>You must authorize the OAuth app for GitHub Classroom for your organization to manage classrooms for your organization account. For more information, see "<a href="/en/github/authenticating-to-github/authorizing-oauth-apps">Authorizing OAuth Apps</a>."</p>
