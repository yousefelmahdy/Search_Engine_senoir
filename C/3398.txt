        <p>Boom?! 20 characters in HTML gave us validation! Now that we have it working, you might have a couple questions:</p>
        <ol>
         <li>How do I style the <code>&lt;input&gt;</code> when it is valid or invalid?</li>
         <li>Can I specify custom error messages? If so, how?</li>
         <li>How do I style the error message?</li>
        </ol>
        <p>So let's dig into each one of these with an example. We will build step-by-step on the simple example above until we have something more custom and attractive.</p>
        <h2 id="example-2-styling-the-input">Example 2: Styling the input</h2>
        <p>There are a couple CSS pseudo classes supported by each <code>&lt;input&gt;</code>. Two potentially useful selectors in this situation:</p>
        <ul>
         <li><code>:valid</code> - when the input is valid</li>
         <li><code>:invalid</code> - when the input is invalid</li>
        </ul>
        <p>The most basic approach would be to change the border color when the input is invalid on our previous example:</p>
        <pre><code class="lang-css">/* CSS :invalid pseudo selector */
input:invalid {
  border-color: red;
}
</code></pre>
        <p></p>
        <style>.example-form--invalid input:invalid {
    border-color: red;
  }</style>
        <p></p>
        <div class="example-form example-form--invalid example-form--center">
         <div class="example-title">
          basic example + :invalid styling
         </div>
         <form class="pageclip-form" method="post" action="https://send.pageclip.co/yYo14hdvw3pSIJi1Y6maW1LFR7gOAUaI/validation">
          <p class="text_smaller">The box is red the whole time you are typing an email. Weak. :/</p>
          <div class="form-group">
           <label>Your Email</label> 
           <input required class="form-control" type="email" name="email" placeholder="a@b.com">
          </div>
          <input type="hidden" name="ex" value="pseudo"> <button class="button button_cta pageclip-form__submit pageclip-form__submit--dark-loader" type="submit"><span>Submit</span></button>
         </form>
        </div>
        <p>Notice, though, that <strong>it isn't very good</strong>. What is even happening?</p>
        <ol>
         <li>The input is <code>required</code> and initially empty, so on page load the input is <code>:invalid</code> and border is red. :( :(</li>
         <li>The input is <code>:invalid</code> until you type a legit email address, so it is red when typing. :( :(</li>
        </ol>
        <p>Ok, so the <code>:invalid</code> pseudo selector is kind of useless. Ideally we would not show an error up front or the whole time they are typing, just when they submit something unacceptable.</p>
        <p>Dave Rupert has a <a href="http://daverupert.com/2017/11/happier-html5-forms/">good solution</a> that uses an input's <code>invalid</code> event. Instead of using the <code>:invalid</code> pseudo selector, he adds a CSS class to the input when it becomes invalid. We will extend this approach by removing the CSS class when the element becomes valid again.</p>
        <p>The CSS</p>
        <pre><code class="lang-css">/* .invalid CSS class instead or pseudo selector */
input.invalid {
  border-color: red;
}
</code></pre>
        <p>And the JavaScript will add the invalid class when the input is invalid</p>
        <pre><code class="lang-js">var invalidClassName = 'invalid'
var inputs = document.querySelectorAll('input, select, textarea')
inputs.forEach(function (input) {
  // Add a css class on submit when the input is invalid.
  input.addEventListener('invalid', function () {
    input.classList.add(invalidClassName)
  })

  // Remove the class when the input becomes valid.
  // 'input' will fire each time the user types
  input.addEventListener('input', function () {
    if (input.validity.valid) {
      input.classList.remove(invalidClassName)
    }
  })
})
</code></pre>

    border-color: red;
  }</style>

        <h2 id="example-3-custom-messages">Example 3: Custom Messages</h2>
        <p>The browser provides default error messages. But they are super generic—"Please fill out this field", etc.—and may not be what you want.</p>
        <p>Generic as they may be, there is an upside to sticking with the default error messages: <strong>default errors are translated into the user's local language</strong>. e.g. If a user is reading your site in German, the default error messages will be in German also.</p>
        <p>The key to custom messages is using an input HTMLElement's <code>setCustomValidity</code> function, e.g. <code>input.setCustomValidity('My custom message')</code>. You can call this function at any time. If you pass a non-empty string, it will treat the input as invalid and not allow form submission. It is a flexible function. For example, it can be used to build your own <a href="https://developer.mozilla.org/en-US/docs/Web/Guide/HTML/HTML5/Constraint_validation#Complex_constraints_using_HTML5_Constraint_API">validation for custom types</a>.</p>
        <p>But <code>setCustomValidity</code> is a bit cumbersome for custom messages as you can't just override a <em>specific</em> message. It's "show this error no matter what" or nothing.</p>
        <p><code>&lt;input&gt;</code> HTMLElements also have a <code>validity</code> object that tells you what, if anything, on the element is invalid. We can use <code>input.validity</code> in conjunction with <code>input.setCustomValidity()</code> to override specific messages.</p>
        <pre><code class="lang-js">// The keys (e.g. valueMissing) map onto
// a key in the `input.validity` object
const customMessages = {
  valueMissing:    'Custom required!',       // `required` attr
  emailMismatch:   'Custom email mismatch',  // Invalid email
  patternMismatch: 'Custom pattern mismatch',// `pattern` attr
}

function getCustomMessage (type, validity) {
  if (validity.typeMismatch) {
    return customMessages[`${type}Mismatch`]
  } else {
    for (const invalidKey in customMessages) {
      if (validity[invalidKey]) {
        return customMessages[invalidKey]
      }
    }
  }
}

var inputs = document.querySelectorAll('input, select, textarea')
inputs.forEach(function (input) {
  // Each time the user types or submits, this will
  // check validity, and set a custom message if invalid.
  function checkValidity () {
    const message = input.validity.valid
      ? null
      : getCustomMessage(input.type, input.validity, customMessages)
    input.setCustomValidity(message || '')
  }
  input.addEventListener('input', checkValidity)
  input.addEventListener('invalid', checkValidity)
})
</code></pre>
