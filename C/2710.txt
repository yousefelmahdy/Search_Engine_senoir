       <h3 id="about-graphql"><a href="#about-graphql">About GraphQL</a></h3> 
       <p>The <a href="https://graphql.github.io/">GraphQL</a> data query language is:</p> 
       <ul> 
        <li> <p><strong>A <a href="https://graphql.github.io/graphql-spec/June2018/">specification</a>.</strong> The spec determines the validity of the <a href="/en/graphql/guides/introduction-to-graphql#schema">schema</a> on the API server. The schema determines the validity of client calls.</p> </li> 
        <li> <p><strong><a href="#about-the-graphql-schema-reference">Strongly typed</a>.</strong> The schema defines an API's type system and all object relationships.</p> </li> 
        <li> <p><strong><a href="/en/graphql/guides/introduction-to-graphql#discovering-the-graphql-api">Introspective</a>.</strong> A client can query the schema for details about the schema.</p> </li> 
        <li> <p><strong><a href="/en/graphql/guides/forming-calls-with-graphql">Hierarchical</a>.</strong> The shape of a GraphQL call mirrors the shape of the JSON data it returns. <a href="/en/graphql/guides/migrating-from-rest-to-graphql#example-nesting">Nested fields</a> let you query for and receive only the data you specify in a single round trip.</p> </li> 
        <li> <p><strong>An application layer.</strong> GraphQL is not a storage model or a database query language. The <em>graph</em> refers to graph structures defined in the schema, where <a href="/en/graphql/guides/introduction-to-graphql#node">nodes</a> define objects and <a href="/en/graphql/guides/introduction-to-graphql#edge">edges</a> define relationships between objects. The API traverses and returns application data based on the schema definitions, independent of how the data is stored.</p> </li> 
       </ul> 
       <h3 id="why-github-is-using-graphql"><a href="#why-github-is-using-graphql">Why GitHub is using GraphQL</a></h3> 
       <p>GitHub chose GraphQL for our API v4 because it offers significantly more flexibility for our integrators. The ability to define precisely the data you want—and <em>only</em> the data you want—is a powerful advantage over the REST API v3 endpoints. GraphQL lets you replace multiple REST requests with <em>a single call</em> to fetch the data you specify.</p> 
       <p>For more details about why GitHub has moved to GraphQL, see the original <a href="https://githubengineering.com/the-github-graphql-api/">announcement blog post</a>.</p> 
       <h3 id="about-the-graphql-schema-reference"><a href="#about-the-graphql-schema-reference">About the GraphQL schema reference</a></h3> 
       <p>The docs in the sidebar are generated from the GitHub GraphQL <a href="/en/graphql/guides/introduction-to-graphql#discovering-the-graphql-api">schema</a>. All calls are validated and executed against the schema. Use these docs to find out what data you can call:</p> 
       <ul> 
        <li> <p>Allowed operations: <a href="/en/graphql/reference/queries">queries</a> and <a href="/en/graphql/reference/mutations">mutations</a>.</p> </li> 
        <li> <p>Schema-defined types: <a href="/en/graphql/reference/scalars">scalars</a>, <a href="/en/graphql/reference/objects">objects</a>, <a href="/en/graphql/reference/enums">enums</a>, <a href="/en/graphql/reference/interfaces">interfaces</a>, <a href="/en/graphql/reference/unions">unions</a>, and <a href="/en/graphql/reference/input-objects">input objects</a>.</p> </li> 
       </ul> 
       <p>You can access this same content via the <a href="/en/graphql/guides/using-the-explorer#accessing-the-sidebar-docs">Explorer Docs sidebar</a>. Note that you may need to rely on both the docs and the schema validation to successfully call the GraphQL API.</p> 
       <p>For other information, such as authentication and rate limit details, check out the <a href="/en/graphql/guides">guides</a>.</p> 
       <h3 id="requesting-support"><a href="#requesting-support">Requesting support</a></h3> 
       <p>For questions, bug reports, and discussions about GitHub Apps, OAuth Apps, and API development, explore the <a href="https://github.community/c/github-api-development-and-support/37">GitHub API Development and Support Forum</a>. The forum is moderated and maintained by GitHub staff, but questions posted to the forum are not guaranteed to receive a reply from GitHub staff.</p> 
       <p>Consider reaching out to <a href="https://github.com/contact">GitHub Support</a> directly using the contact form for:</p> 
       <ul> 
        <li>guaranteed response from GitHub staff</li> 
        <li>support requests involving sensitive data or private concerns</li> 
        <li>feature requests</li> 
        <li>feedback about GitHub products</li> 
       </ul>
