       <h3 id="about-commits"><a href="#about-commits">About commits</a></h3> 
       <p>Similar to saving a file that's been edited, a commit records changes to one or more files in your branch. Git assigns each commit a unique ID, called a SHA or hash, that identifies:</p> 
       <ul> 
        <li>The specific changes</li> 
        <li>When the changes were made</li> 
        <li>Who created the changes</li> 
       </ul> 
       <p>When you make a commit, you must include a commit message that briefly describes the changes. You can also add a co-author on any commits you collaborate on.</p> 
       <p>If the commits you make in GitHub Desktop are associated with the wrong account on GitHub, update the email address in your Git configuration using GitHub Desktop. For more information, see <a href="/en/desktop/getting-started-with-github-desktop/configuring-git-for-github-desktop">"Configuring Git for GitHub Desktop</a>."</p> 
       <h3 id="choosing-a-branch-and-making-changes"><a href="#choosing-a-branch-and-making-changes">Choosing a branch and making changes</a></h3> 
       <ol> 
        <li> <p><a href="/en/desktop/guides/contributing-to-projects/managing-branches">Create a new branch</a>, or select an existing branch by clicking

          </svg> <strong>Current Branch</strong> on the toolbar and selecting the branch from the list.</p> <p><span class="procedural-image-wrapper"><img src="/assets/images/help/desktop/select-branch-from-dropdown.png" alt="Drop down menu to switch your current branch"></span></p> </li> 
        <li> <p>Using your favorite text editor, such as <a href="https://atom.io/">Atom</a>, make the necessary changes to files in your project.</p> </li>

       <h4 id="creating-a-partial-commit"><a href="#creating-a-partial-commit">Creating a partial commit</a></h4> 
       <p>If one file contains multiple changes, but you only want some of those changes to be included in a commit, you can create a partial commit. The rest of your changes will remain intact, so that you can make additional modifications and commits. This allows you to make separate, meaningful commits, such as keeping line break changes in a commit separate from code or prose changes.</p> 
       <p>To exclude changed lines from your commit, click one or more changed lines so the blue disappears. The lines that are still highlighted in blue will be included in the commit.</p> 
       <p> <img src="/assets/images/help/desktop/partial-commit.png" alt="Unselected lines in a file"></p> 
       <h3 id="discarding-changes"><a href="#discarding-changes">Discarding changes</a></h3> 
       <p>If you have uncommitted changes that you don't want to keep, you can discard the changes. This will remove the changes from the files on your computer. You can discard all uncommitted changes in one or more files, or you can discard specific lines you added.</p> 
       <p>Discarded changes are saved in a dated file in the Trash. You can recover discarded changes until the Trash is emptied.</p> 
       <h4 id="discarding-changes-in-one-or-more-files"><a href="#discarding-changes-in-one-or-more-files">Discarding changes in one or more files</a></h4> 
       <ol> 
        <li> <p>In the list of changed files, select the files where you want to discard the changes since the last commit. To select multiple files, click <code>shift</code> and click on the range of files you want to discard changes from. <span class="procedural-image-wrapper"><img src="/assets/images/help/desktop/select-multiple-files-discard.png" alt="Select multiple files with changes to discard"></span></p> </li> 
        <li> <p>Click <strong>Discard Changes</strong> or <strong>Discard Selected Changes</strong> to discard changes to one or more files, or <strong>Discard All Changes</strong> to discard changes to all files since the last commit.</p> <p><span class="procedural-image-wrapper"><img src="/assets/images/help/desktop/discard-changes-mac.png" alt="Discard Changes option in context menu"></span></p> </li> 
        <li> <p>To confirm the changes, review the files affected and click <strong>Discard Changes</strong>.</p> <p><span class="procedural-image-wrapper"><img src="/assets/images/help/desktop/discard-changes-confirm-mac.png" alt="Discard Changes button in the confirmation dialog"></span></p> </li> 
       </ol> 
       <h4 id="discarding-changes-in-one-or-more-lines"><a href="#discarding-changes-in-one-or-more-lines">Discarding changes in one or more lines</a></h4> 
       <p>You can discard one or more changed lines that are uncommitted.</p> 
       <div class="extended-markdown note border rounded-1 mb-4 p-3 color-border-info color-bg-info f5"> 
        <p><strong>Note:</strong> Discarding single lines is disabled in a group of changes that adds and removes lines.</p> 
       </div> 
       <p>To discard one added line, in the list of changed lines, right click on the line you want to discard and select <strong>Discard added line</strong>.</p> 
       <p> <img src="/assets/images/help/desktop/discard-single-line.png" alt="Discard single line in the confirmation dialog"></p> 
       <p>To discard a group of changed lines, right click the vertical bar to the right of the line numbers for the lines you want to discard, then select <strong>Discard added lines</strong>.</p> 
       <p> <img src="/assets/images/help/desktop/discard-multiple-lines.png" alt="Discard a group of added lines in the confirmation dialog"></p> 
       <h3 id="write-a-commit-message-and-push-your-changes"><a href="#write-a-commit-message-and-push-your-changes">Write a commit message and push your changes</a></h3> 
       <p>Once you're satisfied with the changes you've chosen to include in your commit, write your commit message and push your changes. If you've collaborated on a commit, you can also attribute a commit to more than one author.</p> 
       <div class="extended-markdown note border rounded-1 mb-4 p-3 color-border-info color-bg-info f5"> 
        <p><strong>Note</strong>: By default, GitHub Desktop will push the tag that you create to your repository with the associated commit. For more information, see "<a href="/en/desktop/contributing-to-projects/managing-tags">Managing tags</a>."</p> 
       </div> 
       <ol> 
        <li> <p>At the bottom of the list of changes, in the Summary field, type a short, meaningful commit message. Optionally, you can add more information about the change in the Description field.</p> <p><span class="procedural-image-wrapper"><img src="/assets/images/help/desktop/commit-message.png" alt="Commit message field"></span></p> </li> 
        <li> <p>Optionally, to attribute a commit to another author, click the add co-authors icon and type the username(s) you want to include.</p> <p><span class="procedural-image-wrapper"><img src="/assets/images/help/desktop/add-co-author-commit.png" alt="Add a co-author to the commit message"></span></p> </li> 
        <li> <p>Under the Description field, click <strong>Commit to <em>BRANCH</em></strong>.</p> <p><span class="procedural-image-wrapper"><img src="/assets/images/help/desktop/commit-button.png" alt="Commit button"></span></p> </li> 
        <li> <p>If the branch you're trying to commit to is protected, Desktop will warn you.</p> 
         <ul> 
          <li>To move your changes, click <strong>switch branches</strong>.</li>
