   <section id="faq" class="section" aria-label="Veelgestelde vragen" data-module="faq-01"> 
    <div class="row"> 
     <div class="column medium-3"> 
      <h2>Veelgestelde vragen</h2> 
     </div> 
     <div class="column medium-8 medium-offset-1"> 
      <ul class="faq"> 
       <li class="faq__item"> <button type="button" class="faq__question" aria-expanded="false" aria-controls="d1e6c7ff" data-bi-slot="1" data-bi-id="faq-1" data-bi-an="faq-01"> Wat is Azure Cosmos DB? </button> 
        <div class="faq__answer" id="d1e6c7ff">
         <p>Azure Cosmos DB is een wereldwijd gedistribueerde multimodeldatabaseservice. De service is vanaf de basis ontworpen voor wereldwijde distributie en horizontaal schalen.</p>
         <p>Azure Cosmos DB biedt kant-en-klare wereldwijde distributie in elk gewenste aantal Azure-regio's door uw gegevens op transparante wijze te schalen en te repliceren op de locaties van uw gebruikers. Schaal uw schrijf- en leesacties wereldwijd elastisch en betaal alleen voor wat u nodig hebt. Azure Cosmos DB biedt systeemeigen ondersteuning voor NoSQL- en OSS-API's, waaronder MongoDB, Cassandra, Gremlin, etcd, Apache Spark en SQL. Het biedt ook meerdere goed gedefinieerde consistentiemodellen, gegarandeerde lees- en schrijflatentie van slechts enkele milliseconden in het 99e percentiel en een hoge beschikbaarheid van 99,999 procent met multihoming waar ook ter wereld. Al deze garanties worden ondersteund door marktleidende, uitgebreide SLAs.</p>
         <p>Azure Cosmos DB is een volledig beheerde service waardoor u geen gedistribueerde databases meer hoeft te schalen en te beheren in Azure. U hoeft zich dus niet te bekommeren om het beheer van VM's, hardware-inrichting, setup en configuratie, capaciteit, replicatie, softwarepatches en clusterschaling.</p>
        </div> </li> 
       <li class="faq__item"> <button type="button" class="faq__question" aria-expanded="false" aria-controls="79255e1a" data-bi-slot="2" data-bi-id="faq-2" data-bi-an="faq-01"> Wat is kant-en-klare wereldwijde distributie? </button> 
        <div class="faq__answer" id="79255e1a">
         <p>Met Azure Cosmos DB kunt u uw database zodanig configureren dat deze wereldwijd in alle gewenste Azure-regio's wordt gedistribueerd en beschikbaar is. U kunt latentie minimaliseren door gegevens dicht bij uw gebruikers te plaatsen.</p>
         <p>Met Azure Cosmos DB worden uw gegevens transparant gerepliceerd in alle regio's die aan uw Azure Cosmos DB-account zijn gekoppeld. Het biedt een enkele systeeminstallatiekopie van uw wereldwijd gedistribueerde Azure Cosmos DB-database en containers waarin gegevens lokaal kunnen worden gelezen en geschreven door uw toepassing.</p>
         <p>Met kant-en-klare wereldwijde distributie kunt u op elk gewenst moment regio's die aan uw account zijn gekoppeld, toevoegen of verwijderen. Uw toepassing hoeft niet te worden onderbroken of opnieuw te worden geïnstalleerd als u een regio wilt toevoegen of verwijderen. Hoge beschikbaarheid wordt gehandhaafd met systeemeigen multihomingmogelijkheden van de service. <a href="https://docs.microsoft.com/nl-nl/azure/cosmos-db/distribute-data-globally">Meer informatie</a>.</p>
         <p></p>
        </div> </li> 
       <li class="faq__item"> <button type="button" class="faq__question" aria-expanded="false" aria-controls="4309260b" data-bi-slot="3" data-bi-id="faq-3" data-bi-an="faq-01"> Hoe wordt de consistentie beheerd in Azure Cosmos DB? </button> 
        <div class="faq__answer" id="4309260b">
         <p>Azure Cosmos DB benadert gegevensconsistentie als een spectrum aan keuzen met meer opties dat de twee uitersten: sterke en mogelijke consistentie.</p>
         <p>Kies uit vijf goed gedefinieerde modellen in het consistentiespectrum:</p> 
         <ul aria-label="Vijf goed gedefinieerde modellen">
          <li>Sterk</li>
          <li>Gebonden veroudering</li>
          <li>Sessie</li>
          <li>Consistent voorvoegsel</li>
          <li>Mogelijk</li>
         </ul>
         <p>Elk model wordt ondersteund door uitgebreide SLA's. <a href="https://docs.microsoft.com/nl-nl/azure/cosmos-db/consistency-levels-tradeoffs">Meer informatie.</a></p>
        </div> </li> 
       <li class="faq__item"> <button type="button" class="faq__question" aria-expanded="false" aria-controls="c247b93e" data-bi-slot="4" data-bi-id="faq-4" data-bi-an="faq-01"> Wat zijn de Azure Cosmos DB-SLA's? </button> 
        <div class="faq__answer" id="c247b93e">
         <p>Als wereldwijd gedistribueerde database biedt Azure Cosmos DB uitgebreide SLA's waarin doorvoer, latentie in het 99e percentiel, consistentie en hoge beschikbaarheid zijn opgenomen. <a href="/nl-nl/support/legal/sla/cosmos-db/">Geef meer informatie weer</a>.</p>
        </div> </li> 
       <li class="faq__item"> <button type="button" class="faq__question" aria-expanded="false" aria-controls="e1bf2db4" data-bi-slot="5" data-bi-id="faq-5" data-bi-an="faq-01"> Hoe worden in Azure Cosmos DB de beschikbaarheid, schaalbaarheid en duurzaamheid ondersteund? </button> 
        <div class="faq__answer" id="e1bf2db4">
         Met Azure Cosmos DB worden uw gegevens transparant gerepliceerd in alle Azure-regio's die aan uw Azure Cosmos DB-account zijn gekoppeld. Er worden meerdere redundantielagen op uw gegevens toegepast. <a href="https://docs.microsoft.com/nl-nl/azure/cosmos-db/high-availability">Meer informatie</a>.
        </div> </li> 
       <li class="faq__item"> <button type="button" class="faq__question" aria-expanded="false" aria-controls="51bfe9e5" data-bi-slot="6" data-bi-id="faq-6" data-bi-an="faq-01"> Wat is de Azure Cosmos DB-API voor MongoDB? </button> 
        <div class="faq__answer" id="51bfe9e5">
         De Azure Cosmos DB-API voor MongoDB biedt u de mogelijkheid om met Azure Cosmos DB te communiceren alsof het een MongoDB-database is, zonder dat u de database-infrastructuur zelf hoeft te beheren. Dit betekent dat bijna al uw favoriete MongoDB-hulpprogramma, SDK's en toepassingen blijven werken. Raadpleeg <a href="https://aka.ms/cosmosdb-mongo-latest-compat">onze documentatie</a> voor gedetailleerde informatie over compatibiliteit.
        </div> </li> 
       <li class="faq__button-container faq__item"> <button type="button" class="faq__question faq__load-more-button">Meer Veelgestelde vragen laden</button> </li> 
       <li class="faq__item"> <button type="button" class="faq__question" aria-expanded="false" aria-controls="739ead24" data-bi-slot="7" data-bi-id="faq-7" data-bi-an="faq-01"> Wat is de v4.0-ondersteuning voor de Azure Cosmos DB-API voor MongoDB? </button> 
        <div class="faq__answer" id="739ead24">
         Met ondersteuning voor serverversie 4.0 in de Azure Cosmos DB-API voor MongoDB is het eenvoudiger om complexe transactionele bedrijfslogica te modelleren zonder de complexe logica voor het verwerken van fouten. Met de ondersteuning van v 4.0 worden nieuwe functies, zoals transacties voor meerdere documenten, herhaalbare schrijfbewerkingen en meer, geïntroduceerd.
        </div> </li> 
       <li class="faq__item"> <button type="button" class="faq__question" aria-expanded="false" aria-controls="e553b780" data-bi-slot="8" data-bi-id="faq-8" data-bi-an="faq-01"> Wat is de Azure Cosmos DB Cassandra-API? </button> 
        <div class="faq__answer" id="e553b780">
         De Azure Cosmos DB Cassandra-API stelt u in staat om gegevens te gebruiken die zijn opgeslagen in Azure Cosmos DB. Dit doet u via de Cassandra-querytaal (CQL), hulpprogramma's op basis van de Cassandra-querytaal (bijvoorbeeld cqlsh) en bestaande Apache-stuurprogramma's die compatibel zijn met CQLv4. Dit betekent dat uw bestaande Cassandra-toepassing nu kan communiceren met de Azure Cosmos DB Cassandra-API. Binnenkort is de API ook verbonden met het beheerde exemplaar van Azure voor Apache Cassandra (bij algemene beschikbaarheid).
        </div> </li> 
      </ul>
