      <p>This page explains how to create the <b>Footnotes</b> section for Wikipedia articles. In this context, the word "Footnotes" refers to the Wikipedia-specific manner of documenting an article's sources and providing tangential information, and should not be confused with the general concept of <a href="/wiki/Note_(typography)" title="Note (typography)">footnotes</a>. This how-to does not cover the formatting of citations within the Footnotes section, which is reviewed in <a href="/wiki/Wikipedia:Citing_sources" title="Wikipedia:Citing sources">Citing sources</a>. </p>
      <p>Footnotes are used most commonly to provide: </p> 
      <ul>
       <li><a href="/wiki/Wikipedia:Citing_sources" title="Wikipedia:Citing sources">references (bibliographic citations) to reliable sources</a>,</li> 
       <li><a href="/wiki/Help:Explanatory_notes" title="Help:Explanatory notes">explanatory information</a> or</li> 
       <li>source information for tables and other elements.</li>
      </ul> 
      <p>Footnotes or <a href="/wiki/Help:Shortened_footnotes" title="Help:Shortened footnotes">shortened footnotes</a> may be used at the editor's discretion in accordance with the guideline on <a href="/wiki/Wikipedia:CITEVAR" class="mw-redirect" title="Wikipedia:CITEVAR">Variation in citation methods</a>. </p>
      <p>Only certain types of material on the English Wikipedia are required to have an inline citation. There is <a href="/wiki/Wikipedia:Inline_citation#Citation_density" title="Wikipedia:Inline citation">no requirement to provide a citation for every sentence</a>, because multiple sentences may be supported by the same footnote. For advice on which material should be cited, see the guidelines on <a href="/wiki/Wikipedia:MINREF" class="mw-redirect" title="Wikipedia:MINREF">When you must use inline citations</a>, the <a href="/wiki/Wikipedia:Good_article_criteria" title="Wikipedia:Good article criteria">Good article criteria</a> and <a href="/wiki/Wikipedia:When_to_cite" title="Wikipedia:When to cite">When to cite</a>. For advice on how to organize and format bibliographic citations, see the guideline on <a href="/wiki/Wikipedia:Citing_sources" title="Wikipedia:Citing sources">Citing sources</a> and examples of <a href="/wiki/Wikipedia:Citation_templates#Examples" title="Wikipedia:Citation templates">Citation templates</a>. </p>
      <p>Footnotes are created using the <a href="https://www.mediawiki.org/wiki/Extension:Cite" class="extiw" title="mw:Extension:Cite">Cite</a> software extension. This extension adds the HTML-like elements <code class="nowrap" style="">&lt;ref&gt;...&lt;/ref&gt;</code>, <code class="nowrap" style="">&lt;references /&gt;</code> and <code class="nowrap" style="">&lt;references&gt;...&lt;/references&gt;</code>. The elements are also used in a number of templates; for example, it is becoming more common to use {{<a href="/wiki/Template:Reflist" title="Template:Reflist">reflist</a>}} rather than <code class="nowrap" style="">&lt;references /&gt;</code> as it can style the reference list. </p>

      <h2><span class="mw-headline" id="Overview">Overview</span></h2> 
      <div class="plainlinks" style="border:1px solid #a7d7f9; padding: 0.8em; width: 25%; font-size: 87%; float:right;"> 
       <div class="floatleft">
        <a href="/wiki/File:VisualEditor-logo.svg" class="image"><img alt="VisualEditor-logo.svg" src="//upload.wikimedia.org/wikipedia/commons/thumb/1/12/VisualEditor-logo.svg/200px-VisualEditor-logo.svg.png" decoding="async" width="200" height="70" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/1/12/VisualEditor-logo.svg/300px-VisualEditor-logo.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/1/12/VisualEditor-logo.svg/400px-VisualEditor-logo.svg.png 2x" data-file-width="641" data-file-height="225"></a>

      <p>The Footnotes system shows two elements on the page: </p> 
      <ul>
       <li>A <b>Footnote marker</b> is displayed in the article's content as a bracketed, superscripted number, letter, or word. Examples shown respectively are: <sup class="reference nowrap ltr" style="color:#002bb8;">[1]</sup><sup class="reference nowrap ltr" style="color:#002bb8;">[a]</sup><sup class="reference nowrap ltr" style="color:#002bb8;">[Note 1]</sup>. This footnote label is linked to the full footnote. Clicking on the footnote marker jumps the page down to the footnote and highlights the citation. If you are using the desktop site and have Javascript enabled, then hovering your cursor over a footnote marker (or touching it on touch devices) will show a pop-up box containing the footnote.</li> 
       <li>A <b>Footnote</b> displays the full citation for the source. Together the footnotes are displayed in an ordered list wherever the reference list markup {{<a href="/wiki/Template:Reflist" title="Template:Reflist">reflist</a>}} is placed. Each entry begins with the footnote label in plain text. The entire reference list is formatted in a slightly smaller font.</li>
      </ul> 
      <dl>
       <dd>
        Each successive footnote label is automatically incremented. Thus the first footnote marker would be <sup class="reference nowrap ltr" style="color:#002bb8;">[1]</sup>, the second would be <sup class="reference nowrap ltr" style="color:#002bb8;">[2]</sup> and so on. Custom labels are also incremented: <sup class="reference nowrap ltr" style="color:#002bb8;">[a]</sup><sup class="reference nowrap ltr" style="color:#002bb8;">[b]</sup><sup class="reference nowrap ltr" style="color:#002bb8;">[c]</sup>, <sup class="reference nowrap ltr" style="color:#002bb8;">[Note 1]</sup> <sup class="reference nowrap ltr" style="color:#002bb8;">[Note 2]</sup> <sup class="reference nowrap ltr" style="color:#002bb8;">[Note 3]</sup>.
       </dd> 
       <dd>
        For a single use footnote, the label is followed by a caret (^) that is a backlink to the matching footnote marker. For example:
       </dd>
      </dl>

      <div class="dummy-footnote" style="margin-left: 5em;">
       1. <sup>^</sup> <span class="citation">Citation</span>
      </div> 
      <dl>
       <dd>
        If a named footnote is used in the text multiple times, then the footnote has multiple backlinks shown as letters:
       </dd>
      </dl> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1009970892"> 
      <div class="dummy-footnote" style="margin-left: 5em;">
       2. ^ <sup class="dummy-fn-ital">a b c</sup> <span class="citation">Citation</span>
      </div> 
      <dl>
       <dd>
        Clicking on the backlink or pressing <kbd class="keyboard-key nowrap" style="border: 1px solid #aaa; border-radius: 0.2em; box-shadow: 0.1em 0.1em 0.2em rgba(0,0,0,0.1); background-color: #f9f9f9; background-image: linear-gradient(to bottom, #eee, #f9f9f9, #eee); color: #000; padding: 0.1em 0.3em; font-family: inherit; font-size: 0.85em;">Alt</kbd>+<kbd class="keyboard-key nowrap" style="border: 1px solid #aaa; border-radius: 0.2em; box-shadow: 0.1em 0.1em 0.2em rgba(0,0,0,0.1); background-color: #f9f9f9; background-image: linear-gradient(to bottom, #eee, #f9f9f9, #eee); color: #000; padding: 0.1em 0.3em; font-family: inherit; font-size: 0.85em;">?</kbd> returns to the footnote marker.

      <h2><span class="mw-headline" id="Footnotes:_using_a_source_more_than_once">Footnotes: using a source more than once</span></h2> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:REFNAME"></span><span id="WP:NAMEDREFS"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcuts</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:REFNAME&amp;redirect=no">WP:REFNAME</a></span></li>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:NAMEDREFS&amp;redirect=no">WP:NAMEDREFS</a></span></li>
       </ul>
      </div> 
      <p><span class="anchor" id="Reference_name_(naming_a_ref_tag_so_it_can_be_used_more_than_once)"></span> <span class="anchor" id="WP:NAMEDREF"></span> </p> 
      <div role="note" class="hatnote navigation-not-searchable">
       See also: <a href="/wiki/Wikipedia:Citing_sources#Citing_multiple_pages_of_the_same_source" title="Wikipedia:Citing sources">Wikipedia:Citing sources §&nbsp;Citing multiple pages of the same source</a>
      </div> 
      <p>You can cite the same source more than once on a page by using named footnotes. The syntax to define a named footnote is: </p> 
      <dl>
       <dd>
        <code class="nowrap" style="">&lt;ref name="<var>name</var>"&gt;<var>content</var>&lt;/ref&gt;</code>
       </dd>
      </dl> 
      <p>To invoke the named footnote: </p> 
      <dl>
       <dd>
        <code class="nowrap" style="">&lt;ref name="<var>name</var>" /&gt;</code>
       </dd>
      </dl> 
      <p>Names for footnotes and groups must follow these rules: </p> 
      <ul>
       <li>Names are case-sensitive. Please do not use raNdOM capitalization.</li> 
       <li>Names <strong>must not</strong> be purely numeric; the software will accept something like <code>":1"</code> (which is punctuation plus a number), but it will ignore <code>"1"</code> (purely numeric).</li> 
       <li>Names should have semantic value, so that they can be more easily distinguished from each other by human editors who are looking at the wikitext. This means that ref names like <code>"Nguyen 2010"</code> are preferred to names like <code>":1"</code>.</li> 
       <li>Names must be unique. You may not use the same name to define different groups or footnotes.</li> 
       <li>Please consider keeping reference names simple and restricted to the standard <a href="/wiki/English_alphabet" title="English alphabet">English alphabet</a> and numerals. Failing that, if spaces are used, the following technical restrictions become relevant: 
        <ul>
         <li>Quotation marks are preferred but optional if the only characters used are letters <code>A–Z</code>, <code>a–z</code>, digits <code>0–9</code>, and the symbols <code style="word-break: break-all;">!$%&amp;()*,-.:;&lt;@[]^_`{|}~</code>. That is, all printable ASCII characters except <code style="word-break: break-all;">#"'/=&gt;?\</code>.</li> 
         <li>Inclusion of any other characters, <em>including spaces</em>, requires that the reference name be enclosed in quotes; for example, <code>name="John Smith"</code>.</li> 
         <li>The quote marks must be the standard, straight, double quotation marks (<code>"</code>); curly or other quotes will be parsed as part of the reference name.</li> 
         <li>Quote-enclosed reference names may not include a less-than sign (<code>&lt;</code>) or a double straight quote symbol (<code>"</code>). These may be escaped with <code>&amp;lt;</code> and <code>&amp;quot;</code>, respectively.</li>
        </ul></li> 
       <li>You may optionally provide reference names even when the reference name is not required. This makes later re-use of the sourced reference easier.</li>
      </ul>

      </dl>

        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:CITELABEL&amp;redirect=no">WP:CITELABEL</a></span></li>
       </ul>
      </div> 
      <p>Sometimes it is useful to group the footnotes into separate lists, for example to separate explanatory notes from references, or to list references for tables, image captions, infoboxes and navboxes. The sequence of footnote labels is independent in each group. </p>
      <p>The syntax to define a footnote with a group is: </p> 
      <dl>
       <dd>
        <code class="nowrap" style="">&lt;ref group=<var>groupname</var>&gt;Content&lt;/ref&gt;</code>
       </dd>
      </dl> 
      <p>The footnote marker group name <i>must</i> be enclosed in quotes if the group name includes a space, else a cite error will occur, otherwise quotes are optional. </p>
      <p>The syntax for the reference list with a group is: </p> 
      <dl>
       <dd>
        <code>{{<a href="/wiki/Template:Reflist" title="Template:Reflist">reflist</a>|group=<var>groupname</var>}}</code>
       </dd>
      </dl> 
      <p>The reference list group name does not need to be enclosed in quotes. </p>

      <p><span class="anchor" id="footnotes_predefined_groups"></span> There are several predefined groups that can have a reference list styled so that the label (a superscripted character within square brackets, e.g., <sup class="reference nowrap ltr" style="color:#002bb8;">[1]</sup>) of an <a href="/wiki/Help:Explanatory_notes" title="Help:Explanatory notes">explanatory note</a> or citation (aka footnote, reference) matches and links to the note marker label located in the main text and the label in front of the note's text in the appropriate group's list. There can be <a href="/wiki/Wikipedia:Citing_sources#Separating_citations_from_explanatory_footnotes" title="Wikipedia:Citing sources">more than one of these groups' lists</a> in the Notes, References and other similarly purposed sections. </p>
      <p>These predefined note and citation groups have templates that make the <a href="/wiki/Wikitext" class="mw-redirect" title="Wikitext">wikitext</a> markup simpler. These templates also allow a standard reference to be inserted, so that an explanatory note can have a reference, or citations can be nested. </p>
      <p><b>Note:</b> If the note's text has a reference name that is used more than once, the labels will still match, but the clickable alpha characters that toggle the note's display will be <i>next to</i> the note's label, with links to the multiple locations of its marker in the main text. See <a href="/wiki/Help:Footnotes#Footnotes:_using_a_source_more_than_once" title="Help:Footnotes">WP:REFNAME</a> and the first point below in <a href="#Issues">§&nbsp;Issues</a>. </p>

      <h3><span class="mw-headline" id="Issues">Issues</span></h3> 
      <ul>
       <li>Multiple backlink labels (see <a href="/wiki/Wikipedia:REFNAME" class="mw-redirect" title="Wikipedia:REFNAME">WP:REFNAME</a>) are always styled as lower-alpha. This can be confusing when <code>{{<a href="/wiki/Template:Efn" title="Template:Efn">efn</a>}}</code> is used. See the note in the introduction of <a href="#footnotes_predefined_groups">this section</a>, above.</li> 
       <li>The entries in the reference list will show the default decimal styling if: 
        <ul>
         <li>The value for <code class="nowrap" style="">|group=</code> in <code>{{<a href="/wiki/Template:Reflist" title="Template:Reflist">reflist</a>}}</code> or <code class="nowrap" style="">&lt;references /&gt;</code> does not exactly match one of the predefined values; use <code>{{<a href="/wiki/Template:Efn" title="Template:Efn">efn</a>}}</code> or variant as <code class="nowrap" style="">|group=</code> will not be needed.</li> 
         <li>The value for <code class="nowrap" style="">|group=</code> in <code>{{<a href="/wiki/Template:Reflist" title="Template:Reflist">reflist</a>}}</code> or <code class="nowrap" style="">&lt;references /&gt;</code> is enclosed by quotes; use <code>{{<a href="/wiki/Template:Efn" title="Template:Efn">efn</a>}}</code> or variant as <code class="nowrap" style="">|group=</code> will not be needed.</li> 
         <li><code>{{<a href="/wiki/Template:Reflist" title="Template:Reflist">reflist</a>}}</code> is indented with the <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">:</code> markup.</li> 
         <li><code class="nowrap" style="">&lt;references /&gt;</code> is used instead of <code>{{<a href="/wiki/Template:Reflist" title="Template:Reflist">reflist</a>}}</code></li>
        </ul></li> 
       <li>The entries in the reference list will have no styling if: 
        <ul>
         <li>A reference is included in <code>{{<a href="/wiki/Template:Navbox" title="Template:Navbox">navbox</a>}}</code> or one of the variants and the style is set to <code>bodyclass=hlist</code>; to resolve this, replace the style with <code>listclass=hlist</code>.</li>
        </ul></li> 
       <li>If the note text contains an "=" sign, precede the note text with <code class="nowrap" style="">|1=</code>, like this: <code>{{efn|name=fn1|1=Very long footnote with an equation, 2+2=4}}</code></li>
      </ul> 
      <h2><span class="mw-headline" id="Reference_lists:_automatically_generated">Reference lists: automatically generated</span></h2> 
      <div role="note" class="hatnote navigation-not-searchable">
       Main page: <a href="/wiki/Help:Automatically_generated_reference_list" title="Help:Automatically generated reference list">Help:Automatically generated reference list</a>
      </div> 
      <p>A page with <code class="nowrap" style="">&lt;ref&gt;</code> tags but no reference list markup used to display an error, except on talk and user pages. In recent versions of Wikipedia, the error no longer appears; instead an automatically generated reference list (AGRL) is displayed at the bottom of the page. </p>
      <p>Compared to the reference lists on properly formatted pages, an AGRL can be confusing to both readers and editors. But it is easily corrected by adding reference list markup such as the following. Add it at the position where the reference list would normally appear. </p> 
      <pre style="width: 10em;">==References==
{{reflist}}
</pre> 
      <p>On talk pages, the {{<a href="/wiki/Template:Reflist-talk" title="Template:Reflist-talk">reflist-talk</a>}} template can be used to add a reference list to a specific discussion. </p>
      <p><span class="anchor" id="Citing_one_book_repeatedly_with_different_page_numbers"></span> </p> 
      <h2><span class="mw-headline" id="Footnotes:_page_numbers">Footnotes: page numbers</span></h2> 
      <div role="note" class="hatnote navigation-not-searchable">
       Main page: <a href="/wiki/Help:References_and_page_numbers" title="Help:References and page numbers">Help:References and page numbers</a>
      </div> 
      <p>Suppose you would like to cite one book, but different facts appear on different pages. You would like to cite the book again and again, but point each fact to the proper page. Suppose one fact is on page 8, a different fact on page 12, a third fact on page 18, a fourth fact on page 241. You could put a line in the "pages" parameter saying "see pages 8, 12, 18, 241" but a fact-checker might have to check all of them before figuring out the right one. Or, you could duplicate the entire citation for the book in each instance, but that would be redundant. </p>
      <p>One common approach is to use <a href="/wiki/Help:Shortened_footnotes" title="Help:Shortened footnotes">shortened citations</a>. The long citation to support the shortened citations can either be placed as a bullet point in a separate <i>References section</i> after the <i>Footnotes section</i>; or it can be placed in the first footnote to cite the source (with the initial relevant page number[s]). The remaining footnotes will use shortened citations (these usually contain the author's last name, the date of publication, and the relevant page number[s]). </p>
      <p>A less common approach is to attach a <code>{{<a href="/wiki/Template:Rp" title="Template:Rp">rp</a>|8}}</code> right after the footnote marker and replace the "8" with the appropriate page number. For example: </p>

      <h2><span class="mw-headline" id="Flagging_inadequate_referencing">Flagging inadequate referencing</span></h2> 
      <div role="note" class="hatnote navigation-not-searchable">
       See also: <a href="/wiki/Template:Unreferenced/doc#Usage" title="Template:Unreferenced/doc">Template: Unreferenced/doc §&nbsp;Usage</a>
      </div> 
      <ul>
       <li>The template <code>{{<a href="/wiki/Template:Refimprove" class="mw-redirect" title="Template:Refimprove">Refimprove</a>}}</code> should be added to articles where there are <b>some, but insufficient, inline citations to support the material</b> currently in the article. For <a href="/wiki/Wikipedia:BLP" class="mw-redirect" title="Wikipedia:BLP">biographies of living persons</a>, use <code>{{<a href="/wiki/Template:BLP_sources" title="Template:BLP sources">BLP sources</a>}}</code> instead.</li> 
       <li>The template <code>{{<a href="/wiki/Template:Unreferenced" title="Template:Unreferenced">Unreferenced</a>}}</code> should be added to articles with <b>no references at all</b>. For biographies of living persons, use <code>{{<a href="/wiki/Template:BLP_unsourced" title="Template:BLP unsourced">BLP unsourced</a>}}</code> instead.</li> 
       <li>The template <code>{{<a href="/wiki/Template:Unreferenced_section" title="Template:Unreferenced section">Unreferenced section</a>}}</code> is for <b>individual sections that have no citations</b> in articles that have at least one citation somewhere else. For biographies of living persons, use <code>{{<a href="/wiki/Template:BLP_unsourced_section" title="Template:BLP unsourced section">BLP unsourced section</a>}}</code> instead.</li> 
       <li>The template <code>{{<a href="/wiki/Template:Medref" class="mw-redirect" title="Template:Medref">Medref</a>}}</code> is specifically for articles <b>needing additional <i>medical</i> references</b>.</li>
      </ul>
