         <h1 id="groups">Groups <span class="badge-trigger free"></span><a href="#groups" title="Permalink" class="anchor"></a> </h1> 
         <p>In GitLab, you can put related projects together in a group. </p>
         <p>For example, you might create a group for your company members and a subgroup for each individual team. You can name the group <code class="highlighter-rouge">company-team</code>, and the subgroups <code class="highlighter-rouge">backend-team</code>, <code class="highlighter-rouge">frontend-team</code>, and <code class="highlighter-rouge">production-team</code>. </p>
         <p>Then you can: </p>
         <ul> 
          <li>Grant members access to multiple projects at once. </li>
          <li>Add to-do items for all of the group members at once. </li>
          <li>View the <a href="../project/issues/index.html#issues-list">issues</a> and <a href="../project/merge_requests/reviewing_and_managing_merge_requests.html#view-merge-requests-for-all-projects-in-a-group">merge requests</a> for all projects in the group, together in a single list view. </li>
          <li> <a href="../group/bulk_editing/index.html">Bulk edit</a> issues, epics, and merge requests. </li>
         </ul> 
         <p>You can also create <a href="subgroups/index.html">subgroups</a>. </p>
         <h2 id="view-groups">View groups<a href="#view-groups" title="Permalink" class="anchor"></a> </h2> 
         <p>To view groups: </p>
         <ol> 
          <li>In the top menu, select <strong>Groups &gt; Your Groups</strong>. All groups you are a member of are displayed. </li>
          <li>To view a list of public groups, select <strong>Explore public groups</strong>. </li>
         </ol> 
         <p>You can also view groups by namespace. </p>
         <h3 id="namespaces">Namespaces<a href="#namespaces" title="Permalink" class="anchor"></a> </h3> 
         <p>In GitLab, a namespace is a unique name and URL for a user, a group, or subgroup. </p>
         <ul> 
          <li><code class="highlighter-rouge">http://gitlab.example.com/username</code> </li>
          <li><code class="highlighter-rouge">http://gitlab.example.com/groupname</code> </li>
          <li><code class="highlighter-rouge">http://gitlab.example.com/groupname/subgroup_name</code> </li>
         </ul> 
         <p>For example, consider a user named Alex: </p>
         <ol> 
          <li>Alex creates an account with the username <code class="highlighter-rouge">alex</code>: <code class="highlighter-rouge">https://gitlab.example.com/alex</code> </li>
          <li>Alex creates a group for their team with the group name <code class="highlighter-rouge">alex-team</code>. The group and its projects are available at: <code class="highlighter-rouge">https://gitlab.example.com/alex-team</code> </li>
          <li>Alex creates a subgroup of <code class="highlighter-rouge">alex-team</code> with the subgroup name <code class="highlighter-rouge">marketing</code>. The subgroup and its projects are available at: <code class="highlighter-rouge">https://gitlab.example.com/alex-team/marketing</code> </li>
         </ol> 
         <h2 id="create-a-group">Create a group<a href="#create-a-group" title="Permalink" class="anchor"></a> </h2> 
         <p>To create a group: </p>
         <ol> 
          <li>From the top menu, either: 
           <ul> 
            <li>Select <strong>Groups &gt; Your Groups</strong>, and on the right, select the <strong>New group</strong> button. </li>
            <li>To the left of the search box, select the plus sign and then <strong>New group</strong>. </li>
           </ul> </li>
          <li>For the <strong>Group name</strong>, use only: 
           <ul> 
            <li>Alphanumeric characters </li>
            <li>Emojis </li>
            <li>Underscores </li>
            <li>Dashes, dots, spaces, and parentheses (however, it cannot start with any of these characters) </li>
           </ul> <p>For a list of words that cannot be used as group names, see <a href="../reserved_names.html">reserved names</a>. </p></li>
          <li>For the <strong>Group URL</strong>, which is used for the <a href="#namespaces">namespace</a>, use only: 
           <ul> 
            <li>Alphanumeric characters </li>
            <li>Underscores </li>
            <li>Dashes and dots (it cannot start with dashes or end in a dot) </li>
           </ul> </li>
          <li>Choose the <a href="../../public_access/public_access.html">visibility level</a>. </li>
          <li>Invite GitLab members or other users to join the group. </li>
         </ol> 
         <p><i class="fa fa-youtube-play youtube" aria-hidden="true"></i> For details about groups, watch <a href="https://youtu.be/r0sJgjR2f5A">GitLab Namespaces (users, groups and subgroups)</a>. </p>
         <h2 id="add-users-to-a-group">Add users to a group<a href="#add-users-to-a-group" title="Permalink" class="anchor"></a> </h2> 
         <p>You can give a user access to all projects in a group. </p>
         <ol> 
          <li>From the top menu, select <strong>Groups &gt; Your Groups</strong>. </li>
          <li>Find your group and select it. </li>
          <li>From the left sidebar, select <strong>Members</strong>. </li>
          <li>Fill in the fields. 
           <ul> 
            <li>The role applies to all projects in the group. <a href="../permissions.html#permissions">Learn more about permissions</a>. </li>
            <li>On the <strong>Access expiration date</strong>, the user can no longer access projects in the group. </li>
           </ul> </li>
         </ol> 
         <h2 id="request-access-to-a-group">Request access to a group<a href="#request-access-to-a-group" title="Permalink" class="anchor"></a> </h2> 
         <p>As a user, you can request to be a member of a group, if an administrator allows it. </p>
         <ol> 
          <li>From the top menu, select <strong>Groups &gt; Your Groups</strong>. </li>
          <li>Find the group and select it. </li>
          <li>Under the group name, select <strong>Request Access</strong>. </li>
         </ol> 
         <p>As many as ten of the most-recently-active group owners receive an email with your request. Any group owner can approve or decline the request. </p>
         <p>If you change your mind before your request is approved, select <strong>Withdraw Access Request</strong>. </p>
         <h2 id="prevent-users-from-requesting-access-to-a-group">Prevent users from requesting access to a group<a href="#prevent-users-from-requesting-access-to-a-group" title="Permalink" class="anchor"></a> </h2> 
         <p>As a group owner, you can prevent non-members from requesting access to your group. </p>
         <ol> 
          <li>From the top menu, select <strong>Groups &gt; Your Groups</strong>. </li>
          <li>Find the group and select it. </li>
          <li>From the left menu, select <strong>Settings &gt; General</strong>. </li>
          <li>Expand the <strong>Permissions, LFS, 2FA</strong> section. </li>
          <li>Clear the <strong>Allow users to request access</strong> checkbox. </li>
          <li>Select <strong>Save changes</strong>. </li>
         </ol> 
         <h2 id="change-the-owner-of-a-group">Change the owner of a group<a href="#change-the-owner-of-a-group" title="Permalink" class="anchor"></a> </h2> 
         <p>You can change the owner of a group. Each group must always have at least one member with <a href="../permissions.html#group-members-permissions">Owner permission</a>. </p>
         <ul> 
          <li>As an administrator: 
           <ol> 
            <li>Go to the group and from the left menu, select <strong>Members</strong>. </li>
            <li>Give a different member <strong>Owner</strong> permissions. </li>
            <li>Refresh the page. You can now remove <strong>Owner</strong> permissions from the original owner. </li>
           </ol> </li>
          <li>As the current groups owner: 
           <ol> 
            <li>Go to the group and from the left menu, select <strong>Members</strong>. </li>
            <li>Give a different member <strong>Owner</strong> permissions. </li>
            <li>Have the new owner sign in and remove <strong>Owner</strong> permissions from you. </li>
           </ol> </li>
         </ul> 
         <h2 id="remove-a-member-from-the-group">Remove a member from the group<a href="#remove-a-member-from-the-group" title="Permalink" class="anchor"></a> </h2> 
         <p>Prerequisites: </p>
         <ul> 
          <li>You must have <a href="../permissions.html#group-members-permissions">Owner permissions</a>. </li>
          <li>The member must have direct membership in the group. If membership is inherited from a parent group, then the member can be removed from the parent group only. </li>
         </ul> 
         <p>To remove a member from a group: </p>
         <ol> 
          <li>Go to the group. </li>
          <li>From the left menu, select <strong>Members</strong>. </li>
          <li>Next to the member you want to remove, select <strong>Delete</strong>. </li>
          <li>Optional. On the <strong>Remove member</strong> confirmation box, select the <strong>Also unassign this user from related issues and merge requests</strong> checkbox. </li>
          <li>Select <strong>Remove member</strong>. </li>
         </ol> 
         <h2 id="filter-and-sort-members-in-a-group">Filter and sort members in a group<a href="#filter-and-sort-members-in-a-group" title="Permalink" class="anchor"></a> </h2> 
         <div class="introduced-in mb-3">
          Version history<button class="text-expander" data-toggle="collapse" href="#release_version_notes_1" role="button" aria-expanded="false"></button>
          <div class="introduced-in-content collapse" id="release_version_notes_1"> 
           <ul> 
            <li> <a href="https://gitlab.com/gitlab-org/gitlab/-/issues/21727">Introduced</a> in GitLab 12.6. </li>
            <li> <a href="https://gitlab.com/gitlab-org/gitlab/-/issues/228675">Improved</a> in GitLab 13.7. </li>
            <li> <a href="https://gitlab.com/gitlab-org/gitlab/-/issues/289911">Feature flag removed</a> in GitLab 13.8. </li>
           </ul> 
          </div> 
         </div> 
         <p>To find members in a group, you can sort, filter, or search. </p>
         <h3 id="filter-a-group">Filter a group<a href="#filter-a-group" title="Permalink" class="anchor"></a> </h3> 
         <p>Filter a group to find members. By default, all members in the group and subgroups are displayed. </p>
         <ol> 
          <li>Go to the group and select <strong>Members</strong>. </li>
          <li>Above the list of members, in the <strong>Filter members</strong> box, enter filter criteria. 
           <ul> 
            <li>To view members in the group only, select <strong>Membership = Direct</strong>. </li>
            <li>To view members of the group and its subgroups, select <strong>Membership = Inherited</strong>. </li>
            <li>To view members with two-factor authentication enabled or disabled, select <strong>2FA = Enabled</strong> or <strong>Disabled</strong>. </li>
           </ul> </li>
         </ol> 
         <h3 id="search-a-group">Search a group<a href="#search-a-group" title="Permalink" class="anchor"></a> </h3> 
         <p>You can search for members by name, username, or email. </p>
         <ol> 
          <li>Go to the group and select <strong>Members</strong>. </li>
          <li>Above the list of members, in the <strong>Filter members</strong> box, enter search criteria. </li>

         <h2 id="transfer-a-group">Transfer a group<a href="#transfer-a-group" title="Permalink" class="anchor"></a> </h2> 
         <p>You can transfer groups in the following ways: </p>
         <ul> 
          <li>Transfer a subgroup to a new parent group. </li>
          <li>Convert a top-level group into a subgroup by transferring it to the desired group. </li>
          <li>Convert a subgroup into a top-level group by transferring it out of its current group. </li>
         </ul> 
         <p>When transferring groups, note: </p>
         <ul> 
          <li>Changing a groups parent can have unintended side effects. See <a href="../project/repository/index.html#redirects-when-changing-repository-paths">Redirects when changing repository paths</a>. </li>
          <li>You can only transfer groups to groups you manage. </li>
          <li>You must update your local repositories to point to the new location. </li>
          <li>If the immediate parent groups visibility is lower than the groups current visibility, visibility levels for subgroups and projects change to match the new parent groups visibility. </li>
          <li>Only explicit group membership is transferred, not inherited membership. If the groups owners have only inherited membership, this leaves the group without an owner. In this case, the user transferring the group becomes the groups owner. </li>
          <li>Transfers fail if <a href="../packages/index.html">packages</a> exist in any of the projects in the group, or in any of its subgroups. </li>
         </ul>

         <h2 id="group-file-templates">Group file templates <span class="badge-trigger premium"></span><a href="#group-file-templates" title="Permalink" class="anchor"></a> </h2> 
         <p>Use group file templates to share a set of templates for common file types with every project in a group. It is analogous to the <a href="../admin_area/settings/instance_template_repository.html">instance template repository</a>. The selected project should follow the same naming conventions as are documented on that page. </p>

         <p>To learn how to create templates for issues and merge requests, see <a href="../project/description_templates.html">Description templates</a>. </p>

         <h2 id="prevent-project-forking-outside-group">Prevent project forking outside group <span class="badge-trigger premium"></span><a href="#prevent-project-forking-outside-group" title="Permalink" class="anchor"></a> </h2>

         <p>By default, projects in a group can be forked. Optionally, on <a href="https://about.gitlab.com/pricing/">Premium</a> or higher tiers, you can prevent the projects in a group from being forked outside of the current top-level group. </p>
         <p>Previously this setting was available only for groups enforcing group managed account. This setting will be removed from SAML setting page and migrated to group settings. In the interim period, both of these settings are taken into consideration.           <li>Go to the top-level groups <strong>Settings &gt; General</strong> page. </li>
          <li>Expand the <strong>Permissions, LFS, 2FA</strong> section. </li>
          <li>Check <strong>Prevent project forking outside current group</strong>. </li>
          <li>Select <strong>Save changes</strong>. </li>
