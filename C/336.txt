      <p>A <b>template</b> is a Wikipedia page created to be included in other pages. Templates usually contain repetitive material that might need to show up on any number of articles or pages. They are commonly used for <a href="/wiki/Wikipedia:Glossary#Boilerplate_text" title="Wikipedia:Glossary">boilerplate</a> messages, standardized warnings or notices, <a href="/wiki/Help:Infobox" title="Help:Infobox">infoboxes</a>, <a href="/wiki/Wikipedia:Navbox" class="mw-redirect" title="Wikipedia:Navbox">navigational boxes</a>, and similar purposes. </p>
      <p>The most common method of inclusion is called <a href="/wiki/Wikipedia:Transclusion" class="mw-redirect" title="Wikipedia:Transclusion">transclusion</a>, where the <a href="/wiki/Wikipedia:Wikitext" class="mw-redirect" title="Wikipedia:Wikitext">wikitext</a> of the target page contains a reference to the template, using the <code>{{<var style="padding-right: 1px;">Template&nbsp;name</var>}}</code> syntax. Another method is <a href="/wiki/Help:Substitution" title="Help:Substitution">substitution</a>, where the content of the template is copied into the wikitext of the target page, just once, when it is saved. </p>
      <p><a href="/wiki/Help:A_quick_guide_to_templates" title="Help:A quick guide to templates">Help:A quick guide to templates</a> gives a brief introduction to the subject. There is further help from MediaWiki and Wikimedia at <a href="https://www.mediawiki.org/wiki/Help:Templates" class="extiw" title="mw:Help:Templates">mw:Help:Templates</a>, <a href="https://meta.wikimedia.org/wiki/Help:Template" class="extiw" title="m:Help:Template">m:Help:Template</a>, and <a href="https://meta.wikimedia.org/wiki/Help:Advanced_templates" class="extiw" title="m:Help:Advanced templates">m:Help:Advanced templates</a>. </p>

      <h2><span class="mw-headline" id="Template_link_to_template">Template link to template</span></h2> 
      <p>To include the actual full name of the template as text (including the surrounding braces) in a Wikipedia article, apply the <a href="/wiki/Template:Template_link" title="Template:Template link"><b>template link</b> template</a>. The primary motivation to do this is in instruction and documentation. A short example is the code: </p> 
      <dl>
       <dd>
        <dl>
         <dd>
          <code>{{tl|foo}}</code>
         </dd>
        </dl>
       </dd> 
       <dd>
        generates

      <p>Most templates are pages in the <a href="/wiki/Wikipedia:Template_namespace" title="Wikipedia:Template namespace">Template namespace</a>, which means that they have titles in the form "Template:<var style="padding-right: 1px;">XXXX</var>". It is possible, however, to transclude and substitute from any namespace,<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[a]</a></sup> and so some template pages are placed in other namespaces, such as the <a href="/wiki/Wikipedia:User_namespace" class="mw-redirect" title="Wikipedia:User namespace">User namespace</a>. Template pages have associated <a href="/wiki/Wikipedia:Talk_page" class="mw-redirect" title="Wikipedia:Talk page">talk pages</a>. </p>
      <p>Templates can contain any desired <a href="/wiki/Wikipedia:Wikitext" class="mw-redirect" title="Wikipedia:Wikitext">wikitext</a>, including calls to other templates. They have some limited programming capacities: customizable values (called <a href="#Parameters">parameters</a>); calculation and branchings (using <a href="/wiki/Help:Parser_functions" class="mw-redirect" title="Help:Parser functions">parser functions</a>); and access to wiki-specific variables (<a href="/wiki/Help:Magic_words" title="Help:Magic words">magic words</a>), such as dates, times, and page names. They may also contain <a href="/wiki/HTML_element" title="HTML element">tags</a> which define which parts of the wikitext are to be included when the template is transcluded or substituted. This means that the appearance of the template page itself need not be the same as that of the transcluded content (for example, it can contain documentation, categories, etc. for the template). </p>
      <p><strong>How to do it:</strong> To transclude a template into an article or page, type <code>{{<var style="padding-right: 1px;">Template&nbsp;name</var>}}</code> in the wikitext at the place where the template is to appear. The first letter may be indifferently lower- or upper-case. </p>
      <p>The prefix <code>Template:</code> before the template name is the default one and need not be included. However, for templates stored in other namespaces, the prefix, such as <code>User:</code>, must be specified. To transclude a page in <a href="/wiki/Wikipedia:Mainspace#Namespace" class="mw-redirect" title="Wikipedia:Mainspace">mainspace</a>, precede its title with a colon, as <code>{{:<var style="padding-right: 1px;">Page name</var>}}</code>. </p>
      <p><img alt="" src="//upload.wikimedia.org/wikipedia/commons/thumb/4/4a/Pictogram_voting_info.svg/16px-Pictogram_voting_info.svg.png" decoding="async" width="16" height="16" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/4/4a/Pictogram_voting_info.svg/24px-Pictogram_voting_info.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/4/4a/Pictogram_voting_info.svg/32px-Pictogram_voting_info.svg.png 2x" data-file-width="250" data-file-height="250"> <b>Note:</b> Attempting to transclude a template that does not exist produces a <a href="/wiki/Wikipedia:Redlink" class="mw-redirect" title="Wikipedia:Redlink">red link</a>, just like linking to any other nonexistent page. Following the link allows one to create that particular template. It is not possible to transclude pages between projects (such as different-language Wikipedias or MediaWiki)—to use a template on another language project, a copy of the template must be created in that project. </p> 
      <h2><span class="mw-headline" id="Usage_syntax">Usage syntax</span></h2> 
      <h3><span class="mw-headline" id="Parameters">Parameters</span></h3> 
      <div role="note" class="hatnote navigation-not-searchable">
       This section is about template parameters. For search parameters, see <a href="/wiki/Help:Searching#Parameters" title="Help:Searching">Help:Searching §&nbsp;Parameters</a>.
      </div> 
      <div role="note" class="hatnote navigation-not-searchable">
       Further information: <a href="https://meta.wikimedia.org/wiki/Help:Templates#Parameters" class="extiw" title="m:Help:Templates">m:Help:Templates §&nbsp;Parameters</a>
      </div>

        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=H:PARAMETER&amp;redirect=no">H:PARAMETER</a></span></li>

      <p>The basic transclusion syntax given above can be extended by the addition of <a href="/wiki/Parameter" title="Parameter">parameters</a>, which are used to control the template's output. The syntax for this is <code>{{<var style="padding-right: 1px;">Template name</var>|<var style="padding-right: 1px;">parameter</var>|<var style="padding-right: 1px;">parameter</var>|<var style="padding-right: 1px;">...</var>}}</code> where <code><var style="padding-right: 1px;">Template name</var></code> is the name of the template, and each <code><var style="padding-right: 1px;">parameter</var></code> may either contain just a value (these are called <i> <dfn>unnamed parameters</dfn></i>) or be of the form <code><var style="padding-right: 1px;">name</var>=<var style="padding-right: 1px;">value</var></code> (<i> <dfn>named parameters</dfn></i>). The first, second, third, etc. unnamed parameters will be given the names <code>1</code>, <code>2</code>, <code>3</code>, etc. </p>
      <p>Whitespace characters (spaces, tabs, returns) are stripped from the beginnings and ends of <em>named</em> parameter names and values, but not from the middle: thus <code>{{ ... | myparam = this is a test }}</code> has the same effect as <code>{{ ... |myparam=this is a test}}</code>. This does not apply to <em>unnamed</em> parameters, where all whitespace characters are preserved. </p>
      <p>Which parameters (if any) can or should be passed to a template and how they are to be named depends on the coding of that template. Named parameters can be defined in any order. Superfluous or misnamed parameters will be ignored; undefined parameters will be assigned default values. If a parameter is defined more than once, the last value takes effect. </p>
      <p>The value of a parameter can be the empty string, such as when the pipe or equals sign is followed immediately by the next pipe or the closing braces. This is different from omitting the parameter altogether, which leaves it undefined, although templates are often coded so as to behave the same in both cases. </p>
      <p>Parameters can be specified (and will do nothing) even if not represented in the template's code. For example, <code class="nowrap" style="">|reason=</code> is frequently used as a pseudo-parameter to explain briefly in the wikisource why the template was placed.<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">[b]</a></sup> Some templates call <a href="/wiki/Module:Check_for_unknown_parameters" title="Module:Check for unknown parameters">Module:Check for unknown parameters</a> to warn the editor if a parameter is being used that is not accounted for in the template's code; this is mostly used for <a href="/wiki/Wikipedia:Infobox" class="mw-redirect" title="Wikipedia:Infobox">infoboxes</a> and other templates with a large number of complicated parameters, where the presence of an unknown one is usually an unintentional error. If you update such a template to include a new parameter, its call to the module must also be updated to include the new parameter. </p> 
      <h3><span class="mw-headline" id="Calling">Calling</span></h3> 
      <div role="note" class="hatnote navigation-not-searchable">
       Further information: <a href="https://www.mediawiki.org/wiki/Help:Magic_words" class="extiw" title="mw:Help:Magic words">mw:Help:Magic words</a>
      </div> 
      <p>Using a template is much like calling a <a href="/wiki/Function_(computer_science)" class="mw-redirect" title="Function (computer science)">function</a> in a programming language – call it and it returns a value (the <a href="/wiki/Input/output" title="Input/output">output</a>). Like functions, some templates accept parameters that change the output. </p>
      <p>In <a href="/wiki/MediaWiki" title="MediaWiki">MediaWiki</a>, the <a href="/wiki/Wiki_software" title="Wiki software">wiki software</a> that Wikipedia uses, <a href="/wiki/Help:Magic_words#Variables" title="Help:Magic words">variables</a> have a more specific meaning that distinguishes them from templates, but they are both identified by double <a href="/wiki/Curly_brackets" class="mw-redirect" title="Curly brackets">braces</a> <code>{{ }}</code> and they both return a value. </p>
      <p>Whereas MediaWiki variable names are all uppercase, template names have the same basic features and <a href="/wiki/Wikipedia:NCHASHTAG" class="mw-redirect" title="Wikipedia:NCHASHTAG">limitations</a> as all <a href="/wiki/Help:Page_name" title="Help:Page name">page names</a>: they are case-sensitive (except for the first character); underscores are <a href="/wiki/Parsed" class="mw-redirect" title="Parsed">parsed</a> as spaces; and they cannot contain any of these characters: <code># &lt; &gt; [ ] | { }</code>. This is because those are reserved for <a href="/wiki/Wiki_markup" class="mw-redirect" title="Wiki markup">wiki markup</a> and <a href="/wiki/HTML" title="HTML">HTML</a>. </p>
      <p>The <a href="/wiki/Number_sign" title="Number sign">number sign</a> <code>#</code> is called a <a href="/wiki/Fragment_identifier" class="mw-redirect" title="Fragment identifier">fragment identifier</a> because it denotes a <em>fragment</em> or section of a document (such as a <a href="/wiki/Help:Section" title="Help:Section">section</a> in a Wikipedia article). Although it <a href="/wiki/Wikipedia:ANCHOR" class="mw-redirect" title="Wikipedia:ANCHOR">can be used</a> to link to a section of a template page (like <a href="/wiki/Template:Portal#Example" title="Template:Portal">Template:Portal#Example</a>), there is no reason to put a fragment identifier or fragment name in a template reference. In <code>{{Portal<var style="padding-right: 1px;">#Location</var>|Books}}</code>, for example, the string <code><var style="padding-right: 1px;">#Location</var></code> has no purpose and is thus ignored. </p> 
      <h3><span class="mw-headline" id="Substitution">Substitution</span></h3> 
      <div role="note" class="hatnote navigation-not-searchable">
       Further information: <a href="/wiki/Help:Substitution" title="Help:Substitution">Help:Substitution</a>
      </div> 
      <p>When a template is substituted, its content is <a href="/wiki/Hard-coded" class="mw-redirect" title="Hard-coded">hard-coded</a> in the page rather than <a href="/wiki/Wikipedia:Transclusion" class="mw-redirect" title="Wikipedia:Transclusion">transcluded</a>. To learn how and <a href="/wiki/Help:Substitution#When_to_use_substitution" title="Help:Substitution">when</a> to substitute a template, see <a href="/wiki/Help:Substitution" title="Help:Substitution">Help:Substitution</a>. </p> 
      <h2><span class="mw-headline" id="Basic_template_usage_examples">Basic template usage examples</span></h2> 
      <p><img alt="" src="//upload.wikimedia.org/wikipedia/commons/thumb/4/4a/Pictogram_voting_info.svg/16px-Pictogram_voting_info.svg.png" decoding="async" width="16" height="16" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/4/4a/Pictogram_voting_info.svg/24px-Pictogram_voting_info.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/4/4a/Pictogram_voting_info.svg/32px-Pictogram_voting_info.svg.png 2x" data-file-width="250" data-file-height="250"> <b>Note:</b> If you wish to experiment with any of these, please use either the <a href="/wiki/Template:Template_sandbox" title="Template:Template sandbox">template sandbox</a>, <a href="/wiki/Special:ExpandTemplates" title="Special:ExpandTemplates">Special:ExpandTemplates</a> or your user page or sandbox. </p>
      <p>An example of a very simple template can be found at <a href="/wiki/Template:Lambda" title="Template:Lambda">Template:Lambda</a>, which expands out to place the lambda symbol (?) at that point in the text. A programmer would say that the template <em>returns</em> the lambda symbol (?). </p>
      <p>Click on <a href="/wiki/Template:Lambda" title="Template:Lambda">Template:Lambda</a>, then click on the "Edit source" tab to see the template code (its wikitext). The "active" part of that code, called the <dfn>expansion</dfn> of the template, is the single word <code>&amp;lambda;</code>.<sup id="cite_ref-3" class="reference"><a href="#cite_note-3">[c]</a></sup> The remainder of the wikitext is enclosed between <code class="nowrap">&lt;<a href="/wiki/Help:Template#Noinclude,_includeonly,_and_onlyinclude" title="Help:Template">noinclude</a>&gt;</code> tags so it is displayed on the template page itself but will not be displayed when the template is used (or called) on another page. </p>
      <p>To transclude Template:Lambda onto another page (i.e., to use it on another page), type <code>{{<a href="/wiki/Template:Lambda" title="Template:Lambda">lambda</a>}}</code> (or <code>{{<a href="/wiki/Template:Lambda" title="Template:Lambda">Lambda</a>}}</code> – the case of the first letter is not significant) into the wikitext of the target page and press <kbd class="nowrap button" title="This is not a clickable button; it illustrates the button one should find." style="padding:.2em .6em; border:1px solid; border-color:#AAA #555 #555 #AAA; border-radius:3px; background-color: #F2F2F2; color: #000; background-image: linear-gradient(to bottom, #FCFCFC, #E0E0E0); font-family:sans-serif;">Show preview</kbd>. The page will be displayed with the <dfn>template call</dfn> replaced by the expansion of the template, as if the wikitext actually contained <code>&amp;lambda;</code> at that point. The displayed page will therefore contain the text "?". </p>
      <p>For example, type <code>The 11th letter of the Greek alphabet is the lambda ({{<a href="/wiki/Template:Lambda" title="Template:Lambda">lambda</a>}})</code> and you will see "The 11th letter of the Greek alphabet is the lambda (?)" when previewing the page or after saving the change. You can use templates without knowing the details of their code; you only need to remember what result they produce, which is usually described on the template page. </p>
      <p>Another way to use this template is to <a href="/wiki/Help:Substitution" title="Help:Substitution">substitute</a> it. If you type <code>The 11th letter of the Greek alphabet is the lambda ({{<a href="/wiki/Template:Lambda" title="Template:Lambda">subst:lambda</a>}})</code> and preview or save the page, you will again see "The 11th letter of the Greek alphabet is the lambda (?)". If you look again at the saved wikitext,<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">[d]</a></sup> however, you will see that the template calls really were replaced by the expansion of the template when you saved the page. The link between the output text and the template is now broken, and the output will not be affected by future changes to the template (as it would be in the case of transclusion). </p> 
      <h3><span class="mw-headline" id="Examples_with_parameters"><span class="anchor" id="Parameters"></span> Examples with parameters</span></h3> 
      <p>An example of a template that takes parameters is the template <code>{{<a href="/wiki/Template:About" title="Template:About">about</a>}}</code>. Try typing <code>{{about|how to use templates|how to use modules|Help:Module}}</code> in the sandbox—it will produce the following text: </p> 
      <div role="note" class="hatnote navigation-not-searchable">
       This page is about how to use templates. For how to use modules, see <a href="/wiki/Help:Module" class="mw-redirect" title="Help:Module">Help:Module</a>.
      </div>

For example, <code>{{about||how to use modules|Help:Module}}</code>. Note the usage of an empty parameter—in this instance, the consecutive pipes mean that the first parameter that was "passed" to the template is an empty string, which in this template will cause it to omit the initial "about" sentence. This produces: </p>

      <h2><span class="mw-headline" id="Usage_hints_and_workarounds">Usage hints and workarounds</span></h2> 
      <p>The following points may be worth noting when using templates: </p> 
      <ul>
       <li><span class="anchor" id="Equals"></span>An unnamed parameter cannot contain an ordinary equals sign, as this would be interpreted as setting off a named parameter.<sup id="cite_ref-5" class="reference"><a href="#cite_note-5">[e]</a></sup> To pass an equals sign in an unnamed parameter (for example in a <a href="/wiki/URL" title="URL">URL</a> with <a href="/wiki/Attribute%E2%80%93value_pair" class="mw-redirect" title="Attribute–value pair">attribute–value pairs</a>), replace the equals sign with the special template <code>{{<a href="/wiki/Template:%3D" title="Template:=">=</a>}}</code>, which returns an equals sign that will not be interpreted. Another method is to replace the unnamed parameter (and any subsequent unnamed parameters) with named parameters—the first unnamed parameter is equivalent to a named parameter <code class="nowrap" style="">|1=</code> and so on. To call template <code>{{<a href="/wiki/Template:Done" title="Template:Done">done</a>}}</code> with <code>a=b</code> as the literal value for the first parameter, type either <code>{{done|a{{=}}b}}</code> or <code>{{done|1=a=b}}</code>.</li> 
       <li>Similarly, it is not possible to use an ordinary pipe character <code>|</code> in template parameters, as it will be interpreted as a separator.<sup id="cite_ref-6" class="reference"><a href="#cite_note-6">[f]</a></sup> This time, the problem can be solved by using the <a href="https://www.mediawiki.org/wiki/Help:Magic_words#Other" class="extiw" title="mw:Help:Magic words">magic word</a> <code>{{<a href="/wiki/Template:!" title="Template:!">!</a>}}</code> in place of the pipe, or—if the pipe is not intended to be parsed at a higher level—using the <a href="/wiki/List_of_XML_and_HTML_character_entity_references" title="List of XML and HTML character entity references">HTML entity</a> <code>&amp;#124;</code>. Alternatively, for embedding <a href="/wiki/Help:Tables" class="mw-redirect" title="Help:Tables">wikitables</a> in templates, you may use <code>{{<a href="/wiki/Template:Wikitable" title="Template:Wikitable">Wikitable</a>}}</code> to avoid excessive <code>{{<a href="/wiki/Template:!" title="Template:!">!</a>}}</code>.</li> 
       <li>Remember that whitespace characters (spaces, tabs, carriage returns, and line feeds) are not automatically stripped from the start and end of unnamed parameters, unlike with named parameters. Including such characters (or any other non-visible characters in any parameters) may in some cases affect the template's behaviour in unexpected ways. (Template designers can use <code>{{<a href="/wiki/Template:Strip_whitespace" class="mw-redirect" title="Template:Strip whitespace">Strip whitespace</a>}}</code> to remove unwanted whitespace in unnamed parameters.)</li> 
       <li>In documentation and discussions, it is often convenient to be able to produce the template call syntax with a link to the template in question, but without actually calling the template. This can be done easily using the <code>{{<a href="/wiki/Template:Tl" class="mw-redirect" title="Template:Tl">tl</a>}}</code> template (the "<b>t</b>emplate <b>l</b>ink" template). For example, <code>{{tl|Example}}</code> produces {{<a href="/wiki/Template:Example" title="Template:Example">Example</a>}}. There are multiple other <a href="/wiki/Template:Template-linking_templates" title="Template:Template-linking templates">template-linking templates</a> available with different functionalities.</li> 
       <li>When a template is changed (when the template or one of its subtemplates is edited), the change will be reflected on all pages on which the template is transcluded. However, the change may not become visible on all pages immediately; a previously cached version of a page, based on the previous version of the template, may continue to be displayed for some time. Use the <a href="/wiki/Help:Purge" class="mw-redirect" title="Help:Purge">purge</a> function to force a page to be displayed using the latest versions of templates—including on the template page itself, if it contains usage examples.</li> 
       <li>When viewing old versions of pages, remember that templates will be transcluded as they are now, not necessarily as they were when the old page version was active.</li> 
       <li>To list all pages onto which a template is transcluded, use <a href="/wiki/Help:What_links_here" title="Help:What links here">What links here</a> on the template page. This will <em>not</em> include pages where the template has been substituted.</li> 
       <li>To get a list of templates transcluded on a page, click "Edit" and find the list below the edit window. This list also includes the subtemplates used by the templates that are directly transcluded. To get such a list for a page section, an old version of the page,<sup id="cite_ref-7" class="reference"><a href="#cite_note-7">[g]</a></sup> or your newly edited version prior to saving, click <kbd class="nowrap button" title="This is not a clickable button; it illustrates the button one should find." style="padding:.2em .6em; border:1px solid; border-color:#AAA #555 #555 #AAA; border-radius:3px; background-color: #F2F2F2; color: #000; background-image: linear-gradient(to bottom, #FCFCFC, #E0E0E0); font-family:sans-serif;">Show preview</kbd> on the appropriate edit page.</li> 
       <li>There are limits to the number and complexity of the templates that an article may have. See the "<a href="#Expand_limits">§ Expand limits</a>" section for help in resolving this.</li> 
       <li>If you want the template to leave a <a href="/wiki/Wikipedia:~" class="mw-redirect" title="Wikipedia:~">time stamp or signature</a>, you can write <code>&lt;noinclude&gt;&lt;nowiki&gt;&lt;/noinclude&gt;~~~~~&lt;noinclude&gt;&lt;/nowiki&gt;&lt;/noinclude&gt;</code>, but this will only work if you substitute the template. If you transclude it, you'll just get ~~~~~.</li> 
       <li>To improve readability, usually programmers like to split the code with newlines and indent it. Unfortunately, <a href="/wiki/MediaWiki" title="MediaWiki">MediaWiki</a> software does not allow this functionality; in many instances, these purpose-built newlines are treated by the software as content. One possible workaround is to add <code>&lt;!--</code> before each newline character and <code>--&gt;</code> after it, which produces an <a href="/wiki/HTML_comment" class="mw-redirect" title="HTML comment">HTML comment</a>.</li>
      </ul> 
      <h2><span class="mw-headline" id="Creating_and_editing_templates">Creating and editing templates</span></h2> 
      <p>Templates are created and edited in much the same way as any other page: choose an appropriate name, navigate to that page, then click the "Edit" tab or create a new page as needed. As mentioned above, templates are normally placed in the <a href="/wiki/Wikipedia:Template_namespace" title="Wikipedia:Template namespace">Template namespace</a>, though templates intended for your own personal use or for experimentation can be created in your own <a href="/wiki/Wikipedia:User_space" class="mw-redirect" title="Wikipedia:User space">user space</a>. Anything that can be included on a normal page or article can be included on a template, including other templates (called <i> <dfn>subtemplates</dfn></i>). Templates often make use of programming features—parameters, parser functions, and other <a href="/wiki/Help:Magic_words" title="Help:Magic words">magic words</a>—which allow the transcluded content to vary depending on context. There are also special tags to control which information is transcluded and which is not. </p>
      <p>Before creating a template, do a quick search for existing templates (such as by exploring <a href="/wiki/Category:Wikipedia_templates" title="Category:Wikipedia templates">Category:Wikipedia templates</a>) to see if there is already a template that does what you want or a similar template whose code can be copied and modified (or left in place and expanded). Look for generic templates on which the new template can be based; for example, <a href="/wiki/Wikipedia:Navbox" class="mw-redirect" title="Wikipedia:Navbox">navbox</a> templates can be easily created by calling the generic <a href="/wiki/Template:Navbox" title="Template:Navbox">Template:Navbox</a>. </p>
      <p>There is no hard rule about what name to choose for a template—make it short but reasonably descriptive. If similar templates exist, try to follow a consistent naming pattern. Templates can be renamed without breaking existing <a href="/wiki/Wikipedia:Transclusion" class="mw-redirect" title="Wikipedia:Transclusion">transclusions</a> (what is called <b>breakage</b>), provided a <a href="/wiki/Wikipedia:Redirect" title="Wikipedia:Redirect">redirect</a> to the new template name is left behind. </p>
      <p>Be extremely careful when editing existing templates—changes made can affect a large number of pages, often in ways you might not expect. For this reason many high-use templates are <a href="/wiki/Wikipedia:Protection" class="mw-redirect" title="Wikipedia:Protection">protected</a> against editing except by <a href="/wiki/Wikipedia:Administrator" class="mw-redirect" title="Wikipedia:Administrator">administrators</a> and <a href="/wiki/Wikipedia:Template_editor" title="Wikipedia:Template editor">template editors</a>; other editors can propose changes on the talk page. Some templates offer a sandbox and <a href="/wiki/Wikipedia:Template_test_cases" class="mw-redirect" title="Wikipedia:Template test cases">test cases</a> for experimentation. </p>
      <p>To propose the deletion or merger of unused or inappropriate templates or templates that can easily be merged, go to <a href="/wiki/Wikipedia:Templates_for_discussion" title="Wikipedia:Templates for discussion">Templates for discussion</a> (TfD). </p> 
      <h3><span class="mw-headline" id="Handling_parameters">Handling parameters</span></h3> 
      <p>The values of the parameters which can be fed to a template are represented in the template code by items enclosed between <em>triple</em> braces: </p> 
      <ul>
       <li>The code <code>{{{xxx}}}</code> will be replaced by the value of the parameter named <code>xxx</code>. these are known as <a href="/wiki/Named_parameter" title="Named parameter"><i> <dfn>named parameter</dfn>s</i></a>.</li> 
       <li>The codes <code>{{{1}}}</code>, <code>{{{2}}}</code>, and so on will be replaced by the first, second, and so on <i>unnamed parameter</i> (or the value of a parameter named <code>1</code>, <code>2</code>, etc.); these are known as <a href="/wiki/Positional_parameter" class="mw-redirect" title="Positional parameter"><i> <dfn>positional parameter</dfn>s</i></a>.</li>
      </ul> 
      <p>If a parameter is not assigned a value, then no replacement will take place; this means that if no value is passed for parameter <code class="nowrap" style="">|xxx=</code>, the value of the expression <code>{{{xxx}}}</code> inside the template will literally be <i>{{{xxx}}}</i>, not the 'blank' you may have expected. A more intuitive behaviour can be achieved by specifying default parameter values. This is done with the <i>pipe syntax</i>: <code>{{{xxx|dflt}}}</code> specifies the default value <code>dflt</code> for the named parameter <code class="nowrap" style="">|xxx=</code>, and <code>{{{1|dflt}}}</code> specifies the default value <code>dflt</code> for the first unnamed parameter. Most often, this is used to specify null default values, such as <code>{{{1|}}}</code> or <code>{{{xxx|}}}</code>. </p>
      <p>Parameter aliases are a special case of default values. For example, if parameters <code class="nowrap" style="">|1=</code>, <code class="nowrap" style="">|text=</code>, and <code class="nowrap" style="">|message=</code> are names for the same parameter, then wikitext <code>{{{message|{{{text|{{{1|}}}}}}}}}</code> could be used. If more than one of those parameters are given, then <code>message</code> will have priority, followed by <code>text</code>, and finally by the first unnamed parameter. That is, if a template containing that wikitext is passed the parameters <code>|message=A|text=B</code>, the wikitext will expand to <code>A</code>. </p>
      <p>Because of the multiple uses of double-brace and triple-brace syntax, expressions can sometimes be ambiguous. It may be helpful or necessary to include spaces to resolve such ambiguity. For example, <code>{{ {{{xxx}}} }}</code> or <code>{{{ {{xxx}} }}}</code>, rather than typing five consecutive braces, may be more human-readable. However, watch out for unwanted whitespace appearing in template expansions. </p> 
      <h4><span class="mw-headline" id="Special_case:_parameters_within_an_XML-style_opening_tag">Special case: parameters within an XML-style opening tag</span></h4> 
      <p>Parameters do not get expanded when they are wrapped in <code class="nowrap">&lt;<a href="/wiki/Help:Nowiki" class="mw-redirect" title="Help:Nowiki">nowiki</a>&gt;</code> tags. They aren't expanded either if placed within the actual <a href="/wiki/XML" title="XML">XML</a>-style opening tag. Thus, the following will not work within a template: </p> 
      <ul>
       <li><span class="example deprecated-content example-bad monospaced example-mono" style="font-family: monospace, monospace; color: #8B0000;">&lt;ref name={{{param}}}&gt; Smith, Adam (1776)...&lt;/ref&gt;</span></li>
      </ul> 
      <p>because the parameter is not expanded. Instead, you can use the <code>{{#tag:}}</code> <a href="/wiki/Wikipedia:PF" class="mw-redirect" title="Wikipedia:PF">parser function</a>, which is—for example—used in <code>{{<a href="/wiki/Template:Sfn" title="Template:Sfn">sfn</a>}}</code> to generate the <code class="nowrap" style="">&lt;ref&gt;...&lt;/ref&gt;</code> element; see also <a href="/wiki/Help:Magic_words#Formatting" title="Help:Magic words">Help:Magic words § Formatting</a>. Therefore, the following example will work: </p> 
      <ul>
       <li><span class="example monospaced example-mono" style="font-family: monospace, monospace; color: #006400;">{{#tag:ref | Smith, Adam (1776)... | name={{{param}}} }}</span></li>
      </ul> 
      <h4><span class="mw-headline" id="Caution:_overextending_URLs">Caution: overextending URLs</span></h4> 
      <p>If a parameter's value is (or ends with) a <a href="/wiki/URL" title="URL">URL</a>, check whether it is displayed in Wikipedia with the link overextending by one or more characters after the URL so that clicking the link causes an error or failure. Ensure that, after processing by the software, a soft space (<em>not</em> a <a href="/wiki/Non-breaking_space" title="Non-breaking space">hard or non-breaking space</a>) follows the URL, regardless of whether you or a user supplied the URL or whether it was generated by automated processing. Possibly, the source code could contain or generate a space that is discarded in the processing or there might not be any space there. Correct the source code, perhaps by forcing a soft space to appear after the URL. The <code>{{<a href="/wiki/Template:Spaces" title="Template:Spaces">spaces</a>}}</code> template may be useful. </p> 
      <h3><span class="mw-headline" id="System_variables_and_conditional_logic">System variables and conditional logic</span></h3>

They can also be used for arithmetical calculations, but certain standard programming features such as loops and variable assignment are not available. Full string manipulation is also not available; some templates providing such functionality have been created, but they are inefficient and imperfect. </p>

      <h3><span class="mw-headline" id="Nesting_templates">Nesting templates</span></h3> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:NEST"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:NEST&amp;redirect=no">WP:NEST</a></span></li>
       </ul>
      </div> 
      <p>Templates may contain other templates—this is usually called <i> <dfn>nesting</dfn></i>. As the template is processed, the wikitext produced by any nested templates is transcluded into the nesting template, so that the final product is essentially processed from the most deeply nested template out. While fairly straightforward in application, it involves some noteworthy quirks and tricks. </p>
      <p>To pass a parameter value to a nested template, place a parameter tag as the value of one of the nested template's parameters. </p> 
      <dl>
       <dt>
        Examples:
       </dt> 
       <dd>
        Template:A contains <code>"the quick brown {{B|{{{3}}} }} jumps over..."</code> This takes the value passed to the third positional parameter of Template:A and passes it as the first positional parameter of Template:B, then returns the wikitext produced by B as part of the phrase.
       </dd> 
       <dd>
        Template:A contains <code>"the quick brown {{B|waldo={{{3}}} }} jumps over..."</code> As previously, except the third positional parameter of Template:A is passed to the named parameter "waldo" of Template:B.
       </dd>
      </dl> 
      <p>Template parameters themselves can be chosen conditionally. </p> 
      <dl>
       <dt>
        Examples:
       </dt> 
       <dd>
        Template:A contains <code>the quick brown {{B|{{{3}}}=fox}} jumps over...</code> Template:A passes the word "fox" as a named parameter of Template:B whose name is A's third positional parameter.
       </dd> 
       <dd>
        <code>{{#if: <var style="padding-right: 1px;">test string</var> | <var style="padding-right: 1px;">value if test string is not empty</var> | {{#if: <var style="padding-right: 1px;">test string</var> | <var style="padding-right: 1px;">value if test string is not empty</var> | <var style="padding-right: 1px;">value if test string is empty (or only white space)</var> }} }}</code>
       </dd>
      </dl> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:TEMPLATE_LOOP"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcut</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:TEMPLATE_LOOP&amp;redirect=no">WP:TEMPLATE LOOP</a></span></li>
       </ul>
      </div> 
      <p>A template can call itself but will stop after one iteration to prevent an infinite loop. </p>
      <p>When a nested template contains unmatched braces—as in <code>{{<a href="/wiki/Template:Lb" title="Template:Lb">lb</a>}}}</code>—the unmatched braces are treated as text during processing, and do not affect the parsing of braces in the nesting template. If the nested template is substituted, however, the substitution is processed first, and this <em>will</em> change how braces are parsed in the nesting template. This has little practical use, but can occasionally introduce unexpected errors. </p>
      <p>See <a href="https://meta.wikimedia.org/wiki/Help:Advanced_templates" class="extiw" title="meta:Help:Advanced templates">meta:Help:Advanced templates</a> and <a href="https://meta.wikimedia.org/wiki/Help:Recursive_conversion_of_wikitext" class="extiw" title="meta:Help:Recursive conversion of wikitext">meta:Help:Recursive conversion of wikitext</a> for more information. These pages also contain information on unusual calls such as <code>{{template {{{parameter|}}} }}</code>. </p> 
      <h3><span id="Inclusion_control:_noinclude.2C_includeonly.2C_and_onlyinclude"></span><span class="mw-headline" id="Inclusion_control:_noinclude,_includeonly,_and_onlyinclude"><span id="Noinclude,_includeonly,_and_onlyinclude"></span>Inclusion control: noinclude, includeonly, and onlyinclude</span></h3> 
      <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r1007196855">
      <div class="module-shortcutanchordiv">
       <span id="WP:NOINCLUDE"></span><span id="WP:INCLUDEONLY"></span><span id="WP:ONLYINCLUDE"></span>
      </div>
      <div class="module-shortcutboxplain plainlist noprint" role="note">
       <div class="module-shortcutlist">
        <a href="/wiki/Wikipedia:Shortcut" title="Wikipedia:Shortcut">Shortcuts</a>
       </div>
       <ul>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:NOINCLUDE&amp;redirect=no">WP:NOINCLUDE</a></span></li>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:INCLUDEONLY&amp;redirect=no">WP:INCLUDEONLY</a></span></li>
        <li><span class="plainlinks"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Wikipedia:ONLYINCLUDE&amp;redirect=no">WP:ONLYINCLUDE</a></span></li>
       </ul>
      </div> 
      <p>By default, when a template is <a href="/wiki/Wikipedia:Transclusion" class="mw-redirect" title="Wikipedia:Transclusion">transcluded</a> (or <a href="/wiki/Wikipedia:Substitution" title="Wikipedia:Substitution">substituted</a>), the entire wikitext (code) of the template page gets included in that of the target page. However, it is possible to modify that behaviour, using tags that specify which parts of the template code are to be included. This makes it possible to avoid transcluding information intended for display only on the template page itself, such as the template's {{<a href="/wiki/Template:Documentation" title="Template:Documentation">documentation</a>}}, <a href="/wiki/Wikipedia:Category" class="mw-redirect" title="Wikipedia:Category">categories</a>, and <a href="/wiki/Wikipedia:Interwiki_links" class="mw-redirect" title="Wikipedia:Interwiki links">interwiki links</a>. It is also possible to have parts of the code be transcluded, but <em>not</em> be processed on the template page itself (e.g., categories to be applied to the target pages which do not apply to the template). The tags are as follows: </p> 
      <ul>
       <li><code class="nowrap">&lt;<a href="/wiki/Help:Template#Noinclude,_includeonly,_and_onlyinclude" title="Help:Template">noinclude</a>&gt;...&lt;/noinclude&gt;</code>&nbsp;– The text between the tags will <em>not</em> be included when the template is transcluded (substituted), but <em>will</em> be processed on the template's page; a common use is for <a href="/wiki/Wikipedia:Template_documentation" title="Wikipedia:Template documentation">documentation in templates</a>.</li> 
       <li><code class="nowrap">&lt;<a href="/wiki/Help:Template#Noinclude,_includeonly,_and_onlyinclude" title="Help:Template">onlyinclude</a>&gt;...&lt;/onlyinclude&gt;</code>&nbsp;– This specifies that nothing on the page <em>except</em> what appears between the tags will be transcluded (substituted).</li> 
       <li><code class="nowrap">&lt;<a href="/wiki/Help:Template#Noinclude,_includeonly,_and_onlyinclude" title="Help:Template">includeonly</a>&gt;...&lt;/includeonly&gt;</code>&nbsp;– The text between the tags <em>will</em> be transcluded (substituted), but will <em>not</em> be processed on the template's own page.</li>
      </ul> 
      <table class="wikitable" style="text-align: center;"> 
       <tbody>
        <tr> 
         <th>Wikitext </th> 
         <th>What is rendered <em>here</em> (source page) </th> 
         <th>What is transcluded <em>there</em> (destination page) </th>
        </tr> 
        <tr> 
         <td><code>&lt;noinclude&gt; text1 &lt;/noinclude&gt; text2</code> </td> 
         <td><code>text1 text2</code> </td> 
         <td><code>text2</code> </td>
        </tr> 
        <tr> 
         <td><code>&lt;onlyinclude&gt; text1 &lt;/onlyinclude&gt; text2</code> </td> 
         <td><code>text1 text2</code> </td> 
         <td><code>text1</code> </td>
        </tr> 
        <tr> 
         <td><code>&lt;includeonly&gt; text1 &lt;/includeonly&gt; text2</code> </td> 
         <td><code>text2</code> </td> 
         <td><code>text1 text2</code> </td>
        </tr> 
        <tr> 
         <td><code>&lt;onlyinclude&gt;&lt;includeonly&gt; text1 &lt;/includeonly&gt;&lt;/onlyinclude&gt; text2</code> </td> 
         <td><code>text2</code> </td> 
         <td><code>text1</code> </td>
        </tr>
       </tbody>
      </table> 
      <p>Perhaps the most common issue with the use of these blocks is unwanted spaces or lines. It is important to remember that the effect of these tags begins immediately before the first angle bracket, not on the previous line or at the previous visible character; similarly the effect ends immediately after the last angle bracket, not on the next line or with the next visible character. For example: </p> 
      <dl>
       <dt>
        <code class="nowrap">&lt;<a href="/wiki/Help:Template#Noinclude,_includeonly,_and_onlyinclude" title="Help:Template">includeonly</a>&gt;</code>
       </dt> 
       <dd>
        <span class="example monospaced example-mono" style="font-family: monospace, monospace; color: #006400;">}}&lt;includeonly&gt;</span>
       </dd> 
       <dd>
        <span class="example deprecated-content example-bad monospaced example-mono" style="font-family: monospace, monospace; color: #8B0000;">}}<br>&lt;includeonly&gt;</span>
       </dd> 
       <dt>
        <code class="nowrap">&lt;<a href="/wiki/Help:Template#Noinclude,_includeonly,_and_onlyinclude" title="Help:Template">noinclude</a>&gt;</code>
       </dt> 
       <dd>
        <span class="example monospaced example-mono" style="font-family: monospace, monospace; color: #006400;">}}&lt;noinclude&gt;</span>
       </dd> 
       <dd>
        <span class="example deprecated-content example-bad monospaced example-mono" style="font-family: monospace, monospace; color: #8B0000;">}}<br>&lt;noinclude&gt;</span>
       </dd> 
       <dt>
        <code class="nowrap">&lt;/includeonly&gt;</code><code class="nowrap">&lt;<a href="/wiki/Help:Template#Noinclude,_includeonly,_and_onlyinclude" title="Help:Template">noinclude</a>&gt;</code>
       </dt> 
       <dd>
        <span class="example monospaced example-mono" style="font-family: monospace, monospace; color: #006400;">{{template}}&lt;/includeonly&gt;&lt;noinclude&gt;</span>
       </dd> 
       <dd>
        <span class="example deprecated-content example-bad monospaced example-mono" style="font-family: monospace, monospace; color: #8B0000;">{{template}}&lt;/includeonly&gt;<br>&lt;noinclude&gt;</span>
       </dd>
      </dl> 
      <p>These tags can be nested inside each other, though (for a given page) this really only applies to the <code class="nowrap">&lt;<a href="/wiki/Help:Template#Noinclude,_includeonly,_and_onlyinclude" title="Help:Template">onlyinclude</a>&gt;</code> tag; nesting <code class="nowrap">&lt;<a href="/wiki/Help:Template#Noinclude,_includeonly,_and_onlyinclude" title="Help:Template">includeonly</a>&gt;</code> and <code class="nowrap">&lt;<a href="/wiki/Help:Template#Noinclude,_includeonly,_and_onlyinclude" title="Help:Template">noinclude</a>&gt;</code> tags is fairly pointless. Be careful to properly nest the tags, however. Constructions like <code class="nowrap">&lt;<a href="/wiki/Help:Template#Noinclude,_includeonly,_and_onlyinclude" title="Help:Template">onlyinclude</a>&gt;</code>abc<code class="nowrap">&lt;<a href="/wiki/Help:Template#Noinclude,_includeonly,_and_onlyinclude" title="Help:Template">includeonly</a>&gt;</code>def<code class="nowrap">&lt;/onlyinclude&gt;</code>ghi<code class="nowrap">&lt;/includeonly&gt;</code> will <em>not</em> work as expected. Use the "first opened, last closed" rule that is standard for HTML/XML. </p> 
      <h3><span class="mw-headline" id="Problems_and_workarounds">Problems and workarounds</span></h3> 
      <ul>
       <li>If the first character produced by a template or <a href="/wiki/Wikipedia:PF" class="mw-redirect" title="Wikipedia:PF">parser function</a> is one of four wiki markup characters—<code>:</code>, <code>;</code>, <code>*</code>, <code>#</code><sup id="cite_ref-8" class="reference"><a href="#cite_note-8">[h]</a></sup>—then it is processed as though it were at the beginning of a line, even when the template tag is not. This allows the creation of various kinds of lists in templates where the template may not always be in the correct place for a list. To avoid this, either use <code class="nowrap">&lt;<a href="/wiki/Help:Nowiki" class="mw-redirect" title="Help:Nowiki">nowiki</a> /&gt;</code> before the markup or use the <a href="/wiki/HTML" title="HTML">HTML</a> entities <code>&amp;#58;</code>, <code>&amp;#59;</code>, <code>&amp;#42;</code>, and <code>&amp;#35;</code> respectively. In some cases, the HTML entities will work when the <code class="nowrap" style="">&lt;nowiki /&gt;</code> does not. The problem often occurs when a parameter value in a template call starts with one of the four characters.</li> 
       <li>For issues involving the substitution of templates, such as how to control whether subtemplates are substituted as well when the parent template is substituted, see <a href="/wiki/Help:Substitution" title="Help:Substitution">Help:Substitution</a>.</li> 
       <li>The template <code>{{<a href="/wiki/Template:Strip_whitespace" class="mw-redirect" title="Template:Strip whitespace">Strip whitespace</a>}}</code> can be used to strip any initial or final whitespace from unnamed parameter values if this would cause problems; named parameter values are automatically stripped in this way.</li> 
       <li>For debugging templates, the following techniques are sometimes helpful: 
        <ul>
         <li>Use <code>subst:</code> to substitute a template (rather than transcluding it), which can show more clearly what is happening when the template is transcluded; see <a href="/wiki/Help:Substitution" title="Help:Substitution">Help:Substitution</a>.</li> 
         <li>Use <code>msgnw:</code> (short for "<b>m</b>e<b>s</b>sa<b>g</b>e, <b>n</b>o<b>w</b>iki") to more-or-less transclude the wikitext of the template page rather than the processed contents. It is not perfect: lists are rendered, comments are removed, and single newlines are replaced with spaces (which is particularly confounding when transcluding wikitext tables).</li> 
         <li>Use <a href="/wiki/Special:ExpandTemplates" title="Special:ExpandTemplates">Special:ExpandTemplates</a> to see the full recursive expansion of one or more templates.</li>
        </ul></li> 
       <li>To protect server resources and avoid infinite loops, the parser imposes certain limits on the depth transclusion nesting and on the page size with expanded templates. This may cause pages to break when using very complex templates, particularly if there are multiple such templates on the same page. For more information, see <a href="/wiki/Wikipedia:Template_limits" title="Wikipedia:Template limits">Wikipedia:Template limits</a>. A page's overall load on the server can be checked by examining the generated HTML for a page and looking for the <code>NewPP limit report</code> comments.</li> 
       <li>Do not use <code>=</code> wikimarkup to create section headers within a template which is intended for use in article space; this will create an <code>[edit]</code> link that, when transcluded, will confusingly open the template for editing. 
        <ul>
         <li>You may avoid <code>[edit]</code> links to the template by including <code>&lt;includeonly&gt;__NOEDITSECTION__&lt;/includeonly&gt;</code>.</li>
        </ul></li>
      </ul> 
      <h3><span class="mw-headline" id="Documentation">Documentation</span></h3> 
      <div role="note" class="hatnote navigation-not-searchable">
       Further information: <a href="/wiki/Wikipedia:Template_documentation" title="Wikipedia:Template documentation">Wikipedia:Template documentation</a>
      </div> 
      <p>Categorizing your template and documenting its proper usage will make it easier for other editors to find and use. </p>
      <p>Documentation for users, together with the template's categories, is normally placed after the template code, inside <code class="nowrap" style="">&lt;noinclude&gt;...&lt;/noinclude&gt;</code> tags. It is normally necessary to put the opening <code class="nowrap" style="">&lt;noinclude&gt;</code> tag immediately after the end of the code, with no intervening spaces or newlines, to avoid transcluding unwanted whitespace. </p>
      <p>In the case of complex templates, the documentation (together with categories) is often kept on a separate <a href="/wiki/Wikipedia:Subpage" class="mw-redirect" title="Wikipedia:Subpage">subpage</a> of the template page (named "Template:<var style="padding-right: 1px;">XXX</var>/doc"). This also applies to many <a href="/wiki/Wikipedia:Protection" class="mw-redirect" title="Wikipedia:Protection">protected</a> templates, which allows the information to be edited by non-administrators. This is achieved by placing the <code>{{<a href="/wiki/Template:Documentation" title="Template:Documentation">Documentation</a>}}</code> template after the main template code and within <code class="nowrap" style="">&lt;noinclude&gt;...&lt;/noinclude&gt;</code> tags. If the "/doc" subpage does not exist, a link will then appear enabling it to be created. </p> 
      <h3><span class="mw-headline" id="Categorization">Categorization</span></h3> 
      <div role="note" class="hatnote navigation-not-searchable">
       See also: <a href="/wiki/Wikipedia:Categorization#Template_categorization" title="Wikipedia:Categorization">Wikipedia:Categorization §&nbsp;Template categorization</a>
      </div> 
      <h4><span class="mw-headline" id="Categorize_pages_by_template_inclusion">Categorize pages by template inclusion</span></h4> 
      <p>Some templates contain category definitions in their transcluded code, since they are intended to place the target pages in particular categories. This is often done with maintenance categories (placing articles into ordinary content categories in this way is discouraged). When doing this, it may be necessary to use <code class="nowrap" style="">&lt;includeonly&gt;...&lt;/includeonly&gt;</code> tags to keep the template itself out of the category. While developing, testing, sandboxing, or demonstrating a template intended to apply a category, either temporarily replace each category with a test category (starting with <a href="/wiki/Category:X1" title="Category:X1">X1</a>, <a href="/wiki/Category:X2" title="Category:X2">X2</a>, or <a href="/wiki/Category:X3" title="Category:X3">X3</a>) or suppress categorization (see <a href="/wiki/Wikipedia:CATSUP" class="mw-redirect" title="Wikipedia:CATSUP">category suppression in templates</a>). </p> 
      <h4><span class="mw-headline" id="Categorize_templates">Categorize templates</span></h4> 
      <p>Categorization declaration <code>[[Category:Some-topic templates]]</code> should be placed on the template's documentation page (or inside <code class="nowrap" style="">&lt;noinclude&gt;...&lt;/noinclude&gt;</code> <a href="/wiki/Html_tag" class="mw-redirect" title="Html tag">tags</a> if there is no documentation page) to avoid polluting the transcluding pages. </p> 
      <h3><span class="mw-headline" id="Aliases">Aliases</span></h3> 
      <p>Aliases can be created with <a href="/wiki/Wikipedia:Redirect" title="Wikipedia:Redirect">redirects</a>. For example, <a href="/wiki/Template:Tsh" class="mw-redirect" title="Template:Tsh">Template:Tsh</a> redirects to <a href="/wiki/Template:Template_shortcut" title="Template:Template shortcut">Template:Template shortcut</a>. You can then write <code>{{<a href="/wiki/Template:Tsh" class="mw-redirect" title="Template:Tsh">tsh</a>|<var style="padding-right: 1px;">foo</var>}}</code> instead of <code>{{<a href="/wiki/Template:Template_shortcut" title="Template:Template shortcut">Template shortcut</a>|<var style="padding-right: 1px;">foo</var>}}</code>. </p>
      <p>It is good to prepare template aliases which only differ in whitespaces and capitalization. For example, there is a template called <code>{{<a href="/wiki/Template:See_Wiktionary" title="Template:See Wiktionary">See Wiktionary</a>}}</code>. The "W" is capital, since the word "Wiktionary" is so, but a redirect <code>{{<a href="/wiki/Template:See_wiktionary" class="mw-redirect" title="Template:See wiktionary">See wiktionary</a>}}</code> with lower "w" exists because users may type the latter instead. </p> 
      <h2><span class="mw-headline" id="Template_limits">Template limits</span></h2> 
      <p><span class="anchor" id="Expand_limits"></span> </p> 
      <div role="note" class="hatnote navigation-not-searchable">
       Further information: <a href="/wiki/Wikipedia:Template_limits" title="Wikipedia:Template limits">Wikipedia:Template limits</a>
      </div> 
      <p><b>"Post-expand include size" limit.</b> When templates are rendered or expanded to HTML for viewing in your browser, they use memory. This is called the "post-expand include size" and has a limit of 2,048,000&nbsp;bytes. This size is included as an invisible comment in the HTML output—use your browser's view source feature to show the raw HTML and search for "newpp". The report will look like: </p> 
      <div class="mw-highlight mw-highlight-lang-html mw-content-ltr" dir="ltr">
       <pre><span></span><span class="c">&lt;!--</span>
<span class="c">NewPP limit report</span>
<span class="c">Preprocessor node count: 2382/1000000</span>
<span class="c">Post-expand include size: 63476/2048000 bytes</span>
<span class="c">Template argument size: 9517/2048000 bytes</span>
<span class="c">Expensive parser function count: 2/500</span>
<span class="c">--&gt;</span>
</pre>
      </div> 
      <p>The example shows that template expansion is using 63,476&nbsp;bytes out of 2,048,000&nbsp;bytes of available memory. </p>
      <p><b>Display problem.</b> If too many templates are included on a page, the post-expand include size may exceed the limit. When this happens, templates after the limit will no longer expand and will instead display as a wikilink (for example, <span style="color: red;"><u>Template:<var style="padding-right: 1px;">Template name</var></u></span>). Common causes are the inclusion of too many citation templates and/or flag templates. To resolve this problem <a href="/wiki/Wikipedia:Substitution" title="Wikipedia:Substitution">substitute</a> templates, remove templates, or <a href="/wiki/Wikipedia:Splitting" title="Wikipedia:Splitting">split</a> the page. </p>
      <p><b>Non-rendered tranclusions still count towards limit.</b> For example, a page which contains only <code>{{#if:{{:Main Page}}}}</code> would still have a post-expand include size even though it would have no output at all. </p>
      <p>The same applies to <a href="/wiki/Wikipedia:Lua" title="Wikipedia:Lua">Scribunto modules</a>. For example, <code>{{#invoke:Test|main}}</code> would still increase post-expand include size even if Module:Test were simply: </p> 
      <div class="mw-highlight mw-highlight-lang-lua mw-content-ltr" dir="ltr">
       <pre><span></span><span class="n">mw</span><span class="p">.</span><span class="n">getCurrentFrame</span><span class="p">():</span><span class="n">preprocess</span> <span class="s2">"{{msgnw::Main Page}}"</span> <span class="c1">-- remove this line and post-expand include size becomes zero</span>
<span class="kr">return</span> <span class="p">{</span><span class="n">main</span> <span class="o">=</span> <span class="kr">function</span><span class="p">()</span> <span class="kr">end</span><span class="p">}</span> <span class="c1">-- p.main() has no return value</span>
</pre>
      </div> 
      <h2><span class="mw-headline" id="Lua_programming_language">Lua programming language</span></h2> 
      <div role="note" class="hatnote navigation-not-searchable">
       Main page: <a href="/wiki/Wikipedia:Lua" title="Wikipedia:Lua">Wikipedia:Lua</a>
      </div> 
      <p>Since February 2013, the <a href="/wiki/Lua_(programming_language)" title="Lua (programming language)">Lua programming language</a> is available for use through the <a href="https://www.mediawiki.org/wiki/Extension:Scribunto" class="extiw" title="mw:Extension:Scribunto">Scribunto</a> MediaWiki extension. Lua code can be embedded into templates by employing the <code>{{#invoke:}}</code> functionality of the Scribunto MediaWiki extension. The Lua <a href="/wiki/Source_code" title="Source code">source code</a> is stored in pages called modules, and these individual modules are then invoked on template pages. For example, <a href="/wiki/Module:Example" title="Module:Example">Module:Example</a> can be invoked using the code <code>{{#invoke:Example|hello}}</code> to print the text "Hello World!". </p>
