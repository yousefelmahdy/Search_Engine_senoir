       <p>People with admin permissions for a repository can configure a custom domain for a GitHub Pages site.</p> 
       <h3 id="about-custom-domain-configuration"><a href="#about-custom-domain-configuration">About custom domain configuration</a></h3> 
       <p>Make sure you add your custom domain to your GitHub Pages site before configuring your custom domain with your DNS provider. Configuring your custom domain with your DNS provider without adding your custom domain to GitHub could result in someone else being able to host a site on one of your subdomains.</p> 
       <div class="extended-markdown windows"> 
        <p>The <code>dig</code> command, which can be used to verify correct configuration of DNS records, is not included in Windows. Before you can verify that your DNS records are configured correctly, you must install <a href="https://www.isc.org/bind/">BIND</a>.</p> 
       </div> 
       <div class="extended-markdown note border rounded-1 mb-4 p-3 color-border-info color-bg-info f5"> 
        <p><strong>Note:</strong> DNS changes can take up to 24 hours to propagate.</p> 
       </div>

       <h3 id="configuring-an-apex-domain"><a href="#configuring-an-apex-domain">Configuring an apex domain</a></h3>

       <p>If you are using an apex domain as your custom domain, we recommend also setting up a <code>www</code> subdomain. Note that automatic redirects only apply to the <code>www</code> subdomain. Automatic redirects do not apply to any other subdomains, such as <code>blog</code>. For more information, see "<a href="#configuring-a-subdomain">Configuring a subdomain</a>."</p>

       <h3 id="configuring-an-apex-domain-and-the-www-subdomain-variant"><a href="#configuring-an-apex-domain-and-the-www-subdomain-variant">Configuring an apex domain and the <code>www</code> subdomain variant</a></h3> 
       <p>When using an apex domain, we recommend configuring your GitHub Pages site to host content at both the apex domain and that domain's <code>www</code> subdomain variant.</p> 
       <p>To set up a <code>www</code> subdomain alongside the apex domain, you must first configure an apex domain, which will create an <code>ALIAS</code>, <code>ANAME</code>, or <code>A</code> record with your DNS provider. For more information, see "<a href="#configuring-an-apex-domain">Configuring an apex domain</a>."</p> 
       <p>After you configure the apex domain, you must configure a CNAME record with your DNS provider.</p> 
       <ol> 
        <li>Navigate to your DNS provider and create a <code>CNAME</code> record that points <code>www.example.com</code> to the default domain for your site: <code>&lt;user&gt;.github.io</code> or <code>&lt;organization&gt;.github.io</code>. Do not include the repository name. For more information about how to create the correct record, see your DNS provider's documentation. For more information about the default domain for your site, see "<a href="/en/articles/about-github-pages#types-of-github-pages-sites">About GitHub Pages</a>."</li>
