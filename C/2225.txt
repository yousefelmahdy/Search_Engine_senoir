       <h3 id="about-sarif-support"><a href="#about-sarif-support">About SARIF support</a></h3> 
       <p>SARIF (Static Analysis Results Interchange Format) is an <a href="https://docs.oasis-open.org/sarif/sarif/v2.1.0/sarif-v2.1.0.html">OASIS Standard</a> that defines an output file format. The SARIF standard is used to streamline how static analysis tools share their results. Code scanning supports a subset of the SARIF 2.1.0 JSON schema.</p> 
       <p>To upload a SARIF file from a third-party static code analysis engine, you'll need to ensure that uploaded files use the SARIF 2.1.0 version. GitHub will parse the SARIF file and show alerts using the results in your repository as a part of the code scanning experience. For more information, see "<a href="/en/code-security/secure-coding/uploading-a-sarif-file-to-github">Uploading a SARIF file to GitHub</a>." For more information about the SARIF 2.1.0 JSON schema, see <a href="https://github.com/oasis-tcs/sarif-spec/blob/master/Schemata/sarif-schema-2.1.0.json"><code>sarif-schema-2.1.0.json</code></a>.</p> 
       <p>If you're using GitHub Actions with the CodeQL analysis workflow or using the CodeQL runner, then the code scanning results will automatically use the supported subset of SARIF 2.1.0. For more information, see "<a href="/en/code-security/secure-coding/setting-up-code-scanning-for-a-repository">Setting up code scanning for a repository</a>" or "<a href="/en/code-security/secure-coding/running-codeql-runner-in-your-ci-system">Running CodeQL runner in your CI system</a>."</p> 
       <p>If you're using the CodeQL CLI, then you can specify the version of SARIF to use. For more information, see "<a href="/en/code-security/secure-coding/running-codeql-cli-in-your-ci-system#uploading-results-to-github">Running CodeQL CLI in your CI system</a>."</p> 
       <p>You can upload multiple SARIF files for the same tool and commit, and analyze each file using code scanning. You can indicate a "category" for each analysis by specifying a <code>runAutomationDetails.id</code> in each file. Only SARIF files with the same category will overwrite each other. For more information about this property, see <a href="#runautomationdetails-object"><code>runAutomationDetails</code> object</a> below.</p> 
       <p>GitHub uses properties in the SARIF file to display alerts. For example, the <code>shortDescription</code> and <code>fullDescription</code> appear at the top of a code scanning alert. The <code>location</code> allows GitHub to show annotations in your code file. For more information, see "<a href="/en/code-security/secure-coding/managing-code-scanning-alerts-for-your-repository">Managing code scanning alerts for your repository</a>."</p> 
       <p>If you're new to SARIF and want to learn more, see Microsoft's <a href="https://github.com/microsoft/sarif-tutorials"><code>SARIF tutorials</code></a> repository.</p> 
       <h3 id="preventing-duplicate-alerts-using-fingerprints"><a href="#preventing-duplicate-alerts-using-fingerprints">Preventing duplicate alerts using fingerprints</a></h3> 
       <p>Each time the results of a new code scan are uploaded, the results are processed and alerts are added to the repository. To prevent duplicate alerts for the same problem, code scanning uses fingerprints to match results across various runs so they only appear once in the latest run for the selected branch. This makes it possible to match alerts to the right line of code when files are edited.</p> 
       <p>GitHub uses the <code>partialFingerprints</code> property in the OASIS standard to detect when two results are logically identical. For more information, see the "<a href="https://docs.oasis-open.org/sarif/sarif/v2.1.0/cs01/sarif-v2.1.0-cs01.html#_Toc16012611">partialFingerprints property</a>" entry in the OASIS documentation.</p> 
       <p>SARIF files created by the CodeQL analysis workflow or using the CodeQL runner include fingerprint data. If you upload a SARIF file using the <code>upload-sarif</code> action and this data is missing, GitHub attempts to populate the <code>partialFingerprints</code> field from the source files. For more information about uploading results, see "<a href="/en/code-security/secure-coding/uploading-a-sarif-file-to-github#uploading-a-code-scanning-analysis-with-github-actions">Uploading a SARIF file to GitHub</a>."</p> 
       <p>If you upload a SARIF file without fingerprint data using the <code>/code-scanning/sarifs</code> API endpoint, the code scanning alerts will be processed and displayed, but users may see duplicate alerts. To avoid seeing duplicate alerts, you should calculate fingerprint data and populate the <code>partialFingerprints</code> property before you upload the SARIF file. You may find the script that the <code>upload-sarif</code> action uses a helpful starting point: <a href="https://github.com/github/codeql-action/blob/main/src/fingerprints.ts">https://github.com/github/codeql-action/blob/main/src/fingerprints.ts</a>. For more information about the API, see "<a href="/en/rest/reference/code-scanning#upload-an-analysis-as-sarif-data">Upload an analysis as SARIF data</a>."</p> 
       <h3 id="validating-your-sarif-file"><a href="#validating-your-sarif-file">Validating your SARIF file</a></h3> 
       <p>You can check a SARIF file is compatible with code scanning by testing it against the GitHub ingestion rules. For more information, visit the <a href="https://sarifweb.azurewebsites.net/">Microsoft SARIF validator</a>.</p> 
       <div class="extended-markdown note border rounded-1 mb-4 p-3 color-border-info color-bg-info f5"> 
        <p><strong>Note:</strong> SARIF upload supports a maximum of 5000 results per upload. Any results over this limit are ignored. If a tool generates too many results, you should update the configuration to focus on results for the most important rules or queries.</p> 
       </div> 
       <h3 id="supported-sarif-output-file-properties"><a href="#supported-sarif-output-file-properties">Supported SARIF output file properties</a></h3> 
       <p>If you use a code analysis engine other than CodeQL, you can review the supported SARIF properties to optimize how your analysis results will appear on GitHub.</p> 
       <p>Any valid SARIF 2.1.0 output file can be uploaded, however, code scanning will only use the following supported properties.</p>

         </tr>
         <tr>
          <td><code>partialFingerprints</code></td>
          <td><strong>Required.</strong> A set of strings used to track the unique identity of the result.          <tr>

          <td><strong>Optional.</strong> An array of <code>location</code> objects for a <code>threadFlow</code> object, which describes the progress of a program through a thread of execution. For more information, see the <a href="#location-object"><code>location</code> object</a>.</td>
         </tr>
         <tr>
          <td><code>relatedLocations[]</code></td>
          <td>A set of locations relevant to this result. Code scanning will link to related locations when they are embedded in the result message. For more information, see the <a href="#location-object"><code>location</code> object</a>.</td>
         </tr>

       <h4 id="runautomationdetails-object"><a href="#runautomationdetails-object"><code>runAutomationDetails</code> object</a></h4> 
       <p>The <code>runAutomationDetails</code> object contains information that specifies the identity of a run.</p> 
       <div class="extended-markdown note border rounded-1 mb-4 p-3 color-border-info color-bg-info f5"> 
        <p><strong>Note:</strong> <code>runAutomationDetails</code> is a SARIF v2.1.0 object. If you're using the CodeQL CLI, you can specify the version of SARIF to use. The equivalent object to <code>runAutomationDetails</code> is <code>&lt;run&gt;.automationId</code> for SARIF v1 and <code>&lt;run&gt;.automationLogicalId</code> for SARIF v2.</p> 
       </div> 
       <table>
        <thead>
         <tr>
          <th>Name</th>
          <th>Description</th>
         </tr>
        </thead>
        <tbody>
         <tr>
          <td><code>id</code></td>
          <td><strong>Optional.</strong> A string that identifies the category of the analysis and the run ID. Use if you want to upload multiple SARIF files for the same tool and commit, but performed on different languages or different parts of the code.</td>
         </tr>
        </tbody>
       </table> 
       <p>The use of the <code>runAutomationDetails</code> object is optional.</p> 
       <p>The <code>id</code> field can include an analysis category and a run ID. We don't use the run ID part of the <code>id</code> field, but we store it.</p> 
       <p>Use the category to distinguish between multiple analyses for the same tool or commit, but performed on different languages or different parts of the code. Use the run ID to identify the specific run of the analysis, such as the date the analysis was run.</p> 
       <p><code>id</code> is interpreted as <code>category/run-id</code>. If the <code>id</code> contains no forward slash (<code>/</code>), then the entire string is the <code>run_id</code> and the <code>category</code> is empty. Otherwise, <code>category</code> is everything in the string until the last forward slash, and <code>run_id</code> is everything after.</p> 
       <table>
        <thead>
         <tr>
          <th><code>id</code></th>
          <th>category</th>
          <th><code>run_id</code></th>
         </tr>
        </thead>
        <tbody>
         <tr>
          <td>my-analysis/tool1/2021-02-01</td>
          <td>my-analysis/tool1</td>
          <td>2021-02-01</td>
         </tr>
         <tr>
          <td>my-analysis/tool1/</td>
          <td>my-analysis/tool1</td>
          <td><em>no <code>run-id</code></em></td>
         </tr>
         <tr>
          <td>my-analysis for tool1</td>
          <td><em>no category</em></td>
          <td>my-analysis for tool1</td>
         </tr>
        </tbody>
       </table> 
       <ul> 
        <li>The run with an <code>id</code> of "my-analysis/tool1/2021-02-01" belongs to the category "my-analysis/tool1". Presumably, this is the run from February 2, 2021.</li> 
        <li>The run with an <code>id</code> of "my-analysis/tool1/" belongs to the category "my-analysis/tool1" but is not distinguished from other runs in that category.</li> 
        <li>The run whose <code>id</code> is "my-analysis for tool1 " has a unique identifier but cannot be inferred to belong to any category.</li> 
       </ul> 
       <p>For more information about the <code>runAutomationDetails</code> object and the <code>id</code> field, see <a href="https://docs.oasis-open.org/sarif/sarif/v2.1.0/cs01/sarif-v2.1.0-cs01.html#_Toc16012479">runAutomationDetails object</a> in the OASIS documentation. </p> 
       <p>Note that the rest of the supported fields are ignored.</p>
