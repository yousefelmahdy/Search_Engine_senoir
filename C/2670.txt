       <h3 id="prerequisites"><a href="#prerequisites">Prerequisites</a></h3> 
       <p>To develop in a codespace directly in Visual Studio Code, you must sign into the GitHub Codespaces extension. The GitHub Codespaces extension requires Visual Studio Code October 2020 Release 1.51 or later.</p> 
       <p>Use the Visual Studio Marketplace to install the <a href="https://marketplace.visualstudio.com/items?itemName=GitHub.codespaces">GitHub Codespaces</a> extension. For more information, see <a href="https://code.visualstudio.com/docs/editor/extension-gallery">Extension Marketplace</a> in the Visual Studio Code documentation.</p> 
       <div class="extended-markdown mac"> 
        <ol> 
         <li>In Visual Studio Code, in the left sidebar, click the Remote Explorer icon. <span class="procedural-image-wrapper"><img src="/assets/images/help/codespaces/click-remote-explorer-icon-vscode.png" alt="The Remote Explorer icon in Visual Studio Code"></span></li> 
         <li>Click <strong>Sign in to view GitHub...</strong>. <span class="procedural-image-wrapper"><img src="/assets/images/help/codespaces/sign-in-to-view-codespaces-vscode-mac.png" alt="Signing in to view Codespaces"></span></li> 
         <li>To authorize Visual Studio Code to access your account on GitHub, click <strong>Allow</strong>.</li> 
         <li>Sign in to GitHub to approve the extension.</li> 
        </ol> 
       </div> 
       <div class="extended-markdown windows"> 
        <ol> 
         <li>In Visual Studio Code, in the left sidebar, click the Remote Explorer icon. <span class="procedural-image-wrapper"><img src="/assets/images/help/codespaces/click-remote-explorer-icon-vscode.png" alt="The Remote Explorer icon in Visual Studio Code"></span></li> 
         <li>Use the "REMOTE EXPLORER" drop-down, then click <strong>GitHub Codespaces</strong>. <span class="procedural-image-wrapper"><img src="/assets/images/help/codespaces/codespaces-header-vscode.png" alt="The Codespaces header"></span></li> 
         <li>Click <strong>Sign in to view Codespaces...</strong>. <span class="procedural-image-wrapper"><img src="/assets/images/help/codespaces/sign-in-to-view-codespaces-vscode.png" alt="Signing in to view Codespaces"></span></li> 
         <li>To authorize Visual Studio Code to access your account on GitHub, click <strong>Allow</strong>.</li> 
         <li>Sign in to GitHub to approve the extension.</li> 
        </ol> 
       </div> 
       <h3 id="creating-a-codespace-in-visual-studio-code"><a href="#creating-a-codespace-in-visual-studio-code">Creating a codespace in Visual Studio Code</a></h3> 
       <p>After you connect your GitHub account to the GitHub Codespaces extension, you can develop in a codespace that you created on GitHub or in Visual Studio Code.</p> 
       <ol> 
        <li>In Visual Studio Code, in the left sidebar, click the Remote Explorer icon. <span class="procedural-image-wrapper"><img src="/assets/images/help/codespaces/click-remote-explorer-icon-vscode.png" alt="The Remote Explorer icon in Visual Studio Code"></span></li> 
        <li>Click the Add icon, then click <strong>Create New Codespace</strong>. <span class="procedural-image-wrapper"><img src="/assets/images/help/codespaces/create-codespace-vscode.png" alt="The Create new Codespace option in Codespaces"></span></li> 
        <li>Type, then click the repository's name you want to develop in. <span class="procedural-image-wrapper"><img src="/assets/images/help/codespaces/choose-repository-vscode.png" alt="Searching for repository to create a new Codespaces"></span></li> 
        <li>Click the branch you want to develop on. <span class="procedural-image-wrapper"><img src="/assets/images/help/codespaces/choose-branch-vscode.png" alt="Searching for a branch to create a new Codespaces"></span></li> 
        <li>Click the machine type you want to develop in. <span class="procedural-image-wrapper"><img src="/assets/images/help/codespaces/choose-sku-vscode.png" alt="Instance types for a new Codespaces"></span></li> 
       </ol> 
       <h3 id="opening-a-codespace-in-visual-studio-code"><a href="#opening-a-codespace-in-visual-studio-code">Opening a codespace in Visual Studio Code</a></h3> 
       <ol> 
        <li>In Visual Studio Code, in the left sidebar, click the Remote Explorer icon. <span class="procedural-image-wrapper"><img src="/assets/images/help/codespaces/click-remote-explorer-icon-vscode.png" alt="The Remote Explorer icon in Visual Studio Code"></span></li> 
        <li>Under "Codespaces", click the codespace you want to develop in.</li> 
        <li>Click the Connect to Codespace icon. <span class="procedural-image-wrapper"><img src="/assets/images/help/codespaces/click-connect-to-codespace-icon-vscode.png" alt="The Connect to Codespace icon in Visual Studio Code"></span></li> 
       </ol> 
       <h3 id="changing-the-machine-type-in-visual-studio-code"><a href="#changing-the-machine-type-in-visual-studio-code">Changing the machine type in Visual Studio Code</a></h3> 
       <p>You can run your codespace on a choice of three types of remote machine: Basic, Standard, or Premium. Each of these has a different level of resources and a different billing tier. <!-- For information, see "[About billing for Codespaces](/github/developing-online-with-codespaces/about-billing-for-codespaces)." --></p> 
       <div class="extended-markdown note border rounded-1 mb-4 p-3 color-border-info color-bg-info f5"> 
        <p><strong>Note:</strong> Codespaces is free to use during the beta. When Codespaces becomes generally available, you will be billed for storage and compute usage.</p> 
       </div> 
       <p>Switching machine types gives you the flexibility to choose the most appropriate resources for your work. Certain tasks may require different machine types: </p> 
       <ul> 
        <li><strong>Basic</strong> for updating a file and raising a pull request</li> 
        <li><strong>Standard</strong> for editing a website</li> 
        <li><strong>Premium</strong> for high compute tasks such as data analysis</li> 
       </ul> 
       <p>You can change the machine type of your codespace at any time.</p> 
       <ol> 
        <li>In Visual Studio Code, open the Command Palette (<code>shift command P</code> / <code>shift control P</code>).</li> 
        <li>Search for and select "Codespaces: Change Machine Type." <span class="procedural-image-wrapper"><img src="/assets/images/help/codespaces/vscode-change-machine-type-option.png" alt="Searching for a branch to create a new Codespaces"></span></li> 
        <li>Click the codespace that you want to change. <span class="procedural-image-wrapper"><img src="/assets/images/help/codespaces/vscode-change-machine-choose-repo.png" alt="Searching for a branch to create a new Codespaces"></span></li> 
        <li>Choose the machine type you want to use. <span class="procedural-image-wrapper"><img src="/assets/images/help/codespaces/vscode-change-machine-choose-type.png" alt="Searching for a branch to create a new Codespaces"></span></li> 
       </ol> 
       <p>If the codespace is currently running, a message is displayed asking if you would like to restart and reconnect to your codespace now. Click <strong>Yes</strong> if you want to change the machine type used for this codespace immediately. If you click <strong>No</strong>, or if the codespace is not currently running, the change will take effect the next time the codespace restarts.</p> 
       <h3 id="deleting-a-codespace-in-visual-studio-code"><a href="#deleting-a-codespace-in-visual-studio-code">Deleting a codespace in Visual Studio Code</a></h3> 
       <ol> 
        <li>Under "Codespaces", right-click the codespace you want to delete.</li> 
        <li>In the drop-down menu, click <strong>Delete Codespace</strong>. <span class="procedural-image-wrapper"><img src="/assets/images/help/codespaces/delete-codespace-vscode.png" alt="Deleting a codespace in GitHub"></span></li> 
       </ol>
