         <h1 id="epics">Epics <span class="badge-trigger premium"></span><a href="#epics" title="Permalink" class="anchor"></a> </h1> 
         <div class="introduced-in mb-3">
          Version history<button class="text-expander" data-toggle="collapse" href="#release_version_notes_1" role="button" aria-expanded="false"></button>
          <div class="introduced-in-content collapse" id="release_version_notes_1"> 
           <ul> 
            <li>Introduced in <a href="https://about.gitlab.com/pricing/">GitLab Ultimate</a> 10.2. </li>
            <li>Single-level Epics <a href="https://gitlab.com/gitlab-org/gitlab/-/issues/37081">were moved</a> to <a href="https://about.gitlab.com/pricing/">GitLab Premium</a> in 12.8. </li>
           </ul> 
          </div> 
         </div> 
         <p>Epics let you manage your portfolio of projects more efficiently by tracking groups of <a href="../../project/issues/index.html">issues</a> that share a theme across projects and milestones. </p>
         <p>An epics page contains the following tabs: </p>
         <ul> 
          <li> <strong>Issues</strong>: issues added to this epic. </li>
          <li> <p><strong>Epics and Issues</strong>: epics and issues added to this epic. Appears instead of the <strong>Issues</strong> tab if you have access to <a href="#multi-level-child-epics">multi-level epics</a>. Child epics and their issues are shown in a tree view. </p>
           <ul> 
            <li>To reveal the child epics and issues, select the chevron (<strong>&gt;</strong>) next to a parent epic. </li>
            <li> <p>To see a breakdown of open and closed items, hover over the total counts. </p><p>The number provided here includes all epics associated with this project. The number includes epics for which users may not yet have permission. </p></li>
           </ul> </li>
          <li> <a href="#roadmap-in-epics"><strong>Roadmap</strong></a>: a roadmap view of child epics which have start and due dates. Appears if you have access to <a href="#multi-level-child-epics">multi-level epics</a>. </li>
         </ul> 
         <p><a href="img/epic_view_v13.0.png" target="_blank" rel="noopener noreferrer"><img src="img/epic_view_v13.0.png" alt="epic view"></a> </p>
         <h2 id="use-cases">Use cases<a href="#use-cases" title="Permalink" class="anchor"></a> </h2> 
         <ul> 
          <li>Suppose your team is working on a large feature that involves multiple discussions throughout different issues created in distinct projects within a <a href="../index.html">Group</a>. With Epics, you can track all the related activities that together contribute to that single feature. </li>
          <li>Track when the work for the group of issues is targeted to begin, and when its targeted to end. </li>
          <li>Discuss and collaborate on feature ideas and scope at a high level. </li>
         </ul> 
         <h2 id="manage-epics">Manage epics<a href="#manage-epics" title="Permalink" class="anchor"></a> </h2> 
         <p>To learn what you can do with an epic, see <a href="manage_epics.html">Manage epics</a>. Possible actions include: </p>
         <ul> 
          <li><a href="manage_epics.html#create-an-epic">Create an epic</a> </li>
          <li><a href="manage_epics.html#edit-an-epic">Edit an epic</a> </li>
          <li><a href="../bulk_editing/index.html#bulk-edit-epics">Bulk-edit epics</a> </li>
          <li><a href="manage_epics.html#delete-an-epic">Delete an epic</a> </li>
          <li><a href="manage_epics.html#close-an-epic">Close an epic</a> </li>
          <li><a href="manage_epics.html#reopen-a-closed-epic">Reopen a closed epic</a> </li>
          <li><a href="manage_epics.html#go-to-an-epic-from-an-issue">Go to an epic from an issue</a> </li>
          <li><a href="manage_epics.html#search-for-an-epic-from-epics-list-page">Search for an epic from epics list page</a> </li>
          <li><a href="manage_epics.html#make-an-epic-confidential">Make an epic confidential</a> </li>
          <li><a href="manage_epics.html#manage-issues-assigned-to-an-epic">Manage issues assigned to an epic</a> </li>

         <h2 id="relationships-between-epics-and-issues">Relationships between epics and issues<a href="#relationships-between-epics-and-issues" title="Permalink" class="anchor"></a> </h2> 
         <p>The possible relationships between epics and issues are: </p>
         <ul> 
          <li>An epic is the parent of one or more issues. </li>
          <li>An epic is the parent of one or more child epics. For details see <a href="#multi-level-child-epics">Multi-level child epics</a>. <span class="badge-trigger ultimate"></span> </li>
         </ul> 
         <div class="mermaid">
          graph TD Parent_epic --&gt; Issue1 Parent_epic --&gt; Child_epic Child_epic --&gt; Issue2 
         </div> 
         <p>See <a href="manage_epics.html#manage-issues-assigned-to-an-epic">Manage issues assigned to an epic</a> for steps to: </p>
         <ul> 
          <li>Add an issue to an epic </li>
          <li>Reorder issues </li>
          <li>Move an issue between epics </li>
          <li>Promote an issue to an epic </li>
         </ul> 
         <h2 id="issue-health-status-in-epic-tree">Issue health status in Epic tree <span class="badge-trigger ultimate"></span><a href="#issue-health-status-in-epic-tree" title="Permalink" class="anchor"></a> </h2> 
         <div class="introduced-in mb-3">
          Version history<button class="text-expander" data-toggle="collapse" href="#release_version_notes_2" role="button" aria-expanded="false"></button>
          <div class="introduced-in-content collapse" id="release_version_notes_2"> 
           <ul> 
            <li> <a href="https://gitlab.com/gitlab-org/gitlab/-/issues/199184">Introduced</a> in <a href="https://about.gitlab.com/pricing/">GitLab Ultimate</a> 12.10. </li>
            <li>The health status of a closed issue <a href="https://gitlab.com/gitlab-org/gitlab/-/issues/220867">is hidden</a> in <a href="https://about.gitlab.com/pricing/">GitLab Ultimate</a> 13.3 or later. </li>
            <li> <a href="https://gitlab.com/gitlab-org/gitlab/-/issues/213567">Feature flag removed</a> in GitLab 13.7. </li>
           </ul> 
          </div> 
         </div> 
         <p>Report or respond to the health of issues and epics by setting a red, amber, or green <a href="../../project/issues/index.html#health-status">health status</a>, which then appears on your Epic tree. </p>
         <h2 id="multi-level-child-epics">Multi-level child epics <span class="badge-trigger ultimate"></span><a href="#multi-level-child-epics" title="Permalink" class="anchor"></a> </h2> 
         <div class="introduced-in"> 
          <p><a href="https://gitlab.com/gitlab-org/gitlab/-/issues/8333">Introduced</a> in GitLab Ultimate 11.7. </p>
         </div> 
         <p>Any epic that belongs to a group, or subgroup of the parent epics group, is eligible to be added. New child epics appear at the top of the list of epics in the <strong>Epics and Issues</strong> tab. </p>
         <p>When you add an epic thats already linked to a parent epic, the link to its current parent is removed. </p>
         <p>An epic can have multiple child epics up to the maximum depth of seven. </p>
         <p>See <a href="manage_epics.html#manage-multi-level-child-epics">Manage multi-level child epics</a> for steps to create, move, reorder, or delete child epics. </p>
         <h2 id="start-date-and-due-date">Start date and due date<a href="#start-date-and-due-date" title="Permalink" class="anchor"></a> </h2> 
         <p>To set a <strong>Start date</strong> and <strong>Due date</strong> for an epic, select one of the following: </p>
         <ul> 
          <li> <strong>Fixed</strong>: enter a fixed value. </li>
          <li> <strong>Inherited</strong>: inherit a dynamic value from the epics issues, child epics, and milestones. </li>
         </ul> 
         <h3 id="inherited">Inherited<a href="#inherited" title="Permalink" class="anchor"></a> </h3> 
         <div class="introduced-in"> 
          <p><a href="https://gitlab.com/gitlab-org/gitlab/-/issues/7332">Introduced</a> in GitLab 12.5 to replace <strong>From milestones</strong>. </p>
         </div> 
         <p>If you select <strong>Inherited</strong>: </p>
         <ul> 
          <li>For the <strong>start date</strong>: GitLab scans all child epics and issues assigned to the epic, and sets the start date to match the earliest found start date or milestone. </li>
          <li>For the <strong>due date</strong>: GitLab sets the due date to match the latest due date or milestone found among its child epics and issues. </li>
         </ul> 
         <p>These are dynamic dates and recalculated if any of the following occur: </p>
         <ul> 
          <li>A child epics dates change. </li>
          <li>Milestones are reassigned to an issue. </li>
          <li>A milestones dates change. </li>
          <li>Issues are added to, or removed from, the epic. </li>
         </ul> 
         <p>Because the epics dates can inherit dates from its children, the start date and due date propagate from the bottom to the top. If the start date of a child epic on the lowest level changes, that becomes the earliest possible start date for its parent epic. The parent epics start date then reflects this change and propagates upwards to the top epic. </p>
         <h2 id="roadmap-in-epics">Roadmap in epics <span class="badge-trigger ultimate"></span><a href="#roadmap-in-epics" title="Permalink" class="anchor"></a> </h2> 
         <div class="introduced-in"> 
          <p><a href="https://gitlab.com/gitlab-org/gitlab/-/issues/7327">Introduced</a> in <a href="https://about.gitlab.com/pricing/">GitLab Ultimate</a> 11.10. </p>
         </div> 
         <p>If your epic contains one or more <a href="#multi-level-child-epics">child epics</a> which have a <a href="#start-date-and-due-date">start or due date</a>, a <a href="../roadmap/index.html">roadmap</a> view of the child epics is listed under the parent epic. </p>
         <p><a href="img/epic_view_roadmap_v12_9.png" target="_blank" rel="noopener noreferrer"><img src="img/epic_view_roadmap_v12_9.png" alt="Child epics roadmap"></a> </p>
         <h2 id="permissions">Permissions<a href="#permissions" title="Permalink" class="anchor"></a> </h2> 
         <p>If you have access to view an epic and an issue added to that epic, you can view the issue in the epics issue list. </p>
         <p>If you have access to edit an epic and an issue added to that epic, you can add the issue to or remove it from the epic. </p>
         <p>Note that for a given group, the visibility of all projects must be the same as the group, or less restrictive. That means if you have access to a groups epic, then you already have access to its projects issues. </p>
         <p>You can also consult the <a href="../../permissions.html#group-members-permissions">group permissions table</a>. </p>
         <h2 id="thread">Thread<a href="#thread" title="Permalink" class="anchor"></a> </h2> 
         <ul> 
          <li>Comments: collaborate on that epic by posting comments in its thread. These text fields also fully support <a href="../../markdown.html#gitlab-flavored-markdown">GitLab Flavored Markdown</a>. </li>
         </ul> 
         <h2 id="comment-or-start-a-thread">Comment or start a thread<a href="#comment-or-start-a-thread" title="Permalink" class="anchor"></a> </h2> 
         <p>Once you write your comment, you can either: </p>
         <ul> 
          <li>Click <strong>Comment</strong> to publish your comment. </li>
          <li>Click <strong>Start thread</strong> to start a thread within that epics discussion. </li>
         </ul> 
         <h3 id="activity-sort-order">Activity sort order<a href="#activity-sort-order" title="Permalink" class="anchor"></a> </h3> 
         <div class="introduced-in"> 
          <p><a href="https://gitlab.com/gitlab-org/gitlab/-/issues/214364">Introduced</a> in <a href="https://about.gitlab.com/pricing/">GitLab Premium</a> 13.2. </p>
         </div> 
         <p>You can reverse the default order and interact with the activity feed sorted by most recent items at the top. Your preference is saved via local storage and automatically applied to every issue you view. </p>
         <p>To change the activity sort order, click the <strong>Oldest first</strong> dropdown menu and select either oldest or newest items to be shown first. </p>
         <p><a href="img/epic_activity_sort_order_v13_2.png" target="_blank" rel="noopener noreferrer"><img src="img/epic_activity_sort_order_v13_2.png" alt="Issue activity sort order dropdown button"></a> </p>
         <h2 id="award-emoji">Award emoji<a href="#award-emoji" title="Permalink" class="anchor"></a> </h2> 
         <p>You can <a href="../../award_emojis.html">award an emoji</a> to that epic or its comments. </p>
         <h2 id="notifications">Notifications<a href="#notifications" title="Permalink" class="anchor"></a> </h2> 
         <p>You can <a href="../../profile/notifications.html">turn on notifications</a> to be alerted about epic events.</p>
