       <div class="d-flex flex-items-baseline flex-justify-between mt-3"> 
        <h1 class="border-bottom-0">Viewing and updating vulnerable dependencies in your repository</h1> 
        <div class="d-none d-lg-block ml-2"> <button class="btn-link Link--secondary js-print tooltipped tooltipped-n" aria-label="Print this article"> <!-- From https://heroicons.dev/ --> 
          <svg fill="none" height="18" width="18" stroke-linecap="round" stroke-linejoin="round" stroke-width="2" viewbox="0 0 24 24" stroke="currentColor">
           <path d="M17 17h2a2 2 0 002-2v-4a2 2 0 00-2-2H5a2 2 0 00-2 2v4a2 2 0 002 2h2m2 4h6a2 2 0 002-2v-4a2 2 0 00-2-2H9a2 2 0 00-2 2v4a2 2 0 002 2zm8-12V5a2 2 0 00-2-2H9a2 2 0 00-2 2v4h10z"></path>
          </svg> </button> 
        </div> 
       </div> 
       <div class="lead-mktg">
        <p>If GitHub discovers vulnerable dependencies in your project, you can view them on the Dependabot alerts tab of your repository. Then, you can update your project to resolve or dismiss the vulnerability.</p>
       </div> 
       <div class="permissions-statement">
        <p>Repository administrators and organization owners can view and update dependencies.</p>
       </div> 
      </div> 
      <div class="article-grid-toc border-bottom border-xl-0 pb-4 mb-5 pb-xl-0 mb-xl-0"> 
       <div class="article-grid-toc-content"> 
        <h2 id="in-this-article" class="f5 mb-2"><a class="Link--primary" href="#in-this-article">In this article</a></h2> 
        <ul class="list-style-none pl-0 f5 mb-0"> 
         <li class="ml-0  mb-2 lh-condensed"><a href="#about-updates-for-vulnerable-dependencies-in-your-repository">About updates for vulnerable dependencies in your repository</a></li> 
         <li class="ml-0  mb-2 lh-condensed"><a href="#viewing-and-updating-vulnerable-dependencies">Viewing and updating vulnerable dependencies</a></li> 
         <li class="ml-0  mb-2 lh-condensed"><a href="#further-reading">Further reading</a></li> 
        </ul> 
       </div> 
      </div> 
      <div id="article-contents" class="article-grid-body"> 
       <p>Your repository's Dependabot alerts tab lists all open and closed Dependabot alerts and corresponding Dependabot security updates. You can sort the list of alerts using the drop-down menu, and you can click into specific alerts for more details. For more information, see "<a href="/en/code-security/supply-chain-security/about-alerts-for-vulnerable-dependencies">About alerts for vulnerable dependencies</a>."</p> 
       <p>You can enable automatic security updates for any repository that uses Dependabot alerts and the dependency graph. For more information, see "<a href="/en/github/managing-security-vulnerabilities/about-dependabot-security-updates">About Dependabot security updates</a>."</p> 
       <p>Additionally, GitHub can review any dependencies added, updated, or removed in a pull request made against the default branch of a repository, and flag any changes that would introduce a vulnerability into your project. This allows you to spot and deal with vulnerable dependencies before, rather than after, they reach your codebase. For more information, see "<a href="/en/github/collaborating-with-issues-and-pull-requests/reviewing-dependency-changes-in-a-pull-request">Reviewing dependency changes in a pull request</a>."</p> 
       <h3 id="about-updates-for-vulnerable-dependencies-in-your-repository"><a href="#about-updates-for-vulnerable-dependencies-in-your-repository">About updates for vulnerable dependencies in your repository</a></h3> 
       <p>GitHub generates Dependabot alerts when we detect that your codebase is using dependencies with known vulnerabilities. For repositories where Dependabot security updates are enabled, when GitHub detects a vulnerable dependency in the default branch, Dependabot creates a pull request to fix it. The pull request will upgrade the dependency to the minimum possible secure version needed to avoid the vulnerability.</p> 
       <h3 id="viewing-and-updating-vulnerable-dependencies"><a href="#viewing-and-updating-vulnerable-dependencies">Viewing and updating vulnerable dependencies</a></h3> 
       <ol> 
        <li>On GitHub, navigate to the main page of the repository.</li> 
        <li>Under your repository name, click <strong>
          <svg version="1.1" width="16" height="16" viewbox="0 0 16 16" class="octicon octicon-shield" aria-label="The shield symbol" role="img">
           <path fill-rule="evenodd" d="M7.467.133a1.75 1.75 0 011.066 0l5.25 1.68A1.75 1.75 0 0115 3.48V7c0 1.566-.32 3.182-1.303 4.682-.983 1.498-2.585 2.813-5.032 3.855a1.7 1.7 0 01-1.33 0c-2.447-1.042-4.049-2.357-5.032-3.855C1.32 10.182 1 8.566 1 7V3.48a1.75 1.75 0 011.217-1.667l5.25-1.68zm.61 1.429a.25.25 0 00-.153 0l-5.25 1.68a.25.25 0 00-.174.238V7c0 1.358.275 2.666 1.057 3.86.784 1.194 2.121 2.34 4.366 3.297a.2.2 0 00.154 0c2.245-.956 3.582-2.104 4.366-3.298C13.225 9.666 13.5 8.36 13.5 7V3.48a.25.25 0 00-.174-.237l-5.25-1.68zM9 10.5a1 1 0 11-2 0 1 1 0 012 0zm-.25-5.75a.75.75 0 10-1.5 0v3a.75.75 0 001.5 0v-3z"></path>
          </svg> Security</strong>. <span class="procedural-image-wrapper"><img src="/assets/images/help/repository/security-tab.png" alt="Security tab"></span></li> 
        <li>In the security sidebar, click <strong>Dependabot alerts</strong>. <span class="procedural-image-wrapper"><img src="/assets/images/help/repository/dependabot-alerts-tab.png" alt="Dependabot alerts tab"></span></li> 
        <li>Click the alert you'd like to view. <span class="procedural-image-wrapper"><img src="/assets/images/help/graphs/click-alert-in-alerts-list.png" alt="Alert selected in list of alerts"></span></li> 
        <li>Review the details of the vulnerability and, if available, the pull request containing the automated security update.</li> 
        <li>Optionally, if there isn't already a Dependabot security updates update for the alert, to create a pull request to resolve the vulnerability, click <strong>Create Dependabot security update</strong>. <span class="procedural-image-wrapper"><img src="/assets/images/help/repository/create-dependabot-security-update-button.png" alt="Create Dependabot security update button"></span></li> 
        <li>When you're ready to update your dependency and resolve the vulnerability, merge the pull request. Each pull request raised by Dependabot includes information on commands you can use to control Dependabot. For more information, see "<a href="/en/github/administering-a-repository/managing-pull-requests-for-dependency-updates#managing-dependabot-pull-requests-with-comment-commands">Managing pull requests for dependency updates</a>."</li> 
        <li>Optionally, if the alert is being fixed, if it's incorrect, or located in unused code, use the "Dismiss" drop-down, and click a reason for dismissing the alert. <span class="procedural-image-wrapper"><img src="/assets/images/help/repository/dependabot-alert-dismiss-drop-down.png" alt="Choosing reason for dismissing the alert via the &quot;Dismiss&quot; drop-down"></span></li> 
       </ol>
